Introduction
------------

The Spin command provides a shell script interface for displaying a spinner animation while executing a command in the background. This command is particularly useful for long-running operations where you want to show users that the system is active while optionally controlling the visibility of the command's output.

Overview
--------

The Spin command runs a specified command in the background while displaying a customizable spinner animation. The spinner automatically exits when the background command completes, returning the command's exit code and optionally displaying its output.

Sources: [spin/spin.go1-15](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L1-L15)

Command Execution & PTY Support
-------------------------------

The command execution logic in `commandStart` handles cross-platform differences and TTY detection to ensure output is captured correctly.

1.  **Command Initialization**: Uses `exec.CommandContext` with a background context [spin/spin.go75](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L75-L75)
    
2.  **Windows Fallback**: On Windows, it uses `io.MultiWriter` with standard buffers to avoid PTY issues [spin/spin.go84-87](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L84-L87)
    
3.  **PTY Allocation**: On Unix systems with a terminal, it opens pseudo-terminals for both `stdout` and `stderr` using `openPty` [spin/spin.go88-106](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L88-L106)
    
4.  **Output Capture**: `io.Copy` is used to pipe PTY output into internal buffers (`outbuf`, `errbuf`, and `bothbuf`) [spin/spin.go108-109](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L108-L109)
    
5.  **Process Completion**: The exit code is captured, defaulting to `1` if the process state is unavailable [spin/spin.go121-124](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L121-L124)
    

### PTY Utility

The `openPty` function retrieves the current terminal size using `term.GetSize` and allocates a new PTY via `xpty.NewPty` to ensure the background command perceives the correct terminal dimensions [spin/pty.go10-22](https://github.com/charmbracelet/gum/blob/30cc5169/spin/pty.go#L10-L22)

Output Handling
---------------

After the Bubble Tea program terminates, the `Run()` method determines what to print to the user's actual `STDOUT` based on the command's success and the provided flags.

| Condition | Action |
| --- | --- |
| `m.err != nil` | Prints internal error to `os.Stderr` and exits with code 1 [spin/command.go54-58](https://github.com/charmbracelet/gum/blob/30cc5169/spin/command.go#L54-L58) |
| `m.status == 0` | Prints requested output (stdout, stderr, or both) to `os.Stdout` [spin/command.go59-72](https://github.com/charmbracelet/gum/blob/30cc5169/spin/command.go#L59-L72) |
| `o.ShowError` (status != 0) | Prints combined output (`m.output`) to `os.Stdout` for debugging [spin/command.go73-79](https://github.com/charmbracelet/gum/blob/30cc5169/spin/command.go#L73-L79) |

Sources: [spin/command.go54-81](https://github.com/charmbracelet/gum/blob/30cc5169/spin/command.go#L54-L81)

Configuration Options
---------------------

### Display & Style

*   **Spinner Type**: Controlled by `--spinner`. Supported types include `line`, `dot`, `minidot`, `jump`, `pulse`, `points`, `globe`, `moon`, `monkey`, `meter`, and `hamburger` [spin/options.go17](https://github.com/charmbracelet/gum/blob/30cc5169/spin/options.go#L17-L17)
     [spin/spinners.go5-17](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spinners.go#L5-L17)
    
*   **Alignment**: The spinner can be placed to the `left` or `right` of the title [spin/options.go21](https://github.com/charmbracelet/gum/blob/30cc5169/spin/options.go#L21-L21)
    
*   **Styling**: Supports `SpinnerStyle` and `TitleStyle` using the `lipgloss` framework [spin/options.go18-20](https://github.com/charmbracelet/gum/blob/30cc5169/spin/options.go#L18-L20)
    

### Execution Control

*   **Timeout**: A `time.Duration` after which the command is aborted [spin/options.go22](https://github.com/charmbracelet/gum/blob/30cc5169/spin/options.go#L22-L22)
    
*   **Interruption**: Pressing `ctrl+c` triggers `commandAbort`, which sends `syscall.SIGINT` to the background process [spin/spin.go135-140](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L135-L140)
     [spin/spin.go188-190](https://github.com/charmbracelet/gum/blob/30cc5169/spin/spin.go#L188-L190)
