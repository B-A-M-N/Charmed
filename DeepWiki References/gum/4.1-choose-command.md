Choose Command
==============

Purpose and Overview
--------------------

The Choose Command provides an interactive terminal user interface for selecting one or more options from a list. It is designed for scenarios where users need to make selections from predefined options within shell scripts or command-line workflows. Unlike the [Filter Command](https://deepwiki.com/charmbracelet/gum/4.3-filter-command)
, the Choose Command does not provide fuzzy finding capability, making it better suited for smaller, curated lists of options.

Sources: [choose/choose.go1-11](https://github.com/charmbracelet/gum/blob/30cc5169/choose/choose.go#L1-L11)

Basic Usage
-----------

The command returns the selected option(s) to stdout, allowing the output to be captured in shell scripts.

Command Structure and Execution Flow
------------------------------------

The `choose` command follows this execution flow through the `Options.Run()` method:

The command creates a `model` struct containing `items []item`, `paginator paginator.Model`, and various styling configurations. The model is then passed to `tea.NewProgram()` which manages the TUI lifecycle until the user submits or quits.

Options and Parameters
----------------------

The Choose Command provides numerous options to customize its behavior, defined in the `Options` struct:

| Parameter | Type | Default | Description | Environment Variable |
| --- | --- | --- | --- | --- |
| `--limit` | int | 1   | Maximum number of options that can be selected | \-  |
| `--no-limit` | bool | false | Allow unlimited selections (ignores `--limit`) | \-  |
| `--ordered` | bool | false | Maintain the order of selected options | `GUM_CHOOSE_ORDERED` |
| `--height` | int | 10  | Height of the displayed list | `GUM_CHOOSE_HEIGHT` |
| `--cursor` | string | "> " | Prefix shown at cursor position | `GUM_CHOOSE_CURSOR` |
| `--show-help` | bool | true | Show help keybinds | `GUM_CHOOSE_SHOW_HELP` |
| `--timeout` | duration | 0s  | Timeout until choose returns selected element | `GUM_CHOOSE_TIMEOUT` |
| `--header` | string | "Choose:" | Text displayed above the options | `GUM_CHOOSE_HEADER` |
| `--cursor-prefix` | string | "• " | Prefix for the item under the cursor (hidden if limit is 1) | `GUM_CHOOSE_CURSOR_PREFIX` |
| `--selected-prefix` | string | "✓ " | Prefix for selected items (hidden if limit is 1) | `GUM_CHOOSE_SELECTED_PREFIX` |
| `--unselected-prefix` | string | "• " | Prefix for unselected items (hidden if limit is 1) | `GUM_CHOOSE_UNSELECTED_PREFIX` |
| `--selected` | \[\]string | \[\] | Options that should start as selected (selects all if given `*`) | `GUM_CHOOSE_SELECTED` |
| `--select-if-one` | bool | false | Automatically select if there is only one option | \-  |
| `--input-delimiter` | string | "\\n" | Delimiter for options when reading from stdin | `GUM_CHOOSE_INPUT_DELIMITER` |
| `--output-delimiter` | string | "\\n" | Delimiter for options when writing to stdout | `GUM_CHOOSE_OUTPUT_DELIMITER` |
| `--label-delimiter` | string | ""  | Allows setting a delimiter for `label:value` format | `GUM_CHOOSE_LABEL_DELIMITER` |
| `--strip-ansi` | bool | true | Strip ANSI sequences when reading from stdin | `GUM_CHOOSE_STRIP_ANSI` |
| `--padding` | string | (default) | Padding around the chooser | `GUM_CHOOSE_PADDING` |

Additional styling options are available with `--cursor.`, `--header.`, `--item.`, and `--selected.` prefixes for controlling colors and text formatting via Lipgloss styles.

Sources: [choose/options.go9-35](https://github.com/charmbracelet/gum/blob/30cc5169/choose/options.go#L9-L35)

Bubbletea Model Implementation
------------------------------

The Choose command implements the Bubble Tea `tea.Model` interface with the following structure:

The `model.Update()` method handles `tea.KeyMsg` events and updates the `index`, `numSelected`, and `items` state based on keyboard input. The `paginator.Model` calculates visible slice bounds via `GetSliceBounds()` for rendering.

Pagination System
-----------------

For lists exceeding the `--height` option, the Choose command uses the Bubbles `paginator.Model`:

The paginator calculates which page contains the current `index` and maintains the `pager.Page` field. When the cursor moves beyond visible bounds, `model.Update()` calls `pager.NextPage()` or `pager.PrevPage()`. The `View()` method uses `GetSliceBounds()` to render only visible items.

Total pages are calculated as `(len(items) + o.Height - 1) / o.Height` to ensure all items fit across pages.

Sources: [choose/command.go111-119](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L111-L119)

Selection Processing
--------------------

The Choose command processes selections differently based on the `Limit` configuration:

### Selection Modes

### Selection State Management

The `model` tracks selection state through:

*   `numSelected` - count of selected items [choose/choose.go112](https://github.com/charmbracelet/gum/blob/30cc5169/choose/choose.go#L112-L112)
    
*   `currentOrder` - tracks selection sequence [choose/choose.go113](https://github.com/charmbracelet/gum/blob/30cc5169/choose/choose.go#L113-L113)
    
*   `item.selected` - boolean flag per item [choose/choose.go128](https://github.com/charmbracelet/gum/blob/30cc5169/choose/choose.go#L128-L128)
    
*   `item.order` - sequence number for ordered output [choose/choose.go129](https://github.com/charmbracelet/gum/blob/30cc5169/choose/choose.go#L129-L129)
    

When `--ordered` is set and `Limit > 1`, the final output is sorted by the `order` field using `sort.Slice()` [choose/command.go168-172](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L168-L172)

### Pre-selection Handling

The `--selected` option allows pre-selecting items [choose/command.go80-106](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L80-L106)
:

*   If `--selected="*"`, all items are marked as selected (up to limit).
*   Otherwise, items matching values in the `Selected` array are marked.
*   For `Limit == 1`, pre-selection moves the cursor instead of selecting (`startingIndex = i`).

Error Handling
--------------

The Choose command handles several error conditions:

*   Returns error if no options are provided [choose/command.go34](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L34-L34)
    
*   Returns error if nothing is selected (when ESC or Ctrl+C is pressed) [choose/command.go165-167](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L165-L167)
    
*   Returns error for invalid option formats when using label delimiters [choose/command.go50](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L50-L50)
    
*   Handles timeouts gracefully using `timeout.Context(o.Timeout)` [choose/command.go152-153](https://github.com/charmbracelet/gum/blob/30cc5169/choose/command.go#L152-L153)
