Pager Command
=============

The Pager Command in `gum` provides a terminal pager functionality (similar to tools like `less`) that allows users to scroll through and search within text content. It facilitates viewing large text files or command outputs with features like line numbers, soft wrapping, and vim-like search capabilities.

Overview
--------

The Pager Command takes input either from standard input (stdin) or a direct content argument, and displays it in a scrollable, interactive interface. It leverages the Bubble Tea framework to create a terminal user interface with keyboard navigation and search features.

Command Structure
-----------------

The Pager Command follows the standard pattern used throughout `gum`, consisting of an `Options` struct for configuration, a `Run` method for bootstrapping, and a `model` implementation for the Bubble Tea framework.

Usage
-----

### Input Methods

The Pager Command accepts content in two ways:

1.  **Stdin Input**: Pipe content directly into the command. The `Run` function uses `stdin.Read()` to capture the stream and sanitizes it by removing backspace sequences using a regular expression `.\x08` [pager/command.go21-28](https://github.com/charmbracelet/gum/blob/30cc5169/pager/command.go#L21-L28)
    
2.  **Direct Content Argument**: Provide content as a command argument.

Search Functionality
--------------------

The `pager.search` struct manages the state of the search interface, including the `textinput.Model` and the compiled regex `query`.

### Search Implementation Details

*   **Regex Execution**: `Execute` compiles the user input into a regex and replaces matches in `m.content` with styled strings using `m.matchStyle` [pager/search.go37-60](https://github.com/charmbracelet/gum/blob/30cc5169/pager/search.go#L37-L60)
    
*   **Cyclic Navigation**: `NextMatch` and `PrevMatch` navigate through the `FindAllStringIndex` results cyclically using the modulo operator [pager/search.go85](https://github.com/charmbracelet/gum/blob/30cc5169/pager/search.go#L85-L85)
     [pager/search.go122](https://github.com/charmbracelet/gum/blob/30cc5169/pager/search.go#L122-L122)
    
*   **Auto-Scrolling**: The pager calculates the line number of the match by counting newlines in a soft-wrapped version of the content (`softWrapEm`) and updates the viewport's `YOffset` if the match is outside the current view [pager/search.go94-104](https://github.com/charmbracelet/gum/blob/30cc5169/pager/search.go#L94-L104)
    

Sources: [pager/search.go14-168](https://github.com/charmbracelet/gum/blob/30cc5169/pager/search.go#L14-L168)
 [pager/pager.go176-221](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L176-L221)

Text Processing
---------------

The `processText` function is the core rendering logic that prepares the content for the viewport whenever the window is resized or content changes.

1.  **Width Calculation**: Calculates `maxWidth` by subtracting borders, margins, padding, and line number gutters (if enabled) from the viewport width [pager/pager.go135-142](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L135-L142)
    
2.  **Line Iteration**: Splits content by newlines and replaces tabs with four spaces [pager/pager.go144-145](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L144-L145)
    
3.  **Line Numbering**: Prepends `i+1` formatted to 4 digits with a separator [pager/pager.go146-148](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L146-L148)
    
4.  **Soft Wrapping**: If `softWrap` is enabled and a line exceeds `maxWidth`, it uses `ansi.Cut` to split the line into multiple segments, ensuring ANSI escape codes are handled correctly [pager/pager.go150-159](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L150-L159)
    
5.  **Empty Space Fill**: If the content is shorter than the viewport, it fills the remaining height with `~` symbols (similar to vim) if line numbers are enabled [pager/pager.go167-170](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L167-L170)
    

Sources: [pager/pager.go128-172](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L128-L172)
 [pager/search.go150-168](https://github.com/charmbracelet/gum/blob/30cc5169/pager/search.go#L150-L168)

Internal Implementation
-----------------------

The command uses the Bubble Tea Model-View-Update (MVU) pattern:

*   **Init**: Returns `nil` as no initial commands are needed [pager/pager.go106](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L106-L106)
    
*   **Update**:
    *   `tea.WindowSizeMsg`: Triggers `processText` to recalculate wrapping and viewport dimensions [pager/pager.go110-111](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L110-L111)
        
    *   `tea.KeyMsg`: Routes to `keyHandler` which manages search state vs. navigation state [pager/pager.go112-113](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L112-L113)
         [pager/pager.go176-221](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L176-L221)
        
*   **View**:
    *   If search is active, it appends the `search.input.View()` to the bottom of the viewport [pager/pager.go224-226](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L224-L226)
        
    *   Otherwise, it appends the `help.View()` [pager/pager.go228](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L228-L228)
        

Sources: [pager/pager.go106-229](https://github.com/charmbracelet/gum/blob/30cc5169/pager/pager.go#L106-L229)
