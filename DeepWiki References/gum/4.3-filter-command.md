Overview
--------

The Filter Command is an interactive TUI component in Gum that provides fuzzy searching capabilities for filtering and selecting items from a list. It enables users to quickly narrow down options using real-time filtering, with support for both fuzzy and exact matching modes.

The command displays a list of options (from arguments, stdin, or file listing) and allows users to type to filter the list interactively. As the user types, matching items are highlighted and the list is filtered in real-time.

Sources: [filter/filter.go1-11](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L1-L11)

Basic Usage and Functionality
-----------------------------

By default, if no items are provided, the filter command will recursively list all files in the current directory for selection [filter/command.go37-43](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L37-L43)
 Alternatively, items can be provided as arguments or piped through stdin.

The selected item(s) are output to stdout, making it easy to capture the result in a variable [filter/command.go161-163](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L161-L163)

Architecture
------------

### Component Structure

The Filter Command follows the Model-View-Update (MVU) pattern implemented through the Bubble Tea framework. The main components are:

1.  **Options Struct** (`filter.Options`): Contains all customization options for the filter command [filter/options.go10-48](https://github.com/charmbracelet/gum/blob/30cc5169/filter/options.go#L10-L48)
    
2.  **Filter Model** (`filter.model`): Implements the Bubble Tea model interface [filter/filter.go126-158](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L126-L158)
    
3.  **Input Components**: Uses `textinput.Model` for the search input field [filter/filter.go127](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L127-L127)
    
4.  **Display Components**: Uses `viewport.Model` for scrollable list display [filter/filter.go128](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L128-L128)
    
5.  **Matching Engine**: Uses the `sahilm/fuzzy` package for searching/filtering [filter/command.go20](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L20-L20)

### Filter Model State

The `model` struct tracks the interactive state:

*   `choices`: A map of stripped-text to original-text (preserving ANSI) [filter/filter.go129](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L129-L129)
    
*   `filteringChoices`: A slice of plain text used for the matching engine [filter/filter.go130](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L130-L130)
    
*   `matches`: The current subset of `fuzzy.Match` results [filter/filter.go131](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L131-L131)
    
*   `selected`: A map tracking multi-selection state [filter/filter.go134](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L134-L134)
    

Sources: [filter/filter.go126-158](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L126-L158)

### Matching Algorithms

The command switches matching logic based on the `Fuzzy` flag:

1.  **Fuzzy Matching**: Uses `fuzzy.Find` [filter/command.go75](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L75-L75)
    
2.  **Exact Matching**: Uses `exactMatches`, which filters based on `strings.Contains` [filter/filter.go484-499](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L484-L499)
    
3.  **Match Highlighting**: Matches are rendered by converting byte positions to visible character positions using `bytePosToVisibleCharPos` and then applying styles via `lipgloss.StyleRanges` [filter/filter.go216-225](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L216-L225)

Implementation Details
----------------------

### Multi-Selection Logic

When `Limit > 1` or `NoLimit` is true, the model enables selection toggling [filter/command.go92-97](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L92-L97)

*   `ToggleAndNext`: Toggles the current item and moves the cursor [filter/filter.go310-316](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L310-L316)
    
*   `ToggleAll`: Iterates through all current `matches` and adds them to the `selected` map, respecting the `limit` [filter/filter.go262-282](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L262-L282)
    

Sources: [filter/command.go92-97](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L92-L97)
 [filter/filter.go262-316](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L262-L316)

### ANSI and Grapheme Awareness

To ensure highlights appear in the correct position on strings containing ANSI codes or multi-byte graphemes:

1.  Input is stripped of ANSI for matching: `ansi.Strip(opt)` [filter/command.go69](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L69-L69)
    
2.  Byte offsets from the fuzzy matcher are converted to visible character positions: `bytePosToVisibleCharPos(match.Str, rng)` [filter/filter.go221](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L221-L221)
    
3.  Styles are applied to the original string (with ANSI) using `lipgloss.StyleRanges` [filter/filter.go225](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L225-L225)
    

Sources: [filter/command.go69](https://github.com/charmbracelet/gum/blob/30cc5169/filter/command.go#L69-L69)
 [filter/filter.go221-225](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L221-L225)

### Viewport and Layout

The `View()` method handles complex layout logic:

*   **Reverse Mode**: If `Reverse` is true, the input field and header are rendered at the bottom, and the viewport content is offset to align with the bottom edge [filter/filter.go173-175](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L173-L175)
     [filter/filter.go237-249](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L237-L249)
    
*   **Cursor Tracking**: `m.viewport.SetContent` is called every frame to refresh the list, and `m.viewport.LineDown`/`LineUp` are used to keep the cursor in view [filter/filter.go406-438](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter.go#L406-L438)
