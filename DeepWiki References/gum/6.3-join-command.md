Join Command
============

Relevant source files

*   [join/command.go](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go)
    
*   [join/options.go](https://github.com/charmbracelet/gum/blob/30cc5169/join/options.go)
    

The `join` command is a formatting utility used to combine multi-line text blocks horizontally or vertically. It leverages the `lipgloss` library's layout capabilities to allow shell scripts to build complex terminal user interfaces by arranging styled components into grids or columns.

Purpose and Functionality
-------------------------

The `join` command acts as a layout engine for `gum`. It takes multiple strings as inputâ€”often pre-styled using `gum style`â€”and merges them into a single block of text while maintaining alignment and structure.

Sources: [join/command.go1-15](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L1-L15)
 [join/options.go4-10](https://github.com/charmbracelet/gum/blob/30cc5169/join/options.go#L4-L10)

Implementation Details
----------------------

The command is implemented as a simple wrapper around the `lipgloss` joining functions. It processes input via an `Options` struct and executes the join logic in the `Run` method.

### Data Flow and Logic

1.  **Configuration**: The `Options` struct captures the text to be joined and the desired orientation (horizontal vs. vertical) [join/options.go4-10](https://github.com/charmbracelet/gum/blob/30cc5169/join/options.go#L4-L10)
    
2.  **Alignment Mapping**: The `Align` string from the CLI is mapped to a `lipgloss.Position` using the `decode.Align` utility [join/command.go32](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L32-L32)
    
3.  **Function Selection**: Based on the `--vertical` flag, the command selects either `lipgloss.JoinHorizontal` or `lipgloss.JoinVertical` [join/command.go28-31](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L28-L31)
    
4.  **Execution**: The selected function is called with the alignment constant and the slice of text arguments [join/command.go32](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L32-L32)
    

### Component Interaction Diagram

Sources: [join/command.go27-34](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L27-L34)
 [join/options.go4-10](https://github.com/charmbracelet/gum/blob/30cc5169/join/options.go#L4-L10)

Configuration Options
---------------------

The command behavior is controlled by the `Options` struct:

| Field | CLI Flag | Type | Description |
| --- | --- | --- | --- |
| `Text` | `[arg]` | `[]string` | The text blocks to be combined. |
| `Align` | `--align` | `string` | Alignment: `left`, `center`, `right`, `bottom`, `middle`, `top`. |
| `Horizontal` | `--horizontal` | `bool` | Joins strings side-by-side (default). |
| `Vertical` | `--vertical` | `bool` | Joins strings on top of each other. |

Sources: [join/options.go4-10](https://github.com/charmbracelet/gum/blob/30cc5169/join/options.go#L4-L10)

Technical Considerations
------------------------

### Alignment Behavior

The alignment behavior depends on the axis of the join:

*   **Horizontal Join**: The `Align` parameter (e.g., `top`, `center`, `bottom`) determines the vertical position of text blocks relative to each other if they have different heights [join/command.go32](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L32-L32)
    
*   **Vertical Join**: The `Align` parameter (e.g., `left`, `center`, `right`) determines the horizontal position of text blocks relative to each other if they have different widths [join/command.go32](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L32-L32)
    

### Stdin and Arguments

Unlike many other `gum` commands, `join` primarily operates on positional arguments (`o.Text...`) rather than a continuous stdin stream, as it needs distinct blocks of text to perform the join operation [join/options.go5](https://github.com/charmbracelet/gum/blob/30cc5169/join/options.go#L5-L5)

### Preserving Structure

When passing variables to `gum join` in a shell script, it is critical to wrap them in double quotes. This ensures that newlines within the variables are preserved and treated as part of a single argument rather than being split by the shell's word splitting [join/command.go5-9](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L5-L9)

Sources: [join/command.go5-15](https://github.com/charmbracelet/gum/blob/30cc5169/join/command.go#L5-L15)
