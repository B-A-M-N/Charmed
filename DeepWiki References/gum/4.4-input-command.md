Input Command

Purpose and Scope
-----------------

The Input Command provides single-line text input functionality for shell scripts through an interactive TUI interface. It allows users to enter text with features like password masking, character limits, placeholders, and pre-filled values from stdin. This command leverages the `textinput` component from the Bubbles library to render an interactive text entry field.

For multi-line text input, see [Write Command](https://deepwiki.com/charmbracelet/gum/4.5-write-command)
. For general command patterns and configuration, see [Command Patterns](https://deepwiki.com/charmbracelet/gum/3.3-command-patterns)
.

* * *

Command Overview
----------------

The Input Command is implemented in the `input` package and presents a focused text input field where users can type a single line of text. The input is rendered to `os.Stderr` (preserving stdout for data), and the final value is written to `os.Stdout` when the user submits.

### Key Characteristics

| Characteristic | Description |
| --- | --- |
| **Package** | `input` |
| **Entry Point** | `Options.Run()` method |
| **Bubbletea Model** | `model` struct |
| **Bubbles Component** | `textinput.Model` |
| **Output Channel** | stdout (data), stderr (UI) |
| **Input Sources** | Command options, stdin, user typing |
| **Exit Behavior** | Error if not submitted (Ctrl+C/timeout/Esc) |

Configuration Options
---------------------

The `Options` struct defines all configurable parameters for the Input Command. These can be set via command-line flags or environment variables following the `GUM_INPUT_*` pattern.

### Core Input Options

| Field | Type | Purpose | Default Behavior |
| --- | --- | --- | --- |
| `Prompt` | `string` | Text displayed before the input field | `"> "` |
| `Placeholder` | `string` | Hint text shown when input is empty | `"Type something..."` |
| `Value` | `string` | Pre-filled value for the input field | Can also be read from stdin |
| `CharLimit` | `int` | Maximum number of characters allowed | `400` |
| `Width` | `int` | Width of the input field in cells | `0` (Auto-width) |

Sources: [input/options.go10-27](https://github.com/charmbracelet/gum/blob/30cc5169/input/options.go#L10-L27)
 [input/command.go33-40](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L33-L40)

### Password Mode

| Field | Type | Purpose | Implementation |
| --- | --- | --- | --- |
| `Password` | `bool` | Enable password masking | Sets `textinput.EchoMode` to `EchoPassword` |

When password mode is enabled, typed characters are displayed as '•' instead of the actual characters.

Sources: [input/command.go42-45](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L42-L45)

| `PlaceholderStyle` | `style.Styles` | Styling for placeholder text |
| `CursorStyle` | `style.Styles` | Styling for the cursor |

Key Features
------------

### Pre-filling Values

The Input Command supports pre-filling the text input from multiple sources with the following precedence:

1.  **Options.Value** - Specified via the `--value` flag.
2.  **stdin** - Read from standard input if the value is empty.

Sources: [input/command.go20-31](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L20-L31)

### Password Masking

When `Password` is true, the command configures the `textinput.Model`:

*   Sets `EchoMode` to `textinput.EchoPassword`.
*   Sets `EchoCharacter` to `'•'`.

Sources: [input/command.go42-45](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L42-L45)

### Auto-Width Behavior

When `Options.Width` is less than 1, `autoWidth` is set to `true`. The input field then dynamically calculates its width during `tea.WindowSizeMsg`: `m.textinput.Width = msg.Width - 1 - lipgloss.Width(m.textinput.Prompt) - paddingLeft - paddingRight`.

Sources: [input/command.go53](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L53-L53)
 [input/input.go76-81](https://github.com/charmbracelet/gum/blob/30cc5169/input/input.go#L76-L81)

### Keybindings and Help

The command provides a minimal keymap for the user:

*   **Enter**: Submits the input and exits.
*   **Esc / Ctrl+C**: Aborts without submitting.

If `ShowHelp` is enabled, the help component renders the "enter: submit" hint at the bottom.

Sources: [input/input.go29-36](https://github.com/charmbracelet/gum/blob/30cc5169/input/input.go#L29-L36)
 [input/input.go63-65](https://github.com/charmbracelet/gum/blob/30cc5169/input/input.go#L63-L65)

Output and Exit Behavior
------------------------

### Standard Output

The final value is printed to `os.Stdout` only if `m.submitted` is true.

Sources: [input/command.go74-78](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L74-L78)

### Error Conditions

| Condition | Error | Exit Code |
| --- | --- | --- |
| User presses `Esc` or `Ctrl+C` | `"not submitted"` | Non-zero |
| Timeout expires | `"not submitted"` | Non-zero |
| Bubbletea failure | `"failed to run input: ..."` | Non-zero |

Sources: [input/command.go68-76](https://github.com/charmbracelet/gum/blob/30cc5169/input/command.go#L68-L76)
