File Command

Purpose and Scope
-----------------

The `file` command provides an interactive file browser interface for selecting files or directories from the filesystem. It presents a TUI (text user interface) that allows users to navigate directory trees, view file metadata, and select files or directories for use in shell scripts. The command outputs the absolute path of the selected item to stdout.

This document covers the file command's configuration options, architecture, model lifecycle, and styling system. For other interactive selection commands, see [Choose Command](https://deepwiki.com/charmbracelet/gum/4.1-choose-command)
 and [Filter Command](https://deepwiki.com/charmbracelet/gum/4.3-filter-command)
. For multi-line text editing, see [Write Command](https://deepwiki.com/charmbracelet/gum/4.5-write-command)
.

Overview
--------

The file command wraps the `bubbles/filepicker` component to provide a file manager-like interface. Users can:

*   Navigate directory hierarchies [file/file.go1-3](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L1-L3)
    
*   View file permissions and sizes [file/options.go16-17](https://github.com/charmbracelet/gum/blob/30cc5169/file/options.go#L16-L17)
    
*   Show or hide hidden files [file/options.go15](https://github.com/charmbracelet/gum/blob/30cc5169/file/options.go#L15-L15)
    
*   Select either files or directories (or both) [file/options.go18-19](https://github.com/charmbracelet/gum/blob/30cc5169/file/options.go#L18-L19)
    
*   Apply custom styling to different file types [file/options.go25-32](https://github.com/charmbracelet/gum/blob/30cc5169/file/options.go#L25-L32)
    

The command renders the interactive UI to stderr and outputs the selected path to stdout, making it composable in shell pipelines [file/command.go66-77](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L66-L77)

**Sources:** [file/file.go1-13](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L1-L13)
 [file/command.go16-79](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L16-L79)

Configuration Architecture
--------------------------

The file command's configuration is managed through the `Options` struct, which provides both command-line flags and environment variable support:

**Sources:** [file/options.go9-34](https://github.com/charmbracelet/gum/blob/30cc5169/file/options.go#L9-L34)
 [file/command.go31-49](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L31-L49)

### Configuration Options

| Option | Flag | Environment Variable | Default | Description |
| --- | --- | --- | --- | --- |
| Path | `path` (arg) | `GUM_FILE_PATH` | `.` | Starting directory for file picker |
| Cursor | `-c` | `GUM_FILE_CURSOR` | `>` | Character displayed before current item |
| All | `-a` | `GUM_FILE_ALL` | `false` | Show hidden and dot files |
| Permissions | `-p` | `GUM_FILE_PERMISSION` | `true` | Show file permissions |
| Size | `-s` | `GUM_FILE_SIZE` | `true` | Show file sizes |
| File | none | `GUM_FILE_FILE` | `true` | Allow file selection |
| Directory | none | `GUM_FILE_DIRECTORY` | `false` | Allow directory selection |
| ShowHelp | none | `GUM_FILE_SHOW_HELP` | `true` | Display help keybindings |
| Timeout | none | `GUM_FILE_TIMEOUT` | `0s` | Timeout before aborting |
| Header | none | `GUM_FILE_HEADER` | `""` | Header text above picker |
| Height | none | `GUM_FILE_HEIGHT` | `10` | Maximum files to display (0=auto) |
| Padding | none | `GUM_FILE_PADDING` | default | Padding around the picker |

**Sources:** [file/options.go11-23](https://github.com/charmbracelet/gum/blob/30cc5169/file/options.go#L11-L23)

Component Architecture
----------------------

The file command integrates the `bubbles/filepicker` component within a custom Bubbletea model:

**Sources:** [file/file.go55-65](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L55-L65)
 [file/command.go31-78](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L31-L78)

### Model Structure

The `model` struct maintains the state for the file picker interface:

*   **`filepicker`**: The embedded `filepicker.Model` from the bubbles library that handles file system navigation [file/file.go58](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L58-L58)
    
*   **`selectedPath`**: Stores the absolute path of the selected file/directory [file/file.go59](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L59-L59)
    
*   **`quitting`**: Boolean flag indicating whether the user has quit the picker [file/file.go60](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L60-L60)
    
*   **`showHelp`**: Controls whether help keybindings are displayed [file/file.go61](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L61-L61)
    
*   **`padding`**: Four-element array `[top, right, bottom, left]` for layout spacing [file/file.go62](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L62-L62)
    
*   **`help`**: The bubbles `help.Model` for rendering keybindings [file/file.go63](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L63-L63)
    
*   **`keymap`**: Custom keymap wrapping `filepicker.KeyMap` [file/file.go64](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L64-L64)
    
*   **`header`** and **`headerStyle`**: Optional header text and its styling [file/file.go56-57](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L56-L57)
    

**Sources:** [file/file.go55-65](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L55-L65)

Execution Flow
--------------

The `Run()` method orchestrates the file picker execution:

**Sources:** [file/command.go17-79](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L17-L79)

### Validation and Initialization

The `Run()` method performs several validation and initialization steps:

1.  **Flag Validation** [file/command.go18-20](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L18-L20)
    : At least one of `--file` or `--directory` must be set.
2.  **Path Resolution** [file/command.go22-29](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L22-L29)
    : Defaults to current directory (`.`) and resolves to absolute path using `filepath.Abs()`.
3.  **Filepicker Configuration** [file/command.go31-49](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L31-L49)
    : Creates a new `filepicker.Model` and configures it based on `Options`.
4.  **Style Application** [file/command.go43-49](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L43-L49)
    : Converts `style.Styles` to `lipgloss.Style` and applies them to the `filepicker.Styles` struct.
5.  **Padding Parsing** [file/command.go50](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L50-L50)
    : Uses `style.ParsePadding()` to convert the padding string into individual side values.

**Sources:** [file/command.go17-59](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L17-L59)

### Init Method

This starts the file system reading process for the initial directory.

**Sources:** [file/file.go67](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L67-L67)

### Update Method

The `Update()` method handles three categories of messages:

**Sources:** [file/file.go69-95](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L69-L95)

#### Window Size Handling

When the terminal window is resized [file/file.go71-76](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L71-L76)
:

1.  Available height is calculated by subtracting top and bottom padding from `msg.Height`.
2.  If `showHelp` is enabled, the height of the help view is also subtracted.
3.  `m.filepicker.SetHeight()` is called to adjust the component's internal display.

#### Key Event Handling

The model recognizes two quit keys [file/file.go77-85](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L77-L85)
:

*   **Ctrl+C** (`keyAbort`): Sets `quitting = true` and returns `tea.Interrupt`.
*   **Esc or Q** (`keyQuit`): Sets `quitting = true` and returns `tea.Quit`.

Both prevent the selected path from being printed by standardizing the `quitting` state.

#### File Selection Detection

After delegating to `filepicker.Update()`, the model checks if a selection event occurred [file/file.go88-93](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L88-L93)
:

**Sources:** [file/file.go69-95](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L69-L95)

### View Method

The `View()` method constructs the visual representation:

**Sources:** [file/file.go97-115](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L97-L115)

The view is composed of three optional sections stacked vertically [file/file.go101-114](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L101-L114)
:

1.  **Header**: Rendered with `headerStyle` if `header` is non-empty.
2.  **File Picker**: The main `filepicker.View()` output.
3.  **Help**: Keybinding help rendered via `help.View(keymap)` [file/file.go117-119](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L117-L119)
    

All sections are joined vertically with `lipgloss.JoinVertical()` and wrapped in padding using `lipgloss.Style.Padding()` [file/file.go109-114](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L109-L114)

**Sources:** [file/file.go97-119](https://github.com/charmbracelet/gum/blob/30cc5169/file/file.go#L97-L119)

Output and Error Handling
-------------------------

The file command separates UI (stderr) from data (stdout):

### Standard Output (stdout)

When a file is successfully selected [file/command.go77](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L77-L77)
:

The absolute path of the selected item is printed to stdout for script capture.

### Standard Error (stderr)

The interactive UI is rendered to stderr via `tea.WithOutput(os.Stderr)` [file/command.go66](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L66-L66)
 ensuring it doesn't pollute data pipelines.

### Error Cases

The `Run()` method returns errors in the following cases [file/command.go17-79](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L17-L79)
:

| Error | Condition | Source |
| --- | --- | --- |
| `"at least one between --file and --directory must be set"` | Both `--file` and `--directory` flags are false | [file/command.go19](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L19-L19) |
| `"file not found: %w"` | `filepath.Abs()` fails on the provided start path | [file/command.go28](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L28-L28) |
| `"unable to pick selection: %w"` | The Bubbletea program fails to run | [file/command.go70](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L70-L70) |
| `"no file selected"` | The program exited without a selection (e.g., via Esc) | [file/command.go74](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L74-L74) |

**Sources:** [file/command.go17-79](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L17-L79)

Integration with bubbles/filepicker
-----------------------------------

The file command maps `Options` fields directly to `filepicker.Model` properties during initialization [file/command.go31-49](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L31-L49)
:

| Options Field | filepicker Property | Description |
| --- | --- | --- |
| `Path` | `CurrentDirectory`, `Path` | Starting directory |
| `Height` | `SetHeight()`, `AutoHeight` | Display height (0=auto) |
| `Cursor` | `Cursor` | Cursor character |
| `Directory` | `DirAllowed` | Allow directory selection |
| `File` | `FileAllowed` | Allow file selection |
| `Permissions` | `ShowPermissions` | Display file permissions |
| `Size` | `ShowSize` | Display file sizes |
| `All` | `ShowHidden` | Show hidden files |

The filepicker handles file system traversal and metadata retrieval, while `gum` provides the wrapper logic for CLI integration.

**Sources:** [file/command.go31-49](https://github.com/charmbracelet/gum/blob/30cc5169/file/command.go#L31-L49)
