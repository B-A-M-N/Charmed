Confirm Command
===============

Relevant source files

*   [confirm/command.go](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go)

Purpose and Overview
--------------------

The Confirm Command (`gum confirm`) provides a yes/no dialog prompt designed for shell scripts. It allows developers to create interactive confirmation prompts with customizable styling, default behaviors, and timeouts.

The command's primary integration point with shell scripts is its exit code:

*   **Exit 0**: The user selected the affirmative answer (e.g., "Yes").
*   **Exit 1**: The user selected the negative answer (e.g., "No"), aborted the prompt, or the prompt timed out with a negative default.

This design allows for concise shell conditionals like `gum confirm "Are you sure?" && rm file.txt` [confirm/confirm.go1-10](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L1-L10)

Basic Usage
-----------

The command can be used interactively or via pipeline for non-interactive automation.

Architecture and Flow
---------------------

The `confirm` command follows the standard `gum` command pattern, utilizing an `Options` struct for configuration and a Bubble Tea `model` for the TUI lifecycle.

### Execution Logic

The `Run()` function in [confirm/command.go](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go)
 handles the transition from shell execution to the interactive TUI.

 [confirm/confirm.go99-131](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L99-L131)

Component Structure
-------------------

The implementation is split between CLI configuration (`Options`) and TUI state management (`model`).

Options and Customization
-------------------------

The `Options` struct defines the CLI flags and environment variables available to the user.

| Option | Flag | Default | Description |
| --- | --- | --- | --- |
| `Default` | `--default` | `true` | The pre-selected action and result if timeout occurs. |
| `Affirmative` | `--affirmative` | `"Yes"` | Text for the "yes" option. |
| `Negative` | `--negative` | `"No"` | Text for the "no" option. |
| `ShowOutput` | `--show-output` | `false` | Prints the prompt and choice to stdout on exit. |
| `Timeout` | `--timeout` | `0s` | Duration to wait before exiting with the default value. |
| `PromptStyle` | `--prompt.*` | Bold, #7571F9 | Lipgloss style for the prompt text. |
| `SelectedStyle` | `--selected.*` | Pink bg | Lipgloss style for the active button. |

Sources: [confirm/options.go10-25](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/options.go#L10-L25)
 [confirm/command.go32-47](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go#L32-L47)

Interaction Model
-----------------

### Key Bindings

The `defaultKeymap` defines how users interact with the TUI [confirm/confirm.go21-57](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L21-L57)

*   **Affirmative**: `y`, `Y` sets `m.confirmation = true` and quits [confirm/confirm.go124-127](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L124-L127)
    
*   **Negative**: `n`, `N`, `q` sets `m.confirmation = false` and quits [confirm/confirm.go112-115](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L112-L115)
    
*   **Toggle**: `left`, `right`, `h`, `l`, `tab`, `shift+tab` flips the boolean `m.confirmation` state [confirm/confirm.go116-120](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L116-L120)
    
*   **Submit**: `enter` exits with the currently highlighted state [confirm/confirm.go121-123](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L121-L123)
    
*   **Abort**: `ctrl+c` or `esc` exits with `m.confirmation = false` [confirm/confirm.go105-111](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L105-L111)
    

### UI Rendering

The `View()` method constructs the interface using `lipgloss`. It joins the prompt and the two buttons (affirmative and negative) either horizontally or vertically depending on padding and style configuration [confirm/confirm.go133-168](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L133-L168)

If `m.negative` is explicitly set to an empty string, the UI will only display the affirmative option [confirm/confirm.go149-152](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/confirm.go#L149-L152)

Implementation Details
----------------------

### Stdin Handling

Before launching the Bubble Tea program, `Run()` attempts to read from stdin using `stdin.Read(stdin.SingleLine(true))` [confirm/command.go19](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go#L19-L19)
 If input is detected (e.g., piped from another command), it bypasses the TUI entirely. If the input is "yes" or "y", it returns `nil` (exit 0); otherwise, it returns an exit error of 1 [confirm/command.go20-27](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go#L20-L27)

### Timeout Management

If a `--timeout` is provided, `gum` uses `timeout.Context(o.Timeout)` to create a context that will automatically cancel the Bubble Tea program [confirm/command.go29-30](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go#L29-L30)
 If the program exits due to a timeout, it falls back to the `m.confirmation` state which was initialized from `o.Default` [confirm/command.go37-38](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go#L37-L38)

### Styling Integration

Styles are parsed from the `Options` struct using `ToLipgloss()` and applied during the `View()` call [confirm/command.go43-45](https://github.com/charmbracelet/gum/blob/30cc5169/confirm/command.go#L43-L45)

*   `selectedStyle`: Applied to the button currently matching `m.confirmation`.
*   `unselectedStyle`: Applied to the other button.
*   `promptStyle`: Applied to the question text.
