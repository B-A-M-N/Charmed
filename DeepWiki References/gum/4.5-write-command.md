Write Command
=============

Relevant source files

*   [write/command.go](https://github.com/charmbracelet/gum/blob/30cc5169/write/command.go)
    
*   [write/options.go](https://github.com/charmbracelet/gum/blob/30cc5169/write/options.go)

The Write Command (`gum write`) provides a multi-line text editor interface in the terminal for capturing user input. It is designed for shell scripts and command-line workflows that require multi-line input, such as notes, descriptions, or code blocks. For single-line input, see page 4.4 (Input Command).

Overview
--------

The `gum write` command launches an interactive TUI (text user interface) using the Bubble Tea framework and the Bubbles `textarea` component [write/command.go18-26](https://github.com/charmbracelet/gum/blob/30cc5169/write/command.go#L18-L26)
 User input is collected in a multi-line text area and output to standard output upon submission [write/command.go85](https://github.com/charmbracelet/gum/blob/30cc5169/write/command.go#L85-L85)

Command Architecture
--------------------

The Write Command is implemented as a Bubble Tea application that wraps the Bubbles `textarea.Model`. When executed via `Run()`, it processes configuration options, initializes the TUI, and manages user interaction until the text is submitted or the session is aborted [write/command.go20-87](https://github.com/charmbracelet/gum/blob/30cc5169/write/command.go#L20-L87)

### Key Features

*   **Multi-line Editing**: Supports multi-line text entry with cursor navigation [write/write.go1-7](https://github.com/charmbracelet/gum/blob/30cc5169/write/write.go#L1-L7)
    
*   **Auto-Width Behavior**: If width is not specified (set to 0), the component can automatically adjust to the terminal width [write/command.go51-59](https://github.com/charmbracelet/gum/blob/30cc5169/write/command.go#L51-L59)

Configuration Options
---------------------

The Write Command provides a range of options for customizing appearance and behavior:

| Option | Description | Default | Environment Variable |
| --- | --- | --- | --- |
| `--width` | Text area width (0 = auto/terminal width) | 0   | `GUM_WRITE_WIDTH` |
| `--height` | Text area height | 5   | `GUM_WRITE_HEIGHT` |
| `--header` | Header text above textarea | ""  | `GUM_WRITE_HEADER` |
| `--placeholder` | Placeholder text | "Write something..." | `GUM_WRITE_PLACEHOLDER` |
| `--prompt` | Prompt string | "┃ " | `GUM_WRITE_PROMPT` |
| `--show-cursor-line` | Highlight cursor line | false | `GUM_WRITE_SHOW_CURSOR_LINE` |
| `--show-line-numbers` | Show line numbers | false | `GUM_WRITE_SHOW_LINE_NUMBERS` |
| `--value` | Initial value | ""  | `GUM_WRITE_VALUE` |
| `--char-limit` | Max value length (0 = unlimited) | 0   | \-  |
| `--max-lines` | Max number of lines (0 = unlimited) | 0   | \-  |
| `--show-help` | Show help bar | true | `GUM_WRITE_SHOW_HELP` |
| `--cursor.mode` | Cursor mode (blink/hide/static) | "blink" | `GUM_WRITE_CURSOR_MODE` |
| `--timeout` | Timeout before exit | 0s  | `GUM_WRITE_TIMEOUT` |
| `--strip-ansi` | Strip ANSI from stdin | true | `GUM_WRITE_STRIP_ANSI` |

Sources: [write/options.go10-36](https://github.com/charmbracelet/gum/blob/30cc5169/write/options.go#L10-L36)

Implementation Details
----------------------

### External Editor Integration

The `gum write` command allows users to jump into their preferred system editor (e.g., Vim, Nano). This is triggered by `km.OpenInEditor` (`Ctrl+E`) [write/write.go59-62](https://github.com/charmbracelet/gum/blob/30cc5169/write/write.go#L59-L62)

1.  **Temporary File**: The application creates a temporary file with the current buffer content [write/write.go159-175](https://github.com/charmbracelet/gum/blob/30cc5169/write/write.go#L159-L175)
    
2.  **Execution**: It uses `tea.ExecProcess` to launch the editor [write/write.go201](https://github.com/charmbracelet/gum/blob/30cc5169/write/write.go#L201-L201)
    
3.  **Synchronization**: After the editor exits, the file content is read back into the `textarea` model [write/write.go120-125](https://github.com/charmbracelet/gum/blob/30cc5169/write/write.go#L120-L125)

Key Bindings
------------

The following key bindings are available in the Write Command [write/write.go43-67](https://github.com/charmbracelet/gum/blob/30cc5169/write/write.go#L43-L67)
:

| Key | Action |
| --- | --- |
| `Enter` | Submit text (triggers `km.Submit`) |
| `Ctrl+J` | Insert new line (triggers `km.InsertNewline`) |
| `Ctrl+C` | Cancel and exit (triggers `km.Abort`) |
| `Esc` | Quit (triggers `km.Quit`) |
| `Ctrl+E` | Open in external editor (triggers `km.OpenInEditor`) |
| Arrow keys | Navigate text (Default textarea behavior) |
| Backspace/Delete | Delete characters (Default textarea behavior) |

Summary
-------

The Write Command provides a robust interface for multi-line text input. By wrapping the `textarea` bubble and adding features like external editor support and shell-friendly output, it bridges the gap between simple shell prompts and full-blown text editors. For single-line inputs, the [Input Command](https://deepwiki.com/charmbracelet/gum/4.4-input-command)
 is recommended.
