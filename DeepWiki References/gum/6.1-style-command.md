Purpose and Scope
-----------------

The Style Command provides a shell script interface for applying [Lipgloss](https://github.com/charmbracelet/gum/blob/30cc5169/Lipgloss)
 styling to text without writing Go code. It is a non-interactive formatting command that reads input from stdin or command arguments and outputs styled text to stdout [style/command.go1-5](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L1-L5)

This page documents the style command's structure, available styling options, and execution model. For rendering markdown with syntax highlighting, see [Format Command](https://deepwiki.com/charmbracelet/gum/6.2-format-command)
. For combining styled text layouts, see [Join Command](https://deepwiki.com/charmbracelet/gum/6.3-join-command)
.

Command Overview
----------------

The Style Command enables shell scripts to apply terminal styling including colors, borders, spacing, and alignment to text. Unlike interactive commands that use Bubbletea, the style command operates as a pure text transformer.

**Key Characteristics:**

*   **Non-interactive**: No Bubbletea TUI; directly transforms input to output [style/command.go35-36](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L35-L36)
    
*   **Stateless**: Each invocation is independent.
*   **Composable**: Designed for use in shell pipelines [style/command.go23](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L23-L23)
    
*   **Declarative**: Styling configured entirely through CLI flags [style/options.go11-40](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L11-L40)

Command Structure
-----------------

### Options Configuration

The `style.Options` struct defines the command's behavior and embeds the style flags [style/options.go4-9](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L4-L9)

| Field | Type | Help Description |
| --- | --- | --- |
| `Text` | `[]string` | Text to which to apply the style [style/options.go5](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L5-L5) |
| `Trim` | `bool` | Trim whitespaces on every input line [style/options.go6](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L6-L6) |
| `StripANSI` | `bool` | Strip ANSI sequences when reading from STDIN [style/options.go7](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L7-L7) |
| `Style` | `StylesNotHidden` | Embedded styling flags [style/options.go8](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L8-L8) |

### Input Processing Logic

The command prioritizes positional arguments over standard input. If `Text` is provided, it joins them with newlines [style/command.go20-21](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L20-L21)
 If not, it reads from `stdin` using `stdin.Read` [style/command.go23](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L23-L23)

If the `--trim` flag is set, the command iterates through every line of the input text and removes leading/trailing whitespace [style/command.go28-34](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L28-L34)

Sources: [style/command.go18-37](https://github.com/charmbracelet/gum/blob/30cc5169/style/command.go#L18-L37)

### Spacing and Layout

Spacing options (padding and margin) support a shorthand syntax similar to CSS, parsed by `ParsePadding` [style/spacing.go16-47](https://github.com/charmbracelet/gum/blob/30cc5169/style/spacing.go#L16-L47)

*   **1 token**: Applied to all sides (top, right, bottom, left) [style/spacing.go34-36](https://github.com/charmbracelet/gum/blob/30cc5169/style/spacing.go#L34-L36)
    
*   **2 tokens**: First is vertical (top/bottom), second is horizontal (left/right) [style/spacing.go38-40](https://github.com/charmbracelet/gum/blob/30cc5169/style/spacing.go#L38-L40)
    
*   **4 tokens**: Applied as top, right, bottom, left respectively [style/spacing.go42-44](https://github.com/charmbracelet/gum/blob/30cc5169/style/spacing.go#L42-L44)
    

Layout flags include:

*   `--align`: Horizontal/Vertical alignment using `decode.Align` map [style/options.go58](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L58-L58)
     [internal/decode/align.go7-13](https://github.com/charmbracelet/gum/blob/30cc5169/internal/decode/align.go#L7-L13)
    
*   `--width`: Fixed text width [style/options.go60](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L60-L60)
    
*   `--height`: Fixed text height [style/options.go59](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L59-L59)

Integration with Other Commands
-------------------------------

The `style.Styles` struct is designed for embedding. Other commands (like `choose` or `filter`) use `Styles` to provide consistent styling for their UI elements (e.g., cursor styles, header styles) [style/options.go11-17](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L11-L17)
 While `StylesNotHidden` is used for the standalone `gum style` command to ensure all flags are visible in help output, `Styles` is used for embedding where some flags might be hidden by default [style/options.go42-47](https://github.com/charmbracelet/gum/blob/30cc5169/style/options.go#L42-L47)
