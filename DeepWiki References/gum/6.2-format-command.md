Purpose and Scope
-----------------

The `format` command is a non-interactive utility in `gum` designed to process and render bodies of text for display in the terminal. It supports four distinct formatting types: markdown rendering, code syntax highlighting, template string processing with styling helpers, and emoji name expansion. It serves as a text transformer within shell pipelines, bridging raw data and human-readable terminal output.

**Sources:** [format/command.go1-11](https://github.com/charmbracelet/gum/blob/30cc5169/format/command.go#L1-L11)
 [format/README.md1-10](https://github.com/charmbracelet/gum/blob/30cc5169/format/README.md?plain=1#L1-L10)

Implementation Overview
-----------------------

The command is implemented as a standard `Run()` method on the `Options` struct. Unlike interactive commands in `gum` (e.g., `choose`, `filter`), `format` does not initialize a Bubble Tea tea.Model. Instead, it acts as a synchronous processor that reads input, applies a transformation via external libraries like `glamour` or `termenv`, and prints the result to `stdout`.

**Sources:** [format/command.go21-46](https://github.com/charmbracelet/gum/blob/30cc5169/format/command.go#L21-L46)
 [format/formats.go13-67](https://github.com/charmbracelet/gum/blob/30cc5169/format/formats.go#L13-L67)

Input Handling
--------------

The command handles input from two sources, managed within `format/command.go`:

1.  **Positional Arguments**: If `o.Template` contains strings, they are joined with newlines [format/command.go24-25](https://github.com/charmbracelet/gum/blob/30cc5169/format/command.go#L24-L25)
    
2.  **Standard Input**: If no arguments are provided, it uses the `internal/stdin` package to read from `os.Stdin` [format/command.go27](https://github.com/charmbracelet/gum/blob/30cc5169/format/command.go#L27-L27)
    

The `stdin.Read` function provides the ability to strip ANSI escape sequences from the input if `o.StripANSI` is enabled (default is true) [format/options.go10](https://github.com/charmbracelet/gum/blob/30cc5169/format/options.go#L10-L10)
 [internal/stdin/stdin.go73-76](https://github.com/charmbracelet/gum/blob/30cc5169/internal/stdin/stdin.go#L73-L76)

**Sources:** [format/command.go21-28](https://github.com/charmbracelet/gum/blob/30cc5169/format/command.go#L21-L28)
 [internal/stdin/stdin.go37-77](https://github.com/charmbracelet/gum/blob/30cc5169/internal/stdin/stdin.go#L37-L77)

Format Types and Renderers
--------------------------

### Markdown

Renders input as markdown using the `glamour` library. It supports custom themes via the `--theme` flag (defaulting to "pink"). Word wrapping is explicitly disabled (`WithWordWrap(0)`) to allow the terminal to handle layout or for piping to other tools [format/formats.go42-55](https://github.com/charmbracelet/gum/blob/30cc5169/format/formats.go#L42-L55)

### Code

Provides syntax highlighting by wrapping the input in markdown code blocks (` ```language ... ``` `) and passing it to `glamour` with `WithAutoStyle()` [format/formats.go13-26](https://github.com/charmbracelet/gum/blob/30cc5169/format/formats.go#L13-L26)
 The programming language is specified via the `--language` flag [format/options.go8](https://github.com/charmbracelet/gum/blob/30cc5169/format/options.go#L8-L8)

### Template

Uses Go's `text/template` package combined with `termenv.TemplateFuncs` [format/formats.go57-59](https://github.com/charmbracelet/gum/blob/30cc5169/format/formats.go#L57-L59)
 This enables users to use terminal-specific helpers like `{{ Bold "text" }}` or `{{ Color "99" "0" "text" }}` directly in their strings [format/README.md40-44](https://github.com/charmbracelet/gum/blob/30cc5169/format/README.md?plain=1#L40-L44)

### Emoji

Utilizes `glamour` with the `WithEmoji()` option to parse and render `:emoji_name:` sequences into Unicode characters [format/formats.go28-40](https://github.com/charmbracelet/gum/blob/30cc5169/format/formats.go#L28-L40)

| Type | Library | Key Functions |
| --- | --- | --- |
| `markdown` | `github.com/charmbracelet/glamour` | `glamour.NewTermRenderer`, `renderer.Render` |
| `code` | `github.com/charmbracelet/glamour` | `glamour.WithAutoStyle`, `fmt.Sprintf` (wrapping) |
| `template` | `github.com/muesli/termenv` | `termenv.TemplateFuncs`, `tpl.Execute` |
| `emoji` | `github.com/charmbracelet/glamour` | `glamour.WithEmoji` |
