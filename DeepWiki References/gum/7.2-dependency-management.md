Dependency Management
=====================

Dependency management is a critical component of the Gum project, with an importance score of 355.86 in the codebase analysis—the highest of any system component. This reflects the foundational role that external dependencies play in Gum's functionality. The project relies heavily on the Charmbracelet ecosystem for terminal UI capabilities, along with supporting libraries for CLI parsing, text processing, and system interaction.

This document covers the Go module system, dependency versioning strategy, the structure of direct and indirect dependencies, and best practices for managing updates.

Go Module System
----------------

Gum uses Go's standard module system for dependency management. The primary files that define and control dependencies are:

*   `go.mod`: Defines the module path and lists direct dependencies with their versions.
*   `go.sum`: Contains cryptographic checksums of specific dependency versions to ensure build reproducibility.

### go.mod Structure

The `go.mod` file defines the module path (`github.com/charmbracelet/gum`), minimum Go version requirements, and dependency declarations. The file is structured into three main sections:

1.  **Module declaration and toolchain** [go.mod1-3](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L1-L3)
    
2.  **Direct dependencies** (`require` block) [go.mod5-24](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L5-L24)
    
3.  **Indirect dependencies** [go.mod26-61](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L26-L61)
    

#### Module Declaration and Go Version

The `go 1.25.8` directive specifies the Go version required to build Gum [go.mod3](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L3-L3)

Indirect Dependencies
---------------------

Dependency Versioning and Locking
---------------------------------

Gum uses semantic versioning and cryptographic locking to ensure build reproducibility.

### Version Locking with go.sum

The `go.sum` file contains SHA256 checksums for each dependency and its source code [go.sum1-81](https://github.com/charmbracelet/gum/blob/30cc5169/go.sum#L1-L81)
 This ensures that the same dependency versions are used across all builds and that dependencies haven't been tampered with.

**Example entries from go.sum:**

    github.com/charmbracelet/bubbletea v1.3.10 h1:otUDHWMMzQSB0Pkc87rm691KZ3SWa4KUlvF9nRvCICw=
    github.com/charmbracelet/bubbletea v1.3.10/go.mod h1:ORQfo0fk8U+po9VaNvnV95UPWA1BitP1E0N6xJPlHr4=
    

### Pseudo-Versions

Some dependencies use pseudo-versions to reference specific Git commits before official releases:

*   `github.com/charmbracelet/lipgloss v1.1.1-0.20250404203927-76690c660834` [go.mod12](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L12-L12)
    
*   `github.com/charmbracelet/x/errors v0.0.0-20240508181413-e8d8b6e2de86` [go.mod34](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L34-L34)
    

Automated Management
--------------------

Gum employs automated tools to keep dependencies secure and up-to-date.

### Dependabot Configuration

The project uses GitHub Dependabot to scan for updates weekly on Mondays at 05:00 EST [.github/dependabot.yml6-10](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L6-L10)
 It monitors:

*   **Go Modules**: Updates for libraries in `go.mod` [.github/dependabot.yml4-5](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L4-L5)
    
*   **GitHub Actions**: Updates for CI/CD workflows [.github/dependabot.yml25-26](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L25-L26)
    
*   **Docker**: Updates for container images [.github/dependabot.yml42-43](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L42-L43)
    

Specific versions are ignored to prevent breaking changes, such as `bubbletea/v2` beta versions [.github/dependabot.yml21-23](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L21-L23)

### Dependabot Sync

A dedicated workflow `dependabot-sync.yml` runs every Sunday at midnight to synchronize Dependabot configurations across the Charmbracelet organization using a shared meta-workflow [.github/workflows/dependabot-sync.yml1-18](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/dependabot-sync.yml#L1-L18)
