Purpose and Scope
-----------------

This document details the external framework dependencies that form the foundation of `gum`'s interactive terminal user interface capabilities. Specifically, it covers the Charmbracelet ecosystem libraries—Bubbletea, Bubbles, Lipgloss, Glamour, and Log—explaining their roles, versions, and how `gum` commands leverage them to build consistent, styled TUI experiences.

For information about how these components are integrated into the application bootstrap process, see [Application Entry Point](https://deepwiki.com/charmbracelet/gum/3.1-application-entry-point)
. For patterns on how individual commands structure their usage of these frameworks, see [Command Patterns](https://deepwiki.com/charmbracelet/gum/3.3-command-patterns)
.

Bubbletea: TUI Framework
------------------------

Bubbletea (`github.com/charmbracelet/bubbletea`) is the foundational framework providing the event-driven Model-View-Update (MVU) architecture. All interactive `gum` commands implement the Bubbletea pattern.

### Core Interfaces and Types

Every interactive command creates a struct implementing the `tea.Model` interface:

*   **`Init() tea.Cmd`**: Initializes the model and returns initial commands.
*   **`Update(tea.Msg) (tea.Model, tea.Cmd)`**: Handles incoming messages and updates state.
*   **`View() string`**: Renders the current state to a string.

### Program Lifecycle

Commands instantiate Bubbletea programs using `tea.NewProgram()`. Common configuration options include routing UI to `os.Stderr` to preserve `stdout` for the command's functional output, and integrating context for timeout handling.

### Message Types

Bubbletea uses message passing for events:

*   **`tea.KeyMsg`**: Keyboard input events.
*   **`tea.MouseMsg`**: Mouse interaction events.
*   **`tea.WindowSizeMsg`**: Terminal resize events.
*   Custom messages for command-specific logic.

Sources: [go.mod10](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L10-L10)

Bubbles: UI Components Library
------------------------------

Bubbles (`github.com/charmbracelet/bubbles`) provides pre-built, composable UI components that implement the `tea.Model` interface. `gum` commands embed these components to build complex interactions.

### Component Mapping by Command

### Component Usage Patterns

#### textinput.Model

Used for single-line text input with customization. Commands like `filter` use it for search queries, while `input` uses it for primary data entry.

#### viewport.Model

Provides scrollable content areas. It is the primary component for the `pager` command and is used in `filter` and `choose` to handle lists that exceed terminal height.

#### paginator.Model

Implements multi-page navigation with visual indicators (dots or numbers). Used by the `choose` command for handling large option lists.

#### help.Model

Provides consistent keybinding help display. Embedded in interactive commands to display available keyboard shortcuts to the user.

Sources: [go.mod9](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L9-L9)

Lipgloss: Styling Engine
------------------------

Lipgloss (`github.com/charmbracelet/lipgloss`) provides the styling system for colors, borders, padding, margins, and alignment. All visual presentation in `gum` commands uses Lipgloss styles.

### Style Creation Patterns

Commands define styles using `lipgloss.NewStyle()` with chainable methods. Adaptive colors are frequently used to support both light and dark terminal themes.

Additional Charmbracelet Libraries
----------------------------------

### glamour: Markdown Rendering

Used by the `format` command to render markdown content with syntax highlighting. It integrates with Lipgloss for consistent styling across the CLI.

### log: Structured Logging

Used by the `log` command to provide leveled, structured logging output with formatting.

### charmbracelet/x Utilities

Additional utility packages from the extended Charmbracelet ecosystem provide specialized functionality:

| Package | Purpose |
| --- | --- |
| `charmbracelet/x/ansi` | ANSI escape sequence handling |
| `charmbracelet/x/editor` | External editor integration (used by `write`) |
| `charmbracelet/x/term` | Terminal capability detection |
| `charmbracelet/x/xpty` | Pseudo-terminal (PTY) management (used by `spin`) |
