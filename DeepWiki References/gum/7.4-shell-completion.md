Shell Completion
================

This document describes how shell completion is generated and installed for the `gum` CLI, covering the supported shells (Bash, Zsh, and Fish), the code entities responsible for completion, and the process for users and developers to enable completion in their environments.

Purpose and Scope
-----------------

Shell completion enables users to auto-complete `gum` commands, subcommands, and flags in their shell environment. This improves usability and discoverability of the CLI. This document covers:

*   The code structure for shell completion in `gum`
*   How completion scripts are generated for different shells
*   How users can install and activate completion for their shell
*   The relationship between the completion system and the `kong` CLI parser

* * *

Overview of Shell Completion in gum
-----------------------------------

Shell completion in `gum` is implemented as a set of generators that traverse the `kong` command tree. While `gum` uses the `kong` CLI parser, it provides custom generator logic for Bash, Zsh, and Fish to ensure high-quality completion scripts.

Implementation Details
----------------------

### 1\. Command Registration

The `Completion` struct registers the subcommands using `kong` tags. Each shell has its own dedicated struct and `Run` method.

Sources: [completion/command.go13-17](https://github.com/charmbracelet/gum/blob/30cc5169/completion/command.go#L13-L17)

### 2\. Common Utilities

The `completion` package provides shared helper functions to process the `kong` command tree:

*   `commandName(cmd *kong.Node)`: Formats the command path into a shell-safe string (e.g., replacing spaces with underscores) [completion/command.go19-24](https://github.com/charmbracelet/gum/blob/30cc5169/completion/command.go#L19-L24)
    
*   `flagPossibleValues(flag *kong.Flag)`: Extracts allowed enum values for a flag to provide precise completions [completion/command.go52-60](https://github.com/charmbracelet/gum/blob/30cc5169/completion/command.go#L52-L60)
    
*   `hasCommands(cmd *kong.Node)`: Determines if a node has non-hidden child commands [completion/command.go26-33](https://github.com/charmbracelet/gum/blob/30cc5169/completion/command.go#L26-L33)
    

### 3\. Shell-Specific Logic

#### Zsh Completion

The Zsh generator uses a recursive `gen` function [completion/zsh.go105-146](https://github.com/charmbracelet/gum/blob/30cc5169/completion/zsh.go#L105-L146)
 to build a completion function for every command and subcommand. It leverages Zsh's `_arguments` and `_values` utilities.

*   `writeFlags`: Iterates through `cmd.Flags` to generate Zsh flag specifications, including help text and enum values [completion/zsh.go76-86](https://github.com/charmbracelet/gum/blob/30cc5169/completion/zsh.go#L76-L86)
    
*   `writeCommands`: Generates the list of available subcommands for the current context [completion/zsh.go92-103](https://github.com/charmbracelet/gum/blob/30cc5169/completion/zsh.go#L92-L103)
    

#### Fish Completion

The Fish generator produces `complete` commands. It disables file completion by default using `complete -c gum -f` [completion/fish.go21](https://github.com/charmbracelet/gum/blob/30cc5169/completion/fish.go#L21-L21)
 and then selectively enables completions based on the command context.

*   It uses `__fish_use_subcommand` to suggest subcommands [completion/fish.go44](https://github.com/charmbracelet/gum/blob/30cc5169/completion/fish.go#L44-L44)
    
*   It uses `__fish_seen_subcommand_from` to provide context-aware flags [completion/fish.go64](https://github.com/charmbracelet/gum/blob/30cc5169/completion/fish.go#L64-L64)
    

#### Bash Completion

The Bash generator is the most complex, involving a large preamble [completion/bash.go96-182](https://github.com/charmbracelet/gum/blob/30cc5169/completion/bash.go#L96-L182)
 that defines helper functions like `__gum_init_completion` and `__gum_handle_go_custom_completion`. It handles:

*   Debugging via `BASH_COMP_DEBUG_FILE` [completion/bash.go101](https://github.com/charmbracelet/gum/blob/30cc5169/completion/bash.go#L101-L101)
    
*   Custom completion directives (e.g., `ShellCompDirectiveNoFileComp`) [completion/bash.go34-72](https://github.com/charmbracelet/gum/blob/30cc5169/completion/bash.go#L34-L72)

Installation and Usage
----------------------

To use these completions, the generated script must be sourced or placed in a directory recognized by the shell.

| Shell | Command to Generate | Typical Installation Path |
| --- | --- | --- |
| **Bash** | `gum completion bash` | `/etc/bash_completion.d/gum` |
| **Zsh** | `gum completion zsh` | Any directory in `$fpath` as `_gum` |
| **Fish** | `gum completion fish` | `~/.config/fish/completions/gum.fish` |

Man Page Generation
-------------------

While not strictly shell completion, the `man` package follows a similar pattern for generating documentation. It uses `mangokong` to convert the `kong` model into a Roff-formatted man page.

*   **Entity**: `Man` struct [man/command.go13](https://github.com/charmbracelet/gum/blob/30cc5169/man/command.go#L13-L13)
    
*   **Logic**: The `BeforeApply` hook intercepts the command execution, builds the man page using `mangokong.NewManPage`, and prints it to `Stdout` before exiting [man/command.go16-25](https://github.com/charmbracelet/gum/blob/30cc5169/man/command.go#L16-L25)

Maintenance Notes
-----------------

*   **Hidden Commands**: Generators explicitly check for `f.Hidden` or `c.Hidden` to avoid exposing internal or undocumented features in completion scripts [completion/zsh.go78](https://github.com/charmbracelet/gum/blob/30cc5169/completion/zsh.go#L78-L78)
     [completion/fish.go53](https://github.com/charmbracelet/gum/blob/30cc5169/completion/fish.go#L53-L53)
     [completion/command.go48-50](https://github.com/charmbracelet/gum/blob/30cc5169/completion/command.go#L48-L50)
    
*   **Enum Support**: If a flag has an `enum` tag in the `Options` struct, the generators will automatically include those specific values in the suggestions via `flagPossibleValues` [completion/command.go52-60](https://github.com/charmbracelet/gum/blob/30cc5169/completion/command.go#L52-L60)
