Purpose and Scope
-----------------

The Table Command in Gum CLI allows users to render tabular data (specifically CSV data) in the terminal with an interactive interface. It supports both static display and interactive selection of table rows, making it useful for displaying structured data and allowing users to select specific rows or columns for use in shell scripts.

Sources: [table/table.go1-15](https://github.com/charmbracelet/gum/blob/30cc5169/table/table.go#L1-L15)

Overview
--------

The Table Command reads CSV data (either from a file or standard input), parses it, and displays it as a formatted table. In interactive mode (default), users can navigate through the table rows and select one, with the selected row or a specific column from it being returned to stdout.

Command-Line Options
--------------------

The `Options` struct defines the CLI interface, including parsing logic for CSV and styling for the TUI components.

| Option | Description | Default |
| --- | --- | --- |
| `--separator`, `-s` | Row separator character | `,` |
| `--columns`, `-c` | Column names (overrides CSV header) | (from first row) |
| `--widths`, `-w` | Explicit column widths | (auto-calculated) |
| `--height` | Table height in rows | 0 (auto) |
| `--print`, `-p` | Static print mode (non-interactive) | `false` |
| `--file`, `-f` | Path to CSV file | (stdin) |
| `--border`, `-b` | Border style (rounded, thick, etc.) | `rounded` |
| `--return-column`, `-r` | Specific column index to return (1-based) | 0 (whole row) |
| `--lazy-quotes` | Allow unquoted quotes in CSV fields | `false` |

Sources: [table/options.go10-30](https://github.com/charmbracelet/gum/blob/30cc5169/table/options.go#L10-L30)

Core Implementation
-------------------

### CSV Parsing and BOM Handling

The command uses a `unicode.BOMOverride` transformer to ensure that files with Byte Order Marks (common in Windows-exported CSVs) are handled correctly before passing the stream to `csv.NewReader`.

Sources: [table/command.go38-41](https://github.com/charmbracelet/gum/blob/30cc5169/table/command.go#L38-L41)

### Data Normalization

The implementation ensures that every row matches the number of columns, appending empty strings if a row is shorter than the header set.

Sources: [table/command.go97-99](https://github.com/charmbracelet/gum/blob/30cc5169/table/command.go#L97-L99)

### Interactive Table Model

The `model` struct manages the state of the interactive TUI. It wraps the `bubbles/table` component and adds a help view and row counter.

*   **Update**: Handles row selection via `km.Select` (Enter key), storing the selected row in `m.selected` and triggering `tea.Quit`.
*   **View**: Renders the table within a `lipgloss` style that applies padding configured via `--padding`.

Sources: [table/table.go95-117](https://github.com/charmbracelet/gum/blob/30cc5169/table/table.go#L95-L117)
 [table/table.go119-130](https://github.com/charmbracelet/gum/blob/30cc5169/table/table.go#L119-L130)

### Selection Output

After the `tea.Program` finishes, the command retrieves the selection from the model. If `--return-column` is set, it extracts only that specific field; otherwise, it writes the full row using a `csv.Writer` to ensure proper quoting in the output.

Sources: [table/command.go167-175](https://github.com/charmbracelet/gum/blob/30cc5169/table/command.go#L167-L175)

Display Modes
-------------

### Static Print Mode

When `--print` is enabled, the command bypasses the Bubble Tea loop entirely. It uses `lipgloss/table` to render a high-fidelity static table directly to stdout. This mode supports the same border and style configurations as the interactive mode.

Sources: [table/command.go113-128](https://github.com/charmbracelet/gum/blob/30cc5169/table/command.go#L113-L128)

### Keyboard Navigation

The interactive mode uses a standard keymap for navigation:

| Key | Action | Code Binding |
| --- | --- | --- |
| `Up`/`Down` | Navigate between rows | `Navigate` |
| `Enter` | Confirm selection | `Select` |
| `Esc`/`Q` | Quit without selection | `Quit` |
| `Ctrl+C` | Abort (Interrupt) | `Abort` |

Sources: [table/table.go47-66](https://github.com/charmbracelet/gum/blob/30cc5169/table/table.go#L47-L66)
