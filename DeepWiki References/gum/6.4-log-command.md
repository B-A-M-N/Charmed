Log Command
===========

Relevant source files

*   [log/command.go](https://github.com/charmbracelet/gum/blob/30cc5169/log/command.go)
    
*   [log/options.go](https://github.com/charmbracelet/gum/blob/30cc5169/log/options.go)
    

Purpose and Scope
-----------------

The `log` command provides a shell script interface for structured logging using the `charmbracelet/log` library. It allows shell scripts to emit formatted, leveled log messages with timestamps and styling without requiring Go code. The command outputs directly to `stderr` by default (or a file), making it suitable for continuous logging in shell pipelines and scripts.

**Sources:** [log/command.go1-17](https://github.com/charmbracelet/gum/blob/30cc5169/log/command.go#L1-L17)
 [log/options.go1-28](https://github.com/charmbracelet/gum/blob/30cc5169/log/options.go#L1-L28)

Command Structure
-----------------

The log command follows a non-interactive execution pattern. It initializes a logger instance, configures its output and styling based on CLI flags, and then dispatches the message to the appropriate log level method.

**Sources:** [log/command.go16-135](https://github.com/charmbracelet/gum/blob/30cc5169/log/command.go#L16-L135)
 [log/options.go8-28](https://github.com/charmbracelet/gum/blob/30cc5169/log/options.go#L8-L28)

Log Levels and Filtering
------------------------

The command supports standard log levels. It uses a minimum level filter (`MinLevel`) to decide which messages are actually emitted. By default, the logger is initialized to log all levels internally, but the `MinLevel` flag (or `GUM_LOG_LEVEL` environment variable) provides the threshold for output.

| Level String | `log.Level` Constant | Purpose |
| --- | --- | --- |
| `none` | `math.MaxInt32` | Disables logging or used as a default |
| `debug` | `log.DebugLevel` | Detailed diagnostic information |
| `info` | `log.InfoLevel` | Informational messages |
| `warn` | `log.WarnLevel` | Warning messages |
| `error` | `log.ErrorLevel` | Error messages |
| `fatal` | `log.FatalLevel` | Critical errors (calls `log.Fatal`) |

**Sources:** [log/command.go142-149](https://github.com/charmbracelet/gum/blob/30cc5169/log/command.go#L142-L149)
 [log/options.go14-19](https://github.com/charmbracelet/gum/blob/30cc5169/log/options.go#L14-L19)

Output Format and Styling
-------------------------

The command allows deep customization of the log appearance using `lipgloss` styles embedded within the `Options` struct.

### Components of a Log Line

The `charmbracelet/log.Styles` struct is populated from the `Options` struct before the log is emitted.

| Option Field | Target Style Component |
| --- | --- |
| `LevelStyle` | The log level indicator (e.g., "INFO") |
| `TimeStyle` | The timestamp |
| `PrefixStyle` | The message prefix |
| `MessageStyle` | The main log message text |
| `KeyStyle` | Keys in structured `key=value` pairs |
| `ValueStyle` | Values in structured `key=value` pairs |
| `SeparatorStyle` | The character separating keys and values |

**Sources:** [log/command.go69-93](https://github.com/charmbracelet/gum/blob/30cc5169/log/command.go#L69-L93)
 [log/options.go21-27](https://github.com/charmbracelet/gum/blob/30cc5169/log/options.go#L21-L27)

### Time Formatting

The command supports standard Go time layouts and several named aliases for convenience.

**Sources:** [log/command.go40-67](https://github.com/charmbracelet/gum/blob/30cc5169/log/command.go#L40-L67)

Implementation Details
----------------------

The core logic resides in `log.Options.Run()`. It handles the transition from shell arguments to the structured logging library.

### Logger Initialization and Dispatch

The command uses a helper `logger` struct to abstract the difference between standard printing and `printf`\-style formatting.

Depending on the flags provided, the command chooses one of three logging modes:

1.  **Formatted (`--format`)**: Uses `printf` logic.
2.  **Structured (`--structured`)**: Passes arguments as key-value pairs.
3.  **Simple**: Joins all text arguments with spaces and logs them as a single string.
