This page provides detailed guidelines for building, testing, and developing the `gum` CLI tool. It covers build processes, test frameworks, continuous integration workflows, and quality assurance tools used in the project.

Prerequisites
-------------

Before building or testing `gum`, ensure you have the following prerequisites installed:

*   **Go 1.21.0 or later** (minimum version specified in [go.mod3](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L3-L3)
    )
*   **Toolchain: Go 1.21.0** (specified in [go.mod5](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L5-L5)
    )
*   Git

Sources: [go.mod3-5](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L3-L5)

-------------

`gum` can be built using standard Go tooling. The project structure allows for building the entire suite of subcommands at once.

Testing Infrastructure
----------------------

`gum` utilizes Go's standard library `testing` package for unit and integration tests. The test suite is designed to run quickly and reliably across different environments.

### Running Tests Locally

The test suite should be executed with the same flags used in the CI environment to ensure consistency:

| Flag | Purpose |
| --- | --- |
| `-v` | Verbose output showing individual test results. |
| `-cover` | Generate test coverage metrics. |
| `-timeout=30s` | Abort tests that run longer than 30 seconds to prevent hanging CI runners. |
| `./...` | Recursively test all packages in the repository. |

Sources: [.github/workflows/build.yml28-29](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/build.yml#L28-L29)

### Test Organization and Coverage

Tests are located alongside the code they verify, following the `_test.go` naming convention.

**Example: Filter Command Testing** The `filter` package includes tests for internal logic such as fuzzy match range calculation and coordinate conversion between byte positions and visible character positions.

*   `TestMatchedRanges`: Validates the `matchedRanges` function which groups individual character indices into contiguous ranges for highlighting ([filter/filter\_test.go10-43](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter_test.go#L10-L43)
    ).
*   `TestByteToChar`: Validates `bytePosToVisibleCharPos`, ensuring that ANSI escape sequences and multi-byte characters (like emojis) do not break text highlighting ([filter/filter\_test.go45-59](https://github.com/charmbracelet/gum/blob/30cc5169/filter/filter_test.go#L45-L59)
    ).

### Manual Testing with Examples

The repository includes a `test.sh` script in the `examples/` directory. This script serves as a manual smoke test, executing various `gum` commands with complex flags to verify visual output and interactivity.

Continuous Integration (CI)
---------------------------

`gum` uses GitHub Actions for automated quality assurance. The CI pipeline is split into build/test and linting workflows.

### Build and Test Matrix

The `build.yml` workflow ensures cross-platform compatibility by running the test suite on Linux, macOS, and Windows.

Sources: [.github/workflows/build.yml1-29](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/build.yml#L1-L29)

### Linting and Code Quality

`gum` maintains high code quality standards through automated linting.

*   **Static Analysis**: The project uses `golangci-lint` with a comprehensive set of linters including `gosec` (security), `revive` (style), and `misspell` ([.golangci.yml4-27](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L4-L27)
    ).
*   **Formatting**: Code formatting is enforced via `gofumpt` and `goimports` ([.golangci.yml44-45](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L44-L45)
    ).
*   **Automated Sync**: A `lint-sync.yml` workflow runs weekly to ensure linting configurations stay up-to-date with the upstream `charmbracelet/meta` standards ([.github/workflows/lint-sync.yml1-15](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/lint-sync.yml#L1-L15)
    ).

Troubleshooting Build Issues
----------------------------

| Symptom | Cause | Resolution |
| --- | --- | --- |
| `go build` fails with version errors | Go version < 1.21 | Update Go to the version specified in `go.mod` ([go.mod3](https://github.com/charmbracelet/gum/blob/30cc5169/go.mod#L3-L3)<br>). |
| Tests timeout locally | Heavy I/O or TTY issues | Ensure the environment supports TTY or increase the `-timeout` flag beyond 30s. |
| Linting failures on PR | Style or security violations | Run `golangci-lint run` locally to identify specific issues identified in `.golangci.yml` ([.golangci.yml4-27](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L4-L27)<br>). |
