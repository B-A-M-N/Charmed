This document describes the Continuous Integration and Continuous Deployment (CI/CD) infrastructure for the Gum project, implemented using GitHub Actions. The pipeline ensures code quality through automated linting, validates functionality across multiple operating systems, generates release artifacts, and automates dependency updates.

Overview
--------

The Gum CI/CD pipeline consists of several specialized workflows. While some workflows handle per-commit validation (linting and building), others manage periodic maintenance (dependency synchronization) or release orchestration.

| Workflow | File | Primary Purpose |
| --- | --- | --- |
| **lint** | [.github/workflows/lint.yml1-9](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/lint.yml#L1-L9) | Code quality enforcement using `golangci-lint`. |
| **build** | [.github/workflows/build.yml1-14](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/build.yml#L1-L14) | Cross-platform builds and testing. |
| **goreleaser** | [.github/workflows/goreleaser.yml1-30](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/goreleaser.yml#L1-L30) | Production release management and artifact distribution. |
| **nightly** | [.github/workflows/nightly.yml1-20](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/nightly.yml#L1-L20) | Automated nightly builds for the `main` branch. |
| **dependabot-sync** | [.github/workflows/dependabot-sync.yml1-18](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/dependabot-sync.yml#L1-L18) | Syncing Dependabot configurations from central meta-repo. |

Workflow Architecture
---------------------

The project leverages the `charmbracelet/meta` repository to maintain standardized CI/CD logic across the organization. Most local workflow files act as wrappers that call reusable workflows.

Linting and Quality Control
---------------------------

Gum uses `golangci-lint` with a specific configuration defined in [.golangci.yml1-48](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L1-L48)
 The linting process is triggered on every push and pull request [.github/workflows/lint.yml2-5](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/lint.yml#L2-L5)

### Linter Configuration

The project enables a wide array of linters to ensure code safety and consistency:

*   **Security:** `gosec` [.golangci.yml12](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L12-L12)
    
*   **Performance:** `prealloc` [.golangci.yml19](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L19-L19)
    
*   **Correctness:** `bodyclose` [.golangci.yml6](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L6-L6)
     `nilerr` [.golangci.yml16](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L16-L16)
     `rowserrcheck` [.golangci.yml21](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L21-L21)
    
*   **Style:** `gofumpt` [.golangci.yml44](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L44-L44)
     `goimports` [.golangci.yml45](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L45-L45)
     `misspell` [.golangci.yml13](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L13-L13)
    

Specific rules are excluded, such as context checks for logging functions [.golangci.yml30-32](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L30-L32)
 and generated code is treated with lax enforcement [.golangci.yml33](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L33-L33)

**Sources:** [.golangci.yml1-48](https://github.com/charmbracelet/gum/blob/30cc5169/.golangci.yml#L1-L48)
 [.github/workflows/lint.yml1-9](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/lint.yml#L1-L9)

Build and Release Management
----------------------------

### The Build Workflow

The build workflow validates the codebase on the `main` branch and in pull requests [.github/workflows/build.yml3-8](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/build.yml#L3-L8)
 It requires a `PERSONAL_ACCESS_TOKEN` passed as `gh_pat` to interact with organization-level resources [.github/workflows/build.yml12-13](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/build.yml#L12-L13)

### Release Orchestration (GoReleaser)

Gum uses **GoReleaser** to manage the lifecycle of a versioned release. This is triggered when a semantic version tag (e.g., `v1.2.3`) is pushed [.github/workflows/goreleaser.yml5-8](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/goreleaser.yml#L5-L8)

The release configuration [.goreleaser.yml1-20](https://github.com/charmbracelet/gum/blob/30cc5169/.goreleaser.yml#L1-L20)
 includes:

*   **Base Config:** Inherits from `charmbracelet/meta/main/goreleaser-full.yaml` [.goreleaser.yml7](https://github.com/charmbracelet/gum/blob/30cc5169/.goreleaser.yml#L7-L7)
    
*   **Variables:** Defines project-specific metadata like the Scoop bucket name `charm-gum` [.goreleaser.yml11](https://github.com/charmbracelet/gum/blob/30cc5169/.goreleaser.yml#L11-L11)
     and the maintainer info [.goreleaser.yml14-16](https://github.com/charmbracelet/gum/blob/30cc5169/.goreleaser.yml#L14-L16)
    
*   **Milestones:** Automatically closes GitHub milestones associated with the release [.goreleaser.yml19](https://github.com/charmbracelet/gum/blob/30cc5169/.goreleaser.yml#L19-L19)
    

**Sources:** [.github/workflows/goreleaser.yml1-30](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/goreleaser.yml#L1-L30)
 [.goreleaser.yml1-20](https://github.com/charmbracelet/gum/blob/30cc5169/.goreleaser.yml#L1-L20)

Dependency Automation
---------------------

Gum utilizes GitHub Dependabot to keep dependencies up to date. The configuration is split into three ecosystems, all scheduled for weekly updates on Monday mornings [.github/dependabot.yml6-10](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L6-L10)

| Ecosystem | Directory | Target |
| --- | --- | --- |
| `gomod` | `/` | Go modules (dependencies) [.github/dependabot.yml4-5](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L4-L5) |
| `github-actions` | `/` | CI/CD workflow actions [.github/dependabot.yml25-26](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L25-L26) |
| `docker` | `/` | Container base images [.github/dependabot.yml42-43](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L42-L43) |

### Special Rules

*   **Versioning:** Updates use the `chore(scope):` commit message prefix [.github/dependabot.yml13-15](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L13-L15)
    
*   **Grouping:** All updates are grouped into a single PR per ecosystem to reduce noise [.github/dependabot.yml16-19](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L16-L19)
    
*   **Exclusions:** `github.com/charmbracelet/bubbletea/v2` is ignored for version `v2.0.0-beta1` to avoid unstable major version migrations [.github/dependabot.yml21-23](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L21-L23)
    

**Sources:** [.github/dependabot.yml1-58](https://github.com/charmbracelet/gum/blob/30cc5169/.github/dependabot.yml#L1-L58)

Maintenance Workflows
---------------------

### Configuration Syncing

To maintain consistency with the Charmbracelet organization's standards, Gum includes "sync" workflows that periodically pull the latest CI/CD configurations:

*   **dependabot-sync:** Runs every Sunday at midnight to update Dependabot settings [.github/workflows/dependabot-sync.yml1-18](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/dependabot-sync.yml#L1-L18)
    
*   **lint-sync:** Runs every Sunday at midnight to update linting rules and configurations [.github/workflows/lint-sync.yml1-15](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/lint-sync.yml#L1-L15)
    

### Nightly Builds

A nightly workflow runs on every push to `main` [.github/workflows/nightly.yml3-6](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/nightly.yml#L3-L6)
 It builds and signs the latest development version of Gum, including macOS notarization using secrets like `MACOS_NOTARY_KEY` [.github/workflows/nightly.yml15-19](https://github.com/charmbracelet/gum/blob/30cc5169/.github/workflows/nightly.yml#L15-L19)
