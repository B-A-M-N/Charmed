Terminal Events
===============

Relevant source files

*   [options.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go)
    
*   [tea.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go)
    
*   [tty.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go)
    
*   [xterm.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/xterm.go)
    

This document covers terminal-specific events that Bubble Tea programs receive from the terminal emulator and operating system. These events include window resize notifications, terminal capability detection, focus events, and OS signals.

For keyboard and mouse input events, see [Keyboard Input](https://deepwiki.com/charmbracelet/bubbletea/3.1-keyboard-input)
 and [Mouse Input](https://deepwiki.com/charmbracelet/bubbletea/3.2-mouse-input)
. For platform-specific input handling differences, see [Cross-platform Input](https://deepwiki.com/charmbracelet/bubbletea/3.4-cross-platform-input)
.

* * *

Overview
--------

Terminal events are messages that originate from the terminal emulator or operating system, rather than from user input. Bubble Tea automatically handles the detection and dispatching of these events through the event loop in [tea.go722-864](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L722-L864)

The framework manages several categories of terminal events:

| Event Category | Message Types | Source |
| --- | --- | --- |
| Window Size | `WindowSizeMsg`, `windowSizeMsg` | SIGWINCH signal (Unix) or manual detection |
| Terminal Version | `TerminalVersionMsg` | XTVERSION response |
| Terminal Focus | `FocusMsg`, `BlurMsg` | Terminal focus reporting mode |
| Process Control | `SuspendMsg`, `ResumeMsg`, `InterruptMsg` | OS signals (SIGTSTP, SIGCONT, SIGINT) |
| Capabilities | `CapabilityMsg`, `ModeReportMsg` | Terminal capability queries |

**Sources:** [tea.go722-864](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L722-L864)
 [tea.go49-50](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L49-L50)

Window Resize Events
--------------------

### WindowSizeMsg Structure

When the terminal window is resized, Bubble Tea sends a `WindowSizeMsg` to the program's `Update` method. This message contains the new dimensions:

**Diagram: Window Resize Event Flow**

The resize detection system operates at two levels:

1.  **Signal-based detection** (Unix): The `handleResize` goroutine listens for SIGWINCH signals
2.  **Manual polling**: The `checkResize` function queries the terminal size on demand

**Sources:** [tea.go665-677](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L665-L677)
 [tty.go109-127](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L109-L127)
 [tea.go830-831](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L830-L831)

### Handling WindowSizeMsg

Applications receive `WindowSizeMsg` in their `Update` method:

The message is processed in the event loop at [tea.go830-831](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L830-L831)
 where it:

1.  Updates the renderer's internal size via `renderer.resize()`
2.  Forwards the message to the Model's `Update` method

**Sources:** [tea.go830-831](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L830-L831)

### Initial Size Detection

The initial window size is detected during program startup in [tea.go1023-1035](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1023-L1035)
:

**Diagram: Initial Size Detection Sequence**

**Sources:** [tea.go1023-1035](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1023-L1035)
 [tea.go1072-1074](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1072-L1074)

### Resize Detection Implementation

The resize handler is started in [tea.go1118](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1118-L1118)
 and managed by the `handleResize` function:

| Platform | Mechanism | Implementation |
| --- | --- | --- |
| Unix (Linux, macOS) | SIGWINCH signal | `listenForResize` in platform files |
| Windows | Not supported | Channel closed immediately |

On Unix systems, the `listenForResize` goroutine monitors SIGWINCH signals and triggers resize checks. On Windows, resize events are not automatically detected.

**Sources:** [tea.go665-677](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L665-L677)
 [tea.go1118](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1118-L1118)

Focus Events
------------

### Focus Reporting

Bubble Tea supports terminal focus reporting, which sends events when the terminal gains or loses focus. This is enabled via the `View.ReportFocus` field:

When focus reporting is enabled, the program receives:

*   `FocusMsg` - When the terminal gains focus
*   `BlurMsg` - When the terminal loses focus

**Sources:** [tea.go163-170](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L163-L170)

### Focus Event Support

Focus reporting support varies by terminal:

| Terminal/Multiplexer | Support | Notes |
| --- | --- | --- |
| Most modern terminals | Yes | Standard support |
| tmux | Configurable | Requires `focus-events on` in tmux.conf |
| Some terminals | No  | Check terminal documentation |

The focus events are parsed by the `uv.TerminalReader` in the ultraviolet library and dispatched through the event loop.

**Sources:** [tea.go163-170](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L163-L170)

OS Signal Handling
------------------

### Signal Handler Goroutine

Bubble Tea runs a dedicated goroutine to handle OS signals, implemented in [tea.go624-663](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L624-L663)
:

**Diagram: OS Signal Handling Flow**

**Sources:** [tea.go624-663](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L624-L663)

### Signal Types

The signal handler monitors these OS signals:

| Signal | Message Type | Behavior | Line Reference |
| --- | --- | --- | --- |
| SIGINT | `InterruptMsg` | Typically from Ctrl+C (when not in raw mode) | [tea.go651-652](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L651-L652) |
| SIGTERM | `QuitMsg` | Sent by `kill` command | [tea.go654](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L654-L654) |
| SIGWINCH | Internal `windowSizeMsg` | Window resize (Unix only) | See resize section |
| SIGTSTP | N/A | Handled by suspend system | See suspension section |

**Sources:** [tea.go624-663](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L624-L663)
 [tea.go636-637](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L636-L637)

### Signal Handling Control

The signal handler can be disabled or signals can be ignored:

The `ignoreSignals` flag is checked at [tea.go649](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L649-L649)
 before processing any signal.

**Sources:** [options.go64-70](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L64-L70)
 [tea.go513](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L513-L513)
 [tea.go649](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L649-L649)

### Signal Handler Lifecycle

The signal handler goroutine:

1.  Starts in [tea.go1002-1004](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1002-L1004)
     unless disabled
2.  Returns a channel tracked in `p.handlers` for graceful shutdown
3.  Stops when `p.ctx` is cancelled or a signal is received
4.  Cleanup handled in [tea.go638-641](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L638-L641)
    

**Sources:** [tea.go624-663](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L624-L663)
 [tea.go1002-1004](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L1002-L1004)

Process Suspension
------------------

### Suspend and Resume Flow

On Unix systems, Bubble Tea supports process suspension (Ctrl+Z functionality):

**Diagram: Process Suspension Sequence**

**Sources:** [tty.go12-22](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L12-L22)
 [tea.go752-755](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L752-L755)

### Suspend Message Types

Bubble Tea provides three message types for suspension control:

**Sources:** [tea.go544-559](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L544-L559)

### Platform Support

Process suspension is only supported on Unix-like systems:

| Platform | Support | Implementation |
| --- | --- | --- |
| Unix (Linux, macOS) | Yes | `suspendProcess()` sends SIGTSTP |
| Windows | No  | `suspendSupported` is false |

The `suspendSupported` constant is defined in platform-specific files and checked at [tea.go753](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L753-L753)

**Sources:** [tea.go12-22](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L12-L22)
 [tea.go752-755](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L752-L755)

### Terminal Release During Suspension

The suspension process at [tty.go12-22](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L12-L22)
 follows this sequence:

1.  Call `releaseTerminal(true)` to restore terminal state
2.  Call platform-specific `suspendProcess()` to send SIGTSTP
3.  Wait for process to resume (SIGCONT)
4.  Call `RestoreTerminal()` to reinitialize terminal
5.  Send `ResumeMsg` to the application

**Sources:** [tty.go12-22](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L12-L22)
