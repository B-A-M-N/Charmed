Purpose and Scope
-----------------

The `cursedRenderer` is the primary rendering implementation in Bubble Tea that manages terminal output using the ultraviolet library for efficient cell-level diffing and ANSI sequence generation. It implements view-based differential rendering, maintains terminal state through the `lastView` cache, and integrates with `uv.TerminalRenderer` for screen buffer management and optimized terminal updates.

For information about the renderer interface it implements, see page 4.1. For terminal state management (raw mode, TTY initialization), see page 4.3. For terminal feature configuration, see page 4.4.

Sources: [cursed\_renderer.go1-831](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L1-L831)

The `cursedRenderer` struct integrates with the ultraviolet library for efficient terminal rendering:

**Structure Overview**: The renderer uses `uv.TerminalRenderer` for ANSI sequence generation and `uv.ScreenBuffer` for cell-level rendering. The `lastView` field caches the previous frame for diff detection, while `view` holds the current pending frame. All shared state is protected by `mu`.

Terminal Feature Configuration
------------------------------

The `flush()` method conditionally configures terminal features when they change between views:

**Feature Configuration in flush()**

**Conditional Feature Updates**

Each feature is configured only when it changes from the previous view:

| Feature | Condition | ANSI Sequences |
| --- | --- | --- |
| **Alt Screen** | `lastView.AltScreen != view.AltScreen` | `SetModeAltScreenSaveCursor` / `ResetModeAltScreenSaveCursor` |

| **Keyboard** | `KeyboardEnhancements` changed or alt screen toggle | `SetModifyOtherKeys2`, `KittyKeyboard()` |

Cursor Visibility Management
----------------------------

Cursor visibility is handled specially to avoid flickering during updates:

**Cursor Visibility Strategy**

**Two Scenarios**:

1.  **With Synchronized Output Support** [cursed\_renderer.go525-552](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L525-L552)
    :
    
    *   If hiding cursor: Add hide sequence BEFORE entering sync mode
    *   Wrap all updates in `SetModeSynchronizedOutput` ... `ResetModeSynchronizedOutput`
    *   If showing cursor: Add show sequence INSIDE sync block (after updates)
    *   Synchronized output ensures atomic rendering without flicker
2.  **Without Synchronized Output** [cursed\_renderer.go535-555](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L535-L555)
    :
    
    *   Hide cursor before updates if needed
    *   Write updates
    *   Show cursor after updates if needed
    *   Best effort to reduce flicker, but terminal-dependent

Sources: [cursed\_renderer.go484-555](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L484-L555)

Optimization Strategies
-----------------------

The `cursedRenderer` employs several optimization techniques for efficient rendering:

**1\. View-Level Diff Detection**

If `viewEquals()` returns true and bounds match, no terminal writes occur.

Sources: [cursed\_renderer.go281-284](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L281-L284)
 [cursed\_renderer.go790-830](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L790-L830)

**2\. Cell-Level Diffing via Ultraviolet**

The ultraviolet library handles low-level diffing, generating only the sequences needed to update changed cells.

Sources: [cursed\_renderer.go458](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L458-L458)

**3\. Synchronized Output Mode**

When `syncdUpdates` is enabled, all updates are wrapped in synchronized output sequences:

| Without Sync | With Sync |
| --- | --- |
| Updates may be visible during rendering | Updates appear atomically |
| Cursor may flicker | No flicker (terminal buffers all changes) |
| Sequences sent as generated | All sequences buffered until sync end |

Sources: [cursed\_renderer.go525-552](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L525-L552)
 [cursed\_renderer.go676-680](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L676-L680)

**4\. Conditional Feature Updates**

Terminal features are only configured when they change:

This avoids redundant ANSI sequences for features that haven't changed.

Sources: [cursed\_renderer.go313-455](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L313-L455)

**5\. Cursor Movement Optimization**

The renderer supports tab stop and backspace optimizations:

| Optimization | Flag | Purpose |
| --- | --- | --- |
| `hardTabs` | `cursedRenderer.hardTabs` | Use tab characters for cursor movement |
| `backspace` | `cursedRenderer.backspace` | Use backspace for leftward movement |
| `mapnl` | `cursedRenderer.mapnl` | Map newline behavior |

These are configured on the underlying `uv.TerminalRenderer`.

Sources: [cursed\_renderer.go59-68](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L59-L68)

**6\. Resize Handling**

When frame area changes, the renderer:

1.  Calls `scr.Erase()` to force full redraw
2.  Clears `cellbuf.Touched` to reset touched lines
3.  Resizes the cell buffer to new dimensions

This prevents artifacts when dimensions change.

Sources: [cursed\_renderer.go289-300](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L289-L300)

Performance Characteristics
---------------------------

| Aspect | Behavior | Impact |
| --- | --- | --- |
| **Framerate** | Configurable 1-120 FPS (default 60) | Lower FPS reduces CPU usage but may appear less smooth |
| **Mutex Locking** | Every `write()`, `flush()`, and feature method locks `mtx` | Ensures thread safety; minimal contention in normal usage |
| **Differential Rendering** | Line-by-line comparison on every flush | Significantly reduces terminal writes for unchanged content |
| **String Comparison** | Full string equality check via `r.buf.String() == r.lastRender` | Early exit when entire frame unchanged |
| **ANSI Compressor** | Optional post-processing of ANSI sequences | Reduces output size but adds processing overhead (deprecated) |
| **Ticker Overhead** | Goroutine wakes on every tick regardless of changes | Minimal; typical Go ticker performance |

**Optimization Trade-offs**:

*   **Higher FPS**: Smoother animations, higher CPU usage, more frequent terminal writes
*   **Lower FPS**: Lower CPU usage, less smooth updates, may feel sluggish
*   **Width Truncation**: Prevents line wrapping issues but requires character width calculation
*   **Line Skipping**: Saves terminal writes but requires slice comparison overhead

Sources: [standard\_renderer.go15-20](https://github.com/charmbracelet/bubbletea/blob/f25595a8/standard_renderer.go#L15-L20)
 [standard\_renderer.go65-83](https://github.com/charmbracelet/bubbletea/blob/f25595a8/standard_renderer.go#L65-L83)

Replaces the standard renderer with `nilRenderer`, which implements all methods as no-ops. Useful for non-TUI applications or testing.
