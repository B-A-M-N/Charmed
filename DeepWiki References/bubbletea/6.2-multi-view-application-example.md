Purpose and Scope
-----------------

This page demonstrates how to build a Bubble Tea application with multiple distinct views that switch based on application state. The example application [examples/views/main.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go)
 showcases state-based view switching, custom progress bar implementation with animations, and advanced styling using lipgloss v2 with color gradients.

For basic single-view examples, see [Simple Examples](https://deepwiki.com/charmbracelet/bubbletea/6.1-simple-examples)
. For async operations with HTTP requests, see [HTTP and Async Operations Tutorial](https://deepwiki.com/charmbracelet/bubbletea/6.3-http-and-async-operations-tutorial)
.

* * *

Overview
--------

The views example implements a two-screen application: a selection menu and a progress screen. The application demonstrates key patterns for building multi-view TUIs:

*   **State-driven view selection**: A boolean field determines which view renders
*   **Separate update functions**: Different logic for each view state
*   **Custom progress bar**: Built from scratch using character cells and gradients
*   **Smooth animations**: Using easing functions with frame-based timing
*   **Advanced styling**: Lipgloss color gradients and style composition

**Sources**: [examples/views/main.go68-113](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L68-L113)

* * *

Model Structure and State Management
------------------------------------

The model uses multiple fields to track application state across both views:

| Field | Type | Purpose |
| --- | --- | --- |
| `Choice` | `int` | Currently selected menu item (0-3) |
| `Chosen` | `bool` | Whether user has made a selection (view switch trigger) |
| `Ticks` | `int` | Countdown timer in seconds |
| `Frames` | `int` | Animation frame counter for progress |
| `Progress` | `float64` | Progress value from 0.0 to 1.0 |
| `Loaded` | `bool` | Whether progress animation has completed |
| `Quitting` | `bool` | Application exit state |

[examples/views/main.go68-76](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L68-L76)

The `Chosen` field is the primary state switch: when `false`, the application displays the choice menu; when `true`, it displays the progress screen. The `model.Update()` function delegates to different update handlers based on this field:

    func (m model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
        // Global quit keys
        if msg, ok := msg.(tea.KeyMsg); ok {
            k := msg.String()
            if k == "q" || k == "esc" || k == "ctrl+c" {
                m.Quitting = true
                return m, tea.Quit
            }
        }
        
        // Delegate to view-specific update function
        if !m.Chosen {
            return updateChoices(msg, m)
        }
        return updateChosen(msg, m)
    }
    

[examples/views/main.go83-99](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L83-L99)

**Sources**: [examples/views/main.go68-99](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L68-L99)

View Switching Pattern
----------------------

The application implements a clean separation between views using dedicated render functions:

**Sources**: [examples/views/main.go83-177](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L83-L177)

### Choice View Update Logic

The `updateChoices()` function handles navigation and selection in the menu:

*   **Navigation**: `j`/`down` and `k`/`up` keys adjust the `Choice` field (0-3 bounds)
*   **Selection**: `enter` key sets `Chosen = true` and returns `frame()` command to start animation
*   **Timeout**: `tickMsg` decrements the countdown; exits when reaches zero

[examples/views/main.go118-147](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L118-L147)

### Progress View Update Logic

The `updateChosen()` function manages the progress animation:

*   **Animation frames**: `frameMsg` triggers progress calculation using easing function
*   **Progress calculation**: `ease.OutBounce(frames/100)` provides bounce animation
*   **Completion**: When `Progress >= 1`, sets `Loaded = true` and starts exit countdown
*   **Exit timer**: `tickMsg` counts down 3 seconds before quitting

[examples/views/main.go150-177](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L150-L177)

**Sources**: [examples/views/main.go118-177](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L118-L177)

Custom Progress Bar Implementation
----------------------------------

The example implements a progress bar from scratch rather than using the bubbles progress component (which didn't exist when this example was created). The implementation demonstrates character-based rendering and color gradients.

### Progress Bar Constants

    progressBarWidth  = 71
    progressFullChar  = "█"
    progressEmptyChar = "░"
    

[examples/views/main.go22-26](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L22-L26)

### Progress Bar Rendering Function

The `progressbar()` function generates the visual progress display:

**Sources**: [examples/views/main.go233-246](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L233-L246)

The function:

1.  Calculates how many cells should be filled based on percentage
2.  Renders filled cells using the gradient color ramp (`ramp[i]`)
3.  Renders empty cells with subtle styling
4.  Appends the numeric percentage

[examples/views/main.go233-246](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L233-L246)

### Color Gradient Generation

The gradient is pre-calculated at startup using the `makeRampStyles()` function:

    ramp = makeRampStyles("#B14FFF", "#00FFA3", progressBarWidth)
    

[examples/views/main.go40](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L40-L40)

This creates an array of 71 lipgloss styles, each with a color that smoothly transitions from purple (`#B14FFF`) to green (`#00FFA3`). The gradient uses the `colorful.BlendLuv()` method for perceptually uniform color interpolation:

    func makeRampStyles(colorA, colorB string, steps float64) (s []lipgloss.Style) {
        cA, _ := colorful.Hex(colorA)
        cB, _ := colorful.Hex(colorB)
        
        for i := 0.0; i < steps; i++ {
            c := cA.BlendLuv(cB, i/steps)
            s = append(s, lipgloss.NewStyle().Foreground(lipgloss.Color(colorToHex(c))))
        }
        return s
    }
    

[examples/views/main.go251-260](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L251-L260)

**Sources**: [examples/views/main.go40-260](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L40-L260)

Animation with Easing Functions
-------------------------------

The progress bar animation uses the `github.com/fogleman/ease` library to create smooth, natural-looking motion:

**Sources**: [examples/views/main.go150-163](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L150-L163)

### Frame Command

The `frame()` command triggers at 60 FPS (every 1/60th of a second):

    func frame() tea.Cmd {
        return tea.Tick(time.Second/60, func(time.Time) tea.Msg {
            return frameMsg{}
        })
    }
    

[examples/views/main.go62-66](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L62-L66)

### Easing Calculation

The `ease.OutBounce()` function creates a bouncing effect at the end of the animation:

    m.Progress = ease.OutBounce(float64(m.Frames) / float64(100))
    

[examples/views/main.go155](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L155-L155)

This converts the linear frame counter (0-100) into a non-linear progress value that bounces as it approaches 1.0, creating a playful visual effect.

**Sources**: [examples/views/main.go62-163](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L62-L163)

Message Flow and Timing
-----------------------

The application uses two custom message types to coordinate timing:

**Sources**: [examples/views/main.go52-177](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L52-L177)

### Tick Command

Used for second-based timing (countdown timers):

    func tick() tea.Cmd {
        return tea.Tick(time.Second, func(time.Time) tea.Msg {
            return tickMsg{}
        })
    }
    

[examples/views/main.go56-60](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L56-L60)

### Frame Command

Used for animation timing at 60 frames per second:

    func frame() tea.Cmd {
        return tea.Tick(time.Second/60, func(time.Time) tea.Msg {
            return frameMsg{}
        })
    }
    

[examples/views/main.go62-66](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L62-L66)

### Timing Coordination

The application uses different timing strategies for different phases:

1.  **Choice menu**: `tick()` provides a 10-second countdown before auto-quit
2.  **Progress animation**: `frame()` updates at 60 FPS for smooth animation
3.  **Completion countdown**: `tick()` provides a 3-second delay before exit

This demonstrates how multiple timing sources can coexist in a single application by using different message types.

**Sources**: [examples/views/main.go52-172](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L52-L172)

Key Patterns Summary
--------------------

The views example demonstrates several important patterns for building complex Bubble Tea applications:

| Pattern | Implementation | Benefit |
| --- | --- | --- |
| **State-driven view selection** | Boolean `Chosen` field controls which view function renders | Clean separation of UI logic |
| **Delegated update functions** | `updateChoices()` and `updateChosen()` handle state-specific logic | Reduced complexity in main `Update()` |
| **Multiple message types** | `tickMsg` for seconds, `frameMsg` for frames | Coordinated timing at different rates |
| **Custom rendering** | Hand-built progress bar with character cells | Fine control over visual appearance |
| **Easing animations** | `ease.OutBounce()` for non-linear progress | Natural, playful motion |
| **Gradient styling** | Pre-computed color ramp array | Smooth color transitions |
| **Global quit handling** | Check for quit keys before delegation | Consistent exit behavior across views |

**Sources**: [examples/views/main.go1-276](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/views/main.go#L1-L276)
