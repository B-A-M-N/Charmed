Mouse Input
===========

Relevant source files

*   [key.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/key.go)
    
*   [mouse.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go)
    

Purpose and Scope
-----------------

This document covers the mouse input system in Bubbletea, including mouse event types, coordinate handling, button codes, and patterns for processing mouse events in the Update function. Mouse input enables applications to respond to clicks, drags, wheel scrolling, and motion events.

For keyboard input handling, see [Keyboard Input](https://deepwiki.com/charmbracelet/bubbletea/3.1-keyboard-input)
. For terminal-specific events like window resize, see [Terminal Events](https://deepwiki.com/charmbracelet/bubbletea/3.3-terminal-events)
. For configuring mouse modes and terminal features, see [Terminal Features and Capabilities](https://deepwiki.com/charmbracelet/bubbletea/4.4-terminal-features-and-capabilities)
.

* * *

Mouse Event Type Hierarchy
--------------------------

Bubbletea provides four distinct mouse event message types, all implementing the `MouseMsg` interface. Each event type represents a different category of mouse interaction.

### Mouse Message Types

**Diagram 1: Mouse Event Type Structure**

**Sources:** [mouse.go44-144](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L44-L144)

| Message Type | Purpose | Typical Use Case |
| --- | --- | --- |
| `MouseClickMsg` | Button press event | Detecting button clicks, initiating drags |
| `MouseReleaseMsg` | Button release event | Completing clicks, ending drags |
| `MouseWheelMsg` | Wheel scroll event | Scrolling content, zooming |
| `MouseMotionMsg` | Cursor movement event | Hover effects, drag operations, motion tracking |

* * *

Mouse Structure
---------------

The `Mouse` struct contains all information about a mouse event. All mouse message types wrap this structure.

### Mouse Fields

**Diagram 2: Mouse Struct Anatomy**

**Sources:** [mouse.go71-75](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L71-L75)

### Coordinate System

The coordinate system is **zero-based** with the origin (0,0) at the **upper left corner** of the terminal:

*   X increases from left to right (column position)
*   Y increases from top to bottom (row position)
*   Maximum X = terminal width - 1
*   Maximum Y = terminal height - 1

### Modifier Keys

The `Mod` field contains keyboard modifiers held during the mouse event, using the same `KeyMod` type as keyboard events. This allows detection of Ctrl+Click, Shift+Click, etc.

**Sources:** [mouse.go71-75](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L71-L75)

Mouse Buttons
-------------

Mouse buttons are represented by the `MouseButton` type, based on the X11 mouse button encoding standard.

### Button Constants

**Diagram 3: Mouse Button Mapping**

**Sources:** [mouse.go9-42](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L9-L42)

| Constant | Value | Description |
| --- | --- | --- |
| `MouseNone` | 0   | No button (motion without button) |
| `MouseLeft` | 1   | Left button |
| `MouseMiddle` | 2   | Middle button (scroll wheel press) |
| `MouseRight` | 3   | Right button |
| `MouseWheelUp` | 4   | Scroll wheel up |
| `MouseWheelDown` | 5   | Scroll wheel down |
| `MouseWheelLeft` | 6   | Scroll wheel left (horizontal scroll) |
| `MouseWheelRight` | 7   | Scroll wheel right (horizontal scroll) |
| `MouseBackward` | 8   | Browser backward button |
| `MouseForward` | 9   | Browser forward button |
| `MouseButton10` | 10  | Additional button 10 |
| `MouseButton11` | 11  | Additional button 11 |

**Note:** The button encoding follows the X11 standard. Buttons beyond 11 are not supported.

**Sources:** [mouse.go9-42](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L9-L42)

Mouse Event String Representation
---------------------------------

All mouse message types implement `String()` for debugging and logging purposes.

**Diagram 5: String Formatting Logic**

**Sources:** [mouse.go77-144](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L77-L144)

The `String()` method produces human-readable representations:

*   Click/Release/Wheel events delegate to the ultraviolet library's string formatter
*   Motion events append "+motion" if a button is held, or just "motion" otherwise
*   The underlying `uv.Mouse(m).String()` includes coordinate and button information

**Sources:** [mouse.go77-144](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L77-L144)

Enabling Mouse Input
--------------------

Mouse input is not enabled by default. It must be configured through the rendering View's mouse mode setting.

### Mouse Modes

| Mode | Description | Events Generated |
| --- | --- | --- |
| Not enabled | Default, no mouse tracking | None |
| Button mode | Track button press/release | `MouseClickMsg`, `MouseReleaseMsg` |
| Drag mode | Track button + motion while pressed | Click, Release, `MouseMotionMsg` (with button) |
| Motion mode | Track all motion | Click, Release, `MouseMotionMsg` (all) |
| All modes | Combined tracking | `MouseClickMsg`, `MouseReleaseMsg`, `MouseWheelMsg`, `MouseMotionMsg` |

### Configuration

Mouse mode is set in the `View` struct returned by the `View()` method:

Refer to [Terminal Features and Capabilities](https://deepwiki.com/charmbracelet/bubbletea/4.4-terminal-features-and-capabilities)
 for detailed information about mouse mode configuration and terminal support.

Implementation Details
----------------------

### Type Casting and Interface Satisfaction

All mouse message types are type aliases of `Mouse` that satisfy the `MouseMsg` interface:

**Diagram 6: Type Relationships**

**Sources:** [mouse.go44-144](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L44-L144)

Each type alias:

1.  Wraps the `Mouse` struct
2.  Implements `String() string` [mouse.go86-137](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L86-L137)
    
3.  Implements `Mouse() Mouse` to satisfy the `MouseMsg` interface [mouse.go90-144](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L90-L144)
    

### Dependency on Ultraviolet

The mouse system delegates to the `charmbracelet/ultraviolet` package for low-level functionality:

*   `MouseButton` type is imported from ultraviolet [mouse.go10](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L10-L10)
    
*   All button constants are aliases to ultraviolet constants [mouse.go29-42](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L29-L42)
    
*   String formatting uses `uv.Mouse(m).String()` [mouse.go79](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L79-L79)
    

This design allows Bubbletea to leverage ultraviolet's ANSI parsing capabilities while providing a clean API surface.

**Sources:** [mouse.go1-145](https://github.com/charmbracelet/bubbletea/blob/f25595a8/mouse.go#L1-L145)

Platform Support
----------------

Mouse input works across Unix-like systems (Linux, macOS) and Windows. The platform abstraction layer handles terminal-specific mouse tracking sequences.

### Cross-Platform Considerations

*   **Coordinate System:** Consistent across platforms (zero-based, upper-left origin)
*   **Button Codes:** Follow X11 standard on all platforms
*   **ANSI Parsing:** Handled by ultraviolet's `TerminalReader`
*   **Terminal Features:** Mouse mode configuration is platform-independent

For platform-specific terminal initialization details, see [Cross-platform Input](https://deepwiki.com/charmbracelet/bubbletea/3.4-cross-platform-input)
 and [Platform-Specific Features](https://deepwiki.com/charmbracelet/bubbletea/5.6-platform-specific-features)
.

**Sources:** Based on high-level platform abstraction diagram and architecture overview

* * *

Summary
-------

The mouse input system in Bubbletea provides:

*   Four message types for different mouse interactions (`MouseClickMsg`, `MouseReleaseMsg`, `MouseWheelMsg`, `MouseMotionMsg`)
*   A unified `Mouse` struct with coordinates, button information, and modifiers
*   Standard X11 button encoding supporting up to 11 buttons plus wheel actions
*   Zero-based coordinate system with (0,0) at the terminal's upper-left corner
*   Flexible event handling patterns in the Update function
*   Cross-platform support through the ultraviolet library

Mouse events flow from terminal ANSI sequences through the input processing pipeline to the Update function, where applications can handle clicks, drags, scrolls, and motion with type-safe message matching.
