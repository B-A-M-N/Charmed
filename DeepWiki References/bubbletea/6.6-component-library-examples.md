This page demonstrates integration patterns with the bubbles/v2 component library. It covers how to embed Bubble Tea components in your Model, delegate messages, manage focus across multiple components, and handle cursor positioning. For basic program structure, see [Simple Examples](https://deepwiki.com/charmbracelet/bubbletea/6.1-simple-examples)
. For command patterns, see [Command Orchestration Examples](https://deepwiki.com/charmbracelet/bubbletea/6.4-command-orchestration-examples)
.

Overview of Component Integration
---------------------------------

Bubble Tea components from the bubbles library implement a pattern where they:

1.  Expose a `Model` struct that is embedded in your application's model
2.  Provide an `Update(tea.Msg) (Model, tea.Cmd)` method for message delegation
3.  Provide a `View() string` method for rendering
4.  May return initialization commands from helper functions like `Blink` or `Tick`

Pager Example: Viewport Component
---------------------------------

The pager example demonstrates the `viewport.Model` component, which provides scrollable content viewing with features like line numbers, search highlighting, and scroll tracking.

### Model Structure and Lazy Initialization

The viewport component uses a lazy initialization pattern to wait for terminal dimensions:

| Field | Type | Purpose |
| --- | --- | --- |
| `content` | `string` | Content to display (loaded from file) |
| `ready` | `bool` | Flag indicating viewport is initialized |
| `viewport` | `viewport.Model` | The bubbles viewport component |

The initialization happens in the `Update` method when receiving `WindowSizeMsg`:

**Sources:** [examples/pager/main.go31-35](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/pager/main.go#L31-L35)
 [examples/pager/main.go53-85](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/pager/main.go#L53-L85)

The viewport is configured with:

     that displays line numbers or `~` for lines beyond content

Spinner Example: Tick Command Pattern
-------------------------------------

The spinner example demonstrates the simplest component integration pattern with time-based animation.

### Component Lifecycle

The spinner requires returning its `Tick` command from `Init()` to start the animation loop:

| Method | Implementation | Purpose |
| --- | --- | --- |
| `Init()` | `return m.spinner.Tick` | Starts the ticker for animation frames |
| `Update()` | `m.spinner, cmd = m.spinner.Update(msg)` | Delegates all messages to spinner |
| `View()` | `m.spinner.View()` | Gets the current animation frame |

**Sources:** [examples/spinner/main.go23-65](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/spinner/main.go#L23-L65)

### Spinner Configuration

The spinner is configured with visual properties before use:

*   **Spinner type**: `s.Spinner = spinner.Dot` selects the animation style [examples/spinner/main.go25](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/spinner/main.go#L25-L25)
    
*   **Style**: `s.Style = lipgloss.NewStyle().Foreground(...)` sets the color [examples/spinner/main.go26](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/spinner/main.go#L26-L26)
    

The spinner internally manages its animation state and returns commands to schedule the next frame update.

**Sources:** [examples/spinner/main.go24-27](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/spinner/main.go#L24-L27)

Text Input Examples
-------------------

The text input examples demonstrate focus management, cursor positioning, and batch command patterns.

### Single Text Input Pattern

The simplest text input integration follows this structure:

**Key pattern**: The cursor position is retrieved from the text input and adjusted for the layout:

*   Get cursor: `c = m.textInput.Cursor()` [examples/textinput/main.go65](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinput/main.go#L65-L65)
    
*   Adjust Y position: `c.Y += lipgloss.Height(m.headerView())` [examples/textinput/main.go66](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinput/main.go#L66-L66)
    
*   Set on view: `v.Cursor = c` [examples/textinput/main.go75](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinput/main.go#L75-L75)
    

The `VirtualCursor()` method determines whether the cursor is managed by the terminal or drawn as part of the content.

**Sources:** [examples/textinput/main.go25-81](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinput/main.go#L25-L81)

### Multiple Text Inputs with Focus Management

The multiple text inputs example demonstrates focus management across components:

**Sources:** [examples/textinputs/main.go29-189](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L29-L189)

### Focus Management Implementation

The focus management pattern involves:

1.  **Focus index tracking**: `focusIndex int` field tracks which component (0-2 for inputs, 3 for submit button)
2.  **Navigation logic**: Tab/arrow keys increment/decrement with wrapping [examples/textinputs/main.go100-120](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L100-L120)
    
3.  **Focus application**: Loop through inputs, call `Focus()` on target, `Blur()` on others [examples/textinputs/main.go122-132](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L122-L132)
    
4.  **Batch commands**: Collect focus commands from all inputs [examples/textinputs/main.go122-133](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L122-L133)
    

| Key | Action | Implementation |
| --- | --- | --- |
| `tab`, `down`, `enter` | Next input | `m.focusIndex++` |
| `shift+tab`, `up` | Previous input | `m.focusIndex--` |
| `enter` on submit | Exit | Check `m.focusIndex == len(m.inputs)` |
| `ctrl+r` | Change cursor mode | Cycle through `cursor.Mode` values |

**Sources:** [examples/textinputs/main.go100-133](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L100-L133)

### Input Delegation and Cursor Positioning

The update delegation follows this pattern:

Only the focused text input responds to character input messages, but all inputs can respond to their internal commands (like cursor blink).

Cursor positioning accounts for multiple lines:

*   Loop through inputs rendering each on a separate line [examples/textinputs/main.go159-163](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L159-L163)
    
*   When input is focused, get its cursor: `c = in.Cursor()` [examples/textinputs/main.go165](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L165-L165)
    
*   Adjust Y coordinate: `c.Y += i` to account for line number [examples/textinputs/main.go167](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L167-L167)
    
*   Set on view: `v.Cursor = c` [examples/textinputs/main.go187](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinputs/main.go#L187-L187)

### Best Practices

When integrating bubbles components:

1.  **Always delegate messages**: Even if your app doesn't handle a message, pass it to the component
2.  **Return component commands**: Capture and return commands from component `Update()` calls
3.  **Initialize with component commands**: Return `Tick`, `Blink`, or other initialization commands from your `Init()`
4.  **Adjust cursor positions**: If components return cursors, adjust Y coordinate for your layout
5.  **Batch multiple commands**: Use `tea.Batch()` when multiple components return commands
6.  **Handle focus carefully**: Only one input should be focused; explicitly manage focus transitions
7.  **Lazy initialization**: For size-dependent components like viewport, wait for `WindowSizeMsg`
