Purpose and Scope
-----------------

This document covers the terminal state management layer of Bubble Tea, which handles platform-specific terminal initialization, raw mode configuration, virtual terminal processing, and state restoration. This layer provides a unified abstraction over Unix-like systems (Linux, macOS, BSD) and Windows, ensuring that the framework can properly control terminal behavior across platforms.

For information about terminal feature configuration (alternate screen, mouse modes, keyboard enhancements), see [Terminal Features and Capabilities](https://deepwiki.com/charmbracelet/bubbletea/4.4-terminal-features-and-capabilities)
. For the high-level rendering process, see [Cursed Renderer Implementation](https://deepwiki.com/charmbracelet/bubbletea/4.2-cursed-renderer-implementation)
.

Terminal Initialization
-----------------------

The `Program.initInput()` method initializes the terminal for raw input processing. This is called during program startup to configure the terminal for interactive TUI operation.

### Unix Initialization

On Unix-like systems, initialization involves setting the terminal to raw mode using `term.MakeRaw()`:

The raw mode configuration disables:

*   **Canonical mode** (`ICANON`): Input is available immediately, not line-buffered
*   **Echo** (`ECHO`): Input characters are not echoed to the terminal
*   **Signal generation** (`ISIG`): Special characters like Ctrl+C don't generate signals

**Key Code Locations:**

*   [tty\_unix.go15-35](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L15-L35)
     - `initInput()` implementation

     - `term.MakeRaw()` call to enter raw mode
*   [tty\_unix.go27](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L27-L27)

**Sources:** [tty\_unix.go15-35](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L15-L35)

### Windows Initialization

Windows requires enabling Virtual Terminal (VT) processing to support ANSI escape sequences:

The Windows-specific flags:

*   **`ENABLE_VIRTUAL_TERMINAL_INPUT`**: Enables processing of ANSI input escape sequences (key press/release, mouse events)
*   **`ENABLE_VIRTUAL_TERMINAL_PROCESSING`**: Enables ANSI output escape sequences (colors, cursor movement)
*   **`DISABLE_NEWLINE_AUTO_RETURN`**: Prevents automatic carriage return on newline

**Key Code Locations:**

### Cursor Movement Optimization Detection

Both platforms call `checkOptimizedMovements()` to detect whether the terminal supports optimized cursor movements using hard tabs and backspace characters:

| Optimization | Unix Detection | Windows Detection | Usage |
| --- | --- | --- | --- |
| Hard Tabs | Checks `termios` `OXTABS` flag | TODO comment indicates not yet implemented | Allows using `\t` for horizontal cursor movement |
| Backspace | Checks `termios` `ECHOE` flag | TODO comment indicates not yet implemented | Allows using `\b` for backward cursor movement |
| Map Newline | Checks `termios` `ONLCR` flag | Always disabled on Windows | Maps `\n` to `\r\n` |

These optimizations are applied to the renderer via `cursedRenderer.setOptimizations()` at [cursed\_renderer.go59-68](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L59-L68)

Process Suspension and Resume (Unix Only)
-----------------------------------------

Unix-like systems support suspending the program with `Ctrl+Z` (SIGTSTP) and resuming with `fg` (SIGCONT). Bubble Tea handles this gracefully by cleaning up the terminal on suspension and restoring state on resume.

### Suspension Flow

**Implementation Details:**

The `suspendProcess()` function at [tty\_unix.go40-46](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L40-L46)
 performs these steps:

1.  Creates a channel to listen for SIGCONT
2.  Sends SIGTSTP to the entire process group (`syscall.Kill(0, SIGTSTP)`)
3.  Blocks until SIGCONT is received
4.  Returns to allow program execution to continue

The `const suspendSupported = true` at [tty\_unix.go37](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L37-L37)
 enables suspension support, while Windows defines this as `false` at [tty\_windows.go62](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L62-L62)
 with a no-op implementation at [tty\_windows.go64](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L64-L64)

**Sources:** [tty\_unix.go37-46](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L37-L46)
 [tty\_windows.go62-64](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L62-L64)

Terminal Resize Handling
------------------------

Terminal resize events are handled differently across platforms due to operating system capabilities.

### Unix: SIGWINCH Signal

Unix systems use the `SIGWINCH` signal to notify the program when the terminal size changes:

The `listenForResize()` goroutine at [signals\_unix.go15-33](https://github.com/charmbracelet/bubbletea/blob/f25595a8/signals_unix.go#L15-L33)
:

1.  Creates a channel for SIGWINCH signals: [16](https://github.com/charmbracelet/bubbletea/blob/f25595a8/16)
    
2.  Registers for signal notifications: [17](https://github.com/charmbracelet/bubbletea/blob/f25595a8/17)
    
3.  Loops waiting for signals or context cancellation: [24-32](https://github.com/charmbracelet/bubbletea/blob/f25595a8/24-32)
    
4.  Calls `p.checkResize()` when SIGWINCH is received: [31](https://github.com/charmbracelet/bubbletea/blob/f25595a8/31)
    

**Sources:** [signals\_unix.go15-33](https://github.com/charmbracelet/bubbletea/blob/f25595a8/signals_unix.go#L15-L33)

### Windows: No Resize Support

Windows does not support the SIGWINCH signal. The `listenForResize()` implementation at [signals\_windows.go8-10](https://github.com/charmbracelet/bubbletea/blob/f25595a8/signals_windows.go#L8-L10)
 immediately closes the done channel without listening for any events.

| Feature | Unix | Windows |
| --- | --- | --- |
| Signal | `SIGWINCH` | None |
| Implementation | Active listener goroutine | Immediate no-op |
| WindowSizeMsg | Sent on resize | Only sent on startup |
| Runtime Detection | Automatic | Manual polling required |

**Note:** This is documented in `WindowSizeMsg` at [screen.go6-8](https://github.com/charmbracelet/bubbletea/blob/f25595a8/screen.go#L6-L8)
: "Note that Windows does not have support for reporting when resizes occur as it does not support the SIGWINCH signal."

**Sources:** [signals\_windows.go8-10](https://github.com/charmbracelet/bubbletea/blob/f25595a8/signals_windows.go#L8-L10)
 [screen.go6-12](https://github.com/charmbracelet/bubbletea/blob/f25595a8/screen.go#L6-L12)

Error Handling
--------------

Terminal management operations can fail in several ways. The framework handles these scenarios:

### Initialization Errors

Both `tty_unix.go` and `tty_windows.go` wrap errors with context:

    "error entering raw mode: %w"          // Unix: term.MakeRaw failure
    "error getting console mode: %w"        // Windows: GetConsoleMode failure  
    "error setting console mode: %w"        // Windows: SetConsoleMode failure
    "error getting terminal state: %w"      // Windows: term.GetState failure
    

These errors propagate up to `Program.Run()` and cause program termination with the error returned to the caller.

### Cleanup Errors

The `cursedRenderer.close()` method can fail when flushing or writing to output:

    "bubbletea: error closing screen writer: %w"    // <FileRef file-url="https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L222-L222" min=222  file-path="cursed_renderer.go">Hii</FileRef>

These errors are returned from `close()` and typically cause the program to exit with an error.

Best Practices
--------------

When working with terminal management in Bubble Tea:

1.  **Never skip `defer` cleanup**: Always let the framework handle terminal restoration through normal program shutdown
2.  **Test on both platforms**: Terminal behavior differs significantly between Unix and Windows
3.  **Handle missing features gracefully**: Check `suspendSupported` constant and handle resize events conditionally
4.  **Preserve cursor position**: The framework preserves cursor position during suspension at [cursed\_renderer.go235-242](https://github.com/charmbracelet/bubbletea/blob/f25595a8/cursed_renderer.go#L235-L242)
    
5.  **Avoid direct terminal manipulation**: Let the framework manage terminal state to prevent conflicts
