=============================

This document covers the installation process for Bubble Tea, the structure of Go modules in the repository, and the dependency ecosystem. It explains the direct and transitive dependencies required by the framework, their purposes, and how the repository uses `replace` directives for local development.

For information about setting up a local development environment, see [Development Setup](https://deepwiki.com/charmbracelet/bubbletea/7.1-development-setup)
. For details about integrating with the Bubbles component library, see [Component Library Examples](https://deepwiki.com/charmbracelet/bubbletea/6.6-component-library-examples)
.

Installation
------------

### Installing Bubble Tea

To install Bubble Tea in your Go project, use the standard `go get` command:

This installs the latest version of Bubble Tea v2. The framework requires Go 1.24.2 or later as specified in [go.mod5](https://github.com/charmbracelet/bubbletea/blob/f25595a8/go.mod#L5-L5)

Module Structure
----------------

The Bubble Tea repository contains three separate Go modules with different purposes:

| Module | Module Path | Replace Directive | Purpose |
| --- | --- | --- | --- |
| Main Framework | `charm.land/bubbletea/v2` | None | Core Bubble Tea framework |
| Examples | `examples` | `=> ../` | Demonstration applications |
| Tutorials | `tutorial` | `=> ../` | Step-by-step learning materials |

### Module Relationships

**Replace Directives for Local Development**

Both examples and tutorials use `replace` directives to reference the local Bubble Tea source:

*   Examples: `replace charm.land/bubbletea/v2 => ../` [examples/go.mod5](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L5-L5)
    
*   Tutorials: `replace charm.land/bubbletea/v2 => ../` [tutorials/go.mod29](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/go.mod#L29-L29)
    

This allows developers to modify the framework and immediately test changes in examples without publishing a new version.

**Sources:** [go.mod1](https://github.com/charmbracelet/bubbletea/blob/f25595a8/go.mod#L1-L1)
 [examples/go.mod1-5](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L1-L5)

Using Bubble Tea in Your Project
--------------------------------

### Basic Project Setup

Create a new Go module and add Bubble Tea:

### Minimal go.mod

After installation, your `go.mod` will include:

    module github.com/username/my-tui-app
    
    go 1.24
    
    require charm.land/bubbletea/v2 v2.0.0-rc.1
    

All transitive dependencies are automatically managed. The `go.sum` file will contain checksums for all direct and indirect dependencies.

Version Constraints and Retractions
-----------------------------------

### Retracted Versions

The module retracts version `v2.0.0-beta1` due to incorrect version numbering:

    retract v2.0.0-beta1 // We add a "." after the "beta" in the version number.
    

This version should not be used. Use `v2.0.0-beta.1` or later instead.

### Version Compatibility

Bubble Tea v2 is in beta/release candidate phase. The examples and tutorials use pre-release versions:

*   Examples: `v2.0.0-rc.1.0.20251106192006` [examples/go.mod9](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L9-L9)
    
*   Tutorials: Uses local source via replace directive [tutorials/go.mod29](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/go.mod#L29-L29)

Dependency Management Patterns
------------------------------

### Replace Directives for Development

The repository uses `replace` directives in subdirectories to reference the local framework:

    examples/go.mod:
      replace charm.land/bubbletea/v2 => ../
    
    tutorials/go.mod:
      replace charm.land/bubbletea/v2 => ../
    

This pattern enables:

*   Testing framework changes immediately in examples
*   Ensuring examples always use the current codebase
*   Avoiding version management during development

### Pseudo-Versions in Replace Directives

The tutorials module requires a pseudo-version that is resolved by the replace directive:

    require charm.land/bubbletea/v2 v2.0.0-00010101000000-000000000000
    

This zero-timestamp pseudo-version indicates the dependency is satisfied by the replace directive [tutorials/go.mod7](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/go.mod#L7-L7)

**Sources:** [examples/go.mod5](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L5-L5)
