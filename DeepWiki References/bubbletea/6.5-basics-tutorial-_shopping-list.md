Purpose and Scope
-----------------

This page walks through the **Basics Tutorial** located in [tutorials/basics/](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/)
 which introduces the fundamental Model-Update-View (MVU) pattern in Bubble Tea through a grocery list application. The tutorial demonstrates keyboard navigation, checkbox selection, and the complete program lifecycle without performing any I/O operations.

For information about performing I/O operations (HTTP requests, timers, file operations), see the [HTTP and Async Operations Tutorial](https://deepwiki.com/charmbracelet/bubbletea/6.3-http-and-async-operations-tutorial)
 and [Command System](https://deepwiki.com/charmbracelet/bubbletea/2.4-command-system)
.

**Sources:** [tutorials/basics/README.md1-14](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/README.md?plain=1#L1-L14)

Tutorial Overview
-----------------

The Basics Tutorial builds a simple **grocery list application** where users can:

*   Navigate through items using arrow keys or vi-style keys (j/k)
*   Select/deselect items with Enter or Space
*   Quit the program with q or Ctrl+C

The application consists of approximately 87 lines of code and demonstrates the three core methods required by the `tea.Model` interface: `Init()`, `Update()`, and `View()`.

**Application Requirements:**

| Feature | Implementation |
| --- | --- |
| Display list of items | Iterate over `choices` slice in View() |
| Show cursor position | Track with `cursor` int field |
| Track selections | Use `selected` map as a set |
| Handle keyboard input | Process `tea.KeyMsg` in Update() |
| Render UI | Build string in View() |

**Sources:** [tutorials/basics/main.go1-86](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L1-L86)
 [tutorials/basics/README.md16-32](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/README.md?plain=1#L16-L32)

The Model Structure
-------------------

The application state is encapsulated in a `model` struct with three fields:

    Model State Diagram
    

### Field Descriptions

| Field | Type | Purpose |
| --- | --- | --- |
| `cursor` | `int` | Zero-based index indicating which item the cursor points to |
| `choices` | `[]string` | Slice containing the grocery items to display |
| `selected` | `map[int]struct{}` | Map used as a mathematical set; keys are indices of selected items |

The `selected` map uses `struct{}` as the value type because it requires zero memory allocation. The presence of a key in the map indicates that item is selected.

The Update Method
-----------------

The `Update()` method is the heart of the application. It receives messages (events) and returns an updated model and an optional command.

### Message Flow Architecture

    Update Method Message Processing
    

### Keyboard Input Handling

The `Update()` method handles `tea.KeyMsg` messages with the following key bindings:

| Key(s) | Action | Implementation |
| --- | --- | --- |
| `ctrl+c`, `q` | Quit program | Return `tea.Quit` command |
| `up`, `k` | Move cursor up | Decrement `m.cursor` if > 0 |
| `down`, `j` | Move cursor down | Increment `m.cursor` if < len(choices)-1 |
| `enter`, `space` | Toggle selection | Add/remove from `m.selected` map |

The View Method
---------------

The `View()` method constructs the UI as a single string by building it incrementally using string concatenation.

### View Rendering Process

    View Method Rendering Pipeline
    

### UI Output Format

The rendered output has this structure:

    ┌─────────────────────────────────────────┐
    │ What should we buy at the market?       │
    │                                         │
    │ > [x] Buy carrots                       │ ← cursor + selected
    │   [ ] Buy celery                        │
    │   [ ] Buy kohlrabi                      │
    │                                         │
    │ Press q to quit.                        │
    └─────────────────────────────────────────┘
    

Each line is formatted using `fmt.Sprintf()` with three components:

1.  Cursor indicator: `">"` or `" "`
2.  Checkbox: `"[x]"` or `"[ ]"`
3.  Item text: the choice string

**Visual Indicator Logic:**

| Condition | Cursor | Checkbox | Example |
| --- | --- | --- | --- |
| cursor == i, selected | `>` | `[x]` | `> [x] Buy carrots` |
| cursor == i, not selected | `>` | `[ ]` | `> [ ] Buy celery` |
| cursor != i, selected | (space) | `[x]` | `[x] Buy kohlrabi` |
| cursor != i, not selected | (space) | `[ ]` | `[ ] Buy carrots` |

Complete Program Lifecycle
--------------------------

This diagram shows how all components work together in the basics tutorial:

    Complete Basics Tutorial Architecture
    

### Key Interactions

1.  **Initialization**: `main()` creates the initial model and passes it to `tea.NewProgram()`
2.  **Startup**: `Program.Run()` calls `model.Init()` to get the initial command
3.  **Event Loop**: Input events are converted to messages and sent to `Update()`
4.  **State Updates**: `Update()` modifies the model and returns it
5.  **Rendering**: After each update, `View()` is called and output is sent to the renderer
6.  **Termination**: When `Update()` returns `tea.Quit`, the event loop exits and `Run()` returns

**Sources:** [tutorials/basics/main.go1-86](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L1-L86)

Key Concepts Summary
--------------------

### The MVU Pattern in Practice

| Component | Purpose | Tutorial Implementation |
| --- | --- | --- |
| **Model** | Application state | `model` struct with cursor, choices, selected |
| **Init** | Initial command | Returns `SetWindowTitle` command |
| **Update** | Event handler | Handles keyboard input, updates state |
| **View** | UI renderer | Builds string representation of grocery list |

### Message Types Used

This tutorial only uses one message type:

*   **`tea.KeyMsg`**: Delivered when keys are pressed, contains the key information accessible via `msg.String()`

The framework automatically creates and delivers `tea.KeyMsg` messages when keyboard input is detected. No custom message types are needed for this basic tutorial.

### Command Usage

The tutorial uses two built-in commands:

| Command | Where Used | Purpose |
| --- | --- | --- |
| `tea.SetWindowTitle` | `Init()` | Sets terminal window title to "Grocery List" |
| `tea.Quit` | `Update()` | Signals the program to exit gracefully |

Both commands are created by calling the respective functions from the `tea` package. They return `tea.Msg` values that the framework processes internally.
