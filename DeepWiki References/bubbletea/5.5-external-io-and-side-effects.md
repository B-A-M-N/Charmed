Purpose and Scope
-----------------

This document explains how to run external commands and perform blocking I/O operations within a Bubble Tea program. It covers the `ExecProcess` and `Exec` functions, which allow spawning interactive applications like editors, shells, or other terminal-based programs while temporarily pausing the Bubble Tea runtime.

For non-blocking asynchronous operations, see [Command System](https://deepwiki.com/charmbracelet/bubbletea/2.4-command-system)
. For general concurrency patterns, see [Concurrency and Goroutines](https://deepwiki.com/charmbracelet/bubbletea/5.1-concurrency-and-goroutines)
.

* * *

Overview
--------

Bubble Tea provides mechanisms to execute external commands in a blocking fashion, effectively pausing the program's event loop while the external process runs. This is essential for launching interactive applications that need direct terminal control, such as text editors (`vim`, `nano`) or shells.

The framework handles:

*   **Terminal state release**: Raw mode is disabled and terminal state is saved
*   **I/O redirection**: stdin/stdout/stderr are properly connected to the external process
*   **Terminal restoration**: After execution completes, raw mode is re-enabled and the program resumes

**Key Components:**

| Component | Purpose |
| --- | --- |
| `ExecProcess` | High-level function to execute an `*exec.Cmd` |
| `Exec` | Low-level function accepting any `ExecCommand` implementation |
| `ExecCommand` | Interface for blocking command execution |
| `ExecCallback` | Callback type returning a message with execution results |
| `execMsg` | Internal message type triggering execution |

The ExecCommand Interface
-------------------------

The `ExecCommand` interface defines the contract for any command that needs blocking terminal execution.

**Interface Methods:**

| Method | Description |
| --- | --- |
| `Run() error` | Execute the command and return any error |
| `SetStdin(io.Reader)` | Configure the input stream for the command |
| `SetStdout(io.Writer)` | Configure the output stream for the command |
| `SetStderr(io.Writer)` | Configure the error stream for the command |

The framework provides `osExecCommand` [exec.go73-99](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L73-L99)
 a wrapper around `*exec.Cmd` that implements this interface.

ExecProcess: Running External Commands
--------------------------------------

`ExecProcess` is the primary function for executing external commands. It wraps an `*exec.Cmd` and handles all terminal state management automatically.

**Function Signature:**

**Parameters:**

*   `c *exec.Cmd`: The command to execute (from `os/exec` package)
*   `fn ExecCallback`: Optional callback invoked with execution results (can be `nil`)

**Example: Opening Vim Editor**

**Callback Behavior:**

*   If `fn` is `nil`, no message is sent after execution completes
*   If `fn` is provided, it receives any error from execution (or `nil` on success)
*   The callback message is delivered to `Update` via the normal message flow

Internal Execution Implementation
---------------------------------

The `Program.exec` method [exec.go102-129](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L102-L129)
 orchestrates the entire execution process:

**Execution Steps:**

1.  **Terminal Release** [exec.go103-109](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L103-L109)
    : Attempts to release terminal control
    
    *   If release fails, abort and optionally send error via callback
    *   Returns early to prevent corruption of terminal state
2.  **I/O Configuration** [exec.go111-113](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L111-L113)
    :
    
    *   `SetStdin`: Defaults to program's input (usually stdin)
    *   `SetStdout`: Defaults to program's output (usually stdout)
    *   `SetStderr`: Always uses `os.Stderr`
3.  **Command Execution** [exec.go116](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L116-L116)
    : Calls `c.Run()` which blocks until complete
    
4.  **Error Handling** [exec.go116-122](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L116-L122)
    :
    
    *   If execution fails, attempts terminal restoration
    *   Sends error via callback if provided
    *   Returns early on error
5.  **Terminal Restoration** [exec.go125-128](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L125-L128)
    :
    
    *   Re-enables raw mode and restarts input reader
    *   Sends final result via callback (error or `nil`)

Sources: [exec.go102-129](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L102-L129)

Comparison: Exec vs Regular Commands
------------------------------------

| Aspect | ExecProcess/Exec | Regular Cmd (tea.Cmd) |
| --- | --- | --- |
| **Execution** | Blocking (pauses event loop) | Non-blocking (runs in goroutine) |
| **Terminal Control** | External process gets terminal | Program maintains terminal |
| **Use Case** | Interactive programs (editors, shells) | Background operations (HTTP, timers) |
| **Message Delivery** | After completion only | Anytime during execution |
| **I/O Streams** | Direct terminal access | Captured/redirected |
| **Raw Mode** | Disabled during execution | Remains enabled |

**When to Use ExecProcess:**

*   Launching text editors (vim, nano, emacs)
*   Starting interactive shells
*   Running terminal-based tools that need full terminal control (less, more, top)
*   Any program that requires raw terminal input/output

**When to Use Regular Cmd:**

*   HTTP requests
*   File I/O operations
*   Background computations
*   Database queries
*   Any non-interactive operation

Platform Considerations
-----------------------

External command execution works across Unix-like systems and Windows, but behavior may differ:

**Unix/Linux/macOS:**

*   Full support for terminal state management
*   Shells: `$SHELL`, `/bin/sh`, `/bin/bash`
*   Editors: `$EDITOR`, `vim`, `nano`, `emacs`

**Windows:**

*   Virtual terminal processing enabled via [tty\_windows.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go)
    
*   Shell: `cmd.exe`, `powershell.exe`
*   Limited signal support (see [Platform-Specific Features](https://deepwiki.com/charmbracelet/bubbletea/5.6-platform-specific-features)
    )

**Terminal Release** [exec.go103](https://github.com/charmbracelet/bubbletea/blob/f25595a8/exec.go#L103-L103)
: The `releaseTerminal(false)` call uses platform-specific implementations to properly disable raw mode and restore terminal state before external execution.

Summary
-------

External I/O and side effects in Bubble Tea are handled through:

1.  **ExecProcess**: High-level function for running `*exec.Cmd` with automatic terminal management
2.  **Exec**: Low-level function accepting custom `ExecCommand` implementations
3.  **Terminal State Management**: Automatic release and restoration of terminal control
4.  **Blocking Execution**: Event loop pauses during external process execution
5.  **Callback Pattern**: Results delivered as messages to `Update` after completion

This architecture enables seamless integration of interactive external programs while maintaining Bubble Tea's message-driven architecture and ensuring proper terminal state throughout the program lifecycle.
