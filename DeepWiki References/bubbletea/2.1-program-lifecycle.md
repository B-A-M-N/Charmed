This page documents the complete lifecycle of a `tea.Program` instance, from construction through initialization, event loop execution, and termination. It covers how to create a program with `NewProgram`, start it with `Run`, the phases of program execution, and the various ways to terminate a program.

For information about the Model interface that programs execute, see [Model Interface](https://deepwiki.com/charmbracelet/bubbletea/2.2-model-interface)
. For details about the command execution system, see [Command System](https://deepwiki.com/charmbracelet/bubbletea/2.4-command-system)
. For program configuration options, see [Program Options](https://deepwiki.com/charmbracelet/bubbletea/2.5-program-options)
.

* * *

Overview
--------

A Bubble Tea program follows a well-defined lifecycle with four distinct phases:

1.  **Construction**: Creating a `Program` instance with `NewProgram`
2.  **Initialization**: Setting up terminal state and executing initial commands
3.  **Event Loop**: Processing messages, updating state, and rendering
4.  **Termination**: Cleaning up and restoring terminal state

The lifecycle is managed by the `Program` struct [tea.go135-198](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L135-L198)
 which serves as the central orchestrator for all framework operations.

**Sources**: [tea.go135-198](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L135-L198)
 [tea.go240-271](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L240-L271)
 [tea.go578-746](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L578-L746)

Program Construction
--------------------

### Creating a Program

Programs are created using the `NewProgram` function [tea.go240-271](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L240-L271)
 which accepts a `Model` implementation and optional `ProgramOption` functions:

During construction, `NewProgram` performs the following steps:

| Step | Action | Code Reference |
| --- | --- | --- |
| 1   | Initialize `Program` struct with initial model | [tea.go242-245](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L242-L245) |
| 2   | Apply all `ProgramOption` functions | [tea.go248-250](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L248-L250) |
| 3   | Set external context (or default to `context.Background()`) | [tea.go254-256](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L254-L256) |
| 4   | Create internal context with cancellation | [tea.go258](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L258-L258) |
| 5   | Set default output to `os.Stdout` if not specified | [tea.go261-263](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L261-L263) |
| 6   | Set default environment to `os.Environ()` if not specified | [tea.go266-268](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L266-L268) |

The `Program` struct maintains critical state including:

*   **Model reference**: The initial model to execute
*   **Context management**: Both external (user-provided) and internal contexts
*   **I/O streams**: Input reader and output writer
*   **Channel infrastructure**: Message, error, and command channels
*   **Terminal state**: File descriptors and saved terminal states
*   **Renderer**: The output rendering implementation
*   **Configuration**: Startup options and feature flags

**Sources**: [tea.go240-271](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L240-L271)
 [tea.go135-198](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L135-L198)

Program Initialization Phase
----------------------------

### Starting the Program

The `Run` method [tea.go578-746](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L578-L746)
 starts the program and blocks until termination. This method orchestrates the entire program lifecycle:

**Sources**: [tea.go578-746](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L578-L746)

### Initialization Steps

The initialization phase performs these operations in sequence:

#### 1\. Input Stream Configuration

The program determines the input source based on `inputType` [tea.go590-626](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L590-L626)
:

*   **`defaultInput`**: Uses `os.Stdin`, but if stdin is not a TTY (e.g., piped input), opens `/dev/tty` or `CONIN$` instead
*   **`ttyInput`**: Explicitly opens a new TTY for input
*   **`customInput`**: Uses the user-provided `io.Reader`

#### 2\. Handler Setup

Three background goroutines are started [tea.go629-713](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L629-L713)
:

| Handler | Function | Purpose |
| --- | --- | --- |
| Signal Handler | `handleSignals()` [tea.go273-312](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L273-L312) | Listens for `SIGINT` and `SIGTERM` |
| Resize Handler | `handleResize()` [tea.go314-329](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L314-L329) | Monitors terminal size changes via `SIGWINCH` |
| Command Handler | `handleCommands()` [tea.go333-372](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L333-L372) | Executes `Cmd` functions in goroutines |

#### 3\. Panic Recovery

If `withoutCatchPanics` is not set, a `defer recover()` wrapper is installed [tea.go634-641](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L634-L641)
 that catches panics and calls `recoverFromPanic` [tea.go840-848](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L840-L848)

#### 4\. Renderer Initialization

If no custom renderer is set, a `standardRenderer` is created [tea.go644-646](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L644-L646)
 The renderer is then started [tea.go681](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L681-L681)

#### 5\. Terminal Setup

The `initTerminal` method [tty.go25-37](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L25-L37)
 is called, which:

1.  Calls platform-specific `initInput()` ([tty\_unix.go15-30](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L15-L30)
     or [tty\_windows.go14-55](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L14-L55)
    )
2.  Saves the current terminal state
3.  Enables raw mode via `term.MakeRaw()`
4.  Hides the cursor via `renderer.hideCursor()`

#### 6\. Startup Options Application

The program applies configured startup options [tea.go654-678](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L654-L678)
:

*   Window title via `setWindowTitle()` if specified
*   Alt screen buffer if `withAltScreen` is set
*   Bracketed paste mode if not disabled
*   Mouse tracking if `withMouseCellMotion` or `withMouseAllMotion` is set
*   Focus reporting if `withReportFocus` is set

#### 7\. Initial Model Execution

The initial model's `Init()` method is called [tea.go685](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L685-L685)
 and if it returns a `Cmd`, that command is sent to the command handler. The initial view is then rendered [tea.go700](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L700-L700)

#### 8\. Input Reader Activation

The `initCancelReader` method [tty.go78-94](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L78-L94)
 creates a cancelable input reader and starts the `readLoop` goroutine [tty.go96-106](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L96-L106)
 to process input.

**Sources**: [tea.go578-746](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L578-L746)
 [tty.go25-37](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L25-L37)
 [tty.go78-94](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L78-L94)
 [tty\_unix.go15-30](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L15-L30)
 [tty\_windows.go14-55](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L14-L55)

Event Loop Execution
--------------------

### The Central Event Loop

The `eventLoop` method [tea.go382-505](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L382-L505)
 is the heart of the program. It continuously processes messages until termination:

**Sources**: [tea.go382-505](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L382-L505)

### Message Processing

Each iteration of the event loop [tea.go383-504](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L383-L504)
:

1.  **Blocks** on `select` waiting for context cancellation, errors, or messages
2.  **Filters** the message through the optional filter function [tea.go393-398](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L393-L398)
    
3.  **Handles special messages** like `QuitMsg`, `InterruptMsg`, screen control messages [tea.go401-486](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L401-L486)
    
4.  **Executes batch/sequence commands** asynchronously [tea.go473-479](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L473-L479)
    
5.  **Calls** `model.Update(msg)` for all other messages [tea.go494](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L494-L494)
    
6.  **Sends** returned `Cmd` to command handler [tea.go496-500](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L496-L500)
    
7.  **Renders** the view via `renderer.write(model.View())` [tea.go502](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L502-L502)

### Command Execution

Commands returned from `Update` are sent to the command handler goroutine [tea.go333-372](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L333-L372)
 which:

1.  **Spawns a new goroutine** for each `Cmd` [tea.go354-366](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L354-L366)
    
2.  **Catches panics** in command goroutines via `recoverFromGoPanic` [tea.go852-860](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L852-L860)
    
3.  **Executes the command** by calling `cmd()` [tea.go364](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L364-L364)
    
4.  **Sends the result** back to the message channel via `p.Send(msg)` [tea.go365](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L365-L365)
    

Special command types `BatchMsg` [tea.go533-573](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L533-L573)
 and `sequenceMsg` [tea.go507-531](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L507-L531)
 are handled with dedicated execution functions:

*   **`execBatchMsg`**: Executes commands concurrently using `sync.WaitGroup` [tea.go533-573](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L533-L573)
    
*   **`execSequenceMsg`**: Executes commands sequentially in order [tea.go507-531](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L507-L531)
    

**Sources**: [tea.go333-372](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L333-L372)

Program Termination
-------------------

### Termination Methods

Bubble Tea provides three ways to terminate a program:

**Sources**: [tea.go200-203](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L200-L203)
 [tea.go795-797](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L795-L797)
 [tea.go722-745](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L722-L745)

#### Graceful Termination with Quit

The `Quit` function [tea.go200-203](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L200-L203)
 returns a `QuitMsg` that can be returned from `Update`:

Or sent externally via `Program.Quit()` [tea.go788-790](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L788-L790)
:

**Behavior**:

*   Event loop exits gracefully [tea.go402-403](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L402-L403)
    
*   Final view is rendered [tea.go739](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L739-L739)
    
*   `Run()` returns `(model, nil)`

#### Forceful Termination with Kill

The `Kill` method [tea.go795-797](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L795-L797)
 immediately cancels the internal context:

**Behavior**:

*   Context cancellation propagates to all goroutines
*   Event loop exits immediately [tea.go385-386](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L385-L386)
    
*   Final view is **not** rendered [tea.go722-740](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L722-L740)
    
*   `Run()` returns `(model, ErrProgramKilled)` [tea.go722-735](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L722-L735)

### Shutdown Process

| 1   | Cancel internal context | [tea.go811](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L811-L811) |
| 2   | Wait for all handler goroutines to finish | [tea.go814](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L814-L814) |
| 3   | Cancel the input reader | [tea.go817-825](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L817-L825) |
| 4   | Stop or kill the renderer | [tea.go827-833](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L827-L833) |
| 5   | Restore terminal state | [tea.go835](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L835-L835) |

5.  Exit alt screen buffer if active [tty.go51-56](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L51-L56)
    
6.  Restore original terminal state via `term.Restore()` [tty.go64-73](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L64-L73)

### Program Errors

| Error | Condition | Code Reference |

| `ErrInterrupted` | Received `InterruptMsg` | [tea.go405-406](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L405-L406) |

### Panic Recovery

By default, panics are caught and handled gracefully:

**Main Loop Panics** [tea.go634-641](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L634-L641)
:

The `recoverFromPanic` method [tea.go840-848](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L840-L848)
:

1.  Sends `ErrProgramPanic` to the errors channel
2.  Calls `shutdown(true)` to clean up
3.  Prints the panic message and stack trace

**Command Goroutine Panics** [tea.go356-362](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L356-L362)
:

The `recoverFromGoPanic` method [tea.go852-860](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L852-L860)
:

1.  Sends `ErrProgramPanic` to the errors channel
2.  Calls `p.cancel()` to initiate shutdown
3.  Prints the panic message and stack trace

Panic catching can be disabled with `WithoutCatchPanics()` [options.go81-85](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L81-L85)

Synchronization and Testing
---------------------------

### Wait Method

The `Wait` method [tea.go799-802](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L799-L802)
 blocks until the program completes shutdown:

This is useful for coordinating shutdown when the program is running in a goroutine. The method waits on the `finished` channel, which is closed when `Run()` returns [tea.go583-586](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L583-L586)

### Send Method

The `Send` method [tea.go774-779](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L774-L779)
 allows external code to inject messages into the program:

**Behavior**:

*   Before `Run()` is called: **Blocks** until the program starts
*   After `Run()` returns: **No-op** (safe to call)
*   During execution: Sends message to `msgs` channel

This enables testing and external control of the program.

### Testing Support

Tests can use `WithoutRenderer()` [options.go177-181](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L177-L181)
 to disable output rendering, making testing easier:

The `Wait()` method allows tests to synchronize on program completion [tea\_test.go102-150](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea_test.go#L102-L150)
