Repository Multi-Module Structure
---------------------------------

The Bubble Tea repository uses a multi-module Go structure that separates the core library from examples. This organization allows independent versioning and dependency management while enabling coordinated local development through `replace` directives.

### Module Organization

**Sources:** [examples/go.mod1-20](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L1-L20)
 [go.mod1-32](https://github.com/charmbracelet/bubbletea/blob/f25595a8/go.mod#L1-L32)

### Module Purposes

| Module | Path | Purpose | Module Name |
| --- | --- | --- | --- |
| **Main Library** | `/` | Core Bubble Tea framework implementation | `charm.land/bubbletea/v2` |
| **Examples** | `/examples` | Sample applications demonstrating framework usage | `examples` |

**Sources:** [go.mod1](https://github.com/charmbracelet/bubbletea/blob/f25595a8/go.mod#L1-L1)
 [examples/go.mod1](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L1-L1)

Go Sum Files
------------

Each module maintains its own `go.sum` file containing cryptographic checksums for all dependencies.

| Module | Sum File | Purpose |
| --- | --- | --- |
| Main Library | `go.sum` | Checksums for core library dependencies |
| Examples | `examples/go.sum` | Checksums for examples + transitive dependencies |

### Checksum Verification

The `go.sum` files ensure reproducible builds by verifying that downloaded dependencies match expected checksums. When dependencies are updated, the corresponding `go.sum` file is automatically updated by the Go toolchain.

**Sources:** [go.sum1-39](https://github.com/charmbracelet/bubbletea/blob/f25595a8/go.sum#L1-L39)
 [examples/go.sum1-93](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.sum#L1-L93)

Module Independence
-------------------

Each module can be built, tested, and run independently:

The examples depend on the main library through the `replace` directive, so changes to the main library will immediately affect examples. This dependency is intentional and enables rapid testing of library changes.

**Sources:** [go.mod1-32](https://github.com/charmbracelet/bubbletea/blob/f25595a8/go.mod#L1-L32)
 [examples/go.mod1-20](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/go.mod#L1-L20)

Removing Replace Directives
---------------------------

The `replace` directives should **not** be removed during normal development. They are intentionally committed to the repository to facilitate local development. Only remove them when:

1.  Testing that the examples/tutorials work with a published version of the library
2.  Preparing for release to verify version compatibility

After testing without the `replace` directive, it should be restored before committing changes.
