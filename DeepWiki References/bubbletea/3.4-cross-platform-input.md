Purpose and Scope
-----------------

This page documents the platform-specific input handling implementations in Bubble Tea, focusing on the differences between Unix and Windows terminal initialization, raw mode setup, and TTY management. For general keyboard input processing (ANSI parsing, key detection) which is platform-agnostic, see [Keyboard Input](https://deepwiki.com/charmbracelet/bubbletea/3.1-keyboard-input)
. For mouse and terminal events, see [Mouse Input](https://deepwiki.com/charmbracelet/bubbletea/3.2-mouse-input)
 and [Terminal Events](https://deepwiki.com/charmbracelet/bubbletea/3.3-terminal-events)
.

Platform Abstraction Strategy
-----------------------------

Bubble Tea uses Go build tags to provide platform-specific implementations while maintaining a consistent interface. The framework defines three key functions that have platform-specific implementations:

**Sources:** [tty.go25-36](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L25-L36)
 [tty\_unix.go1-2](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L1-L2)
 [tty\_windows.go1-2](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L1-L2)

The build tags ensure only one implementation is compiled for each platform:

*   Unix: `//go:build darwin || dragonfly || freebsd || linux || netbsd || openbsd || solaris || aix || zos`
*   Windows: `//go:build windows`

Unix Implementation
-------------------

### TTY Initialization

The Unix implementation of `initInput()` performs the following operations:

**Sources:** [tty\_unix.go15-30](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L15-L30)

Key characteristics:

*   **TTY Device:** Uses `/dev/tty` for input when needed via `openInputTTY()` [tty\_unix.go32-38](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L32-L38)
    
*   **Raw Mode:** Applied via `term.MakeRaw()` which configures the terminal for character-by-character input without echoing [tty\_unix.go19](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L19-L19)
    
*   **State Preservation:** Saves `previousTtyInputState` for restoration on exit [tty\_unix.go19](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L19-L19)
    

### Suspend Support

Unix systems support process suspension through signals:

**Sources:** [tty\_unix.go40-49](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L40-L49)

The `suspendProcess()` function:

1.  Creates a channel to receive `SIGCONT`
2.  Registers the signal handler via `signal.Notify()`
3.  Sends `SIGTSTP` to the entire process group (`pid=0`)
4.  Blocks until `SIGCONT` is received
5.  Returns control to allow terminal restoration

The constant `suspendSupported = true` indicates this platform supports suspension [tty\_unix.go40](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L40-L40)

Windows Implementation
----------------------

### Console Initialization

Windows requires additional setup to enable Virtual Terminal processing:

**Sources:** [tty\_windows.go14-55](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L14-L55)

Key differences from Unix:

| Aspect | Unix | Windows |
| --- | --- | --- |
| **TTY Device** | `/dev/tty` | `CONIN$` |
| **Input Mode** | Raw mode via termios | Raw mode + VT input flag |
| **Output Mode** | Not modified separately | VT processing flag required |
| **State Variables** | `previousTtyInputState` | `previousTtyInputState` + `previousOutputState` |
| **Open Mode** | `os.Open()` | `os.OpenFile()` with `O_RDWR` |

**Sources:** [tty\_unix.go32-38](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_unix.go#L32-L38)
 [tty\_windows.go58-64](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L58-L64)

### Virtual Terminal Modes

Windows requires explicit enablement of Virtual Terminal features:

**Input Console Mode:**

*   Flag: `windows.ENABLE_VIRTUAL_TERMINAL_INPUT`
*   Effect: Allows ANSI escape sequences in input stream
*   Applied to: `ttyInput` file descriptor
*   Code: [tty\_windows.go25-33](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L25-L33)
    

**Output Console Mode:**

*   Flag: `windows.ENABLE_VIRTUAL_TERMINAL_PROCESSING`
*   Effect: Interprets ANSI escape sequences in output
*   Applied to: `ttyOutput` file descriptor
*   Code: [tty\_windows.go43-51](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L43-L51)
    

Without these flags, Windows console would not recognize or generate ANSI sequences, breaking all keyboard/mouse/cursor operations.

### No Suspend Support

Windows implementation provides a no-op for process suspension:

**Sources:** [tty\_windows.go66-68](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty_windows.go#L66-L68)

The `Suspend()` command (see [Command System](https://deepwiki.com/charmbracelet/bubbletea/2.4-command-system)
) checks `suspendSupported` before attempting suspension, effectively making it a no-op on Windows.

Terminal State Management
-------------------------

Both platforms share common state management logic in `tty.go`:

**Sources:** [tty.go13-60](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L13-L60)

### Initialization Flow

The `initTerminal()` function orchestrates platform-specific initialization:

1.  Check if using `nilRenderer` (skip if true) [tty.go26-29](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L26-L29)
    
2.  Call platform-specific `initInput()` [tty.go31-33](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L31-L33)
    
3.  Hide cursor via renderer [tty.go35](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L35-L35)
    

### Restoration Flow

The `restoreTerminalState()` function ensures clean terminal state on exit:

**Sources:** [tty.go41-75](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L41-L75)

The restoration process:

1.  Disables all rendering features (bracketed paste, mouse, focus, alt screen)
2.  Shows the cursor
3.  Restores original terminal state via `term.Restore()`
4.  On Windows, also restores output console state [tty.go69-73](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L69-L73)
    

### Suspend and Resume (Unix Only)

The suspend mechanism uses terminal state management:

**Sources:** [tty.go13-23](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L13-L23)

The `Program.suspend()` method:

1.  Calls `ReleaseTerminal()` to clean up terminal state
2.  Calls platform-specific `suspendProcess()` (blocks on Unix, no-op on Windows)
3.  Calls `RestoreTerminal()` to reinitialize after resume
4.  Sends `ResumeMsg` to notify the application [tty.go22](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L22-L22)
    

### Input Reader Lifecycle

The cancel reader is managed through the terminal state:

**Sources:** [tty.go78-117](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L78-L117)

The `initCancelReader()` function:

*   Creates a new `cancelreader` instance [tty.go85](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L85-L85)
    
*   Launches the `readLoop()` goroutine [tty.go91](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L91-L91)
    
*   If `cancel=true`, stops the existing reader first [tty.go79-82](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L79-L82)
    
*   Uses `waitForReadLoop()` with 500ms timeout [tty.go109-117](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L109-L117)
    

The `readLoop()` continuously reads input and handles errors:

*   Calls `readInputs()` which eventually calls `readAnsiInputs()` from `key.go` [tty.go99](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L99-L99)
    
*   Ignores `io.EOF` and `cancelreader.ErrCanceled` [tty.go100](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L100-L100)
    
*   Sends other errors to `p.errs` channel [tty.go103](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L103-L103)
    
*   Closes `readLoopDone` on exit [tty.go97](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L97-L97)
    

**Sources:** [tty.go96-117](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tty.go#L96-L117)
