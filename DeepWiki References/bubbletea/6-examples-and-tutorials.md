======================

This section provides an overview of the practical examples and tutorials included in the Bubbletea repository. These materials demonstrate framework usage patterns, component integration, and common application architectures.

**Scope**: This page covers the organization and structure of the tutorials and examples directories, their learning progression, and the key patterns they demonstrate. For detailed coverage of specific example categories, see subsections [6.1](https://deepwiki.com/charmbracelet/bubbletea/6.1-simple-examples)
 through [6.6](https://deepwiki.com/charmbracelet/bubbletea/6.6-component-library-examples)
. For core framework concepts, see [Core Components](https://deepwiki.com/charmbracelet/bubbletea/2-core-components)
. For command execution patterns, see [Command System](https://deepwiki.com/charmbracelet/bubbletea/2.4-command-system)
.

* * *

Repository Structure
--------------------

The Bubbletea repository organizes learning materials into two primary directories: `tutorials/` for step-by-step guides with annotated code, and `examples/` for standalone reference implementations.

### Directory Organization

**Sources**: [README.md24-25](https://github.com/charmbracelet/bubbletea/blob/f25595a8/README.md?plain=1#L24-L25)
 [tutorials/basics/README.md1-13](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/README.md?plain=1#L1-L13)
 [tutorials/commands/README.md1-10](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/commands/README.md?plain=1#L1-L10)

Both the `tutorials/` and `examples/` directories contain their own `go.mod` files with `replace` directives pointing to the parent module, enabling local development and testing of changes to the core framework.

### Tutorial 1: Basics (Shopping List)

The basics tutorial ([tutorials/basics/](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/)
) introduces the three core methods required by the `tea.Model` interface:

| Method | Purpose | Implementation |
| --- | --- | --- |
| `Init() tea.Cmd` | Returns initial command | [tutorials/basics/main.go28-30](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L28-L30) |
| `Update(tea.Msg) (tea.Model, tea.Cmd)` | Processes messages, updates state | [tutorials/basics/main.go32-57](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L32-L57) |
| `View() tea.View` | Renders UI based on model | [tutorials/basics/main.go59-82](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L59-L82) |

The model structure stores application state:

    type model struct {
        cursor   int                // Current selection
        choices  []string           // Available items
        selected map[int]struct{}   // Selected items
    }
    

**Key concepts demonstrated**:

*   Handling `tea.KeyPressMsg` events
*   State management with struct fields
*   Returning `tea.Quit` command to exit
*   Building string-based UI content
*   Wrapping content in `tea.NewView()`

**Sources**: [tutorials/basics/main.go10-90](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L10-L90)
 [tutorials/basics/README.md40-208](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/README.md?plain=1#L40-L208)

### Tutorial 2: Commands (HTTP Requests)

The commands tutorial ([tutorials/commands/](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/commands/)
) extends the basics with asynchronous I/O:

| Component | Type | Purpose |
| --- | --- | --- |
| `checkServer()` | `tea.Cmd` | Performs HTTP GET request |
| `statusMsg` | `int` | Reports HTTP status code |
| `errMsg` | `struct{ err error }` | Reports errors |

**Command execution pattern**:

1.  `Init()` returns `checkServer` function reference
2.  Framework executes `checkServer()` in goroutine
3.  `checkServer()` returns `statusMsg` or `errMsg`
4.  `Update()` receives message and updates model
5.  `View()` renders result

**Sources**: [tutorials/commands/main.go1-79](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/commands/main.go#L1-L79)
 [tutorials/commands/README.md36-176](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/commands/README.md?plain=1#L36-L176)

### Error Handling Pattern

Commands tutorial demonstrates the idiomatic error handling approach:

    // Define error message type
    type errMsg struct{ err error }
    
    // Implement error interface
    func (e errMsg) Error() string { return e.err.Error() }
    
    // Return from Cmd
    if err != nil {
        return errMsg{err}
    }
    
    // Handle in Update
    case errMsg:
        m.err = msg
        return m, tea.Quit
    

**Sources**: [tutorials/commands/main.go32-36](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/commands/main.go#L32-L36)
 [tutorials/commands/main.go48-50](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/commands/main.go#L48-L50)

Integration with Component Libraries
------------------------------------

Examples demonstrate integration with the Charmbracelet ecosystem:

### Component Usage Pattern

**Sources**: [README.md34-39](https://github.com/charmbracelet/bubbletea/blob/f25595a8/README.md?plain=1#L34-L39)
 [README.md315-327](https://github.com/charmbracelet/bubbletea/blob/f25595a8/README.md?plain=1#L315-L327)

### Example Component Integration

Pager example ([examples/pager](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/pager)
) demonstrates viewport integration:

*   Model embeds `viewport.Model`
*   Init calls `viewport.New(width, height)`
*   Update forwards messages to `viewport.Update()`
*   View calls `viewport.View()`

Spinner example ([examples/spinner](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/spinner)
) demonstrates animation:

*   Model embeds `spinner.Model`
*   Init calls `spinner.New()` and sets style
*   Update forwards `spinner.TickMsg` to `spinner.Update()`
*   View calls `spinner.View()`

TextInput example ([examples/textinput](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/textinput)
) demonstrates focus management:

*   Model embeds `textinput.Model`
*   Init calls `textinput.New()` and `Focus()`
*   Update forwards key events to `textinput.Update()`
*   View calls `textinput.View()`

**Sources**: [README.md315-327](https://github.com/charmbracelet/bubbletea/blob/f25595a8/README.md?plain=1#L315-L327)

Running Examples and Tutorials
------------------------------

### Building and Execution

Each tutorial and example can be executed independently:

### Module Dependencies

Both directories use `replace` directives to reference the parent module:

    // tutorials/go.mod or examples/go.mod
    replace charm.land/bubbletea/v2 => ../
    

This enables:

*   Testing framework changes against examples
*   Developing new features with immediate validation
*   Ensuring examples stay synchronized with API changes

**Sources**: [README.md27-30](https://github.com/charmbracelet/bubbletea/blob/f25595a8/README.md?plain=1#L27-L30)
