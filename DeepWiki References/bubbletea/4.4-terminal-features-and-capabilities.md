This document describes the terminal features and capabilities that Bubbletea can detect, configure, and utilize during rendering. These features are controlled through fields on the `View` struct and are managed by the renderer implementations, primarily `cursedRenderer`.

This page focuses on **terminal-level features** such as alternate screen mode, mouse tracking, keyboard protocols, and cursor styling. For information about the renderer interface and View structure, see [Renderer Interface and View Structure](https://deepwiki.com/charmbracelet/bubbletea/4.1-renderer-interface-and-view-structure)
. For details on the rendering implementation and diff-based updates, see [Cursed Renderer Implementation](https://deepwiki.com/charmbracelet/bubbletea/4.2-cursed-renderer-implementation)
. For terminal state management and platform-specific setup, see [Terminal Management](https://deepwiki.com/charmbracelet/bubbletea/4.3-terminal-management)
.

Alternate Screen Mode
---------------------

The alternate screen buffer is a separate screen buffer that allows programs to take over the entire terminal display without disturbing the user's scrollback history. When enabled, the program's output is rendered in this separate buffer, and when disabled, the terminal returns to the main screen buffer.

### Configuration

Set `View.AltScreen = true` to enable alternate screen mode:

### Implementation Details

The renderer uses the `DECSET/DECRST` mode 1049, which saves and restores the cursor position:

Cursor Configuration\
--------------------\
\
The cursor can be configured for visibility, shape, blink state, position, and color.\
\
### Cursor Structure\
\
### Cursor Shapes\
\
### ANSI Cursor Style Encoding\
\
The cursor style is encoded as a single integer combining shape and blink state:\
\
| Shape | Blink | Value | ANSI Sequence |\
| --- | --- | --- | --- |\
| Block | Yes | 1   | `\x1b[1 q` |\
| Block | No  | 2   | `\x1b[2 q` |\
| Underline | Yes | 3   | `\x1b[3 q` |\
| Underline | No  | 4   | `\x1b[4 q` |\
| Bar | Yes | 5   | `\x1b[5 q` |\
| Bar | No  | 6   | `\x1b[6 q` |\
| Reset | \-  | 0   | `\x1b[0 q` |\
\
The encoding is: `style = (shape * 2) + 1 + (blink ? 0 : 1)`\
\
### Cursor Visibility\
\
Cursor visibility is controlled by whether `View.Cursor` is nil:\
\
| View Field | Effect | ANSI Sequence |\
| --- | --- | --- |\
| `Cursor = nil` | Hide cursor | `\x1b[?25l` |\
| `Cursor = &Cursor{...}` | Show cursor | `\x1b[?25h` |\
\
### Cursor Color\

If `Cursor.Color` is set, the cursor color is changed using OSC sequences:\

| --- | --- | --- |\

| Reset Color | `\x1b]112\x1b\\` | `\x1b]112\x1b\\` |\

Keyboard Enhancement Protocols\

\
Bubbletea supports modern keyboard enhancement protocols that provide better key disambiguation and event reporting. Two protocols are enabled:\
\
1.  **modifyOtherKeys** (xterm protocol 2): Disambiguates modified keys\
2.  **Kitty Keyboard Protocol**: Provides advanced key reporting with multiple flags\
\
### Configuration\
\
### Protocol Details\
\
Both protocols are always enabled, but with different flag configurations:\
\
| Protocol | ANSI Sequence | Purpose |\
| --- | --- | --- |\
| modifyOtherKeys mode 2 | `\x1b[>4;2m` | Enable enhanced key disambiguation |\
| Kitty (basic) | `\x1b[=1;1u` | Enable basic key disambiguation |\
| Kitty (with releases) | `\x1b[=3;1u` | Enable key disambiguation + press/release events |\
| modifyOtherKeys reset | `\x1b[>4m` | Disable modifyOtherKeys |\
| Kitty reset | `\x1b[=0;1u` | Disable Kitty protocol |\
| Kitty query | `\x1b[?u` | Request current Kitty keyboard state |\

Synchronized Output\
-------------------\
\
Synchronized output mode (also called synchronized updates) is a terminal feature that allows atomic screen updates, eliminating tearing and flicker by buffering all output until a complete frame is ready.\
\
### Configuration\
\
Synchronized output is controlled internally by the renderer and can be enabled via:\
\
This is typically configured through program options during initialization.\
\
### ANSI Sequences\
\
| Operation | ANSI Sequence |\
| --- | --- |\
| Begin synchronized output | `\x1b[?2026h` |\
| End synchronized output | `\x1b[?2026l` |\
\
### Rendering Strategy\
\
The renderer uses synchronized output when available, with a fallback to cursor hide/show wrapping:\
\
This strategy ensures:\
\
1.  When synchronized output is supported, all updates are atomic\
2.  When not supported, cursor hide/show minimizes flicker (best effort)\
3.  Cursor visibility updates are placed inside (syncd) or outside (hide/show) the buffering region appropriately\

Window Title\
------------\
\
The terminal window title can be set programmatically using the OSC 0 or OSC 2 sequences.\

### Configuration\

### ANSI Sequences\
