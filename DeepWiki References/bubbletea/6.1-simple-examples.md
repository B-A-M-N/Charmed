This page walks through the simplest applications in the Bubble Tea repository, demonstrating the minimal implementation of the Model-View-Update pattern, timer-based commands, and basic user interaction. These examples serve as the most accessible entry point to understanding Bubble Tea's core architecture.

Overview
--------

The simple examples demonstrate fundamental Bubble Tea concepts through minimal implementations:

*   **Simple Counter** ([examples/simple/main.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go)
    ): A basic countdown timer running in inline mode
*   **Fullscreen Counter** ([examples/fullscreen/main.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go)
    ): Same countdown logic using alternate screen buffer
*   **Shopping List** ([tutorials/basics/main.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go)
    ): An interactive to-do list application with keyboard navigation

The counter examples implement a countdown from 5 seconds to 0, then exit automatically. The shopping list implements a simple interactive checklist. Together they showcase:

*   Minimal model implementation (single integer and struct types)
*   Timer-based commands using `tea.Tick`
*   Keyboard input handling (`tea.KeyMsg`)
*   State management with user interaction
*   The three required methods: `Init()`, `Update()`, `View()`
*   Program lifecycle from initialization to termination

Simple Counter Example
----------------------

The simple counter is the most basic Bubble Tea program, implementing a countdown timer that runs inline in the terminal. The entire program is 79 lines including comments and demonstrates the absolute minimum required to build a functional TUI application.

**Simple Counter Program Flow**

### Model Definition

The model is defined as a simple integer type at [examples/simple/main.go34](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L34-L34)
:

This is the simplest possible model type in Bubble Tea. The integer value directly represents the countdown state. The program initializes with `model(5)` at [examples/simple/main.go25](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L25-L25)

### Init Method

The `Init()` method at [examples/simple/main.go38-40](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L38-L40)
 returns the initial command:

This starts the timer by returning the `tick` command function, which executes immediately when the program starts.

### Update Method

The `Update()` method at [examples/simple/main.go45-63](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L45-L63)
 handles two message types:

| Message Type | Handler Logic | Result |
| --- | --- | --- |
| `tea.KeyMsg` | Checks for quit keys (`q`, `ctrl+c`) or suspend (`ctrl+z`) | Returns `tea.Quit` or `tea.Suspend` |
| `tickMsg` | Decrements counter, checks if <= 0 | Returns `tea.Quit` or `tick` |

Key handling supports:

*   `q` or `ctrl+c`: Immediate quit
*   `ctrl+z`: Process suspension (Unix signal)

On each tick, the model decrements and checks if the countdown has reached zero. If not, it returns another `tick` command to continue the countdown loop.

### View Method

The `View()` method at [examples/simple/main.go67-69](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L67-L69)
 renders the current state:

The view simply displays the current counter value with usage instructions.

### Tick Command

The `tick` function at [examples/simple/main.go75-78](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L75-L78)
 implements a blocking 1-second timer:

This pattern executes in a separate goroutine spawned by Bubble Tea. After sleeping for one second, it returns a `tickMsg` that flows back into the `Update()` method. The custom message type `tickMsg` is defined at [examples/simple/main.go73](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L73-L73)
 as `type tickMsg time.Time`.

### Logging Configuration

The example includes optional logging setup at [examples/simple/main.go17-22](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L17-L22)
:

This demonstrates the recommended pattern for debugging TUI applications, since stdout is occupied by the interface.

Fullscreen Counter Example
--------------------------

The fullscreen counter implements the same countdown logic as the simple counter but runs in alternate screen buffer mode. This is the standard mode for fullscreen terminal applications.

**Fullscreen Counter Program Flow**

### Key Differences from Simple Counter

The fullscreen version differs in three main aspects:

| Aspect | Simple Counter | Fullscreen Counter |
| --- | --- | --- |
| Screen Mode | Inline (normal terminal) | Alternate screen buffer |
| Program Options | None | `tea.WithAltScreen()` |
| Timer Implementation | Custom `time.Sleep` | `tea.Tick()` helper |
| View Formatting | Left-aligned | Centered with spacing |

### Model and Messages

The model type at [examples/fullscreen/main.go14](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L14-L14)
 and message type at [examples/fullscreen/main.go16](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L16-L16)
 are identical to the simple counter:

### Program Initialization

The program is created with the `tea.WithAltScreen()` option at [examples/fullscreen/main.go19](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L19-L19)
:

This option causes Bubble Tea to:

1.  Switch to the alternate screen buffer on startup
2.  Hide the cursor
3.  Restore the previous terminal state on exit

### Tick Implementation

The fullscreen version uses `tea.Tick()` at [examples/fullscreen/main.go52-56](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L52-L56)

This is the recommended pattern for timer-based commands. `tea.Tick()` internally uses `time.NewTimer()` and handles cleanup automatically. The callback function receives the actual time and can convert it to any message type.

### Update Logic

The update method at [examples/fullscreen/main.go29-46](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L29-L46)

*   Uses `q`, `esc`, or `ctrl+c` for quitting (no suspend support)
*   Calls `tick()` as a function instead of returning it directly

### View Rendering

The view at [examples/fullscreen/main.go48-50](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L48-L50)

Comparison of Implementation Patterns
-------------------------------------

### Timer Command Patterns

The two examples demonstrate different approaches to timer-based commands:

**Simple Counter: Custom Blocking Timer**

Advantages:

*   Simple and direct
*   Full control over sleep behavior
*   Useful for learning the command pattern

Disadvantages:

*   Blocks the goroutine for the entire duration
*   No built-in cancellation
*   Resources held during sleep

**Fullscreen Counter: tea.Tick Helper**

Advantages:

*   Uses `time.NewTimer()` internally (more efficient)
*   Automatic resource cleanup
*   Can be cancelled if program exits
*   Recommended pattern for production code

Sources: [examples/simple/main.go75-78](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L75-L78)
 [examples/fullscreen/main.go52-56](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L52-L56)

### Update Method Patterns

Both examples follow the same message handling pattern but with slight variations:

| Pattern Element | Simple Counter | Fullscreen Counter |
| --- | --- | --- |
| Type switch on message | ✓   | ✓   |
| KeyMsg handling | ✓   | ✓   |
| Quit keys | `q`, `ctrl+c` | `q`, `esc`, `ctrl+c` |
| Suspend support | ✓ (`ctrl+z`) | ✗   |
| Custom message handling | `tickMsg` | `tickMsg` |
| Return command directly | `return m, tick` | `return m, tick()` |

### Screen Mode Comparison

| Feature | Inline Mode | Alternate Screen |
| --- | --- | --- |
| Terminal state | Shares with shell | Separate buffer |
| Previous output | Visible above app | Hidden |
| Exit behavior | Output remains | Restores previous screen |
| Cursor | Visible | Hidden by default |
| Use case | Status displays, prompts | Full TUI applications |

Sources: [examples/simple/main.go45-63](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L45-L63)
 [examples/fullscreen/main.go29-46](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L29-L46)

MVU Pattern Implementation
--------------------------

Both counter examples demonstrate the complete MVU cycle:

**MVU Cycle in Counter Examples**

### Init Phase

Both examples initialize with the same pattern:

1.  Create model with initial value (`model(5)`)
2.  Call `Init()` which returns first command
3.  Execute `tick` command to start timer loop

### Update Phase

The update cycle processes two message sources:

*   **Keyboard input**: Generates `tea.KeyMsg` for user interaction
*   **Timer events**: Generates `tickMsg` from the `tick` command

Each message flows through the type switch, updates state, and optionally returns a new command.

### View Phase

The view is called after every update and renders the current model state. Both examples use simple string formatting with `fmt.Sprintf()`.

Sources: [examples/simple/main.go38-69](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/simple/main.go#L38-L69)
 [examples/fullscreen/main.go25-50](https://github.com/charmbracelet/bubbletea/blob/f25595a8/examples/fullscreen/main.go#L25-L50)

Shopping List Example
---------------------

The shopping list tutorial ([tutorials/basics/main.go](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go)
) demonstrates the first interactive Bubble Tea application, implementing a simple to-do list with keyboard navigation and item selection. This is the canonical "hello world" example for Bubble Tea, covered extensively in the main README.

**Shopping List Application Flow**

### Model Structure

The shopping list uses a struct model defined at [tutorials/basics/main.go10-14](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L10-L14)
:

| Field | Type | Purpose |
| --- | --- | --- |
| `cursor` | `int` | Index of currently selected item |
| `choices` | `[]string` | List of available items |
| `selected` | `map[int]struct{}` | Set of selected item indices |

The `selected` field uses Go's idiomatic "map as set" pattern, where the keys are indices and the values are empty structs.

### Initialization

The `initialModel()` function at [tutorials/basics/main.go16-25](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L16-L25)
 creates the initial state:

The `Init()` method at [tutorials/basics/main.go27-29](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L27-L29)
 sets the window title:

This demonstrates how commands can perform I/O operations (terminal control) during initialization.

### Update Logic

The `Update()` method at [tutorials/basics/main.go31-56](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L31-L56)
 handles keyboard input exclusively:

**Key Bindings**

| Key | Action | Implementation |
| --- | --- | --- |
| `q`, `ctrl+c` | Quit program | Returns `tea.Quit` |
| `up`, `k` | Move cursor up | Decrements `m.cursor`, bounds check |
| `down`, `j` | Move cursor down | Increments `m.cursor`, bounds check |
| `enter`, `space` | Toggle selection | Insert/delete from `m.selected` map |

Navigation logic at [tutorials/basics/main.go37-44](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L37-L44)
:

*   Cursor movement is bounded by `0` and `len(m.choices)-1`
*   No wrapping behavior (stops at edges)

Selection toggle logic at [tutorials/basics/main.go45-51](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L45-L51)
:

*   Checks if current index exists in `m.selected` map
*   If exists, deletes the entry (unselect)
*   If not exists, inserts empty struct (select)

### View Rendering

The `View()` method at [tutorials/basics/main.go58-78](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tutorials/basics/main.go#L58-L78)
 renders the shopping list with visual indicators:

**View Rendering Logic**

Visual indicators:

*   Cursor: `>` for current item, (space) for others
*   Checkbox: `x` for selected, (space) for unselected
*   Format: `{cursor} [{checkbox}] {item text}`

Example output:

    What should we buy at the market?
    
      [ ] Buy carrots
    > [x] Buy celery
      [ ] Buy kohlrabi
    
    Press q to quit.
