Program Options
===============

Program Options provide a flexible configuration system for customizing the behavior of a `tea.Program` instance. These options are passed to `NewProgram` and control various aspects of program initialization, including input/output sources, terminal features, mouse modes, signal handling, and message processing.

For information about the Program lifecycle itself, see [Program Lifecycle](https://deepwiki.com/charmbracelet/bubbletea/2.1-program-lifecycle)
. For details on the rendering system that some options configure, see [Renderer Interface](https://deepwiki.com/charmbracelet/bubbletea/4.1-renderer-interface-and-view-structure)
.

ProgramOption Type
------------------

`ProgramOption` is a functional option type that modifies a `Program` instance during initialization. Each option is a function that accepts a pointer to a `Program` and configures specific fields or behaviors.

**Sources:** [options.go9-15](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L9-L15)
 [tea.go240-250](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L240-L250)

The basic pattern is defined at [options.go9-15](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L9-L15)
:

Options are applied in `NewProgram` at [tea.go247-250](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L247-L250)
:

Input and Output Options
------------------------

### WithInput

Sets a custom input source for reading user input. By default, programs use `os.Stdin`. Pass `nil` to disable input entirely.

| Field Modified | Type | Default |
| --- | --- | --- |
| `p.input` | `io.Reader` | `os.Stdin` |
| `p.inputType` | `inputType` | `customInput` |

**Sources:** [options.go38-43](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L38-L43)

### WithInputTTY

Opens a new TTY (or console input device on Windows) for input, regardless of whether stdin is a terminal. This is useful when stdin is redirected but you still need terminal input.

| Field Modified | Type | Value |
| --- | --- | --- |
| `p.inputType` | `inputType` | `ttyInput` |

**Sources:** [options.go46-50](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L46-L50)

### WithOutput

Sets the output destination for rendering. By default, programs use `os.Stdout`.

| Field Modified | Type | Default |
| --- | --- | --- |
| `p.output` | `io.Writer` | `os.Stdout` |

**Sources:** [options.go28-32](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L28-L32)
 [tea.go260-263](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L260-L263)

### WithEnvironment

Configures environment variables for the program. This is particularly useful for remote sessions (e.g., SSH) where you need to pass environment from the remote session.

| Field Modified | Type | Default |
| --- | --- | --- |
| `p.environ` | `[]string` | `os.Environ()` |

**Sources:** [options.go63-67](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L63-L67)
 [tea.go265-268](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L265-L268)

Context and Lifecycle Options
-----------------------------

### WithContext

Provides an external context for controlling program execution. When the context is cancelled, the program terminates with `ErrProgramKilled` wrapping the context error.

**Sources:** [options.go20-24](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L20-L24)
 [tea.go254-258](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L254-L258)

The internal context is derived from the external context at [tea.go258](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L258-L258)
:

If no external context is provided, the program uses `context.Background()` as seen at [tea.go254-256](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L254-L256)

Signal Handling Options
-----------------------

### WithoutSignalHandler

Disables the default signal handler that captures `SIGINT` and `SIGTERM`. Use this when you want to handle OS signals yourself.

| Startup Option Modified | Behavior |
| --- | --- |
| `withoutSignalHandler` | Signal handler goroutine not started |

**Sources:** [options.go71-75](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L71-L75)
 [tea.go629-631](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L629-L631)

The signal handler check occurs during `Run()` at [tea.go629-631](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L629-L631)

### WithoutSignals

Immediately sets the program to ignore all OS signals. This is primarily used for testing.

| Field Modified | Type | Value |
| --- | --- | --- |
| `p.ignoreSignals` | `uint32` | `1` |

**Sources:** [options.go89-93](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L89-L93)

### WithoutCatchPanics

Disables automatic panic recovery. When disabled, panics will leave the terminal in an unusable state because cleanup operations won't execute.

| Startup Option Modified | Behavior |
| --- | --- |
| `withoutCatchPanics` | Panic recovery disabled in event loop and command execution |

**Sources:** [options.go81-85](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L81-L85)
 [tea.go634-641](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L634-L641)

Panic catching is implemented at multiple points: [tea.go634-641](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L634-L641)

Terminal Feature Options
------------------------

### WithAltScreen

Starts the program with the alternate screen buffer enabled, providing full-window mode. The alternate screen is automatically exited when the program terminates.

**Sources:** [options.go109-113](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L109-L113)
 [tea.go658-660](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L658-L660)

The alternate screen is activated during `Run()` at [tea.go658-660](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L658-L660)

### WithoutBracketedPaste

Disables bracketed paste mode. By default, Bubble Tea enables bracketed paste to distinguish pasted content from typed content.

| Startup Option Modified | Behavior |
| --- | --- |
| `withoutBracketedPaste` | Bracketed paste not enabled on startup |

**Sources:** [options.go116-120](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L116-L120)
 [tea.go661-663](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L661-L663)

### WithReportFocus

Enables focus reporting, which sends `FocusMsg` and `BlurMsg` when the terminal gains or loses focus. Not all terminals support this feature, and tmux requires configuration.

| Startup Option Modified | Behavior |
| --- | --- |
| `withReportFocus` | Focus events reported to Update method |

**Sources:** [options.go248-252](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L248-L252)
 [tea.go676-678](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L676-L678)

Mouse Input Options
-------------------

Mouse options control how mouse events are captured and reported. They are mutually exclusive - only one can be active at a time.

### WithMouseCellMotion

Enables mouse in "cell motion" mode, which captures:

*   Click, release, and wheel events
*   Movement events when a button is pressed (drag events)

This mode has broader terminal support than all motion mode.

**Sources:** [options.go137-142](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L137-L142)
 [tea.go664-670](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L664-L670)
 [tea.go674](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L674-L674)

The mutual exclusion is handled at [options.go139-140](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L139-L140)
:

### WithMouseAllMotion

Enables mouse in "all motion" mode, which captures all mouse events including hover (movement without buttons pressed). This enables richer interactions but is not supported by all terminals.

| Startup Option Modified | Behavior |
| --- | --- |
| `withMouseAllMotion` | All mouse movement tracked, including hover |
| `withMouseCellMotion` | Cleared (mutually exclusive) |

**Sources:** [options.go162-167](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L162-L167)
 [tea.go667-670](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L667-L670)

Rendering Options
-----------------

### WithoutRenderer

Disables the renderer entirely, causing output to be sent plainly to the output writer without any terminal control sequences. This is useful for non-TUI modes or daemon-like behavior.

| Field Modified | Type | Value |
| --- | --- | --- |
| `p.renderer` | `renderer` | `&nilRenderer{}` |

**Sources:** [options.go177-181](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L177-L181)
 [nil\_renderer\_test.go5-24](https://github.com/charmbracelet/bubbletea/blob/f25595a8/nil_renderer_test.go#L5-L24)

### WithANSICompressor

Enables ANSI sequence compression to reduce output size, at the cost of processing overhead. This feature is deprecated due to performance concerns.

| Startup Option Modified | Status |
| --- | --- |
| `withANSICompressor` | Deprecated |

**Sources:** [options.go191-195](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L191-L195)
 [tea.go645](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L645-L645)

### WithFPS

Sets a custom maximum frames per second for the renderer. The value is capped between 1 and 120 FPS. Default is 60 FPS.

| Field Modified | Type | Range |
| --- | --- | --- |
| `p.fps` | `int` | 1-120 |

**Sources:** [options.go235-239](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L235-L239)

Message Processing Options
--------------------------

### WithFilter

Installs a message filter function that intercepts messages before they reach the `Update` method. The filter can modify, replace, or suppress messages by returning a different message or `nil`.

**Sources:** [options.go226-230](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L226-L230)
 [tea.go190](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L190-L190)
 [tea.go392-398](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L392-L398)

The filter is invoked in the event loop at [tea.go392-398](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L392-L398)
:

Example use case: preventing program shutdown when there are unsaved changes, as shown in the documentation at [options.go207-218](https://github.com/charmbracelet/bubbletea/blob/f25595a8/options.go#L207-L218)

Startup Options Implementation
------------------------------

The `startupOptions` type is implemented as a bitfield for efficient storage and checking of multiple boolean options.

**Sources:** [tea.go89-108](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L89-L108)

The type definition and constants are at [tea.go89-108](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L89-L108)
:

Options are stored in the `Program.startupOptions` field at [tea.go144](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L144-L144)
 and checked throughout the initialization and event loop.

Option Application Flow
-----------------------

**Sources:** [tea.go240-271](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L240-L271)
 [tea.go578-679](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L578-L679)

The full flow is:

1.  **Program Creation** [tea.go242-245](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L242-L245)
     - Initialize `Program` struct
2.  **Option Application** [tea.go247-250](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L247-L250)
     - Apply each `ProgramOption`
3.  **Default Values** [tea.go254-268](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L254-L268)
     - Set unspecified defaults
4.  **Context Derivation** [tea.go258](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L258-L258)
     - Create internal context
5.  **Startup Application** [tea.go654-678](https://github.com/charmbracelet/bubbletea/blob/f25595a8/tea.go#L654-L678)
     - Apply `startupOptions` during `Run()`
