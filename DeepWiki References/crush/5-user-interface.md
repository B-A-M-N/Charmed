Overview
--------

The TUI is built on [Bubble Tea](https://charm.land/bubbletea)
 (`charm.land/bubbletea/v2`). The root model is `UI` in [internal/ui/model/ui.go163](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L163-L163)
 It owns all visual sub-components and processes every `tea.Msg` that arrives from the Bubble Tea runtime. Screen layout uses the `ultraviolet` (`github.com/charmbracelet/ultraviolet`) screen-drawing library [internal/ui/model/ui.go56](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L56-L56)
 while styling is handled by `lipgloss` (`charm.land/lipgloss/v2`) [internal/ui/model/ui.go27](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L27-L27)

The TUI layer sits between the backend `app.App` (event source) and the terminal. Backend events are pushed through typed messages; the `UI.Update` method converts them into rendering state changes.

Dialog System
-------------

### `Dialog` Interface

Every modal dialog implements a shared interface that defines its ID, message handling logic, and drawing routine. This allows the `Overlay` to manage them generically.

### `Overlay` — Dialog Stack

The `Overlay` manages a stack of dialogs. The top of the stack is the active dialog that receives input and is rendered last (on top).

### Concrete Dialog Implementations

Crush uses several specialized dialogs:

*   **Commands**: Command palette for quick actions.
*   **Models**: Selection list for AI providers and models.
*   **Sessions**: Manager for conversation history.
*   **API Key**: Input field for provider credentials.
