This page documents the chat display subsystem: the `Chat` model that manages the scrollable list of messages, how `MessageItem`s are managed, the follow-mode auto-scroll behavior, the animation pipeline, and the high-performance rendering cache.

* * *

Overview
--------

The chat display is a high-level UI component that bridges the gap between raw `message.Message` entities and the Bubble Tea rendering loop. It is primarily defined in `internal/ui/model/chat.go` and utilizes a specialized `list.List` for lazy rendering of chat bubbles and tool outputs.

| Component | Role |
| --- | --- |
| `Chat` | The controller model that manages a `list.List` and maps IDs to indices. [internal/ui/model/chat.go35-73](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L35-L73) |
| `MessageItem` | Interface for any renderable item in the chat list. [internal/ui/list/item.go21-39](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L21-L39) |
| `List` | Core layout engine for vertical stacking and viewport management. [internal/ui/list/list.go10-54](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L10-L54) |
| `chatDrawCache` | Memoization layer for decoded ANSI output to optimize redraw performance. [internal/ui/model/chat.go88-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L88-L92) |

* * *

The `Chat` Model
----------------

The `Chat` struct manages the state of the message list, including visibility-based animation optimizations and mouse interaction state. It maintains an `idInxMap` to allow direct updates to messages even as their positions change in the list.

### Key Functions

*   **`NewChat`**: Initializes the `list.List` with a gap of 1 and registers render callbacks for highlighting and focus borders. [internal/ui/model/chat.go96-110](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L96-L110)
    
*   **`SetMessages` / `AppendMessages`**: Populates the list and maintains the `idInxMap` for O(1) lookups by ID. It handles `NestedToolContainer` types to map sub-tool IDs to the parent message index. [internal/ui/model/chat.go317-353](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L317-L353)
    
*   **`UpdateMessage`**: Updates an existing message by ID, utilizing the `idInxMap` to find the correct `list.Item`. [internal/ui/model/chat.go368-384](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L368-L384)

### Viewport and Scroll State

The `list.List` manages scrolling using `offsetIdx` (the index of the first visible item) and `offsetLine` (the number of lines within that item that are scrolled out of view). [internal/ui/list/list.go27-32](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L27-L32)

*   **`ScrollBy(lines)`**: Moves the viewport up or down, correctly handling item boundaries and gaps. [internal/ui/list/list.go214-273](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L214-L273)
    
*   **`VisibleItemIndices()`**: Calculates which items are currently on screen, used for animation throttling. [internal/ui/list/list.go277-305](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L277-L305)

Animation System
----------------

Animations (spinners, thinking indicators) are managed via `anim.StepMsg`. To save CPU, `Chat` only propagates animation ticks to items currently visible in the viewport.

### Visibility Optimization

1.  **`Animate(msg anim.StepMsg)`**: Checks if the target item's index is within `list.VisibleItemIndices()`. [internal/ui/model/chat.go415-440](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L415-L440)
    
2.  **`pausedAnimations`**: If an item is off-screen, its ID is added to this map, and the animation tick chain is broken by returning `nil` instead of a command. [internal/ui/model/chat.go430-435](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L430-L435)
    
3.  **`RestartPausedVisibleAnimations`**: When the user scrolls, this method checks the `pausedAnimations` map and restarts ticks for any IDs that have entered the viewport by calling `animatable.StartAnimation()`. [internal/ui/model/chat.go444-475](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L444-L475)

High-Performance Rendering
--------------------------

To handle long chat histories with complex ANSI styling, the `Chat` component implements a two-tier caching strategy.

### 1\. List-Level Cache (`list.List`)

The `list.List` maintains a `cache` map keyed by the `Item` pointer. [internal/ui/list/list.go45](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L45-L45)

*   **`Versioned.Bump()`**: Items must increment their version on mutation to invalidate the cache. [internal/ui/list/item.go68-70](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L68-L70)
    
*   **`Finished()`**: Once an item returns `true`, the list freezes its rendered output, skipping `Render()` calls entirely for future frames. [internal/ui/list/item.go32-38](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L32-L38)
    

### 2\. Chat-Level Draw Cache (`chatDrawCache`)

The `Chat.Draw` method uses a `chatDrawCache` to memoize the decoded form of the `list.Render()` output. [internal/ui/model/chat.go88-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L88-L92)

*   **ANSI Decoding**: `uv.StyledString.Draw` is expensive because it reparses ANSI sequences every frame. `Chat` performs this once and stores the result in a `uv.ScreenBuffer`. [internal/ui/model/chat.go134-140](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L134-L140)
    
*   **Invalidation**: The cache is invalidated if the rendered string changes or the terminal's `ansi.Method` (graphemes vs wcwidth) changes. [internal/ui/model/chat.go134-137](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L134-L137)
    

Sources: [internal/ui/model/chat.go124-173](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L124-L173)

Follow-Mode and Auto-Scroll
---------------------------

The `follow` flag in `Chat` determines if the view should automatically scroll to the bottom when new messages arrive.

| Action | Follow State Change |
| --- | --- |
| `ScrollToBottom()` | Set to `true`. [internal/ui/model/chat.go516](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L516-L516) |
| `ScrollToTop()` | Set to `false`. [internal/ui/model/chat.go522](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L522-L522) |
| Manual Scroll Up | Set to `false` if the scroll results in the view leaving the bottom. [internal/ui/model/chat.go530-532](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L530-L532) |
| `SetSize()` | If `AtBottom()`, calls `ScrollToBottom()` to maintain the anchor. [internal/ui/model/chat.go250-253](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L250-L253) |

The `AtBottom()` check in the underlying list determines if the current viewport is already at the end of the content. [internal/ui/list/list.go117-138](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L117-L138)
