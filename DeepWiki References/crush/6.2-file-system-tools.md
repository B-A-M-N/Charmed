Purpose and Scope
-----------------

This document describes the file system manipulation tools available in Crush's tool system. These tools enable AI agents to read, write, and modify files within the working directory and its subdirectories. The primary tools covered are:

*   `view` — Read file contents with optional line ranges, image support, and LSP diagnostic integration. [internal/agent/tools/view.go75-79](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L75-L79)
     [internal/agent/tools/view.go98-101](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L98-L101)
    
*   `edit` — Apply targeted string replacements to existing files, producing unified diffs. [internal/agent/tools/edit.go44-46](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L44-L46)
     [internal/agent/tools/edit.go101-106](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L101-L106)
    
*   `multiedit` — Batch editing of a single file with sequential operations, allowing partial success reporting. [internal/agent/tools/multiedit.go55-59](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L55-L59)
     [internal/agent/tools/multiedit.go151-164](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L151-L164)
    
*   `write` — Create new files or overwrite existing ones entirely. [internal/agent/tools/write.go44-46](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/write.go#L44-L46)
    
*   `ls` — List directory contents with tree visualization and depth control. [internal/agent/tools/ls.go52-54](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L52-L54)
     [internal/agent/tools/ls.go138-141](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L138-L141)
    

These tools integrate with the permission system to ensure user approval for file modifications and use `internal/diff` for transparent change visibility.

View Tool
---------

The `view` tool provides read access to file contents. It includes specialized logic for handling images, large files, and LSP diagnostics.

### Implementation Details

*   **Line Ranges:** Handles `Offset` (0-based line start) and `Limit` (number of lines). [internal/agent/tools/view.go47-51](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L47-L51)
    
*   **Image Support:** Detects image extensions and returns base64 encoded data if the model supports it. [internal/agent/tools/view.go203-218](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L203-L218)
    
*   **LSP Integration:** Ensures LSP servers are aware of the file via `openInLSPs` and waits for diagnostics via `waitForLSPDiagnostics`. [internal/agent/tools/view.go252-253](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L252-L253)
     [internal/agent/tools/diagnostics.go45-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/diagnostics.go#L45-L62)
     [internal/agent/tools/diagnostics.go67-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/diagnostics.go#L67-L87)
    
*   **Safety:** Enforces a `MaxViewSize` (200KB) for the returned section and checks if the file is outside the working directory. [internal/agent/tools/view.go76](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L76-L76)
     [internal/agent/tools/view.go127-136](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L127-L136)
    
*   **Tracking:** Records the read operation in `filetracker.Service` to enable staleness checks during subsequent edits. [internal/agent/tools/view.go264](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L264-L264)
    

**Sources:** [internal/agent/tools/view.go1-265](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/view.go#L1-L265)
 [internal/agent/tools/diagnostics.go45-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/diagnostics.go#L45-L87)

Edit and Multi-Edit Tools
-------------------------

These tools apply targeted changes using string substitution. They are designed to be "safe" by requiring the agent to provide the exact `old_string` to be replaced.

### The Edit Flow

1.  **Read & Validate:** Ensures the `OldString` exists. If it appears multiple times and `ReplaceAll` is false, it returns `oldStringMultipleMatchesErr`. [internal/agent/tools/edit.go47-49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L47-L49)
     [internal/agent/tools/edit.go232-241](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L232-L241)
    
2.  **Staleness Check:** Compares the file's `ModTime` against the `LastReadTime` from `filetracker`. If the file was changed manually since the agent last read it, the edit is rejected. [internal/agent/tools/edit.go215-223](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L215-L223)
    
3.  **Diff Generation:** Uses `diff.GenerateDiff` to create a unified diff and calculate `additions` and `removals`. [internal/agent/tools/edit.go261-265](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L261-L265)
     [internal/diff/diff.go10-29](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/diff/diff.go#L10-L29)
    
4.  **Permissions:** Calls `permissions.Request`. The UI displays the generated diff to the user. [internal/agent/tools/edit.go267-287](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L267-L287)
    
5.  **Persistence:** Writes to disk and creates a new entry in the `history.Service`. [internal/agent/tools/edit.go292-307](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L292-L307)
    

### Multi-Edit Batching

`multiedit` allows sequential edits on a single file. It validates all edits via `validateEdits` before applying any. [internal/agent/tools/multiedit.go82-84](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L82-L84)
 [internal/agent/tools/multiedit.go117-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L117-L125)

*   **Sequential Application:** Edits are applied in the provided order; each edit operates on the result of the previous one. [internal/agent/tools/multiedit.go152-164](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L152-L164)
    
*   **Partial Success:** If some edits fail, successful edits are still applied. The tool returns `MultiEditResponseMetadata` containing `EditsApplied` and `EditsFailed`. [internal/agent/tools/multiedit.go46-53](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L46-L53)
     [internal/agent/tools/multiedit.go175-181](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L175-L181)
    
*   **File Creation:** If the first edit has an empty `OldString`, it treats the operation as a file creation. [internal/agent/tools/multiedit.go128-132](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L128-L132)
    

**Sources:** [internal/agent/tools/edit.go1-309](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L1-L309)
 [internal/agent/tools/multiedit.go1-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/multiedit.go#L1-L125)
 [internal/diff/diff.go1-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/diff/diff.go#L1-L30)

Write and LS Tools
------------------

### Write Tool

The `write` tool is used for full-file creation or replacement. It performs a staleness check and records the change in file history. It automatically creates parent directories using `os.MkdirAll`. [internal/agent/tools/write.go76-79](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/write.go#L76-L79)
 [internal/agent/tools/write.go90-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/write.go#L90-L92)
 [internal/agent/tools/write.go137-156](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/write.go#L137-L156)

### LS Tool

The `ls` tool lists directory contents. It uses `fsext.ListDirectory` to handle `.gitignore` awareness and depth limits. [internal/agent/tools/ls.go138-153](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L138-L153)

*   **Tree Visualization:** Converts flat file lists into a hierarchical `TreeNode` structure for rendering. [internal/agent/tools/ls.go159-168](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L159-L168)
     [internal/agent/tools/ls.go171-233](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L171-L233)
    
*   **Limits:** Enforces a default limit of 1000 files to prevent context window overflow. [internal/agent/tools/ls.go53](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L53-L53)
     [internal/agent/tools/ls.go144-145](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/ls.go#L144-L145)

Diff and Diagnostics Integration
--------------------------------

### Unified Diff Generation

The `internal/diff` package provides a wrapper around `go-udiff`. It returns the diff string along with metadata about additions and removals. [internal/diff/diff.go10-29](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/diff/diff.go#L10-L29)

### LSP Diagnostics

After any modification (`edit`, `multiedit`, `write`), the tools call `notifyLSPs`. This triggers a `textDocument/didChange` notification to running LSP servers and waits for updated diagnostics. [internal/agent/tools/edit.go101-106](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/edit.go#L101-L106)
 [internal/agent/tools/write.go160-164](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/write.go#L160-L164)
