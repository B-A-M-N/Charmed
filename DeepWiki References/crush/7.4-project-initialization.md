Purpose and Scope
-----------------

This document covers Crush's project initialization system, which automatically detects when a new project needs onboarding and provides the mechanism for loading specialized domain knowledge via **Agent Skills**. This system ensures users have a smooth first-run experience while avoiding repeated prompts for projects that have already been initialized.

For general configuration loading and merging, see **3.2 Configuration System**. For setting up AI providers and models, see **2.4 Provider Setup**.

* * *

Overview
--------

Project initialization in Crush serves three primary purposes:

1.  **First-run detection**: Determines whether a project has been used with Crush before by checking for a persistent flag file via `MarkProjectInitialized` [internal/ui/model/onboarding.go18-29](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/onboarding.go#L18-L29)
    
2.  **Context file creation**: Guides the agent to examine the project and write a context file (defined by the `initialize_as` option, default: `AGENTS.md`) containing project-specific conventions, build systems, and patterns [internal/agent/templates/initialize.md.tpl1-5](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/templates/initialize.md.tpl#L1-L5)
    
3.  **Skill Discovery**: Discovers and validates specialized instructions formatted according to the `agentskills.io` standard [internal/skills/skills.go1-3](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L1-L3)
    

The system is non-intrusive: it only prompts when genuinely needed and can be triggered manually via `ActionInitializeProject` from the commands palette [internal/ui/dialog/actions.go57](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/actions.go#L57-L57)

### Initialization Prompting

When a project is identified as needing initialization, the agent uses the `internal/agent/templates/initialize.md.tpl` template. This template instructs the agent to:

1.  Check if the directory is empty or contains only config files. If so, it stops and prompts the user to add source code [internal/agent/templates/initialize.md.tpl3](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/templates/initialize.md.tpl#L3-L3)
    
2.  Discover project types, build commands, and architecture patterns by reading representative source files [internal/agent/templates/initialize.md.tpl7-14](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/templates/initialize.md.tpl#L7-L14)
    
3.  Create or update the file specified in `Config.Options.InitializeAs` [internal/agent/templates/initialize.md.tpl1](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/templates/initialize.md.tpl#L1-L1)

Agent Skills (agentskills.io)
-----------------------------

Crush implements the **Agent Skills** open standard to allow users and the community to provide structured domain-specific instructions to the AI.

### SKILL.md Format

A skill is defined by a `SKILL.md` file containing YAML frontmatter and Markdown instructions [internal/skills/skills.go22-23](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L22-L23)

| Field | Description | Constraints |
| --- | --- | --- |
| `name` | Unique identifier for the skill | Max 64 chars, alphanumeric/hyphens [internal/skills/skills.go24-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L24-L30) |
| `description` | Summary of what the skill provides | Max 1024 chars [internal/skills/skills.go25](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L25-L25) |
| `user-invocable` | If true, appears in command palette | Boolean [internal/skills/skills.go41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L41-L41) |
| `compatibility` | Runtime or tool requirements | Max 500 chars [internal/skills/skills.go26](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L26-L26) |
| `instructions` | The actual Markdown content | Extracted from body [internal/skills/skills.go178](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L178-L178) |

### Discovery and Validation

Skills are discovered through a recursive scan of specific paths [internal/skills/skills.go213-221](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L213-L221)

1.  **Discovery Paths**: Skills are loaded from `Options.SkillsPaths`. Default paths include `.agents/skills`, `.crush/skills`, `.claude/skills`, and `.cursor/skills` [internal/skills/builtin/crush-config/SKILL.md208-210](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/builtin/crush-config/SKILL.md?plain=1#L208-L210)
    
2.  **Validation**: `Skill.Validate()` ensures the name matches the directory name and lengths are within limits [internal/skills/skills.go118-146](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/skills/skills.go#L118-L146)
    
3.  **Invocable Skills**: Skills marked `user-invocable: true` are loaded as `CustomCommand` objects with prefixes `user:` (global) or `project:` (local) [internal/commands/commands.go64-86](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/commands/commands.go#L64-L86)

TUI Implementation
------------------

### Onboarding View

The onboarding flow is rendered via `initializeView`, which displays a prompt asking the user if they want to initialize the current working directory [internal/ui/model/onboarding.go82-117](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/onboarding.go#L82-L117)
 It uses the `Initialize` keymap for navigation (Yes/No/Switch) [internal/ui/model/keys.go52-57](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/keys.go#L52-L57)

### Landing Page

After initialization (or skipping), the `landingView` renders the project state [internal/ui/model/landing.go24-57](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/landing.go#L24-L57)
 This includes:

*   **Model Info**: Current large model being used [internal/ui/model/landing.go14-20](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/landing.go#L14-L20)
    
*   **Skills Info**: Status and list of discovered skills [internal/ui/model/landing.go46](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/landing.go#L46-L46)
    
*   **LSP/MCP Status**: Active language servers and MCP tools [internal/ui/model/landing.go44-45](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/landing.go#L44-L45)
    

### Pills and Tasks

Active agent tasks are rendered as "Pills" at the bottom of the TUI [internal/ui/model/pills.go1-12](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L1-L12)

*   **Todo Pill**: Shows progress of todos (e.g., "1/3") and the current in-progress task [internal/ui/model/pills.go70-111](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L70-L111)
    
*   **Queue Pill**: Displays the number of queued prompts [internal/ui/model/pills.go55-67](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L55-L67)
