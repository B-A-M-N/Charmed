This document explains how Crush achieves cross-platform compatibility across multiple operating systems and architectures. It covers the POSIX shell emulation strategy, build system configuration, Windows-specific considerations, and distribution mechanisms.

Overview
--------

Crush supports execution on seven operating systems (Linux, macOS, Windows, FreeBSD, OpenBSD, NetBSD, and Android) across multiple architectures (amd64, arm64, 386, arm) [.goreleaser.yml48-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L48-L62)
 The core cross-platform strategy is **POSIX shell emulation on all platforms**, including Windows, combined with a **pure-Go build strategy** (`CGO_ENABLED=0`) to ensure maximum portability without external library dependencies [.goreleaser.yml46](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L46-L46)

POSIX Shell Emulation Architecture
----------------------------------

Crush uses the `mvdan.cc/sh/v3` interpreter to provide POSIX shell emulation on all platforms [internal/shell/shell.go7-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L7-L9)
 On Windows, this eliminates the need for native shell implementations (`cmd.exe` or PowerShell), ensuring that AI-generated scripts behave consistently regardless of the host OS.

### Windows Compatibility Features

The shell implementation explicitly instructs that commands should use forward slashes (/) as path separators to work correctly on all platforms, including Windows [internal/shell/shell.go8-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L8-L9)
 The `Shell` struct maintains an internal environment and working directory to persist state between executions [internal/shell/shell.go66-67](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L66-L67)

### Shell Environment Markers

Crush unconditionally sets specific environment variables on every shell it spawns to allow tools to detect AI agent invocation:

*   `CRUSH=1`
*   `AGENT=crush`
*   `AI_AGENT=crush`

These markers are applied both to the interactive bash tool and the hook runner [internal/shell/shell.go37-49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L37-L49)

File System and Path Normalization
----------------------------------

Cross-platform file handling is managed by the `fsext` package, which addresses differences in path separators, line endings, and metadata.

### Path and Drive Handling

Crush normalizes patterns to forward slashes on Windows so that configurations can use backslashes while the underlying globbing logic remains consistent [internal/fsext/fileutil.go98-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L98-L100)

### Line-Ending Normalization

The `fsext` package provides utilities for converting between CRLF (Windows) and LF (Unix) line endings to ensure the AI agent can process files consistently across environments:

| `ToUnixLineEndings` | Converts `\r\n` to `\n` | [internal/fsext/fileutil.go224-229](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L224-L229) |
| `ToWindowsLineEndings` | Converts `\n` to `\r\n` | [internal/fsext/fileutil.go231-237](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L231-L237) |

 It provides `Short` and `Long` functions to toggle between tilde (`~`) notation and absolute paths [internal/home/home.go34-47](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/home/home.go#L34-L47)

Build Strategy: CGO\_ENABLED=0
------------------------------

Crush uses a strictly pure-Go build strategy to ensure maximum portability and ease of installation.

**Pure-Go Build and Distribution Strategy**

### Advantages of CGO\_ENABLED=0

1.  **No System Dependencies**: The resulting binary does not depend on `glibc` or other system-level C libraries [.goreleaser.yml46](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L46-L46)
    
2.  **Simplified Distribution**: Allows for easy cross-compilation from any host to any target (e.g., building Windows binaries on Linux).
3.  **Static Linking**: All dependencies, including the shell interpreter, are compiled into a single executable.

Hook System Cross-Platform Execution
------------------------------------

The hook system executes user-defined scripts using the same embedded POSIX shell used by the agent tools [internal/hooks/runner.go158-160](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L158-L160)

*   **Execution Context**: Hooks are run via `shell.Run`, providing a consistent environment across platforms [internal/hooks/runner.go180](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L180-L180)
    
*   **Timeout Handling**: The runner implements an `abandonGrace` period (1 second) to handle hooks that do not yield after a context cancellation, preventing the application from hanging on unresponsive platform-specific commands [internal/hooks/runner.go20-21](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L20-L21)
     [internal/hooks/runner.go198-208](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L198-L208)
    
*   **Exit Code Semantics**: The runner translates standard exit codes (e.g., 2 for Deny/Block) into internal `Decision` types [internal/hooks/runner.go221-223](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L221-L223)

TUI and Terminal Compatibility
------------------------------

While primarily handled by the `styles` package, terminal interaction adapts to OS-specific requirements:

*   **Windows Input**: Specific handlers for Windows-specific terminal events are located in `internal/cmd/root_windows.go` and `internal/client/dial_windows.go`.
*   **Server Communication**: On Windows, the client connects to the backend using named pipes instead of Unix sockets [internal/client/dial\_windows.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/client/dial_windows.go)
    
*   **Clipboard Support**: Support varies by platform; Crush detects the available clipboard provider (e.g., `pbcopy` on macOS, `xclip`/`wl-copy` on Linux, or internal Windows APIs).
