The User Interface in Crush is built using the **Bubble Tea** framework, following a "smart main model, components" architecture. The `UI` struct acts as the central orchestrator, managing state transitions, focus, and message routing, while sub-components (chat, editor, dialogs) handle specific rendering tasks and expose methods for the main model to call.

UI Model Structure
------------------

The `UI` struct is the root of the TUI hierarchy. It maintains the application's lifecycle state, layout calculations, and references to all sub-components.

### Core UI Struct

The `UI` struct manages several categories of state:

*   **Application State (`uiState`)**: Determines if the user is in the onboarding flow, project initialization, landing screen, or active chat. [internal/ui/model/ui.go102-110](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L102-L110)
    
*   **Focus State (`uiFocusState`)**: Controls which component (Editor or Main Chat) receives keyboard input. [internal/ui/model/ui.go92-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L92-L100)
    
*   **Layout & Sizing**: Tracks terminal dimensions (`width`, `height`) and calculates responsive layout rectangles via the `uiLayout` struct. [internal/ui/model/ui.go179-182](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L179-L182)
    
*   **Component References**: Holds instances of `Chat`, `textarea.Model`, `attachments.Attachments`, `header`, and the `dialog.Overlay`. [internal/ui/model/ui.go191-224](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L191-L224)

Lifecycle: Init, Update, and View
---------------------------------

The `UI` model implements the standard Bubble Tea interface: `Init`, `Update`, and `View`.

### 1\. Initialization (`Init`)

The `Init` function triggers initial commands such as querying terminal capabilities, starting spinners, and loading the initial session. [internal/ui/model/ui.go318-331](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L318-L331)

### 2\. Message Routing (`Update`)

The `Update` function is the central dispatcher. It processes incoming `tea.Msg` instances and routes them based on priority:

1.  **Dialogs**: If a dialog is open in the `Overlay`, it intercepts messages first via `m.dialog.HandleMsg(msg)`. [internal/ui/model/ui.go384-406](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L384-L406)
    
2.  **Global Keys**: Handles quit (`Ctrl+C`), help (`Ctrl+G`), and global toggle commands like `Models` (`Ctrl+L`) or `Sessions` (`Ctrl+S`). [internal/ui/model/ui.go580-620](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L580-L620)
     [internal/ui/model/keys.go69-99](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/keys.go#L69-L99)
    
3.  **Focus-Based Routing**: If the editor is focused (`uiFocusEditor`), keys are sent to the `textarea`. If the chat is focused (`uiFocusMain`), keys handle scrolling and selection. [internal/ui/model/ui.go622-754](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L622-L754)
    

### 3\. Rendering (`View`)

The `View` function composes the final string for the terminal. It uses `lipgloss` for styling and layout, joining components like the header, chat viewport, and editor. It handles both compact and standard layout modes, rendering specialized views for onboarding if the state is `uiOnboarding`. [internal/ui/model/ui.go1130-1178](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L1130-L1178)

Component Coordination
----------------------

The `UI` model forwards events to sub-components by calling their specific methods or passing the message down.

### Sub-Components and Their Roles

*   **Chat**: Manages the message history list, viewport scrolling, and follow-mode. [internal/ui/model/ui.go224](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L224-L224)
    
*   **Editor (`textarea`)**: Handles multi-line user input and history navigation. [internal/ui/model/ui.go208](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L208-L208)
    
*   **Attachments**: Manages the list of files/images attached to the current prompt. [internal/ui/model/ui.go211](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L211-L211)
    
*   **Header**: Displays the current session title, model information, and context window usage. [internal/ui/model/ui.go197](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L197-L197)
     [internal/ui/model/header.go25-33](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/header.go#L25-L33)
    
*   **Pills**: Displays active todos and the prompt queue status at the bottom of the chat. [internal/ui/model/pills.go209-250](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L209-L250)
    
*   **Dialog Overlay**: Manages a stack of modal dialogs using the `dialog.Overlay` struct. [internal/ui/model/ui.go191](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L191-L191)
    

### Bridging Backend Events

The TUI stays in sync with backend services (LSP, MCP, Agent) via a PubSub system. The `app.Subscribe` function bridges backend events into the Bubble Tea program.

1.  **Subscription**: The application subscribes to the event bus, forwarding internal events to the Bubble Tea program's message loop.
2.  **Event Mapping**: Backend events (e.g., `session.Updated`, `message.Created`) are received as typed messages. [internal/ui/model/ui.go473-545](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L473-L545)
    
3.  **TUI Consumption**: The `UI.Update` method receives these events (like `pubsub.Event[session.Session]`) and updates its internal state, such as updating the current session or appending messages to the chat view. [internal/ui/model/ui.go480-495](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L480-L495)

Dialog System
-------------

The `dialog.Overlay` manages a stack of components implementing the `Dialog` interface. [internal/ui/model/ui.go191](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L191-L191)

### Key Dialog Entities

*   **Commands**: Palette for system, user, and MCP commands. [internal/ui/model/ui.go247-248](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L247-L248)
    
*   **Models**: Selection for Large/Small task models. [internal/ui/model/keys.go83-86](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/keys.go#L83-L86)
    
*   **Sessions**: Management of conversation history. [internal/ui/model/keys.go91-94](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/keys.go#L91-L94)
    
*   **API Key Input**: Secure entry for provider credentials.

Dialogs communicate back to the `UI` model via `Action` types returned from `HandleMsg`.

Styling and Theme System
------------------------

Crush uses a centralized `Styles` struct defined in `internal/ui/styles/styles.go`. This struct contains `lipgloss` styles for every UI element.

*   **Header Styles**: Defines styles for diagonals, context percentages, and Hypercredit counts. [internal/ui/styles/styles.go64-77](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/styles.go#L64-L77)
    
*   **Component Styles**: Specific styles for `Editor`, `Pills`, `LSP`, and `Resource` lists. [internal/ui/styles/styles.go114-217](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/styles.go#L114-L217)
    
*   **Icons**: Standardized icons for statuses (e.g., `CheckIcon`, `LSPErrorIcon`, `TodoInProgressIcon`). [internal/ui/styles/styles.go19-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/styles.go#L19-L55)
    
*   **Model Info**: Renders token counts, costs, and reasoning effort levels. [internal/ui/common/elements.go44-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/elements.go#L44-L92)
