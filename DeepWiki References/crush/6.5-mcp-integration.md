This page documents how crush connects to Model Context Protocol (MCP) servers: the configuration types, the client lifecycle state machine, transport creation, the event broker, tool and prompt retrieval, and how MCP tools are surfaced to the agent as `fantasy.AgentTool` implementations. For LSP integration (a separate extensibility subsystem), see [LSP Integration](https://deepwiki.com/charmbracelet/crush/6.4-lsp-integration)
. For the permission system that gates MCP tool calls, see [Permission System](https://deepwiki.com/charmbracelet/crush/3.6-permission-system)
. For agent tool architecture in general, see [Tool Architecture](https://deepwiki.com/charmbracelet/crush/4.6-tool-architecture)
.

* * *

Overview
--------

MCP servers extend the agent with capabilities beyond the built-in tools. Each named entry in the `mcp` section of `crush.json` becomes one managed client. Crush connects to all enabled servers at startup, discovers their tools, prompts, and resources, and makes each item available to the agent under a namespaced name.

**Configuration source:** [internal/config/config.go166-186](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L166-L186)
 (referenced via `config.MCPConfig`)

| Field | Type | Purpose |
| --- | --- | --- |
| `type` | `MCPType` | Transport: `stdio`, `sse`, or `http` |
| `command` | `string` | Executable for `stdio` servers |
| `args` | `[]string` | Arguments for `stdio` command |
| `env` | `map[string]string` | Environment variables (supports shell expansion) |
| `url` | `string` | Endpoint for `http`/`sse` servers |
| `headers` | `map[string]string` | HTTP headers (supports shell expansion) |
| `timeout` | `int` | Connection timeout in seconds (default: 15) |
| `disabled` | `bool` | Skip this server entirely |
| `enabled_tools` | `[]string` | Allow list of tool names to include |
| `disabled_tools` | `[]string` | Deny list of tool names to exclude |

Sources: [internal/config/config.go166-186](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L166-L186)
