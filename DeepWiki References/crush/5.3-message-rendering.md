This document describes how chat messages are rendered in the Crush TUI. It covers the message item type system, rendering pipeline, tool-specific renderers, and optimization strategies including caching and animation management. For information about the overall chat display and list management, see [Chat Display](https://deepwiki.com/charmbracelet/crush/5.2-chat-display)
. For styling configuration, see [Styling System](https://deepwiki.com/charmbracelet/crush/5.8-styling-system)
.

Overview
--------

The message rendering system transforms `message.Message` domain objects into visual representations in the terminal. The system supports three primary message roles:

*   **User messages**: User input with attachments rendered as markdown and support for skill invocation XML. [internal/ui/chat/user.go25-34](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/user.go#L25-L34)
    
*   **Assistant messages**: AI responses with thinking/reasoning content and text output, utilizing per-section caching for performance. [internal/ui/chat/assistant.go119-147](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/assistant.go#L119-L147)
    
*   **Tool messages**: Tool calls and results with tool-specific rendering logic via the `ToolRenderer` interface. [internal/ui/chat/tools.go45-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/tools.go#L45-L55)
    

All message items implement the `MessageItem` interface and support features like text highlighting, keyboard shortcuts, caching, and animations. [internal/ui/chat/messages.go53-57](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/messages.go#L53-L57)

Optimization & Caching
----------------------

### Multi-Level Caching

Crush employs several caching layers to maintain 60FPS in the TUI:

1.  **`cachedMessageItem`**: Stores the raw string for a specific width. [internal/ui/chat/messages.go172-192](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/messages.go#L172-L192)
    
2.  **`prefixedRendered`**: Caches the final styled string (with focus prefixes) to avoid splitting/joining lines on every frame. [internal/ui/chat/messages.go189-192](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/messages.go#L189-L192)
    
3.  **`streamingMarkdown`**: Used in `AssistantMessageItem` to cache a "stable prefix" of markdown rendering so that streaming only re-renders the partial trailing block. [internal/ui/chat/assistant.go141-146](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/assistant.go#L141-L146)
    

### Animation Logic

Animation is handled via `anim.Anim`. Items bump their `Versioned` counter on every animation tick to invalidate the parent list's memoization, ensuring the UI refreshes for each spinner frame.
