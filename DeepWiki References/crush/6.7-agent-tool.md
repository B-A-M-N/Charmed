Agent Tool
==========

The Agent Tool and Agentic Fetch Tool are specialized capabilities that allow a primary AI agent to delegate complex tasks or research to sub-agents. These tools are defined in `internal/agent/agent_tool.go` and `internal/agent/agentic_fetch_tool.go`. They facilitate a hierarchical agent structure where a "Parent" session can spawn "Child" sessions to perform isolated units of work.

* * *

Overview
--------

Crush implements two distinct agentic delegation tools:

1.  **Agent Tool (`agent`)**: Spawns a general-purpose sub-agent using the `config.AgentTask` configuration to perform a specific prompt or task [internal/agent/agent\_tool.go23-27](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent_tool.go#L23-L27)
    
2.  **Agentic Fetch Tool (`agentic_fetch`)**: Spawns a specialized sub-agent equipped with web search and retrieval tools to answer questions using online information [internal/agent/agentic\_fetch\_tool.go67-81](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L67-L81)
    

Both tools utilize the `runSubAgent` orchestration logic within the `coordinator` to manage session lifecycles, cost propagation, and result surfacing.

Implementation Details
----------------------

### The Agent Tool (`agent`)

The `agent` tool allows the model to offload tasks. It uses the `config.AgentTask` configuration [internal/agent/agent\_tool.go27-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent_tool.go#L27-L30)

*   **Prompting**: It uses a specific `taskPrompt` which is built with the current working directory [internal/agent/agent\_tool.go31-34](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent_tool.go#L31-L34)
    
*   **Execution**: It calls `runSubAgent` which handles the synchronous execution of the child agent [internal/agent/agent\_tool.go58-65](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent_tool.go#L58-L65)
     It requires a `SessionID` and `AgentMessageID` from the context to link the sub-session [internal/agent/agent\_tool.go48-56](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent_tool.go#L48-L56)
    

### The Agentic Fetch Tool (`agentic_fetch`)

This tool is designed for deep web research. Unlike the standard `fetch` tool, it spawns an agent that can reason about search results and iterate on findings.

*   **Modes**: It supports both direct URL analysis and general web searching [internal/agent/agentic\_fetch\_tool.go75-81](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L75-L81)
    
*   **URL Handling**: If a URL is provided, it fetches the content first using `tools.FetchURLAndConvert` [internal/agent/agentic\_fetch\_tool.go110-115](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L110-L115)
     If the content is large (exceeds `tools.LargeContentThreshold`), it saves it to a temporary file and instructs the sub-agent to use `view` and `grep` [internal/agent/agentic\_fetch\_tool.go117-135](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L117-L135)
    
*   **Tooling**: The sub-agent is provided with a specialized toolset including `NewWebFetchTool`, `NewWebSearchTool`, `NewGrepTool`, `NewGlobTool`, `NewSourcegraphTool`, and `NewViewTool` [internal/agent/agentic\_fetch\_tool.go165-174](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L165-L174)
    
*   **Model Selection**: It typically uses the "small" model for both large and small roles since research tasks often don't require the highest parameter count models [internal/agent/agentic\_fetch\_tool.go182-183](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L182-L183)
    
*   **Hook Interception**: Sub-agent tools run without hook interception to avoid redundant execution of user hooks for every inner tool call [internal/agent/agentic\_fetch\_tool.go176-179](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L176-L179)

 [internal/agent/agentic\_fetch\_tool.go53-199](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L53-L199)

Session and Message Relationship
--------------------------------

Sub-agents do not operate on the parent's message history directly. Instead, a new session is created to maintain isolation.

**Diagram: Entity Mapping**

| Entity | Code Reference | Description |
| --- | --- | --- |
| **Deterministic ID** | `CreateAgentToolSessionID` | Derived from Parent Message ID and Tool Call ID to ensure idempotency. |
| **Task Session** | `CreateTaskSession` | A session record where `ParentSessionID` is populated to maintain the hierarchy. |
| **Cost Sync** | `runSubAgent` | Logic that adds `childSession.Cost` to `parentSession.Cost` upon completion. |

**Sources:** [internal/agent/agent\_tool.go58-65](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent_tool.go#L58-L65)
 [internal/agent/agentic\_fetch\_tool.go193-200](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L193-L200)

Sub-Agent Configuration
-----------------------

The behavior of sub-agents is modified via the `NewSessionAgent` constructor.

*   **System Prompt**: Sub-agents receive a specialized system prompt generated via templates like `agentic_fetch_prompt.md.tpl` [internal/agent/agentic\_fetch\_tool.go50-51](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L50-L51)
    
*   **Permissions**: For `agentic_fetch`, the sub-agent is initialized with `IsYolo` based on whether the parent session has `SkipRequests` enabled [internal/agent/agentic\_fetch\_tool.go187](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L187-L187)
    
*   **Temporary Workspace**: Sub-agents often work in a temporary directory (e.g., for `agentic_fetch` results) which is cleaned up after the tool returns [internal/agent/agentic\_fetch\_tool.go102-106](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L102-L106)

Comparison with Standard Network Tools
--------------------------------------

The `agentic_fetch` tool is more powerful than the basic `fetch` or `download` tools.

| Tool | File Path | Behavior | Use Case |
| --- | --- | --- | --- |
| `fetch` | `internal/agent/tools/fetch.go` | Single HTTP GET, optional HTML-to-Markdown conversion [internal/agent/tools/fetch.go143-180](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L143-L180) | Quick retrieval of a known page. |
| `download` | `internal/agent/tools/download.go` | Streams a URL directly to a local file path [internal/agent/tools/download.go149-155](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/download.go#L149-L155) | Downloading binaries or large assets. |
| `agentic_fetch` | `internal/agent/agentic_fetch_tool.go` | Spawns a `SessionAgent` that can search, fetch, and grep through results [internal/agent/agentic\_fetch\_tool.go165-174](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L165-L174) | Complex research or multi-step information gathering. |
