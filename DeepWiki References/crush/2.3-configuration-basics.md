Crush works with no configuration file at all. When you do need to customize it, configuration is stored as JSON files at well-known locations that are merged together at startup.

This page covers:

*   Where config files live and how to override those paths.
*   How multiple files are merged and in what priority order.
*   The most commonly used configuration keys and variable resolution.

For a deep dive into the loading pipeline and the `Config` struct internals, see page 3.2. For provider-specific setup (API keys, OAuth, etc.), see page 2.4.

* * *

File Locations and Priority
---------------------------

Crush loads configuration from multiple locations, merging them with later sources taking precedence over earlier ones. The loading order is managed by the `Load` function in `internal/config/load.go` [internal/config/load.go35-129](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L35-L129)

| Priority | Location | Description | Environment Override |
| --- | --- | --- | --- |
| 1 (lowest) | Global config | `~/.config/crush/crush.json` | `CRUSH_GLOBAL_CONFIG` |
| 2   | Global data | `~/.local/share/crush/crush.json` (runtime state) | `CRUSH_GLOBAL_DATA` |
| 3   | Project configs | `crush.json` in current directory or parents | N/A |
| 4   | Project local | `.crush.json` in current directory or parents | N/A |
| 5 (highest) | Workspace data | `.crush/crush.json` (persistence for UI changes) | N/A |

### Config File Path Resolution

The loader determines the exact file paths based on environment variables and OS.

**Config file path resolution logic:**

*   If `CRUSH_GLOBAL_CONFIG` is set, it uses `$CRUSH_GLOBAL_CONFIG/crush.json`.
*   Otherwise, it respects `XDG_CONFIG_HOME` or defaults to `~/.config/crush/crush.json` on Unix.

**Data file path resolution logic:**

*   If `CRUSH_GLOBAL_DATA` is set, it uses `$CRUSH_GLOBAL_DATA/crush.json`.
*   Otherwise, it respects `XDG_DATA_HOME` or defaults to `~/.local/share/crush/crush.json` on Unix.

**Project config discovery**: starting from the working directory, the loader walks up the directory tree looking for `crush.json` and `.crush.json`. This climb is bounded by the Git worktree root; it will not adopt `crush.json` files found above a Git repository [internal/config/load\_test.go67-96](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load_test.go#L67-L96)

File Format
-----------

All config files are JSON objects. You can add a `$schema` field to enable editor auto-completion and validation. The schema is generated from the `Config` struct [schema.json54-106](https://github.com/charmbracelet/crush/blob/1ceeaa93/schema.json#L54-L106)

The schema is also available locally via `crush schema` [schema.json1-4](https://github.com/charmbracelet/crush/blob/1ceeaa93/schema.json#L1-L4)

How Configs Are Merged
----------------------

The `loadFromBytes` function (called via `loadFromConfigPaths`) uses `jsons.Merge` to deep-merge the list of loaded config files [internal/config/load.go38-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L38-L41)
 Later files override earlier ones. **Arrays are replaced entirely, not appended.**

Variable Resolution in Config Values
------------------------------------

String values in the config file are resolved through a `VariableResolver` [internal/config/load.go103-104](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L103-L104)
 Crush uses a `ShellVariableResolver` which supports environment variables and command substitution [internal/config/load.go103](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L103-L103)

| Pattern | Description | Example |
| --- | --- | --- |
| `$VAR` or `${VAR}` | Standard shell variable syntax | `$OPENAI_API_KEY` |
| `$(command)` | Shell command substitution | `$(echo $SECRET)` |

The `VariableResolver` is used in several contexts:

| Context | Where resolution happens |
| --- | --- |
| Provider `api_key` | `configureProviders()` at startup [internal/config/load.go111-113](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L111-L113) |
| Provider `extra_headers` | `configureProviders()` during header mapping [internal/config/load.go234-237](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L234-L237) |
| Provider `base_url` | Overridden via `Providers.Get()` [internal/config/load.go187-189](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L187-L189) |
