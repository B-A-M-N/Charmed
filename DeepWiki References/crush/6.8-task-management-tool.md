The Task Management Tool provides a structured way for the AI agent to track its objectives, progress, and pending operations within a session. It bridges the gap between high-level natural language goals and the discrete tool executions required to achieve them.

Overview
--------

The system consists of a backend tracking mechanism within the session state and a front-end "Pills" UI component. The agent uses the `todos` tool to manage a list of `TodoItem` objects, which are persisted in the session. These items are then rendered in the TUI to provide the user with real-time visibility into the agent's internal plan.

Task Data Structures
--------------------

Tasks are represented as `Todo` items within a session. Each item tracks its content, current status, and an "active form" used for display during execution.

### session.Todo

| Field | Type | Description |
| --- | --- | --- |
| `Content` | `string` | The primary description of the task (e.g., "Run tests"). [internal/ui/chat/todos.go162](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L162-L162) |
| `ActiveForm` | `string` | A present continuous version for display (e.g., "Running tests"). [internal/ui/chat/todos.go163-165](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L163-L165) |
| `Status` | `TodoStatus` | The current state: `pending`, `in_progress`, or `completed`. [internal/ui/chat/todos.go153-160](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L153-L160) |

### Status Transitions

The agent manages transitions by calling the `todos` tool. The UI reflects these states using different icons and styles:

*   **Pending**: Task not yet started. Uses `styles.TodoPendingIcon`. [internal/ui/chat/todos.go159-160](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L159-L160)
    
*   **In Progress**: Accompanied by a spinner (`todoSpinner`) or `styles.SpinnerIcon` in the pills UI. [internal/ui/model/pills.go234-237](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L234-L237)
    
*   **Completed**: Task finished successfully. Uses `styles.TodoCompletedIcon`. [internal/ui/chat/todos.go154-155](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L154-L155)

TUI Implementation: The Pills Component
---------------------------------------

The "Pills" component is a specialized UI element in `internal/ui/model/pills.go` that displays active tasks and the prompt queue at the bottom of the chat interface.

### Rendering Logic

The component calculates its height dynamically based on whether it is collapsed (showing only the summary pill) or expanded (showing the full list of tasks).

### Key Functions

*   `todoPill`: Renders the summary pill showing progress (e.g., "To-Do 2/5") and the currently active task name. It truncates tasks longer than `maxTaskDisplayLength` (40 characters). [internal/ui/model/pills.go70-111](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L70-L111)
     [internal/ui/model/pills.go26](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L26-L26)
    
*   `pillsAreaHeight`: Determines the vertical space required. If expanded, it adds the count of todos (or queue items) to the base pill height. [internal/ui/model/pills.go186-206](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L186-L206)
    
*   `togglePillsExpanded`: Handles the transition between the summary view and the detailed task list, triggered by `ctrl+t`. [internal/ui/model/pills.go138-162](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L138-L162)
    
*   `renderPills`: The main draw loop for the component, which handles padding, width calculation, and selection of sub-renderers based on focus state. [internal/ui/model/pills.go209-246](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/pills.go#L209-L246)
    
*   `FormatTodosList`: Formats the list of todos with appropriate status icons and handles truncation for the TUI width. [internal/ui/chat/todos.go139-173](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L139-L173)

Tool Rendering in Chat
----------------------

When the agent calls the `todos` tool, the results are rendered in the chat history via `TodosToolMessageItem`. This provides a historical record of plan updates.

*   **Metadata Usage**: If the tool call result includes metadata, the renderer uses it to show richer information like "created 5 todos" or "completed all". [internal/ui/chat/todos.go77-118](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L77-L118)
    
*   **Status Ratios**: Displays the progress as a ratio (e.g., 1/3) using `sty.Tool.TodoRatio`. [internal/ui/chat/todos.go70-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L70-L71)
    
*   **Compact Mode**: Supports a compact rendering for nested tool calls or reduced UI density. [internal/ui/chat/todos.go123-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/chat/todos.go#L123-L125)
