Purpose and Scope
-----------------

This document provides a comprehensive overview of Crush, a terminal-based AI assistant for software development. It covers the system's architecture, core components, data flow, and integration points. The overview is intended for developers who need to understand how the major subsystems fit together before diving into specific implementation details.

For detailed information about specific subsystems, refer to:

*   Installation and CLI usage: **Getting Started**
*   Configuration file format and options: **Configuration Basics** and **Configuration System**
*   Database schema and service layer: **Database and Services**
*   Agent execution and tool system: **Agent System** and **Tool System**
*   TUI components and interaction: **User Interface**
*   Build and deployment: **Development**

What is Crush?
--------------

Crush is a terminal-based AI coding assistant built in Go that provides direct access to your development environment. It combines a rich TUI (Terminal User Interface) built with [Bubble Tea](https://github.com/charmbracelet/crush/blob/1ceeaa93/Bubble%20Tea)
 multi-provider AI integration through the `fantasy` abstraction layer, and an extensive tool system for file manipulation, shell execution, LSP queries, and MCP resource access. The system maintains conversation history in a local SQLite database and supports both interactive and non-interactive modes [internal/cmd/root.go77-105](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L77-L105)

System Architecture
-------------------

### High-Level Component Diagram

**Description:** Crush follows a layered architecture with clear separation of concerns. The CLI entry point initializes the `ConfigStore` and `app.App` orchestrator [internal/cmd/root.go106-144](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L106-L144)
 The orchestrator manages all services and subsystems. The agent system handles AI interactions through the `fantasy.Agent` abstraction [internal/agent/agent.go206-211](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L206-L211)
 while the TUI subscribes to service events for real-time updates [internal/app/app.go109](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L109-L109)
 All persistent state is stored in SQLite via service abstractions [internal/app/app.go84-103](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L84-L103)

Data Flow
---------

### Message Processing Pipeline

**Description:** User input flows from the TUI to the `AgentCoordinator.Run` method [internal/agent/coordinator.go171](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L171-L171)
 The coordinator delegates to a `SessionAgent`, which uses `fantasy.NewAgent` to interact with LLM providers [internal/agent/agent.go206-211](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L206-L211)
 During execution, the agent may call various tools. These tool results are fed back into the model context until a final response is generated. All updates are broadcast via the `App.events` channel to the TUI [internal/app/app.go166-173](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L166-L173)

Summary
-------

Crush is a terminal-based AI assistant designed for software development workflows. It integrates:

1.  **Orchestration Layer**: `app.App` coordinates services like sessions, messages, and permissions [internal/app/app.go54-78](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L54-L78)
    
2.  **Agent Layer**: `agent.Coordinator` manages multiple `SessionAgent` instances that execute the LLM loop [internal/agent/coordinator.go89-108](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L89-L108)
    
3.  **Tool System**: Extensible tools (Shell, LSP, MCP) allow the AI to interact with the local environment [internal/agent/agent.go201-204](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L201-L204)
    
4.  **Interactive TUI**: Built with Bubble Tea v2, providing a real-time chat interface with rich rendering [internal/cmd/root.go130-135](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L130-L135)
    
5.  **Non-Interactive Mode**: Supports automation and piping via `crush run` [internal/app/app.go211-236](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L211-L236)
