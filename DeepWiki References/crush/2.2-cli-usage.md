This page documents every command and flag exposed by the `crush` binary. It covers the interactive root command, the `crush run` non-interactive subcommand, and utility subcommands for session management, statistics, and system inspection.

Global Flags
------------

These flags are declared as `PersistentFlags` on `rootCmd` and are available to every subcommand.

| Flag | Short | Type | Default | Description |
| --- | --- | --- | --- | --- |
| `--cwd` | `-c` | string | `""` | Override the current working directory |
| `--data-dir` | `-D` | string | `""` | Override the crush data directory (where the DB and logs live) |
| `--debug` | `-d` | bool | `false` | Enable debug logging |
| `--host` | `-H` | string | `server.DefaultHost()` | Connect to a specific crush server host |

Sources: [internal/cmd/root.go53-56](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L53-L56)

### Root-Only Flags

| Flag | Short | Type | Default | Description |
| --- | --- | --- | --- | --- |
| `--yolo` | `-y` | bool | `false` | Skip all permission prompts (auto-approve everything) |
| `--session` | `-s` | string | `""` | Continue a previous session by ID |
| `--continue` | `-C` | bool | `false` | Continue the most recent session |

The `--yolo` flag is passed into the application state to bypass user confirmation for tool execution [internal/cmd/root.go58](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L58-L58)

`crush run` — Non-Interactive Mode
----------------------------------

Sends a single prompt to the agent and prints the response to stdout. The prompt can be provided as CLI arguments or piped from stdin via `MaybePrependStdin` [internal/cmd/run.go78-82](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/run.go#L78-L82)

**Execution Logic:** If `CRUSH_CLIENT_SERVER` is enabled, it uses `runNonInteractive` to stream events from a remote server [internal/cmd/run.go158-166](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/run.go#L158-L166)
 Otherwise, it calls `appWs.App().RunNonInteractive` locally [internal/cmd/run.go142](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/run.go#L142-L142)

Sources: [internal/cmd/run.go30-61](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/run.go#L30-L61)
 [internal/cmd/run.go97-123](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/run.go#L97-L123)

### `run` Flags

| Flag | Short | Type | Default | Description |
| --- | --- | --- | --- | --- |
| `--quiet` | `-q` | bool | `false` | Hide the spinner during execution |
| `--verbose` | `-v` | bool | `false` | Show logs on stderr |
| `--model` | `-m` | string | `""` | Specify large model (e.g., `openai/gpt-4o`) |

`crush session` — Session Management
------------------------------------

Manage conversation history. Commands support `--json` for automation.

| Subcommand | Description |
| --- | --- |
| `list` | Lists all sessions with ID and Title. |
| `show <id>` | Displays full message history for a specific session. |
| `last` | Shows the most recent session. |
| `delete <id>` | Deletes a session from the database. |
| `rename <id> <t>` | Updates the session title. |

**Implementation:** These subcommands interact with `session.Service` and `message.Service` [internal/cmd/session.go101-105](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/session.go#L101-L105)
 The `resolveSessionID` function handles resolving short hashes or UUIDs to full session entities by listing sessions and checking prefixes [internal/cmd/session.go217-242](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/session.go#L217-L242)

Sources: [internal/cmd/session.go33-98](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/session.go#L33-L98)
 [internal/cmd/session.go217-242](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/session.go#L217-L242)

`crush stats` — Usage Analytics
-------------------------------

Generates a local HTML dashboard of usage statistics including token usage, costs, and activity patterns.

*   **Data Gathering:** Queries the SQLite database via `db.Queries` [internal/cmd/stats.go181-237](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L181-L237)
    
*   **Output:** Generates an HTML file in the data directory and opens it in the default browser [internal/cmd/stats.go166-176](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L166-L176)
    
*   **Assets:** Embeds HTML, CSS, and JS templates [internal/cmd/stats.go24-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L24-L41)

Utility Commands
----------------

### `crush login`

Authenticates with external platforms like Charm Hyper or GitHub Copilot.

*   `crush login hyper`: Performs OAuth device flow via `hyper.InitiateDeviceAuth` [internal/cmd/login.go72-132](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/login.go#L72-L132)
    
*   `crush login copilot`: Refreshes or requests GitHub tokens [internal/cmd/login.go134-200](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/login.go#L134-L200)
    

### `crush dirs`

Displays system paths for configuration and data storage.

*   `crush dirs config`: Prints the global configuration directory [internal/cmd/dirs.go27-42](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/dirs.go#L27-L42)
    
*   `crush dirs data`: Prints the data storage directory [internal/cmd/dirs.go44-59](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/dirs.go#L44-L59)
    

### `crush schema` (Hidden)

Generates the JSON schema for `crush.json` using `jsonschema.Reflector` on the `config.Config` struct [internal/cmd/schema.go12-26](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/schema.go#L12-L26)
 This is used by CI to auto-update `schema.json` [.github/workflows/schema-update.yml21](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/schema-update.yml#L21-L21)

Telemetry and Logs
------------------

### `crush logs`

Tail the application logs. It supports `--follow` and `--tail N` flags [internal/cmd/logs.go74-75](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/logs.go#L74-L75)

*   **File Location:** Logs are stored in the data directory as `crush.log` [internal/cmd/logs.go58](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/logs.go#L58-L58)
    
*   **Formatting:** Parsed JSON logs are re-rendered using `charm.land/log/v2` via `printLogLine` [internal/cmd/logs.go167-216](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/logs.go#L167-L216)
    

### Telemetry (PostHog)

Crush initializes telemetry if metrics are enabled in the configuration [internal/cmd/stats.go118-120](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L118-L120)

*   **Events:** Tracks session listings [internal/cmd/session.go145](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/session.go#L145-L145)
     stats viewing [internal/cmd/stats.go138](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L138-L138)
     and application initialization [internal/cmd/root.go124](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L124-L124)
    
*   **Implementation:** The `internal/event` package handles PostHog client initialization and event queuing [internal/event/event.go50-93](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/event/event.go#L50-L93)
