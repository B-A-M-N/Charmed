This page covers how crush routes LLM calls through the `charm.land/fantasy` abstraction library, how `config.ProviderConfig` entries are mapped to live provider connections, and how the Catwalk model registry supplies provider and model metadata. For user-facing configuration (API keys, environment variables, OAuth token flow), see [Provider Setup](https://deepwiki.com/charmbracelet/crush/2.4-provider-setup)
. For how selected models are resolved and per-model parameters applied, see [Model Configuration](https://deepwiki.com/charmbracelet/crush/4.3-model-configuration)
.

* * *

Abstraction Layers
------------------

Two libraries handle provider concerns:

| Library | Module Path | Role |
| --- | --- | --- |
| **Catwalk** | `charm.land/catwalk/pkg/catwalk` | Model registry — provider/model descriptors, cost tracking, context windows, and capability flags. |
| **Fantasy** | `charm.land/fantasy` | Provider abstraction — uniform `Provider` and `LanguageModel` interfaces over all LLM backends. |

The `coordinator` in `internal/agent/coordinator.go` bridges them: it reads configuration, creates `fantasy.Provider` instances, and packages the resulting `fantasy.LanguageModel` together with Catwalk metadata into a `Model` struct. [internal/agent/agent.go99-103](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L99-L103)

Sources: [internal/agent/agent.go99-103](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L99-L103)
 [internal/agent/coordinator.go76-85](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L76-L85)

Supported Backends
------------------

`buildProvider` (in `internal/agent/coordinator.go`) dispatches on `ProviderConfig.Type` to a dedicated builder function. Each builder accepts the resolved API key, base URL, custom headers, and debug flags, returning a `fantasy.Provider`.

| `ProviderConfig.Type` | Builder Function | Fantasy Provider Package |
| --- | --- | --- |
| `anthropic` | `buildAnthropicProvider` | `charm.land/fantasy/providers/anthropic` |
| `openai` | `buildOpenaiProvider` | `charm.land/fantasy/providers/openai` |
| `openrouter` | `buildOpenrouterProvider` | `charm.land/fantasy/providers/openrouter` |
| `vercel` | `buildVercelProvider` | `charm.land/fantasy/providers/vercel` |
| `azure` | `buildAzureProvider` | `charm.land/fantasy/providers/azure` |
| `bedrock` | `buildBedrockProvider` | `charm.land/fantasy/providers/bedrock` |
| `google` | `buildGoogleProvider` | `charm.land/fantasy/providers/google` |
| `google-vertex` | `buildGoogleVertexProvider` | `charm.land/fantasy/providers/google` |
| `openaicompat` | `buildOpenaiCompatProvider` | `charm.land/fantasy/providers/openaicompat` |
| `hyper` | `buildHyperProvider` | `internal/agent/hyper` |

### Special Cases: Hyper

The `hyper` provider type is handled by the `internal/agent/hyper` package. It implements the `fantasy.Provider` and `fantasy.LanguageModel` interfaces by proxying requests to the Hyper service API. [internal/agent/hyper/provider.go1-2](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.go#L1-L2)

*   **Embedded Data**: The package embeds a `provider.json` file containing default model definitions (e.g., `kimi-k2.6`, `deepseek-v4-flash`) and costs. [internal/agent/hyper/provider.go21-22](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.go#L21-L22)
     [internal/agent/hyper/provider.json1-10](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.json#L1-L10)
    
*   **Base URL**: The default endpoint is `https://hyper.charm.land`, which can be overridden via the `HYPER_URL` environment variable. [internal/agent/hyper/provider.go45-48](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.go#L45-L48)
    
*   **Credits**: The provider supports checking user balance via `FetchCredits` which queries the `/v1/credits` endpoint. [internal/agent/hyper/provider.go52-83](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.go#L52-L83)

Provider Initialization Flow
----------------------------

Both a large model and a small model are initialized per agent. `buildAgentModels` performs this for each. `UpdateModels` is called at the start of every `coordinator.Run` to refresh model state.

**buildAgentModels → buildProvider → LanguageModel**

Sources: [internal/agent/coordinator.go494-575](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L494-L575)

Detailed steps:

1.  `NewCoordinator` calls `buildAgent` → `buildAgentModels`. [internal/agent/coordinator.go126-132](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L126-L132)
    
2.  `buildAgentModels` reads `cfg.Models[config.SelectedModelTypeLarge]` and `cfg.Models[config.SelectedModelTypeSmall]`. [internal/agent/coordinator.go496-511](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L496-L511)
    
3.  For each, it fetches the corresponding `ProviderConfig` via `cfg.Providers.Get`. [internal/agent/coordinator.go514-522](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L514-L522)
    
4.  `buildProvider` inspects `ProviderConfig.Type` and calls the matching `buildXxxProvider`. [internal/agent/coordinator.go577-600](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L577-L600)
    
5.  The builder constructs the `fantasy.Provider` with API key, base URL, and custom headers. [internal/agent/coordinator.go601-610](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L601-L610)
    
6.  `provider.LanguageModel(ctx, modelID)` returns the live `fantasy.LanguageModel`. [internal/agent/coordinator.go541-555](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L541-L555)
    
7.  The matching `catwalk.Model` entry is located inside `providerCfg.Models` and bundled into the returned `Model` struct. [internal/agent/coordinator.go560-573](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L560-L573)

### Cache Management

The `cache[T]` struct handles persistence of provider data to disk. [internal/config/provider.go199-220](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L199-L220)

*   **Pathing**: `cachePathFor` determines the storage location based on `XDG_DATA_HOME` or platform-specific defaults. On Windows, it uses `%LOCALAPPDATA%/crush/`. On Linux/macOS, it defaults to `$HOME/.local/share/crush/`. [internal/config/provider.go37-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L37-L55)
    
*   **Validation**: It uses `etag.Of(data)` to perform conditional updates against remote sources. [internal/config/provider.go213-219](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L213-L219)
    
*   **Manual Update**: Users can trigger a refresh via `crush update-providers`. This command calls `config.UpdateProviders` or `config.UpdateHyper` depending on the `--source` flag. [internal/cmd/update\_providers.go48-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/update_providers.go#L48-L55)
     [internal/config/provider.go58-123](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L58-L123)
    

### Update Mechanism

Crush checks for its own updates via the `internal/update` package, which queries the GitHub Releases API. [internal/update/update.go14-118](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/update/update.go#L14-L118)

*   **Version Comparison**: `Info.Available()` compares the current version against the latest tag from GitHub. It handles pre-release logic, returning `true` if a stable version is available for a pre-release install. [internal/update/update.go43-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/update/update.go#L43-L55)
    
*   **Check**: `update.Check(ctx, current, client)` returns an `Info` struct containing the current and latest version strings, stripping "v" prefixes for consistency. [internal/update/update.go58-73](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/update/update.go#L58-L73)
    
*   **Development Builds**: `IsDevelopment()` identifies versions installed via `go install` (matching `goInstallRegexp`) or built from dirty git trees. [internal/update/update.go29-35](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/update/update.go#L29-L35)
