This page documents how the crush TUI handles mouse events: the event types processed, the routing priority between dialogs and chat, coordinate translation, click-timing logic for highlight copy, focus switching, and scroll behavior.

For context on the broader TUI model and focus states, see page [5.1](https://github.com/charmbracelet/crush/blob/1ceeaa93/5.1)
 For the chat component that receives the translated mouse coordinates, see page [5.2](https://github.com/charmbracelet/crush/blob/1ceeaa93/5.2)

* * *

Overview
--------

Mouse handling is orchestrated within `UI.Update()` in [internal/ui/model/ui.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go)
 The `UI` struct intercepts four Bubble Tea mouse message types — `tea.MouseClickMsg`, `tea.MouseMotionMsg`, `tea.MouseReleaseMsg`, and `tea.MouseWheelMsg` — plus the internally-defined `DelayedClickMsg` [internal/ui/model/chat.go27-31](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L27-L31)
 All raw mouse types follow a strict priority rule: **active dialogs via the Overlay absorb mouse events first** [internal/ui/model/ui.go571-574](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L571-L574)
; the chat area only receives events when no dialog is open and the UI state is `uiChat` [internal/ui/model/ui.go582-583](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L582-L583)

Message Types and Their Handlers
--------------------------------

| Bubble Tea message | Condition guard | Primary action |
| --- | --- | --- |
| `tea.MouseClickMsg` | Not in dialog, state `uiChat` | `handleClickFocus`, then `Chat.HandleMouseDown(x, y)` |
| `tea.MouseMotionMsg` | Not in dialog, state `uiChat` | Edge-scroll detection, then `Chat.HandleMouseDrag(x, y)` |
| `tea.MouseReleaseMsg` | Not in dialog, state `uiChat` | `Chat.HandleMouseUp(x, y)`, schedule highlight copy check |
| `tea.MouseWheelMsg` | Not in dialog, state `uiChat` | `Chat.ScrollByAndAnimate(±MouseScrollThreshold)` |
| `DelayedClickMsg` | None | `Chat.HandleDelayedClick(msg)` |

Coordinate Translation
----------------------

The chat component occupies a specific sub-region of the terminal defined by the layout. Before forwarding mouse positions to the `Chat` model, `UI.Update()` translates terminal coordinates into the chat's local coordinate space by subtracting the origin of the `layout.main` rectangle [internal/ui/model/ui.go588-589](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L588-L589)

**Coordinate translation into chat local space**

The translation logic ensures that the `Chat` component remains decoupled from the global layout. Sidebar clicks are explicitly filtered out using `image.Pt(msg.X, msg.Y).In(m.layout.sidebar)` [internal/ui/model/ui.go584](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L584-L584)

Focus Management on Click
-------------------------

The `handleClickFocus` function determines which sub-component (`uiFocusEditor` or `uiFocusMain`) should receive keyboard focus based on the click location [internal/ui/model/ui.go956-972](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L956-L972)

Mouse Wheel Scrolling
---------------------

`MouseScrollThreshold` is a constant set to `5` lines [internal/ui/model/ui.go65](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L65-L65)
 Each `tea.MouseWheelMsg` triggers `Chat.ScrollByAndAnimate`. If the user scrolls past the current selection, the chat automatically updates the selected item to keep a valid item in focus [internal/ui/model/ui.go670-684](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L670-L684)

| Button | Delta passed | Selection update |
| --- | --- | --- |
| `tea.MouseWheelUp` | `-MouseScrollThreshold` | `Chat.SelectPrev()` if current selection scrolled out |
| `tea.MouseWheelDown` | `+MouseScrollThreshold` | `Chat.SelectNext()` if current selection scrolled out |

The `Chat` component uses `list.List` for underlying scroll management [internal/ui/model/chat.go37](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L37-L37)

Text Highlight and Multi-Click Detection
----------------------------------------

Crush implements single, double, and triple click detection to facilitate text selection and interaction within the chat.

1.  **Click Timing**: `doubleClickThreshold` is set to `400ms` [internal/ui/model/chat.go21](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L21-L21)
    
2.  **Coordinate Mapping**: The `Chat` model tracks `mouseDownX` and `mouseDownY` as character/line offsets within a specific `MessageItem` [internal/ui/model/chat.go48-49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L48-L49)
    
3.  **Highlight Copy**: On `tea.MouseReleaseMsg`, if a highlight exists, a `tea.Tick` is scheduled [internal/ui/model/ui.go649-650](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L649-L650)
     If the threshold passes without another click, `copyChatHighlightMsg` is sent to copy the selection to the clipboard [internal/ui/model/ui.go651-654](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L651-L654)
    
4.  **DelayedClickMsg**: This internal message is used to trigger actions (like expanding/collapsing tool results) only after ensuring the user isn't performing a double-click for text selection [internal/ui/model/chat.go27-31](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/chat.go#L27-L31)

Dialog Mouse Passthrough
------------------------

The `dialog.Overlay` component manages modal dialogs. When an overlay is active, it intercepts all mouse messages via `Overlay.Update(msg)` [internal/ui/model/ui.go571-574](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L571-L574)
 This prevents background interactions with the chat or editor while a dialog (like the Commands palette or Models picker) is open.
