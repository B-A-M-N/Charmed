This page documents the JSON schema for `crush.json` configuration files: how it is generated from Go source types, how the CI workflow maintains it, and how to utilize it in editors for validation and autocomplete. For a description of the config loading process and how files merge, see [3.2](https://github.com/charmbracelet/crush/blob/1ceeaa93/3.2)
 For a user-facing explanation of common config options, see [2.3](https://github.com/charmbracelet/crush/blob/1ceeaa93/2.3)

* * *

Overview
--------

Crush exposes its configuration structure as a JSON Schema (Draft 2020-12). The schema is derived directly from the Go `Config` struct in `internal/config/config.go` [internal/config/config.go432-463](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L432-L463)
 using struct-tag annotations and the `github.com/invopop/jsonschema` reflector. This allows any editor supporting JSON Schema (VS Code, JetBrains IDEs, Neovim with LSP) to provide real-time validation and autocomplete for `crush.json` files.

The schema is maintained in the repository at:

| File | Purpose |
| --- | --- |
| `schema.json` | The canonical machine-generated schema at the repository root [schema.json1-5](https://github.com/charmbracelet/crush/blob/1ceeaa93/schema.json#L1-L5) |
| `.github/crush-schema.json` | A copy used for schema serving, marked as generated in [.gitattributes2](https://github.com/charmbracelet/crush/blob/1ceeaa93/.gitattributes#L2-L2) |

* * *

Schema Generation
-----------------

The schema is produced by the `crush schema` subcommand defined in `internal/cmd/schema.go` [internal/cmd/schema.go12-26](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/schema.go#L12-L26)

The command uses `jsonschema.Reflector` to analyze the `config.Config` struct. It reflects the type information and struct tags into a JSON schema, which is then marshaled with indentation and printed to stdout [internal/cmd/schema.go18-23](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/schema.go#L18-L23)

Automated CI Update
-------------------

The schema is kept in sync with the code via a dedicated GitHub Actions workflow. When changes are made to the configuration structures or the hyper provider embedding, the CI regenerates the schema to ensure documentation and editor support never drift from the implementation.

**Workflow Logic (`schema-update.yml`):**

1.  **Trigger**: Runs on pushes to `main` affecting `internal/config/**` or `internal/agent/hyper/**` [.github/workflows/schema-update.yml3-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/schema-update.yml#L3-L9)
    
2.  **Execution**: Runs `go run . schema > ./schema.json` to generate the latest schema [.github/workflows/schema-update.yml21](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/schema-update.yml#L21-L21)
    
3.  **Hyper Generation**: Also runs `go generate ./internal/agent/hyper/...` [.github/workflows/schema-update.yml22](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/schema-update.yml#L22-L22)
    
4.  **Persistence**: Uses `stefanzweifel/git-auto-commit-action` to commit the updated `schema.json` back to the repository [.github/workflows/schema-update.yml23-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/schema-update.yml#L23-L30)
    

The file `.gitattributes` marks these files as `linguist-generated` to exclude them from diff statistics [.gitattributes1-5](https://github.com/charmbracelet/crush/blob/1ceeaa93/.gitattributes#L1-L5)

 Every exported field that participates in serialization carries a `json` tag and a `jsonschema` tag providing metadata like `description`, `enum`, `default`, and `example`.

Using the Schema in Editors
---------------------------

To enable validation and autocomplete, include the `$schema` property at the root of your `crush.json` file.

The `Config` struct explicitly includes the `Schema` field with the `json:"$schema"` tag to ensure it is preserved during programmatic configuration updates [internal/config/config.go433](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L433-L433)

### Editor Support

*   **VS Code**: Automatically detects the `$schema` field and provides intellisense.
*   **JetBrains (GoLand/IntelliJ)**: Maps `crush.json` to the schema via the JSON Schema settings or the inline `$schema` key.
*   **Neovim**: Requires `jsonls` (JSON Language Server) to be configured to follow the schema URL.
