Purpose and Scope
-----------------

This document explains Crush's variable resolution system, which handles environment variable expansion, shell command substitution, and templated API keys throughout the configuration system. The variable resolver enables secure credential management by allowing users to reference external credential stores or environment variables instead of hardcoding sensitive values.

* * *

Overview
--------

Crush's variable resolution system supports two primary resolution patterns in configuration string values:

1.  **Environment variable expansion**: `$VAR` or `${VAR}` syntax expands to environment variable values.
2.  **Shell command substitution**: `$(command)` syntax executes shell commands and captures stdout.

This design allows configuration files to reference external credential stores rather than containing sensitive values directly. Examples include `$(pass show api-key)` for Unix password managers or `$(security find-generic-password -w -a user -s service)` for macOS Keychain.

CRUSH\_ Prefix Environment Variables
------------------------------------

Crush supports a `CRUSH_` prefix convention for environment variables. This enables Crush-specific credentials without affecting other tools that might read the same environment variables. For example, setting `CRUSH_OPENAI_API_KEY` will override `OPENAI_API_KEY` only within the Crush process.

**PushPopCrushEnv() Function**

Title: Environment Variable Prefix Mapping Logic

**Mechanism:**

1.  **Scan**: Iterates through `os.Environ()` looking for variables prefixed with `CRUSH_` [internal/config/load.go142-144](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L142-L144)
    
2.  **Backup**: Stores original values of unprefixed variable names in a map [internal/config/load.go152-155](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L152-L155)
    
3.  **Override**: Sets unprefixed variables to the values found in their `CRUSH_` counterparts using `os.Setenv()` [internal/config/load.go157-159](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L157-L159)
    
4.  **Return**: Returns a cleanup function that restores original values from the backup map [internal/config/load.go161-166](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L161-L166)

Resolution Logic
----------------

### Shell Expansion

The `shellVariableResolver` delegates the actual parsing and execution to the `shell.ExpandValue` function. This provides a rich set of bash-like features.

*   **Variable Substitution**: Supports `$VAR`, `${VAR}`, and defaulting syntax like `${VAR:-default}` [internal/config/resolve.go59-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L59-L62)
    
*   **Command Substitution**: Supports `$(command)` for dynamic execution [internal/config/resolve.go78-80](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L78-L80)
    
*   **Timeout**: Single `ResolveValue` calls are bounded by a 5-minute timeout [internal/config/resolve.go14](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L14-L14)
    
*   **Validation**: A lone `$` is rejected as an invalid value format [internal/config/resolve.go93-95](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L93-L95)
    

Sources: [internal/config/resolve.go12-105](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L12-L105)

### Error Sanitization

To prevent sensitive information from leaking into logs or error messages, the resolver scrubs error output.

*   **Scrubbing**: The `sanitizeResolveError` function ensures the resolved (post-expansion) value is never included in the error message [internal/config/resolve.go118-120](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L118-L120)
    
*   **Bounding**: Error messages are truncated to 512 bytes [internal/config/resolve.go111](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L111-L111)
    
*   **Sanitization**: Non-printable characters (except tab and newline) are replaced with `?` [internal/config/resolve.go160-174](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L160-L174)
    

Sources: [internal/config/resolve.go107-174](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L107-L174)

Templated API Keys
------------------

`ProviderConfig` supports re-resolution of credentials to handle scenarios like auth errors where a token might have expired.

| Field | Type | Purpose |
| --- | --- | --- |

| `APIKeyTemplate` | `string` | The original unresolved string (e.g. `$(gh auth token)`) [internal/config/config.go102](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L102-L102) |

**Resolution Flow in `configureProviders`:**

1.  The `APIKeyTemplate` is populated from the `APIKey` field during initial load [internal/config/load.go248](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L248-L248)
    
2.  The `ResolveValue` method is called on the template [internal/config/load.go250](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L250-L250)
    
3.  The result is written back to the `APIKey` field for use in requests [internal/config/load.go251](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L251-L251)

Configuration-Specific Resolution Methods
-----------------------------------------

### LSP and MCP Environment Resolution

LSP and MCP configurations often require environment variables to be passed to their respective servers.

Title: Environment Map Resolution Data Flow

*   **LSP**: `LSPConfig.ResolvedEnv()` resolves the `Env` map into a `KEY=VALUE` slice [internal/config/config.go412-414](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L412-L414)
    
*   **MCP**: `MCPConfig.ResolvedEnv()` performs the same resolution for MCP servers [internal/config/config.go416-418](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L416-L418)
    
*   **Logic**: Both delegate to `resolveEnvs`, which uses a `VariableResolver` to expand values for each key [internal/config/config.go1021-1037](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L1021-L1037)
    

### MCP Header Resolution

For MCP servers using `http` or `sse` transports, headers can contain resolved variables.

*   **Method**: `MCPConfig.ResolvedHeaders()` [internal/config/config.go420-431](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L420-L431)
    
*   **Behavior**: It iterates through the `Headers` map and resolves each value. If a resolution fails, the error is logged and that specific header is skipped [internal/config/config.go427-430](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L427-L430)
