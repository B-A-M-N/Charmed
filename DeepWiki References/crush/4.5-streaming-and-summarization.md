This document explains how Crush streams responses from AI providers in real-time and automatically summarizes conversations when they approach context window limits. For information about message storage and persistence, see [Message System](https://deepwiki.com/charmbracelet/crush/4.4-message-system)
. For details on tool execution within the streaming flow, see [Tool Architecture](https://deepwiki.com/charmbracelet/crush/4.6-tool-architecture)
.

Overview
--------

The streaming system enables incremental response delivery, updating the UI and database as the AI provider generates text, makes tool calls, and executes tools. The automatic summarization feature prevents conversations from exceeding model context windows by condensing message history when token usage reaches configured thresholds.

------------------------

 It delegates to `SessionAgent.Run`, which:

Streaming Callbacks
-------------------

The `fantasy.Agent.Stream` method accepts an `AgentStreamCall` structure that defines multiple callback functions for handling incremental updates during response generation.

### Callback Reference

| Callback | Purpose | Updates | Error Handling |
| --- | --- | --- | --- |
| `PrepareStep` | Prepare messages before provider call | Creates assistant message, processes queued messages, applies caching | Returns error to abort streaming |
| `OnReasoningStart` | Reasoning content begins | Appends reasoning to assistant message | Returns error to abort |
| `OnReasoningDelta` | Incremental reasoning text | Appends reasoning delta to assistant message | Returns error to abort |
| `OnReasoningEnd` | Reasoning completes | Stores reasoning signatures, marks thinking finished | Returns error to abort |
| `OnTextDelta` | Incremental response text | Appends text to assistant message | Returns error to abort |
| `OnToolInputStart` | Tool input begins | Creates tool call with pending state | Returns error to abort |
| `OnToolCall` | Tool call completes | Updates tool call with full input | Returns error to abort |
| `OnToolResult` | Tool execution completes | Creates tool result message | Returns error to abort |
| `OnStepFinish` | Provider turn completes | Updates finish reason, session usage | Returns error to abort |
| `OnRetry` | Provider error, retrying | Currently no-op | N/A |
| `StopWhen` | Check stopping conditions | N/A | Return `true` to stop iteration |

Sources: [internal/agent/agent.go252-427](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L252-L427)

Automatic Summarization
-----------------------

Crush automatically summarizes conversations when they approach the model's context window limit, preventing token exhaustion and enabling indefinite conversation continuity.

### Summarization Trigger

**Diagram: Automatic summarization trigger logic**

Sources: [internal/agent/agent.go406-426](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L406-L426)
 [internal/agent/agent.go49-56](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L49-L56)

The summarization trigger uses a `StopWhen` condition that evaluates after each step:

1.  **Calculate Token Usage** [internal/agent/agent.go409](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L409-L409)
    : Sum completion and prompt tokens from current session.
2.  **Determine Remaining Capacity** [internal/agent/agent.go410](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L410-L410)
    : Subtract from context window size.
3.  **Calculate Threshold** [internal/agent/agent.go411-416](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L411-L416)
    :
    *   Large models (>200k context): Use 20k token buffer (`largeContextWindowThreshold`, `largeContextWindowBuffer`) [internal/agent/agent.go53-54](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L53-L54)
        
    *   Smaller models: Use 20% of context window (`smallContextWindowRatio`) [internal/agent/agent.go55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L55-L55)
        
4.  **Check Condition** [internal/agent/agent.go417](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L417-L417)
    : If remaining <= threshold and auto-summarize is enabled, return `true` to stop streaming and trigger summarization.

Sources: [internal/agent/agent.go406-423](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L406-L423)

### Summarization Process

**Diagram: Summarization execution sequence**

Sources: [internal/agent/agent.go564-675](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L564-L675)

The `sessionAgent.Summarize` method performs the following steps. Note that `Coordinator.Summarize` [internal/agent/coordinator.go84](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L84-L84)
 wraps this call, providing the current provider options before delegating.

1.  **Validation** [internal/agent/agent.go564-567](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L564-L567)
    : Return `ErrSessionBusy` if session is active.
2.  **Message Retrieval** [internal/agent/agent.go572-583](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L572-L583)
    : Fetch all session messages via `getSessionMessages`.
3.  **Agent Creation** [internal/agent/agent.go593-595](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L593-L595)
    : Create a new `fantasy.Agent` with `summaryPrompt` as the system prompt.
4.  **Summary Message Creation** [internal/agent/agent.go596-604](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L596-L604)
    : Create a message with `IsSummaryMessage: true` flag in the database.
5.  **Prompt Construction** [internal/agent/agent.go606](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L606-L606)
    : Call `buildSummaryPrompt` to build the prompt text, including any current todos [internal/agent/agent.go1146-1158](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L1146-L1158)
    
6.  **Streaming** [internal/agent/agent.go608-637](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L608-L637)
    : Stream the summary, handling `OnReasoningDelta`, `OnReasoningEnd`, and `OnTextDelta`.
7.  **Usage Update** [internal/agent/agent.go666](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L666-L666)
    : Accumulate token usage and cost into the session via `updateSessionUsage`.
8.  **Session State Update** [internal/agent/agent.go669-673](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L669-L673)
    : Set `SummaryMessageID`, reset `CompletionTokens` to current output, and zero `PromptTokens`.

### Summary Prompt Template

 and instructs the model to provide a thorough briefing for continuing work, covering Current State, Files & Changes, Technical Context, Strategy, and Exact Next Steps.

### Post-Summarization Message Handling

After summarization completes, the session's message history is effectively reset [internal/agent/agent.go749-769](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L749-L769)
:

1.  When loading messages, if `session.SummaryMessageID` is set, only messages from that point forward are loaded [internal/agent/agent.go755-761](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L755-L761)
    
2.  The summary message's role is reassigned from `assistant` to `user` [internal/agent/agent.go762](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L762-L762)
     so it serves as the starting context for subsequent turns.
3.  Earlier messages are hidden from the LLM but preserved in the database.

If the agent was interrupted mid-execution (had pending tool calls), the original request is re-queued [internal/agent/agent.go539-547](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L539-L547)
 with a modified prompt explaining the interruption.

Sources: [internal/agent/agent.go749-769](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L749-L769)
 [internal/agent/agent.go533-548](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L533-L548)

Title Generation
----------------

A related feature generates session titles using the small model for efficiency. This occurs asynchronously during the first message [internal/agent/agent.go225-230](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L225-L230)
:

1.  **Async Execution**: Launches via `sync.WaitGroup.Go` so title generation runs concurrently with the main streaming call [internal/agent/agent.go228-230](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L228-L230)
    
2.  **Model Selection**: Attempts with `smallModel` first; falls back to `largeModel` if it fails [internal/agent/agent.go807-828](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L807-L828)
    
3.  **Prompt Template**: Uses `internal/agent/templates/title.md` loaded at [internal/agent/agent.go60-61](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L60-L61)
     with instructions to produce a concise title; `<think>` tags are stripped via `thinkTagRegex` [internal/agent/agent.go68](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L68-L68)
     [internal/agent/agent.go849](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L849-L849)
    
4.  **Atomic Update**: Uses `sessions.UpdateTitleAndUsage` [internal/agent/agent.go883](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L883-L883)
     to atomically write only the title and usage fields, avoiding races with concurrent session updates.

Sources: [internal/agent/agent.go772-888](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L772-L888)

Error Handling in Streaming
---------------------------

When streaming encounters errors, the agent ensures data consistency [internal/agent/agent.go431-531](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L431-L531)
:

1.  **Finish Thinking** [internal/agent/agent.go438](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L438-L438)
    : Close any open reasoning state via `FinishThinking()`.
2.  **Complete Unfinished Tool Calls** [internal/agent/agent.go439-452](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L439-L452)
    : Mark incomplete tool calls as finished with empty input `{}`.
3.  **Create Error Tool Results** [internal/agent/agent.go453-491](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L453-L491)
    : For tool calls without corresponding results, create tool result messages with appropriate error text.
4.  **Add Finish Status** [internal/agent/agent.go492-521](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L492-L521)
    : Set `FinishReasonCanceled`, `FinishReasonPermissionDenied`, `FinishReasonError`, etc., with provider-specific error details.
5.  **Update Message** [internal/agent/agent.go522-530](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L522-L530)
    : Persist final state to database using the parent `ctx` (since `genCtx` has been cancelled).

Sources: [internal/agent/agent.go431-531](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L431-L531)
