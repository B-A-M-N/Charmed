This page documents the `SelectedModel` and `SelectedModelType` types, how models are resolved at runtime via `GetModel`, `GetModelByType`, `LargeModel`, and `SmallModel`, the `UpdatePreferredModel` persistence flow, and how per-model generation parameters (`MaxTokens`, `Temperature`, `ReasoningEffort`, `Think`, etc.) are structured and applied.

For details on provider initialization and the `ProviderConfig` type, see page 3.2. For agent configuration and which model type each agent uses, see page 4.1.

Model Configuration Structure
-----------------------------

Crush maintains two active model selections: a "large" model for complex tasks and a "small" model for simple tasks. These selections are stored in the `Config.Models` map with keys `"large"` and `"small"`.

### Diagram: Model Configuration Hierarchy

**Selected Model Data Structure**

Each `SelectedModel` contains the following fields:

| Field | Type | Purpose | Example |
| --- | --- | --- | --- |
| `Model` | `string` | Model ID as used by provider API | `"gpt-4o"`, `"claude-3-5-sonnet-20241022"` |
| `Provider` | `string` | Provider ID matching key in providers config | `"openai"`, `"anthropic"` |
| `ReasoningEffort` | `string` | OpenAI reasoning models only: controls depth of reasoning | `"low"`, `"medium"`, `"high"` |
| `Think` | `bool` | Anthropic models: enables extended thinking mode | `true` |
| `MaxTokens` | `int64` | Maximum output tokens (overrides provider default) | `4096` |
| `Temperature` | `*float64` | Sampling temperature (0=deterministic, 1=random) | `0.7` |
| `TopP` | `*float64` | Nucleus sampling parameter | `0.9` |
| `TopK` | `*int64` | Top-k sampling parameter | `40` |
| `FrequencyPenalty` | `*float64` | Penalize frequent tokens to reduce repetition | `0.5` |
| `PresencePenalty` | `*float64` | Penalize tokens already present to increase diversity | `0.5` |
| `ProviderOptions` | `map[string]any` | Additional provider-specific options | —   |

Per-Model Generation Parameters
-------------------------------

The `SelectedModel` struct carries both identity fields (`Model`, `Provider`) and optional generation parameter overrides. When a field is set, it overrides the provider/model default used by the agent during inference. Pointer types (`*float64`, `*int64`) distinguish "not set" from a zero value [internal/config/config.go80-84](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L80-L84)

`ReasoningEffort` and `Think` are convenience fields that map to provider-specific API parameters. `ReasoningEffort` is used for OpenAI `o1`/`o3`\-family models [internal/config/config.go73](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L73-L73)
 `Think` enables Anthropic's extended thinking (reasoning) mode and is only meaningful when the underlying model supports it [internal/config/config.go76](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L76-L76)

The `ProviderOptions` map allows arbitrary provider-specific overrides. These are merged with any `ProviderOptions` set on the `ProviderConfig` during agent initialization [internal/config/config.go87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L87-L87)

Sources: [internal/config/config.go64-88](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L64-L88)

Model Types and Agent Assignment
--------------------------------

Crush uses two model type classifications defined as `SelectedModelType` [internal/config/config.go47](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L47-L47)

*   **Large Model (`SelectedModelTypeLarge`)**: Used by the `coder` agent for complex coding tasks requiring deep reasoning and context [internal/config/config.go55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L55-L55)

`UpdatePreferredModel` and Persistence
--------------------------------------

`UpdatePreferredModel` is the primary method for changing the active model selection. It updates the in-memory `Config.Models` map, persists the selection to the data config file via `ConfigStore.SetConfigField`, and records the choice in the recent-models list.

### Diagram: UpdatePreferredModel Flow

`SetConfigField` writes only the targeted JSON path — it reads the current config file, patches the field, and writes the result back. After a successful write, it triggers an `autoReload` to keep all in-memory state fresh.

Recent Models Tracking
----------------------

Crush tracks the 5 most recently used models for each model type (large and small).

### Recent Model Recording Process

When a user selects a model, the system:

1.  Adds the model to the front of the recent models list for its type.
2.  Removes any duplicate entries (deduplication).
3.  Trims the list to maximum 5 entries.
4.  Persists the updated list to the config file via `SetConfigField`.

Sources: [internal/config/config.go660-701](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L660-L701)

 It establishes baseline large and small selections, then overlays any user-specified values from `Config.Models`.

Model Resolution Methods
------------------------

Methods on `Config` resolve the active model to the canonical `catwalk.Model` record, which carries metadata such as `ContextWindow`, `DefaultMaxTokens`, `CanReason`, and `SupportsImages`.

### Diagram: Resolution Method Mapping

| Method | Input | Returns |
| --- | --- | --- |
| `GetModel` | `provider` ID, `model` ID | `*catwalk.Model` or `nil` if not found |
| `GetModelByType` | `SelectedModelType` | `*catwalk.Model` via `GetModel` |
| `LargeModel` | —   | `*catwalk.Model` for `SelectedModelTypeLarge` |
| `SmallModel` | —   | `*catwalk.Model` for `SelectedModelTypeSmall` |
| `GetProviderForModel` | `SelectedModelType` | `*ProviderConfig` for the model's provider |
