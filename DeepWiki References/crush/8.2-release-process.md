Purpose and Scope
-----------------

This document describes the release process for Crush, including version tagging, multi-platform builds, code signing, and distribution across multiple package managers. The release system is built on GoReleaser and GitHub Actions, automating the creation and publication of releases to over 8 distribution channels.

For information about the CI/CD workflows that run on every commit (build, security scanning, schema updates), see **8.3 CI/CD Workflows**. For local development build tasks, see **8.1 Build System**.

* * *

Release Trigger
---------------

Releases are initiated by pushing a Git tag matching the pattern `v*.*.*` to the repository [\[.github/workflows/release.yml:5-8\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. The conventional entry point is the `release` Taskfile task, which handles the tag creation. This triggers the GitHub Actions workflow defined in `release.yml` [\[.github/workflows/release.yml:3\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

The workflow delegates to the shared `charmbracelet/meta` goreleaser workflow using Go 1.26 and GoReleaser nightly [\[.github/workflows/release.yml:16-21\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. The `.goreleaser.yml` file in the repository root drives the entire build and distribution pipeline.

Pre-Release Preparation
-----------------------

Before building binaries, the `before.hooks` section of `.goreleaser.yml` generates shell completions and man pages by invoking the binary itself [\[.goreleaser.yml:30-39\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. These artifacts are then bundled into every release archive and distributed to the appropriate system locations by each package format.

Package Distribution Details
----------------------------

### `brews` — Homebrew

Releases are committed to the `charmbracelet/homebrew-tap` repository [\[.goreleaser.yml:187-189\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. Shell completions and man pages are installed via the `extra_install` directive [\[.goreleaser.yml:196-200\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

### `scoops` — Scoop (Windows)

Windows packages are published to the `charmbracelet/scoop-bucket` repository [\[.goreleaser.yml:203-205\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

### `npms` — npm

Available as `@charmland/crush` on the npm registry [\[.goreleaser.yml:213\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. The package wraps the native binary.

### `nfpms` — Native Linux Packages

The `nfpms` key generates packages for multiple Linux distributions [\[.goreleaser.yml:219-241\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
:

*   **.deb**: Debian, Ubuntu (GPG signed)
*   **.rpm**: RHEL, Fedora (GPG signed)
*   **.apk**: Alpine Linux
*   **archlinux**: Arch Linux
*   **termux.deb**: Android Termux

### `aur_sources` / `aurs` — Arch User Repository

Two AUR packages are maintained:

*   `aur_sources`: Builds from source using Arch-specific `GOFLAGS` [\[.goreleaser.yml:99-142\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
    .
*   `aurs`: Installs pre-compiled binaries from release archives (`crush-bin`) [\[.goreleaser.yml:143-179\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
    .

### `nix` — Nix User Repository

Releases are published to `charmbracelet/nur` with `license: fsl11Mit` [\[.goreleaser.yml:262-269\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

### `winget` — Windows Package Manager

GoReleaser opens a pull request targeting `microsoft/winget-pkgs@master` [\[.goreleaser.yml:277-299\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

Code Signing and Security
-------------------------

**Signing pipeline — secrets, files, and goreleaser keys**

### macOS Code Signing

macOS binaries are signed and notarized. The entitlements file grants `com.apple.security.cs.allow-unsigned-executable-memory`, required for Go binaries on macOS [\[.github/entitlements.plist:1-8\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

### Linux Package Signing

Debian and RPM packages are GPG-signed using the `NFPM_GPG_KEY` and `NFPM_PASSPHRASE` secrets [\[.github/workflows/release.yml:28-29\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
 via the `nfpms` signature configuration [\[.goreleaser.yml:236-241\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

### Cosign Artifact Signing (`signs`)

The `signs` section runs `cosign sign-blob --bundle` on `checksums.txt`, producing `checksums.txt.sigstore.json` for downstream verification [\[.goreleaser.yml:243-252\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

### SBOM Generation (`sboms`)

SBOM entries are generated for `archive` and `source` artifacts for supply chain auditing [\[.goreleaser.yml:257-260\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

Nightly and Snapshot Releases
-----------------------------

The `nightly` section configures a rolling release published to GitHub with `keep_single_release: true` [\[.goreleaser.yml:21-24\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. Snapshot builds use the timestamp-based version format `0.0.0-{{ .Timestamp }}` [\[.goreleaser.yml:27\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

The `nightly.yml` workflow runs every day at midnight [\[.github/workflows/nightly.yml:4-5\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
. It checks if a successful run already exists for the current HEAD before triggering the shared nightly workflow [\[.github/workflows/nightly.yml:21-29\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

* * *

Archive Naming Convention
-------------------------

Archives use the template: `crush_{{.Version}}_{{title .Os}}_{{arch}}` [\[.goreleaser.yml:78-85\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.

*   **amd64** → `x86_64`
*   **386** → `i386`
*   **Windows** archives use `.zip` [\[.goreleaser.yml:94\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
    .

Every archive includes the binary, README, LICENSE, man pages, and shell completions [\[.goreleaser.yml:87-91\]](https://deepwiki.com/charmbracelet/crush/8.2-release-process)
.
