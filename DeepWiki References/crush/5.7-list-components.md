This page documents the list component system in Crush's TUI, which provides a flexible abstraction for displaying and navigating scrollable, selectable content. The system supports lazy rendering, memoized performance, and fuzzy filtering.

* * *

Purpose and Architecture
------------------------

The list component system consists of a core `List` type that provides viewport-based rendering with lazy evaluation and an F6-level render memo (cache). Lists render items that implement the `Item` interface.

Core List Component
-------------------

The `List` struct [internal/ui/list/list.go10-54](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L10-L54)
 manages items stacked vertically. It uses an internal cache to avoid re-rendering items that haven't changed.

### Item Interface and Memoization

Items participate in the list-level render memo (F6) [internal/ui/list/item.go11-20](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L11-L20)
 The cache key for each item is a combination of its pointer, the viewport width, and its version.

*   **`Version()`**: Returns a monotonic counter. Items must increment this (via `Versioned.Bump()`) on any state change that alters output [internal/ui/list/item.go26-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L26-L30)
    
*   **`Finished()`**: Reports if the item has reached a terminal state. Once an item is finished and rendered, the list freezes its cache entry [internal/ui/list/item.go32-38](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L32-L38)
    
*   **`Versioned`**: A helper struct that can be embedded in items to provide thread-safe version tracking [internal/ui/list/item.go50-57](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L50-L57)
    

**Sources:** [internal/ui/list/item.go9-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/item.go#L9-L71)

### List Rendering and Caching

The `List` maintains a `cache map[Item]*listCacheEntry` [internal/ui/list/list.go45](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L45-L45)

*   A width change via `SetSize` drops the entire cache because wrapped output depends on width [internal/ui/list/list.go98-104](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L98-L104)
    
*   Height-only changes are no-ops for the cache [internal/ui/list/list.go97](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L97-L97)
    
*   `renderItemEntry` populates the cache on miss and checks versions to detect mutations [internal/ui/list/list.go236-250](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L236-L250)

FilterableList and Fuzzy Search
-------------------------------

`FilterableList` wraps a standard `List` and adds fuzzy filtering capabilities using `sahilm/fuzzy` [internal/ui/list/filterable.go22-26](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/filterable.go#L22-L26)

### Filtering Data Flow

When `SetFilter` is called, the list re-evaluates which items from its master `items` slice match the query [internal/ui/list/filterable.go70-74](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/filterable.go#L70-L74)
 If an item implements `MatchSettable`, it is updated with the match metadata (like `MatchedIndexes`) to allow the item to render highlights [internal/ui/list/filterable.go108-113](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/filterable.go#L108-L113)

Specialized Implementations
---------------------------

### ModelsList (Model Picker)

`ModelsList` handles the selection of AI models, organized by provider groups [internal/ui/dialog/models\_list.go15-20](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/models_list.go#L15-L20)

*   **Grouped Rendering**: Items are interleaved with `ModelGroup` headers and `SpacerItem` separators [internal/ui/dialog/models\_list.go43-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/models_list.go#L43-L55)
    
*   **Smart Navigation**: `SelectNext`, `SelectPrev`, `SelectFirst`, and `SelectLast` are overridden to skip non-selectable items like headers and spacers [internal/ui/dialog/models\_list.go103-154](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/models_list.go#L103-L154)
    
*   **Enhanced Filtering**: The filter query matches against a combination of the provider title and the model name [internal/ui/dialog/models\_list.go205-213](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/models_list.go#L205-L213)

### Completions List

The `Completions` component uses a `FilterableList` to display file and MCP resource suggestions [internal/ui/completions/completions.go61](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/completions/completions.go#L61-L61)

*   **Tiered Sorting**: It implements a `namePriorityTier` system to sort results. Exact matches rank highest (`tierExactName`), followed by prefix matches (`tierPrefixName`), path segments (`tierPathSegment`), and finally fuzzy fallbacks [internal/ui/completions/completions.go72-96](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/completions/completions.go#L72-L96)
    
*   **Reverse Rendering**: The list is rendered in reverse (`SetReverse(true)`) so that suggestions grow upwards from the input area [internal/ui/completions/completions.go102](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/completions/completions.go#L102-L102)

Scrolling and Viewport Mechanics
--------------------------------

The list supports line-accurate scrolling through two offsets:

1.  `offsetIdx`: The index of the first visible item [internal/ui/list/list.go28](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L28-L28)
    
2.  `offsetLine`: The number of lines of the `offsetIdx` item that are scrolled above the viewport [internal/ui/list/list.go29-32](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/list/list.go#L29-L32)
    

### Scrollbar Calculation

The TUI renders a vertical scrollbar using `common.Scrollbar` [internal/ui/common/scrollbar.go11](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/scrollbar.go#L11-L11)
 It calculates `thumbSize` and `thumbPos` based on the ratio of `viewportSize` to `contentSize` [internal/ui/common/scrollbar.go17-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/scrollbar.go#L17-L30)
