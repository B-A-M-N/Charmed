Purpose and Scope
-----------------

This page documents the visual styling infrastructure for the crush TUI: the `Styles` struct, the `charmtone`\-based color themes, gradient rendering utilities, terminal capability detection, and the notification subsystem. These primitives are consumed by every UI component described in the user interface sections.

For how TUI components render using these styles, see TUI Architecture ([5.1](https://github.com/charmbracelet/crush/blob/1ceeaa93/5.1)
). For dialog-specific layout helpers, see Dialog System ([5.5](https://github.com/charmbracelet/crush/blob/1ceeaa93/5.5)
).

Theme System
------------

Crush uses a theme system based on `charmtone`. Themes are constructed using the `quickStyle` helper, which maps a semantic palette of colors to the complex `Styles` struct [internal/ui/styles/quickstyle.go64-330](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/quickstyle.go#L64-L330)

### Provider-Based Themes

The system selects themes based on the active LLM provider:

*   **Hypercrush**: Uses the `HypercrushObsidiana` theme [internal/ui/styles/themes.go10-11](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/themes.go#L10-L11)
    
*   **Default**: Uses `CharmtonePantera` for all other providers [internal/ui/styles/themes.go13-14](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/themes.go#L13-L14)
    

### Semantic Palette (`quickStyleOpts`)

Themes are defined using semantic roles:

*   `primary`/`secondary`: Brand accents used for gradients and cursors [internal/ui/styles/quickstyle.go75-76](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/quickstyle.go#L75-L76)
    
*   `fgBase`/`bgBase`: Primary text and window background [internal/ui/styles/quickstyle.go66-72](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/quickstyle.go#L66-L72)
    
*   `error`/`warning`/`success`: Status colors for diagnostics and tool results [internal/ui/styles/quickstyle.go47-54](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/quickstyle.go#L47-L54)

Logo Rendering
--------------

The `logo` package renders the stylized "Crush" wordmark. It supports two modes:

1.  **Wide Render**: Used on the landing screen; features diagonal "fields" and a randomly stretched letterform to prevent visual staleness [internal/ui/logo/logo.go42-147](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/logo/logo.go#L42-L147)
    
2.  **Small Render**: A compact version used in the sidebar or header that combines "Charm™" with a gradient wordmark [internal/ui/logo/logo.go151-168](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/logo/logo.go#L151-L168)

Gradient and Markdown Rendering
-------------------------------

### Foreground Gradients

`internal/ui/styles/grad.go` provides utilities for interpolating colors across strings. It uses `go-colorful` for linear interpolation and `uniseg` to ensure that multi-byte grapheme clusters (like emojis or combined characters) are treated as single units during color application [internal/ui/styles/grad.go15-115](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/grad.go#L15-L115)

### Markdown Cache

Markdown is rendered using `glamour`. To maintain performance during TUI updates, Crush implements a shared renderer cache indexed by terminal width [internal/ui/common/markdown.go28-32](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/markdown.go#L28-L32)

*   **Concurrency**: Because `glamour` (goldmark) is not safe for concurrent `Render` calls, the system uses `LockMarkdownRenderer` to provide per-renderer mutexes [internal/ui/common/markdown.go130-140](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/markdown.go#L130-L140)
    
*   **Invalidation**: When styles change (e.g., theme toggle), `InvalidateMarkdownRendererCache` clears all cached renderers and mutexes [internal/ui/common/markdown.go91-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/markdown.go#L91-L100)
    

Sources: [internal/ui/styles/grad.go1-120](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/styles/grad.go#L1-L120)
 [internal/ui/common/markdown.go1-140](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/markdown.go#L1-L140)

DiffView and Notifications
--------------------------

### DiffView Component

The `DiffView` component (`internal/ui/diffview/diffview.go`) handles the display of file changes. It supports:

*   **Unified Diff**: Standard `+`/`-` line display [internal/ui/diffview/diffview.go29](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/diffview/diffview.go#L29-L29)
    
*   **Syntax Highlighting**: Integration with `chroma` for colored code within the diff [internal/ui/diffview/diffview.go48](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/diffview/diffview.go#L48-L48)
    

### Notifications

The TUI tracks window focus via `m.notifyWindowFocused` [internal/ui/model/ui.go245](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L245-L245)
 If the agent completes a task while the window is blurred, the `notification.Backend` (e.g., desktop alerts) is triggered to notify the user [internal/ui/model/ui.go244](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/model/ui.go#L244-L244)
