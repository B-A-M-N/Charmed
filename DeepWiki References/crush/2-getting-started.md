What is crush?
--------------

`crush` is a terminal-based AI coding assistant. It runs as either an interactive TUI (default) or a non-interactive CLI tool (`crush run`). The agent has direct access to your file system and shell, enabling it to read, write, and execute code on your behalf under a permission system. It supports multiple LLM providers, Model Context Protocol (MCP), and Language Server Protocol (LSP) for enhanced context [README.md1-22](https://github.com/charmbracelet/crush/blob/1ceeaa93/README.md?plain=1#L1-L22)

Interactive Mode (TUI)
----------------------

Running `crush` with no subcommand starts the Bubble Tea TUI. The TUI provides:

*   A chat window for conversation history.
*   An editor for typing prompts.
*   Modal dialogs for model selection and permissions.
*   Mouse support for scrolling and clicking, filtered via `ui.MouseEventFilter` [internal/cmd/root.go134](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L134-L134)
    

The entry point is `rootCmd.RunE` in [internal/cmd/root.go106-145](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L106-L145)

* * *

Non-Interactive Mode
--------------------

Use `crush run` to send a single prompt and exit. This is useful for scripts and pipes.

The `crush run` command is defined in the `cmd` package and allows for quick, single-turn interactions with the AI agent without entering the full TUI environment.

Configuration Basics
--------------------

`crush` uses JSON for configuration. It looks for a global config and project-level overrides. It also supports loading settings from environment variables prefixed with `CRUSH_` [internal/config/load.go141-167](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L141-L167)

A minimal config for OpenAI:

The `Config` struct defines all available fields, including `providers`, `models`, `mcp`, and `lsp` [schema.json54-93](https://github.com/charmbracelet/crush/blob/1ceeaa93/schema.json#L54-L93)
 See [Configuration Basics](https://deepwiki.com/charmbracelet/crush/2.3-configuration-basics)
 for details.

* * *

Permission System
-----------------

By default, `crush` asks for permission before running shell commands or modifying files. This is managed by the permission service and exposed via the workspace. Passing `--yolo` allows the application to automatically accept all permission requests, which is useful for non-interactive automation but carries security risks [internal/cmd/root.go58](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L58-L58)

* * *

Data Directory
--------------

`crush` stores its SQLite database, logs, and session data in a dedicated directory. The default is `.crush` [internal/config/config.go24](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L24-L24)
 This directory is used to store workspace-specific state like session files [internal/config/load.go49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L49-L49)
 Use `crush dirs` to see your local paths.
