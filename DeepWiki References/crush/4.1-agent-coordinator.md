The Agent Coordinator is the orchestration layer for AI agent execution in Crush. This document details the `Coordinator` interface, its primary implementation, the `SessionAgent` architecture, message queuing, concurrency control, and loop detection mechanisms.

For provider integration and model building details, see page 4.2 (Provider Integration). For tool system architecture, see page 4.6 (Tool Architecture). For message streaming and callbacks, see page 4.5 (Streaming and Summarization).

* * *

Architecture Overview
---------------------

The agent system consists of two primary components: the **Coordinator**, which manages the lifecycle of multiple agents and sessions, and the **SessionAgent**, which handles the actual execution of a single session's interaction with an LLM.

**Coordinator and SessionAgent Architecture**

**Sources:**

*   [internal/agent/coordinator.go73-108](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L73-L108)
    
*   [internal/agent/coordinator.go110-168](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L110-L168)
    
*   [internal/agent/agent.go86-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L86-L125)
    
*   [internal/agent/agent.go141-159](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L141-L159)
    

**Coordinator Responsibilities** ([internal/agent/coordinator.go73-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L73-L87)
):

*   Build and configure `SessionAgent` instances via `buildAgent`.
*   Manage provider credentials and handle OAuth token refresh via `refreshTokenIfExpired` ([internal/agent/coordinator.go205-209](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L205-L209)
    ).
*   Route requests to appropriate agents in `Run`.
*   Discover and track "Skills" (specialized agent capabilities) at session start ([internal/agent/coordinator.go126-133](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L126-L133)
    ).
*   Update agent configurations when models change via `UpdateModels`.

**SessionAgent Responsibilities** ([internal/agent/agent.go86-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L86-L100)
):

*   Execute conversation turns with streaming responses in `Run`.
*   Manage per-session message queues for sequential processing via `messageQueue`.
*   Handle cancellation via `activeRequests` context cancellation functions.
*   Coordinate tool execution and trigger auto-summarization.

Coordinator Interface and Implementation
----------------------------------------

### Coordinator Interface

The `Coordinator` interface at [internal/agent/coordinator.go73-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L73-L87)
 provides the public API for the application layer:

| Method | Purpose |
| --- | --- |
| `Run(ctx, sessionID, prompt, attachments)` | Execute user prompt with attachments, returns `*fantasy.AgentResult`. |
| `Cancel(sessionID)` | Cancel active request for specific session. |
| `CancelAll()` | Cancel all active requests across all sessions. |
| `IsSessionBusy(sessionID) bool` | Check if a specific session has an active request. |
| `IsBusy() bool` | Check if any session managed by the coordinator is busy. |
| `QueuedPrompts(sessionID) int` | Count queued prompts for a session. |
| `QueuedPromptsList(sessionID) []string` | List the text of queued prompts. |
| `ClearQueue(sessionID)` | Clear the message queue for a session. |
| `Summarize(ctx, sessionID)` | Trigger conversation summarization for a session. |
| `Model() Model` | Get the current large model configuration. |
| `UpdateModels(ctx)` | Rebuild models and tools from the latest configuration. |

**Sources:** [internal/agent/coordinator.go73-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L73-L87)

### Initialization and buildAgent

The `NewCoordinator` function initializes the `coordinator` struct and wires the primary `AgentCoder` instance. It performs skill discovery once during initialization ([internal/agent/coordinator.go126-133](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L126-L133)
).

**NewCoordinator Initialization**

**Sources:** [internal/agent/coordinator.go110-168](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L110-L168)
 [internal/agent/coordinator.go452-491](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L452-L491)

The `buildAgent` method at [internal/agent/coordinator.go452-491](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L452-L491)
 is the core factory for `SessionAgent` instances. It:

1.  Builds model instances via `buildAgentModels` ([internal/agent/coordinator.go458-462](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L458-L462)
    ).
2.  Creates the `SessionAgent` via `NewSessionAgent` ([internal/agent/coordinator.go474-484](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L474-L484)
    ).
3.  Uses an `errgroup.Group` (`readyWg`) to asynchronously build the system prompt and the toolset, ensuring initialization doesn't block the main thread but is ready before `Run` completes its first call ([internal/agent/coordinator.go486-503](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L486-L503)
    ).

SessionAgent Implementation
---------------------------

### SessionAgent Structure

The `sessionAgent` struct at [internal/agent/agent.go109-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L109-L125)
 uses thread-safe `csync` data structures to handle concurrent configuration updates and per-session state.

| Field | Type | Purpose |
| --- | --- | --- |
| `largeModel` | `*csync.Value[Model]` | The primary LLM for reasoning and coding. |
| `smallModel` | `*csync.Value[Model]` | LLM for lighter tasks like title generation. |
| `tools` | `*csync.Slice[fantasy.AgentTool]` | The set of tools available to the agent. |
| `messageQueue` | `*csync.Map[string, []SessionAgentCall]` | Stores prompts sent while the agent is busy. |
| `activeRequests` | `*csync.Map[string, context.CancelFunc]` | Tracks active runs for cancellation. |

**Sources:** [internal/agent/agent.go109-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L109-L125)

### Execution Flow and Queueing

The `Run` method at [internal/agent/agent.go161-563](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L161-L563)
 implements sequential processing per session.

1.  **Empty Check**: Validates that the prompt or attachments are not empty ([internal/agent/agent.go162-164](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L162-L164)
    ).
2.  **Busy Check**: If `IsSessionBusy(call.SessionID)` is true, the call is appended to the `messageQueue` and the method returns `nil, nil` ([internal/agent/agent.go170-178](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L170-L178)
    ).
3.  **Snapshot**: It takes a snapshot of tools, models, and prompts to ensure consistency during the run ([internal/agent/agent.go181-184](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L181-L184)
    ).
4.  **Fantasy Integration**: It initializes a `fantasy.NewAgent` and starts a stream ([internal/agent/agent.go206-243](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L206-L243)
    ).
5.  **Callbacks**: It defines callbacks for `PrepareStep`, `OnTextDelta`, and `OnToolCall` to persist data to the `message.Service` ([internal/agent/agent.go246-428](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L246-L428)
    ).
6.  **Recursion**: When a stream finishes, it checks the `messageQueue`. If more prompts are waiting, it pops the next one and calls `Run` recursively ([internal/agent/agent.go555-562](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L555-L562)
    ).

**Sources:** [internal/agent/agent.go161-563](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L161-L563)

Loop Detection
--------------

To prevent the agent from entering infinite loops (e.g., repeatedly calling the same tool with the same input and getting the same error), the `sessionAgent` utilizes loop detection logic.

**Loop Detection Logic**

**Sources:** [internal/agent/loop\_detection.go19-39](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/loop_detection.go#L19-L39)

*   **`hasRepeatedToolCalls`**: Examines a window of the last 10 steps (defined by `loopDetectionWindowSize`). If any tool interaction signature repeats more than 5 times (`loopDetectionMaxRepeats`), it returns true ([internal/agent/loop\_detection.go11-39](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/loop_detection.go#L11-L39)
    ).
*   **`getToolInteractionSignature`**: Computes a SHA-256 hash of the tool name, the input JSON, and the tool's output string ([internal/agent/loop\_detection.go45-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/loop_detection.go#L45-L71)
    ).
*   **`toolResultOutputString`**: Converts various `fantasy.ToolResultOutputContent` types (text, error, media) into a stable string for hashing ([internal/agent/loop\_detection.go75-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/loop_detection.go#L75-L92)
    ).

Sub-Agents and Tool Integration
-------------------------------

The Coordinator supports spawning sub-agents via specific tools, notably the `AgentTool`.

### Agent Tool (Recursive Invocation)

The `AgentTool` allows the primary agent to delegate tasks. It:

1.  Builds a child `SessionAgent` via the coordinator's `buildAgent` method ([internal/agent/coordinator.go452-491](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L452-L491)
    ).
2.  Uses the child session to execute specific tasks via `runSubAgent` ([internal/agent/coordinator\_test.go99-110](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator_test.go#L99-L110)
    ).
3.  Rolls up costs and results from the child session back to the parent session.

**Sources:** [internal/agent/coordinator.go452-491](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L452-L491)
 [internal/agent/agent.go116-118](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L116-L118)

Cancellation and State Management
---------------------------------

### Cancellation Logic

Cancellation is handled at both the Coordinator and SessionAgent levels.

*   **`Cancel(sessionID)`**: Retrieves the `context.CancelFunc` from the `activeRequests` map and invokes it ([internal/agent/agent.go91-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L91-L92)
    ).
*   **`CancelAll()`**: Iterates through all active requests and cancels them ([internal/agent/agent.go92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L92-L92)
    ).
*   **`ClearQueue(sessionID)`**: Removes all pending `SessionAgentCall` objects from the `messageQueue` for the given session ([internal/agent/agent.go97](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L97-L97)
    ).

**Sources:** [internal/agent/agent.go86-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L86-L100)
 [internal/agent/coordinator.go77-83](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L77-L83)

### Busy State Tracking

The busy state is tracked via the `activeRequests` map. A session is considered busy if an entry exists for its `sessionID`.

*   **`IsSessionBusy(sessionID)`**: Performs a lookup in `activeRequests` ([internal/agent/agent.go93](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L93-L93)
    ).
*   **`IsBusy()`**: Checks if `activeRequests` has any entries ([internal/agent/agent.go94](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L94-L94)
    ).

**Sources:** [internal/agent/agent.go93-94](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L93-L94)
