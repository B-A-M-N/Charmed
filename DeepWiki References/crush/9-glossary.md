This glossary defines the technical terms, abbreviations, and domain-specific jargon used throughout the Crush codebase. It serves as a bridge between high-level concepts and the specific implementation details found in the Go source code.

System-Wide Terms
-----------------

### Agent

The high-level entity responsible for processing user prompts, orchestrating tool calls, and generating responses using LLMs.

*   **Code Entities:** `internal/agent.SessionAgent` [internal/agent/agent.go86-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L86-L100)
     `internal/agent.Coordinator` [internal/agent/coordinator.go73-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L73-L87)
    
*   **Implementation:** Crush uses a `Coordinator` to manage multiple sessions, each handled by a `SessionAgent` which wraps the `fantasy.Agent` framework [internal/agent/agent.go206-211](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L206-L211)
     The `Coordinator` also handles model refreshing and OAuth token rotation [internal/agent/coordinator.go171-209](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/coordinator.go#L171-L209)
    

### App

The central orchestrator struct that wires together all services (Sessions, Messages, History, Permissions, FileTracker) and manages the application lifecycle.

*   **Code Entities:** `internal/app.App` [internal/app/app.go54-78](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L54-L78)
     `app.New` [internal/app/app.go84-152](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L84-L152)
    
*   **Implementation:** The `App` struct owns the `AgentCoordinator`, `LSPManager`, and a `pubsub.Broker` for distributing agent notifications to the TUI [internal/app/app.go71-77](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L71-L77)
    

### Session

A persistent conversation context. Sessions can have parent-child relationships (e.g., when a primary agent spawns a sub-agent via the `AgentTool`).

*   **Code Entities:** `internal/session.Session` [internal/session/session.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/session/session.go)
     `internal/session.Service` [internal/session/session.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/session/session.go)
    
*   **Implementation:** Sessions are stored in a SQLite database. The `App` resolves whether to continue an existing session or create a new one during startup [internal/app/app.go184-209](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L184-L209)
    

### Tool

A capability provided to the Agent to interact with the external world (e.g., file system, shell, LSP, MCP).

*   **Code Entities:** `fantasy.AgentTool` [internal/agent/agent.go114](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L114-L114)
     `internal/agent/tools` [internal/agent/tools/](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/)
     (package).
*   **Implementation:** Tools implement the `fantasy.AgentTool` interface, allowing the LLM to call them via JSON-schema defined parameters. Tool execution is gated by the `permission.Service` [internal/app/app.go100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L100-L100)

Configuration & Infrastructure
------------------------------

### ConfigStore

A runtime wrapper around the static `Config` struct that handles dynamic variable resolution, provider setup, and persistence.

*   **Code Entities:** `internal/config.ConfigStore` [internal/config/config.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go)
     `config.Load` [internal/config/load.go35-129](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L35-L129)
    
*   **Implementation:** Manages the merging of global config (`~/.config/crush/crush.json`) and project-local config (`.crush.json`). It also detects terminal-specific capabilities like Apple Terminal's transparent mode [internal/config/load.go89-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L89-L92)
    

### VariableResolver

An interface for resolving environment variables or secrets within configuration fields (e.g., resolving `$OPENAI_API_KEY` from the environment).

*   **Code Entities:** `internal/config.VariableResolver` [internal/config/config.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go)
     `internal/config.ShellVariableResolver` [internal/config/load.go103-104](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L103-L104)
    

### YOLO Mode

A security bypass flag that causes the `PermissionService` to automatically approve all tool execution requests without prompting the user.

*   **Code Entities:** `rootCmd` flag `--yolo` [internal/cmd/root.go58](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L58-L58)
     `config.Permissions.SkipRequests` [internal/app/app.go90-91](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L90-L91)

Subsystem Specifics
-------------------

### LSP (Language Server Protocol)

Used by Crush to provide code intelligence (diagnostics, references) to the Agent.

*   **Code Entities:** `internal/lsp.Manager` [internal/lsp/manager.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/lsp/manager.go)
     `internal/lsp.Client` [internal/lsp/client.go32-66](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/lsp/client.go#L32-L66)
    
*   **Implementation:** Managed by `lsp.Manager`, which tracks configured servers and handles their lifecycle. It uses callbacks to update the UI with diagnostics [internal/app/app.go141-149](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L141-L149)
    

### MCP (Model Context Protocol)

A standard for connecting AI models to external data sources and tools.

*   **Code Entities:** `internal/agent/tools/mcp.Initialize` [internal/agent/tools/mcp/init.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/mcp/init.go)
     `internal/config.MCPConfig` [internal/config/config.go211-255](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L211-L255)
    
*   **Implementation:** Crush supports `stdio`, `sse`, and `http` transports for MCP servers [internal/config/config.go234-243](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L234-L243)
     MCP instructions are injected into the agent's system prompt [internal/agent/agent.go187-199](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agent.go#L187-L199)
    

### Catwalk

The Charm-specific service/library used for managing model provider metadata and configurations.

*   **Code Entities:** `charm.land/catwalk` [go.mod8](https://github.com/charmbracelet/crush/blob/1ceeaa93/go.mod#L8-L8)
     `internal/config.Providers` [internal/config/load.go95-99](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L95-L99)
    

### Hooks

User-defined shell commands that execute before tool usage (`PreToolUse`).

*   **Code Entities:** `internal/hooks.HookConfig` [internal/config/config.go310-322](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L310-L322)
     `internal/agent.HookedTool` [internal/agent/hooked\_tool.go](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hooked_tool.go)
    
*   **Implementation:** Hooks allow blocking or modifying tool inputs via external scripts [internal/config/load.go73-77](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L73-L77)
