Variable Resolution
-------------------

Crush supports dynamic configuration values through a variable resolution system. This allows users to keep secrets out of plain-text config files by referencing environment variables or shell-resolved values.

The `VariableResolver` interface defines how these placeholders are expanded. During configuration loading, a `ShellVariableResolver` is initialized to handle these lookups [internal/config/load.go101-104](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L101-L104)

**Diagram: Configuration Variable Resolution**

### Resolution Flow

1.  **Template Storage**: When a provider is configured, the original value (e.g., `$OPENAI_API_KEY`) is stored in `APIKeyTemplate` [internal/config/config.go101-102](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L101-L102)
    
2.  **Initial Resolution**: The `configureProviders` method uses the `VariableResolver` to populate the active `APIKey` and `ExtraHeaders` [internal/config/load.go234-245](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L234-L245)
    
3.  **Re-resolution**: If an authentication error occurs, Crush can re-resolve the template to pick up updated environment variables without a restart.

Cross-Platform Support
----------------------

Crush is designed for first-class support across macOS, Linux, and Windows. This requires several abstraction layers to handle platform-specific quirks.

### Terminal and Shell Abstractions

*   **Windows Emulation**: On Windows, Crush uses `mvdan.cc/sh/v3` to provide POSIX-like shell behavior for agent tools and hooks [docs/hooks/README.md93-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L93-L100)
    
*   **Apple Terminal**: Crush detects if it is running in Apple Terminal and automatically enables "transparent mode" to improve visual compatibility [internal/config/load.go89-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L89-L92)
    
*   **Line Endings**: The `fsext` package handles normalization of line endings across different operating systems.

### Build Strategy

Crush utilizes a `CGO_ENABLED=0` pure-Go build strategy to ensure maximum portability and ease of installation across different architectures and operating systems. This allows binaries to run on systems without specific C library versions [README.md21-22](https://github.com/charmbracelet/crush/blob/1ceeaa93/README.md?plain=1#L21-L22)

Hooks System
------------

Hooks are user-defined shell scripts that execute during the agent lifecycle. They provide a mechanism to enforce policies, rewrite tool inputs, or inject additional context [docs/hooks/README.md6-12](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L6-L12)

### PreToolUse Execution

The `PreToolUse` hook fires before any tool call is executed. The `hookedTool` wrapper intercepts the `Run` call, executes the configured hooks via the `hooks.Runner`, and processes the result [internal/agent/hooked\_tool.go54-60](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hooked_tool.go#L54-L60)

**Diagram: Hook Interception Flow**

### Hook Protocol

Hooks communicate with Crush via exit codes and JSON on `stdout`:

*   **Exit 0**: Success/Allow.
*   **Exit 2**: Block/Deny the tool call [docs/hooks/README.md84-85](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L84-L85)
    
*   **Exit 49**: Halt the entire agent turn.
*   **JSON Envelope**: Hooks can return an `updated_input` to rewrite the arguments passed to the tool [internal/agent/hooked\_tool.go75-77](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hooked_tool.go#L75-L77)

Project Initialization
----------------------

When Crush is started, it performs several checks to determine the project context and configure itself appropriately.

### Onboarding and Detection

*   **Git Detection**: Crush checks if it is inside a git worktree using `isInsideWorktree()`. If not, it applies more restrictive defaults to tools like `ls` to prevent performance issues in large non-git directories [internal/config/load.go79-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L79-L87)
    
*   **Context Files**: Crush looks for common project instruction files such as `.cursorrules`, `CLAUDE.md`, or `AGENTS.md` (defined in `defaultContextPaths`) to provide the LLM with project-specific context [internal/config/config.go28-45](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L28-L45)
