Purpose and Scope
-----------------

This page covers the local development build system for Crush, managed through [Task](https://taskfile.dev/)
 (Taskfile v3). The build system provides tasks for linting, building, testing, running, profiling, and preparing releases. For CI/CD automation and GitHub Actions workflows, see page 8.3. For the goreleaser-driven release pipeline, see page 8.2.

Sources: [Taskfile.yaml1-216](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L1-L216)

Task Runner Architecture
------------------------

Crush uses [Task](https://taskfile.dev/)
 (Taskfile v3) as its build automation tool, providing a simple alternative to Make with better cross-platform support. All development operations are defined in the root `Taskfile.yaml`.

### Environment Configuration

The build system configures several environment variables that affect compilation behavior:

| Variable | Value | Scope | Purpose |
| --- | --- | --- | --- |
| `CGO_ENABLED` | `0` | Global | Produces static binaries with no C dependencies. [Taskfile.yaml12](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L12-L12)<br> [.goreleaser.yml46](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L46-L46) |
| `GOEXPERIMENT` | `greenteagc` | Global | Enables the GreenTea garbage collector experiment. [Taskfile.yaml13](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L13-L13)<br> [.goreleaser.yml47](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L47-L47) |
| `GOTOOLCHAIN` | `go1.25.0` | `lint:install` | Pins Go version for linter installation. [Taskfile.yaml21](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L21-L21) |

The system also uses dynamic Taskfile variables computed at startup:

*   `VERSION`: Output of `git describe --long`, injected via `-ldflags` into `internal/version.Version`. [Taskfile.yaml6-7](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L6-L7)
     [Taskfile.yaml46](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L46-L46)
    
*   `RACE`: Set to `"1"` when `race.log` exists in the repo root; enables `-race` flag on `build` and `run`. [Taskfile.yaml8-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L8-L9)
    

Sources: [Taskfile.yaml5-22](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L5-L22)
 [.goreleaser.yml44-47](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L44-L47)

Core Development Tasks
----------------------

### Linting

The linting system uses `golangci-lint` with custom configuration and additional checks:

**Task: `lint:install`**

*   Installs `golangci-lint` using the specified Go 1.25.0 toolchain. [Taskfile.yaml16-21](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L16-L21)
    

**Task: `lint`**

*   Runs `golangci-lint` with `.golangci.yml` configuration. [Taskfile.yaml23-27](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L23-L27)
    
*   **Note:** Explicitly nullifies `GOEXPERIMENT` to avoid GreenTea GC during linting to ensure compatibility with the linter. [Taskfile.yaml29](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L29-L29)
    
*   Includes `lint:log` as a prerequisite to check log capitalization. [Taskfile.yaml26](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L26-L26)
    

**Task: `lint:log`**

*   Custom script check to enforce that log messages start with capital letters. [Taskfile.yaml31-34](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L31-L34)
    
*   Script: `./scripts/check_log_capitalization.sh`

**Task: `lint:fix`**

*   Runs `golangci-lint` with the `--fix` flag for automatic correction of issues. [Taskfile.yaml36-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L36-L41)
    

Sources: [Taskfile.yaml16-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L16-L41)

### Building

The `build` task compiles the main application binary with version information embedded:

**Task: `build`**

*   Output: `crush` (or `crush.exe` on Windows). [Taskfile.yaml56](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L56-L56)
    
*   Includes race detector if `RACE=1`. [Taskfile.yaml48](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L48-L48)
    
*   Injects version via ldflags: `-X github.com/charmbracelet/crush/internal/version.Version={{.VERSION}}`. [Taskfile.yaml46](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L46-L46)
     [.goreleaser.yml72-73](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L72-L73)
    
*   Task-level source tracking includes all `.go` files, `go.mod`, agent templates, and the Hyper provider schema. [Taskfile.yaml49-54](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L49-L54)
    

Sources: [Taskfile.yaml43-57](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L43-L57)
 [.goreleaser.yml44-76](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L44-L76)

### Testing

**Task: `test`**

*   Runs all tests with race detection and fail-fast enabled. [Taskfile.yaml82-85](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L82-L85)
    
*   Command: `go test -race -failfast ./... {{.CLI_ARGS}}`

**Task: `test:record`** (alias: `record`)

*   Re-records VCR cassettes for HTTP mocking in agent tests. [Taskfile.yaml87-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L87-L92)
    
*   Removes existing test data: `rm -r internal/agent/testdata`. [Taskfile.yaml91](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L91-L91)
    
*   Runs agent tests with an extended 1-hour timeout. [Taskfile.yaml92](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L92-L92)
    

Sources: [Taskfile.yaml82-92](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L82-L92)

### Running

**Task: `run`**

*   Builds and executes the binary, forwarding arguments. [Taskfile.yaml58-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L58-L62)
    
*   If `RACE=1`, redirects stderr to `race.log`. [Taskfile.yaml62](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L62-L62)
    

**Task: `run:catwalk`**

*   Runs against a local Catwalk instance for provider testing. [Taskfile.yaml64-70](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L64-L70)
    
*   Sets `CATWALK_URL=http://localhost:8080`. [Taskfile.yaml67](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L67-L67)
    

**Task: `run:onboarding`**

*   Tests the onboarding flow by using isolated temporary data and config directories. [Taskfile.yaml72-80](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L72-L80)
    
*   Sets `CRUSH_GLOBAL_DATA` and `CRUSH_GLOBAL_CONFIG` to `tmp/onboarding/`. [Taskfile.yaml75-76](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L75-L76)
    

Sources: [Taskfile.yaml58-80](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L58-L80)

Profiling and Development
-------------------------

### Enabling Profiling

**Task: `dev`**

*   Runs the application with `CRUSH_PROFILE=true`. [Taskfile.yaml109-114](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L109-L114)
    

In `main.go`, the application checks for this environment variable. If present, it starts a goroutine to serve `pprof` endpoints via `http.ListenAndServe("localhost:6060", nil)`. [main.go23-31](https://github.com/charmbracelet/crush/blob/1ceeaa93/main.go#L23-L31)

### Profiling Tasks

Crush provides tasks to capture different profile types and serve them via a web UI on port `6061`: [Taskfile.yaml130-143](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L130-L143)

| Task | Endpoint | Description |
| --- | --- | --- |
| `profile:cpu` | `/debug/pprof/profile?seconds=10` | 10-second CPU profile. |
| `profile:heap` | `/debug/pprof/heap` | Current heap memory profile. |
| `profile:allocs` | `/debug/pprof/allocs` | Memory allocations profile. |

Sources: [main.go23-31](https://github.com/charmbracelet/crush/blob/1ceeaa93/main.go#L23-L31)
 [Taskfile.yaml130-143](https://github.com/charmbracelet/crush/blob/1ceeaa93/Taskfile.yaml#L130-L143)
