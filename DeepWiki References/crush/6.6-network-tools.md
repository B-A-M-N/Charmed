This page documents the network-enabled tools in `crush`: the `fetch` tool for retrieving web content, the `agentic_fetch` tool for AI-powered web research, the `web_fetch` tool for sub-agents, and the `web_search` tool for DuckDuckGo integration. These tools allow the agent to access real-time information and documentation from the internet, converting complex formats like HTML into LLM-friendly Markdown.

* * *

Overview
--------

Network tools extend the agent's reach beyond the local file system. They handle the complexities of HTTP communication, content-type negotiation, and data transformation.

1.  **`fetch`**: A parallel-capable tool that retrieves a URL and converts HTML to Markdown. [internal/agent/tools/fetch.go20-43](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L20-L43)
    
2.  **`agentic_fetch`**: Spawns a sub-agent to search, fetch, and analyze web content to answer complex questions. [internal/agent/agentic\_fetch\_tool.go53-69](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L53-L69)
    
3.  **`web_search`**: Integration with DuckDuckGo to find relevant URLs, returning titles and snippets. [internal/agent/tools/web\_search.go23-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/web_search.go#L23-L62)
    
4.  **`web_fetch`**: A specialized version of fetch used by sub-agents that handles large content by saving to temp files. [internal/agent/tools/web\_fetch.go25-80](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/web_fetch.go#L25-L80)
    
5.  **`download`**: A tool specifically for saving remote assets to the local disk. [internal/agent/tools/download.go33-51](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/download.go#L33-L51)
    
6.  **`sourcegraph`**: Queries the Sourcegraph GraphQL API for code search. [internal/agent/tools/sourcegraph.go30-48](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/sourcegraph.go#L30-L48)

The Fetch Tool (`fetch`)
------------------------

The `fetch` tool is the primary mechanism for the agent to "read" a webpage. It is defined as a `fantasy.NewParallelAgentTool`, allowing multiple fetch requests to execute concurrently. [internal/agent/tools/fetch.go58-61](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L58-L61)

### Request Options and Timeouts

The tool accepts `FetchParams` including the `URL`, a preferred `Format` (`text`, `markdown`, or `html`), and an optional `Timeout`. [internal/agent/tools/fetch\_types.go39-43](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch_types.go#L39-L43)

*   **Default Timeout**: 30 seconds (configured in the `http.Client`). [internal/agent/tools/fetch.go52-56](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L52-L56)
    
*   **Maximum Timeout**: 120 seconds. [internal/agent/tools/fetch.go99-100](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L99-L100)
    
*   **User-Agent**: `crush/1.0`. [internal/agent/tools/fetch.go118](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L118-L118)
    

Sources: [internal/agent/tools/fetch.go52-118](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L52-L118)
 [internal/agent/tools/fetch\_types.go39-43](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch_types.go#L39-L43)

### Content-Type and Conversion Logic

The tool inspects the `Content-Type` header to determine how to process the body:

| Format Requested | Content-Type | Transformation Logic |
| --- | --- | --- |
| `text` | `text/html` | Uses `goquery` to extract text from the `<body>` and collapses whitespace. [internal/agent/tools/fetch.go192-202](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L192-L202) |
| `markdown` | `text/html` | Uses `github.com/JohannesKaufmann/html-to-markdown` to convert DOM to Markdown. [internal/agent/tools/fetch.go204-213](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L204-L213) |
| `html` | `text/html` | Returns only the inner HTML of the `<body>` tag using `goquery`. [internal/agent/tools/fetch.go164-180](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L164-L180) |

Sources: [internal/agent/tools/fetch.go143-180](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L143-L180)
 [internal/agent/tools/fetch.go192-213](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L192-L213)

### Large Content Handling

To prevent context window overflow, `fetch` enforces a `MaxFetchSize` of **100KB**. [internal/agent/tools/fetch.go22](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L22-L22)

*   The response body is read via `io.LimitReader`. [internal/agent/tools/fetch.go130](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L130-L130)
    
*   If the final processed content exceeds this limit, it is truncated. [internal/agent/tools/fetch.go182-185](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/fetch.go#L182-L185)

Agentic Fetch and Web Search
----------------------------

The `agentic_fetch` tool provides a higher-level interface for research. It spawns a sub-agent equipped with specialized tools. [internal/agent/agentic\_fetch\_tool.go181-191](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L181-L191)

### Implementation of `agentic_fetch`

*   **Sub-Agent Tools**: The sub-agent has access to `web_search`, `web_fetch`, `glob`, `grep`, `view`, and `sourcegraph`. [internal/agent/agentic\_fetch\_tool.go165-174](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L165-L174)
    
*   **Modes**:
    *   **URL Mode**: Fetches the provided URL using `FetchURLAndConvert`. [internal/agent/agentic\_fetch\_tool.go112](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L112-L112)
        
    *   **Search Mode**: Instructs the agent to use `web_search` to find relevant info. [internal/agent/agentic\_fetch\_tool.go138](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L138-L138)
        
*   **Large Content Redirection**: If content exceeds `LargeContentThreshold` (50KB), it is saved to a temp file, and the agent is instructed to use `view` and `grep` on that file. [internal/agent/agentic\_fetch\_tool.go117-132](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/agentic_fetch_tool.go#L117-L132)
    

### Web Fetch (`web_fetch`)

Used primarily by sub-agents, `web_fetch` automatically converts HTML to Markdown using `FetchURLAndConvert` and saves content larger than 50KB to temporary files to avoid flooding the sub-agent's context. [internal/agent/tools/web\_fetch.go51-75](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/web_fetch.go#L51-L75)

--------------------------
