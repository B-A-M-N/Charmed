The Hooks system in Crush allows for user-defined shell commands to intercept, validate, and modify agent behavior at key lifecycle events. Currently, the system focuses on the `PreToolUse` event, enabling use cases such as blocking dangerous commands, rewriting tool inputs, and injecting additional context into the LLM's prompt.

Overview
--------

Hooks are implemented as external executables or inline shell commands that communicate with Crush via environment variables and JSON over `stdin`/`stdout`. They are executed through an embedded POSIX shell interpreter, ensuring cross-platform consistency.

### Key Characteristics

*   **Parallel Execution**: All matching hooks for an event run concurrently to minimize latency [internal/hooks/runner.go111-120](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L111-L120)
    
*   **Deterministic Composition**: Results are aggregated based on the order they appear in the configuration files [internal/hooks/runner.go122-123](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L122-L123)
    
*   **Cross-Platform**: Uses `mvdan.cc/sh` for POSIX emulation on Windows and Unix-like systems [internal/shell/shell.go6-10](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L6-L10)
    
*   **Language Agnostic**: Hooks can be written in any language (Bash, Python, Node, Go) as long as they follow the execution contract [docs/hooks/README.md15-17](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L15-L17)

Configuration
-------------

Hooks are configured in `crush.json` or `.crush.json` within the `hooks` object. They can be defined at both global and project levels, with project-level hooks taking precedence during the merge process [internal/config/load.go58-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L58-L71)

Execution Model
---------------

The `Runner` manages the execution of matching hooks for a specific event.

### 1\. Input Payload

Hooks receive data via two channels:

*   **Environment Variables**: Prefixed with `CRUSH_`, providing context like `CRUSH_TOOL_NAME`, `CRUSH_SESSION_ID`, and tool-specific inputs (e.g., `CRUSH_TOOL_INPUT_COMMAND`) [internal/hooks/input.go38-79](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L38-L79)
    
*   **JSON Stdin**: A JSON object containing the `event`, `session_id`, `tool_name`, and the full `tool_input` object [internal/hooks/input.go81-98](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L81-L98)
    

### 2\. Output Envelope

Hooks communicate back to Crush by printing a JSON object to `stdout`:

*   `decision`: `"allow"`, `"deny"`, or `"none"`.
*   `reason`: A string explaining a denial (shown to the user/model).
*   `context`: String or array of strings to inject into the LLM's prompt.
*   `updated_input`: A modified JSON object to replace the original tool input.
*   `halt`: Boolean to stop subsequent agent actions.

Sources: [internal/hooks/input.go157-185](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L157-L185)
 [docs/hooks/README.md234-260](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L234-L260)

### 3\. Exit Code Semantics

While JSON `stdout` is preferred, hooks can use exit codes for quick decisions:

*   **0**: Success / No opinion (proceed to JSON parsing).
*   **2**: Block/Deny (prevents tool execution).
*   **49**: Halt (stops the agent loop entirely).

Sources: [internal/hooks/runner.go221-235](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L221-L235)
 [docs/hooks/README.md213-232](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L213-L232)

Data Flow: Result Aggregation
-----------------------------

When multiple hooks match a single tool call, the `Runner` aggregates results using the following priority logic:

1.  **Halt**: If any hook returns `halt: true`, the final result is halted [internal/hooks/input.go138-141](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L138-L141)
    
2.  **Deny**: If any hook returns a `deny` decision, the tool is blocked. Multiple denial reasons are concatenated [internal/hooks/input.go121-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L121-L125)
    
3.  **Allow**: If no hook denies, but at least one allows, the tool proceeds [internal/hooks/input.go131-133](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L131-L133)
    
4.  **Input Rewrite**: The `updated_input` from the _last_ hook in the config order that provides one is used [internal/hooks/input.go143-146](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L143-L146)
    
5.  **Context**: Context strings from all hooks are concatenated [internal/hooks/input.go148-151](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/input.go#L148-L151)

Integration with Permissions
----------------------------

Hooks run _before_ the standard permission system prompts the user.

*   If a hook returns `deny`, the tool call fails immediately with the provided `reason`, and the user is never prompted.
*   If a hook returns `allow`, it can be used to "auto-approve" commands that would otherwise require a manual prompt, provided the permission system is configured to respect hook decisions [docs/hooks/README.md33-34](https://github.com/charmbracelet/crush/blob/1ceeaa93/docs/hooks/README.md?plain=1#L33-L34)
    

Timeouts and Safety
-------------------

*   **Timeout**: If a hook exceeds its configured `timeout`, the `Runner` cancels its context. It waits for a 1-second `abandonGrace` before abandoning the goroutine and proceeding with "no opinion" [internal/hooks/runner.go16-208](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L16-L208)
    
*   **Isolation**: Hooks run in the embedded shell but do not have access to the `BlockFuncs` that restrict the agent's `bash` tool, as they are considered trusted user configuration [internal/hooks/runner.go160-161](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L160-L161)
