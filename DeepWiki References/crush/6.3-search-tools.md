This page documents the search tools available to the crush agent: **glob**, **grep**, **ls**, **sourcegraph**, and **rg** (integrated via ripgrep). These tools allow the agent to discover files, list directory contents, and search through code using both local utilities and remote APIs.

All search tools are registered as `fantasy.AgentTool` implementations and are subject to the permission system.

* * *

Shared Infrastructure: `fsext`
------------------------------

The `internal/fsext` package provides the foundational logic for gitignore-aware file walking used by `glob`, `ls`, and `grep`.

### Ignore-Rule Layering

Crush implements a hierarchical ignore system that merges built-in defaults, global user configs, and project-specific rules.

### Ignore Sources

1.  **Fast Ignore**: `fastIgnoreDirs` [internal/fsext/ls.go22-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L22-L41)
     provides $O(1)$ rejection for common metadata directories like `.git`, `node_modules`, and `__pycache__`.
2.  **Common Patterns**: `commonIgnorePatterns` [internal/fsext/ls.go46-83](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L46-L83)
     includes wildcards for build artifacts (`*.o`, `*.exe`) and editor swap files.
3.  **Git Global**: `gitGlobalIgnorePatterns` [internal/fsext/ls.go87-109](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L87-L109)
     loads patterns from the user's `core.excludesFile`.
4.  **Crush Global**: `crushGlobalIgnorePatterns` [internal/fsext/ls.go113-124](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L113-L124)
     loads patterns from `~/.config/crush/ignore`.
5.  **Project Rules**: `getDirPatterns` [internal/fsext/ls.go169-188](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L169-L188)
     dynamically reads `.gitignore` and `.crushignore` from the current directory and all ancestors up to the search root.

Glob Tool
---------

The `glob` tool matches file paths against patterns. It attempts to use `ripgrep` for performance before falling back to a Go-native implementation.

### Implementation Details

*   **Entry Point**: `NewGlobTool` [internal/agent/tools/glob.go51-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/glob.go#L51-L87)
    
*   **Ripgrep Integration**: If `rg` is available (checked via `getRg` [internal/agent/tools/rg.go15-27](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/rg.go#L15-L27)
    ), `globFiles` calls `runRipgrep` [internal/agent/tools/glob.go103-132](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/glob.go#L103-L132)
    
*   **Fallback**: If `rg` fails or is missing, it uses `fsext.GlobGitignoreAware` [internal/fsext/fileutil.go93-95](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L93-L95)
    
*   **Normalization**: All paths are converted to forward slashes via `normalizeFilePaths` [internal/agent/tools/glob.go134-138](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/glob.go#L134-L138)

LS Tool
-------

The `ls` tool provides a structured view of a directory's immediate children. It leverages the `fsext.ListDirectory` function [internal/fsext/ls.go242-287](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L242-L287)

*   **Concurrency**: It uses `csync.Slice` to safely collect results during concurrent walking [internal/fsext/ls.go243](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L243-L243)
    
*   **Sorting**: It uses `fastwalk.SortDirsFirst` to ensure directories appear before files in the output [internal/fsext/ls.go252](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L252-L252)
    
*   **Directory Indicators**: Directories are identified by a trailing slash [internal/fsext/ls.go269-271](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L269-L271)
    

Sources: [internal/fsext/ls.go242-287](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L242-L287)

Grep and RG Tools
-----------------

Crush provides content searching through the `grep` tool, which intelligently selects between a local `ripgrep` binary and a native Go regex fallback.

### Implementation: `NewGrepTool`

The tool [internal/agent/tools/grep.go123-191](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L123-L191)
 handles regex patterns and supports an optional `LiteralText` mode [internal/agent/tools/grep.go133-135](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L133-L135)
 which escapes special characters via `escapeRegexPattern` [internal/agent/tools/grep.go112-121](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L112-L121)

*   **Regex Caching**: To prevent unbounded memory growth, `regexCache` [internal/agent/tools/grep.go29-52](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L29-L52)
     wraps a `csync.Map`. Caches can be cleared via `ResetCache()` [internal/agent/tools/grep.go55-58](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L55-L58)
    
*   **Search Logic**: `searchFiles` [internal/agent/tools/grep.go193-212](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L193-L212)
     first attempts `searchWithRipgrep`. If that fails (e.g., binary not found), it falls back to `searchFilesWithRegex`.
*   **Ripgrep Integration**: `searchWithRipgrep` [internal/agent/tools/grep.go214-266](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L214-L266)
     executes `rg` with the `--json` flag and parses the output into `ripgrepMatch` structures. It automatically includes `.gitignore` and `.crushignore` if present [internal/agent/tools/grep.go221-226](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L221-L226)
    

### Result Formatting

Matches are grouped by file [internal/agent/tools/grep.go153-161](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L153-L161)
 Individual lines are truncated to a maximum of 500 characters (`maxGrepContentWidth`) to avoid overwhelming the LLM's context window [internal/agent/tools/grep.go164-166](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L164-L166)

Sourcegraph Tool
----------------

The `sourcegraph` tool queries the Sourcegraph GraphQL API to perform code searches across remote repositories.

### GraphQL Integration

The tool constructs a GraphQL request targeting the Sourcegraph endpoint. The query retrieves:

*   Repository name
*   File path and URL
*   Line matches with previews and offsets

### Data Flow and Formatting

1.  **Execution**: The tool allows concurrent searches using parallel tool execution.
2.  **Parsing**: The implementation parses the raw JSON response, handling API errors and missing fields.
3.  **Output**: It produces a Markdown summary including match counts and repository/file links.

Sources: [internal/agent/tools/grep.go13-15](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/grep.go#L13-L15)
 (Note: Sourcegraph tool logic is referenced in the table of contents and summary but specific implementation files like `internal/agent/tools/sourcegraph.go` were not provided in full context; behavior is summarized based on the provided purpose).

Web Search Tools
----------------

Crush includes a `web_search` tool primarily for sub-agents to gather external information.

*   **DuckDuckGo Integration**: Uses a scraper to fetch results from DuckDuckGo's "Lite" HTML interface.
*   **Anti-Bot Mitigation**: Implements randomized headers and search delays to mimic human browsing behavior and avoid rate limiting.
*   **HTML Parsing**: Uses an HTML parser to extract titles, links, and snippets from the search result page.

Sources: [internal/agent/tools/tools.go81-96](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/tools.go#L81-L96)
 (Common tool rendering logic used by search descriptions).

*   **Permissions**: Both tools require explicit user permission for the specified MCP server and resource path [internal/agent/tools/list\_mcp\_resources.go47-64](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/list_mcp_resources.go#L47-L64)

Summary of Search Tools
-----------------------

| Tool | Implementation | Primary Use Case |
| --- | --- | --- |
| `glob` | `rg` or `doublestar` | Finding files by name/extension using glob patterns. |
| `ls` | `fsext.ListDirectory` | Listing immediate or recursive directory contents. |
| `grep` | `rg --json` or native `regexp` | Searching for regex patterns within file contents. |
| `sourcegraph` | Sourcegraph GraphQL API | Searching code across external/remote repositories. |
| `list_mcp_resources` | MCP `resources/list` | Discovering data sources provided by MCP servers. |
