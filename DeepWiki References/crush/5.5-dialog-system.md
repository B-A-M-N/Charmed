This page documents the modal dialog infrastructure in crush's TUI: the `Dialog` interface, the `Overlay` stack manager, the `Action` dispatch mechanism, and the layout utilities. For documentation of specific dialog implementations (Commands, Models, Sessions, Permissions, etc.), see [5.6 Commands and Models Dialogs](https://github.com/charmbracelet/crush/blob/1ceeaa93/5.6%20Commands%20and%20Models%20Dialogs)
 For the broader TUI architecture that hosts the overlay, see [5.1 TUI Architecture](https://github.com/charmbracelet/crush/blob/1ceeaa93/5.1%20TUI%20Architecture)

* * *

Overview
--------

All modal dialogs in crush live under `internal/ui/dialog/`. The system has three main responsibilities:

1.  **Interface contract** — a `Dialog` interface that every modal panel must implement.
2.  **Stack management** — an `Overlay` that holds an ordered stack of open dialogs and routes messages to the topmost one.
3.  **Layout composition** — a set of `Draw*` functions that position dialog views onto the terminal screen using `ultraviolet`.

The `UI` struct in `internal/ui/model/ui.go` (not provided, but referenced by common patterns) owns a single `*dialog.Overlay` and is responsible for intercepting Bubble Tea messages, forwarding them to the overlay, and dispatching the resulting `Action` values back into the application.

The `Overlay` Stack
-------------------

**Source:** [internal/ui/dialog/dialog.go51-213](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L51-L213)

`Overlay` holds a `[]Dialog` slice. The last element is the _front_ (active) dialog. [internal/ui/dialog/dialog.go51-53](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L51-L53)

**Overlay lifecycle methods:**

| Method | Behavior |
| --- | --- |
| `NewOverlay(dialogs ...Dialog) *Overlay` | Constructs the overlay, optionally with pre-seeded dialogs. [internal/ui/dialog/dialog.go56-60](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L56-L60) |
| `OpenDialog(dialog Dialog)` | Appends a dialog to the back of the stack (makes it front). [internal/ui/dialog/dialog.go78-80](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L78-L80) |
| `CloseDialog(dialogID string)` | Removes the dialog with the given ID from anywhere in the stack. [internal/ui/dialog/dialog.go83-90](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L83-L90) |
| `CloseFrontDialog()` | Removes the last element (the active dialog). [internal/ui/dialog/dialog.go93-98](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L93-L98) |
| `BringToFront(dialogID string)` | Moves a dialog to the end of the slice. [internal/ui/dialog/dialog.go119-128](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L119-L128) |
| `Dialog(dialogID string) Dialog` | Looks up a dialog by ID; returns `nil` if not found. [internal/ui/dialog/dialog.go101-108](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L101-L108) |
| `DialogLast() Dialog` | Returns the front dialog without removing it. [internal/ui/dialog/dialog.go111-116](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L111-L116) |
| `HasDialogs() bool` | Used by the UI to decide whether to intercept events. [internal/ui/dialog/dialog.go63-65](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L63-L65) |

**`Update` routing:** The `Overlay.Update` method forwards the message to the active dialog and returns an `Action`. [internal/ui/dialog/dialog.go131-143](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L131-L143)

**`Draw` iteration:** The `Overlay.Draw` method iterates through all dialogs in the stack, rendering them in order. It returns the cursor position from the frontmost dialog. [internal/ui/dialog/dialog.go199-205](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L199-L205)

Layout and Composition
----------------------

**Source:** [internal/ui/dialog/dialog.go166-196](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L166-L196)
 [internal/ui/common/common.go66-84](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/common.go#L66-L84)

The system provides utility functions for positioning dialogs consistently within the terminal area.

*   **`DrawCenter(scr, area, view)`**: Renders a string view centered in the specified rectangle. [internal/ui/dialog/dialog.go177-179](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L177-L179)
    
*   **`DrawCenterCursor(scr, area, view, cur)`**: Centers the view and adjusts the terminal cursor position relative to the centered coordinates. [internal/ui/dialog/dialog.go166-174](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L166-L174)
    
*   **`DrawOnboardingCursor(scr, area, view, cur)`**: Positions the view at the bottom-left of the screen, used for onboarding and initialization flows. [internal/ui/dialog/dialog.go188-196](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/dialog/dialog.go#L188-L196)
    
*   **`CenterRect(area, width, height)`**: Calculates a `uv.Rectangle` of the given dimensions centered within a parent area. [internal/ui/common/common.go66-74](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/ui/common/common.go#L66-L74)
