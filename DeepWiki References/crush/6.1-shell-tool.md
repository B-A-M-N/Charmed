Purpose and Scope
-----------------

This document covers the `bash` tool and its underlying `shell.Shell` component, which provides cross-platform shell command execution capabilities with POSIX emulation and security controls. It also details the `BackgroundShellManager` for long-running processes, the management tools `job_output` and `job_kill`, and the integration of shell hooks for pre-execution validation.

The shell tool enables AI agents to execute shell commands in a controlled environment, maintaining state across multiple executions while enforcing security policies through command blocking mechanisms.

Overview
--------

The shell tool is implemented as `NewBashTool` [internal/agent/tools/bash.go193-194](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/bash.go#L193-L194)
 and uses the `shell.Shell` component for command execution. Key features include:

*   **POSIX Shell Emulation**: Uses `mvdan.cc/sh/v3` for consistent cross-platform behavior, even on Windows [internal/shell/shell.go6-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L6-L9)
    
*   **State Persistence**: Maintains working directory and environment variables across command executions [internal/shell/shell.go3-5](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L3-L5)
    
*   **Security Controls**: `BlockFunc` mechanism to prevent dangerous command execution [internal/shell/shell.go61-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L61-L62)
    
*   **Background Execution**: Support for long-running jobs that automatically background after a timeout [internal/agent/tools/bash.go26-28](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/bash.go#L26-L28)
    
*   **Streaming Support**: Real-time output streaming for long-running commands via `ExecStream` [internal/shell/shell.go122-127](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L122-L127)
    
*   **Builtin Utilities**: Includes a Go-native implementation of `jq` via `gojq` [internal/shell/run.go130-133](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/run.go#L130-L133)
    
*   **Crush Environment Markers**: Unconditionally sets `CRUSH=1`, `AGENT=crush`, and `AI_AGENT=crush` so child processes can detect the agent [internal/shell/shell.go43-49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L43-L49)

Command Execution Flow
----------------------

The shell supports both buffered and streaming command execution. After each command, it synchronizes the internal state (CWD and Env) with the interpreter's final state via `updateShellFromRunner` [internal/shell/shell.go243-251](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L243-L251)

1.  **Locking**: The `Shell` uses a `sync.Mutex` to ensure that state updates (CWD/Env) are thread-safe [internal/shell/shell.go115-116](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L115-L116)
    
2.  **Interpreter Setup**: `newInterp` constructs an `interp.Runner` configured with standard handlers (builtins, block list, coreutils) [internal/shell/run.go91-99](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/run.go#L91-L99)
    
3.  **Execution**: `execCommon` handles parsing and execution, ensuring the state is updated even if the command fails [internal/shell/shell.go254-266](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L254-L266)
    
4.  **State Management**: `updateShellFromRunner` iterates through `runner.Vars` and persists exported variables back to the `Shell` instance [internal/shell/shell.go243-251](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L243-L251)

Security and Command Blocking
-----------------------------

Security is enforced via `BlockFunc`, which inspects command arguments before the interpreter executes them [internal/shell/run.go144-158](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/run.go#L144-L158)

### Helper Blockers

*   **`CommandsBlocker(cmds []string)`**: Creates a filter that blocks exact matches of the primary command name using a lookup set [internal/shell/shell.go184-197](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L184-L197)
    
*   **`ArgumentsBlocker(cmd, args, flags)`**: A granular filter that blocks specific subcommands or flags by splitting input into arguments and flags and performing subset matching [internal/shell/shell.go199-216](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L199-L216)
    

### Banned Commands List

The `bash` tool includes a comprehensive list of banned commands including network tools (`curl`, `ssh`, `wget`), system administration (`sudo`, `su`), and package managers (`apt`, `pacman`, `yum`) [internal/agent/tools/bash.go72-143](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/bash.go#L72-L143)
 These are wired into the shell via `blockFuncs()` [internal/agent/tools/bash.go161-191](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/bash.go#L161-L191)

### Background Logic

*   **`Start`**: Launches a command in a separate goroutine using `ExecStream`. It returns a unique 3-digit hex ID generated from an atomic counter [internal/shell/background.go89-129](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/background.go#L89-L129)
    
*   **`MaxBackgroundJobs`**: Limited to 50 concurrent jobs to prevent resource exhaustion [internal/shell/background.go17-93](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/background.go#L17-L93)
    
*   **`AutoBackgroundAfter`**: In the `bash` tool, if a command takes longer than a specific duration (default 60s), it is automatically moved to a background job [internal/agent/tools/bash.go27-51](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/bash.go#L27-L51)

*   **`KillAll`**: Gracefully terminates all active background shells during application shutdown, respecting a context deadline [internal/shell/background.go196-211](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/background.go#L196-L211)

Hook Integration
----------------

The shell execution environment is shared with the hooks system. The `hooks.Runner` uses `shell.Run` for stateless execution of user-defined hook commands [internal/hooks/runner.go26-187](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L26-L187)

*   **Shared Environment**: Hooks receive the same `CrushEnvMarkers` as the `bash` tool, allowing hook-authored scripts to detect they are running under Crush [internal/hooks/hooks\_test.go199-205](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/hooks_test.go#L199-L205)
    
*   **Timeout and Abandonment**: Hooks use a `context.WithTimeout`. If a hook does not yield after cancellation, it is abandoned after a 1-second `abandonGrace` [internal/hooks/runner.go16-207](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L16-L207)
    
*   **Result Aggregation**: Multiple hooks can run in parallel; their results (Decision, Reason, Context) are aggregated into a single `AggregateResult` [internal/hooks/runner.go110-122](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/hooks/runner.go#L110-L122)

POSIX Emulation and Windows Support
-----------------------------------

Crush provides a consistent shell environment across OSs by using a pure-Go interpreter.

*   **`mvdan/sh` Interpreter**: Provides Bash-compatible behavior on all platforms, including Windows [internal/shell/shell.go6-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L6-L9)
    
*   **Path Normalization**: Commands are encouraged to use forward slashes (/) as path separators to work correctly on all platforms [internal/shell/shell.go8-9](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/shell.go#L8-L9)
    
*   **Go CoreUtils**: On Windows, Crush defaults to using pure-Go implementations of common core utilities (ls, cp, mv, etc.) via `mvdan.cc/sh/moreinterp/coreutils` [internal/shell/coreutils.go9-19](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/coreutils.go#L9-L19)

Management Tools
----------------

### job\_output

Retrieves output from a background shell. It can optionally wait for the process to complete before returning.

*   **Implementation**: Calls `GetOutput()` on the `BackgroundShell` which returns `stdout`, `stderr`, and the `done` status [internal/agent/tools/job\_output.go52-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/job_output.go#L52-L71)
    
*   **Parameters**: `shell_id` (string), `wait` (boolean) [internal/agent/tools/job\_output.go20-23](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/job_output.go#L20-L23)
    

### job\_kill

Terminates a background process.

*   **Implementation**: Calls `Kill(id)` on the `BackgroundShellManager`, which invokes the shell's `cancel()` function and waits for the `done` channel [internal/shell/background.go147-156](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/shell/background.go#L147-L156)
    
*   **Tool Wrapper**: Wrapped as `NewJobKillTool` for agent use [internal/agent/tools/job\_kill.go27-38](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/tools/job_kill.go#L27-L38)
