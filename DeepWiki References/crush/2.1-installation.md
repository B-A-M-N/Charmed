This page documents the supported installation methods for Crush, the build system configuration, and the release pipeline. Crush is a terminal-based AI assistant designed for cross-platform compatibility, supporting major package managers and manual builds from source.

* * *

Installation Methods
--------------------

Crush uses **GoReleaser** as its authoritative release engine, as defined in [.goreleaser.yml1-349](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L1-L349)
 The build pipeline produces statically linked binaries (`CGO_ENABLED=0`) with the `greenteagc` garbage collection experiment enabled for optimized performance [.goreleaser.yml45-47](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L45-L47)

### Package Managers

| Platform | Method | Command / Reference |
| --- | --- | --- |
| **macOS / Linux** | Homebrew | `brew install charmbracelet/tap/crush` [.goreleaser.yml186-201](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L186-L201) |
| **Windows** | Scoop | `scoop bucket add charmbracelet https://github.com/charmbracelet/scoop-bucket`  <br>`scoop install crush` [.goreleaser.yml202-211](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L202-L211) |
| **Windows** | Winget | Distributed via `winget` using standard Charm publishing [.goreleaser.yml262-275](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L262-L275) |
| **Arch Linux** | AUR | `yay -S crush` (source) [.goreleaser.yml99-142](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L99-L142)<br> or `yay -S crush-bin` (binary) [.goreleaser.yml143-180](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L143-L180) |
| **Cross-platform** | npm | `npm install -g @charmland/crush` [.goreleaser.yml212-217](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L212-L217) |
| **Nix** | Nixpkgs | `nix-shell -p crush` (via `charmbracelet/nur`) [.goreleaser.yml276-299](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L276-L299) |
| **Linux (Native)** | nfpm | `.deb`, `.rpm`, and `.apk` packages [.goreleaser.yml219-242](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L219-L242) |

### Distribution Targets

The release matrix covers a wide array of operating systems and architectures:

*   **OS**: Linux, Darwin (macOS), Windows, FreeBSD, OpenBSD, NetBSD, Android [.goreleaser.yml48-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L48-L55)
    
*   **Arch**: amd64, arm64, 386, arm (v7) [.goreleaser.yml56-62](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L56-L62)
    

Native Linux packages are generated using **nfpm**, producing `.deb`, `.rpm`, and `.apk` files [.goreleaser.yml219-242](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L219-L242)

Building from Source
--------------------

To build Crush manually, you can use the provided `Taskfile.yaml` or standard Go commands.

### Using Task

The project uses `go-task` for development automation. The `install` task handles fetching tags and building with the correct `ldflags` [.Taskfile.yaml116-122](https://github.com/charmbracelet/crush/blob/1ceeaa93/.Taskfile.yaml#L116-L122)

### Manual Go Build

The version string is injected into `github.com/charmbracelet/crush/internal/version.Version` via `ldflags` during the build process [.goreleaser.yml73](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L73-L73)
 [.Taskfile.yaml46](https://github.com/charmbracelet/crush/blob/1ceeaa93/.Taskfile.yaml#L46-L46)

### Build Configuration Details

The authoritative build configuration in [.goreleaser.yml44-76](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L44-L76)
 and [.Taskfile.yaml11-14](https://github.com/charmbracelet/crush/blob/1ceeaa93/.Taskfile.yaml#L11-L14)
 specifies:

*   **CGO\_ENABLED=0**: Ensures the binary is statically linked and portable.
*   **GOEXPERIMENT=greenteagc**: Uses an optimized garbage collection strategy.
*   **Trimpath**: Removes local file system paths from the compiled binary for privacy and reproducibility [.goreleaser.yml75](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L75-L75)

Release Pipeline and Automation
-------------------------------

The release process is automated via GitHub Actions, specifically the `goreleaser` workflow [.github/workflows/release.yml3](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/release.yml#L3-L3)

### 1\. Release Trigger

The workflow is triggered by pushing a semantic version tag (e.g., `v1.2.3`) [.github/workflows/release.yml5-8](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/release.yml#L5-L8)
 A `release` task in `Taskfile.yaml` automates the tagging and pushing process, ensuring CI checks pass before a tag is created [.Taskfile.yaml160-181](https://github.com/charmbracelet/crush/blob/1ceeaa93/.Taskfile.yaml#L160-L181)

### 2\. Nightly Builds

Nightly releases are published every day at midnight if there are new changes [.github/workflows/nightly.yml1-32](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/nightly.yml#L1-L32)
 These are versioned with a `-nightly` suffix [.goreleaser.yml21-24](https://github.com/charmbracelet/crush/blob/1ceeaa93/.goreleaser.yml#L21-L24)

### 3\. macOS Signing and Notarization

For Darwin targets, the pipeline performs code signing using entitlements specified in `.github/entitlements.plist` [.github/workflows/release.yml19](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/workflows/release.yml#L19-L19)
 This enables `com.apple.security.cs.allow-unsigned-executable-memory` [.github/entitlements.plist5-7](https://github.com/charmbracelet/crush/blob/1ceeaa93/.github/entitlements.plist#L5-L7)
