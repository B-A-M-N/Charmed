This page guides users through configuring AI providers (OpenAI, Anthropic, Google, etc.), setting up authentication (API keys, OAuth), and selecting models for use with Crush.

**Key Topics:**

*   Provider authentication methods (environment variables, config files, OAuth)
*   Provider-specific setup (Hyper, Copilot, Custom OpenAI-compat)
*   Model selection and parameter configuration
*   Provider auto-update and caching mechanism

For information about the overall configuration system and file loading, see [Configuration Basics](https://deepwiki.com/charmbracelet/crush/2.3-configuration-basics)
.

Provider Types
--------------

Crush supports multiple provider types. The `type` field in `ProviderConfig` determines the API format used for communication:

| Type | Description | Use Case |
| --- | --- | --- |
| `openai` | OpenAI native format | OpenAI API |
| `openai-compat` | OpenAI-compatible format | Local models (Ollama, LM Studio), Groq, OpenRouter |
| `anthropic` | Anthropic native format | Anthropic API, Claude models |
| `gemini` | Google Gemini format | Google Gemini API |
| `azure` | Azure OpenAI format | Azure-hosted models |
| `vertexai` | Google Vertex AI format | GCP-hosted models |
| `hyper` | Charm Hyper format | Proxied requests to Charm's Hyper service |

**Sources:** [internal/config/config.go97-98](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L97-L98)
 [internal/agent/hyper/provider.json5](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.json#L5-L5)

Authentication Methods
----------------------

### Standard Environment Variables

Crush recognizes standard environment variables for common providers. If these are set, they are automatically mapped to providers during `config.Load()`.

| Variable | Provider |
| --- | --- |
| `HYPER_API_KEY` | Charm Hyper |
| `ANTHROPIC_API_KEY` | Anthropic |
| `OPENAI_API_KEY` | OpenAI |
| `GEMINI_API_KEY` | Google Gemini |
| `OPENROUTER_API_KEY` | OpenRouter |
| `HF_TOKEN` | Hugging Face |

**Sources:** [README.md182-200](https://github.com/charmbracelet/crush/blob/1ceeaa93/README.md?plain=1#L182-L200)
 [internal/config/load.go101-113](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L101-L113)

### Variable Resolution

Crush uses a `shellVariableResolver` to handle secrets in configuration files. This allows users to reference environment variables via `$VAR` or `${VAR:?msg}` syntax. The resolver delegates to `shell.ExpandValue`, supporting bash-style substitution and command expansion.

**Sources:** [internal/config/resolve.go52-87](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L52-L87)

Provider Auto-Update Mechanism
------------------------------

Crush maintains model metadata (pricing, context windows, capabilities) via an auto-update mechanism that fetches data from **Catwalk** and **Hyper**.

### The `Providers()` Function

The `Providers(cfg *Config)` function manages this lifecycle:

1.  **Check Options**: Respects `DisableProviderAutoUpdate` and `DisableDefaultProviders` [internal/config/provider.go144-145](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L144-L145)
    
2.  **Concurrent Fetch**: Uses a `sync.WaitGroup` to fetch from Catwalk (via `catwalkSyncer`) and the Hyper API (via `hyperSyncer`) simultaneously [internal/config/provider.go153-187](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L153-L187)
    
3.  **Caching**: Fetched data is cached locally in JSON files using the `cache[T]` struct [internal/config/provider.go199-221](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L199-L221)
    
4.  **Fallback**: If remote fetching fails, it falls back to the local cache or embedded providers bundled with the binary [internal/config/provider.go165-180](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L165-L180)
    

### Cache Locations

Cache files are stored in platform-specific directories determined by `cachePathFor`:

*   **Linux/macOS**: `$HOME/.local/share/crush/`
*   **Windows**: `%LOCALAPPDATA%/crush/`

**Sources:** [internal/config/provider.go37-55](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L37-L55)
 [internal/config/provider.go139-197](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/provider.go#L139-L197)

Specific Provider Setup Notes
-----------------------------

### Charm Hyper

Hyper is a meta-provider proxying various models. It is the default provider if no others are configured. It supports credit tracking via the `/v1/credits` endpoint.

*   **Base URL**: `https://hyper.charm.land`
*   **Environment**: `HYPER_API_KEY` or `HYPER_URL` for custom endpoints.

**Sources:** [internal/agent/hyper/provider.go36-52](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.go#L36-L52)
 [internal/agent/hyper/provider.json1-8](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/agent/hyper/provider.json#L1-L8)

### GitHub Copilot

Authentication for Copilot is handled via `crush login copilot`. Crush uses a specialized token refresh mechanism and checks for existing tokens on disk before prompting for device authentication.

**Sources:** [internal/cmd/login.go149-218](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/login.go#L149-L218)
 [internal/config/config.go18](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L18-L18)

### Custom OpenAI-Compatible

For local models (e.g., Ollama), set the `type` to `openai-compat` and provide a `base_url`. Users can manually define the `models` list in the config to override or extend discovered models.

**Sources:** [internal/config/config.go90-138](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L90-L138)
 [internal/config/load.go184-220](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L184-L220)
