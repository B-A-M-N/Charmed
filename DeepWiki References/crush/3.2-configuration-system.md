This page covers the internal mechanics of crush's configuration system: how config files are discovered and merged, the full type hierarchy of the `Config` struct, how environment variables are resolved inside config values, and how individual fields are persisted back to disk at runtime.

For a user-facing overview of the config file format and common options, see page [2.3](https://github.com/charmbracelet/crush/blob/1ceeaa93/2.3)
 For provider-specific setup (API keys, OAuth, etc.), see page [2.4](https://github.com/charmbracelet/crush/blob/1ceeaa93/2.4)
 For the JSON schema generation process, see page [8.6](https://github.com/charmbracelet/crush/blob/1ceeaa93/8.6)

Purpose and Scope
-----------------

The config package (`internal/config`) is responsible for:

1.  **Discovery**: finding all applicable config files from global through project-level [internal/config/load.go34-36](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L34-L36)
    
2.  **Merging**: combining multiple JSON files with predictable priority order using `loadFromBytes` which wraps `go-jsons` [internal/config/load.go38-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L38-L41)
     [internal/config/load.go62-63](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L62-L63)
    
3.  **Defaults**: applying runtime defaults and environment-variable overrides via `setDefaults()` [internal/config/load.go43](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L43-L43)
    
4.  **Variable Resolution**: expanding shell-style references like `$OPENAI_API_KEY` and `$(cmd)` in config values using `VariableResolver` [internal/config/resolve.go16-18](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/resolve.go#L16-L18)
    
5.  **Runtime Mutation**: writing individual fields back to the data-dir config without overwriting user files via `SetConfigField` [internal/config/store.go128-130](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/store.go#L128-L130)
    
6.  **Provider Bootstrapping**: loading the known-provider catalog from Catwalk and configuring active providers [internal/config/load.go95-113](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L95-L113)

Config File Locations and Priority
----------------------------------

Config is loaded by `Load()` in [internal/config/load.go33-129](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L33-L129)
 which initializes a `ConfigStore` [internal/config/store.go40-53](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/store.go#L40-L53)
 that owns both the pure-data `Config` and runtime state.

**Load order (lowest to highest priority)**

| Priority | Path | Purpose |
| --- | --- | --- |
| 1 (lowest) | `~/.config/crush/crush.json` | Global user config (via `GlobalConfig()`) [internal/config/load\_test.go127](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load_test.go#L127-L127) |
| 2   | `crush.json` found walking up from CWD | Project config (via `lookupConfigs()`) [internal/config/load.go36](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L36-L36) |
| 3   | `.crush.json` found walking up from CWD | Project-local config (via `lookupConfigs()`) [internal/config/load.go36](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L36-L36) |
| 4 (highest) | `$DATA_DIR/crush.json` | Workspace-specific overrides [internal/config/load.go49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L49-L49)<br> [internal/config/load.go58-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L58-L71) |

`lookupConfigs()` [internal/config/load.go36](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L36-L36)
 collects applicable files, and `Load` applies them in sequence. Workspace data is loaded last via `os.ReadFile(store.workspacePath)` and merged to ensure it has highest priority [internal/config/load.go58-71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L58-L71)

`SelectedModel` and `SelectedModelType`
---------------------------------------

`SelectedModelType` [internal/config/config.go47-57](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L47-L57)
 is a string type with two recognized values:

*   `SelectedModelTypeLarge` = `"large"`
*   `SelectedModelTypeSmall` = `"small"`

`SelectedModel` [internal/config/config.go64-88](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L64-L88)
 holds the active model choice for one slot. The `Model` and `Provider` fields are required; sampling parameters like `Temperature` are optional pointers to distinguish unset values from explicit zeros [internal/config/config.go80](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L80-L80)

The `configureSelectedModels()` method [internal/config/load.go120-122](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/load.go#L120-L122)
 reconciles user preferences against the provider catalog, falling back to provider defaults if needed.

Convenience accessors on `Config`:

*   `LargeModel()`: returns the `catwalk.Model` for the large slot [internal/config/config.go428-430](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L428-L430)
    
*   `SmallModel()`: returns the `catwalk.Model` for the small slot [internal/config/config.go432-434](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/config/config.go#L432-L434)
