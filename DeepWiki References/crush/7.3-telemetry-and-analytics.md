This document describes Crush's telemetry and analytics system, which collects usage data and error reports to improve the product. The system uses PostHog for event tracking and provides privacy controls to allow users to opt out. It also details the `crush stats` command, which provides a local, private dashboard for personal usage statistics.

For information about configuration options that control telemetry behavior, see page 3.2 (Configuration System) and page 2.3 (Configuration Basics).

* * *

Overview
--------

Crush includes an optional telemetry system implemented in the `internal/event` package that collects:

*   Application lifecycle events (initialization, exit)
*   Error reports with type information and messages
*   System metadata (OS, architecture, shell, terminal type)
*   Version information
*   Feature usage (session management, file picker, stats viewing)

All telemetry is **opt-out**. If the `DisableMetrics` option is set in the configuration, the telemetry client is never initialized. No personally identifiable information (PII) is collected—users are identified by a hashed machine ID generated via `getDistinctId()`.

Device Identification
---------------------

### Distinct ID Generation

Each device is assigned a distinct ID used to track events across sessions without identifying the user. The identification system uses a multi-tier fallback strategy in `internal/event/identifier.go`.

**Distinct ID Generation Flow**

**Sources:** [internal/event/identifier.go20-49](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/event/identifier.go#L20-L49)

### User Aliasing

The system supports aliasing machine IDs to user IDs (e.g., after an OAuth login event) using `event.Alias(userID)`. This function ensures that if a user logs in, their anonymous machine activity can be linked to their account.

| Event Name | Trigger Site | Purpose |
| --- | --- | --- |
| `app initialized` | `AppInitialized()` | Start of a session |

| `stats viewed` | `StatsViewed()` | Tracks usage of the `stats` command |
| `session created` | `SessionCreated()` | Tracks new session creation |
| `session listed` | `SessionListed(json)` | Tracks session management via CLI |
| `session renamed` | `SessionRenamed(json)` | Tracks session metadata updates |

Usage Statistics (crush stats)
------------------------------

While PostHog handles aggregate telemetry, the `crush stats` command provides a local-first, private view of a user's own data. It queries the local SQLite database to generate an HTML dashboard.

### Data Flow for Stats

**Sources:** [internal/cmd/stats.go123-176](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L123-L176)
 [internal/cmd/stats.go181-240](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L181-L240)

### Key Components

*   **`gatherStats(ctx, conn)`**: Aggregates data from the local database using `db.Queries`. It calculates total tokens, cost, and usage patterns by day, model, and hour. [internal/cmd/stats.go181-240](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L181-L240)
    
*   **Embedded Assets**: The HTML (`index.html`), CSS (`index.css`), and JS (`index.js`) for the dashboard are embedded into the binary using `go:embed`. [internal/cmd/stats.go24-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L24-L41)
    
*   **HTML Generation**: The `generateHTML` function uses `html/template` to inject the gathered `Stats` struct as JSON into the dashboard template. [internal/cmd/stats.go167-169](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L167-L169)
    
*   **Dashboard Features**:
    *   **Activity Heatmap**: Visualizes session frequency by hour and day. [internal/cmd/stats.go63](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L63-L63)
        
    *   **Tool Usage**: Tracks how often different agent tools (e.g., `bash`, `edit`) are called. [internal/cmd/stats.go112-115](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L112-L115)
        
    *   **Cost Estimation**: Calculates total cost based on token usage per model stored in the DB. [internal/cmd/stats.go71](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/stats.go#L71-L71)

Event Flushing and Shutdown
---------------------------

To ensure events are not lost when the CLI exits, the `event.Flush()` function is invoked during the application shutdown sequence via `AppExited()`. This closes the PostHog client and waits for the configured timeout to send pending events.

**Sources:** [internal/event/event.go118-125](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/event/event.go#L118-L125)
