Purpose and Scope
-----------------

The `fsext` package provides file system utilities for discovering and filtering files in a manner consistent with git's behavior. The package implements gitignore-aware directory traversal, glob pattern matching with `**` support, and cross-platform path utilities.

For file manipulation tools that use these utilities, see [File System Tools](https://deepwiki.com/charmbracelet/crush/6.2-file-system-tools)
. For database-backed file tracking, see [Database and Services](https://deepwiki.com/charmbracelet/crush/3.3-database-and-services)
.

Directory Listing
-----------------

### ListDirectory Function

`ListDirectory` performs depth-limited directory traversal using `fastwalk`. It applies hierarchical ignore rules discovered during the walk.

**Parameters:**

| Parameter | Type | Purpose |
| --- | --- | --- |
| `initialPath` | string | Root directory to list |
| `ignorePatterns` | \[\]string | Additional glob patterns to ignore |
| `depth` | int | Maximum traversal depth (-1 for unlimited) |
| `limit` | int | Maximum results (0 for unlimited) |

Glob Pattern Matching
---------------------

### Public API

`Glob()` and `GlobGitignoreAware()` delegate to `globWithDoubleStar()`. This implementation uses `fastwalk` for efficient traversal and `doublestar` for pattern matching. Results are absolute paths sorted by modification time (newest first) [internal/fsext/fileutil.go154-156](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L154-L156)

---------------------

### directoryLister

`directoryLister` manages hierarchical ignore patterns with per-directory caching using `csync.Map` [internal/fsext/ls.go143-146](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L143-L146)

| Method | Purpose |

| `getDirPatterns(dir)` | Reads `.gitignore` and `.crushignore` from `dir` [internal/fsext/ls.go169-188](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L169-L188) |
| `getCombinedMatcher(dir)` | Assembles a `gitignore.Matcher` from common + global + ancestor patterns [internal/fsext/ls.go193-222](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L193-L222) |
| `shouldIgnore(path, ...)` | Main ignore decision; uses `fastIgnoreDirs` for O(1) lookups [internal/fsext/ls.go226-232](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L226-L232) |

### Fast Ignore Directories

The `fastIgnoreDirs` map provides O(1) lookup for commonly ignored directories to avoid expensive pattern matching. It includes entries like `.git`, `node_modules`, `.crush`, and `__pycache__` [internal/fsext/ls.go22-41](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L22-L41)

### Global and Common Patterns

*   **Common**: Hardcoded patterns like `*.swp`, `dist`, `bin`, and lockfiles [internal/fsext/ls.go46-83](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L46-L83)
    
*   **Git Global**: Patterns from git's global excludes file, usually found via `core.excludesFile` or `~/.config/git/ignore` [internal/fsext/ls.go87-109](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L87-L109)
    
*   **Crush Global**: Patterns from `~/.config/crush/ignore` [internal/fsext/ls.go113-124](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/ls.go#L113-L124)

Path and Content Utilities
--------------------------

### Path Display

*   **PrettyPath**: Uses `home.Short()` to replace the home directory with `~` [internal/fsext/fileutil.go175-177](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L175-L177)
    
*   **DirTrim**: Abbreviates paths for TUI display, preserving the last directory while shortening parents to initials (e.g., `~/.../c/internal/app`) [internal/fsext/fileutil.go179-201](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L179-L201)
    
*   **HasPrefix**: Checks if a path is within a prefix using `filepath.Rel` to ensure the path doesn't escape the prefix via `..` [internal/fsext/fileutil.go214-221](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L214-L221)
    

### Line Ending Normalization

*   **ToUnixLineEndings**: Replaces Windows-style `\r\n` with Unix `\n` [internal/fsext/fileutil.go224-229](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L224-L229)
    
*   **ToWindowsLineEndings**: Replaces Unix-style `\n` with Windows `\r\n` [internal/fsext/fileutil.go231-237](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/fileutil.go#L231-L237)
    

### Paste Buffer Parsing

`ParsePastedFiles` handles file paths pasted into the terminal. It detects the environment (Windows Terminal vs. Unix) and applies specific parsing logic to handle spaces and quotes [internal/fsext/paste.go8-22](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/paste.go#L8-L22)

*   **Windows Terminal**: Handles space-separated paths enclosed in double quotes [internal/fsext/paste.go33-78](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/paste.go#L33-L78)
    
*   **Unix**: Handles space-separated paths with backslash escapes [internal/fsext/paste.go80-129](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/paste.go#L80-L129)
    

### Home Directory Utilities

*   `Config()`: Returns the user config directory, respecting `XDG_CONFIG_HOME` [internal/home/home.go26-31](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/home/home.go#L26-L31)
    
*   `Short(p)`: Replaces the home directory path with `~` [internal/home/home.go34-39](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/home/home.go#L34-L39)

Traversal and Lookup
--------------------

The `fsext` package provides utilities for walking up the file system tree to locate specific files or boundaries.

| Function | Logic |
| --- | --- |
| `Lookup()` | Searches for targets starting from a directory and walking up to the root [internal/fsext/lookup.go19-51](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/lookup.go#L19-L51) |
| `LookupClosest()` | Similar to `Lookup` but returns only the first match and stops at the home directory [internal/fsext/lookup.go59-83](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/lookup.go#L59-L83) |
| `LookupClosestBounded()` | Constrains the upward search to a `stopDir` boundary [internal/fsext/lookup.go96-120](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/lookup.go#L96-L120) |
| `traverseUp()` | Internal engine for upward walking; provides absolute paths and owner IDs to callbacks [internal/fsext/lookup.go163-192](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/lookup.go#L163-L192) |

### Ownership Boundaries

The lookup utilities check file ownership to ensure searches do not cross security or environment boundaries (e.g., walking into a directory owned by another user) [internal/fsext/lookup.go169-172](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/lookup.go#L169-L172)

*   **Unix**: Uses `syscall.Stat_t` to retrieve the UID [internal/fsext/owner\_others.go12-24](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/owner_others.go#L12-L24)
    
*   **Windows**: Always returns `-1` as UID is not applicable in the same way [internal/fsext/owner\_windows.go9-15](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/fsext/owner_windows.go#L9-L15)
