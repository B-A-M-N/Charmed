This page traces the complete startup and shutdown sequence of `crush`: from `main()` through configuration loading, database initialization, service wiring, and into either interactive TUI mode or non-interactive run mode, ending with graceful teardown.

For detail on configuration loading internals, see **3.2 Configuration System**. For the database schema and migration system, see **3.3 Database and Services**. For the event bus wiring that `setupEvents()` produces, see **3.4 Event System**.

* * *

Overview
--------

The lifecycle has two execution paths that share a common setup phase:

| Mode | Entry | Output |
| --- | --- | --- |
| Interactive (TUI) | `crush` | Bubble Tea program |
| Non-interactive | `crush run <prompt>` | Streamed text to stdout |

Both paths utilize the `App` struct as the central orchestrator [internal/app/app.go54-78](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L54-L78)

### Step 1: `main()` and `Execute()`

`main()` optionally starts a pprof HTTP server if `CRUSH_PROFILE` is set, then calls `cmd.Execute()` [main.go24-35](https://github.com/charmbracelet/crush/blob/1ceeaa93/main.go#L24-L35)

`Execute()` uses `fang.Execute()` to dispatch to subcommands [internal/cmd/root.go189-197](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L189-L197)
 It handles colored version output via a custom template and sets up signal handling for `os.Interrupt`.

Sources: [main.go24-35](https://github.com/charmbracelet/crush/blob/1ceeaa93/main.go#L24-L35)
 [internal/cmd/root.go165-197](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L165-L197)

### Step 2: `setupWorkspace()` — Common Initialization

The initialization logic is encapsulated in `setupLocalWorkspace` (called by `setupWorkspaceWithProgressBar`) [internal/cmd/root.go218-248](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L218-L248)
:

1.  **Config Loading**: Calls `config.Load()` to merge global and project-level configurations [internal/cmd/root.go252-264](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L252-L264)
    
2.  **Database Connection**: Calls `db.Connect()`, which ensures the SQLite database exists and runs `goose` migrations [internal/cmd/root.go266-274](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L266-L274)
    
3.  **Skill Discovery**: Discovers workspace skills via `skills.DiscoverFromConfig` [internal/cmd/root.go276-281](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L276-L281)
    
4.  **App Construction**: Calls `app.New()` to wire services [internal/cmd/root.go283-287](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L283-L287)
    
5.  **Telemetry**: Initializes the `event` package for metrics if not disabled [internal/cmd/root.go289-291](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L289-L291)
    

Sources: [internal/cmd/root.go218-291](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/cmd/root.go#L218-L291)

After constructing services, `app.New()` performs background tasks:

1.  **Event Setup**: Calls `app.setupEvents()` to wire internal service events to the `events` channel [internal/app/app.go115](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L115-L115)

4.  **LSP Tracking**: Starts `LSPManager.TrackConfigured()` to monitor configured servers [internal/app/app.go149](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L149-L149)

### Step 4b: Non-Interactive Path (`crush run`)

In `runCmd.RunE` (from `internal/cmd/run.go` - logic referenced in `App`):

1.  **Prompt Resolution**: Reads prompt from arguments or stdin.
2.  **Execution**: Calls `app.RunNonInteractive()` [internal/app/app.go213](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L213-L213)
    
3.  **Lifecycle**: `RunNonInteractive` waits for the agent to become ready [internal/app/app.go225](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L225-L225)
     resolves the session [internal/app/app.go184-209](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L184-L209)
     and runs the agent coordinator [internal/app/app.go270](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L270-L270)

Shutdown Sequence
-----------------

Shutdown is managed by `app.Shutdown()` [internal/app/app.go547-590](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L547-L590)
 ensuring graceful teardown of active components.

**Shutdown Sequence**

1.  **Agent Cancellation**: Call to `AgentCoordinator.CancelAll()` to stop LLM requests [internal/app/app.go558](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L558-L558)
    
2.  **Cleanup Functions**: Executes registered cleanup functions including `db.Release` and `mcp.Close` [internal/app/app.go125-129](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L125-L129)
    
3.  **LSP Termination**: Calls `LSPManager.KillAll()` to stop language servers [internal/app/app.go565](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L565-L565)
    
4.  **Parallel Teardown**: Uses a 5-second timeout context to ensure processes exit [internal/app/app.go564-589](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L564-L589)
    

Sources: [internal/app/app.go547-590](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L547-L590)

Event Bus Setup
---------------

`app.setupEvents()` wires service-level publishers to the application's internal `events` channel [internal/app/app.go426-442](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L426-L442)
 This allows the TUI or non-interactive runners to receive typed messages from the backend services.

**Key Event Sources:**

*   **Sessions**: Updates to session state or titles [internal/app/app.go446-450](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L446-L450)
    
*   **Messages**: New messages or tool execution results [internal/app/app.go451-455](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L451-L455)
    
*   **Permissions**: Requests for user approval [internal/app/app.go456-465](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L456-L465)
    
*   **LSP/MCP**: Diagnostic updates and connection status [internal/app/app.go477-489](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/app/app.go#L477-L489)
