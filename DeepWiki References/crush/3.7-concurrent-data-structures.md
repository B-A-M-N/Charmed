The `csync` package (`internal/csync`) provides generic, thread-safe data structures used throughout the codebase wherever shared state must be accessed from multiple goroutines. This package provides primitives that wrap standard Go maps and slices with appropriate locking mechanisms, lazy-loading capabilities, and support for modern Go features like iterators and JSON marshaling.

`Value[T]`
----------

`Value` is a generic thread-safe wrapper for any value type using a `sync.RWMutex` [internal/csync/value.go11-14](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/value.go#L11-L14)
 It is designed for simple scalars or structs.

**Restrictions:** To prevent shallow-copy issues and race conditions on nested data, `NewValue` will **panic** if the provided type is a pointer, slice, or map [internal/csync/value.go19-30](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/value.go#L19-L30)
 Users are directed to use `Slice[T]` or `Map[K, V]` instead.

| Method | Lock acquired | Behavior |
| --- | --- | --- |
| `Get()` | Read lock | Returns the current value [internal/csync/value.go33-37](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/value.go#L33-L37) |
| `Set(t)` | Write lock | Updates the inner value [internal/csync/value.go40-44](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/value.go#L40-L44) |

`VersionedMap[K, V]`
--------------------

`VersionedMap` wraps `*Map[K, V]` and adds a monotonically incrementing version counter using `atomic.Uint64` [internal/csync/versionedmap.go16-19](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/versionedmap.go#L16-L19)

Every mutation (`Set` or `Del`) increments the version [internal/csync/versionedmap.go27-36](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/versionedmap.go#L27-L36)
 This allows consumers to track if the map has changed without performing a full comparison or deep copy.

Lazy Loading Strategies
-----------------------

### `NewLazyMap`

The `NewLazyMap` function locks the map's mutex **before** spawning the goroutine [internal/csync/maps.go34-39](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L34-L39)
 The goroutine performs the load and then unlocks the mutex. This ensures that any subsequent calls to `Get` or `Set` will block until the loading is complete.

### `LazySlice[T]`

`LazySlice` uses a `sync.WaitGroup` to coordinate loading [internal/csync/slices.go9-12](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/slices.go#L9-L12)
 The `NewLazySlice` constructor starts the load goroutine immediately [internal/csync/slices.go16-22](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/slices.go#L16-L22)
 The only way to access the data is via `Seq()`, which calls `s.wg.Wait()` before providing the iterator [internal/csync/slices.go25-34](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/slices.go#L25-L34)

Iterator Interfaces (Seq, Seq2)
-------------------------------

All structures implement the Go 1.23 `iter` patterns.

*   **Snapshot Semantics:** `Map` and `Slice` iterators take a snapshot (via `Copy()`) before yielding [internal/csync/maps.go108](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L108-L108)
     [internal/csync/slices.go104](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/slices.go#L104-L104)
     This prevents deadlocks if the loop body attempts to modify the same structure.
*   **Early Exit:** Iterators respect the `yield` return value, allowing `break` statements in loops to stop iteration [internal/csync/maps.go111-113](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L111-L113)
     [internal/csync/slices.go29-31](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/slices.go#L29-L31)
    

| Method | Type | Yields |

| `Map.Seq()` | `iter.Seq[V]` | Values only [internal/csync/maps.go119-127](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L119-L127) |
| `Map.Seq2()` | `iter.Seq2[K, V]` | Key-Value pairs [internal/csync/maps.go107-116](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L107-L116) |

JSON Marshaling
---------------

`Map[K, V]` provides built-in support for `encoding/json`.

*   **UnmarshalJSON:** Acquires a write lock and replaces the existing map with the unmarshaled data [internal/csync/maps.go143-148](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L143-L148)
    
*   **MarshalJSON:** Acquires a read lock and marshals the inner map [internal/csync/maps.go151-155](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L151-L155)
    
*   **JSONSchemaAlias:** Used for automated schema generation to represent the `Map` as a standard JSON object [internal/csync/maps.go137-140](https://github.com/charmbracelet/crush/blob/1ceeaa93/internal/csync/maps.go#L137-L140)

--------------
