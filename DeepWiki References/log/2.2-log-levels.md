Log Levels
==========

Relevant source files

*   [level.go](https://github.com/charmbracelet/log/blob/cf6e8671/level.go)
    
*   [level\_test.go](https://github.com/charmbracelet/log/blob/cf6e8671/level_test.go)
    

This page documents the log levels supported by the charmbracelet/log library, explaining how they function within the logging system and how to use them effectively. For information about configuring loggers, see [Configuration Options](https://deepwiki.com/charmbracelet/log/6-configuration-options)
.

Overview of Log Levels
----------------------

Log levels provide a way to categorize log messages by their importance and severity. They allow you to filter which messages are displayed based on a minimum threshold, helping to control the verbosity of your application's logging output.

Sources: [level.go10-26](https://github.com/charmbracelet/log/blob/cf6e8671/level.go#L10-L26)

Available Log Levels
--------------------

The charmbracelet/log library defines five standard log levels as constants:

| Level | Numeric Value | Purpose |
| --- | --- | --- |
| `DebugLevel` | \-4 | Detailed information for debugging purposes |
| `InfoLevel` | 0   | General operational information |
| `WarnLevel` | 4   | Warning conditions that might need attention |
| `ErrorLevel` | 8   | Error conditions that don't stop the application |
| `FatalLevel` | 12  | Critical errors that terminate the application |

Additionally, there's an internal `noLevel` constant (with a value of `math.MaxInt`) used with the `log.Print` family of functions.

Sources: [level.go13-26](https://github.com/charmbracelet/log/blob/cf6e8671/level.go#L13-L26)

Log Level Filtering
-------------------

When a log level is set on a logger, only messages at that level or higher are emitted. Messages with lower levels are silently discarded.

For example, if a logger's level is set to `WarnLevel`, then:

*   Debug and Info messages will be discarded
*   Warn, Error, and Fatal messages will be processed and output

By default, a new logger uses `InfoLevel` if not explicitly configured otherwise.

Sources: [level\_test.go10-13](https://github.com/charmbracelet/log/blob/cf6e8671/level_test.go#L10-L13)

Log Level String Representation
-------------------------------

Each log level has a string representation, which is useful for configuration and display purposes.

The `String()` method on the `Level` type returns the lowercase string representation of each level.

Sources: [level.go28-44](https://github.com/charmbracelet/log/blob/cf6e8671/level.go#L28-L44)

Parsing Log Levels from Strings
-------------------------------

The library provides a `ParseLevel` function to convert string representations back to `Level` values. This is case-insensitive, so "INFO", "info", and "Info" all parse to `InfoLevel`.

Sources: [level.go49-65](https://github.com/charmbracelet/log/blob/cf6e8671/level.go#L49-L65)
 [level\_test.go15-73](https://github.com/charmbracelet/log/blob/cf6e8671/level_test.go#L15-L73)

Internal Implementation
-----------------------

The `Level` type in charmbracelet/log is implemented as an integer, with larger values representing more severe log levels. This allows for simple comparison operations when filtering messages.

In the internal implementation, log level filtering is performed through a simple integer comparison: if the message level is greater than or equal to the logger's configured level, the message is processed.

Sources: [level.go10-11](https://github.com/charmbracelet/log/blob/cf6e8671/level.go#L10-L11)

Default Behavior
----------------

A few important default behaviors to note:

1.  The zero value of `Level` is `InfoLevel` (0), making it the default level when no level is specified.
2.  When parsing an invalid level string, `ParseLevel` returns an error and the `InfoLevel` value.
3.  The `noLevel` constant is used internally for logging methods that don't specify a level, like `log.Print()`.
