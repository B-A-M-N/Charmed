Logger Structure
================

Relevant source files

*   [logger.go](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go)
    

This document details the core `Logger` type in the charmbracelet/log library, its fields, and primary methods. The `Logger` is the central component that handles all logging operations, including message formatting, level filtering, and output writing. For information about log levels and filtering, see [Log Levels](https://deepwiki.com/charmbracelet/log/2.2-log-levels)
. For the global logger, see [Global Logger](https://deepwiki.com/charmbracelet/log/2.3-global-logger)
.

Logger Definition
-----------------

The `Logger` structure is defined in `logger.go` and provides a comprehensive logging implementation with support for different output formats, styling, and contextual logging.

Sources: [logger.go24-48](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L24-L48)

Core Fields
-----------

The `Logger` structure contains several important fields that determine its behavior:

| Field | Type | Purpose |
| --- | --- | --- |
| `w` | `io.Writer` | Destination for log output (default: stderr) |
| `b` | `bytes.Buffer` | Buffer used to construct log messages |
| `mu` | `*sync.RWMutex` | Mutex for thread-safe operations |
| `level` | `int64` | Current log level (stored as atomic value) |
| `prefix` | `string` | Prefix added to log messages |
| `formatter` | `Formatter` | Format type for log output (Text, JSON, Logfmt) |
| `reportCaller` | `bool` | Whether to include caller information |
| `reportTimestamp` | `bool` | Whether to include timestamps |
| `fields` | `[]interface{}` | Contextual key-value pairs for this logger |
| `styles` | `*Styles` | Styling configuration for text output |

Sources: [logger.go24-48](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L24-L48)

Configuration Methods
---------------------

The `Logger` provides several methods to configure its behavior:

| Method | Purpose |
| --- | --- |
| `SetLevel(level Level)` | Set minimum log level |
| `SetOutput(w io.Writer)` | Set output destination |
| `SetFormatter(f Formatter)` | Set output format |
| `SetPrefix(prefix string)` | Set message prefix |
| `SetReportCaller(report bool)` | Enable/disable caller reporting |
| `SetReportTimestamp(report bool)` | Enable/disable timestamp reporting |
| `SetTimeFormat(format string)` | Set timestamp format |
| `SetTimeFunction(f TimeFunction)` | Set function to generate timestamps |
| `SetCallerFormatter(f CallerFormatter)` | Set function to format caller info |
| `SetCallerOffset(offset int)` | Set offset for caller frame calculation |
| `SetStyles(s *Styles)` | Set text output styles |

Implementation Details
----------------------

### Thread Safety

The `Logger` is designed to be thread-safe using a read-write mutex for most operations and atomic operations for level checking:

*   Level checks use atomic loads to avoid locking
*   The `isDiscard` flag uses atomic operations for fast rejection
*   All configuration changes use mutex locking
*   Log writing operations use mutex locking

This allows the logger to be safely used from multiple goroutines simultaneously.

Sources: [logger.go56-82](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L56-L82)
 [logger.go214-290](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L214-L290)

### Custom Formatting

The log message formatting process involves several steps:

1.  Collection of log metadata (timestamp, level, caller information)
2.  Addition of logger prefix if set
3.  Addition of the log message
4.  Addition of logger context fields
5.  Addition of call-specific key-value pairs
6.  Selection of formatter (Text, JSON, Logfmt)
7.  Formatting and writing to output

For more details about the formatters, see [Formatters](https://deepwiki.com/charmbracelet/log/4-formatters)
.

Sources: [logger.go84-140](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L84-L140)

Logger Creation
---------------

The `Logger` structure is typically not created directly. Instead, use the `New` function with options to create a properly initialized logger. For information about configuration options, see [Configuration Options](https://deepwiki.com/charmbracelet/log/6-configuration-options)
