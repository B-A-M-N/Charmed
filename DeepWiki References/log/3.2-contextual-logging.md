Contextual Logging
==================

*   [context.go](https://github.com/charmbracelet/log/blob/cf6e8671/context.go)

Contextual logging allows you to enhance log messages with additional structured information that persists across multiple log calls. The `charmbracelet/log` package provides powerful capabilities for adding context to your logs, either through key-value pairs or prefixes, and for associating loggers with Go's standard `context.Context`.

Key Features
------------

*   Add persistent key-value pairs to loggers with the `With` method
*   Add prefixes to log messages with `WithPrefix`
*   Store and retrieve loggers from Go's `context.Context`
*   Built-in parent-child logger hierarchy with context inheritance

Sources: [logger.go328-350](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L328-L350)
 [context.go5-18](https://github.com/charmbracelet/log/blob/cf6e8671/context.go#L5-L18)

Logger Context Hierarchy
------------------------

Loggers in `charmbracelet/log` form a hierarchy, where child loggers inherit all settings and contextual fields from their parents.

When a log message is generated from a child logger, all fields from the parent are automatically included.

Sources: [logger.go329-343](https://github.com/charmbracelet/log/blob/cf6e8671/logger.go#L329-L343)

Internal Implementation Details
-------------------------------

When you create a new logger with `With` or `WithPrefix`, the package:

1.  Creates a shallow copy of the parent logger
2.  Copies the style settings
3.  Creates new buffer, mutex, and helpers map
4.  For `With`: appends the new key-value pairs to the fields slice
5.  For `WithPrefix`: sets the prefix value

This ensures that the new logger has its own state but inherits all the appropriate settings.

Log Message Generation with Context
-----------------------------------

When a log message is generated, the contextual fields are included in this order:

1.  Standard fields (timestamp, level, caller if enabled)
2.  Prefix (if set)
3.  Message
4.  Logger's persistent fields (from `With`)
5.  Message-specific key-value pairs

Context Integration
-------------------

The package provides integration with Go's `context.Context` through the `WithContext` and `FromContext` functions.

This allows you to:

1.  Store a logger in a context
2.  Pass the context through your application
3.  Retrieve the logger wherever you have access to the context

Sources: [context.go5-23](https://github.com/charmbracelet/log/blob/cf6e8671/context.go#L5-L23)

Best Practices
--------------

1.  Create a base logger with application-wide context (app name, version, environment)
2.  Use `With` to add component-specific context (service name, request ID)
3.  Use `WithContext` to pass loggers through your application via context
4.  Use message-specific key-value pairs for information specific to a single log line
5.  Keep key names consistent throughout your application
