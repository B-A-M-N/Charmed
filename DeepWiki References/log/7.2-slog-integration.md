Slog Integration
================

Relevant source files

*   [logger\_121.go](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121.go)
    
*   [logger\_121\_test.go](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121_test.go)
    
*   [logger\_no121.go](https://github.com/charmbracelet/log/blob/cf6e8671/logger_no121.go)
    

This page documents how the `charmbracelet/log` package integrates with Go's structured logging package `slog`. The logger in this package implements the `slog.Handler` interface, allowing it to be used as a drop-in replacement for the default `slog` handlers while maintaining Charm's styling and formatting capabilities.

Overview
--------

Go 1.21 introduced the standard library `log/slog` package for structured logging. For compatibility with both Go 1.21+ and earlier versions, `charmbracelet/log` provides integration with:

*   The standard library `log/slog` package (Go 1.21+)
*   The experimental `golang.org/x/exp/slog` package (Go versions before 1.21)

This integration allows developers to use the structured logging capabilities of `slog` with the visual appeal and customization options of `charmbracelet/log`.

Level Mapping
-------------

The integration maps `slog.Level` values to `log.Level` values automatically:

| slog.Level | log.Level |
| --- | --- |
| slog.LevelDebug | DebugLevel |
| slog.LevelInfo | InfoLevel |
| slog.LevelWarn | WarnLevel |
| slog.LevelError | ErrorLevel |

Custom levels are also supported.

Sources: [logger\_121\_test.go154-184](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121_test.go#L154-L184)

Attribute Support
-----------------

The integration supports all standard `slog` attribute types:

Sources: [logger\_121\_test.go194-276](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121_test.go#L194-L276)

The following attribute types are supported:

*   Any values
*   Booleans
*   Durations
*   Float64
*   Strings
*   Time values
*   Uint64
*   Groups (nested attributes)
*   Custom `LogValuer` implementations

Technical Implementation
------------------------

The integration is implemented through build tags to handle different Go versions:

*   `go1.21`: Uses the standard library `log/slog` package
*   `!go1.21`: Uses the experimental `golang.org/x/exp/slog` package

To maintain compatibility, the code uses type aliases and constants to abstract away the differences between the two versions:

Sources: [logger\_121.go13-20](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121.go#L13-L20)
 [logger\_no121.go14-21](https://github.com/charmbracelet/log/blob/cf6e8671/logger_no121.go#L14-L21)

Cross-Formatter Compatibility
-----------------------------

The `slog` integration works with all formatters provided by `charmbracelet/log`:

*   Text formatter (default)
*   JSON formatter
*   Logfmt formatter

This allows using `slog` with the formatting style that best suits your needs.

Sources: [logger\_121\_test.go258-274](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121_test.go#L258-L274)

Performance Considerations
--------------------------

The integration is designed to be efficient, with minimal overhead compared to using `slog` directly:

*   Log records are only processed if the level is enabled
*   Attributes are efficiently converted to key-value pairs
*   Caller information is extracted from the record's PC

Sources: [logger\_121.go25-46](https://github.com/charmbracelet/log/blob/cf6e8671/logger_121.go#L25-L46)
