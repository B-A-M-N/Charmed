Logfmt Formatter
================

Purpose and Scope
-----------------

The Logfmt Formatter is one of three output formatters provided by the charmbracelet/log library, specifically designed to format log messages in the logfmt format. Logfmt is a widely-used key-value pair format that represents data as `key=value` pairs, similar to HTTP query parameters. It's particularly useful for machine parsing while remaining human-readable.

This document covers the implementation, usage, and behavior of the Logfmt Formatter. For information about other formatters, see [Text Formatter](https://deepwiki.com/charmbracelet/log/4.1-text-formatter)
 and [JSON Formatter](https://deepwiki.com/charmbracelet/log/4.2-json-formatter)
.

Sources: [formatter.go12-13](https://github.com/charmbracelet/log/blob/cf6e8671/formatter.go#L12-L13)
 [logfmt.go1-33](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L1-L33)

Overview
--------

The Logfmt Formatter produces log output in the format:

    level=info msg="log message" key1=value1 key2=value2
    

This format is designed to be:

*   Easy to parse by log processing tools
*   Relatively human-readable
*   Compact without unnecessary formatting

Implementation
--------------

The Logfmt Formatter is implemented as a `Formatter` type constant and a logger method that handles the actual formatting.

Sources: [formatter.go4-14](https://github.com/charmbracelet/log/blob/cf6e8671/formatter.go#L4-L14)
 [logfmt.go11-32](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L11-L32)

### Core Components

The implementation consists of:

1.  **Formatter Type**: Defined as a constant in the formatter.go file
2.  **Logger Method**: The `logfmtFormatter` method that processes and encodes key-value pairs
3.  **External Dependency**: Uses the `github.com/go-logfmt/logfmt` package for encoding

Sources: [formatter.go4-14](https://github.com/charmbracelet/log/blob/cf6e8671/formatter.go#L4-L14)
 [logfmt.go3-9](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L3-L9)

### Special Handling

The Logfmt Formatter includes special handling for:

1.  **Timestamps**: Formatted according to the logger's time format
2.  **Unsupported Types**: Automatically converted to strings using `fmt.Sprintf("%+v", value)`
3.  **Multiline Strings**: Escaped with `\n` to maintain the key-value format
4.  **Complex Data Structures**: Structs, slices, maps, etc. are formatted to maintain readability

Sources: [logfmt.go14-31](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L14-L31)

Data Type Formatting
--------------------

The table below shows how different data types are formatted:

| Data Type | Example Input | Formatted Output |
| --- | --- | --- |
| Simple string | `"foo", "bar"` | `foo=bar` |
| Multiline string | `"foo", "bar\nbaz"` | `foo="bar\nbaz"` |
| Error | `"err", errors.New("foo: bar")` | `err="foo: bar"` |
| Struct | `"foo", struct{bar string}{"baz"}` | `foo={bar:baz}` |
| String Slice | `"foo", []string{"bar", "baz"}` | `foo="[bar baz]"` |
| Map | `"foo", map[string]string{"bar": "baz"}` | `foo=map[bar:baz]` |

Sources: [logfmt\_test.go35-119](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt_test.go#L35-L119)

Comparison with Other Formatters
--------------------------------

The Logfmt Formatter offers a middle ground between:

*   The visually appealing but less machine-parsable Text Formatter
*   The fully structured but verbose JSON Formatter

Sources: [formatter.go4-14](https://github.com/charmbracelet/log/blob/cf6e8671/formatter.go#L4-L14)

Implementation Details
----------------------

### Key Processing Logic

The formatter implements key processing logic to handle edge cases:

1.  **Empty Keys**: Skipped to maintain valid logfmt output
2.  **Odd Number of Arguments**: Handled by adding a "missing value" placeholder
3.  **Non-String Keys**: Converted to strings using `fmt.Sprint()`

Sources: [logfmt.go20-24](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L20-L24)
 [logfmt\_test.go65-70](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt_test.go#L65-L70)

### Error Handling

When the logfmt encoder encounters unsupported value types, the formatter:

1.  Catches the `logfmt.ErrUnsupportedValueType` error
2.  Converts the value to a string using `fmt.Sprintf("%+v", value)`
3.  Re-encodes the key with the string representation of the value

This ensures that all values can be represented in the logfmt format, even complex ones.

Sources: [logfmt.go25-30](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L25-L30)

Integration with Logger
-----------------------

The Logfmt Formatter integrates with standard logger fields and context:

When a log method is called:

1.  The logger checks if the formatter is set to `LogfmtFormatter`
2.  If so, it calls the `logfmtFormatter` method with appropriate key-value pairs
3.  Standard fields (level, timestamp, caller, prefix) are automatically included
4.  User-provided fields are appended to the output

Sources: [formatter.go4-14](https://github.com/charmbracelet/log/blob/cf6e8671/formatter.go#L4-L14)
 [logfmt.go11-32](https://github.com/charmbracelet/log/blob/cf6e8671/logfmt.go#L11-L32)

Conclusion
----------

The Logfmt Formatter provides a balanced approach to logging, offering structured output that is both machine-parsable and reasonably human-readable. It's particularly well-suited for:

*   Production environments where logs need to be processed by automated tooling
*   Systems where log storage efficiency is important
*   Applications where complex data structures need to be represented in a flattened format

For more styling options and human-focused output, consider the [Text Formatter](https://deepwiki.com/charmbracelet/log/4.1-text-formatter)
. For fully structured JSON output, refer to the [JSON Formatter](https://deepwiki.com/charmbracelet/log/4.2-json-formatter)
.
