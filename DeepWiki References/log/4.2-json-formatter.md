JSON Formatter
==============

Relevant source files

*   [json.go](https://github.com/charmbracelet/log/blob/cf6e8671/json.go)
    
*   [json\_test.go](https://github.com/charmbracelet/log/blob/cf6e8671/json_test.go)
    

The JSON Formatter in the charmbracelet/log library provides structured, machine-readable log output in JSON format. This formatter is ideal for environments where logs are consumed by automated systems, log aggregation tools, or any scenario requiring programmatic log analysis. While the [Text Formatter](https://deepwiki.com/charmbracelet/log/4.1-text-formatter)
 offers human-friendly output with styling, the JSON Formatter focuses on machine-readability and consistent structured data.

Overview and Usage
------------------

The JSON Formatter converts log messages and their associated metadata into valid JSON objects. Each log entry becomes a single-line JSON object with standardized fields.

### Standard Fields

The JSON Formatter includes several standard fields in the output:

| Field | Key | Description | Inclusion Condition |
| --- | --- | --- | --- |
| Level | `level` | Log level (debug, info, warn, error, fatal) | Always included |
| Message | `msg` | The log message | Included when a message is provided |
| Timestamp | `time` | Timestamp formatted according to logger's time format | When `SetReportTimestamp(true)` |
| Caller | `caller` | File and line number where the log call was made | When `SetReportCaller(true)` |
| Prefix | `prefix` | Logger prefix | When prefix is set via `SetPrefix` |

### The jsonWriter

The `jsonWriter` is a utility struct that helps build JSON objects. It:

1.  Tracks whether a comma is needed before the next key-value pair
2.  Handles JSON encoding of arbitrary Go values
3.  Manages the structure of the JSON object being built

Key aspects of the implementation:

*   Uses the `encoding/json` package to ensure proper JSON encoding
*   Handles invalid JSON gracefully by replacing problematic values with fallback text
*   Removes trailing newlines added by the JSON encoder

Special Value Handling
----------------------

The JSON formatter provides special handling for different value types:

1.  **Time values**: Formatted according to the logger's configured time format
2.  **Error values**: Serialized using their error message
3.  **fmt.Stringer values**: Serialized using their String() method
4.  **Nested structures (maps, slices, structs)**: Properly serialized as nested JSON objects/arrays
5.  **slog values**: Support for Go's structured logging values (slog)
6.  **Invalid JSON values**: Replaced with "invalid value" if they can't be marshaled to JSON

Integration with Log Analysis Tools
-----------------------------------

The JSON formatter is particularly useful when integrating with log analysis platforms that expect structured data. Common examples include:

*   Elasticsearch/Logstash/Kibana (ELK Stack)
*   Grafana Loki
*   Datadog
*   New Relic

These tools can automatically parse JSON logs and make fields searchable and filterable.
