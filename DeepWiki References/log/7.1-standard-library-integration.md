Standard Library Integration
============================

Relevant source files

*   [stdlog.go](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go)
    
*   [stdlog\_test.go](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog_test.go)
    

This page documents how to integrate charmbracelet/log with Go's standard library logging package. For integration with Go's structured logging package (slog), see [Slog Integration](https://deepwiki.com/charmbracelet/log/7.2-slog-integration)
.

Overview
--------

The charmbracelet/log library provides seamless integration with Go's standard `log` package, allowing you to:

1.  Redirect standard library log output through charmbracelet/log
2.  Apply charmbracelet/log's formatting, styling, and levels to standard library logs
3.  Automatically infer log levels from message prefixes or force specific levels

Sources: [stdlog.go8-45](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go#L8-L45)

Core Components
---------------

### stdLogWriter

The `stdLogWriter` struct is an adapter that implements the `io.Writer` interface to redirect standard library log messages to charmbracelet/log:

When a standard library logger writes to this adapter, the `Write` method:

1.  Trims trailing newlines
2.  Determines the appropriate log level
3.  Forwards the message to the corresponding log method

Sources: [stdlog.go8-45](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go#L8-L45)

### StandardLogOptions

The `StandardLogOptions` struct allows you to configure how standard library logs are processed:

Currently, it supports:

*   `ForceLevel`: Forces all standard library logs to use a specific level, ignoring prefix inference

Sources: [stdlog.go47-50](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go#L47-L50)

Level Inference
---------------

When using the standard library integration without forcing a level, the adapter automatically infers log levels from message prefixes:

| Message Prefix | charmbracelet/log Level |
| --- | --- |
| `DEBUG` | Debug |
| `INFO` | Info |
| `WARN` | Warn |
| `ERROR` or `ERR` | Error |
| _(no prefix)_ | Info (default) |

Sources: [stdlog.go27-41](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go#L27-L41)

Implementation Details
----------------------

### Caller Reporting

When using standard library integration with caller reporting enabled, the adapter adjusts the caller offset to ensure that the correct file and line number are reported:

The caller stack is adjusted by adding 3 to the caller offset to account for:

1.  `log.Printf()` ->
2.  `l.Output()` ->
3.  `l.out.Write(stdLogger.Write)`

Sources: [stdlog.go58-59](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go#L58-L59)

### Message Processing

When processing messages, the adapter:

1.  Removes trailing newlines
2.  Extracts and removes level prefixes (if present)
3.  Trims leading/trailing whitespace
4.  Forwards the processed message to the appropriate logger method

Sources: [stdlog.go13-42](https://github.com/charmbracelet/log/blob/cf6e8671/stdlog.go#L13-L42)

Testing the Integration
-----------------------

To verify the standard library integration is working correctly, you can:

1.  Check if level prefixes are correctly identified
2.  Confirm that forced levels override prefix inference
3.  Verify that caller information is reported correctly when enabled

The test suite provides examples of these verification steps.
