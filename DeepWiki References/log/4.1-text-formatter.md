Text Formatter
==============

Relevant source files

*   [text.go](https://github.com/charmbracelet/log/blob/cf6e8671/text.go)
    
*   [text\_test.go](https://github.com/charmbracelet/log/blob/cf6e8671/text_test.go)
    

The Text Formatter is the default formatter in the `charmbracelet/log` library, providing human-readable, colorful output for log messages. It renders log entries as styled text with support for multiline values, special formatting for common fields, and customizable styling.

For information on other formatters available in the library, see [Formatters](https://deepwiki.com/charmbracelet/log/4-formatters)
 and its related pages for [JSON Formatter](https://deepwiki.com/charmbracelet/log/4.2-json-formatter)
 and [Logfmt Formatter](https://deepwiki.com/charmbracelet/log/4.3-logfmt-formatter)
.

Architecture
------------

The Text Formatter is a core component of the formatter system in the charmbracelet/log library, responsible for converting structured log data into human-readable text output.

Sources: [text.go167-272](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L167-L272)

Implementation Overview
-----------------------

The Text Formatter is implemented in the `textFormatter` method of the `Logger` struct. When a log message is processed, the logger converts the message and any additional key-value pairs into a formatted string.

Sources: [text.go167-272](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L167-L272)

Special Key Handling
--------------------

The Text Formatter processes several special keys differently from regular key-value pairs:

| Special Key | Description | Styling |
| --- | --- | --- |
| `TimestampKey` | Formats the timestamp according to the configured time format | Applied using `styles.Timestamp` |
| `LevelKey` | Renders the log level (INFO, ERROR, etc.) | Applied using `styles.Levels[level]` |
| `CallerKey` | Shows where the log was called from (file and line) | Applied using `styles.Caller` |
| `PrefixKey` | Adds a prefix to the log message | Applied using `styles.Prefix` |
| `MessageKey` | The main log message | Applied using `styles.Message` |

All other keys are treated as regular key-value pairs and formatted as `key=value`.

Sources: [text.go175-216](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L175-L216)

Regular Key-Value Formatting
----------------------------

For non-special keys, the Text Formatter handles three main cases:

1.  **Simple values**: Formatted as `key=value`
2.  **Multiline values**: Formatted with special indentation
3.  **Values requiring quoting**: Formatted as `key="escaped value"`

Multiline Value Handling
------------------------

The Text Formatter preserves multiline values by indenting each line after the first with a special separator. This makes it easy to identify which lines belong to which key.

Example output for a multiline value:

    ERROR info
      key1=
      │ val1
      │ val2
    

This is handled by the `writeIndent` method, which processes multiline strings and adds the correct indentation to each line.

Sources: [text.go18-48](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L18-L48)
 [text.go249-254](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L249-L254)

String Escaping and Quoting
---------------------------

The Text Formatter includes logic to determine when values need escaping or quoting:

1.  `needsEscaping`: Checks if a string contains non-printable characters or quotes
2.  `escapeStringForOutput`: Handles the actual escaping of special characters
3.  `needsQuoting`: Determines if a value should be quoted (contains spaces, special characters, etc.)

This ensures that log output remains valid and readable even with complex values.

Sources: [text.go50-145](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L50-L145)

Styling System
--------------

The Text Formatter uses the lipgloss library to apply styles to different components of the log message. Styles are defined in the `Styles` struct and can be customized:

    styles.Timestamp  - Style for timestamps
    styles.Levels     - Map of styles for each log level
    styles.Caller     - Style for caller information
    styles.Prefix     - Style for the log prefix
    styles.Message    - Style for the main message
    styles.Key        - Default style for keys
    styles.Value      - Default style for values
    styles.Keys       - Map of styles for specific keys
    styles.Values     - Map of styles for specific values
    styles.Separator  - Style for the separator between keys and values
    

Each style can customize attributes like color, boldness, italics, etc.

Sources: [text.go169-265](https://github.com/charmbracelet/log/blob/cf6e8671/text.go#L169-L265)

Output Example
--------------

A fully formatted log message might look like this:

    2023-04-01 14:30:45 INFO <main.go:42> Server starting user=admin port=8080
    

Where:

*   `2023-04-01 14:30:45` is the timestamp (styled with `styles.Timestamp`)
*   `INFO` is the level (styled with `styles.Levels[InfoLevel]`)
*   `<main.go:42>` is the caller (styled with `styles.Caller`)
*   `Server starting` is the message (styled with `styles.Message`)
*   `user=admin port=8080` are key-value pairs (keys styled with `styles.Key`, values with `styles.Value`)

Sources: [text\_test.go101-210](https://github.com/charmbracelet/log/blob/cf6e8671/text_test.go#L101-L210)
