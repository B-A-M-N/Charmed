Global Logger
=============

Relevant source files

*   [pkg.go](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go)
    

The Global Logger component provides a convenient, package-level API for logging in Go applications without requiring developers to instantiate or manage logger instances explicitly. It acts as a singleton that serves as the default entry point to the logging functionality provided by the Charmbracelet log package.

For details about creating custom logger instances, see [Logger Structure](https://deepwiki.com/charmbracelet/log/2.1-logger-structure)
.

Overview
--------

The Global Logger is implemented as a package-level variable with functions that delegate to an underlying `Logger` instance. This pattern allows for simple import-and-use operation while still providing access to the full feature set of the logging system.

_Diagram: Global Logger Architecture_

Sources: [pkg.go16-23](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L16-L23)
 [pkg.go25-37](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L25-L37)

Implementation Details
----------------------

The Global Logger is implemented using Go's atomic package to ensure thread safety, with lazy initialization to create the default logger only when needed.

_Diagram: Global Logger Initialization Flow_

Sources: [pkg.go25-37](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L25-L37)
 [pkg.go185-187](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L185-L187)

### Key Components

*   **`defaultLogger`**: An atomic pointer to the singleton Logger instance
*   **`Default()`**: Returns the default logger, creating it if needed
*   **`SetDefault()`**: Allows replacing the default logger with a custom one

Sources: [pkg.go16-42](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L16-L42)

Package-Level Logging API
-------------------------

The Global Logger exposes a comprehensive set of functions that mirror the methods available on the Logger struct. These functions provide a simple, statically-importable API for logging at various levels.

### Core Logging Functions

| Function | Description | Level |
| --- | --- | --- |
| `Debug()` | Logs a debug message | DebugLevel |
| `Info()` | Logs an informational message | InfoLevel |
| `Warn()` | Logs a warning message | WarnLevel |
| `Error()` | Logs an error message | ErrorLevel |
| `Fatal()` | Logs a fatal message and exits the program | FatalLevel |
| `Print()` | Logs a message with no level | noLevel |

Each of these functions has a corresponding formatted version (e.g., `Debugf()`, `Infof()`) that accepts a format string and arguments.

Sources: [pkg.go175-244](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L175-L244)

### Configuration Functions

The Global Logger provides package-level functions to configure the default logger:

| Function | Purpose |
| --- | --- |
| `SetLevel()` | Sets the minimum log level |
| `SetOutput()` | Sets the output destination |
| `SetFormatter()` | Sets the log format (Text, JSON, Logfmt) |
| `SetReportTimestamp()` | Toggles timestamp reporting |
| `SetReportCaller()` | Toggles caller location reporting |
| `SetPrefix()` | Sets a prefix for log messages |
| `SetStyles()` | Sets styles for the text formatter |
| `SetTimeFormat()` | Sets timestamp format |
| `SetTimeFunction()` | Sets a function to transform timestamps |
| `SetCallerFormatter()` | Sets a function to format caller information |

Sources: [pkg.go86-150](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L86-L150)

Thread Safety
-------------

The Global Logger is designed to be thread-safe. Its implementation uses atomic operations for safely initializing and accessing the default logger instance, allowing it to be used concurrently from multiple goroutines without additional synchronization.

Sources: [pkg.go16-23](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L16-L23)
 [pkg.go25-37](https://github.com/charmbracelet/log/blob/cf6e8671/pkg.go#L25-L37)

Initialization Behavior
-----------------------

By default, the Global Logger is configured to:

*   Write to `os.Stderr`
*   Include timestamps
*   Use the text formatter
*   Apply default styles

These settings can be modified at any time through the configuration functions.
