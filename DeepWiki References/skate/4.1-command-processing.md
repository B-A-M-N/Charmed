Purpose and Scope
-----------------

This document details how Skate processes commands from initial CLI input through execution and output. It covers the command structure, parsing mechanisms, database operations, and terminal output formatting that enable Skate's key-value operations.

For information about how data is physically stored, see [Data Storage](https://deepwiki.com/charmbracelet/skate/4.2-data-storage)
. For details on the terminal UI implementation, see [Terminal Interface](https://deepwiki.com/charmbracelet/skate/4.3-terminal-interface)
.

Command Structure Overview
--------------------------

Skate uses the Cobra library (`github.com/spf13/cobra`) to define and process commands. The command structure forms a tree with a root command and several subcommands for key-value operations.

Sources: [main.go39-109](https://github.com/charmbracelet/skate/blob/09860417/main.go#L39-L109)
 [main.go431-446](https://github.com/charmbracelet/skate/blob/09860417/main.go#L431-L446)

Command Registration
--------------------

Commands are registered with the root command during initialization. Each command is defined with its usage, description, and a function to execute when invoked.

Sources: [main.go420-447](https://github.com/charmbracelet/skate/blob/09860417/main.go#L420-L447)

Command Input Processing
------------------------

### Key Parsing

Skate uses a key parser to extract the key and optional database name from input strings. The format is `KEY@DB` where `DB` is optional.

Sources: [main.go394-407](https://github.com/charmbracelet/skate/blob/09860417/main.go#L394-L407)

### Database Selection

If no database is specified, Skate uses the "default" database. The system follows this logic:

1.  Parse key and optional DB name from command arguments
2.  If DB name is empty, use "default"
3.  Construct the database file path
4.  Open the BadgerDB instance

Sources: [main.go409-418](https://github.com/charmbracelet/skate/blob/09860417/main.go#L409-L418)
 [main.go366-375](https://github.com/charmbracelet/skate/blob/09860417/main.go#L366-L375)

Database Operations
-------------------

### Database Access

All database operations are wrapped in transactions using the `wrap` function, which handles transaction creation, execution, and commitment or discard on error.

Sources: [main.go456-463](https://github.com/charmbracelet/skate/blob/09860417/main.go#L456-L463)

### Command-Specific Operations

Each command implements its own operation pattern:

| Command | Function | Operation Pattern |
| --- | --- | --- |
| set | set() | Parse key → Open DB → Create transaction → Set value → Commit |
| get | get() | Parse key → Open DB → Read-only transaction → Get value → Format output |
| delete | del() | Parse key → Open DB → Create transaction → Delete key → Commit |
| list | list() | Parse DB name → Open DB → Read-only transaction → Iterate keys/values → Format output |

Sources: [main.go123-146](https://github.com/charmbracelet/skate/blob/09860417/main.go#L123-L146)

Output Formatting
-----------------

Skate formats terminal output using the `printFromKV` function, which handles different output formats:

1.  Binary data detection and handling
2.  Formatting with specified delimiters
3.  Keys-only or values-only output modes

The function checks if the output is a terminal and formats accordingly, handling non-UTF8 content appropriately.

Sources: [main.go377-392](https://github.com/charmbracelet/skate/blob/09860417/main.go#L377-L392)
