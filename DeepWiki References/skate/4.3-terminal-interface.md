Terminal Interface
==================

Relevant source files

*   [go.mod](https://github.com/charmbracelet/skate/blob/09860417/go.mod)
    
*   [main.go](https://github.com/charmbracelet/skate/blob/09860417/main.go)
    

This document describes the terminal interface implementation in Skate, focusing on how output is styled, formatted, and presented to users. The terminal interface is responsible for creating a consistent, readable user experience when interacting with the key-value store through the command line.

For information about the command processing flow, see [Command Processing](https://deepwiki.com/charmbracelet/skate/4.1-command-processing)
. For details about the CLI commands and options, see [Command Line Interface](https://deepwiki.com/charmbracelet/skate/6.1-command-line-interface)
.

Overview
--------

Skate uses terminal styling and formatting to ensure that output is presented in a clear, user-friendly manner. The primary components of the terminal interface include:

*   Text styling for warnings and errors
*   Output formatting for key-value pairs
*   Binary data detection and handling
*   Terminal interaction for confirmations

Text Styling with Lipgloss
--------------------------

Skate uses the Charmbracelet Lipgloss library for styling terminal output. Lipgloss provides capabilities for colored text, bold formatting, and layout control.

### Defined Styles

Currently, Skate implements the following style:

*   `warningStyle`: Used for displaying important information in confirmation dialogues, rendered in bold red text (color "204")

Terminal Output Formatting
--------------------------

Skate implements custom output formatting to handle different types of data and display requirements:

### Binary Data Detection and Handling

When displaying data, Skate automatically detects whether the output is valid UTF-8 text or binary data. Binary data is handled differently:

1.  By default, binary data is replaced with "(omitted binary data)" when outputting to a terminal
2.  The `--show-binary` flag (available in `get` and `list` commands) overrides this behavior to display binary data

Sources: [main.go377-392](https://github.com/charmbracelet/skate/blob/09860417/main.go#L377-L392)
 [main.go436](https://github.com/charmbracelet/skate/blob/09860417/main.go#L436-L436)

### Terminal Detection

Skate uses the `term` package to detect if output is being directed to a terminal or redirected to a file/pipe. This affects how data is displayed:

*   When output is to a terminal, binary data may be omitted
*   When output is redirected, raw data is always passed through

Sources: [main.go380](https://github.com/charmbracelet/skate/blob/09860417/main.go#L380-L380)

Command-Specific Output Formatting
----------------------------------

Different commands have specific output formatting requirements:

### List Command Output

The `list` command supports multiple output formats controlled by flags:

| Flag | Description | Effect |
| --- | --- | --- |
| `--keys-only`, `-k` | Show only keys | Outputs only key names |
| `--values-only`, `-v` | Show only values | Outputs only values |
| `--delimiter`, `-d` | Specify delimiter | Sets character(s) between key and value (default: tab) |
| `--reverse`, `-r` | Reverse order | Lists items in reverse lexicographic order |
| `--show-binary`, `-b` | Show binary data | Displays binary data instead of omitting |

Sources: [main.go307-364](https://github.com/charmbracelet/skate/blob/09860417/main.go#L307-L364)
 [main.go431-435](https://github.com/charmbracelet/skate/blob/09860417/main.go#L431-L435)

### Get Command Output

The `get` command outputs the value associated with a key. It also supports the `--show-binary` flag to control binary data display.

User Interaction
----------------

Skate implements user interaction for certain operations that require confirmation:

### Delete Database Confirmation

When deleting a database with the `delete-db` command, Skate:

1.  Presents a confirmation message with the database path highlighted in warning style
2.  Waits for user input (y/n)
3.  Proceeds or aborts based on the response
4.  Provides feedback on the action taken

Sources: [main.go239-274](https://github.com/charmbracelet/skate/blob/09860417/main.go#L239-L274)
 [main.go257-260](https://github.com/charmbracelet/skate/blob/09860417/main.go#L257-L260)

Output Formatting Implementation
--------------------------------

The core of the terminal interface output formatting is implemented in the `printFromKV` function, which handles:

1.  Determining if output is to a terminal
2.  Checking if data is valid UTF-8 text
3.  Substituting binary data with a placeholder when appropriate
4.  Formatting output according to the provided format string

Example Output Format
---------------------

The terminal interface uses different output formats depending on the command and flags. For example:

1.  **Default list output**: Key and value separated by a tab (or custom delimiter)
2.  **Keys-only list**: One key per line
3.  **Values-only list**: One value per line
4.  **Get command**: Just the value, followed by a newline if output is to a terminal
