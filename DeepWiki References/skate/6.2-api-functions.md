API Functions
=============

Relevant source files

*   [main.go](https://github.com/charmbracelet/skate/blob/09860417/main.go)
    

This page documents Skate's internal API functions that power the command-line interface. These functions handle key-value operations, database management, path handling, and transaction processing. For information about the command-line interface itself, see [Command Line Interface](https://deepwiki.com/charmbracelet/skate/6.1-command-line-interface)
.

Overview
--------

Skate's codebase is organized around a set of core API functions that implement the key-value store functionality. These functions are designed to be modular and reusable, each serving a specific purpose in the application's architecture.

Core Key-Value Operations
-------------------------

These functions implement the fundamental CRUD (Create, Read, Update, Delete) operations for key-value pairs.

### set

Sets a value for a key, optionally in a specific database. The function can accept the value either as a command-line argument or read from standard input.

Sources: [main.go123-145](https://github.com/charmbracelet/skate/blob/09860417/main.go#L123-L145)

### get

Retrieves a value for a given key from the specified database and prints it to standard output.

Sources: [main.go147-170](https://github.com/charmbracelet/skate/blob/09860417/main.go#L147-L170)

### del

Deletes a key-value pair from the specified database.

Sources: [main.go172-186](https://github.com/charmbracelet/skate/blob/09860417/main.go#L172-L186)

### list

Lists all key-value pairs in a database with support for various formatting options controlled by flags.

Sources: [main.go307-364](https://github.com/charmbracelet/skate/blob/09860417/main.go#L307-L364)

Database Management Functions

These functions handle operations on entire databases rather than individual key-value pairs.

### listDbs

### getDbs

Usage Examples
--------------

### Setting and Getting Values

The core key-value operations in Skate involve parsing keys, opening the appropriate database, and performing operations within a transaction:

1.  The key is parsed to extract the actual key and database name
2.  The database is opened
3.  The operation is performed within a transaction
4.  For retrieval operations, the result is formatted and printed

### Database Management

Database management operations involve:

1.  Finding the database path
2.  For listing, reading directory contents
3.  For deletion, confirming with the user and removing the database directory

### Transaction Processing

All database operations use the `wrap` function to handle transactions:

1.  A transaction is created (read-only or read-write)
2.  The operation function is executed within the transaction
3.  The transaction is committed on success or discarded on failure
