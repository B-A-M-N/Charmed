Build Process
=============

Relevant source files

*   [.github/workflows/build.yml](https://github.com/charmbracelet/skate/blob/09860417/.github/workflows/build.yml)
    
*   [.goreleaser.yml](https://github.com/charmbracelet/skate/blob/09860417/.goreleaser.yml)
    
*   [go.mod](https://github.com/charmbracelet/skate/blob/09860417/go.mod)
    

This document details the build process for Skate, explaining how to build the project from source and providing insights into the build toolchain. For information about the release process, see [Release Process](https://deepwiki.com/charmbracelet/skate/5.2-release-process)
.

Prerequisites
-------------

Before building Skate from source, ensure you have the following prerequisites installed:

*   Go 1.23 or later (the project uses Go 1.23.0 with toolchain 1.24.1)
*   Git

Sources: [go.mod3-5](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L3-L5)

Build Toolchain
---------------

Skate uses a standard Go build toolchain with additional support from GoReleaser for creating distribution packages. The project's dependencies are managed through Go modules.

Sources: [go.mod3-5](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L3-L5)
 [.goreleaser.yml1-16](https://github.com/charmbracelet/skate/blob/09860417/.goreleaser.yml#L1-L16)

Dependencies
------------

Skate relies on several key dependencies for its functionality:

| Dependency | Purpose |
| --- | --- |
| github.com/dgraph-io/badger/v4 | Underlying storage engine for the key-value store |
| github.com/spf13/cobra | Command-line interface framework |
| github.com/charmbracelet/lipgloss | Terminal styling and UI components |
| github.com/muesli/go-app-paths | Cross-platform application paths management |
| github.com/agnivade/levenshtein | Fuzzy matching for database suggestions |
| github.com/muesli/mango-cobra | Man page generation |

Sources: [go.mod7-16](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L7-L16)

Building From Source
--------------------

### Local Build

To build Skate locally from source:

1.  Clone the repository:
    
        git clone https://github.com/charmbracelet/skate.git
        cd skate
        
    
2.  Build the binary:
    
        go build
        
    
3.  Install to your GOPATH (optional):
    
        go install
        
    

The compiled binary will be named `skate` (or `skate.exe` on Windows).

### Build Configuration

The project uses standard Go build tags and environment variables to configure the build process. No special build tags are required for basic functionality.

Continuous Integration
----------------------

Skate uses GitHub Actions for continuous integration and deployment. The build workflow is triggered on push and pull request events.

The CI process uses workflows from the `charmbracelet/meta` repository, which provides standardized build configurations for Charm projects.

Sources: [.github/workflows/build.yml1-12](https://github.com/charmbracelet/skate/blob/09860417/.github/workflows/build.yml#L1-L12)

Release Build Process
---------------------

For official releases, Skate uses GoReleaser to create binaries for multiple platforms and package them appropriately. The release configuration is defined in `.goreleaser.yml`.

GoReleaser handles:

*   Cross-compilation for multiple platforms
*   Binary signing
*   Package creation (Homebrew, Debian, RPM)
*   GitHub release creation
*   Docker image building

The release configuration is imported from a central template at `charmbracelet/meta/main/goreleaser-simple.yaml` and customized with project-specific variables.

Sources: [.goreleaser.yml1-16](https://github.com/charmbracelet/skate/blob/09860417/.goreleaser.yml#L1-L16)

Build Artifacts
---------------

The build process produces the following artifacts:

*   Binary executables for multiple platforms (Linux, macOS, Windows)
*   Homebrew formula for installation via brew
*   Docker images
*   Package manager configurations

Sources: [.goreleaser.yml9-15](https://github.com/charmbracelet/skate/blob/09860417/.goreleaser.yml#L9-L15)

Development Build vs Release Build
----------------------------------

There are two primary build paths for Skate:

1.  **Development Build**: Used during development and CI for testing
    
    *   Created with standard `go build` command
    *   Includes development-specific information
    *   Generated by GitHub Actions for each commit
2.  **Release Build**: Used for official releases
    
    *   Created with GoReleaser
    *   Optimized for distribution
    *   Generated only for tagged versions
    *   Includes additional metadata and packaging

Sources: [.github/workflows/build.yml5-12](https://github.com/charmbracelet/skate/blob/09860417/.github/workflows/build.yml#L5-L12)
