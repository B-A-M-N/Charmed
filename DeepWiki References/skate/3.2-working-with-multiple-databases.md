Working with Multiple Databases
===============================

Relevant source files

*   [README.md](https://github.com/charmbracelet/skate/blob/09860417/README.md?plain=1)
    
*   [main.go](https://github.com/charmbracelet/skate/blob/09860417/main.go)
    

This document explains how to create, use, and manage multiple databases in Skate. It covers the database organization, syntax for specifying databases in commands, and provides examples of common database operations. For information about basic key-value operations, see [Basic Commands](https://deepwiki.com/charmbracelet/skate/3.1-basic-commands)
.

Database Overview
-----------------

Skate allows you to organize your key-value data into separate databases, which are automatically created on demand when you reference them in commands. Each database is stored as a separate directory on your filesystem and uses BadgerDB as its storage engine.

Sources: [main.go225-236](https://github.com/charmbracelet/skate/blob/09860417/main.go#L225-L236)
 [main.go409-418](https://github.com/charmbracelet/skate/blob/09860417/main.go#L409-L418)

Database Naming and Syntax
--------------------------

To specify a database in Skate, use the `@` symbol followed by the database name. The database name should be placed after the key and before any values.

Sources: [main.go394-407](https://github.com/charmbracelet/skate/blob/09860417/main.go#L394-L407)
 [README.md119-131](https://github.com/charmbracelet/skate/blob/09860417/README.md?plain=1#L119-L131)

### Key-Database Parsing

Skate parses the key and database name from commands using the `keyParser()` function. This function splits the input string by the `@` symbol and returns the key and database name separately.

| Input Format | Parsed Key | Parsed Database | Result |
| --- | --- | --- | --- |
| `mykey` | `mykey` | `""` | Uses default database |
| `mykey@work` | `mykey` | `work` | Uses `work` database |
| Complex inputs with multiple `@` symbols are not supported and will return an error |     |     |     |

Sources: [main.go394-407](https://github.com/charmbracelet/skate/blob/09860417/main.go#L394-L407)

Creating and Using Databases
----------------------------

### Automatic Database Creation

Databases are created automatically when you first reference them in a command. There is no separate command needed to create a database.

When a database is first referenced, Skate:

1.  Parses the database name from the command
2.  Creates the directory structure if it doesn't exist
3.  Initializes a new BadgerDB instance in that directory

Database Storage Structure
--------------------------

Skate stores all databases in the user's data directory under a structured path.

The exact path depends on your operating system:

*   On macOS: `~/Library/Application Support/charm/kv/`
*   On Linux: `~/.local/share/charm/kv/`
*   On Windows: `%APPDATA%\charm\kv\`

Each database is stored as a separate directory containing BadgerDB files.

Sources: [main.go225-236](https://github.com/charmbracelet/skate/blob/09860417/main.go#L225-L236)

Internal Implementation Details
-------------------------------

When you specify a database in a command:

1.  The `keyParser()` function extracts the key and database name
2.  The `openKV()` function takes the database name and either uses it or falls back to "default"
3.  The `getFilePath()` function constructs the filesystem path for the database
4.  BadgerDB is opened at that path with `badger.Open()`

If you provide an invalid database name, the `findDb()` function will:

1.  Check if the database exists
2.  If not, get a list of all databases
3.  Calculate Levenshtein distances to suggest similar database names
4.  Return an error with suggestions

Summary
-------

Skate's multiple database support provides a simple yet powerful way to organize your key-value data. Databases are automatically created when needed, easily managed with dedicated commands, and help you separate concerns in your data storage. The `@db` syntax makes it intuitive to work with different databases across all of Skate's commands.
