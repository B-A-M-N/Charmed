Command Line Interface
======================

Relevant source files

*   [README.md](https://github.com/charmbracelet/skate/blob/09860417/README.md?plain=1)

This page documents the complete command-line interface (CLI) of Skate, a personal key-value store. It provides a comprehensive reference for all available commands, options, flags, and usage patterns. For information about the internal API functions, see [API Functions](https://deepwiki.com/charmbracelet/skate/6.2-api-functions)
.

Command Overview
----------------

Skate's command-line interface is built using the Cobra framework and provides commands for basic key-value operations, database management, and utility functions.

Sources: [main.go38-110](https://github.com/charmbracelet/skate/blob/09860417/main.go#L38-L110)

Root Command
------------

The root command is the entry point to Skate's CLI.

    skate [command] [flags]
    

When executed without any arguments, it displays the help information.

| Flag | Description |
| --- | --- |
| `--help`, `-h` | Display help for the command |
| `--version`, `-v` | Display version information |

Sources: [main.go39-46](https://github.com/charmbracelet/skate/blob/09860417/main.go#L39-L46)

Key-Value Commands
------------------

### set

Sets a value for a specified key, optionally in a specific database.

    skate set KEY[@DB] [VALUE]
    

If `VALUE` is omitted, the value is read from standard input, allowing for storing contents of files or output from other commands.

**Examples:**

    skate set foo bar
    skate set foo@custom bar
    skate set profile-pic < image.jpg
    

Sources: [main.go48-55](https://github.com/charmbracelet/skate/blob/09860417/main.go#L48-L55)
 [main.go123-145](https://github.com/charmbracelet/skate/blob/09860417/main.go#L123-L145)

### get

Retrieves the value of a specified key, optionally from a specific database.

    skate get KEY[@DB] [flags]
    

| Flag | Description |
| --- | --- |
| `--show-binary`, `-b` | Print binary values |

**Examples:**

    skate get foo
    skate get foo@custom
    skate get profile-pic > image.jpg

### delete

Deletes a key-value pair, optionally from a specific database.

**Examples:**

    skate delete foo

Database Management Commands
----------------------------

### list

Lists all key-value pairs, optionally from a specific database.

    skate list [@DB] [flags]
    

| Flag | Description |
| --- | --- |
| `--reverse`, `-r` | List in reverse lexicographic order |
| `--keys-only`, `-k` | Only print keys, don't fetch values |
| `--values-only`, `-v` | Only print values |
| `--delimiter`, `-d` | Delimiter to separate keys and values (default: tab) |
| `--show-binary`, `-b` | Print binary values |

**Examples:**

    skate list
    skate list @custom
    skate list -k
    skate list -v
    skate list -r
    skate list -d "="
    

Sources: [main.go72-77](https://github.com/charmbracelet/skate/blob/09860417/main.go#L72-L77)

### Key Format

Keys in Skate can be specified with or without a database:

*   Simple key: `key`
*   Key with database: `key@database`

The database part specifies which database to store the key-value pair in. If omitted, the key-value pair is stored in the `default` database.

Sources: [main.go394-407](https://github.com/charmbracelet/skate/blob/09860417/main.go#L394-L407)

----------------

The path structure follows this pattern:

*   `<user_data_dir>/charm/kv/<database_name>/`

Each database directory contains BadgerDB files for storing the key-value data.

Binary Data Handling
--------------------

Skate handles binary data intelligently by detecting if data is valid UTF-8 text. When displaying binary data to a terminal:

1.  By default, binary data is displayed as "(omitted binary data)"
2.  Use the `--show-binary` (`-b`) flag to force display of binary data
3.  When redirecting output to a file, binary data is always written as-is

This behavior applies to both the `get` and `list` commands.

Sources: [main.go377-392](https://github.com/charmbracelet/skate/blob/09860417/main.go#L377-L392)

Error Handling and Suggestions
------------------------------

When a specified database does not exist, Skate provides helpful suggestions for similar database names using the Levenshtein distance algorithm:

    "database_name" does not exist, did you mean "similar_database"?
    

This helps users identify typos or remember the correct database names.
