Dependency Management
=====================

Relevant source files

*   [.github/dependabot.yml](https://github.com/charmbracelet/skate/blob/09860417/.github/dependabot.yml)
    
*   [go.mod](https://github.com/charmbracelet/skate/blob/09860417/go.mod)
    
*   [go.sum](https://github.com/charmbracelet/skate/blob/09860417/go.sum)
    

This page documents how dependencies are managed in the Skate project, including key external libraries used, their purposes, and the processes for maintaining and updating dependencies.

Dependency Overview
-------------------

Skate is built using Go modules for dependency management. The project relies on several external libraries to provide its core functionality. All dependencies are tracked in the `go.mod` and `go.sum` files.

### Major Dependencies Structure

Sources: [go.mod7-16](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L7-L16)

### Core Dependencies by Function

| Dependency | Purpose | Usage in Skate |
| --- | --- | --- |
| github.com/dgraph-io/badger/v4 | Embeddable key-value database | Main storage engine for storing and retrieving key-value pairs |
| github.com/spf13/cobra | CLI application framework | Implementation of command-line interface and commands |
| github.com/charmbracelet/lipgloss | Terminal styling library | Formatting output with styles for better readability |
| github.com/muesli/go-app-paths | Application path management | Determining appropriate locations for database files |
| github.com/agnivade/levenshtein | String similarity algorithm | Used for fuzzy matching database names and suggestions |
| github.com/muesli/mango-cobra | Man page generator | Creating man pages from command documentation |
| github.com/muesli/roff | Man page formatter | Formatting man pages for documentation |
| golang.org/x/term | Terminal utilities | Terminal interaction capabilities |

Sources: [go.mod7-16](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L7-L16)

Dependency Declaration
----------------------

Skate uses Go modules for dependency management, which are declared in the `go.mod` file.

### Direct Dependencies

Direct dependencies are explicitly required by the Skate codebase:

    require (
        github.com/agnivade/levenshtein v1.2.1
        github.com/charmbracelet/lipgloss v1.1.0
        github.com/dgraph-io/badger/v4 v4.7.0
        github.com/muesli/go-app-paths v0.2.2
        github.com/muesli/mango-cobra v1.2.0
        github.com/muesli/roff v0.1.0
        github.com/spf13/cobra v1.9.1
        golang.org/x/term v0.31.0
    )
    

Sources: [go.mod7-16](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L7-L16)

### Indirect Dependencies

Indirect dependencies are automatically included when required by direct dependencies and are marked with `// indirect` comments:

    require (
        github.com/aymanbagabas/go-osc52/v2 v2.0.1 // indirect
        github.com/cespare/xxhash/v2 v2.3.0 // indirect
        github.com/charmbracelet/colorprofile v0.2.3-0.20250311203215-f60798e515dc // indirect
        ...
    )
    

Sources: [go.mod18-49](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L18-L49)

### Go Version Requirements

Skate specifies its Go version requirements in the `go.mod` file:

    go 1.23.0
    
    toolchain go1.24.1
    

This indicates that:

*   The minimum required Go version is 1.23.0
*   The code is built with Go toolchain version 1.24.1

Sources: [go.mod3-5](https://github.com/charmbracelet/skate/blob/09860417/go.mod#L3-L5)

Automated Dependency Management
-------------------------------

Skate uses GitHub's Dependabot to automate dependency updates, configured via the `.github/dependabot.yml` file.

### Dependabot Configuration

Dependabot is configured to monitor and update three types of dependencies:

1.  Go modules (`gomod`) - Main application dependencies
2.  GitHub Actions workflows - CI/CD dependencies
3.  Docker dependencies - Container configuration

For each ecosystem, updates are scheduled weekly on Mondays at 5:00 AM Eastern Time.

Sources: [.github/dependabot.yml1-42](https://github.com/charmbracelet/skate/blob/09860417/.github/dependabot.yml#L1-L42)

Manual Dependency Management
----------------------------

For developers contributing to Skate, the following procedures can be used for manual dependency management:

### Adding a New Dependency

To add a new dependency to the project:

1.  Use the `go get` command:
    
        go get github.com/example/dependency@v1.2.3
        
    
2.  The dependency will be automatically added to `go.mod` and `go.sum`
    
3.  Commit the updated `go.mod` and `go.sum` files
    

### Updating Dependencies

To update a specific dependency:

1.  Use `go get` with the `-u` flag:
    
        go get -u github.com/example/dependency
        
    
2.  To update all dependencies:
    
        go get -u ./...
        
    
3.  To update to a specific version:
    
        go get github.com/example/dependency@v1.3.0
        
    

### Dependency Cleanup and Verification

To maintain a clean dependency tree:

1.  Run `go mod tidy` to remove unused dependencies and add missing ones:
    
        go mod tidy
        
    
2.  Verify dependency integrity:
    
        go mod verify
