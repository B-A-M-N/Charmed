This document covers the core commands in Skate, a personal key-value store. It details how to set, get, delete, and list key-value pairs using the command-line interface. For information about working with multiple databases, see [Working with Multiple Databases](https://deepwiki.com/charmbracelet/skate/3.2-working-with-multiple-databases)

Command Syntax and Database Format
----------------------------------

All Skate commands use a consistent key format: `KEY[@DB]`. The `@DB` portion is optional and specifies which database to use. If omitted, Skate uses the `default` database.

Sources: [main.go394-407](https://github.com/charmbracelet/skate/blob/09860417/main.go#L394-L407)

Set Command
-----------

The `set` command stores a value for a given key in the specified database.

### Syntax

    skate set KEY[@DB] [VALUE]
    

If `VALUE` is omitted, the value is read from standard input, allowing you to pipe content or redirect from files.

List Command
------------

The `list` command displays all key-value pairs in the specified database.

### Syntax

    skate list [@DB]

### Options

| `--keys-only` | `-k` | Only print keys (faster) |
| `--values-only` | `-v` | Only print values |

Summary
-------

Skate provides four core commands for managing key-value data:

| Command | Purpose | Example |
| --- | --- | --- |
| `set` | Store a value for a key | `skate set kitty meow` |
| `get` | Retrieve a value by key | `skate get kitty` |
| `delete` | Remove a key-value pair | `skate delete kitty` |
| `list` | Display all key-value pairs | `skate list` |

Each command supports the `KEY[@DB]` format to specify which database to target, with "default" being used if no database is specified.
