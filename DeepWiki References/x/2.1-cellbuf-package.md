Purpose and Scope
-----------------

The `cellbuf` package provides terminal cell buffer management and efficient screen rendering for terminal-based applications. It implements a double-buffering system with diff-based rendering that generates minimal ANSI escape sequences to update only changed portions of the screen. This package is the foundation for building high-performance terminal user interfaces.

For ANSI sequence generation and parsing, see [ANSI Processing and Control](https://deepwiki.com/charmbracelet/x/4-ansi-processing-and-control)
. For terminal input handling, see [Input Processing System](https://deepwiki.com/charmbracelet/x/3-input-processing-system)
. For virtual terminal emulation, see [Virtual Terminal Emulator (vt)](https://deepwiki.com/charmbracelet/x/2.2-virtual-terminal-emulator-(vt))
.

Double-Buffer System
--------------------

The `Screen` uses two buffers to implement efficient diff-based rendering:

### Rendering Workflow

1.  **Application modifies `newbuf`**: Calls like `SetCell(x, y, cell)` update the new buffer and mark changed lines in the `touch` map [cellbuf/screen.go467-487](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L467-L487)
    
2.  **`Render()` performs diff**: The `transformLine(y)` function compares each touched line in `curbuf` vs `newbuf` [cellbuf/screen.go927-1128](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L927-L1128)
    
3.  **Optimization strategies applied**:
    
    *   Use `moveCursor()` to minimize cursor movement sequences [cellbuf/screen.go188-244](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L188-L244)
        
    *   Use `emitRange()` with `ansi.ECH` and `ansi.REP` for repeated cells [cellbuf/screen.go744-811](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L744-L811)
        
    *   Use `clearToEnd()` with `ansi.EL` for trailing blanks [cellbuf/screen.go852-879](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L852-L879)
        
4.  **`Flush()` writes output**: Accumulated ANSI sequences in `buf` are written to the `io.Writer` [cellbuf/screen.go1241-1257](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L1241-L1257)
    
5.  **Sync state**: Changed portions of `curbuf` are updated to match `newbuf` [cellbuf/screen.go1122-1127](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L1122-L1127)
    

**Key Data Structures:**

    touch map[int]lineData  // Tracks which lines changed
      lineData {
        firstCell int  // First changed cell index
        lastCell int   // Last changed cell index
        oldIndex int   // For scrolling optimization
      }

Cursor Movement Optimization
----------------------------

The `moveCursor()` function employs multiple strategies to minimize escape sequence length:

### Cursor Movement Methods

**Method #1: Relative Movement**

Uses `relativeCursorMove()` [cellbuf/screen.go38-186](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L38-L186)
 which combines:

*   **Vertical**: `ansi.VPA`, `ansi.CUU`, `ansi.CUD`, or `\n` / `ansi.RI`
*   **Horizontal**: `ansi.HPA`, `ansi.CUF`, `ansi.CUB`, `\t`, or `\b`
*   **Overwrite optimization**: Write cell content instead of escape sequences [cellbuf/screen.go122-150](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L122-L150)
    

**Method #2: Carriage Return + Relative**

    "\r" + relativeCursorMove(s, 0, fy, x, y, ...)
    

Useful when target is near column 0 [cellbuf/screen.go229-232](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L229-L232)

**Method #3: Home + Relative**

    ansi.CursorHomePosition + relativeCursorMove(s, 0, 0, x, y, ...)
    

Useful for positions near top-left [cellbuf/screen.go236-239](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L236-L239)

### Optimization Flags

| Option | Enables | Benefit |
| --- | --- | --- |
| `opts.HardTabs` | Use `\t` and `ansi.CHT` / `ansi.CBT` | Faster horizontal movement at tab stops |
| `opts.Backspace` | Use `\b` for backward movement | Shorter sequences for small backward moves |
| `opts.RelativeCursor` | Prefer relative over absolute | Better for inline mode |

### Wide Character Handling

Wide characters (width > 1) require special handling:

1.  **Setting a wide cell**: The primary cell stores content, following cells become zero-width `EmptyCell` placeholders [cellbuf/buffer.go177-182](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/buffer.go#L177-L182)
    
2.  **Overwriting wide cells**: When writing to any part of a wide character, the entire character is replaced with blanks [cellbuf/buffer.go143-160](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/buffer.go#L143-L160)
    
3.  **Edge cases**: Wide characters at buffer boundaries are converted to blanks [cellbuf/buffer.go167-172](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/buffer.go#L167-L172)
    

**Example:**

    Before: |A|B|中(w=2)| |  (中 occupies 2 cells)
    Set(2, 0, 'X'):
    After:  |A|B|X| | |     (中 replaced with blanks, X written)
    

**Sources:** [cellbuf/buffer.go134-186](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/buffer.go#L134-L186)

Dependencies
------------

| --- | --- | --- |

| `term` | Terminal utilities | Size detection via `term.GetSize()` |
| `colorprofile` | Color profile management | Downsample colors for terminal capabilities |
| `go-runewidth` | Character width | Calculate display width of runes |
| `uniseg` | Unicode segmentation | Extract grapheme clusters correctly |

Usage Example
-------------

**Key Points:**

1.  `SetCell()` modifies `newbuf` and marks lines as touched
2.  `Render()` diffs `curbuf` vs `newbuf` and generates sequences
3.  `Flush()` writes accumulated sequences to `io.Writer`
4.  Separation allows batching multiple updates before rendering

**Sources:** [cellbuf/screen.go467-487](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/screen.go#L467-L487)
