Windows Console Input
=====================

This document covers the `windows` package, which provides low-level bindings to the Windows Console API for reading input events. It exposes Windows Console input functions (`ReadConsoleInput`, `PeekConsoleInput`) and structures (`InputRecord`, `KeyEventRecord`, `MouseEventRecord`) that enable applications to capture keyboard, mouse, focus, and window resize events directly from the Windows Console. For higher-level cross-platform input handling that uses this package, see page [3.1](https://deepwiki.com/charmbracelet/x/3.1-input-package)
.

Package Overview
----------------

The `windows` package wraps Windows Console API functions through Go syscalls, providing type-safe access to console input functionality. It defines structures that mirror Windows Console API event records and constants for virtual key codes, mouse buttons, and control key states.

**Package Structure**

InputRecord Structure
---------------------

The `InputRecord` type represents the Windows `INPUT_RECORD` structure, which is a discriminated union that can hold different types of console events. The structure contains an event type indicator and a 16-byte event data field.

**InputRecord Layout**

| Field | Type | Purpose |
| --- | --- | --- |
| `EventType` | `uint16` | Identifies which type of event is stored in `Event` |
| `_` | `[2]byte` | Padding to align the structure to 32-bit boundary |
| `Event` | `[16]byte` | Raw event data, decoded based on `EventType` |

The `EventType` field determines how the `Event` byte array should be interpreted. Each event type has a corresponding decoding method that extracts the typed data using little-endian binary encoding.

Console Input Functions
-----------------------

The package provides four syscall wrappers for reading and managing console input events through the Windows Console API.

**Console API Functions**

| Function | Parameters | Returns | Purpose |
| --- | --- | --- | --- |
| `ReadConsoleInput` | `console Handle, buf *InputRecord, toread uint32, read *uint32` | `error` | Reads input events from console buffer, blocking if none available |
| `PeekConsoleInput` | `console Handle, buf *InputRecord, toread uint32, read *uint32` | `error` | Reads input events without removing them from buffer |
| `GetNumberOfConsoleInputEvents` | `console Handle, numevents *uint32` | `error` | Returns count of unread events in buffer |
| `FlushConsoleInputBuffer` | `console Handle` | `error` | Clears all pending input events |

The syscall declarations use `//sys` directives that are processed by `mkwinsyscall` to generate the actual syscall stubs in `zsyscall_windows.go`. All functions use the wide-character (Unicode) versions (`ReadConsoleInputW`, `PeekConsoleInputW`).

Sources: [windows/syscall\_windows.go11-14](https://github.com/charmbracelet/x/blob/7642919e/windows/syscall_windows.go#L11-L14)
 [windows/zsyscall\_windows.go38-77](https://github.com/charmbracelet/x/blob/7642919e/windows/zsyscall_windows.go#L38-L77)

Key Event Records
-----------------

The `KeyEventRecord` structure contains detailed information about keyboard events, including the key pressed, modifier states, and repeat count.

**KeyEventRecord Structure**

The `KeyEvent()` method extracts the key event data from the `Event` byte array:

| Byte Range | Field | Encoding |
| --- | --- | --- |
| `[0:4]` | `KeyDown` | `uint32 > 0` for key press |
| `[4:6]` | `RepeatCount` | Little-endian `uint16` |
| `[6:8]` | `VirtualKeyCode` | Little-endian `uint16` |
| `[8:10]` | `VirtualScanCode` | Little-endian `uint16` |
| `[10:12]` | `Char` | Little-endian `uint16` cast to `rune` |
| `[12:16]` | `ControlKeyState` | Little-endian `uint32` |

**Virtual Key Codes**

The package defines constants for all Windows Virtual Key Codes (VK\_\*). These are device-independent key identifiers:

| VK Constant | Hex Value | Key |
| --- | --- | --- |
| `VK_BACK` | `0x08` | Backspace |
| `VK_TAB` | `0x09` | Tab |
| `VK_RETURN` | `0x0D` | Enter |
| `VK_SHIFT` | `0x10` | Shift (generic) |
| `VK_CONTROL` | `0x11` | Control (generic) |
| `VK_MENU` | `0x12` | Alt (generic) |
| `VK_ESCAPE` | `0x1B` | Escape |
| `VK_SPACE` | `0x20` | Space |
| `VK_LEFT` | `0x25` | Left Arrow |
| `VK_UP` | `0x26` | Up Arrow |
| `VK_RIGHT` | `0x27` | Right Arrow |
| `VK_DOWN` | `0x28` | Down Arrow |
| `VK_F1`\-`VK_F24` | `0x70`\-`0x87` | Function keys |
| `VK_LSHIFT` | `0xA0` | Left Shift |
| `VK_RSHIFT` | `0xA1` | Right Shift |
| `VK_LCONTROL` | `0xA2` | Left Control |
| `VK_RCONTROL` | `0xA3` | Right Control |
| `VK_LMENU` | `0xA4` | Left Alt |
| `VK_RMENU` | `0xA5` | Right Alt |

**Control Key State Flags**

The `ControlKeyState` field uses bitflags to indicate modifier key and lock states:

| Constant | Hex Value | Meaning |
| --- | --- | --- |
| `RIGHT_ALT_PRESSED` | `0x0001` | Right Alt key is down |
| `LEFT_ALT_PRESSED` | `0x0002` | Left Alt key is down |
| `RIGHT_CTRL_PRESSED` | `0x0004` | Right Control key is down |
| `LEFT_CTRL_PRESSED` | `0x0008` | Left Control key is down |
| `SHIFT_PRESSED` | `0x0010` | Shift key is down |
| `NUMLOCK_ON` | `0x0020` | Num Lock is active |
| `SCROLLLOCK_ON` | `0x0040` | Scroll Lock is active |
| `CAPSLOCK_ON` | `0x0080` | Caps Lock is active |
| `ENHANCED_KEY` | `0x0100` | Key is from extended keyboard section |

Mouse Event Records
-------------------

The `MouseEventRecord` structure captures mouse input events including button presses, cursor movement, and wheel scrolling.

**MouseEventRecord Structure**

The `MouseEvent()` method extracts mouse event data from the `Event` byte array:

| Byte Range | Field | Encoding |
| --- | --- | --- |
| `[0:2]` | `MousePositon.X` | Little-endian `int16` |
| `[2:4]` | `MousePositon.Y` | Little-endian `int16` |
| `[4:8]` | `ButtonState` | Little-endian `uint32` |
| `[8:12]` | `ControlKeyState` | Little-endian `uint32` |
| `[12:16]` | `EventFlags` | Little-endian `uint32` |

**Button State Constants**

The `ButtonState` field uses bitflags to indicate which mouse buttons are currently pressed:

| Constant | Hex Value | Button |
| --- | --- | --- |
| `FROM_LEFT_1ST_BUTTON_PRESSED` | `0x0001` | Left mouse button |
| `RIGHTMOST_BUTTON_PRESSED` | `0x0002` | Right mouse button |
| `FROM_LEFT_2ND_BUTTON_PRESSED` | `0x0004` | Middle mouse button |
| `FROM_LEFT_3RD_BUTTON_PRESSED` | `0x0008` | Mouse backward button |
| `FROM_LEFT_4TH_BUTTON_PRESSED` | `0x0010` | Mouse forward button |

**Event Flags**

The `EventFlags` field indicates the type of mouse event:

| Constant | Hex Value | Event Type |
| --- | --- | --- |
| `MOUSE_MOVED` | `0x0001` | Mouse cursor moved |
| `DOUBLE_CLICK` | `0x0002` | Double-click occurred |
| `MOUSE_WHEELED` | `0x0004` | Vertical mouse wheel scrolled |
| `MOUSE_HWHEELED` | `0x0008` | Horizontal mouse wheel scrolled |

When `EventFlags` is 0, it indicates a standard button press or release event. The high word of `ButtonState` contains the wheel delta for wheel events.

Mouse Event Processing
----------------------

Windows mouse events are processed through the Windows Console API mouse event records, handling different mouse actions, wheel events, and modifier key combinations.

The `mouseEventButton` function determines which mouse button was pressed by comparing previous and current button states:

| Button Constant | Windows Flag | Mouse Button |
| --- | --- | --- |
| `FROM_LEFT_1ST_BUTTON_PRESSED` | Left mouse button | `MouseLeft` |
| `RIGHTMOST_BUTTON_PRESSED` | Right mouse button | `MouseRight` |
| `FROM_LEFT_2ND_BUTTON_PRESSED` | Middle mouse button | `MouseMiddle` |
| `FROM_LEFT_3RD_BUTTON_PRESSED` | Mouse backward | `MouseBackward` |
| `FROM_LEFT_4TH_BUTTON_PRESSED` | Mouse forward | `MouseForward` |

State Management
----------------

The Windows input system maintains complex state through the `win32InputState` structure to handle multi-byte sequences, modifier key tracking, and window size change detection.

The state management serves several critical purposes:

| State Field | Purpose | Usage |
| --- | --- | --- |
| `ansiBuf`/`ansiIdx` | ANSI escape sequence collection | Accumulates bytes when VKC=0 until complete sequence |
| `utf16Buf`/`utf16Half` | UTF-16 surrogate pair handling | Manages high/low surrogate pairs for Unicode |
| `lastCks` | Previous control key state | Detects modifier key release events |
| `lastMouseBtns` | Previous mouse button state | Determines button press/release transitions |
| `lastWinsizeX`/`lastWinsizeY` | Previous window dimensions | Prevents duplicate resize events |
