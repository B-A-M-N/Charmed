Purpose and Scope
-----------------

This page documents the **generation** of ANSI and DEC terminal control sequences in the `ansi` package. These functions produce escape sequences for cursor movement, screen manipulation, mode control, and device communication. For parsing incoming ANSI sequences and manipulating styled text, see [Parser and Text Manipulation](https://deepwiki.com/charmbracelet/x/4.1-parser-and-text-manipulation)
. For generating SGR styling sequences (colors, bold, italic), see [Styling and Colors](https://deepwiki.com/charmbracelet/x/4.3-styling-and-colors)
.

Overview
--------

The `ansi` package provides functions and constants for generating terminal control sequences that instruct the terminal to perform operations like moving the cursor, clearing the screen, changing modes, and requesting information. These sequences follow ANSI X3.64, ECMA-48, and DEC VT standards.

Cursor Movement and Positioning
-------------------------------

### Basic Cursor Movement

The package provides functions for relative cursor movement in four directions:

| Function | Alias | Description | Escape Sequence |
| --- | --- | --- | --- |
| `CursorUp(n)` | `CUU(n)` | Move cursor up n cells | `CSI n A` |
| `CursorDown(n)` | `CUD(n)` | Move cursor down n cells | `CSI n B` |
| `CursorForward(n)` | `CUF(n)` | Move cursor right n cells | `CSI n C` |
| `CursorBackward(n)` | `CUB(n)` | Move cursor left n cells | `CSI n D` |

Constants are provided for single-cell movements: `CUU1`, `CUD1`, `CUF1`, `CUB1`.

**Sources:** [ansi/cursor.go62-194](https://github.com/charmbracelet/x/blob/7642919e/ansi/cursor.go#L62-L194)

### Character Insertion and Deletion

`InsertCharacter` inserts n blank characters at the cursor position, shifting existing characters right. `DeleteCharacter` deletes n characters at the cursor, shifting remaining characters left. `RepeatPreviousCharacter` repeats the last printed character n times.

**Sources:** [ansi/screen.go228-270](https://github.com/charmbracelet/x/blob/7642919e/ansi/screen.go#L228-L270)

### Light/Dark Mode Reporting

Primary Device Attributes (DA1) report terminal capabilities like 132-column support, Sixel graphics, and soft character sets. Secondary Device Attributes (DA2) identify the terminal type and version. Tertiary Device Attributes (DA3) report the unit ID.

**Sources:** [ansi/ctrl.go8-157](https://github.com/charmbracelet/x/blob/7642919e/ansi/ctrl.go#L8-L157)

Keypad Control
--------------

Application mode makes the keypad send application-specific sequences (e.g., `SS3 A` for up arrow), while numeric mode sends standard ANSI sequences. The escape sequence method (DECKPAM/DECKPNM) and the mode method (DECNKM) achieve the same result.

**Sources:** [ansi/keypad.go1-29](https://github.com/charmbracelet/x/blob/7642919e/ansi/keypad.go#L1-L29)
