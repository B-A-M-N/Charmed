Colors and Styling
==================

This document covers the comprehensive color management and text styling system in the `ansi` package, which provides terminal applications with support for various color formats, SGR (Select Graphic Rendition) attributes, and ANSI escape sequence generation. The system handles color conversion between different terminal color modes (3/4-bit, 8-bit, 24-bit) and provides a fluent interface for building styled terminal output.

For information about ANSI escape sequence parsing and terminal control, see [ANSI Parser and Decoder](https://deepwiki.com/charmbracelet/x/4.1-parser-and-text-manipulation)
. For string manipulation with ANSI-aware operations, see [String Manipulation and Width](https://deepwiki.com/charmbracelet/x/4.3-styling-and-colors)
.

Color Type System
-----------------

The color system provides a unified interface for representing different terminal color formats through the `Color` interface, with specific implementations for each color mode supported by terminals.

### Basic Colors (3/4-bit)

The `BasicColor` type represents the standard 16 ANSI colors, defined as constants from `Black` (0) to `BrightWhite` (15). These colors provide maximum terminal compatibility.

| Color Range | Constants | Description |
| --- | --- | --- |
| 0-7 | `Black`, `Red`, `Green`, `Yellow`, `Blue`, `Magenta`, `Cyan`, `White` | Standard ANSI colors |
| 8-15 | `BrightBlack`, `BrightRed`, `BrightGreen`, etc. | Bright/bold variants |

### Extended Colors (8-bit)

The `IndexedColor` type supports the 256-color xterm palette, covering:

*   Colors 0-15: Same as BasicColor
*   Colors 16-231: 6×6×6 RGB color cube
*   Colors 232-255: 24-level grayscale ramp

### True Colors (24-bit)

The `RGBColor` type provides full 24-bit color support with individual red, green, and blue components. The deprecated `TrueColor` type serves as a compatibility alias.

**Sources:** [ansi/color.go9-157](https://github.com/charmbracelet/x/blob/7642919e/ansi/color.go#L9-L157)

Style System and SGR Attributes
-------------------------------

The styling system centers around the `Style` type, which provides a fluent interface for building SGR (Select Graphic Rendition) escape sequences that control text appearance.

### Core Style Attributes

The `Style` type implements a builder pattern where each method returns a new style with the additional attribute applied:

| Method | SGR Code | Description |
| --- | --- | --- |
| `Bold()` | 1   | Bold/increased intensity |
| `Faint()` | 2   | Faint/decreased intensity |
| `Italic()` | 3   | Italic text |
| `Underline()` | 4   | Single underline |
| `SlowBlink()` | 5   | Slow blink (≤150/min) |
| `RapidBlink()` | 6   | Rapid blink (≥150/min) |
| `Reverse()` | 7   | Reverse/invert colors |
| `Conceal()` | 8   | Conceal/hide text |
| `Strikethrough()` | 9   | Strikethrough/crossed out |

### Color Encoding Functions

The internal functions `foregroundColorString()`, `backgroundColorString()`, and `underlineColorString()` handle the conversion from color types to SGR parameter strings:

*   **Basic Colors**: Direct mapping to SGR codes (30-37, 40-47, 90-97, 100-107)
*   **Extended Colors**: 256-color format using `38;5;n`, `48;5;n`, `58;5;n`
*   **True Colors**: 24-bit format using `38;2;r;g;b`, `48;2;r;g;b`, `58;2;r;g;b`

**Sources:** [ansi/style.go362-502](https://github.com/charmbracelet/x/blob/7642919e/ansi/style.go#L362-L502)

Color Conversion and Compatibility
----------------------------------

The system provides automatic color conversion to ensure compatibility across different terminal capabilities.

### Conversion Algorithm Details

The `Convert256()` function implements a sophisticated color matching algorithm:

1.  **Color Cube Mapping**: Maps RGB values to the nearest point in xterm's 6×6×6 color cube
2.  **Grayscale Mapping**: Calculates the closest grayscale value from the 24-level ramp
3.  **Distance Comparison**: Uses HSLuv color space distance to choose between cube and grayscale
4.  **Lookup Table**: The `ansi256To16` array provides direct mapping from 256-color to 16-color indices

### XParseColor Support

The `XParseColor()` function provides X11-style color string parsing:

| Format | Example | Description |
| --- | --- | --- |
| `#RGB` | `#f0f` | 3-digit hex |
| `#RRGGBB` | `#ff00ff` | 6-digit hex |
| `rgb:RRRR/GGGG/BBBB` | `rgb:ffff/0000/ffff` | X11 RGB format |
| `rgba:RRRR/GGGG/BBBB/AAAA` | `rgba:ffff/0000/ffff/8000` | X11 RGBA format |

**Sources:** [ansi/color.go174-247](https://github.com/charmbracelet/x/blob/7642919e/ansi/color.go#L174-L247)

Style Sequence Parsing
----------------------

The system includes comprehensive parsing capabilities for reading SGR color information from ANSI escape sequences, supporting multiple color formats and parameter separators.

### Parameter Separator Support

The `ReadStyleColor()` function supports both traditional semicolon-separated and modern colon-separated parameter formats:

*   **Semicolon format**: `38;2;255;128;0` (legacy compatibility)
*   **Colon format**: `38:2:255:128:0` (ITU T.416 standard)
*   **Mixed format**: Not supported (returns error)

### Color Space Extensions

The parser supports optional color space identifiers and tolerance values as defined in ITU T.416, though these are currently ignored in color construction.

**Sources:** [ansi/style.go504-673](https://github.com/charmbracelet/x/blob/7642919e/ansi/style.go#L504-L673)
 [cellbuf/sequence.go127-131](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/sequence.go#L127-L131)

Integration with Cell Buffer System
-----------------------------------

The color and styling system integrates with the cell buffer rendering system through the `ReadStyle()` function, which processes SGR sequences and updates style state.

The `ReadStyle()` function in the cellbuf package handles the complete SGR parameter processing, including:

*   Basic attribute toggling (bold, italic, underline, etc.)
*   Foreground colors (parameters 30-37, 38, 39, 90-97)
*   Background colors (parameters 40-47, 48, 49, 100-107)
*   Underline colors (parameters 58, 59)
*   Advanced underline styles with sub-parameters

**Sources:** [cellbuf/sequence.go10-115](https://github.com/charmbracelet/x/blob/7642919e/cellbuf/sequence.go#L10-L115)
