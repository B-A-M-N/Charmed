Purpose and Scope
-----------------

The `vt` package provides a complete virtual terminal emulator that interprets ANSI escape sequences, maintains terminal state, and manages a virtual screen buffer. It implements terminal emulation compatible with VT220 and modern terminal features including alternate screen, cursor control, scroll regions, and terminal modes.

This package focuses on **emulating** a terminal by processing ANSI sequences and maintaining state. For parsing ANSI sequences, see [ANSI Processing and Control](https://deepwiki.com/charmbracelet/x/4-ansi-processing-and-control)
. For rendering to actual terminals, see [cellbuf Package](https://deepwiki.com/charmbracelet/x/2.1-cellbuf-package)
.

**Sources**: [vt/terminal.go1-57](https://github.com/charmbracelet/x/blob/7642919e/vt/terminal.go#L1-L57)
 [vt/go.mod1-28](https://github.com/charmbracelet/x/blob/7642919e/vt/go.mod#L1-L28)

### UTF-8 and Grapheme Handling

The emulator properly handles multi-byte UTF-8 characters and grapheme clusters (combining characters). ASCII characters (0x20-0x7E) are handled directly, while non-ASCII characters are buffered and processed as complete graphemes.

The `flushGrapheme` method processes the buffered runes using `ansi.FirstGraphemeCluster` to extract proper grapheme clusters with their display widths.

--------------

### Screen Scrolling

Scrolling operations use the line insertion/deletion primitives:

| Operation | Implementation | Behavior |
| --- | --- | --- |
| `ScrollUp(n)` | Move cursor to top margin, delete n lines | Lines scroll up; new blank lines at bottom |
| `ScrollDown(n)` | Move cursor to top margin, insert n lines | Lines scroll down; new blank lines at top |

Both operations temporarily move the cursor to the scroll region's top edge to perform the operation, then restore the original cursor position.

Tab Stop Management
-------------------

The emulator maintains a set of horizontal tab stops that control the behavior of the HT (Horizontal Tab) control character.

### Tab Stop Structure

Tab stops are managed through a tabstop data structure that supports:

*   Setting a tab stop at a specific column
*   Clearing a tab stop at a specific column
*   Clearing all tab stops
*   Finding the next tab stop from a position
*   Finding the previous tab stop from a position
*   Resetting to default (every 8 columns)

### Tab Operations

*   **HTS** (`ESC H` or `0x88`) - Set tab stop at cursor position

Device Reports
--------------

The emulator responds to various device status and capability queries by writing responses to the output pipe.

### Report Types

| Sequence | Report | Response |
| --- | --- | --- |
| `CSI 5 n` | Operating Status | `CSI 0 n` (always ready) |
| `CSI 6 n` | Cursor Position Report | `CSI row;col R` |
| `CSI ? 6 n` | Extended Cursor Position | `CSI ? row;col;page R` |
| `CSI c` or `CSI 0 c` | Primary Device Attributes | `CSI ? 62;1;6;22 c` (VT220 with features) |
| `CSI > c` or `CSI > 0 c` | Secondary Device Attributes | `CSI > 1;10;0 c` (VT220, version 1.0) |
| `CSI ? {mode} $ p` | Request DEC Mode | `CSI ? {mode};{value} $ y` |
| `CSI {mode} $ p` | Request ANSI Mode | `CSI {mode};{value} $ y` |
| `OSC 10 ? ST` | Query Foreground Color | `OSC 10;rgb:rr/gg/bb ST` |
| `OSC 11 ? ST` | Query Background Color | `OSC 11;rgb:rr/gg/bb ST` |
| `OSC 12 ? ST` | Query Cursor Color | `OSC 12;rgb:rr/gg/bb ST` |

Grapheme and Width Handling
---------------------------

The emulator uses the `ansi` package's grapheme cluster support to correctly handle Unicode combining characters and emoji.

### Width Calculation Methods

The terminal supports different width calculation methods via the `WidthMethod` interface:

*   **GraphemeWidth** - Unicode-aware, handles combining characters and emoji
*   **WcWidth** - Traditional wcswidth calculation
*   **Custom methods** - Applications can provide their own

The emulator internally always uses `GraphemeWidth` for accurate rendering, but applications can query and set the method used for their own text processing.

**Sources**: [vt/utf8.go30-41](https://github.com/charmbracelet/x/blob/7642919e/vt/utf8.go#L30-L41)

### Phantom State (Pending Wrap)

When auto-wrap mode is enabled and a character is written at the rightmost column, the cursor enters a "phantom" state. The next printable character will wrap to the next line.

This phantom state is tracked by the `atPhantom` boolean in the emulator and is reset by cursor movement operations.

**Sources**: [vt/utf8.go54-61](https://github.com/charmbracelet/x/blob/7642919e/vt/utf8.go#L54-L61)
 [vt/utf8.go90-97](https://github.com/charmbracelet/x/blob/7642919e/vt/utf8.go#L90-L97)

Summary
-------

The vt package provides a comprehensive virtual terminal emulator with:

*   **Complete VT220 compatibility** with modern extensions
*   **Flexible handler system** for customization and extension
*   **Dual screen buffers** for full-screen applications
*   **Comprehensive callback system** for reactive updates
*   **Proper Unicode handling** including grapheme clusters
*   **Scroll region support** with independent margins
*   **Multiple rendering outputs** for different integration needs
*   **Character set mapping** for compatibility with legacy systems
*   **Tab stop management** matching physical terminal behavior
*   **Device reporting** for terminal capability queries

The architecture cleanly separates sequence parsing (via the `ansi` package), state management (via the `Screen` type), and rendering (via the ultraviolet Buffer system), making it suitable for embedding in various terminal-based applications.
