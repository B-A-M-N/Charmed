Terminal I/O (termios and wcwidth)
==================================

Relevant source files

*   [termios/go.mod](https://github.com/charmbracelet/x/blob/7642919e/termios/go.mod)
    
*   [wcwidth/go.mod](https://github.com/charmbracelet/x/blob/7642919e/wcwidth/go.mod)

Purpose and Scope
-----------------

This document covers two Unix-specific utility packages in the platform abstraction layer:

*   **termios**: Provides Unix/Linux terminal I/O control through system calls like `tcgetattr` and `tcsetattr`
*   **wcwidth**: Calculates character display width for proper terminal layout (deprecated wrapper)

These packages enable low-level terminal manipulation on Unix systems and accurate character width measurement for text rendering. For cross-platform terminal control interfaces, see [Pseudo-Terminal Interface (xpty)](https://deepwiki.com/charmbracelet/x/5.1-pseudo-terminal-interface-(xpty))
. For Windows-specific terminal I/O, see [Windows ConPTY](https://deepwiki.com/charmbracelet/x/5.2-windows-conpty)
. For higher-level input handling that uses these primitives, see [Input Package](https://deepwiki.com/charmbracelet/x/3.1-input-package)
.

Termios Package
---------------

### Purpose

The `termios` package provides Unix-specific terminal I/O control operations. It serves as a thin wrapper around `golang.org/x/sys/unix` to manipulate terminal attributes, enabling applications to:

*   Switch terminals into raw mode for character-by-character input
*   Save and restore terminal state
*   Configure terminal behavior (echo, canonical mode, signal generation)
*   Control terminal size and window dimensions

**Sources:** [termios/go.mod1-6](https://github.com/charmbracelet/x/blob/7642919e/termios/go.mod#L1-L6)

### Module Dependencies

The termios module declares a direct dependency on `golang.org/x/sys v0.40.0`, which provides low-level Unix system call bindings.

| Dependency | Version | Purpose |
| --- | --- | --- |
| golang.org/x/sys | v0.40.0 | POSIX terminal control syscalls |

**Sources:** [termios/go.mod5](https://github.com/charmbracelet/x/blob/7642919e/termios/go.mod#L5-L5)

### Terminal Attribute Operations

The termios package wraps standard POSIX terminal control operations:

**Common terminal flags manipulated:**

| Flag Category | Examples | Purpose |
| --- | --- | --- |
| Input flags (iflag) | ICRNL, IXON | Input processing (CR/LF mapping, flow control) |
| Output flags (oflag) | OPOST, ONLCR | Output processing (newline translation) |
| Control flags (cflag) | CSIZE, PARENB | Hardware control (char size, parity) |
| Local flags (lflag) | ECHO, ICANON, ISIG | Terminal behavior (echo, canonical mode, signals) |

**Sources:** Based on standard POSIX termios operations referenced in system diagrams

### Integration with Higher-Level Packages

The termios package is used by other packages in the platform abstraction layer:

**Key usage patterns:**

1.  **Raw mode initialization**: Before reading input events, applications call `term.MakeRaw()` which uses termios to disable canonical mode and echo
2.  **State restoration**: On application exit, `term.Restore()` uses termios to return the terminal to its original state
3.  **Window size queries**: The `term.GetSize()` function uses termios `ioctl` operations to query terminal dimensions

**Sources:** Based on integration patterns shown in system architecture diagrams

Wcwidth Package
---------------

### Status and Purpose

The `wcwidth` package calculates the display width of Unicode characters and strings in terminal contexts. **This package is deprecated** and now serves as a thin wrapper around `github.com/mattn/go-runewidth`.

**Sources:** [wcwidth/wcwidth.go10-22](https://github.com/charmbracelet/x/blob/7642919e/wcwidth/wcwidth.go#L10-L22)

### Module Dependencies

| Dependency | Version | Purpose |
| --- | --- | --- |
| github.com/mattn/go-runewidth | v0.0.19 | Unicode width calculation tables |
| github.com/clipperhouse/uax29/v2 | v2.2.0 | Unicode segmentation (indirect) |

The indirect dependency on `uax29` is pulled in by `go-runewidth` for proper Unicode grapheme cluster segmentation.

**Sources:** [wcwidth/go.mod1-8](https://github.com/charmbracelet/x/blob/7642919e/wcwidth/go.mod#L1-L8)
 [wcwidth/go.sum1-4](https://github.com/charmbracelet/x/blob/7642919e/wcwidth/go.sum#L1-L4)

### API Surface

The wcwidth package exposes two deprecated functions:

#### RuneWidth Function

Returns the fixed-width display width of a single rune:

*   Returns `0` for combining characters and zero-width characters
*   Returns `1` for standard Latin characters
*   Returns `2` for East Asian Wide and Fullwidth characters

#### StringWidth Function

Returns the total display width of a string by summing the widths of all runes.

**Sources:** [wcwidth/wcwidth.go8-22](https://github.com/charmbracelet/x/blob/7642919e/wcwidth/wcwidth.go#L8-L22)

### Character Width Categories

The underlying `go-runewidth` library categorizes characters according to Unicode East Asian Width properties:

| Width | Return Value | Examples |
| --- | --- | --- |
| Zero-width | 0   | Combining marks, zero-width joiner |
| Narrow | 1   | ASCII, Latin-1, most Unicode |
| Wide | 2   | CJK ideographs, Hangul syllables, Katakana |
| Ambiguous | 1 or 2 | Greek, Cyrillic (context-dependent) |

**Note:** The ansi package provides more sophisticated width calculation via `GraphemeWidth()` which handles ANSI escape sequences. See [Parser and Text Manipulation](https://deepwiki.com/charmbracelet/x/4.1-parser-and-text-manipulation)
 for details.

**Sources:** Based on Unicode East Asian Width standard

### Migration Path

For new code, use `go-runewidth` directly instead of this deprecated wrapper:

| Deprecated | Replacement |
| --- | --- |
| `wcwidth.RuneWidth(r)` | `runewidth.RuneWidth(r)` |
| `wcwidth.StringWidth(s)` | `runewidth.StringWidth(s)` |

The wcwidth package exists for backward compatibility with existing code that depends on the old import path.

**Sources:** [wcwidth/wcwidth.go10-22](https://github.com/charmbracelet/x/blob/7642919e/wcwidth/wcwidth.go#L10-L22)

Build Tag Strategy
------------------

Both packages use build tags to ensure they only compile on appropriate platforms:

The `termios` package is Unix-specific and excluded from Windows builds. Windows applications use the [Windows ConPTY](https://deepwiki.com/charmbracelet/x/5.2-windows-conpty)
 package instead for terminal control.

Package Relationship Summary
----------------------------

| Package | Platform | Purpose | Status | Key Dependency |
| --- | --- | --- | --- | --- |
| termios | Unix/Linux | Terminal attribute control | Active | golang.org/x/sys |
| wcwidth | All | Character width calculation | Deprecated | go-runewidth |

Both packages are foundational components in the platform abstraction layer, with termios enabling low-level terminal control on Unix systems and wcwidth supporting correct text layout across all platforms.
