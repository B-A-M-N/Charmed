The `exp/` directory contains experimental packages with **no guarantees of API stability or backwards compatibility**. These packages are under active development and may undergo breaking changes as they mature. Once stable, they may be promoted to top-level packages or moved to separate repositories.

For information about stable utility packages like SSH keys and color constants, see [SSH Keys and Colors](https://deepwiki.com/charmbracelet/x/6.1-ssh-keys-and-colors)
.

Overview
--------

The experimental packages cover diverse functionality areas including testing utilities, color management, generic data structure operations, and system integration helpers. Each package is independently versioned as a separate Go module.

### Experimental Package Catalog

| Package | Purpose | Key Use Case |
| --- | --- | --- |
| `exp/teatest` | Testing framework for Bubble Tea programs | Automated TUI testing with golden files |
| `exp/golden` | Golden file comparison testing | Snapshot testing for text output |
| `exp/charmtone` | CharmTone color palette utilities | Brand-consistent color management |
| `exp/toner` | Color toning and adjustment utilities | Dynamic color manipulation |
| `exp/slice` | Generic slice manipulation functions | Functional programming patterns |
| `exp/maps` | Generic map utility functions | Map operations and transformations |
| `exp/strings` | String manipulation utilities | Text processing helpers |
| `exp/higherorder` | Generic higher-order functions | Functional programming primitives |
| `exp/ordered` | Generic min/max/clamp functions | Value constraint operations |
| `exp/open` | Cross-platform file/URL opener | System integration for opening resources |

Testing Framework (teatest)
---------------------------

The `teatest` package provides a comprehensive testing harness for Bubble Tea programs, enabling automated testing of interactive terminal applications without requiring a real terminal.

### TestModel Architecture

Sources: [exp/teatest/teatest.go105-163](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L105-L163)
 [exp/teatest/teatest.go19-22](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L19-L22)
 [exp/teatest/teatest.go241-280](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L241-L280)

### Key Testing Patterns

The `teatest` package implements several testing patterns:

#### TestModel Creation

The `NewTestModel` function [exp/teatest/teatest.go120-163](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L120-L163)
 creates a test harness that:

*   Wraps a `tea.Program` with controlled I/O
*   Captures output in a thread-safe buffer using `safeReadWriter` [exp/teatest/teatest.go283-305](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L283-L305)
    
*   Provides input injection via `bytes.Buffer`
*   Handles interruption signals for test cleanup

#### Waiting and Polling

The `WaitFor` function [exp/teatest/teatest.go69-103](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L69-L103)
 implements condition-based polling:

*   Default timeout: 1 second
*   Default check interval: 50ms
*   Configurable via `WithDuration` and `WithCheckInterval` options
*   Returns error with last output on timeout

#### Golden File Testing

Integration with `exp/golden` via `RequireEqualOutput` [exp/teatest/teatest.go278-281](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L278-L281)
 enables snapshot testing for terminal output, comparing rendered output against golden files stored in `testdata/` directories.

Sources: [exp/teatest/teatest.go69-103](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L69-L103)
 [exp/teatest/teatest.go120-163](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L120-L163)
 [exp/teatest/teatest.go278-281](https://github.com/charmbracelet/x/blob/7642919e/exp/teatest/teatest.go#L278-L281)

### Testing Options

| Option | Purpose | Default Value |
| --- | --- | --- |
| `WithInitialTermSize(x, y)` | Set initial terminal dimensions | None (must be set explicitly) |
| `WithDuration(d)` | WaitFor timeout duration | 1 second |
| `WithCheckInterval(d)` | WaitFor polling interval | 50 milliseconds |
| `WithFinalTimeout(d)` | Timeout for final model/output | No timeout |
| `WithTimeoutFn(fn)` | Custom timeout handler | Fatal error |

Slice Utilities
---------------

The `slice` package provides a comprehensive set of generic utility functions for slice manipulation, implementing common functional programming patterns and data structure operations.

### Slice Operations Architecture

Sources: [exp/slice/slice.go8-120](https://github.com/charmbracelet/x/blob/7642919e/exp/slice/slice.go#L8-L120)

### Function Categories

#### Transformation Functions

*   `GroupBy[T, K]` - Groups slice elements by a key function, returning a map of groups
*   `Uniq[T]` - Returns a new slice with duplicate elements removed
*   `Intersperse[T]` - Inserts an element between each pair of existing elements

#### Access Functions

*   `Take[T]` - Returns the first n elements, handling edge cases gracefully
*   `Last[T]` - Returns the last element with existence checking
*   `ContainsAny[T]` - Checks if any of the provided values exist in the slice

#### Mutation Functions

*   `Shift[T]` - Removes and returns the first element (queue-like operation)
*   `Pop[T]` - Removes and returns the last element (stack-like operation)
*   `DeleteAt[T]` - Removes and returns the element at a specific index
