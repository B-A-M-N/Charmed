Damping Types
=============

Relevant source files

*   [README.md](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1)
    
*   [spring.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go)
    

This document explains the different damping behaviors available in the Harmonica spring animation system and their effects on animation. It covers the characteristics and implementation details of the three damping types: under-damped, critically-damped, and over-damped springs. For information about implementing spring animations in your applications, see [Spring Implementation](https://deepwiki.com/charmbracelet/harmonica/3.1-spring-implementation)
.

Overview of Damping in Spring Animation
---------------------------------------

In the Harmonica library, the `dampingRatio` parameter passed to the `NewSpring` function determines how a spring-based animation approaches its target value. This parameter controls the rate at which oscillations diminish and significantly affects the feel of the resulting animation.

Sources: [spring.go132-146](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L146)
 [spring.go216-224](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L216-L224)

The Three Damping Types
-----------------------

The damping ratio determines which of three distinct behaviors the spring animation will exhibit:

Sources: [spring.go118-132](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L118-L132)
 [README.md71-91](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L71-L91)

### Under-Damped Springs (damping ratio < 1)

Under-damped springs exhibit oscillation around the target value. The object will overshoot the target and continue to oscillate with progressively smaller amplitudes until eventually settling at the target position.

This behavior is implemented in [spring.go173-195](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L173-L195)
 where specific coefficients are calculated for under-damped cases. The smaller the damping ratio, the more pronounced and persistent the oscillations will be.

Typical uses include:

*   Bouncy UI elements
*   Playful animations
*   Elastic or springy objects simulation

Sources: [spring.go173-195](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L173-L195)
 [README.md76-81](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L76-L81)

### Critically-Damped Springs (damping ratio = 1)

Critically-damped springs approach the target position as quickly as possible without overshooting or oscillating. This creates a smooth, direct motion.

This behavior is implemented in [spring.go196-209](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L196-L209)
 where coefficients for the critically-damped case are calculated. It represents the precise threshold between under-damping and over-damping.

Typical uses include:

*   UI elements requiring precision
*   Camera movements
*   Natural-feeling transitions

Sources: [spring.go196-209](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L196-L209)
 [README.md82-86](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L82-L86)

### Over-Damped Springs (damping ratio > 1)

Over-damped springs approach the target position without any oscillation, but more slowly than critically-damped springs. This creates a gradual, deliberate motion that can convey weight or resistance.

This behavior is implemented in [spring.go147-171](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L147-L171)
 where coefficients for the over-damped case are calculated. The higher the damping ratio, the slower the approach to the target.

Typical uses include:

*   Heavy object simulations
*   Deliberate, weighty animations
*   Gradual transitions

Sources: [spring.go147-171](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L147-L171)
 [README.md87-91](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L87-L91)

Damping Ratio Selection Guide
-----------------------------

The table below provides guidance on selecting appropriate damping ratio values for different animation needs:

| Animation Need | Damping Type | Recommended Value | Effect |
| --- | --- | --- | --- |
| Bouncy, playful motion | Under-damped | 0.2 - 0.4 | Multiple visible oscillations |
| Slightly elastic motion | Under-damped | 0.5 - 0.8 | Slight overshoot with few oscillations |
| Quick, precise motion | Critically-damped | 1.0 | Direct approach without oscillation |
| Slightly heavy feel | Over-damped | 1.1 - 1.5 | Slower approach |
| Very heavy or resistant feel | Over-damped | 2.0 - 5.0 | Very gradual approach |

The angular frequency parameter (typically between 2.0 and 10.0) affects the overall speed of the animation and should be considered alongside the damping ratio when fine-tuning animations.

Mathematical Foundation
-----------------------

The implementation in Harmonica is based on the equations for a damped harmonic oscillator:

*   Under-damped case corresponds to a complex solution with sinusoidal terms [spring.go173-195](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L173-L195)
    
*   Critically-damped case corresponds to a solution with repeated roots [spring.go196-209](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L196-L209)
    
*   Over-damped case corresponds to a solution with distinct real roots [spring.go147-171](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L147-L171)
    

By pre-computing the coefficients for each case, the library provides an efficient implementation that can update position and velocity values rapidly during animation.
