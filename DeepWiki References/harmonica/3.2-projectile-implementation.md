Projectile Implementation
=========================

Relevant source files

*   [projectile.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go)
    
*   [projectile\_test.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile_test.go)
    

This document provides a step-by-step guide on implementing and using the projectile motion system in the Harmonica library. It covers initialization, updating, and integration into application loops. For information about the theoretical concepts behind projectile motion, see [Projectile Motion](https://deepwiki.com/charmbracelet/harmonica/2.3-projectile-motion)
. For details on implementing spring animations, see [Spring Implementation](https://deepwiki.com/charmbracelet/harmonica/3.1-spring-implementation)
.

Overview
--------

The Projectile system in Harmonica simulates objects moving under constant acceleration, such as items thrown or falling under gravity. This implementation supports both 2D and 3D motion with configurable initial position, velocity, and acceleration.

Sources: [projectile.go73-85](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L73-L85)

Core Components
---------------

The projectile implementation relies on three main structures:

Sources: [projectile.go23-44](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L23-L44)

### Point

`Point` represents a position in 3D space with X, Y, and Z coordinates. This is used to track the projectile's current location.

### Vector

`Vector` represents both direction and magnitude, used for:

*   Velocity: how fast the projectile is moving and in what direction
*   Acceleration: how velocity changes over time (e.g., gravity)

### Projectile

The main struct that encapsulates the projectile's state and behavior:

*   Position (`pos`): current location in space
*   Velocity (`vel`): current movement speed and direction
*   Acceleration (`acc`): forces acting on the projectile
*   Time step (`deltaTime`): how much time passes between updates

Implementation Steps
--------------------

### Step 1: Initialization

Create a new projectile by calling `NewProjectile` with appropriate parameters:

Sources: [projectile.go61-71](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L61-L71)

The parameters are:

*   `deltaTime`: The time step between updates (typically calculated using `FPS()`)
*   `initialPosition`: Starting position of the projectile
*   `initialVelocity`: Starting velocity vector
*   `initialAcceleration`: Constant acceleration vector (often gravity)

### Step 2: Integration into Update Loop

In your application's update loop, call the `Update()` method on each frame to calculate the new position:

Sources: [projectile.go73-85](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L73-L85)

### Step 3: Retrieving State Information

You can access the projectile's current state using these methods:

*   `Position()`: Get the current position
*   `Velocity()`: Get the current velocity
*   `Acceleration()`: Get the current acceleration

Physics Update Calculation
--------------------------

The projectile's position and velocity are updated on each frame according to these equations:

Sources: [projectile.go73-85](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L73-L85)

In the implementation, this is calculated separately for each coordinate (X, Y, Z):

    Position.X += Velocity.X * deltaTime
    Position.Y += Velocity.Y * deltaTime
    Position.Z += Velocity.Z * deltaTime
    
    Velocity.X += Acceleration.X * deltaTime
    Velocity.Y += Acceleration.Y * deltaTime
    Velocity.Z += Acceleration.Z * deltaTime
    

Sources: [projectile.go75-83](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L75-L83)

Performance Considerations
--------------------------

*   The `Update()` method is lightweight and efficient, making it suitable for updating many projectiles per frame
*   For very large numbers of projectiles, consider batch processing or parallel updates
*   The simulation uses a simple Euler integration method, which is fast but may accumulate errors over long simulations

Summary
-------

Implementing projectile motion with Harmonica involves:

1.  Creating a projectile with initial conditions using `NewProjectile()`
2.  Calling `Update()` on each frame to calculate new positions
3.  Using the returned position to render your object
4.  Optionally retrieving additional state information with `Position()`, `Velocity()`, and `Acceleration()`

This implementation provides a simple yet effective way to add realistic projectile motion to your applications.
