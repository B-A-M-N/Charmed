Spring Animation
================

Relevant source files

*   [README.md](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1)
    
*   [spring.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go)
    

Spring Animation in Harmonica provides a simplified damped harmonic oscillator system that creates smooth, natural motion for animated elements. This page covers the implementation, physics, and practical usage of the Spring system in the Harmonica library. For information about different damping behaviors, see [Damping Types](https://deepwiki.com/charmbracelet/harmonica/2.2-damping-types)
. For step-by-step implementation guidance, see [Spring Implementation](https://deepwiki.com/charmbracelet/harmonica/3.1-spring-implementation)
.

Core Concept
------------

Spring animation simulates the physical behavior of a damped spring, allowing elements to move toward a target position with natural-looking acceleration and deceleration. The implementation in Harmonica is based on a simplified damped harmonic oscillator model, offering precise control over motion characteristics.

Sources: [spring.go1-22](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L1-L22)

Spring Structure and Parameters
-------------------------------

The `Spring` struct stores precomputed coefficients that define how the spring behaves when updated. This approach allows efficient updates across multiple animation objects using the same spring characteristics.

Sources: [spring.go104-107](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L104-L107)
 [spring.go132-212](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L212)

The Spring struct uses four coefficient fields to encapsulate the mathematical model:

| Coefficient | Purpose |
| --- | --- |
| `posPosCoef` | How current position affects new position |
| `posVelCoef` | How current velocity affects new position |
| `velPosCoef` | How current position affects new velocity |
| `velVelCoef` | How current velocity affects new velocity |

Sources: [spring.go104-107](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L104-L107)

Creating Springs
----------------

Springs are created using the `NewSpring()` function which takes three key parameters:

Sources: [spring.go132-212](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L212)

### Key Parameters

1.  **deltaTime**: The time step between animation updates (in seconds)
    
    *   Can be calculated using the `FPS()` utility function
    *   Example: `FPS(60)` for 60 frames per second
2.  **angularFrequency**: Controls the speed of the animation
    
    *   Higher values = faster motion
    *   Typical range: 1.0 to 10.0
    *   Must be non-negative
3.  **dampingRatio**: Controls the springiness of the animation
    
    *   Value < 1.0: Under-damped (oscillates)
    *   Value = 1.0: Critically-damped (no oscillation, fastest approach)
    *   Value > 1.0: Over-damped (no oscillation, slower approach)
    *   Must be non-negative

Sources: [spring.go67-69](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L67-L69)
 [spring.go132-145](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L145)

Spring Coefficient Calculation
------------------------------

The `NewSpring()` function calculates coefficients differently based on the damping ratio category:

Sources: [spring.go147-212](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L147-L212)

### Mathematical Implementation

The coefficient calculations involve complex mathematical formulas based on the specific damping category:

*   **Over-damped**: Uses real exponential functions ([spring.go147-172](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L147-L172)
    )
*   **Under-damped**: Uses trigonometric functions with exponential decay ([spring.go173-195](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L173-L195)
    )
*   **Critically-damped**: Uses simplified exponential calculations ([spring.go196-209](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L196-L209)
    )

These mathematical implementations derive from the analytical solution to the damped harmonic oscillator differential equation.

Updating Spring Values
----------------------

The `Update()` method calculates new position and velocity values based on current state and the target position:

Sources: [spring.go214-224](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L214-L224)

The core calculation looks like:

    newPos = (pos-equilibriumPos)*posPosCoef + vel*posVelCoef + equilibriumPos
    newVel = (pos-equilibriumPos)*velPosCoef + vel*velVelCoef
    

This calculates motion in a space relative to the equilibrium (target) position and then transforms it back to the absolute coordinate system.

Sources: [spring.go217-222](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L217-L222)

Integration with Animation Loops
--------------------------------

Spring animations are typically integrated into an application's animation loop:

Sources: [spring.go8-18](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L8-L18)
 [README.md22-52](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L22-L52)

### Typical Usage Pattern

    1. Initialize Spring once with desired parameters
    2. In animation loop:
       a. Call spring.Update() for each animated value
       b. Render the updated values
       c. Wait for next frame
    

The Spring system is designed to be framework-agnostic, working in any context with an animation loop, including 2D, 3D, and terminal-based interfaces.

Sources: [README.md24-52](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L24-L52)

Performance Considerations
--------------------------

The Spring implementation is optimized for performance through several mechanisms:

1.  **Coefficient Precomputation**: Expensive mathematical calculations are performed once during initialization rather than on each update
2.  **Minimal Memory Usage**: The Spring struct only stores four essential coefficient values
3.  **Efficient Update Calculations**: The `Update()` method uses simple multiplication and addition operations
4.  **Special Case Handling**: Special cases (like zero angular frequency) are handled efficiently

Sources: [spring.go104-107](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L104-L107)
 [spring.go138-145](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L138-L145)

Historical Context
------------------

The implementation is a port of Ryan Juckett's damped simple harmonic oscillator, originally written in C++ in 2008. The mathematical model and approach are directly derived from this work, which provides a highly efficient spring animation system.

Sources: [spring.go3-50](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L3-L50)
 [README.md93-100](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L93-L100)
