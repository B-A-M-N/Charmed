Projectile Motion
=================

Relevant source files

*   [harmonica.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/harmonica.go)
    
*   [projectile.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go)
    
*   [projectile\_test.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile_test.go)
    

Purpose and Scope
-----------------

This document details the Projectile Motion system in the Harmonica library, which simulates objects moving under constant acceleration in 2D or 3D space. This system is ideal for particle effects, ballistic simulations, and physics-based animations where objects follow predictable trajectories under forces like gravity.

For information about spring-based animations, see [Spring Animation](https://deepwiki.com/charmbracelet/harmonica/2.1-spring-animation)
.

Sources: [harmonica.go1-4](https://github.com/charmbracelet/harmonica/blob/6f6967c9/harmonica.go#L1-L4)
 [projectile.go3-21](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L3-L21)

Core Components
---------------

The projectile motion system consists of three primary structures that work together to simulate physical motion:

**Component Definitions:**

1.  **Point**: Represents a 3D position in space with X, Y, and Z coordinates.
2.  **Vector**: Represents direction and magnitude in 3D space, used for velocity and acceleration.
3.  **Projectile**: Main simulation object that combines position, velocity, and acceleration with update logic.

Sources: [projectile.go25-44](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L25-L44)

Coordinate System
-----------------

Harmonica supports both standard and terminal-based coordinate systems:

The library provides two predefined gravity vectors to accommodate these systems:

*   `Gravity`: Standard gravity vector `(0, -9.81, 0)` for traditional coordinate systems
*   `TerminalGravity`: Inverted gravity vector `(0, 9.81, 0)` for terminal applications where Y increases downward

Sources: [projectile.go46-59](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L46-L59)

Physics Simulation
------------------

The projectile motion system implements basic Newtonian physics with a fixed time step approach:

**Key Equations:**

*   Position update: `position += velocity × deltaTime`
*   Velocity update: `velocity += acceleration × deltaTime`

This fixed time step approach ensures consistent behavior regardless of frame rate variations. The simulation accuracy depends on the `deltaTime` value provided during initialization.

Sources: [projectile.go73-85](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L73-L85)

Creating and Using Projectiles
------------------------------

### Initialization

Use the `NewProjectile` factory function to create a projectile instance:

**Parameters:**

*   `deltaTime`: Time step for simulation (typically 1/FPS)
*   `initialPosition`: Starting Point (X, Y, Z)
*   `initialVelocity`: Starting Vector for velocity
*   `acceleration`: Constant Vector acting on the projectile (often gravity)

Sources: [projectile.go61-71](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L61-L71)
 [harmonica.go19-26](https://github.com/charmbracelet/harmonica/blob/6f6967c9/harmonica.go#L19-L26)

### Update Loop

After initialization, call `Update()` on each frame to advance the simulation:

**Accessor Methods:**

*   `Position()`: Returns the current Point
*   `Velocity()`: Returns the current velocity Vector
*   `Acceleration()`: Returns the current acceleration Vector

Sources: [projectile.go87-100](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L87-L100)
 [harmonica.go28-31](https://github.com/charmbracelet/harmonica/blob/6f6967c9/harmonica.go#L28-L31)

Trajectory Visualization
------------------------

Projectile motion with constant acceleration produces predictable paths:

For objects under the influence of gravity:

*   With no horizontal acceleration, horizontal velocity remains constant
*   Vertical velocity changes at a constant rate based on gravitational acceleration
*   Results in the classic parabolic trajectory

Sources: [projectile\_test.go71-104](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile_test.go#L71-L104)

Common Scenarios and Behaviors
------------------------------

The following table illustrates common projectile motion scenarios:

| Scenario | Initial Velocity | Acceleration | Resulting Trajectory |
| --- | --- | --- | --- |
| Free fall | (0, 0, 0) | (0, -9.81, 0) | Straight down with increasing speed |
| Horizontal throw | (5, 0, 0) | (0, -9.81, 0) | Parabola with constant horizontal speed |
| Angled throw | (5, 5, 0) | (0, -9.81, 0) | Parabola with peak height determined by initial Y velocity |
| Terminal UI particle | (5, 5, 0) | (0, 9.81, 0) | Inverted parabola (using TerminalGravity) |

Sources: [projectile\_test.go38-104](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile_test.go#L38-L104)

Performance Considerations
--------------------------

The projectile system is designed for simplicity and performance:

*   Fixed calculations per update regardless of projectile count
*   No dependencies between projectiles, enabling parallelization
*   Minimal memory usage with just position, velocity, and acceleration vectors

Sources: [projectile.go73-85](https://github.com/charmbracelet/harmonica/blob/6f6967c9/projectile.go#L73-L85)
