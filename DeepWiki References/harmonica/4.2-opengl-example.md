OpenGL Example
==============

Relevant source files

*   [examples/spring/opengl/JetBrainsMono-Regular.ttf](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/JetBrainsMono-Regular.ttf)
    
*   [examples/spring/opengl/main.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go)
    

This document describes the OpenGL example application included with the Harmonica library. The example demonstrates how to use Harmonica's spring animation system in a graphical context, providing a visual representation of spring physics and an interactive way to experiment with different spring parameters.

For information about other example applications, see [Example Applications](https://deepwiki.com/charmbracelet/harmonica/4-example-applications)
, [Terminal UI Example](https://deepwiki.com/charmbracelet/harmonica/4.1-terminal-ui-example)
, and [Particle Example](https://deepwiki.com/charmbracelet/harmonica/4.3-particle-example)
.

Overview
--------

The OpenGL example is a graphical application that visualizes spring physics by allowing users to create and manipulate circles (sprites) that move using spring-based animation. The application uses the Pixel library with an OpenGL backend for rendering and demonstrates real-time parameter adjustments.

Sources: [examples/spring/opengl/main.go73-93](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L73-L93)

Application Structure
---------------------

The OpenGL example is structured around a game loop pattern with distinct update and render phases. It uses Harmonica's spring physics to animate visual elements.

Sources: [examples/spring/opengl/main.go32-43](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L32-L43)
 [examples/spring/opengl/main.go209-226](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L209-L226)

Spring Integration
------------------

The example demonstrates how to integrate Harmonica's spring system into a graphics application. Spring physics are applied to both the position and size of visual elements.

Sources: [examples/spring/opengl/main.go245-261](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L245-L261)
 [examples/spring/opengl/main.go296-299](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L296-L299)

### Spring Initialization

The spring is configured with three key parameters:

1.  **Delta Time**: The time step for simulation
2.  **Angular Frequency**: Controls the speed of oscillation (referred to as "frequency" in the UI)
3.  **Damping Ratio**: Controls how quickly oscillations diminish

Sources: [examples/spring/opengl/main.go296-299](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L296-L299)

### Applying Spring Motion

The example applies spring physics to three properties of each sprite:

1.  X-position
2.  Y-position
3.  Radius (size)

Each property is animated independently using the same spring instance:

Sources: [examples/spring/opengl/main.go258-260](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L258-L260)

Interactive Controls
--------------------

The example provides interactive controls to experiment with spring parameters in real-time:

| Control | Action |
| --- | --- |
| Left/Right Arrows | Decrease/Increase frequency |
| Up/Down Arrows | Increase/Decrease damping |
| Shift + Arrows | Apply larger adjustments |
| Space | Toggle help text |
| Mouse Click | Create or move sprite |

These controls allow users to observe how different parameter combinations affect the spring animation behavior.

Sources: [examples/spring/opengl/main.go95-162](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L95-L162)

### Special Animation Effects

The example includes special animation effects for showing and hiding sprites:

1.  **Show Animation**: Appears at mouse position and grows to a random size using spring animation
2.  **Hide Animation**: Shrinks to nothing using a faster spring configuration

Sources: [examples/spring/opengl/main.go305-320](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L305-L320)

Usage Patterns
--------------

The OpenGL example demonstrates several important patterns for integrating Harmonica into a graphical application:

1.  **Recalculation on Parameter Change**: When spring parameters change, the spring coefficients are recalculated
2.  **Multiple Properties per Spring**: Using the same spring to animate multiple properties
3.  **State Management**: Tracking animation state to determine when animations are complete
4.  **Timeout Handling**: Automatically hiding sprites after a period of inactivity

These patterns can be applied to other graphical applications that need natural-looking, physics-based animations.

Sources: [examples/spring/opengl/main.go133-139](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L133-L139)
 [examples/spring/opengl/main.go251-255](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/opengl/main.go#L251-L255)

System Requirements
-------------------

The example requires:

1.  OpenGL support
2.  The font file `JetBrainsMono-Regular.ttf` in the working directory
3.  Dependencies:
