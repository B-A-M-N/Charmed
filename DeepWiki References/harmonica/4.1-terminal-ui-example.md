Terminal UI Example
===================

Relevant source files

*   [examples/spring/tui/main.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/tui/main.go)
    

Purpose and Scope
-----------------

This document explains the Terminal UI Example included with Harmonica, which demonstrates how to integrate the library's spring animation system with a terminal-based user interface using the Bubble Tea framework. This example showcases how to create smooth, physics-based animations in a text environment without requiring a graphical display.

For information about implementing spring animations in general, see [Spring Implementation](https://deepwiki.com/charmbracelet/harmonica/3.1-spring-implementation)
. For graphical examples, see [OpenGL Example](https://deepwiki.com/charmbracelet/harmonica/4.2-opengl-example)
.

Overview
--------

The Terminal UI Example creates a simple animation of a text-based sprite that moves horizontally across the terminal using spring physics. The sprite starts at the left edge of the screen and smoothly animates to a target position, demonstrating the natural, organic movement that spring animation provides.

Sources: [examples/spring/tui/main.go1-111](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/tui/main.go#L1-L111)

Architecture
------------

The example integrates three main components:

1.  **Harmonica's Spring System**: Provides the physics-based animation calculations
2.  **Bubble Tea Framework**: Manages the terminal UI application lifecycle and event handling
3.  **Lip Gloss**: Handles terminal styling and formatting

Sources: [examples/spring/tui/main.go49-100](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/tui/main.go#L49-L100)

Key Parameters
--------------

The example is configured with specific parameters that control the animation behavior:

| Parameter | Value | Description |
| --- | --- | --- |
| `fps` | 60  | Frames per second for the animation |
| `frequency` | 7.0 | Angular frequency of the spring (controls speed) |
| `damping` | 0.15 | Damping ratio of the spring (controls bounciness) |
| `spriteWidth` | 12  | Width of the sprite in characters |
| `spriteHeight` | 5   | Height of the sprite in rows |

These parameters create an underdamped spring (damping < 1.0), which produces a slightly bouncy animation as the sprite approaches its target position.

Sources: [examples/spring/tui/main.go15-21](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/spring/tui/main.go#L15-L21)

Running the Example
-------------------

To run this example, navigate to the `examples/spring/tui` directory and execute:

    go run main.go
    

The terminal will display a sprite (represented by forward slashes) that animates from the left side of the terminal to a target position using spring physics.

Summary
-------

The Terminal UI Example demonstrates:

1.  How to integrate Harmonica's spring system with a terminal UI framework
2.  Creating frame-based animations in a terminal environment
3.  Converting physics-based calculations to visual representations
4.  Setting appropriate spring parameters for natural-looking motion

This example serves as a practical starting point for implementing physics-based animations in terminal applications.
