Spring Implementation
=====================

Relevant source files

*   [README.md](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1)

This guide provides a step-by-step approach to implementing spring animations in your applications using the Harmonica library. We focus specifically on practical implementation of the Spring system for creating smooth, natural motion. For conceptual details about springs and damping types, see [Spring Animation](https://deepwiki.com/charmbracelet/harmonica/2.1-spring-animation)
 and [Damping Types](https://deepwiki.com/charmbracelet/harmonica/2.2-damping-types)

Understanding the Spring System
-------------------------------

The Spring system in Harmonica is designed to simulate damped harmonic motion - the same physics that governs real-world springs. It provides a computationally efficient way to animate values with natural-looking motion.

Sources: [spring.go214-224](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L214-L224)

### Spring Parameters

Springs in Harmonica are configured with three key parameters:

| Parameter | Description | Effect on Animation |
| --- | --- | --- |
| `deltaTime` | Time step between updates (seconds) | Controls how time advances, typically based on framerate |
| `angularFrequency` | Angular frequency of motion | Higher values = faster motion |
| `dampingRatio` | Amount of damping | < 1: oscillation, = 1: fastest approach without oscillation, > 1: slower approach |

Sources: [spring.go132-212](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L212)
 [README.md57-69](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L57-L69)

Basic Implementation Steps
--------------------------

### 1\. Initialize a Spring

The first step is to create a Spring instance with appropriate parameters:

Sources: [spring.go132-212](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L212)
 [README.md24-26](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L24-L26)

To create a Spring, use the `NewSpring` function with your desired parameters:

    spring := harmonica.NewSpring(harmonica.FPS(60), 6.0, 0.5)
    

*   The first parameter is the time delta - in this example using the FPS helper to convert from frames-per-second
*   The second parameter (6.0) is the angular frequency - higher values create faster animations
*   The third parameter (0.5) is the damping ratio - this example uses under-damping for some springiness

Detailed Update Process
-----------------------

Each time `Update` is called, the Spring applies its physics model to calculate new position and velocity values:

Sources: [spring.go214-224](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L214-L224)

The update process uses precomputed coefficients to efficiently calculate the next state:

1.  The current position is transformed to be relative to the target position
2.  The spring coefficients (computed during initialization) are applied
3.  The new position and velocity are returned

Performance Considerations
--------------------------

*   Springs are computationally efficient - the heavy math is done once during initialization
*   The `Update` method is very fast, making it suitable for animating many objects
*   Springs with the same parameters can share a single Spring instance
*   Make sure the deltaTime matches your actual frame rate for consistent animation

Sources: [spring.go104-224](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L104-L224)

Troubleshooting
---------------

| Issue | Possible Cause | Solution |
| --- | --- | --- |
| Animation too fast/slow | Incorrect angular frequency | Adjust the angular frequency parameter |
| Animation too bouncy | Damping ratio too low | Increase the damping ratio closer to 1.0 |
| Animation too stiff | Damping ratio too high | Decrease the damping ratio |
| Animation stutters | Inconsistent frame rate | Ensure deltaTime matches actual frame rate |

Sources: [README.md57-69](https://github.com/charmbracelet/harmonica/blob/6f6967c9/README.md?plain=1#L57-L69)
 [spring.go132-212](https://github.com/charmbracelet/harmonica/blob/6f6967c9/spring.go#L132-L212)

Integration with Different Frameworks
-------------------------------------

The Spring system is framework-agnostic and can be used with:

*   Game engines
*   UI frameworks
*   Terminal applications
*   WebGL/OpenGL
*   Custom rendering systems

The same core pattern applies across all frameworks: initialize once, then call Update on each frame.
