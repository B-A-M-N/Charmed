Particle Example
================

Relevant source files

*   [examples/particle/main.go](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go)
    

Purpose
-------

The Particle Example demonstrates the practical application of Harmonica's Projectile Motion system in a terminal-based application. This example showcases how to initialize, animate, and visualize a projectile moving under constant acceleration in a terminal interface. It provides a concrete implementation of the projectile motion concepts detailed in [Projectile Motion](https://deepwiki.com/charmbracelet/harmonica/2.3-projectile-motion)
.

Implementation Overview
-----------------------

The example is built as a terminal application using the Bubble Tea framework, which provides an event-driven architecture for terminal user interfaces. The example simulates a particle moving in a 2D space with physics-based motion.

Sources: [examples/particle/main.go1-88](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L1-L88)

Key Components
--------------

### Model Structure

The central component of the example is the `model` struct, which holds the projectile and its current position:

The `model` struct:

*   Contains a pointer to a `harmonica.Projectile`, which handles the physics calculations
*   Tracks the current position as a `harmonica.Point`
*   Implements the Bubble Tea interface methods for initialization, updates, and rendering

Sources: [examples/particle/main.go33-36](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L33-L36)

### Animation Commands

Two key functions drive the animation loop:

1.  `animate()` - Creates a ticker that generates frame update messages at the specified FPS (60)
2.  `wait()` - Creates a command that delays execution for a specified duration

Sources: [examples/particle/main.go20-31](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L20-L31)

Animation Flow
--------------

The example follows a specific sequence to animate the projectile:

The animation continues until either:

*   The projectile's Y-coordinate exceeds `maxHeight` (100)
*   The user presses any key to exit

Sources: [examples/particle/main.go38-59](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L38-L59)

Projectile Configuration
------------------------

The projectile is initialized in the `main()` function with specific parameters:

| Parameter | Value | Description |
| --- | --- | --- |
| Initial Position | `{X: 0, Y: 0}` | Starting at the origin |
| Initial Velocity | `{X: 5, Y: 0}` | Moving right with no vertical velocity |
| Acceleration | `TerminalGravity` | Constant downward acceleration (predefined constant) |
| Time Step | `FPS(60)` | 1/60th of a second per update |

These parameters create a projectile that follows a classic parabolic trajectory - initially moving horizontally but gradually falling due to gravity.

Sources: [examples/particle/main.go77-82](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L77-L82)

Terminal Visualization
----------------------

The example uses a simple but effective approach to visualize the projectile's position in the terminal:

This visualization technique:

1.  Uses newlines to represent vertical position
2.  Uses spaces to represent horizontal position
3.  Displays the exact coordinates as a string at that position

The result is that the coordinates appear at the correct relative position in the terminal, creating a visual representation of the projectile's path.

Sources: [examples/particle/main.go61-74](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L61-L74)

Complete Animation Cycle
------------------------

The full animation cycle can be represented as:

On each frame:

1.  The projectile's position is updated based on physics equations
2.  The new position is checked against termination conditions
3.  The position is rendered in the terminal
4.  The program waits for the next frame

Sources: [examples/particle/main.go42-59](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L42-L59)

Physics Simulation
------------------

Behind the scenes, the `harmonica.Projectile` type applies the standard equations of motion for constant acceleration:

1.  Position update: `position += velocity * deltaTime + 0.5 * acceleration * deltaTime²`
2.  Velocity update: `velocity += acceleration * deltaTime`

When the model calls `projectile.Update()` in each frame, these equations are applied to calculate the new position of the projectile.

Sources: [examples/particle/main.go49](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L49-L49)

Running the Example
-------------------

To run this example:

1.  Clone the Harmonica repository
2.  Navigate to the `examples/particle` directory
3.  Run `go run main.go`

The terminal will display a particle moving in a parabolic path, with its coordinates shown at each position. The animation continues until the particle reaches a Y position greater than 100, or you press any key to exit.

Sources: [examples/particle/main.go76-88](https://github.com/charmbracelet/harmonica/blob/6f6967c9/examples/particle/main.go#L76-L88)
