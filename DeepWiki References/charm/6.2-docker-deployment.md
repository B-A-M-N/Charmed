Official Docker Images
----------------------

Charm provides official Docker images that are ready to use for deployment. These images contain the Charm binary and are configured with the necessary defaults for running a Charm server.

Sources: [docker.md2-7](https://github.com/charmbracelet/charm/blob/aff3071d/docker.md?plain=1#L2-L7)

Container Architecture
----------------------

The Charm Docker container is designed to be minimal and secure, built on a scratch image with just the essential components needed to run the Charm server.

The Charm container exposes four main ports:

*   35353: SSH server for authentication and connections
*   35354: HTTP server for API access
*   35355: Stats server for metrics (when enabled)
*   35356: Health server for monitoring container health

Sources: [Dockerfile1-22](https://github.com/charmbracelet/charm/blob/aff3071d/Dockerfile#L1-L22)
 [docker.md8-23](https://github.com/charmbracelet/charm/blob/aff3071d/docker.md?plain=1#L8-L23)

Basic Deployment
----------------

### Using Docker Run

The simplest way to deploy Charm with Docker is using the `docker run` command:

This command:

1.  Names the container "charm"
2.  Mounts a local directory to the container's `/data` directory
3.  Maps the required ports to the host
4.  Sets a restart policy
5.  Uses the latest official image

Sources: [docker.md9-23](https://github.com/charmbracelet/charm/blob/aff3071d/docker.md?plain=1#L9-L23)

### Using Docker Compose

For easier management, you can use Docker Compose to deploy Charm:

Save this to a file named `docker-compose.yml` and run `docker-compose up -d` to start the Charm server in detached mode.

Sources: [docker.md25-41](https://github.com/charmbracelet/charm/blob/aff3071d/docker.md?plain=1#L25-L41)

Data Persistence
----------------

All Charm data is stored in the `/data` directory inside the container. To ensure data persistence across container restarts and updates, you must mount this directory as a volume to your host system.

TLS Configuration
-----------------

For production deployments, it's recommended to secure your Charm server with TLS. There are two main approaches to configuring TLS with Docker:

1.  **Using a reverse proxy** (recommended)
2.  **Direct TLS configuration** in the Charm container

### Using a Reverse Proxy

The recommended approach is to use a reverse proxy like Traefik or Caddy to handle TLS termination:

When using a reverse proxy, you need to set the following environment variables in your Charm container:

*   `CHARM_SERVER_HOST`: Set to your public hostname
*   `CHARM_SERVER_PUBLIC_URL`: Set to the full public URL (e.g., `https://cloud.example.com:35354`)

Metrics and Monitoring
----------------------

To enable Prometheus metrics collection for your Charm server, set the `CHARM_SERVER_ENABLE_METRICS` environment variable to `true`. Metrics will be available at `http://<CHARM_SERVER_HOST>:<CHARM_SERVER_STATS_PORT>/metrics`.

You can then configure Prometheus to scrape metrics from this endpoint.

Sources: [README.md280](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L280-L280)
