Introduction
------------

This guide provides comprehensive instructions for setting up and maintaining your own Charm server instance. With the official Charm Cloud service sunsetting on November 29, 2024, self-hosting becomes essential for continued use of Charm-based tools and services.

The Charm server provides several key services:

*   User account management with SSH-based authentication
*   Encrypted key-value storage (Charm KV)
*   Virtual filesystem storage (Charm FS)
*   End-to-end encryption (Charm Crypt)

For information about backing up and restoring user accounts and data, see [Backup and Restore](https://deepwiki.com/charmbracelet/charm/6.3-backup-and-restore)
.

Server Configuration
--------------------

### Environment Variables

| Variable | Description | Default | Example |
| --- | --- | --- | --- |
| `CHARM_SERVER_BIND_ADDRESS` | Network interface to listen to | 0.0.0.0 | 127.0.0.1 |
| `CHARM_SERVER_HOST` | Hostname to advertise | localhost | charm.example.com |
| `CHARM_SERVER_SSH_PORT` | SSH server port | 35353 | 22022 |
| `CHARM_SERVER_HTTP_PORT` | HTTP server port | 35354 | 8080 |
| `CHARM_SERVER_STATS_PORT` | Stats server port | 35355 | 8081 |
| `CHARM_SERVER_HEALTH_PORT` | Health server port | 35356 | 8082 |
| `CHARM_SERVER_DATA_DIR` | Server data directory | ./data\_ | /var/lib/charm |
| `CHARM_SERVER_USE_TLS` | Whether to use TLS | false | true |
| `CHARM_SERVER_TLS_KEY_FILE` | TLS key file path |     | /etc/charm/privkey.pem |
| `CHARM_SERVER_TLS_CERT_FILE` | TLS cert file path |     | /etc/charm/fullchain.pem |
| `CHARM_SERVER_PUBLIC_URL` | Server public URL |     | [https://charm.example.com:35354](https://charm.example.com:35354/) |
| `CHARM_SERVER_ENABLE_METRICS` | Enable Prometheus metrics | false | true |
| `CHARM_SERVER_USER_MAX_STORAGE` | Max storage per user (bytes) | 0 (no limit) | 1073741824 (1GB) |

Sources: [README.md266-282](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L266-L282)

### Storage Considerations

By default, self-hosted servers don't have data storage limits, unlike the 1GB cap for free accounts on the official Charm Cloud. You can enforce storage limits by setting `CHARM_SERVER_USER_MAX_STORAGE` to a byte value.

The server stores all data in the directory specified by `CHARM_SERVER_DATA_DIR`, including user accounts, SSH keys, and encrypted data. Ensure this directory is backed up regularly.

TLS Configuration
-----------------

### Built-in TLS Support

Charm includes built-in TLS support. To enable it:

1.  Set `CHARM_SERVER_USE_TLS=true`
2.  Provide paths to your TLS certificate and key files:
    *   `CHARM_SERVER_TLS_CERT_FILE=/path/to/fullchain.pem`
    *   `CHARM_SERVER_TLS_KEY_FILE=/path/to/privkey.pem`
3.  Set `CHARM_SERVER_HOST` to match your certificate's domain name

Sources: [README.md302-306](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L302-L306)
 [docs/self-hosting.md44-49](https://github.com/charmbracelet/charm/blob/aff3071d/docs/self-hosting.md?plain=1#L44-L49)

### Using a Reverse Proxy

Many deployments use a reverse proxy like Nginx or Caddy to handle TLS termination and forward requests to the Charm server.

When using a reverse proxy:

1.  Configure the proxy to forward HTTP requests to `CHARM_SERVER_HTTP_PORT` (default 35354)
2.  Configure the proxy to forward TCP connections for SSH to `CHARM_SERVER_SSH_PORT` (default 35353)
3.  Set `CHARM_SERVER_PUBLIC_URL` to the full public URL of your reverse proxy

| --- | --- | --- | --- |

| `CHARM_SSH_PORT` | SSH port | 35353 | 35353 |

| `CHARM_DEBUG` | Enable debugging logs | false | true |

| `CHARM_IDENTITY_KEY` | Identity key path |     | ~/.ssh/id\_ed25519 |

### Using a Reverse Proxy with TLS

If you're using a reverse proxy with TLS termination:

1.  Set `CHARM_HOST` to your server's domain
2.  Set `CHARM_HTTP_PORT` to the HTTPS port of your reverse proxy (usually 443)
3.  If using a non-standard SSH port, set `CHARM_SSH_PORT` accordingly

Monitoring and Maintenance
--------------------------

### Health Checks

Charm provides a health endpoint accessible at `http://<host>:<CHARM_SERVER_HEALTH_PORT>/health` (default port 35356). This returns a simple status indicating if the server is running properly.

### Metrics

If you enable metrics with `CHARM_SERVER_ENABLE_METRICS=true`, Prometheus-compatible metrics will be available at `http://<host>:<CHARM_SERVER_STATS_PORT>/metrics` (default port 35355).

Security Considerations
-----------------------

### End-to-End Encryption

All user data stored in Charm (KV store, filesystem, etc.) is encrypted before it leaves the client:

Even as the server administrator, you cannot access the plaintext content of user data, as encryption keys are managed by client devices. This architecture ensures that even if the server is compromised, user data remains protected.

Sources: [crypt/README.md1-29](https://github.com/charmbracelet/charm/blob/aff3071d/crypt/README.md?plain=1#L1-L29)

Troubleshooting
---------------

### Common Issues

#### Server Won't Start

*   Check if ports are already in use: `lsof -i :35353` and `lsof -i :35354`
*   Verify permissions on the data directory
*   Check TLS certificate paths if using TLS

#### Client Can't Connect

*   Verify environment variables are set correctly
*   Check if firewall rules allow connections to the server ports
*   Confirm the server is running with `curl http://<host>:35356/health`

#### TLS Certificate Problems

*   Verify certificate and key files are valid
*   Ensure certificate domain matches the `CHARM_SERVER_HOST`
*   Check certificate expiration date
