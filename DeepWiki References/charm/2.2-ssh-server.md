SSH Server
==========

Relevant source files

*   [server/auth.go](https://github.com/charmbracelet/charm/blob/aff3071d/server/auth.go)
    
*   [server/link.go](https://github.com/charmbracelet/charm/blob/aff3071d/server/link.go)
    
*   [server/ssh.go](https://github.com/charmbracelet/charm/blob/aff3071d/server/ssh.go)
    

The SSH Server is a core component of the Charm infrastructure responsible for handling user authentication, account management, and device linking. It provides the authentication mechanism for clients to securely connect to Charm services and manages the process of linking multiple devices to a single Charm account. For information about HTTP-based API services, see [HTTP Server](https://deepwiki.com/charmbracelet/charm/2.1-http-server)
.

Purpose and Functionality
-------------------------

The SSH Server serves as the primary authentication gateway for Charm clients, providing:

1.  User authentication using SSH public keys
2.  JWT token generation for API access
3.  Device linking to enable multiple devices to access the same account
4.  SSH key management for accounts

Sources: [server/ssh.go32-40](https://github.com/charmbracelet/charm/blob/aff3071d/server/ssh.go#L32-L40)
 [server/auth.go13-38](https://github.com/charmbracelet/charm/blob/aff3071d/server/auth.go#L13-L38)

Server Architecture
-------------------

The SSH Server is built using the `charmbracelet/wish` library, which provides a high-level framework for building SSH applications, and the `charmbracelet/ssh` library for low-level SSH protocol handling.

### Server Initialization

The `NewSSHServer` function creates a new SSH Server instance with the following configuration:

1.  Server address and port
2.  Host key for SSH server
3.  Public key authentication handler
4.  Middleware for command routing
5.  Database connection
6.  Link queue for managing device linking

Sources: [server/ssh.go42-85](https://github.com/charmbracelet/charm/blob/aff3071d/server/ssh.go#L42-L85)

Authentication Flow
-------------------

The SSH Server authenticates clients using SSH public keys, which enables passwordless authentication and provides a secure mechanism for client identification.

Sources: [server/ssh.go110-160](https://github.com/charmbracelet/charm/blob/aff3071d/server/ssh.go#L110-L160)
 [server/auth.go40-125](https://github.com/charmbracelet/charm/blob/aff3071d/server/auth.go#L40-L125)

### JWT Token Generation

The server generates JWT (JSON Web Token) tokens for authenticated users. These tokens are used for authenticating API requests to the HTTP server. The tokens are signed using Ed25519 and include:

*   Subject: User's Charm ID
*   Expiration: 1 hour from generation
*   Issuer: Server's HTTP URL
*   Audience: Specified audience (defaults to "charm")

Sources: [server/ssh.go141-151](https://github.com/charmbracelet/charm/blob/aff3071d/server/ssh.go#L141-L151)

Device Linking
--------------

A key feature of the Charm SSH Server is the ability to link multiple devices to a single Charm account. This is implemented using a token-based system:

Sources: [server/link.go116-326](https://github.com/charmbracelet/charm/blob/aff3071d/server/link.go#L116-L326)

### Token Generation

Tokens are generated using the `toktok` library, which creates short, human-readable tokens. These tokens are designed to be easy to type on another device for linking.

Sources: [server/link.go258-270](https://github.com/charmbracelet/charm/blob/aff3071d/server/link.go#L258-L270)

Key Management
--------------

The SSH Server provides functionality for managing SSH keys associated with a Charm account:

### Listing Keys

The `api-keys` command retrieves all SSH keys associated with a user's account and identifies which key is currently active.

Sources: [server/auth.go74-109](https://github.com/charmbracelet/charm/blob/aff3071d/server/auth.go#L74-L109)

### Unlinking Keys

The `api-unlink` command removes a specific key from a user's account. This is used when a device should no longer have access to the account.

Integration with Charm Architecture
-----------------------------------

The SSH Server plays a crucial role in the overall Charm architecture by providing the authentication layer for all client-server interactions.

The SSH Server coordinates with the HTTP Server by providing authenticated clients with JWT tokens that can be used for API access. This separation of concerns allows the SSH Server to focus on secure authentication and account management, while the HTTP Server handles data operations.

Sources: [server/ssh.go114-151](https://github.com/charmbracelet/charm/blob/aff3071d/server/ssh.go#L114-L151)

Command Summary
---------------

| Command | Handler | Description |
| --- | --- | --- |
| `api-auth` | `handleAPIAuth` | Authenticates user and returns JWT token with encryption keys |
| `api-keys` | `handleAPIKeys` | Returns list of SSH keys associated with user's account |
| `api-link` | `handleAPILink` | Initiates or completes device linking process |
| `api-unlink` | `handleAPIUnlink` | Removes a key from user's account |
| `id` | `handleID` | Returns user's Charm ID |
| `jwt` | `handleJWT` | Returns JWT token for specified audience |
