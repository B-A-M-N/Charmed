Purpose and Scope
-----------------

This document details all configuration options and environment variables for the Charm client. It covers how to configure connection parameters, authentication settings, and data storage locations to customize the Charm client's behavior. For information about server-side configuration, see [Server Configuration](https://deepwiki.com/charmbracelet/charm/2.3-server-configuration)
.

-----------------------

The Charm client configuration is defined in the `Config` struct in the client package. This structure determines how the client connects to the Charm server, where it stores data, and other operational aspects.

Configuration Options
---------------------

The Charm client configuration is controlled by the following environment variables:

| Environment Variable | Description | Default Value |
| --- | --- | --- |
| `CHARM_HOST` | Server public URL | `cloud.charm.sh` |
| `CHARM_SSH_PORT` | SSH port to connect to | `35353` |
| `CHARM_HTTP_PORT` | HTTP port to connect to | `35354` |
| `CHARM_DEBUG` | Enable debugging logs | `false` |
| `CHARM_LOGFILE` | File path to output debug logs | `""` (empty) |
| `CHARM_KEY_TYPE` | Type of key to create for new users | `ed25519` |
| `CHARM_DATA_DIR` | Path to user data storage | `""` (uses XDG path) |
| `CHARM_IDENTITY_KEY` | Path to the identity key used for auth | `""` (auto-discovers) |

Sources: [client/client.go28-37](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L28-L37)
 [README.md245-252](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L245-L252)

### Connection Settings

The `CHARM_HOST`, `CHARM_SSH_PORT`, and `CHARM_HTTP_PORT` variables determine which server the client connects to. These settings are crucial when connecting to a self-hosted Charm server instead of the default cloud.charm.sh server.

Sources: [client/client.go28-32](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L28-L32)
 [client/client.go273-283](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L273-L283)

### Debug and Logging

When troubleshooting the Charm client, you can enable debug logging with `CHARM_DEBUG=true`. Output can be directed to a file by setting `CHARM_LOGFILE` to a file path.

Sources: [client/client.go33-34](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L33-L34)

### Key and Data Settings

The `CHARM_KEY_TYPE` variable determines the type of SSH key created for new users. Currently, supported values are `ed25519` (default) and `rsa`.

The `CHARM_DATA_DIR` variable sets where user data is stored. If not specified, it uses the XDG data path with a subdirectory for the specific host (`$XDG_DATA_HOME/charm/$CHARM_HOST`).

The `CHARM_IDENTITY_KEY` variable can explicitly specify the path to an SSH key for authentication. If not set, the client searches for appropriate keys in the data directory.

Sources: [client/client.go35-37](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L35-L37)

Client Configuration and Data Storage
-------------------------------------

### Data Storage Location Determination

The client uses the `DataPath()` method to determine where to store user data:

Sources: [client/client.go288-298](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L288-L298)

### Key Discovery and Management

When initializing, the client checks for an existing key in this order:

1.  Use the key at `CHARM_IDENTITY_KEY` if specified
2.  Search for keys matching `CHARM_KEY_TYPE` in the data directory
3.  Generate a new key of `CHARM_KEY_TYPE` if none found

Sources: [client/client.go62-115](https://github.com/charmbracelet/charm/blob/aff3071d/client/client.go#L62-L115)

Impact on Other Charm Components
--------------------------------

The client configuration affects all other Charm components:

1.  **Charm KV**: Uses the client configuration to connect to the server for syncing and to determine where to store the local database.
    
2.  **Charm FS**: Uses the client configuration to connect to the server for file operations and for authentication.
    
3.  **Charm Crypt**: Uses the client configuration to manage encryption keys and for authentication with the server.
    

Sources: [README.md102-176](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L102-L176)
