This document provides a comprehensive guide to the Charm command-line interface (CLI). The Charm CLI allows users to interact with various Charm Cloud services, manage accounts, work with the file system, use the key-value store, and manage encryption keys, all from the terminal.

For information about the Terminal User Interface (TUI), see [Terminal UI](https://deepwiki.com/charmbracelet/charm/4-terminal-ui)
.

Overview of Command Structure
-----------------------------

The Charm CLI is structured as a main command (`charm`) with various subcommands that provide different functionality. When run without arguments, it launches an interactive Terminal UI, but it can also be used in command-line mode with specific subcommands.

Sources: [main.go98-117](https://github.com/charmbracelet/charm/blob/aff3071d/main.go#L98-L117)

Root Command
------------

When executed without arguments, the `charm` command launches an interactive Terminal UI. When executed with subcommands, it performs specific actions in command-line mode.

    charm [command] [options]
    

All commands support the `--help` flag for more information.

Sources: [main.go27-61](https://github.com/charmbracelet/charm/blob/aff3071d/main.go#L27-L61)

File System Commands
--------------------

The file system (FS) commands allow you to interact with both the local file system and the Charm Cloud file system. Remote paths are prefixed with `charm:`.

### Basic Commands

| Command | Description | Usage |
| --- | --- | --- |
| `fs cat` | Display file contents | `charm fs cat [charm:]PATH` |
| `fs cp` | Copy file | `charm fs cp [charm:]SOURCE [charm:]DESTINATION` |
| `fs mv` | Move file | `charm fs mv [charm:]SOURCE [charm:]DESTINATION` |
| `fs rm` | Remove file | `charm fs rm [charm:]PATH` |
| `fs ls` | List files | `charm fs ls [charm:]PATH` |
| `fs tree` | Display directory tree | `charm fs tree [charm:]PATH` |

### Examples

    # Display a remote file
    charm fs cat charm:/notes.txt
    
    # Copy from local to remote
    charm fs cp local.txt charm:/remote.txt
    
    # Copy from remote to local
    charm fs cp charm:/remote.txt local.txt
    
    # List files in remote directory
    charm fs ls charm:/
    
    # Remove remote file
    charm fs rm charm:/unwanted.txt
    

### Options

*   `-r`, `--recursive`: Perform operations recursively on directories (for `cp` and `mv`)

Key-Value Store Commands
------------------------

The key-value (KV) store commands allow you to interact with a cloud-synced key-value database. The KV store can have multiple databases, specified with the `@DB` suffix.

### Basic Commands

| Command | Description | Usage |
| --- | --- | --- |
| `kv set` | Set a value for a key | `charm kv set KEY[@DB] VALUE` |
| `kv get` | Get a value for a key | `charm kv get KEY[@DB]` |
| `kv delete` | Delete a key | `charm kv delete KEY[@DB]` |
| `kv list` | List all key-value pairs | `charm kv list [@DB]` |
| `kv sync` | Sync local DB with Cloud | `charm kv sync [@DB]` |
| `kv reset` | Reset local DB | `charm kv reset [@DB]` |

### Examples

    # Set a value
    charm kv set greeting "Hello, world!"
    
    # Set a value in a specific database
    charm kv set greeting@myapp "Hello from myapp!"
    
    # Get a value
    charm kv get greeting
    
    # List all key-value pairs
    charm kv list
    
    # List keys only
    charm kv list --keys-only
    
    # Sync with cloud
    charm kv sync
    

### Options for `list` command

*   `-r`, `--reverse`: List in reverse lexicographic order
*   `-k`, `--keys-only`: Only print keys
*   `-v`, `--values-only`: Only print values
*   `-b`, `--show-binary`: Print binary values
*   `-d`, `--delimiter`: Set delimiter between keys and values (default is tab)

Account Management Commands
---------------------------

These commands allow you to manage your Charm account.

### Basic Commands

| Command | Description | Usage |
| --- | --- | --- |
| `id` | Show your Charm ID | `charm id` |
| `name` | Set or show your username | `charm name [USERNAME]` |
| `bio` | Set or show your bio | `charm bio [BIO]` |

### Key Management

| Command | Description | Usage |
| --- | --- | --- |
| `keys` | List SSH keys linked to your account | `charm keys` |
| `link` | Link a new device to your account | `charm link` |
| `backup-keys` | Backup your keys | `charm backup-keys [DIRECTORY]` |
| `import-keys` | Import keys from a backup | `charm import-keys [DIRECTORY]` |
| `key-sync` | Sync keys with the Cloud | `charm key-sync` |

### Examples

    # View your Charm ID
    charm id
    
    # Set your username
    charm name johndoe
    
    # Add a bio
    charm bio "Charm enthusiast"
    
    # List linked SSH keys
    charm keys
    
    # Link a new device
    charm link
    

Sources: [main.go98-107](https://github.com/charmbracelet/charm/blob/aff3071d/main.go#L98-L107)

Environment Configuration
-------------------------

The Charm CLI respects several environment variables for configuration:

*   `CHARM_HTTP_PORT`: HTTP port for the server
*   `CHARM_SSH_PORT`: SSH port for the server
*   `CHARM_HOST`: Host for the server
*   `CHARM_DATA_DIR`: Directory for storing data
*   `CHARM_LOGFILE`: Path to log file

Sources: [main.go39-51](https://github.com/charmbracelet/charm/blob/aff3071d/main.go#L39-L51)

Implementation Details
----------------------

The Charm CLI is built using the Cobra framework for command-line parsing. It follows a hierarchical command structure, where each subcommand is implemented as a separate Cobra command.

The main entry point is in `main.go`, which defines the root command and adds all subcommands. Each subcommand is defined in a separate file in the `cmd` package.

For file system operations, the CLI uses a combination of Go's standard file system interfaces and custom implementations for remote operations. The Key-Value store uses BadgerDB for local storage, with synchronization to the cloud.
