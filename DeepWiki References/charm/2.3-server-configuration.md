This page documents all configuration options and environment variables for the Charm server. It covers how to configure the server for different deployment scenarios including self-hosting, TLS setup, and containerized environments. For client-side configuration, see [Client Configuration](https://deepwiki.com/charmbracelet/charm/3.1-client-configuration)
.

Overview
--------

The Charm server can be configured through environment variables. This allows for flexible deployment across various environments without modifying code. The server, by default, listens for SSH connections on port 35353 and HTTP connections on port 35354.

Sources: [server/server.go28-52](https://github.com/charmbracelet/charm/blob/aff3071d/server/server.go#L28-L52)
 [README.md266-282](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L266-L282)

Environment Variables
---------------------

The Charm server is configured using the following environment variables:

| Variable | Description | Default Value |
| --- | --- | --- |
| `CHARM_SERVER_BIND_ADDRESS` | Network interface to listen on | `0.0.0.0` |
| `CHARM_SERVER_HOST` | Hostname to advertise | `localhost` |
| `CHARM_SERVER_SSH_PORT` | SSH server port | `35353` |
| `CHARM_SERVER_HTTP_PORT` | HTTP server port | `35354` |
| `CHARM_SERVER_STATS_PORT` | Stats server port | `35355` |
| `CHARM_SERVER_HEALTH_PORT` | Health server port | `35356` |
| `CHARM_SERVER_DATA_DIR` | Server data directory | `./data` |
| `CHARM_SERVER_USE_TLS` | Whether to use TLS | `false` |
| `CHARM_SERVER_TLS_KEY_FILE` | Path to TLS key file | \`\` |
| `CHARM_SERVER_TLS_CERT_FILE` | Path to TLS certificate file | \`\` |
| `CHARM_SERVER_PUBLIC_URL` | Server public URL for reverse proxy setups | \`\` |
| `CHARM_SERVER_ENABLE_METRICS` | Whether to enable Prometheus metrics | `false` |
| `CHARM_SERVER_USER_MAX_STORAGE` | Maximum storage per user (bytes, 0 = unlimited) | `0` |

These environment variables are parsed in the `DefaultConfig()` function in the server package.

Sources: [server/server.go28-41](https://github.com/charmbracelet/charm/blob/aff3071d/server/server.go#L28-L41)
 [README.md266-282](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L266-L282)

Networking Configuration
------------------------

### Basic Network Settings

The server listens on the network interface specified by `CHARM_SERVER_BIND_ADDRESS` (default: `0.0.0.0`). The server advertises itself using the hostname specified by `CHARM_SERVER_HOST` (default: `localhost`).

### Port Configuration

The Charm server requires four ports:

1.  SSH Server: `CHARM_SERVER_SSH_PORT` (default: 35353)
2.  HTTP Server: `CHARM_SERVER_HTTP_PORT` (default: 35354)
3.  Stats Server: `CHARM_SERVER_STATS_PORT` (default: 35355)
4.  Health Server: `CHARM_SERVER_HEALTH_PORT` (default: 35356)

When deploying behind a reverse proxy or load balancer, these ports need to be properly forwarded.

Sources: [server/server.go30-34](https://github.com/charmbracelet/charm/blob/aff3071d/server/server.go#L30-L34)
 [README.md269-272](https://github.com/charmbracelet/charm/blob/aff3071d/README.md?plain=1#L269-L272)

TLS Configuration
-----------------

The Charm server supports TLS for secure communication. TLS can be configured in two ways:

1.  **Built-in TLS**: Using the server's built-in TLS support
2.  **Reverse Proxy**: Terminating TLS at a reverse proxy

### Built-in TLS

To enable built-in TLS:

1.  Set `CHARM_SERVER_USE_TLS` to `true`
2.  Specify the TLS key file path with `CHARM_SERVER_TLS_KEY_FILE`
3.  Specify the TLS certificate file path with `CHARM_SERVER_TLS_CERT_FILE`

### Reverse Proxy TLS

When using a reverse proxy for TLS termination (e.g., Nginx, Caddy, Traefik):

1.  Set `CHARM_SERVER_HOST` to your public hostname
2.  Set `CHARM_SERVER_PUBLIC_URL` to the full public URL (e.g., `https://cloud.charm.sh:35354`)

The reverse proxy should handle TLS termination and forward requests to the Charm server's HTTP port.

Storage Configuration
---------------------

### Data Directory

The `CHARM_SERVER_DATA_DIR` environment variable (default: `./data`) specifies where the server stores its data, including:

*   Database files (stored in a `db` subdirectory)
*   File storage (stored in a `files` subdirectory)

### User Storage Limits

By default, self-hosted Charm servers do not impose storage limits on users. The free cloud-hosted service limits users to 1GB of storage.

To set a storage limit for users on a self-hosted server, use the `CHARM_SERVER_USER_MAX_STORAGE` environment variable. Set it to the maximum number of bytes a user can store, or `0` for unlimited storage.

Metrics and Monitoring
----------------------

The Charm server includes built-in support for Prometheus metrics.

### Enabling Metrics

To enable metrics collection:

1.  Set `CHARM_SERVER_ENABLE_METRICS` to `true`
2.  Access metrics at `http://<CHARM_SERVER_HOST>:<CHARM_SERVER_STATS_PORT>/metrics`

The metrics server listens on the port specified by `CHARM_SERVER_STATS_PORT` (default: 35355).

### Health Checks

The server also provides a health check endpoint at port `CHARM_SERVER_HEALTH_PORT` (default: 35356).

Programmatic Configuration
--------------------------

When embedding the Charm server in your own Go application, you can programmatically configure it using the `Config` struct and its various builder methods:

The main configuration methods are:

*   `DefaultConfig()`: Creates a Config with default values from environment
*   `WithDB()`: Sets a custom database implementation
*   `WithFileStore()`: Sets a custom file storage implementation
*   `WithStats()`: Sets a custom stats implementation
*   `WithKeys()`: Sets custom SSH and JWT keys
*   `WithTLSConfig()`: Sets a custom TLS configuration
*   `WithErrorLogger()`: Sets a custom error logger
*   `WithLinkQueue()`: Sets a custom link queue implementation

Sources: [server/server.go61-114](https://github.com/charmbracelet/charm/blob/aff3071d/server/server.go#L61-L114)

Summary
-------

The Charm server is highly configurable through environment variables, making it adaptable to various deployment scenarios. The key configuration areas are:

1.  **Networking**: Configure bind address, hostname, and ports
2.  **TLS**: Enable secure communication through built-in TLS or a reverse proxy
3.  **Storage**: Specify data directory and user storage limits
4.  **Metrics**: Enable Prometheus metrics for monitoring
5.  **Deployment**: Configure for Docker, systemd, or programmatic usage

When self-hosting, ensure you properly configure the `CHARM_SERVER_HOST` and, if using a reverse proxy, the `CHARM_SERVER_PUBLIC_URL` variables to ensure proper functionality.
