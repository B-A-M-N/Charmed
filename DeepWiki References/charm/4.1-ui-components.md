UI Components
=============

This page documents the common UI components and utilities used throughout the Charm Terminal User Interface (TUI). These components serve as building blocks for creating consistent, interactive terminal experiences across the Charm application.

For information about specific UI implementations like Key Management UI and User Profile UI, see [Key Management UI](https://deepwiki.com/charmbracelet/charm/4.2-key-management-ui)
 and [User Profile UI](https://deepwiki.com/charmbracelet/charm/4.3-user-profile-ui)
 respectively.

Architecture Overview
---------------------

The Charm UI is built using the [Bubble Tea](https://github.com/charmbracelet/charm/blob/aff3071d/Bubble%20Tea)
 framework, which follows the Model-View-Update (MVU) pattern. UI Components in Charm are organized into a hierarchical structure with shared styling and common utilities.

Common UI Components
--------------------

### Styles

The Charm UI uses a consistent styling system through the `common.Styles` struct, which defines colors, text formatting, and layout rules for various UI elements. The default styles are created using the `common.DefaultStyles()` function.

Key style elements include:

*   Text styles (normal, bold, subtle)
*   Color schemes for different states
*   Layout components (padding, margins, borders)

Sources: [ui/ui.go112](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L112-L112)
 [ui/common/views.go23-52](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L23-L52)

### State Indicators

UI components in Charm can exist in different states, represented by the `common.State` enumeration:

| State | Description | Visual Representation |
| --- | --- | --- |
| StateNormal | Default state | Standard appearance |
| StateSelected | Currently selected item | Highlighted (usually in fuschia) |
| StateActive | Active/current item | Emphasized appearance |
| StateSpecial | Special item | Distinctive appearance |
| StateDeleting | Item being deleted | Warning appearance (usually in red) |

Sources: [ui/common/views.go11-28](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L11-L28)

### Vertical Lines

Vertical lines are used to visually separate content and indicate state:

    │ Key: SHA256:abcdef123456...
    │ Added: 02 Jan 2023 15:04:05 UTC
    

The line color changes based on the component state, providing visual feedback about selection or deletion states.

Sources: [ui/common/views.go30-36](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L30-L36)
 [ui/keys/keyview.go66-77](https://github.com/charmbracelet/charm/blob/aff3071d/ui/keys/keyview.go#L66-L77)

### Spinner

The spinner component provides visual feedback during asynchronous operations like loading data or generating keys.

The spinner is rendered with a subtle style that doesn't distract from other UI elements but provides clear indication that an operation is in progress.

Sources: [ui/common/views.go53-59](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L53-L59)
 [ui/ui.go177-181](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L177-L181)

### Key-Value Views

The `KeyValueView` function renders labeled data in a consistent format:

    │ Username: meowgorithm
    │ User ID: 1234abcd
    

This pattern is used throughout the UI to display user information, key details, and other structured data.

Sources: [ui/common/views.go62-84](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L62-L84)

### Help Text

The `HelpView` function renders a standardized help footer that displays available commands and controls:

    j/k, ↑/↓: choose • enter: select
    

Commands are visually separated by bullet points, creating a concise reference for users.

Sources: [ui/common/views.go96-112](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L96-L112)
 [ui/ui.go414-416](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L414-L416)

### Buttons

Several button components provide interactive elements:

| Function | Description | Example |
| --- | --- | --- |
| ButtonView | Standard button | `[ OK ]` |
| YesButtonView | Affirmative button with underlined 'Y' | `[ Yes ]` |
| NoButtonView | Negative button with underlined 'N' | `[ No ]` |
| OKButtonView | Confirmation button | `[ OK ]` |
| CancelButtonView | Cancellation button | `[ Cancel ]` |

Buttons change appearance based on focus state, with focused buttons being visually distinct.

Sources: [ui/common/views.go114-182](https://github.com/charmbracelet/charm/blob/aff3071d/ui/common/views.go#L114-L182)

UI Lifecycle and Rendering
--------------------------

### Initialization

The UI is initialized with a configuration and starts in the `statusInit` state:

    NewProgram(cfg) -> initialModel(cfg) -> Init() -> Update() -> View()
    

The initialization process creates a new Bubble Tea program with the initial model, then starts the event loop.

Sources: [ui/ui.go24-32](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L24-L32)
 [ui/ui.go109-124](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L109-L124)

### Update Cycle

The Update method handles incoming messages and updates the model state based on:

1.  User input (keyboard events)
2.  System events (client initialization, data fetching)
3.  Child component updates

Child components are updated through the `updateChilden` function, which delegates updates to the appropriate submodel based on the current UI status.

Sources: [ui/ui.go126-222](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L126-L222)
 [ui/ui.go225-329](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L225-L329)

### View Rendering

The View method renders the UI based on the current state:

    View() -> [statusSpecific rendering] -> styles.App.Render(...)
    

Each status has a specific rendering approach, and the final output is wrapped and styled using the App style.

Sources: [ui/ui.go332-363](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L332-L363)

Specialized UI Elements
-----------------------

### Menu View

The menu rendering system displays available options and highlights the currently selected option:

      Link a machine
    ❯ Manage linked keys
      Set Username
      Backup
      Exit
    

The selection marker and highlighted text visually indicate the current selection.

Sources: [ui/ui.go365-383](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L365-L383)

### Error View

Error states are displayed with a consistent formatting that makes them visually distinct:

    Error: Something went wrong
    

The error message is styled with a highlighted prefix and subtle text for the error details.

Sources: [ui/ui.go418-423](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L418-L423)

Integration with Charm Client
-----------------------------

UI components interact with the Charm client through the `client.Client` instance stored in the model. This allows UI components to:

1.  Fetch user information
2.  Manage SSH keys
3.  Generate link tokens
4.  Update user settings

The client is initialized during app startup and passed to child components as needed.

Sources: [ui/ui.go120-123](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L120-L123)
 [ui/ui.go196-199](https://github.com/charmbracelet/charm/blob/aff3071d/ui/ui.go#L196-L199)

Summary
-------

The Charm UI Components system provides a consistent, modular approach to building terminal user interfaces. By using shared styles, common components, and a clear state management pattern, the system ensures a cohesive user experience across different parts of the application.

The UI architecture balances separation of concerns with practical integration, allowing components to be developed and tested independently while working together seamlessly in the complete application.
