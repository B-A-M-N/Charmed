Prerequisites
-------------

Before building or testing Charm, ensure you have the following prerequisites installed:

*   **Go**: Version 1.20 or later (latest recommended)
*   **Git**: For source code management
*   **Make**: For running build commands (optional but recommended)

Release Process
---------------

The Charm project uses GoReleaser for creating releases. When a version tag (v\*._._ format) is pushed, the release process automatically:

1.  Builds binaries for multiple platforms
2.  Creates Docker images
3.  Generates release notes
4.  Publishes artifacts to GitHub Releases
5.  Uploads packages to package repositories

To create a new release:

The CI system will automatically handle the release process.

Local Development Tips
----------------------

For efficient local development:

1.  **Use Go Modules**: Ensure you're using Go modules (`GO111MODULE=on`)
2.  **Run Tests Frequently**: Run `go test ./...` to validate changes
3.  **Check Coverage**: Monitor coverage with `go test -coverprofile=profile.cov ./...`
4.  **Lint Before Committing**: Run `golangci-lint run` to catch issues early

Common Issues and Solutions
---------------------------

| Issue | Solution |
| --- | --- |
| Build fails with dependency errors | Run `go mod tidy` to clean up dependencies |
| Tests fail with race conditions | Use `-race` flag to identify race issues |
| Coverage reports show gaps | Add tests for uncovered code paths |
| Linter flags issues | Follow the suggestions to fix style problems |

By following these guidelines, you'll be able to effectively build and test the Charm codebase, ensuring high-quality contributions.
