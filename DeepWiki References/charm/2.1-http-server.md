HTTP Server
===========

*   [server/http.go](https://github.com/charmbracelet/charm/blob/aff3071d/server/http.go)
    
*   [server/jwk.go](https://github.com/charmbracelet/charm/blob/aff3071d/server/jwk.go)

The HTTP Server is a core component of the Charm Cloud backend that provides RESTful API access to Charm's services including user management, file system operations, key-value storage, and encryption services. This page documents the HTTP server's implementation, endpoints, authentication mechanisms, and integration with the overall Charm server architecture. For information about the SSH server component, see [SSH Server](https://deepwiki.com/charmbracelet/charm/2.2-ssh-server)
.

Architecture Overview
---------------------

The HTTP Server is implemented as part of the Charm Server architecture and runs alongside the SSH Server to provide complete backend functionality. It consists of two HTTP servers: the main API server and a health check server.

Server Configuration
--------------------

The HTTP server is configured through the `Config` struct, which can be populated from environment variables or set programmatically:

| Configuration Option | Environment Variable | Default | Description |
| --- | --- | --- | --- |
| BindAddr | CHARM\_SERVER\_BIND\_ADDRESS | ""  | Address to bind the server to |
| Host | CHARM\_SERVER\_HOST | "localhost" | Host name for the server |
| HTTPPort | CHARM\_SERVER\_HTTP\_PORT | 35354 | Port for the main HTTP server |
| HealthPort | CHARM\_SERVER\_HEALTH\_PORT | 35356 | Port for the health check server |
| UseTLS | CHARM\_SERVER\_USE\_TLS | false | Whether to use TLS |
| TLSKeyFile | CHARM\_SERVER\_TLS\_KEY\_FILE |     | Path to TLS key file |
| TLSCertFile | CHARM\_SERVER\_TLS\_CERT\_FILE |     | Path to TLS certificate file |
| PublicURL | CHARM\_SERVER\_PUBLIC\_URL |     | Public URL for the server |
| UserMaxStorage | CHARM\_SERVER\_USER\_MAX\_STORAGE | 0   | Maximum storage per user (0 = unlimited) |

Sources: [server/server.go27-52](https://github.com/charmbracelet/charm/blob/aff3071d/server/server.go#L27-L52)
 [server/server.go61-70](https://github.com/charmbracelet/charm/blob/aff3071d/server/server.go#L61-L70)

API Endpoints
-------------

The HTTP server exposes the following API endpoints:

| Endpoint | Method | Description |
| --- | --- | --- |
| `/v1/id/:id` | GET | Get user by ID |
| `/v1/bio/:name` | GET | Get user by name |
| `/v1/bio` | POST | Update user profile |
| `/v1/encrypt-key` | POST | Add encryption key |
| `/v1/fs/*` | GET | Get file from user's file system |
| `/v1/fs/*` | POST | Create or update file |
| `/v1/fs/*` | DELETE | Delete file |
| `/v1/seq/:name` | GET | Get sequence value |
| `/v1/seq/:name` | POST | Increment sequence value |
| `/v1/news` | GET | Get news list |
| `/v1/news/:id` | GET | Get specific news item |
| `/v1/public/jwks` | GET | Get JSON Web Key Set |
| `/.well-known/openid-configuration` | GET | Get OpenID configuration |

Sources: [server/http.go96-108](https://github.com/charmbracelet/charm/blob/aff3071d/server/http.go#L96-L108)

Error Handling
--------------

The HTTP Server provides standardized error handling through the `renderError` and `renderCustomError` methods:

| Error Function | HTTP Status | Description |
| --- | --- | --- |
| `renderError` | 500 | Generic internal server error |
| `renderCustomError` | Configurable | Custom error with specific message and status code |

Sources: [server/http.go162-170](https://github.com/charmbracelet/charm/blob/aff3071d/server/http.go#L162-L170)

Health Endpoint
---------------

The HTTP Server includes a separate health check server that provides a simple endpoint for monitoring:

| Endpoint | Method | Response | Description |
| --- | --- | --- | --- |
| `/` | GET | "We live!" | Basic health check endpoint |

This endpoint requires no authentication and is accessible on a separate port to allow monitoring systems to check the server's status without affecting the main API server.
