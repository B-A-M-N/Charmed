User Profile UI
===============

Relevant source files

*   [ui/info/info.go](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go)
    
*   [ui/username/username.go](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go)
    

This document describes the User Profile UI components in the Charm system. These components allow users to view their account information and modify their username through terminal-based interfaces. The User Profile UI consists of two main components: the Info UI for displaying user information and the Username UI for changing usernames.

For details on key management interfaces, see [Key Management UI](https://deepwiki.com/charmbracelet/charm/4.2-key-management-ui)
.

Overview
--------

The User Profile UI provides a terminal-based interface for users to interact with their Charm account information. It leverages the Bubble Tea framework to create an interactive TUI (Terminal User Interface) that feels responsive and intuitive to use.

Sources: [ui/info/info.go1-101](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L1-L101)
 [ui/username/username.go1-283](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L1-L283)

Info UI Component
-----------------

The Info UI component is responsible for displaying the user's account information. It provides a read-only view showing details such as the username, the host server, and when the account was created.

### Structure and Flow

The Info UI follows a simple flow where it fetches the user's bio information from the server and then displays it in a formatted view.

Sources: [ui/info/info.go91-101](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L91-L101)
 [ui/info/info.go25-32](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L25-L32)

### Key Components

The Info UI consists of these key components:

1.  **Model**: Stores the UI state including the user data and error information
2.  **Update function**: Handles messages and updates the model accordingly
3.  **View function**: Renders the UI based on the current model state
4.  **GetBio command**: Fetches the user's profile data from the server

Sources: [ui/info/info.go25-32](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L25-L32)
 [ui/info/info.go11-23](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L11-L23)
 [ui/info/info.go91-101](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L91-L101)

### Rendering User Information

The Info UI displays the user's information in a key-value format, showing:

*   The host server
*   The username (or "(none set)" if not defined)
*   The account creation date

If the user information hasn't been loaded yet, it shows "Authenticating..." to indicate that data is being fetched.

Sources: [ui/info/info.go67-89](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L67-L89)

Username UI Component
---------------------

The Username UI component provides an interactive interface for users to change their username. It includes text input with validation, navigable buttons, and feedback for the user.

### Structure and States

The Username UI implements a state machine with two main states:

*   **ready**: The default state where the user can input a new username and navigate UI elements
*   **submitting**: The state during the API call when the username is being updated

Sources: [ui/username/username.go14-19](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L14-L19)

### Navigation and Focus

The UI implements a focus system allowing users to navigate between three UI elements:

1.  Text input for the username
2.  OK button to submit the change
3.  Cancel button to exit without changes

Sources: [ui/username/username.go21-28](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L21-L28)
 [ui/username/username.go59-89](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L59-L89)

### Input Validation and Error Handling

The Username UI validates usernames before submission and handles several types of errors:

1.  **Username taken**: When the chosen username is already in use
2.  **Invalid username**: When the username doesn't meet validation requirements
3.  **General errors**: Unexpected errors during the submission process

The validation process occurs both client-side before submission and server-side during the API call.

Sources: [ui/username/username.go262-282](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L262-L282)
 [ui/username/username.go202-224](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L202-L224)

### Model Structure

The Username UI Model maintains the state of the UI, including:

| Field | Type | Description |
| --- | --- | --- |
| Done | bool | Indicates when to exit this view |
| Quit | bool | Indicates when to quit the entire program |
| cc  | \*client.Client | Reference to the Charm client |
| styles | common.Styles | UI styling information |
| state | state | Current UI state (ready/submitting) |
| newName | string | The new username being entered |
| index | index | Currently focused UI element |
| errMsg | string | Error message to display |
| input | input.Model | Text input model |
| spinner | spinner.Model | Spinner for loading state |

Sources: [ui/username/username.go44-57](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L44-L57)
 [ui/username/username.go92-118](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L92-L118)

Integration with Charm Client
-----------------------------

Both the Info UI and Username UI components integrate with the Charm Client to communicate with the server. This integration follows a message-based pattern common in the Bubble Tea framework:

1.  UI components dispatch commands that make API calls via the Charm Client
2.  API responses are converted to messages that update the UI state
3.  The UI re-renders based on the updated state

The key API methods used are:

*   `cc.Bio()` - Retrieves the user's profile information
*   `cc.SetName(name)` - Updates the user's username

Sources: [ui/info/info.go91-101](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L91-L101)
 [ui/username/username.go262-282](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L262-L282)

User Interface Rendering
------------------------

Both UI components implement a View function that renders the current state as a string which is displayed in the terminal.

### Info UI Rendering

The Info UI renders the user information in a key-value format using the `common.KeyValueView` helper function:

Sources: [ui/info/info.go67-89](https://github.com/charmbracelet/charm/blob/aff3071d/ui/info/info.go#L67-L89)

### Username UI Rendering

The Username UI renders different views based on its current state:

*   In the ready state: Shows the text input and buttons
*   In the submitting state: Shows a spinner and "Submitting..." text
*   When errors occur: Shows appropriate error messages

The UI includes keyboard navigation instructions and visual indicators for the currently focused element.

Sources: [ui/username/username.go239-260](https://github.com/charmbracelet/charm/blob/aff3071d/ui/username/username.go#L239-L260)

Accessibility and User Experience
---------------------------------

The User Profile UI is designed with terminal accessibility in mind:

*   Full keyboard navigation
*   Clear error messages
*   Visual indicators for focused elements
*   Simple, clean layout

This ensures that users can efficiently manage their profile information through a terminal interface that follows standard TUI interaction patterns.
