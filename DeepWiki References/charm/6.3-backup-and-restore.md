Backup and Restore
==================

Relevant source files

This document explains how to backup and restore your Charm account and data. The focus is primarily on backing up and restoring the SSH keys that are essential for account authentication and data access in Charm. Without these keys, you cannot access your encrypted data in the Charm system.

Key Concepts
------------

When you first run the Charm client, it creates a new ED25519 key pair for you. This key pair is crucial as it:

1.  Authenticates your identity with the Charm server
2.  Provides access to your encrypted data
3.  Enables linking to your account from other devices

Therefore, backing up these keys is essential for account recovery.

Sources: [docs/backup-account.md1-5](https://github.com/charmbracelet/charm/blob/aff3071d/docs/backup-account.md?plain=1#L1-L5)
 [cmd/backup\_keys.go1-21](https://github.com/charmbracelet/charm/blob/aff3071d/cmd/backup_keys.go#L1-L21)

Backup Process
--------------

### Backing Up Your Keys

To back up your Charm account keys, use the `backup-keys` command:

    charm backup-keys
    

This creates a tar archive named `charm-keys-backup.tar` in your current directory containing your SSH keys.

### Backup Options

You can specify a custom output path using the `-o` flag:

    charm backup-keys -o ~/my-charm-backup.tar
    

You can also output the private key to stdout by using `-` as the output path, which is useful for piping to other tools:

    charm backup-keys -o - | some-encryption-tool
    

Sources: [cmd/backup\_keys.go22-88](https://github.com/charmbracelet/charm/blob/aff3071d/cmd/backup_keys.go#L22-L88)
 [docs/backup-account.md6-26](https://github.com/charmbracelet/charm/blob/aff3071d/docs/backup-account.md?plain=1#L6-L26)

Restore Process
---------------

### Restoring From a Backup File

To restore your keys from a backup file, use the `import-keys` command:

    charm import-keys charm-keys-backup.tar
    

This extracts the keys from the archive and places them in your Charm data directory, restoring access to your account.

### Restoring From Stdin

You can also restore a key from stdin, which is useful when combined with other tools:

    cat private_key | charm import-keys -f -
    

This is especially helpful when integrating with encryption tools like `melt`:

    cat seed.txt | melt restore - | charm import-keys -f -
    

Sources: [docs/restore-account.md1-14](https://github.com/charmbracelet/charm/blob/aff3071d/docs/restore-account.md?plain=1#L1-L14)

Implementation Details
----------------------

The backup and restore functionality is implemented primarily through two commands in the Charm CLI: `backup-keys` and `import-keys`.

### Backup Keys Command

The `backup-keys` command follows these steps:

1.  Retrieves the Charm client configuration and data path
2.  Validates that the data directory contains valid keys
3.  Creates a tar archive containing only the key files
4.  Saves the archive to the specified location (or outputs to stdout)

The command specifically looks for files matching the pattern `charm_(rsa|ed25519)(.pub)?` in your Charm data directory.

Best Practices
--------------

1.  **Regular Backups**: Create regular backups of your keys, especially after setting up a new account or linking new devices.
    
2.  **Secure Storage**: Store your backup files in a secure location. Consider encrypting the backup file since it contains your private keys.
    
3.  **Multiple Locations**: Keep copies of your backups in multiple secure locations to prevent data loss.
    
4.  **Test Restoration**: Periodically test the restoration process to ensure your backups are valid and you're familiar with the process.
    
5.  **Integration with Melt**: For additional security, consider using `melt` to create an encrypted backup:
    
        charm backup-keys -o - | melt > seed.txt
        
    
    And restore with:
    
        cat seed.txt | melt restore - | charm import-keys -f -
        
    

Limitations and Considerations
------------------------------

1.  The backup process only covers the SSH keys used for authentication and encryption. It does not back up your actual data stored in Charm's File System (FS) or Key-Value (KV) store. For more information on those systems, see [Data Storage](https://deepwiki.com/charmbracelet/charm/5-data-storage)
    .
    
2.  If your private key is compromised, an attacker could gain access to your Charm account and data. Always handle backup files with care.
    
3.  The self-hosting configuration for Charm is not included in the key backup. If you're self-hosting Charm, you should separately back up your server configuration. See [Self-Hosting Guide](https://deepwiki.com/charmbracelet/charm/6.1-self-hosting-guide)
     for more details.
    

Troubleshooting
---------------

| Issue | Possible Cause | Solution |
| --- | --- | --- |
| `backup-keys` fails with "we didn't find any keys to backup" | The Charm data directory doesn't contain valid key files | Ensure you've set up your Charm account properly |
| Backup file already exists | A file with the specified name already exists | Use a different output path or remove the existing file |
| Cannot restore keys | Corrupted backup file | Try a different backup file or restore from another device |
| Keys restored but cannot access account | Server connection issues or account problems | Verify server connection and try authenticating again |
