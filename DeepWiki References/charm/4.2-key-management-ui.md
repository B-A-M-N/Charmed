Key Management UI
=================

Relevant source files

*   [ui/keys/keys.go](https://github.com/charmbracelet/charm/blob/aff3071d/ui/keys/keys.go)
    
*   [ui/link/link.go](https://github.com/charmbracelet/charm/blob/aff3071d/ui/link/link.go)
    
*   [ui/linkgen/linkgen.go](https://github.com/charmbracelet/charm/blob/aff3071d/ui/linkgen/linkgen.go)
    

The Key Management UI in Charm Cloud provides a terminal-based user interface for managing SSH keys associated with a user's Charm account. This includes viewing linked keys, deleting keys, and linking new devices to an existing account through a secure authorization flow. For information about the general Terminal UI architecture, see [Terminal UI](https://deepwiki.com/charmbracelet/charm/4-terminal-ui)
.

Overview
--------

The Key Management UI consists of three primary components:

1.  **Keys UI** - Displays all SSH keys linked to a user's account and allows for key deletion
2.  **Link Generation UI** - Creates and displays linking tokens for connecting new devices
3.  **Link Receiver UI** - Handles the process of linking a new device using a token

Key Components
--------------

### SSH Keys UI

The Keys UI displays all SSH keys linked to a user's Charm account. It provides a paginated view of keys with the ability to navigate between them and delete selected keys.

### Link Generation UI

The Link Generation UI is used to create a token that can be used to link another device to the user's Charm account. It displays the generated token and handles the authorization process when the other device attempts to link.

The Link Receiver UI is used on a new device to connect to an existing Charm account using a token generated by the Link Generation UI.

Key Management Workflows
------------------------

### Viewing and Managing SSH Keys

The Keys UI displays all SSH keys linked to the user's account in a paginated view. Each key shows:

*   The key fingerprint
*   Whether it's the currently active key
*   When it was created

Users can navigate between keys using the up/down arrows or j/k keys. If there are more keys than can fit on one screen, pagination controls are provided.

    Model {
      state: stateNormal,
      activeKeyIndex: 0,  // index of currently active key
      keys: [...]        // array of public keys
      pager: { ... }     // pagination model
      index: 0           // currently selected key on current page
    }
    

Sources: [ui/keys/keys.go44-118](https://github.com/charmbracelet/charm/blob/aff3071d/ui/keys/keys.go#L44-L118)

### Deleting Keys

Keys can be deleted by selecting them and pressing 'x', followed by 'y' to confirm. There are additional confirmation steps when:

1.  Deleting the currently active key (the key being used for the current session)
2.  Deleting the last key (which effectively deletes the account)

Sources: [ui/keys/keys.go162-190](https://github.com/charmbracelet/charm/blob/aff3071d/ui/keys/keys.go#L162-L190)
 [ui/keys/keys.go374-382](https://github.com/charmbracelet/charm/blob/aff3071d/ui/keys/keys.go#L374-L382)

### Linking a New Device

The linking process involves two devices:

1.  **Initiator** - The device that already has access to the Charm account
2.  **Receiver** - The new device being linked to the account

#### Initiator Side (Link Generation)

1.  User runs the link generation UI
2.  A token is generated and displayed
3.  User provides this token to the receiver device
4.  When the receiver attempts to link, the initiator shows the request details
5.  User approves or denies the request

#### Receiver Side (Link)

1.  User runs the link command with the token from the initiator
2.  The UI attempts to connect to the Charm server
3.  If the token is valid, it waits for authorization from the initiator
4.  Upon approval, the device is linked to the account

Sources: [ui/linkgen/linkgen.go314-341](https://github.com/charmbracelet/charm/blob/aff3071d/ui/linkgen/linkgen.go#L314-L341)
 [ui/link/link.go174-189](https://github.com/charmbracelet/charm/blob/aff3071d/ui/link/link.go#L174-L189)

Implementation Details
----------------------

### Keys UI Model

The Keys UI is implemented using the Bubble Tea framework and maintains a model with the following key components:

| Field | Type | Description |
| --- | --- | --- |
| `cc` | `*client.Client` | Pointer to Charm client for API operations |
| `state` | `state` | Current UI state (loading, normal, deleting, etc.) |
| `activeKeyIndex` | `int` | Index of the key currently in use |
| `keys` | `[]*charm.PublicKey` | Slice of all keys linked to the account |
| `index` | `int` | Selected key index on current page |
| `pager` | `pager.Model` | Pagination model for displaying keys |

Sources: [ui/keys/keys.go53-68](https://github.com/charmbracelet/charm/blob/aff3071d/ui/keys/keys.go#L53-L68)

### Link Generation UI Model

The Link Generation UI model maintains:

| Field | Type | Description |
| --- | --- | --- |
| `lh` | `*linkHandler` | Handles communication channels for linking |
| `status` | `status` | Current state of the linking process |
| `token` | `string` | Generated token for linking |
| `linkRequest` | `linkRequest` | Details about the incoming link request |
| `buttonIndex` | `int` | Selected button (accept/reject) |

Sources: [ui/linkgen/linkgen.go50-66](https://github.com/charmbracelet/charm/blob/aff3071d/ui/linkgen/linkgen.go#L50-L66)

### Link Receiver UI Model

The Link Receiver UI model includes:

| Field | Type | Description |
| --- | --- | --- |
| `lh` | `*linkHandler` | Handles communication channels for linking |
| `code` | `string` | Token provided for linking |
| `status` | `status` | Current state of the linking process |
| `alreadyLinked` | `bool` | Whether the key was already linked |

Sources: [ui/link/link.go45-55](https://github.com/charmbracelet/charm/blob/aff3071d/ui/link/link.go#L45-L55)

### Communication Channels

All three components use a channel-based communication system with the Charm client to handle asynchronous operations. The `linkHandler` struct manages channels for:

*   Token creation
*   Link requests
*   Authorization responses
*   Success/failure notifications
*   Timeouts
*   Errors

This allows the UI to remain responsive while waiting for network operations to complete.

 [ui/linkgen/linkgen.go314-382](https://github.com/charmbracelet/charm/blob/aff3071d/ui/linkgen/linkgen.go#L314-L382)

Error Handling
--------------

The Key Management UI handles various error conditions:

1.  **Authentication Failures** - When the SSH key is invalid or not recognized
2.  **Network Errors** - When the Charm server cannot be reached
3.  **Timeout Errors** - When linking requests time out
4.  **Invalid Tokens** - When the provided linking token is invalid

Each error state is displayed to the user with an appropriate message.
