Field Interface

The Field interface is a core building block in the charmbracelet/huh library that defines the contract for all form input elements. It ensures that all input types (text inputs, selectors, confirmation dialogs, etc.) can work consistently within the form system. This page documents the Field interface itself, not its specific implementations which are covered in section [Field Types](https://deepwiki.com/charmbracelet/huh/3-field-types)
.

Interface Purpose and Architecture
----------------------------------

The Field interface sits between the Group container and concrete field implementations (Input, Select, etc.), providing a standardized way to:

1.  Render field components
2.  Handle user interaction
3.  Validate input data
4.  Integrate with the Bubble Tea framework
5.  Support accessibility

 [group.go2-13](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L2-L13)

Interface Definition
--------------------

The Field interface defines a comprehensive set of methods that all field types must implement:

Sources: [form.go132-189](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L132-L189)

### Method Categories

The Field interface methods can be grouped into several functional categories:

| Category | Methods | Purpose |
| --- | --- | --- |
| **Bubble Tea Model** | `Init()`, `Update()`, `View()` | Provide integration with the Bubble Tea framework |
| **Focus Management** | `Focus()`, `Blur()` | Handle when a field gains or loses focus |
| **Error Handling** | `Error()` | Provide validation results |
| **Execution** | `Run()` | Allow running a field independently |
| **Display Behavior** | `Skip()`, `Zoom()`, `KeyBinds()` | Control navigation and display characteristics |
| **Configuration** | `WithTheme()`, `WithAccessible()`, `WithKeyMap()`, `WithWidth()`, `WithHeight()`, `WithPosition()` | Set field appearance and behavior |
| **Data Access** | `GetKey()`, `GetValue()` | Retrieve field data |

Sources: [form.go132-189](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L132-L189)

Method Details
--------------

### Bubble Tea Integration

As a Bubble Tea component, each Field implements the Model interface:

| Method | Description |
| --- | --- |
| `Init() tea.Cmd` | Initializes the field and returns any initial commands |
| `Update(tea.Msg) (tea.Model, tea.Cmd)` | Handles updates based on messages (key presses, etc.) |
| `View() string` | Renders the field as a string for display |

Sources: [form.go140-142](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L140-L142)

### Focus Management

| Method | Description |
| --- | --- |
| `Focus() tea.Cmd` | Called when the field receives focus; can return commands to be executed |
| `Blur() tea.Cmd` | Called when the field loses focus; typically saves the current value |

Sources: [form.go145-146](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L145-L146)

### Error Handling and Execution

| Method | Description |
| --- | --- |
| `Error() error` | Returns any validation error for the field |
| `Run() error` | Runs the field individually (outside a form context) |

Sources: [form.go149-152](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L149-L152)

### Display Behavior

| Method | Description |
| --- | --- |
| `Skip() bool` | When true, the field will be skipped during tab navigation |
| `Zoom() bool` | When true, the field expands to take up the entire group height |
| `KeyBinds() []key.Binding` | Returns the key bindings for help text display |

Sources: [form.go157-164](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L157-L164)

### Configuration Methods

Methods for styling and configuring the field:

| Method | Description |
| --- | --- |
| `WithTheme(*Theme) Field` | Sets the visual styling theme |
| `WithAccessible(bool) Field` | Enables accessibility mode for screen readers |
| `WithKeyMap(*KeyMap) Field` | Sets keyboard shortcuts |
| `WithWidth(int) Field` | Sets the display width |
| `WithHeight(int) Field` | Sets the display height |
| `WithPosition(FieldPosition) Field` | Sets positional information |

Sources: [form.go166-182](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L166-L182)

### Data Access

| Method | Description |
| --- | --- |
| `GetKey() string` | Returns the field's identifier key |
| `GetValue() any` | Returns the field's current value |

Sources: [form.go185-188](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L185-L188)

Field Implementation Pattern
----------------------------

All field implementations follow a similar pattern:

1.  Create a constructor that initializes the field's state
2.  Provide field-specific configuration methods (e.g., `Title()`, `Value()`)
3.  Implement all the Field interface methods

Field Implementations
---------------------

The library provides several implementations of the Field interface, each serving a specific input need:

| Field Type | Description | Key Features |
| --- | --- | --- |
| Input | Single-line text input | Character limits, password mode, autocomplete |
| Text | Multi-line text input | Line numbers, external editor support |
| Select | Single option selector | Filtering, inline mode, dynamic options |
| MultiSelect | Multiple option selector | Option limits, filtering |
| Confirm | Yes/No choice | Custom button labels |
| Note | Display-only text | No input required |
| FilePicker | File system navigator | Directory browsing, file selection |

For details on these field types, see the [Field Types](https://deepwiki.com/charmbracelet/huh/3-field-types)
 section.

Creating Custom Fields
----------------------

To create a custom field, you must implement all methods of the Field interface. A typical approach is to:

1.  Create a struct with necessary state properties
2.  Implement the Bubble Tea Model methods (Init, Update, View)
3.  Implement focus management (Focus, Blur)
4.  Implement error handling and validation
5.  Implement the configuration methods
6.  Implement data access methods (GetKey, GetValue)

The existing field implementations (like Input, Select, etc.) provide excellent reference examples.
