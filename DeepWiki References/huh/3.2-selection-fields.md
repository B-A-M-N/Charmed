Selection fields in Huh provide interactive components for users to choose from a list of options in terminal UIs. This page covers the `Select` and `MultiSelect` field types, which allow selecting single or multiple options respectively. For text input fields, see [Text Input Fields](https://deepwiki.com/charmbracelet/huh/3.1-text-input-fields)
.

Overview of Selection Fields
----------------------------

Huh provides two types of selection fields:

1.  **Select** - Allows users to choose one option from a list
2.  **MultiSelect** - Allows users to choose multiple options from a list

Both field types support features like:

*   Custom titles and descriptions
*   Filtering options with keyboard input
*   Custom validation
*   Dynamic option generation
*   Accessibility mode for screen readers

The Option Type
---------------

Both selection fields use the `Option` type to represent selectable choices. An option consists of:

*   **Key** - The text displayed to the user
*   **Value** - The actual value stored when selected (can be any comparable type)
*   **selected** - Boolean indicating if the option is selected

Huh provides helper functions to create options:

1.  `NewOption(key string, value T)` - Create a single option with a custom key and value
2.  `NewOptions(values ...T)` - Create options from a list of values, using the string representation as the key
3.  `Selected(bool)` - Method to set whether an option is pre-selected

Sources: [option.go1-39](https://github.com/charmbracelet/huh/blob/151ba059/option.go#L1-L39)

Select Field
------------

The `Select` field provides a way to choose a single option from a list. It's implemented as a generic type `Select[T]`, where `T` is the type of the value to be selected.

### Features and Configuration

*   **Value/Accessor** - Set the variable to store the selected value
*   **Key** - Identifier for retrieving the value after form submission
*   **Title/TitleFunc** - Static or dynamic title for the field
*   **Description/DescriptionFunc** - Static or dynamic description
*   **Options/OptionsFunc** - Static or dynamically generated options
*   **Filtering** - Filter options by typing
*   **Inline** - Compact horizontal display mode
*   **Height** - Control the visible height of the options list
*   **Validation** - Custom validation function

MultiSelect Field
-----------------

The `MultiSelect` field allows selection of multiple options from a list. It's implemented as a generic type `MultiSelect[T]`, where `T` is the type of the values to be selected.

### Features and Configuration

*   **Value/Accessor** - Set the variable to store selected values (as a slice)
*   **Key** - Identifier for retrieving values after form submission
*   **Title/TitleFunc** - Static or dynamic title for the field
*   **Description/DescriptionFunc** - Static or dynamic description
*   **Options/OptionsFunc** - Static or dynamically generated options
*   **Filtering** - Filter options by typing
*   **Limit** - Maximum number of selections allowed
*   **Height** - Control the visible height of the options list
*   **Validation** - Custom validation function

Common Features and Behaviors
-----------------------------

Both `Select` and `MultiSelect` share several key behaviors and features that enhance their usability.

### Filtering Options

Both field types support filtering options by typing. When the user presses `/`, they enter filtering mode and can type to narrow down the options shown.

    Title: Choose your burger /burger
    
    > Charmburger Classic
      Chickwich
    

The filter functionality:

*   Is enabled by default
*   Can be toggled with the `/` key
*   Filters case-insensitively
*   Matches substrings anywhere in the option text

### Dynamic Content

Both field types support dynamic content through their `TitleFunc`, `DescriptionFunc`, and `OptionsFunc` methods:

The dynamic content system:

*   Re-evaluates when specified bindings change
*   Caches results to prevent unnecessary recalculations
*   Displays loading indicators for slow operations
*   Works with any comparable value as a binding

### Accessibility

Both field types provide accessibility support for screen readers:

When accessibility mode is enabled:

*   The field is rendered as plain text with numbered options
*   Users input numbers to make selections
*   No TUI or cursor-based navigation is used
*   Works with screen readers and other assistive technologies

Implementation Details
----------------------

### Internal State Management

Selection fields maintain several internal state properties:

*   **Select**:
    
    *   `selected` - Index of the currently selected option
    *   `filtering` - Whether filtering mode is active
    *   `filter` - Text input model for filter text
    *   `filteredOptions` - Options matching the current filter
*   **MultiSelect**:
    
    *   `cursor` - Index of the option under the cursor
    *   `filtering` - Whether filtering mode is active
    *   `filter` - Text input model for filter text
    *   `filteredOptions` - Options matching the current filter

Both types implement the Bubble Tea event loop through their `Update` and `View` methods, handling keyboard inputs and rendering.

### Integration with Bubble Tea

Selection fields implement the Bubble Tea `Model` interface:

This allows them to:

*   Be used in standalone forms or embedded in larger Bubble Tea applications
*   Handle keyboard events and update their internal state
*   Render their current state as a string for display

Comparison of Select vs MultiSelect
-----------------------------------

| Feature | Select | MultiSelect |
| --- | --- | --- |
| Value Type | Single value of type T | Slice of values \[\]T |
| Selection UI | Cursor points to selected item | Selected items have checkmarks |
| Navigation | Move cursor with j/k or arrow keys | Move cursor with j/k or arrow keys |
| Selection | Auto-selects item under cursor | Toggle selection with space |
| Inline Mode | Supports horizontal display | Not supported |
| Limit | N/A | Can limit maximum selections |
| Selection Shortcuts | N/A | Select All (a) / Select None (A) |
