Confirmation Field
==================

Relevant source files

*   [README.md](https://github.com/charmbracelet/huh/blob/151ba059/README.md?plain=1)
    
*   [examples/burger/main.go](https://github.com/charmbracelet/huh/blob/151ba059/examples/burger/main.go)
    
*   [field\_confirm.go](https://github.com/charmbracelet/huh/blob/151ba059/field_confirm.go)
    

The Confirmation Field (`Confirm`) is a specialized field type in the charmbracelet/huh library that enables binary yes/no decisions within forms. This field allows users to toggle between affirmative and negative options, storing their selection as a boolean value. For other selection types with multiple options, see [Selection Fields](https://deepwiki.com/charmbracelet/huh/3.2-selection-fields)
.

Overview
--------

The `Confirm` field displays a title and optional description, followed by two buttons - one affirmative ("Yes" by default) and one negative ("No" by default). When used in a form, it captures a boolean value indicating the user's choice.

Sources: [field\_confirm.go15-386](https://github.com/charmbracelet/huh/blob/151ba059/field_confirm.go#L15-L386)

Key Configuration Options
-------------------------

The `Confirm` field offers several configuration methods to customize its appearance and behavior:

| Method | Purpose | Example |
| --- | --- | --- |
| `Title(string)` | Sets the title text | `Title("Are you sure?")` |
| `TitleFunc(func() string, bindings)` | Sets dynamic title based on other field values | `TitleFunc(func() string { return "Hello, " + name }, &name)` |
| `Description(string)` | Sets descriptive text below the title | `Description("This action cannot be undone")` |
| `Affirmative(string)` | Customizes the affirmative button text (default: "Yes") | `Affirmative("Proceed")` |
| `Negative(string)` | Customizes the negative button text (default: "No") | `Negative("Cancel")` |
| `Value(*bool)` | Binds the field to a boolean variable | `Value(&proceed)` |
| `Validate(func(bool) error)` | Sets validation logic | `Validate(func(b bool) error { if !b { return errors.New("must agree") } return nil })` |
| `WithButtonAlignment(position)` | Sets alignment of the buttons | `WithButtonAlignment(lipgloss.Left)` |
| `Inline(bool)` | Controls if title/description appear inline with buttons | `Inline(true)` |

Rendering and Display
---------------------

The `Confirm` field renders differently based on whether it's focused and what the current value is:

1.  When focused, the currently selected option (Yes/No) is highlighted
2.  The value can be toggled with Space or directly selected with y/n keys
3.  The layout handles title, description, any error messages, and the button row

Sources: [field\_confirm.go235-298](https://github.com/charmbracelet/huh/blob/151ba059/field_confirm.go#L235-L298)

Keyboard Controls
-----------------

The `Confirm` field responds to the following keyboard inputs:

| Key | Action |
| --- | --- |
| Space | Toggle between Yes/No |
| y   | Select affirmative option (Yes) |
| n   | Select negative option (No) |
| Enter | Confirm and move to next field |
| Tab | Move to next field |
| Shift+Tab | Move to previous field |

Sources: [field\_confirm.go155-158](https://github.com/charmbracelet/huh/blob/151ba059/field_confirm.go#L155-L158)

Accessibility Support
---------------------

The `Confirm` field includes built-in accessibility mode for screen readers:

1.  When accessibility is enabled, the field renders as a simple prompt with \[y/N\] or \[Y/n\] notation
2.  The prompt uses high-contrast styling suitable for screen readers
3.  User can respond with 'y' or 'n' in a plain text interface

Sources: [field\_confirm.go300-321](https://github.com/charmbracelet/huh/blob/151ba059/field_confirm.go#L300-L321)

Implementation Details
----------------------

The `Confirm` field implements the `Field` interface and uses the Bubble Tea library for terminal UI rendering. The field maintains its state using an `Accessor[bool]` to store and retrieve the boolean value. Styling is provided by the theme system, with different styles for focused and blurred states.

When the field receives keyboard input, it updates its internal state and bound variable accordingly. The field also supports validation through a user-provided validation function.
