This page documents the text input field types available in the Huh library for collecting textual input from users in terminal forms. These include the single-line `Input` field and multi-line `Text` field components. For information about other field types like selection fields, see [Selection Fields](https://deepwiki.com/charmbracelet/huh/3.2-selection-fields)
.

Input Field
-----------

The `Input` field provides a single-line text input component for collecting brief textual data from users. It's ideal for capturing names, email addresses, phone numbers, or any short text responses.

### Key Features

| Feature | Method | Description |
| --- | --- | --- |
| Title | `Title(string)` | Sets the field's title/label |
| Description | `Description(string)` | Adds explanatory text below the title |
| Placeholder | `Placeholder(string)` | Sets placeholder text shown when field is empty |
| Character Limit | `CharLimit(int)` | Restricts input to specified character count |
| Validation | `Validate(func(string) error)` | Validates user input |
| Prompt | `Prompt(string)` | Sets the input prompt (defaults to ">") |
| Echo Mode | `EchoMode(EchoMode)` | Controls how input is displayed (normal, password, none) |
| Inline Layout | `Inline(bool)` | Places title on same line as input field |
| Suggestions | `Suggestions([]string)` | Provides autocompletion suggestions |

Dynamic Content
---------------

Both field types support dynamic content through function-based setters that allow fields to update based on the state of other fields:

| Static Method | Dynamic Method | Purpose |
| --- | --- | --- |
| `Title(string)` | `TitleFunc(func() string, binding any)` | Dynamic title |
| `Description(string)` | `DescriptionFunc(func() string, binding any)` | Dynamic description |
| `Placeholder(string)` | `PlaceholderFunc(func() string, binding any)` | Dynamic placeholder |
| `Suggestions([]string)` | `SuggestionsFunc(func() []string, binding any)` | Dynamic suggestions |

The dynamic methods take a function that returns the desired value and a binding to another field. When the bound field changes, the function is re-evaluated.

Accessibility Support
---------------------

Both text input field types include accessibility features for screen readers and other assistive technologies. When accessibility mode is enabled, the fields render as standard terminal prompts rather than TUI elements.

In accessible mode, text input fields:

*   Present a simple prompt
*   Accept input directly
*   Validate the input
*   Re-prompt if validation fails

Best Practices
--------------

1.  **Choose the right field type**:
    
    *   Use `Input` for single-line text like names, emails, short answers
    *   Use `Text` for multi-line content like comments, descriptions, paragraphs
2.  **Provide clear guidance**:
    
    *   Use the `Title` to identify what information you're requesting
    *   Use `Description` to provide context or instructions
    *   Use `Placeholder` to show example input
3.  **Validate appropriately**:
    
    *   Validate only what's necessary
    *   Provide clear error messages that explain how to fix the issue
    *   Consider using character limits to prevent extreme inputs
4.  **Be accessible**:
    
    *   Support accessibility mode for screen reader compatibility
    *   Test your forms with accessibility mode enabled
