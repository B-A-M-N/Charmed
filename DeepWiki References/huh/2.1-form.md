Form

The Form component is the primary building block of the `charmbracelet/huh` library. It serves as a container for organizing and managing interactive form elements in terminal user interfaces. This page documents the Form component's structure, capabilities, and configuration options.

For information about the individual fields that can be added to forms, see [Field Types](https://deepwiki.com/charmbracelet/huh/3-field-types)
.

Overview
--------

Form is the top-level structure in the huh library that organizes collections of input fields into logical groups. A Form consists of one or more Groups, each containing one or more Fields. The Form handles navigation between groups, validation, and submission of user input.

Sources: [form.go60-96](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L60-L96)
 [group.go16-44](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L16-L44)

Form Structure
--------------

The Form struct is defined in the library with several key components:

Sources: [form.go60-96](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L60-L96)

The Form struct has the following key components:

*   **selector**: Manages the collection of groups in the form
*   **results**: Stores the values entered by the user, keyed by field identifiers
*   **State**: Tracks the current state of the form (normal, completed, or aborted)
*   **theme**: Controls the visual appearance of the form
*   **keymap**: Defines the keyboard mappings for form navigation and interaction
*   **layout**: Determines how the form is arranged visually
*   **accessible**: Controls whether the form should run in accessible mode for screen readers

Creating Forms
--------------

To create a new Form, use the `NewForm` function and provide one or more Groups:

    form := huh.NewForm(
        huh.NewGroup(
            // Fields for the first group
            huh.NewInput().Title("Name").Value(&name),
            huh.NewSelect<FileRef file-url="https://github.com/charmbracelet/huh/blob/151ba059/string" undefined  file-path="string">Hii</FileRef>Title("Color").Options(...),
        ),
        huh.NewGroup(
            // Fields for the second group
            huh.NewConfirm().Title("Proceed?").Value(&confirm),
        ),
    )

Form Configuration
------------------

The Form struct provides several methods for customization:

| Method | Purpose |
| --- | --- |
| `WithTheme(theme *Theme)` | Apply a theme to the form |
| `WithKeyMap(keymap *KeyMap)` | Set custom key bindings |
| `WithWidth(width int)` | Set the form width |
| `WithHeight(height int)` | Set the form height |
| `WithLayout(layout Layout)` | Set the form layout |
| `WithAccessible(bool)` | Enable/disable accessible mode |
| `WithTimeout(duration)` | Set a timeout for form completion |
| `WithShowHelp(bool)` | Show/hide key binding help |
| `WithShowErrors(bool)` | Show/hide validation errors |
| `WithOutput(io.Writer)` | Specify where to output the form |
| `WithInput(io.Reader)` | Specify where to read input |
| `WithProgramOptions(opts ...tea.ProgramOption)` | Set Bubble Tea program options |

Sources: [form.go234-361](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L234-L361)

### Theming

The Form can be styled using themes:

    form := huh.NewForm(
        // Groups and fields

Available built-in themes include:

*   `ThemeCharm()`
*   `ThemeDracula()`
*   `ThemeBase16()`

Form Navigation and State Management
------------------------------------

Form navigation works by moving between groups and fields. Each group is displayed as a "page" of the form, and users can navigate between fields within the current group.

Sources: [form.go36-47](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L36-L47)
 [form.go472-494](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L472-L494)
 [group.go214-249](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L214-L249)

### Form States

A Form can be in one of three states:

| State | Description |
| --- | --- |
| `StateNormal` | User is interacting with the form |
| `StateCompleted` | User has completed the form (all groups processed) |
| `StateAborted` | User has aborted the form (e.g., pressed Ctrl+C) |

Sources: [form.go36-47](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L36-L47)

### Navigation Methods

The Form provides methods for programmatic navigation:

| Method | Purpose |
| --- | --- |
| `NextGroup()` | Move to the next group |
| `PrevGroup()` | Move to the previous group |
| `NextField()` | Move to the next field in the current group |
| `PrevField()` | Move to the previous field in the current group |
| `GetFocusedField()` | Get the currently focused field |

Sources: [form.go472-499](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L472-L499)

Running Forms
-------------

There are several ways to run a form:

### Standalone Mode

Run a form directly to collect input:

    if err := form.Run(); err != nil {
        // Handle error
    }
    

This method blocks until the form is completed or aborted.

Sources: [form.go654-658](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L654-L658)

### Running with Context

Run a form with a context for cancelation:

    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()
    
    if err := form.RunWithContext(ctx); err != nil {
        // Handle error
    }
    

Sources: [form.go660-677](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L660-L677)

### Accessible Mode

Forms can run in accessible mode, which makes them more compatible with screen readers:

    form := huh.NewForm(
        // Groups and fields
    ).WithAccessible(true)
    

In accessible mode, the form uses simpler terminal prompting rather than a full TUI renderer.

Sources: [form.go234-237](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L234-L237)
 [form.go702-720](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L702-L720)

Examples
--------

### Basic Form Example

    package main
    
    import (
        "fmt"
        "github.com/charmbracelet/huh"
    )
    
    func main() {
        var name string
        var age int
        var confirm bool
        
        form := huh.NewForm(
            huh.NewGroup(
                huh.NewInput().
                    Title("What's your name?").
                    Value(&name),
                    
                huh.NewInput().
                    Title("How old are you?").
                    Value(&age),
            ),
            huh.NewGroup(
                huh.NewConfirm().
                    Title("Is this information correct?").
                    Value(&confirm),
            ),
        )
        
        if err := form.Run(); err != nil {
            fmt.Println("Error:", err)
            return
        }
        
        if confirm {
            fmt.Printf("Hello, %s! You are %d years old.\n", name, age)
        } else {
            fmt.Println("Information not confirmed.")
        }
    }
    

Sources: [examples/multiple-groups/main.go11-79](https://github.com/charmbracelet/huh/blob/151ba059/examples/multiple-groups/main.go#L11-L79)

### Integration with Bubble Tea

Forms can be embedded in larger Bubble Tea applications:

    type Model struct {
        form *huh.Form
        // Other application state...
    }
    
    func NewModel() Model {
        return Model{
            form: huh.NewForm(
                // Groups and fields
            ),
        }
    }
    
    func (m Model) Init() tea.Cmd {
        return m.form.Init()
    }
    
    func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
        // Handle other application messages...
        
        // Process form updates
        form, cmd := m.form.Update(msg)
        if f, ok := form.(*huh.Form); ok {
            m.form = f
        }
        
        // Check if form is completed
        if m.form.State == huh.StateCompleted {
            // Handle form completion
        }
        
        return m, cmd
    }
    
    func (m Model) View() string {
        // Render form and other application elements
        return m.form.View()
    }
    

Sources: [examples/bubbletea/main.go62-167](https://github.com/charmbracelet/huh/blob/151ba059/examples/bubbletea/main.go#L62-L167)
 [examples/bubbletea-options/main.go10-22](https://github.com/charmbracelet/huh/blob/151ba059/examples/bubbletea-options/main.go#L10-L22)

Error Handling
--------------

Forms can return several types of errors:

| Error | Description |
| --- | --- |
| `ErrUserAborted` | User aborted the form (e.g., pressed Ctrl+C) |
| `ErrTimeout` | Form timed out before completion |
| `ErrTimeoutUnsupported` | Timeout requested while in accessible mode (not supported) |

Sources: [form.go50-57](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L50-L57)

Form Layout System
------------------

Forms use a layout system to control how groups and fields are arranged:

    form := huh.NewForm(
        // Groups and fields
    ).WithLayout(huh.LayoutDefault)
    

The layout system determines the visual presentation of the form, including group widths and overall form appearance.

Sources: [form.go354-361](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L354-L361)

Summary
-------

The Form component is the central organizing structure in the huh library. It provides a flexible and customizable way to create interactive terminal forms with multiple pages of input fields. Forms can be run standalone or integrated into larger Bubble Tea applications, and they support theming, keyboard navigation, and accessibility features.
