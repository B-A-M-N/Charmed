This document provides a technical overview of the architecture of the `charmbracelet/huh` library. It explains the core components, their relationships, and the design patterns used throughout the codebase. For installation instructions, see [Installation](https://deepwiki.com/charmbracelet/huh/1.2-installation)

Component Overview
------------------

The `huh` library is built around a hierarchical component architecture that enables the creation and management of interactive forms in terminal user interfaces.

Sources: [form.go59-96](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L59-L96)

Core Components
---------------

### Form

The `Form` is the primary container that manages a collection of `Group` instances. Forms handle user navigation between groups, field value collection, validation, and rendering.

Forms maintain a state (`FormState`) that can be:

*   `StateNormal` - User is completing the form
*   `StateCompleted` - User has completed the form
*   `StateAborted` - User has aborted the form

The `Form` uses a `Selector` to manage its collection of groups and track which group is currently active.

Sources: [form.go36-48](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L36-L48)
 [form.go59-96](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L59-L96)
 [form.go501-517](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L501-L517)
 [form.go520-625](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L520-L625)

### Group

A `Group` represents a logical collection of fields that are displayed together. Groups can be thought of as pages or sections within a form.

Groups are responsible for:

1.  Displaying a title and description
2.  Rendering fields in a viewport
3.  Handling navigation between fields
4.  Showing errors and help text

Sources: [group.go19-44](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L19-L44)
 [group.go217-249](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L217-L249)
 [group.go330-399](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L330-L399)

### Field Interface

The `Field` interface defines the contract for all form input types. It is implemented by various field types such as `Input`, `Text`, `Select`, and `Confirm`.

The `Field` interface implements the Bubble Tea `Model` interface, allowing fields to be used directly in Bubble Tea applications. Fields handle their own state, validation, and rendering.

Sources: [form.go132-189](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L132-L189)

Layout System
-------------

The Layout system determines how groups are arranged within a form. The library provides several built-in layouts:

Layout types:

*   `LayoutDefault` - Shows a single group at a time
*   `LayoutStack` - Stacks all groups vertically
*   `LayoutColumns` - Distributes groups in even columns
*   `LayoutGrid` - Arranges groups in a grid with rows and columns

Sources: [layout.go7-11](https://github.com/charmbracelet/huh/blob/151ba059/layout.go#L7-L11)
 [layout.go13-170](https://github.com/charmbracelet/huh/blob/151ba059/layout.go#L13-L170)

Theme System
------------

Theming in `huh` is managed through a comprehensive system that allows styling of all components. The library includes several predefined themes.

The theme system leverages `lipgloss` for styling and supports styling of forms, groups, fields, and their various states (focused, blurred).

Sources: [theme.go9-18](https://github.com/charmbracelet/huh/blob/151ba059/theme.go#L9-L18)
 [theme.go20-77](https://github.com/charmbracelet/huh/blob/151ba059/theme.go#L20-L77)
 [theme.go84-329](https://github.com/charmbracelet/huh/blob/151ba059/theme.go#L84-L329)

Selector Utility
----------------

The `Selector` is a utility type used throughout the library to manage collections of items (groups in a form, fields in a group) and handle navigation between them.

The `Selector` provides methods for moving through items, getting the currently selected item, and iterating over all items.

Sources: [internal/selector/selector.go4-97](https://github.com/charmbracelet/huh/blob/151ba059/internal/selector/selector.go#L4-L97)

Bubble Tea Integration
----------------------

The library is designed to work seamlessly with the Bubble Tea framework for terminal user interfaces.

Both `Form` and `Field` implement the Bubble Tea `Model` interface, allowing them to be used directly in Bubble Tea applications or standalone with the provided `Run()` method.

Sources: [form.go501-517](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L501-L517)
 [form.go520-625](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L520-L625)
 [form.go647-720](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L647-L720)

Accessibility Support
---------------------

The library includes built-in support for accessibility, particularly for screen readers, through the `WithAccessible()` method.

When running in accessible mode, the form uses simpler rendering techniques that are more compatible with screen readers, avoiding the TUI rendering typically used in terminal applications.

Sources: [form.go234-237](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L234-L237)
 [form.go702-720](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L702-L720)
