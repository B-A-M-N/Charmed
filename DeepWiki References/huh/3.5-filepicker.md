FilePicker
==========

Relevant source files

*   [field\_filepicker.go](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go)
    

The FilePicker component is a field type in the `charmbracelet/huh` library that enables users to browse and select files or directories within terminal-based forms. This component provides a navigable file system interface, allowing users to explore directories and select files based on configurable criteria.

Overview
--------

The FilePicker field creates an interactive file browsing experience similar to traditional file pickers in graphical interfaces but optimized for terminal environments. It integrates with the broader form system to collect file paths as string values.

Sources: [field\_filepicker.go19-42](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L19-L42)
 [field\_filepicker.go362-439](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L362-L439)

Basic Usage
-----------

Creating and configuring a FilePicker field follows the same pattern as other fields in the `huh` library, with additional options specific to file selection.

Sources: [field\_filepicker.go44-58](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L44-L58)
 [field\_filepicker.go177-239](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L177-L239)

### Example Configuration

Here's how to create and configure a FilePicker field:

1.  Create a new FilePicker with `NewFilePicker()`
2.  Set configuration options using method chaining
3.  Add it to a form group

Key configuration methods include:

| Method | Purpose | Default |
| --- | --- | --- |
| `Value(*string)` | Binds a string pointer to store the selected file path | nil |
| `Key(string)` | Sets a key to retrieve the value after form submission | ""  |
| `Title(string)` | Sets a title for the field | ""  |
| `Description(string)` | Sets a description for the field | ""  |
| `AllowedTypes([]string)` | Sets file extensions that can be selected | \[\] (all allowed) |
| `FileAllowed(bool)` | Whether files can be selected | true |
| `DirAllowed(bool)` | Whether directories can be selected | false |
| `CurrentDirectory(string)` | Sets the starting directory | Current working directory |

Sources: [field\_filepicker.go61-159](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L61-L159)

File Selection Interface
------------------------

The FilePicker has two main states:

1.  **Collapsed View**: Shows the currently selected file path (or "No file selected")
2.  **Expanded View**: Shows the file browser interface for navigation and selection

When a user opens the file browser, they can:

*   Navigate up and down through files and directories
*   Enter directories to explore their contents
*   Select files (if allowed and matching the allowed types)
*   Navigate back to parent directories

Sources: [field\_filepicker.go201-239](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L201-L239)
 [field\_filepicker.go291-307](https://github.com/charmbracelet/huh/blob/151ba059/field_filepicker.go#L291-L307)

Styling and Theming
-------------------

The FilePicker integrates with the form theme system to maintain consistent styling. It applies different styles based on whether the field is focused or blurred, and also styles different types of file system entries distinctly.

Key styled elements include:

*   Directory entries
*   File entries
*   Selected entry
*   Disabled entries (files that don't match allowed types)
*   File permissions (when shown)
*   File sizes (when shown)

Keyboard Navigation
-------------------

The FilePicker component has a specialized keymap to support file navigation, which gets integrated with the form's overall keymap system:

| Key Action | Default Binding | Function |
| --- | --- | --- |
| Open | Enter (when collapsed) | Opens the file browser |
| Close | Esc (when expanded) | Closes the file browser without selection |
| Up  | Up Arrow | Navigates up in the file list |
| Down | Down Arrow | Navigates down in the file list |
| PageUp | Page Up | Navigates up a page in the file list |
| PageDown | Page Down | Navigates down a page in the file list |
| GoToTop | Home | Navigates to the top of the file list |
| GoToBottom | End | Navigates to the bottom of the file list |
| Select | Enter (when expanded) | Selects the current file/directory |
| Back | Backspace | Navigates to parent directory |
| Next | Tab | Moves to next field (when collapsed) |
| Prev | Shift+Tab | Moves to previous field (when collapsed) |

The keymap is context-sensitive, with certain keys only being enabled in specific states (expanded vs. collapsed).

Validation
----------

FilePicker supports validation at multiple levels:

1.  **Built-in file type validation**: Only files matching the specified extensions in `AllowedTypes` can be selected
2.  **Custom validation**: User-provided validation function via the `Validate()` method
3.  **File existence validation**: In accessible mode, validates that the entered path exists

When a validation error occurs, it is displayed to the user and the field remains in an error state until corrected.

Accessibility Support
---------------------

FilePicker implements accessible mode support, which provides a text-based interface suitable for screen readers. In accessible mode:

1.  Instead of a navigable interface, a text prompt asks for a file path
2.  The path is validated for existence and file type
3.  The user-provided validation function is also applied

This ensures that the FilePicker can be used effectively with assistive technologies.

Advanced Options
----------------

The FilePicker supports several advanced display options:

| Option | Method | Default | Description |
| --- | --- | --- | --- |
| Show Hidden Files | `ShowHidden(bool)` | false | Whether to display hidden files (starting with `.`) |
| Show File Sizes | `ShowSize(bool)` | false | Whether to display file sizes |

Conclusion
----------

The FilePicker field provides a powerful way to collect file paths in terminal-based forms. It offers a rich navigation experience while maintaining compatibility with the form system and supporting accessibility. Use it when your terminal application needs file system interaction as part of its forms.

For more information about other field types available in the library, see [Field Types](https://deepwiki.com/charmbracelet/huh/3-field-types)
