Note Field
==========

Relevant source files

*   [README.md](https://github.com/charmbracelet/huh/blob/151ba059/README.md?plain=1)
    
*   [examples/burger/main.go](https://github.com/charmbracelet/huh/blob/151ba059/examples/burger/main.go)
    
*   [field\_note.go](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go)
    

The Note Field is a non-interactive component designed to display information to users within forms. Unlike other fields that collect input, Note Fields primarily provide context, instructions, or explanatory text. They can also serve as separators between functional sections of a form or as notification areas for displaying dynamic information that updates based on user interactions with other fields.

Overview
--------

The Note Field implements the Field interface but serves a different purpose than input fields. It doesn't store user inputs and instead focuses on displaying formatted text.

Sources: [field\_note.go13-35](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go#L13-L35)

Customization Options
---------------------

### Setting Text Content

Note Fields have two primary text components:

1.  **Title**: The heading text of the Note Field
2.  **Description**: The main body of text

Both can be set using either static or dynamic methods:

| Method | Description |
| --- | --- |
| `Title(string)` | Sets a static title |
| `TitleFunc(func() string, any)` | Sets a dynamic title that updates based on changes to the binding |
| `Description(string)` | Sets a static description |
| `DescriptionFunc(func() string, any)` | Sets a dynamic description that updates based on changes to the binding |

Sources: [field\_note.go56-106](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go#L56-L106)

### Visual Customization

The appearance of a Note Field can be customized with:

| Method | Description |
| --- | --- |
| `Height(int)` | Sets the height of the Note Field |
| `WithWidth(int)` | Sets the width of the Note Field |
| `WithTheme(*Theme)` | Sets the theme for styling the Note Field |

Sources: [field\_note.go108-112](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go#L108-L112)
 [field\_note.go263-293](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go#L263-L293)

### Navigation Controls

Note Fields can include a navigation button:

| Method | Description |
| --- | --- |
| `Next(bool)` | Controls whether to show a "Next" button |
| `NextLabel(string)` | Customizes the text on the Next button |

By default, Note Fields are "skipped" in form navigation (users can advance past them without interaction), but this behavior can be modified by the form when the Note Field is the only field in a group.

Sources: [field\_note.go114-129](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go#L114-L129)

Text Formatting
---------------

Note Fields support basic text formatting through a simple markup syntax:

| Syntax | Effect |
| --- | --- |
| `_text_` | Italic text |
| `*text*` | Bold text |
| `` `code` `` | Code blocks with gray background |

The formatting is handled by the `render()` function which translates these markers into ANSI escape sequences.

Sources: [field\_note.go315-371](https://github.com/charmbracelet/huh/blob/151ba059/field_note.go#L315-L371)

Integration with Forms
----------------------

Note Fields are often used as:

1.  **Welcome screens**: Introducing users to the form
2.  **Section headers**: Providing context for a group of related fields
3.  **Instructions**: Giving users guidance on how to fill out complex fields
4.  **Dynamic previews**: Showing the effect of user inputs in real-time
