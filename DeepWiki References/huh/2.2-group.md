Group
=====

The Group component is a fundamental building block in the charmbracelet/huh form system. Groups organize related Fields into cohesive sections within a Form, serving as containers that handle field navigation, error management, and visual presentation. For information about the overall Form structure, see [Form](https://deepwiki.com/charmbracelet/huh/2.1-form)
; for details about individual Fields, see [Field Interface](https://deepwiki.com/charmbracelet/huh/2.3-field-interface)
.

Purpose and Scope
-----------------

A Group:

*   Groups logically related fields together
*   Handles navigation between fields within the group
*   Maintains a title and description for context
*   Manages field layout and styling
*   Controls error handling and validation for its fields
*   Can be conditionally shown or hidden

Groups act as pages within your form - users navigate between groups as discrete steps when completing a form.

Group Structure
---------------

The `Group` struct contains the following key components:

| Component | Type | Purpose |
| --- | --- | --- |
| `selector` | `*selector.Selector[Field]` | Manages the collection of fields and selection state |
| `title` | `string` | The group's title text |
| `description` | `string` | The group's description text |
| `viewport` | `viewport.Model` | Handles scrolling for group content |
| `showHelp` | `bool` | Whether to display help text |
| `help` | `help.Model` | The help model for keybinding assistance |
| `showErrors` | `bool` | Whether to display validation errors |
| `width`/`height` | `int` | Dimension constraints |
| `theme` | `*Theme` | Styling configuration |
| `keymap` | `*KeyMap` | Keyboard controls configuration |
| `hide` | `func() bool` | Function determining if group should be hidden |
| `active` | `bool` | Whether the group is currently active |

Sources: [group.go13-44](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L13-L44)

Configuration Methods
---------------------

Groups have a fluent API for configuration:

| Method | Purpose |
| --- | --- |
| `Title(string)` | Sets the group's title |
| `Description(string)` | Sets the group's description |
| `WithShowHelp(bool)` | Controls help text visibility |
| `WithShowErrors(bool)` | Controls error message visibility |
| `WithTheme(*Theme)` | Sets styling theme |
| `WithKeyMap(*KeyMap)` | Sets keyboard controls |
| `WithWidth(int)` | Sets width for the group and its fields |
| `WithHeight(int)` | Sets height for the group |
| `WithHide(bool)` | Sets whether the group should be skipped |
| `WithHideFunc(func() bool)` | Sets a function to dynamically determine if the group should be skipped |

Sources: [group.go66-151](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L66-L151)

Navigation and Field Interaction
--------------------------------

Groups handle navigation between their fields and maintain the current field selection state. When a user interacts with a form, they're interacting with one field at a time within the active group.

When all fields in a group have valid values, the form can progress to the next group. If any field has a validation error, the form cannot progress until the error is resolved.

Error Handling
--------------

Groups collect and display errors from their fields. If any field has a validation error, the group will display those errors in its footer. The form cannot progress to the next group until all errors are resolved.

Sources: [group.go153-163](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L153-L163)
 [form.go571-624](https://github.com/charmbracelet/huh/blob/151ba059/form.go#L571-L624)

Rendering
---------

Groups have several methods for rendering:

| Method | Purpose |
| --- | --- |
| `View()` | Renders the complete group |
| `Header()` | Renders only the title and description |
| `Content()` | Renders only the fields |
| `Footer()` | Renders only the help text and errors |

The rendered output includes:

1.  The group title and description (if set)
2.  The fields in the group
3.  Help text and error messages (if any)

Sources: [group.go330-399](https://github.com/charmbracelet/huh/blob/151ba059/group.go#L330-L399)

Integration with Forms
----------------------

Groups are designed to work within the Form system:

When a form runs, it:

1.  Activates the first visible group
2.  Allows users to navigate through fields in that group
3.  When the last field is completed, moves to the next group
4.  When the last group is completed, the form is submitted

Best Practices
--------------

1.  **Logical Grouping**: Group related fields together to create a logical flow
2.  **Clear Titles and Descriptions**: Provide informative titles and descriptions to guide users
3.  **Appropriate Width**: Set a width that accommodates the longest field to prevent awkward wrapping
4.  **Validation**: Implement field validation to ensure data integrity
5.  **Conditional Display**: Use `WithHideFunc` to dynamically show/hide groups based on context

Summary
-------

The `Group` component is a versatile container for organizing fields within a form. It handles navigation, error collection, and rendering, and can be conditionally displayed based on form state. By properly configuring groups, you can create intuitive multi-step forms with clear sections and logical flow.
