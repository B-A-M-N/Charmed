Rendering Pipeline
==================

Purpose and Scope
-----------------

This document describes the multi-stage rendering process that transforms a `Style` configuration into ANSI-escaped terminal output. The rendering pipeline is the core execution path that applies all style properties in a specific order to produce correctly formatted text.

For information about configuring style properties before rendering, see [Style Configuration](https://deepwiki.com/charmbracelet/lipgloss/2.1-style-configuration)
. For details about individual text attributes, see [Text Attributes and Styling](https://deepwiki.com/charmbracelet/lipgloss/2.2-text-attributes-and-styling)
. For color management and terminal adaptation, see [Colors and Terminal Detection](https://deepwiki.com/charmbracelet/lipgloss/3-colors-and-terminal-detection)
. For border rendering specifics, see [Borders](https://deepwiki.com/charmbracelet/lipgloss/4-borders)
.

Pipeline Architecture
---------------------

The rendering pipeline is implemented in the `Render` method and operates as a pure function that takes input strings and returns a fully-styled ANSI string. The pipeline consists of three major phases:

**Sources:** [style.go267-526](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L267-L526)

Entry Point: Style.Render
-------------------------

The `Render` method is the single entry point for the rendering pipeline. It accepts variadic string arguments and an optional pre-set string value from `Style.value`.

### Method Signature and Initialization

[style.go268-323](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L268-L323)

The method begins by:

1.  Joining all input strings with the pre-set `value` field
2.  Extracting all property values from the style configuration
3.  Initializing ANSI style objects (`te`, `teSpace`, `teWhitespace`)
4.  Computing derived values like border sizes

| Variable | Purpose | Lines |
| --- | --- | --- |
| `te` | Primary ANSI style for regular text | 276 |
| `teSpace` | ANSI style for space characters when `underlineSpaces`/`strikethroughSpaces` differ | 277 |
| `teWhitespace` | ANSI style for padding/margin whitespace when `colorWhitespace` or `reverse` is set | 278 |

**Sources:** [style.go268-323](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L268-L323)

Pipeline Stages
---------------

### Stage 1: Text Transformation

The first stage applies text transformations before any styling:

[style.go320-327](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L320-L327)

**Transform Function:** If set via `Style.Transform()`, the custom transform function executes first, allowing arbitrary string manipulation.

**Tab Conversion:** [style.go528-541](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L528-L541)

*   Default: Tabs convert to 4 spaces
*   Configurable via `Style.TabWidth(n)`
*   Special value `-1` (`NoTabConversion`) preserves tabs

**Carriage Return Removal:** [style.go400](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L400-L400)

*   Converts `\r\n` to `\n` for consistent line handling

**Inline Mode:** [style.go403-405](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L403-L405)

*   If `inline` property is set, strips all newlines
*   Forces single-line rendering

**Sources:** [style.go320-327](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L320-L327)
 [style.go398-405](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L398-L405)
 [style.go528-541](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L528-L541)

### Stage 2: ANSI Style Construction

The pipeline constructs ANSI styling objects by applying text attributes and colors:

[style.go333-395](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L333-L395)

The ANSI style construction follows this pattern:

| Attribute | Method Called | Lines |
| --- | --- | --- |
| Bold | `te.Bold()` | 333-335 |
| Italic | `te.Italic(true)` | 336-338 |
| Underline | `te.Underline(true)` + `te.UnderlineStyle(s.ul)` | 339-341, 383-385 |
| Reverse | `te.Reverse(true)` + `teWhitespace.Reverse(true)` | 342-345 |
| Blink | `te.Blink(true)` | 346-348 |
| Faint | `te.Faint()` | 349-351 |
| Foreground | `te.ForegroundColor(fg)` | 353-361 |
| Background | `te.BackgroundColor(bg)` | 363-371 |
| Underline Color | `te.UnderlineColor(ul)` | 373-381 |
| Strikethrough | `te.Strikethrough(true)` | 386-388 |

**Space Styling Logic:** [style.go310-395](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L310-L395)

When `underlineSpaces` or `strikethroughSpaces` is set, the pipeline uses a separate `teSpace` style object to apply different formatting to space characters versus regular text.

**Sources:** [style.go333-395](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L333-L395)

### Stage 3: Word Wrapping

Word wrapping applies only when not in inline mode and when a width is specified:

[style.go411-415](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L411-L415)

The wrap width calculation accounts for:

*   Specified width from `Style.Width()`
*   Left and right padding values
*   Horizontal border size (already subtracted at line 408-409)

**Sources:** [style.go407-415](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L407-L415)

### Stage 4: Core Text Rendering

The core text rendering stage applies ANSI styling to each character or line:

[style.go417-447](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L417-L447)

**Character-Level vs Line-Level Styling:**

When `useSpaceStyler` is true (line 318), the pipeline processes each rune individually to apply different styling to spaces:

[style.go428-440](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L428-L440)

Otherwise, it applies styling to entire lines at once for better performance.

**Hyperlink Wrapping:** [style.go444-446](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L444-L446)

If a hyperlink is set via `Style.Hyperlink()`, the entire styled text is wrapped with `ansi.SetHyperlink()` and `ansi.ResetHyperlink()` sequences.

**Sources:** [style.go417-447](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L417-L447)

### Stage 5: Padding Application

Padding is applied after text styling but before alignment:

[style.go449-478](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L449-L478)

**Padding Implementation:**

| Direction | Function | Lines |
| --- | --- | --- |
| Left | `padLeft()` | 455-461 |
| Right | `padRight()` | 463-469 |
| Top | `strings.Repeat("\n", topPadding)` | 471-473 |
| Bottom | `strings.Repeat("\n", bottomPadding)` | 475-477 |

The `pad()` function [style.go598-629](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L598-L629)
 applies styling to padding characters when `colorWhitespace` or `styleWhitespace` is enabled, using the `teWhitespace` style object.

**Padding Character:** [style.go451-454](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L451-L454)

Defaults to space ( ) but can be customized via `Style.PaddingChar()`. The padding character is preserved when text is copied and pasted.

**Sources:** [style.go449-478](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L449-L478)
 [style.go583-629](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L583-L629)

### Stage 6: Vertical Alignment

Height constraints and vertical alignment are applied after padding:

[style.go480-483](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L480-L483)

The `alignTextVertical` function (defined in [align.go](https://github.com/charmbracelet/lipgloss/blob/30441468/align.go)
) positions text within the specified height according to the `alignVertical` property (`Top`, `Center`, `Bottom`).

**Sources:** [style.go480-483](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L480-L483)

### Stage 7: Horizontal Alignment

Horizontal alignment is applied to ensure all lines are the same width:

[style.go485-498](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L485-L498)

This stage runs under two conditions:

1.  Multiple lines exist in the text (`numLines != 0`)
2.  A width constraint is specified (`width != 0`)

The `alignTextHorizontal` function (defined in [align.go](https://github.com/charmbracelet/lipgloss/blob/30441468/align.go)
) pads short lines with spaces (styled with `teWhitespace` if applicable) to match the longest line or specified width.

**Sources:** [style.go485-498](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L485-L498)

### Stage 8: Border Application

Borders are applied after alignment and padding but before margins:

[style.go500-503](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L500-L503)

The `applyBorder` method is defined in [borders.go](https://github.com/charmbracelet/lipgloss/blob/30441468/borders.go)
 and handles:

*   Border character rendering
*   Border foreground/background colors
*   Border gradient blending
*   Conditional border sides (top, right, bottom, left)

**Sources:** [style.go500-503](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L500-L503)

### Stage 9: Margin Application

Margins are applied after borders:

[style.go502](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L502-L502)

The `applyMargins` method [style.go543-580](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L543-L580)
 applies margins in all four directions using the `pad()` helper functions:

[style.go558-576](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L558-L576)

**Margin Background Color:** [style.go553-556](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L553-L556)

Margins can have a background color set via `Style.MarginBackground()`, which is applied to the margin whitespace using the ANSI style object.

**Sources:** [style.go502](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L502-L502)
 [style.go543-580](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L543-L580)

### Stage 10: Truncation

The final stage applies maximum width and height constraints:

**Maximum Width:** [style.go505-514](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L505-L514)

When `maxWidth` is set, each line is truncated using `ansi.Truncate()`, which properly handles ANSI sequences to ensure the visual width doesn't exceed the limit.

**Maximum Height:** [style.go516-523](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L516-L523)

When `maxHeight` is set, the output is limited to the specified number of lines by slicing the lines array.

**Sources:** [style.go505-523](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L505-L523)

Complete Execution Order
------------------------

The following table summarizes the complete pipeline execution order with dependencies:

| Stage | Operation | Condition | Lines |
| --- | --- | --- | --- |
| 0   | Join strings | Always | 268-274 |
| 1   | Transform function | If `transform` set | 325-327 |
| 2   | Tab conversion | Always | 398 |
| 3   | Carriage return removal | Always | 400 |
| 4   | Inline mode processing | If `inline` set | 403-405 |
| 5   | Border size calculation | Always | 408-409 |
| 6   | Word wrapping | If `!inline && width > 0` | 412-415 |
| 7   | ANSI style construction | Always | 333-395 |
| 8   | Core text rendering | Always | 417-447 |
| 9   | Left padding | If `!inline && leftPadding > 0` | 455-461 |
| 10  | Right padding | If `!inline && rightPadding > 0` | 463-469 |
| 11  | Top padding | If `!inline && topPadding > 0` | 471-473 |
| 12  | Bottom padding | If `!inline && bottomPadding > 0` | 475-477 |
| 13  | Vertical alignment | If `height > 0` | 481-483 |
| 14  | Horizontal alignment | If `numLines > 0 OR width > 0` | 488-497 |
| 15  | Border application | If `!inline` | 501 |
| 16  | Margin application | If `!inline` | 502 |
| 17  | Width truncation | If `maxWidth > 0` | 506-513 |
| 18  | Height truncation | If `maxHeight > 0` | 517-522 |

Special Cases and Optimizations
-------------------------------

### Early Exit for Empty Styles

[style.go329-331](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L329-L331)

If `s.props == 0` (no properties set), the pipeline returns early with only tab conversion applied, avoiding unnecessary processing.

### Inline Mode Bypass

When `inline` is set to `true`:

*   Margins are skipped
*   Padding is skipped
*   Borders are skipped
*   Newlines are removed from text

This optimization is used for rendering single-line text elements where block-level formatting is unnecessary.

**Sources:** [style.go306](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L306-L306)
 [style.go403-405](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L403-L405)
 [style.go450](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L450-L450)
 [style.go500](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L500-L500)

### Space Styling Optimization

[style.go318](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L318-L318)

The `useSpaceStyler` flag determines whether character-level iteration is required:

When false, the pipeline can style entire lines at once for better performance.

**Sources:** [style.go318](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L318-L318)
 [style.go428-440](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L428-L440)

### Color Whitespace Handling

[style.go305](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L305-L305)
 [style.go315](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L315-L315)

Two flags control how whitespace is styled:

*   `colorWhitespace`: Whether to apply background color to padding (default: true)
*   `styleWhitespace`: Whether to apply reverse styling to whitespace (set when `reverse` is enabled)

These flags determine which ANSI style object to use when rendering padding and margins.

**Sources:** [style.go305](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L305-L305)
 [style.go315](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L315-L315)
 [style.go457-459](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L457-L459)
 [style.go465-467](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L465-L467)

Pipeline Data Flow
------------------

The complete data flow through the rendering pipeline:

Each transformation produces a new string that serves as input to the next stage. The pipeline maintains immutability by building up the result through string concatenation and transformation rather than in-place modification.
