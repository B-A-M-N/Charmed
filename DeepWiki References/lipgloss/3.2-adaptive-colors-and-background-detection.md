**Purpose**: This document describes Lipgloss's adaptive color selection mechanisms that automatically adjust colors based on terminal capabilities and background darkness. This enables applications to provide optimal color choices for different terminal environments without requiring user configuration.

For basic color definitions and manipulation functions, see [Color System](https://deepwiki.com/charmbracelet/lipgloss/3.1-color-system)
. For automatic color downsampling during output, see [Terminal Adaptation and Output](https://deepwiki.com/charmbracelet/lipgloss/3.3-terminal-adaptation-and-output)
.

Overview
--------

Adaptive color selection in Lipgloss operates through two primary patterns:

| Pattern | Purpose | Key Types |
| --- | --- | --- |
| **LightDark** | Choose colors based on terminal background darkness | `LightDarkFunc` |
| **Complete** | Choose colors based on terminal capability profile | `CompleteFunc` |

Both patterns decouple color definition from runtime selection, allowing styles to be defined once and automatically adapt to different terminal environments.

Background Detection
--------------------

### Query Mechanism

The background detection system queries the terminal using ANSI escape sequences to determine if the background is light or dark. This information is then used by the `LightDark` pattern to select appropriate colors.

### Terminal Query Protocol

The low-level query mechanism in [terminal.go25-124](https://github.com/charmbracelet/lipgloss/blob/30441468/terminal.go#L25-L124)
 uses:

1.  **OSC 11** - Request background color escape sequence
2.  **Raw mode** - Terminal must be in raw mode to receive response
3.  **Timeout** - 2-second default timeout via [terminal.go49](https://github.com/charmbracelet/lipgloss/blob/30441468/terminal.go#L49-L49)
    
4.  **Parsing** - Uses `ansi.XParseColor` to convert response to `color.Color`

**Diagram: Terminal Query Protocol**

Sources: [terminal.go14-124](https://github.com/charmbracelet/lipgloss/blob/30441468/terminal.go#L14-L124)

### isDarkColor Implementation

The darkness determination in [color.go225-233](https://github.com/charmbracelet/lipgloss/blob/30441468/color.go#L225-L233)
 uses HSL color space:

1.  Converts `color.Color` to `colorful.Color`
2.  Extracts HSL components (Hue, Saturation, Luminance)
3.  Returns `true` if luminance < 0.5

Pattern Comparison
------------------

| Feature | LightDark | Complete | Compat AdaptiveColor | Compat CompleteColor |
| --- | --- | --- | --- | --- |
| **Background awareness** | Yes | No  | Yes | No  |
| **Profile awareness** | No  | Yes | No  | Yes |
| **Thread-safe** | Yes | Yes | No  | No  |
| **Requires detection** | Manual | Manual | Automatic (global) | Automatic (global) |
| **Flexibility** | High | High | Low | Low |
| **Recommended for new code** | Yes | Yes | No  | No  |

The function-based patterns (`LightDark` and `Complete`) are preferred because they:

1.  Avoid global state
2.  Support explicit dependency injection
3.  Allow runtime reconfiguration
4.  Work with any I/O streams

Best Practices
--------------

### When to Use Each Pattern

**Use `LightDark` when:**

*   Color choice depends on theme (light vs dark)
*   Terminal capability is irrelevant
*   Working with Bubble Tea applications

**Use `Complete` when:**

*   Need different colors for different terminal capabilities
*   Want to provide fallbacks for limited color support
*   Terminal background is irrelevant

**Use both patterns when:**

*   Need both background and capability awareness
*   Building professional TUI applications

### Error Handling

Background detection can fail for several reasons:

*   Redirected I/O streams
*   Terminals that don't support OSC 11
*   Permission issues on Windows

The `HasDarkBackground` function in [query.go83-89](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L83-L89)
 handles this by returning `true` on error, assuming a dark background as the safer default.

Sources: [query.go66-89](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L66-L89)

### Platform Considerations

**Windows**: The query mechanism automatically handles console redirection by opening `CONIN$` and `CONOUT$` when necessary (see [query.go35-54](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L35-L54)
).

**Unix/Linux**: Attempts query on both provided file descriptors, using the first successful result (see [query.go57-63](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L57-L63)
).

Sources: [query.go34-64](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L34-L64)
