Purpose and Scope
-----------------

Terminal adaptation ensures that styled output renders correctly across diverse terminal environments with varying capabilities. Lipgloss achieves this through automatic color downsampling, terminal capability detection, and specialized output writers. This system transparently handles the complexity of terminal differences, from modern 24-bit color terminals to legacy 16-color consoles.

This document covers the output writer system (`Print`, `Println`, `Fprint`, etc.), automatic color downsampling via `colorprofile.Writer`, terminal capability detection mechanisms, and platform-specific ANSI support. For details on color definitions and adaptive color selection, see page 3.1 and 3.2. For information on how terminal detection integrates with the rendering pipeline, see page 2.3.

Terminal Capability Detection
-----------------------------

Terminal adaptation requires detecting both color capabilities and background brightness. Lipgloss provides functions to query the terminal directly.

### Background Color Detection

The `BackgroundColor()` function queries the terminal's background color using ANSI escape sequences:

**Query Mechanism:**

1.  **Set Raw Mode**: Terminal input is set to raw mode to read escape sequences
2.  **Send OSC 11**: Write `\x1b]11;?\x1b\\` (OSC 11 query) to output
3.  **Wait for Response**: Read response with timeout (default 2 seconds)
4.  **Parse Response**: Extract RGB values from response format `rgb:RRRR/GGGG/BBBB`
5.  **Restore State**: Return terminal to normal mode

Sources: [query.go12-26](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L12-L26)
 [terminal.go14-47](https://github.com/charmbracelet/lipgloss/blob/30441468/terminal.go#L14-L47)

### Platform-Specific Handling

**Windows Console Detection:**

On Windows, when input/output is redirected, the function opens the console explicitly:

This handles scenarios where the application's stdin/stdout are redirected while the terminal is still accessible.

Sources: [query.go34-54](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L34-L54)

### HasDarkBackground Function

`HasDarkBackground()` wraps `BackgroundColor()` and determines brightness:

The function returns `true` by default if detection fails, assuming most developers use dark terminals.

**Brightness Calculation:**

The `isDarkColor()` function (from the color profile package) calculates relative luminance:

    L = 0.2126*R + 0.7152*G + 0.0722*B
    

If L < 0.5, the background is considered dark.

Sources: [query.go66-89](https://github.com/charmbracelet/lipgloss/blob/30441468/query.go#L66-L89)

### Query Terminal Timeout

Terminal queries use a timeout to prevent hanging on unresponsive terminals:

The timeout is implemented using a goroutine and `uv.NewCancelReader`, which supports cancellation on both Unix and Windows.

Sources: [terminal.go49-124](https://github.com/charmbracelet/lipgloss/blob/30441468/terminal.go#L49-L124)

Platform-Specific ANSI Support
------------------------------

Lipgloss provides platform-specific functions to enable ANSI support on legacy systems.

### Windows ANSI Enablement

**Platform Build Tags:**

| File | Build Constraint | Implementation |
| --- | --- | --- |
| `ansi_windows.go` | `//go:build windows` | Enables ANSI support via Windows Console API |
| `ansi_unix.go` | `//go:build !windows` | No-op (ANSI supported by default) |

**Windows Implementation:**

The function calls `windows.SetConsoleMode()` to enable the `ENABLE_VIRTUAL_TERMINAL_PROCESSING` flag, which enables ANSI escape sequence processing in cmd.exe and PowerShell. This is only necessary on Windows 10 and earlier; Windows Terminal supports ANSI by default.

Summary
-------

The Renderer component bridges lipgloss's styling abstractions with terminal realities through:

1.  **Color Profile Detection**: Automatically detects terminal color capabilities (Ascii → ANSI → ANSI256 → TrueColor)
2.  **Background Detection**: Determines dark vs. light backgrounds for adaptive color selection
3.  **Lazy Initialization**: Uses `sync.Once` to query terminal capabilities exactly once
4.  **Thread Safety**: `sync.RWMutex` enables safe concurrent access in multi-goroutine TUI applications
5.  **Customization**: Supports custom renderers for SSH, testing, and multi-output scenarios

Every `TerminalColor` receives the Renderer as context during resolution, enabling automatic color degradation and background adaptation. This architecture ensures lipgloss styles "just work" across diverse terminal environments while providing fine-grained control when needed.

Sources: [renderer.go1-182](https://github.com/charmbracelet/lipgloss/blob/30441468/renderer.go#L1-L182)
