Placement
=========

This document describes the placement system in lipgloss, which positions text blocks within defined rectangular dimensions by filling gaps with styled whitespace. Placement functions enable precise positioning of content horizontally, vertically, or in both dimensions simultaneously.

For measuring text dimensions before placement, see [Measurement Utilities](https://deepwiki.com/charmbracelet/lipgloss/5.3-placement#4.1)
. For composing multiple text blocks with alignment, see [Joining and Alignment](https://deepwiki.com/charmbracelet/lipgloss/5.3-placement#4.2)
.

* * *

Overview
--------

The placement system consists of three core functions that position strings within unstyled boxes of specified dimensions:

| Function | Dimensions | Description |
| --- | --- | --- |
| `Place` | 2D (width × height) | Positions text in both horizontal and vertical space |
| `PlaceHorizontal` | 1D (width) | Positions text horizontally, preserves original height |
| `PlaceVertical` | 1D (height) | Positions text vertically, preserves original width |

All placement operations fill gaps between content and target dimensions using a `whitespace` renderer that can apply foreground/background colors and custom characters.

**Sources:** [position.go34-154](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L34-L154)
 [README.md390-407](https://github.com/charmbracelet/lipgloss/blob/30441468/README.md?plain=1#L390-L407)

The Position Type
-----------------

### Position Representation

The `Position` type represents a normalized position along an axis, defined as a `float64` value between 0.0 (start) and 1.0 (end). A value of 0.5 represents the center.

**Position Constants and Their Numeric Values**

| Constant | Value | Axis Usage | Alignment |
| --- | --- | --- | --- |
| `Top` | 0.0 | Vertical | Align to top edge |
| `Bottom` | 1.0 | Vertical | Align to bottom edge |
| `Left` | 0.0 | Horizontal | Align to left edge |
| `Right` | 1.0 | Horizontal | Align to right edge |
| `Center` | 0.5 | Both | Center alignment |

The `value()` method [position.go21-23](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L21-L23)
 clamps any `Position` to the valid range using `math.Min(1, math.Max(0, float64(p)))`, ensuring invalid positions are automatically corrected.

**Sources:** [position.go10-32](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L10-L32)

Placement Functions
-------------------

### Function Signatures and Relationships

**Sources:** [position.go34-154](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L34-L154)

### Place: Two-Dimensional Positioning

The `Place` function positions text within a rectangular region by composing horizontal and vertical placement:

**Execution Flow:**

1.  First applies horizontal placement with `PlaceHorizontal(width, hPos, str, opts...)`
2.  Takes the horizontally-placed result and applies vertical placement
3.  Returns the final positioned text block

This composition means horizontal placement happens first, then vertical placement operates on the full horizontally-placed block.

**Sources:** [position.go34-44](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L34-L44)

### PlaceHorizontal: Horizontal Positioning

The `PlaceHorizontal` function aligns text within a specified width by adding whitespace padding on the left, right, or both sides.

**Algorithm** [position.go56-98](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L56-L98)
:

1.  **Measure Content:** Call `getLines(str)` to get all lines and maximum content width
2.  **Calculate Gap:** `gap = width - contentWidth`
3.  **Early Exit:** If `gap <= 0`, return original string unchanged
4.  **Create Whitespace Renderer:** `ws = newWhitespace(r, opts...)`
5.  **Process Each Line:**
    *   Calculate shortfall: `short = contentWidth - ansi.StringWidth(line)`
    *   Apply position-based rendering:
        *   **Left** (`pos == 0.0`): `line + whitespace(gap + short)`
        *   **Right** (`pos == 1.0`): `whitespace(gap + short) + line`
        *   **Other positions**: Split gap proportionally using `split = round(totalGap * pos.value())`

**Position Calculation for Center Alignment:**

When position is neither left nor right, the gap is split proportionally:

**Sources:** [position.go46-98](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L46-L98)

### PlaceVertical: Vertical Positioning

The `PlaceVertical` function aligns text within a specified height by adding empty lines above, below, or both.

**Algorithm** [position.go110-154](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L110-L154)
:

1.  **Measure Content Height:** Count newlines: `contentHeight = strings.Count(str, "\n") + 1`
2.  **Calculate Gap:** `gap = height - contentHeight`
3.  **Early Exit:** If `gap <= 0`, return original string unchanged
4.  **Create Whitespace Renderer:** `ws = newWhitespace(r, opts...)`
5.  **Get Content Width:** Use `getLines(str)` to determine line width
6.  **Generate Empty Line:** `emptyLine = ws.render(width)`
7.  **Apply Position-Based Rendering:**
    *   **Top** (`pos == 0.0`): Content first, then `gap` empty lines
    *   **Bottom** (`pos == 1.0`): `gap` empty lines, then content
    *   **Other positions**: Split gap proportionally

**Empty Line Generation:**

Each empty line is a full-width whitespace rendering that matches the content width, ensuring consistent block dimensions:

**Sources:** [position.go100-154](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L100-L154)

Integration with Layout System
------------------------------

### Dependency on Measurement Utilities

Placement functions rely on measurement utilities to determine content dimensions:

**Key Measurement Dependencies:**

1.  **`getLines(str)`:** Used by both horizontal and vertical placement to determine content dimensions and split text into lines
2.  **`ansi.StringWidth()`:** Used throughout whitespace rendering to accurately measure cell width, accounting for ANSI escape codes and wide characters
3.  **`strings.Count(str, "\n")`:** Used by `PlaceVertical` to count content height

**Sources:** [position.go57](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L57-L57)

### Relationship to Style Rendering

While placement functions create unstyled boxes with styled whitespace, the `Style.Render` method uses similar positioning logic internally for padding, margins, and alignment:

**Distinction:**

*   **Placement functions** create positioning in **unstyled** boxes (no borders, no inherent styling on content)
*   **Style.Render** applies positioning as part of a **styled** rendering pipeline (with borders, colors, text attributes)

Placement functions are utility primitives that can be used independently or combined with styled components.

Position Value Calculations
---------------------------

### Proportional Gap Splitting

Both horizontal and vertical placement use proportional gap splitting for non-edge positions:

**Horizontal Splitting** [position.go83-85](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L83-L85)
:

**Vertical Splitting** [position.go140-142](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L140-L142)
:

**Example Position Calculations:**

| Position Value | Description | Left/Top Gap | Right/Bottom Gap |
| --- | --- | --- | --- |
| 0.0 (Left/Top) | Align to start | 0 cells | All gap cells |
| 0.25 | Quarter from start | 25% of gap | 75% of gap |
| 0.5 (Center) | Centered | 50% of gap | 50% of gap |
| 0.75 | Three-quarters | 75% of gap | 25% of gap |
| 1.0 (Right/Bottom) | Align to end | All gap cells | 0 cells |

**Rounding Behavior:**

The use of `math.Round` ensures that when the gap is odd, the split is deterministic. For example, with a gap of 7 and position 0.5:

*   `split = round(7 * 0.5) = round(3.5) = 4`
*   `left = 7 - 4 = 3`
*   `right = 7 - 3 = 4`

This results in slightly more space on the right/bottom side for center-aligned odd gaps.
