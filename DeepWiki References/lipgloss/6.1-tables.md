Purpose and Scope
-----------------

The table component provides a flexible, styled table renderer for terminal output. It supports headers, customizable borders, per-cell styling, dynamic resizing, and content wrapping. Tables are built using a declarative API similar to the core Lipgloss styling system.

For general layout primitives, see [Layout and Composition](https://deepwiki.com/charmbracelet/lipgloss/5-layout-and-composition)
. For list rendering, see [Lists](https://deepwiki.com/charmbracelet/lipgloss/6.2-lists)
. For tree rendering, see [Trees](https://deepwiki.com/charmbracelet/lipgloss/6.3-trees)
.

Core Architecture
-----------------

The table system is organized around three main components: the `Table` struct that manages configuration and rendering, a `Data` interface that abstracts data access, and a resizing system that handles dimension calculations.

Data Model
----------

Tables consume data through the `Data` interface, which provides indexed access to cells. The package includes two built-in implementations: `StringData` for direct row/column data and `Filter` for filtered views.

| Interface Method | Description |
| --- | --- |
| `At(row, cell int)` | Returns the content of a specific cell |
| `Rows()` | Returns total number of rows |
| `Columns()` | Returns total number of columns |

**Sources:** [table/rows.go3-13](https://github.com/charmbracelet/lipgloss/blob/30441468/table/rows.go#L3-L13)

### StringData

The default data implementation stores rows as string slices. Columns are automatically counted based on the longest row.

**Sources:** [table/rows.go21-37](https://github.com/charmbracelet/lipgloss/blob/30441468/table/rows.go#L21-L37)

### Filter

The `Filter` type wraps another `Data` implementation and applies a predicate function to determine which rows are visible.

Table Configuration
-------------------

### Creating Tables

Tables are created using `New()` which returns a `*Table` with default settings: `NormalBorder()`, all borders enabled, wrapping enabled, and `DefaultStyles` for styling.

**Sources:** [table/table.go75-92](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L75-L92)

### Setting Data

Data can be provided through three methods:

| Method | Description |
| --- | --- |
| `Data(data Data)` | Sets data using the `Data` interface |
| `Rows(rows ...[]string)` | Appends multiple rows to `StringData` |
| `Row(row ...string)` | Appends a single row to `StringData` |

**Sources:** [table/table.go123-151](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L123-L151)

### Headers

Headers are set using `Headers(headers ...string)` and are rendered in row index `-1` (the `HeaderRow` constant). Headers are always truncated to fit within column widths and never wrap.

**Sources:** [table/table.go153-162](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L153-L162)
 [table/table.go13](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L13-L13)

Dimension Management
--------------------

### Width and Height

| Method | Behavior |
| --- | --- |
| `Width(w int)` | Sets table width; triggers column resizing |
| `Height(h int)` | Sets maximum table height; enables overflow handling |
| `YOffset(o int)` | Sets vertical rendering offset for scrolling |

When no width is specified, the table calculates its width based on content.

**Sources:** [table/table.go253-283](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L253-L283)

### Resizing Algorithm

The resizing system automatically adjusts column widths to fit the specified table width. The algorithm operates in two modes:

**Expansion**: When the table is narrower than the target width, the algorithm iteratively finds the shortest non-fixed column and increments its width by 1 until the target width is reached.

**Shrinking**: When the table is wider than the target width, the algorithm uses a three-phase approach:

1.  Shrink columns that are very large (>= tableWidth/2)
2.  Shrink columns that exceed their median content width
3.  Shrink the largest remaining column

The median-based approach preserves data visibility by shrinking columns with excessive whitespace first.

**Sources:** [table/resizing.go11-108](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L11-L108)
 [table/resizing.go189-305](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L189-L305)

### Column Metadata

The resizer calculates per-column statistics to guide width allocation:

| Statistic | Description |
| --- | --- |
| `min` | Narrowest cell width in column |
| `max` | Widest cell width in column |
| `median` | Median cell width in column |
| `xPadding` | Horizontal padding from styles |
| `fixedWidth` | Fixed width from `Style.Width()` |

**Sources:** [table/resizing.go110-119](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L110-L119)
 [table/resizing.go144-187](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L144-L187)

### Height Management

When `Height()` is set, the table limits visible rows and displays an overflow indicator ("…") for the last visible row. The algorithm accounts for borders, headers, padding, and row separators to determine how many rows fit.

**Sources:** [table/resizing.go425-481](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L425-L481)
 [table/table.go475-546](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L475-L546)

Rendering Pipeline
------------------

### Rendering Flow

**Sources:** [table/table.go310-375](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L310-L375)

### Cell Rendering

Each cell is rendered by:

1.  Getting the style from `styleFunc(row, col)`
2.  Truncating content if `wrap` is false or for headers
3.  Applying the style with calculated width and height
4.  Joining cells horizontally with border characters

Cells inherit margins and padding from their `lipgloss.Style`, which affects the available space for content.

**Sources:** [table/table.go475-546](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L475-L546)
 [table/table.go548-561](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L548-L561)

### Content Wrapping

When `wrap` is true (default), cell content wraps within the calculated column width. The height of each row is determined by the tallest wrapped cell. Headers never wrap regardless of the setting.

**Sources:** [table/table.go302-308](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L302-L308)
 [table/resizing.go307-327](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L307-L327)

Advanced Features
-----------------

### Filtering Data

The `Filter` type enables dynamic row filtering without copying data:

The filter is transparent to the table—it only sees rows that pass the predicate.

**Sources:** [table/rows.go66-113](https://github.com/charmbracelet/lipgloss/blob/30441468/table/rows.go#L66-L113)
 [table/table\_test.go571-611](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L571-L611)

### Vertical Offset

`YOffset()` enables table scrolling by skipping the first N rows during rendering. When combined with `Height()`, this creates a scrollable table viewport.

**Sources:** [table/table.go273-282](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L273-L282)
 [table/table\_test.go112-126](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L112-L126)

### Visibility Tracking

After rendering, the table exposes which rows are visible:

| Method | Description |
| --- | --- |
| `FirstVisibleRowIndex()` | Index of first visible row |
| `LastVisibleRowIndex()` | Index of last visible row (-2 if all visible) |
| `VisibleRows()` | Count of visible rows |

These are useful for implementing scrolling UI logic.

**Sources:** [table/table.go284-300](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L284-L300)

### Empty Tables

Tables handle missing data gracefully:

*   Tables with headers but no rows render only the header row
*   Tables with no headers and no rows render an empty string
*   Rows with fewer cells than headers display empty cells

**Sources:** [table/table.go312-317](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L312-L317)
 [table/table\_test.go89-96](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L89-L96)
