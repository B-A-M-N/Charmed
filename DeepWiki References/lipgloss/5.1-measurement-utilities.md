Measurement Utilities
=====================

**Purpose:** This document describes the measurement utilities that provide ANSI-aware text dimension calculations for terminal rendering. These functions accurately measure the visual width and height of text blocks, accounting for ANSI escape sequences, Unicode grapheme clusters, and wide characters (CJK characters, emojis).

**Scope:** This page covers the three core measurement functions: `Width()`, `Height()`, and `Size()`. For information about using these measurements to compose text blocks, see [Joining and Alignment](https://deepwiki.com/charmbracelet/lipgloss/5.1-measurement-utilities#4.2)
 and [Placement](https://deepwiki.com/charmbracelet/lipgloss/5.1-measurement-utilities#4.3)
. For information about how these measurements integrate with style rendering, see [Core Style System](https://deepwiki.com/charmbracelet/lipgloss/2-core-style-system)
.

* * *

Overview
--------

The measurement utilities provide three public functions that calculate the visual dimensions of text strings:

| Function | Purpose | Returns |
| --- | --- | --- |
| `Width(str string)` | Calculates the maximum cell width across all lines | `int` (cell width) |
| `Height(str string)` | Counts the number of lines in the text | `int` (line count) |
| `Size(str string)` | Calculates both dimensions in a single call | `(width int, height int)` |

These functions are essential because standard Go string operations (`len()`, `len([]rune())`) do not account for:

*   ANSI escape sequences (which occupy bytes but no visual space)
*   Wide characters like Chinese/Japanese/Korean glyphs (占 = 2 cells)
*   Emoji and other grapheme clusters (👨‍👩‍👧‍👦 = 2 cells)

 [README.md370-388](https://github.com/charmbracelet/lipgloss/blob/30441468/README.md?plain=1#L370-L388)

Width Function
--------------

The `Width()` function calculates the maximum visual width of a text string in terminal cells.

### Signature

**Location:** [size.go15-24](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L15-L24)

### Implementation Strategy

**Sources:** [size.go15-24](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L15-L24)

### Behavior Details

1.  **Line Splitting:** The function splits the input string by `\n` characters [size.go16](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L16-L16)
    
2.  **Per-Line Measurement:** Each line is measured using `ansi.StringWidth()` from the `x/ansi` package [size.go17](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L17-L17)
    
3.  **Maximum Selection:** The function returns the width of the widest line [size.go18-20](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L18-L20)
    
4.  **ANSI Handling:** ANSI escape sequences are automatically stripped during measurement, so they contribute zero width

### Example Measurements

| String | Standard `len()` | `Width()` Result | Explanation |
| --- | --- | --- | --- |
| `"Hello"` | 5   | 5   | Simple ASCII |
| `"\x1b<FileRef file-url="https://github.com/charmbracelet/lipgloss/blob/30441468/1mHello\\x1b[0m\"` | 15  | 5   | ANSI bold codes ignored |\

Height Function\
---------------\
\
The `Height()` function calculates the number of lines in a text string by counting newline characters.\
\
### Signature\
\
**Location:** [size.go30-32](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L30-L32)\
\
### Implementation\
\
The implementation is straightforward:\
\
1.  Count all `\n` characters using `strings.Count()` [size.go31](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L31-L31)\
    \
2.  Add 1 to account for the final line (which may not have a trailing newline) [size.go31](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L31-L31)\
    \
\
### Behavior Details\
\
*   **Single Line:** A string with no newlines returns `1` [size.go31](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L31-L31)\
    \
*   **Empty String:** An empty string returns `1` (considered as one empty line)\
*   **Windows Line Endings:** The function only counts `\n` characters. If your input uses `\r\n` (CRLF) line endings, you should convert them to `\n` (LF) first [size.go27-28](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L27-L28)\
    \
*   **ANSI Sequences:** Height measurement is independent of ANSI sequences\
\
### Example Measurements\
\
| String | `Height()` Result |\
| --- | --- |\
| `"Hello"` | 1   |\
| `"Line1\nLine2"` | 2   |\
| `"A\nB\nC\n"` | 4 (3 lines + final empty) |\
| `""` | 1   |\
\
**Sources:** [size.go26-32](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L26-L32)\

ANSI Sequence Handling\
----------------------\
\
The measurement utilities rely on the `x/ansi` package to handle ANSI escape sequences correctly.\
\
### Why ANSI-Awareness Matters\
\
ANSI escape sequences control text formatting and colors but occupy bytes without consuming visual terminal cells:\
\
    "\x1b[1;31mHello\x1b[0m"\
     ^^^^^^^ Bold+Red   ^^^^^ Reset\
             ^^^^^^ The actual visible text (5 cells)\
    \
\
Standard Go functions would count this as:\
\
*   `len()`: 19 bytes\
*   `len([]rune())`: 13 runes (including escape sequence characters)\
\
But visually, it only occupies **5 terminal cells**.\
\
### Integration with x/ansi\
\
The `ansi.StringWidth()` function [size.go17](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L17-L17)\
:\
\
1.  Parses and removes ANSI escape sequences\
2.  Calculates the cell width of remaining characters\
3.  Accounts for wide characters (CJK, emoji) that occupy 2 cells\
\
**Sources:** [size.go6-7](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L6-L7)\
 [size.go17](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L17-L17)\

Wide Character Support\
----------------------\
\
### Unicode Grapheme Clusters\
\
The measurement utilities properly handle characters that occupy more than one terminal cell:\
\
| Character Type | Example | Cell Width | Bytes | Runes |\
| --- | --- | --- | --- | --- |\
| ASCII | `A` | 1   | 1   | 1   |\
| Latin Extended | `é` | 1   | 2   | 1   |\
| CJK | `中` | 2   | 3   | 1   |\
| Emoji | `😀` | 2   | 4   | 1   |\
| Emoji Sequence | `👨‍👩‍👧‍👦` | 2   | 25  | 7   |\
\
### Why Standard Functions Fail\
\
The `Width()` function delegates to `ansi.StringWidth()`, which uses the `displaywidth` library to correctly calculate cell widths according to Unicode's East Asian Width properties.\
\
**Sources:** [size.go9-14](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L9-L14)\
 [README.md728-734](https://github.com/charmbracelet/lipgloss/blob/30441468/README.md?plain=1#L728-L734)\

Performance Characteristics\
---------------------------\
\
### Time Complexity\
\
| Function | Complexity | Explanation |\
| --- | --- | --- |\
| `Width()` | O(n) | Must scan entire string and parse ANSI sequences |\
| `Height()` | O(n) | Single pass counting `\n` characters |\
| `Size()` | O(n) | Delegates to `Width()` and `Height()` |\
\
Where `n` is the byte length of the input string.\
\
### Caching Considerations\
\
The measurement functions are pure functions with no side effects. If you need to measure the same text multiple times, consider caching the results:\
\
**Sources:** [size.go1-42](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L1-L42)\

Limitations and Edge Cases\
--------------------------\
\
### Windows Line Endings\
\
The `Height()` function only counts `\n` characters. If your text uses Windows-style `\r\n` line endings, you should normalize them first:\
\
**Sources:** [size.go27-28](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L27-L28)\
\
### Terminal Locale Configuration\
\
Wide character width calculations depend on terminal locale settings. If you encounter misalignment issues with CJK characters, set the `RUNEWIDTH_EASTASIAN=0` environment variable as documented in the FAQ [README.md728-734](https://github.com/charmbracelet/lipgloss/blob/30441468/README.md?plain=1#L728-L734)\
\
### Empty Strings\
\
An empty string has a height of `1` (representing one empty line) and a width of `0`:\
\
**Sources:** [size.go30-32](https://github.com/charmbracelet/lipgloss/blob/30441468/size.go#L30-L32)\
