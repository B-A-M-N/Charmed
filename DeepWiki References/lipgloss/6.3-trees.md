Purpose and Scope
-----------------

This document covers the Tree component, a sub-package of Lipgloss for rendering hierarchical data structures with automatic indentation, branch characters, and custom styling. Trees provide a visual representation of parent-child relationships, commonly used for file system displays, organizational charts, or any nested data structure.

For rendering tabular data, see [Tables](https://deepwiki.com/charmbracelet/lipgloss/6.1-tables)
. For linear lists with optional nesting, see [Lists](https://deepwiki.com/charmbracelet/lipgloss/6.2-lists)
. For general layout and composition techniques, see [Layout and Composition](https://deepwiki.com/charmbracelet/lipgloss/5-layout-and-composition)
.

**Sources:** [README.md698-797](https://github.com/charmbracelet/lipgloss/blob/30441468/README.md?plain=1#L698-L797)

* * *

Core Concepts
-------------

The Tree component is built around three fundamental elements:

| Element | Description | Configuration Method |
| --- | --- | --- |
| **Root** | The top-level node of the tree | `tree.Root(string)` |
| **Children** | Nodes beneath the root or other nodes | `.Child(...)` |
| **Enumerator** | Visual connector characters (├──, └──, etc.) | `.Enumerator(EnumeratorFunc)` |

Trees support arbitrary nesting depth, where any child can itself be a tree with its own children. The component automatically handles indentation and branch character rendering based on the tree structure.

**Sources:** [README.md706-722](https://github.com/charmbracelet/lipgloss/blob/30441468/README.md?plain=1#L706-L722)

Basic Tree Creation
-------------------

### Simple Trees

A basic tree is created using `tree.Root()` followed by `.Child()` calls:

**Output:**

    .
    ├── A
    ├── B
    └── C
    

The `Child()` method accepts variadic arguments, where each argument becomes a child node at the same level. String arguments are rendered as simple text nodes.

-------------------------

Trees can be built incrementally using the `Child()` method repeatedly:

Tree vs List vs Table
---------------------

| Feature | Tree | List | Table |
| --- | --- | --- | --- |
| **Structure** | Hierarchical with nesting | Linear with optional nesting | Rows and columns |
| **Visual Connectors** | Branch characters (├──, └──) | Bullets or enumerators | Borders |
| **Nesting** | Deep hierarchies | Shallow nesting | None |
| **Use Case** | File systems, org charts | Ordered/unordered lists | Tabular data |
| **Package** | `tree/` | `list/` | `table/` |

Trees are optimized for representing parent-child relationships with visual depth indication. For flat or lightly nested content, Lists may be more appropriate. For structured data with multiple attributes per item, Tables are the better choice.

Implementation Details
----------------------

While the source code for the tree package is not included in the provided files, the architecture follows established patterns from the Lipgloss ecosystem:

1.  **Builder Pattern**: Chainable methods for fluent API design
2.  **Style Delegation**: Uses `lipgloss.Style` for all visual formatting
3.  **Functional Configuration**: Enumerator functions for customization
4.  **Composition**: Trees can contain other trees as children
5.  **Immutability**: Methods return new tree instances rather than mutating

The tree rendering likely follows these steps internally:

1.  Traverse the tree structure depth-first
2.  Track current depth and parent-child relationships
3.  Apply enumerator function to generate branch characters
4.  Apply appropriate styles (root, item, enumerator)
5.  Concatenate styled text with proper indentation
6.  Return final ANSI-escaped string
