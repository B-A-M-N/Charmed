Joining and Alignment
=====================

This document describes the joining and alignment primitives in Lipgloss for arranging multiple text blocks together. These utilities enable horizontal and vertical composition of styled strings with precise control over alignment positioning.

For text measurement utilities used by these functions, see [Measurement Utilities](https://deepwiki.com/charmbracelet/lipgloss/5.1-measurement-utilities)
. For placing single text blocks within defined areas, see [Placement](https://deepwiki.com/charmbracelet/lipgloss/5.3-placement)
. For advanced multi-layer composition, see [Canvas and Layer System](https://deepwiki.com/charmbracelet/lipgloss/5.4-canvas-and-layer-system)
.

Overview
--------

Lipgloss provides two primary joining functions that combine multiple text blocks:

| Function | Purpose | Position Meaning |
| --- | --- | --- |
| `JoinHorizontal` | Arranges blocks side-by-side | Vertical alignment (0=top, 0.5=center, 1=bottom) |
| `JoinVertical` | Stacks blocks vertically | Horizontal alignment (0=left, 0.5=center, 1=right) |

Both functions use the `Position` type to specify alignment along the perpendicular axis. The alignment determines how blocks of different sizes are positioned relative to each other.

**Sources:** [join.go1-176](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L1-L176)
 [position.go10-32](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L10-L32)

Position Type and Constants
---------------------------

The `Position` type represents a fractional position along an axis, with values clamped between 0 and 1:

Lipgloss defines five position constants for common alignments:

| Constant | Value | Horizontal Meaning | Vertical Meaning |
| --- | --- | --- | --- |
| `Top` | 0.0 | N/A | Top edge |
| `Left` | 0.0 | Left edge | N/A |
| `Center` | 0.5 | Horizontal center | Vertical center |
| `Bottom` | 1.0 | N/A | Bottom edge |
| `Right` | 1.0 | Right edge | N/A |

Positions can also be specified as arbitrary float values between 0 and 1 for custom alignment positions (e.g., `0.25` for 25% from the start).

**Sources:** [position.go10-32](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L10-L32)

JoinHorizontal: Side-by-Side Composition
----------------------------------------

The `JoinHorizontal` function arranges text blocks side-by-side with vertical alignment control.

### Function Signature

**Parameters:**

*   `pos`: Vertical alignment position (0=top, 0.5=center, 1=bottom)
*   `strs`: Variable number of strings to join

### Operation Details

**Diagram: JoinHorizontal Processing Stages**

The function performs these steps:

1.  **Line Extraction** [join.go48-53](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L48-L53)
    : Breaks each block into lines and calculates maximum width per block
2.  **Height Normalization** [join.go56-79](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L56-L79)
    : Adds empty lines to shorter blocks based on position
3.  **Width Padding** [join.go82-93](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L82-L93)
    : Pads each line to its block's maximum width using spaces
4.  **Horizontal Merge** [join.go82-95](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L82-L95)
    : Concatenates corresponding lines from each block

### Position Handling

For blocks shorter than the tallest:

*   `Top` (0.0): Extra lines added at bottom [join.go64-65](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L64-L65)
    
*   `Bottom` (1.0): Extra lines added at top [join.go67-68](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L67-L68)
    
*   Custom (0.0-1.0): Lines split proportionally [join.go70-78](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L70-L78)
    

**Sources:** [join.go10-96](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L10-L96)

JoinVertical: Vertical Stacking
-------------------------------

The `JoinVertical` function stacks text blocks vertically with horizontal alignment control.

### Function Signature

**Parameters:**

*   `pos`: Horizontal alignment position (0=left, 0.5=center, 1=right)
*   `strs`: Variable number of strings to join

### Operation Details

**Diagram: JoinVertical Processing Flow**

The function performs these steps:

1.  **Width Calculation** [join.go129-135](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L129-L135)
    : Determines maximum width across all blocks
2.  **Line Processing** [join.go138-172](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L138-L172)
    : Processes each line of each block
3.  **Gap Calculation** [join.go140](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L140-L140)
    : Computes whitespace needed for alignment
4.  **Position Application** [join.go142-164](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L142-L164)
    : Adds whitespace based on position

### Position Handling

For each line:

*   `Left` (0.0): Line followed by spaces [join.go143-145](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L143-L145)
    
*   `Right` (1.0): Spaces followed by line [join.go147-149](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L147-L149)
    
*   Custom (0.0-1.0): Spaces split proportionally [join.go151-163](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L151-L163)
    

**Sources:** [join.go98-175](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L98-L175)

**Diagram: alignTextHorizontal Internal Logic**

This function aligns multi-line text horizontally and normalizes line widths:

1.  **Line Analysis** [align.go13](https://github.com/charmbracelet/lipgloss/blob/30441468/align.go#L13-L13)

Helper Functions and Utilities
------------------------------

### getLines Function

Both joining functions rely on an internal `getLines` helper that:

*   Splits strings by newlines
*   Calculates maximum visual width using `ansi.StringWidth`
*   Returns both line array and maximum width

This function is ANSI-aware, meaning it correctly measures text width excluding escape sequences.

**Sources:** [join.go49](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L49-L49)
 [align.go13](https://github.com/charmbracelet/lipgloss/blob/30441468/align.go#L13-L13)

### Width Calculation

All functions use `ansi.StringWidth` from the `github.com/charmbracelet/x/ansi` package to measure visual width. This ensures correct handling of:

*   Wide Unicode characters (e.g., CJK characters)
*   ANSI escape sequences (which have zero visual width)

**Sources:** [join.go88](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L88-L88)
 [join.go140](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L140-L140)
 [align.go17](https://github.com/charmbracelet/lipgloss/blob/30441468/align.go#L17-L17)

### Empty Input Handling

*   Zero strings: Returns empty string [join.go29-31](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L29-L31)
     [join.go117-119](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L117-L119)

     [join.go120-122](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L120-L122)
    

### Position Clamping

The `Position.value()` method clamps values to \[0, 1\] range [position.go21-23](https://github.com/charmbracelet/lipgloss/blob/30441468/position.go#L21-L23)
:

*   Negative values → 0.0
*   Values > 1.0 → 1.0

### Wide Character Handling

All width calculations use `ansi.StringWidth`, which correctly handles:

*   East Asian wide characters (count as 2)
*   Zero-width characters
*   ANSI escape sequences (zero width)

**Sources:** [join.go29-34](https://github.com/charmbracelet/lipgloss/blob/30441468/join.go#L29-L34)

Integration with Style System
-----------------------------

While alignment functions in [align.go](https://github.com/charmbracelet/lipgloss/blob/30441468/align.go)
 are internal, they are called during the Style rendering pipeline. These functions receive a `*ansi.Style` parameter to apply styling to padding whitespace, ensuring visual consistency.

The public joining functions do not apply styles themselves but work with pre-rendered styled strings. Users typically render strings with styles before joining:
