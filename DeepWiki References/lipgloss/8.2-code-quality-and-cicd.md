Code Quality and CI/CD
======================

Relevant source files

*   [.github/workflows/build.yml](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml)
    
*   [.github/workflows/coverage.yml](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml)

This document describes the automated code quality and continuous integration/continuous deployment (CI/CD) infrastructure for the Lipgloss codebase. It covers the GitHub Actions workflows, linting configuration, code coverage requirements, and quality standards enforced on every push and pull request.

For information about the testing infrastructure and test organization, see [Testing Infrastructure](https://deepwiki.com/charmbracelet/lipgloss/8.1-testing-infrastructure)
. For details about module structure and dependencies, see [Module Architecture and Dependencies](https://deepwiki.com/charmbracelet/lipgloss/9-module-architecture-and-dependencies)
.

* * *

CI/CD Pipeline Overview
-----------------------

The Lipgloss repository implements a comprehensive CI/CD pipeline using GitHub Actions with three primary workflows that run on every push and pull request:

| Workflow | Purpose | Trigger | Configuration |
| --- | --- | --- | --- |
| `lint` | Static code analysis | push, pull\_request | [.github/workflows/lint.yml1-13](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/lint.yml#L1-L13) |
| `coverage` | Test coverage reporting | push, pull\_request | [.github/workflows/coverage.yml1-33](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L1-L33) |
| `build` | Build verification | push to master, pull\_request | [.github/workflows/build.yml1-14](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L1-L14) |

All workflows must pass for a pull request to be mergeable, establishing a zero-tolerance policy for code quality violations.

**Sources:** [.github/workflows/lint.yml1-13](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/lint.yml#L1-L13)
 [.github/workflows/coverage.yml1-33](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L1-L33)
 [.github/workflows/build.yml1-14](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L1-L14)

Workflow Architecture
---------------------

**Workflow Integration Pattern**: The Lipgloss repository uses a centralized workflow pattern where `lint` and `build` workflows delegate to shared workflows in the `charmbracelet/meta` repository. This ensures consistent quality standards across all Charmbracelet projects while allowing project-specific configuration through parameters.

**Sources:** [.github/workflows/lint.yml1-13](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/lint.yml#L1-L13)
 [.github/workflows/coverage.yml1-33](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L1-L33)
 [.github/workflows/build.yml1-14](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L1-L14)

Code Coverage Workflow
----------------------

The coverage workflow runs tests with race detection and generates coverage reports that are uploaded to Coveralls.io for tracking and visualization.

### Coverage Collection Process

**Test Execution Command**: The workflow executes tests with three critical flags:

*   `-race`: Enables data race detector to catch concurrent access bugs [.github/workflows/coverage.yml30](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L30-L30)
    
*   `-covermode atomic`: Uses atomic coverage mode for thread-safe coverage counting [.github/workflows/coverage.yml30](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L30-L30)
    
*   `-coverprofile=profile.cov`: Outputs coverage data to `profile.cov` [.github/workflows/coverage.yml30](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L30-L30)
    

**Sources:** [.github/workflows/coverage.yml26-32](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L26-L32)

### Workflow Configuration

The coverage workflow runs on a matrix strategy but currently only uses a single configuration:

| Parameter | Value | Lines |
| --- | --- | --- |
| Go Version | `^1` (latest Go 1.x) | [.github/workflows/coverage.yml8](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L8-L8) |
| OS  | `ubuntu-latest` | [.github/workflows/coverage.yml9](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L9-L9) |
| Go Module Mode | `on` (GO111MODULE) | [.github/workflows/coverage.yml12](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L12-L12) |

**Authentication Setup**: The workflow configures Git to use a personal access token for accessing private Charmbracelet repositories:

This configuration allows the test suite to fetch dependencies from private Charmbracelet organization repositories [.github/workflows/coverage.yml22-24](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L22-L24)

**Sources:** [.github/workflows/coverage.yml6-24](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L6-L24)

Build Workflow
--------------

The build workflow verifies that the codebase can be successfully built and all tests pass. It runs on pushes to the `master` branch and on all pull requests.

### Build Configuration

The build workflow is minimal, delegating all implementation to a shared workflow:

**Key Configuration Points:**

*   **Shared Workflow**: Uses `charmbracelet/meta/.github/workflows/build.yml@main` [.github/workflows/build.yml11](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L11-L11)
    
*   **Authentication**: Passes `PERSONAL_ACCESS_TOKEN` secret for private repository access [.github/workflows/build.yml12-13](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L12-L13)
    
*   **Trigger Branches**: Only runs on `master` for pushes [.github/workflows/build.yml5-6](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L5-L6)
    
*   **PR Validation**: Runs on all pull requests [.github/workflows/build.yml7-8](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L7-L8)

Quality Gates and Standards
---------------------------

The CI/CD pipeline enforces strict quality standards that must be met before code can be merged:

### Quality Gate Requirements

| Gate | Requirement | Enforcement | Configuration |
| --- | --- | --- | --- |
| **Linting** | Zero linting violations | Hard block | [.golangci.yml44-45](https://github.com/charmbracelet/lipgloss/blob/30441468/.golangci.yml#L44-L45)<br> sets no limits |
| **Build** | Successful compilation | Hard block | [.github/workflows/build.yml7-8](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/build.yml#L7-L8) |
| **Tests** | All tests pass | Hard block | [.github/workflows/coverage.yml30](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L30-L30) |
| **Race Detection** | No data races | Hard block | [.github/workflows/coverage.yml30](https://github.com/charmbracelet/lipgloss/blob/30441468/.github/workflows/coverage.yml#L30-L30)<br> `-race` flag |
| **Coverage** | Coverage reported | Tracking only | Reported to Coveralls.io |

### Zero-Tolerance Policy

The linting configuration enforces a zero-tolerance policy through the following settings:

This means:

*   **No Issue Suppression**: All linting violations are reported, not just the first N [.golangci.yml44-45](https://github.com/charmbracelet/lipgloss/blob/30441468/.golangci.yml#L44-L45)
    
*   **No Duplicate Hiding**: Repeated violations are not collapsed [.golangci.yml44-45](https://github.com/charmbracelet/lipgloss/blob/30441468/.golangci.yml#L44-L45)
    
*   **Fail Fast**: Any violation blocks the merge
*   **Consistent Standards**: All code must meet the same quality bar

Integration with Development Workflow
-------------------------------------

The CI/CD system integrates seamlessly with the GitHub development workflow:

1.  **Pre-Commit**: Developers can run `golangci-lint run` locally using the same `.golangci.yml` configuration
2.  **Push**: All three workflows execute automatically
3.  **Pull Request**: Status checks appear on the PR page, blocking merge if any fail
4.  **Coverage Tracking**: Historical coverage data is available on Coveralls.io
5.  **Merge**: Only possible when all status checks pass
