Text Attributes and Styling
===========================

Relevant source files

*   [set.go](https://github.com/charmbracelet/lipgloss/blob/30441468/set.go)
    
*   [style.go](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go)

This page covers lipgloss's text formatting capabilities including bold, italic, underline, strikethrough, and other text attributes. It also documents rune-level styling with `StyleRunes` and string transformations via the `Transform` function.

For information about color application, see section 3 (Colors and Terminal Detection). For border styling, see section 4 (Borders).

* * *

Overview
--------

Lipgloss provides a comprehensive set of text attributes that can be applied to styled text. These attributes are implemented as boolean properties tracked by the `props` bitmask system and applied during the rendering pipeline. Text attributes can be applied globally to entire strings or selectively to specific runes using the `StyleRunes` function.

**Sources:** [style.go21-91](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L21-L91)
 [style.go142-195](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L142-L195)

Text Attribute Properties
-------------------------

The following text attributes are defined as property keys in the style system:

| Property Key | Description | ANSI Effect |
| --- | --- | --- |
| `boldKey` | Bold/bright text | Increases font weight or brightness |
| `italicKey` | Italic text | Slanted text (may render as reverse on some terminals) |
| `underlineKey` | Underlined text | Line beneath text |
| `strikethroughKey` | Strikethrough text | Line through text |
| `reverseKey` | Inverted colors | Swaps foreground and background colors |
| `blinkKey` | Blinking text | Text blinks (terminal support varies) |
| `faintKey` | Dimmed text | Reduced intensity foreground |
| `underlineSpacesKey` | Controls underline on spaces | Boolean modifier for underline behavior |
| `strikethroughSpacesKey` | Controls strikethrough on spaces | Boolean modifier for strikethrough behavior |

These properties are stored as bit flags in the `props` bitmask and as boolean values in the `attrs` integer field of the `Style` struct.

Rendering Pipeline for Text Attributes
--------------------------------------

When `Style.Render()` is called, text attributes are applied in a specific order as part of a multi-stage rendering pipeline:

The rendering process extracts attribute values using `getAsBool()` and builds `ansi.Style` instances (from the `github.com/charmbracelet/x/ansi` package) that accumulate ANSI escape codes. Attributes are applied by calling methods like `te.Bold()`, `te.Italic()`, etc., which return modified `ansi.Style` instances. The text is then styled using `te.Styled()` which wraps strings with the appropriate ANSI sequences.

**Sources:** [style.go267-526](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L267-L526)

Space Styling Control
---------------------

By default, underlines and strikethroughs are applied to all characters including spaces. The `UnderlineSpaces` and `StrikethroughSpaces` methods provide fine-grained control over this behavior.

### UnderlineSpaces

Controls whether underlines are drawn on spaces between words:

Default behavior: When `Underline(true)` is set, `UnderlineSpaces` defaults to `true`, meaning spaces are underlined. Setting `UnderlineSpaces(false)` explicitly disables underlining of spaces while keeping underlining on other characters.

### StrikethroughSpaces

Controls whether strikethroughs are drawn on spaces:

### Implementation Details

During rendering, these properties control whether a separate `teSpace` styler is used for space characters:

| Condition | Behavior |
| --- | --- |
| `underline && underlineSpaces` | Spaces are underlined with main style |
| `underline && !underlineSpaces` | Spaces use `teSpace` without underline |
| `strikethrough && strikethroughSpaces` | Spaces are struck through with main style |
| `strikethrough && !strikethroughSpaces` | Spaces use `teSpace` without strikethrough |

The rendering loop at [style.go418-442](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L418-L442)
 iterates through lines and runes, checking `unicode.IsSpace(r)` and applying the appropriate styler (`te` or `teSpace`) based on these settings.

Rune-Level Styling with StyleRunes
----------------------------------

The `StyleRunes` function enables applying different styles to specific runes within a string based on their indices.

### Function Signature

**Parameters:**

*   `str`: The input string
*   `indices`: Slice of rune indices to apply the matched style to
*   `matched`: Style to apply to runes at the specified indices
*   `unmatched`: Style to apply to all other runes

### Algorithm

1.  Converts the indices slice to a map for O(1) lookup (lines 12-15)
2.  Converts the string to a rune slice to handle multi-byte characters correctly (line 21)
3.  Iterates through runes, accumulating consecutive runes with the same match status (lines 24-40)
4.  When a boundary is detected (match status changes or end of string), flushes the accumulated group by rendering it with the appropriate style (lines 28-38)
5.  Returns the concatenated result

### Example Usage

This is particularly useful for search highlighting, syntax highlighting, or any scenario where specific characters need different styling.

String Transformations
----------------------

The `Transform` method allows applying arbitrary string transformations at render time. The transformation function is executed before any other rendering operations.

### Transform Method

Sets a transformation function that will be applied to the input string during rendering. The function receives the raw input string and returns a transformed string.

### Execution in Rendering Pipeline

The transform function is applied early in the rendering pipeline:

1.  Input strings are joined with `joinString()` [style.go269-274](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L269-L274)
    
2.  **Transform function is called** (if set) [style.go325-327](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L325-L327)
    
3.  Early return if no properties set [style.go329-330](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L329-L330)
    
4.  Text attributes and colors applied [style.go333-395](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L333-L395)
    
5.  Layout operations applied [style.go411-523](https://github.com/charmbracelet/lipgloss/blob/30441468/style.go#L411-L523)

Property Storage and Retrieval
------------------------------

Text attributes are stored using a dual-layer system:

### Storage Architecture

**Bitmask (`props`):** Each property key is a power of 2, allowing efficient bitwise operations to track which properties have been explicitly set. This enables the `Inherit()` method to selectively copy only unset properties.

**Attributes Field (`attrs`):** Text attributes (boolean properties) are stored as bits in the `attrs` integer. When setting a boolean attribute, the corresponding bit in `attrs` is set or cleared using bitwise operations.

### Key Methods

*   `Style.set(key, value)` - Sets a property, updates `props` bitmask and stores value [set.go9-108](https://github.com/charmbracelet/lipgloss/blob/30441468/set.go#L9-L108)
    
*   `Style.getAsBool(key, default)` - Retrieves boolean attribute value from `attrs` field [get.go](https://github.com/charmbracelet/lipgloss/blob/30441468/get.go)
    
*   `Style.isSet(key)` - Checks if property is set in `props` bitmask [get.go](https://github.com/charmbracelet/lipgloss/blob/30441468/get.go)
    
*   `Style.unset(key)` - Removes property from `props` bitmask [unset.go4-6](https://github.com/charmbracelet/lipgloss/blob/30441468/unset.go#L4-L6)

Unsetting Text Attributes
-------------------------

All text attribute properties can be removed using corresponding `Unset` methods:

| Method | Removes |
| --- | --- |
| `UnsetBold()` | Bold attribute |
| `UnsetItalic()` | Italic attribute |
| `UnsetUnderline()` | Underline attribute |
| `UnsetStrikethrough()` | Strikethrough attribute |
| `UnsetReverse()` | Reverse attribute |
| `UnsetBlink()` | Blink attribute |
| `UnsetFaint()` | Faint attribute |
| `UnsetUnderlineSpaces()` | Underline spaces control |
| `UnsetStrikethroughSpaces()` | Strikethrough spaces control |
| `UnsetTransform()` | Transform function |

Unsetting a property removes it from the `props` bitmask, causing it to be excluded during rendering and allowing it to be inherited from other styles.

Terminal Compatibility
----------------------

Text attributes rely on ANSI escape sequences generated by the `termenv` library. Terminal support varies:

| Attribute | Support Level |
| --- | --- |
| Bold | Universal (SGR 1) |
| Faint | Wide support (SGR 2) |
| Italic | Good (SGR 3, may render as reverse) |
| Underline | Universal (SGR 4) |
| Blink | Varies (SGR 5, often disabled) |
| Reverse | Universal (SGR 7) |
| Strikethrough | Modern terminals (SGR 9) |

The `Renderer` component handles profile detection and adapts output to terminal capabilities. For more information on terminal adaptation, see [Renderer and Terminal Adaptation](https://deepwiki.com/charmbracelet/lipgloss/3-colors-and-terminal-detection)
.
