This page demonstrates practical examples of using Lipgloss's structured components - tables, lists, and trees - with nested structures and advanced styling patterns. These examples illustrate real-world usage patterns from the codebase's test suite and example applications.

For component API documentation, see Tables ([6.1](https://github.com/charmbracelet/lipgloss/blob/30441468/6.1)
), Lists ([6.2](https://github.com/charmbracelet/lipgloss/blob/30441468/6.2)
), and Trees ([6.3](https://github.com/charmbracelet/lipgloss/blob/30441468/6.3)
). For basic styling concepts, see Core Style System ([2](https://github.com/charmbracelet/lipgloss/blob/30441468/2)
).

Table Component Examples
------------------------

The `table` package provides a flexible table renderer with support for headers, borders, styling functions, and dynamic resizing. The test suite and examples demonstrate various usage patterns.

### Basic Table Construction

A basic table uses method chaining to configure headers, rows, borders, and styling:

**Basic Table Construction Pattern**

The simplest table construction pattern is shown in the test suite [table/table\_test.go23-34](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L23-L34)
:

    table := New().
        Border(lipgloss.NormalBorder()).
        StyleFunc(TableStyle).
        Headers("LANGUAGE", "FORMAL", "INFORMAL").
        Row("Chinese", "Nǐn hǎo", "Nǐ hǎo").
        Row("French", "Bonjour", "Salut")
    

| Method | Purpose | Example |
| --- | --- | --- |
| `Headers()` | Set column headers | `Headers("Name", "Age", "City")` |
| `Row()` | Add single row | `Row("Alice", "30", "NYC")` |
| `Rows()` | Add multiple rows | `Rows([][]string{{"Bob", "25", "LA"}})` |
| `Border()` | Set border style | `Border(lipgloss.NormalBorder())` |
| `StyleFunc()` | Define cell styling logic | `StyleFunc(func(row, col int) lipgloss.Style {...})` |

Sources: table/table\_test.go, table/table.go

### StyleFunc Pattern for Dynamic Styling

The `StyleFunc` type [table/table.go15-37](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L15-L37)
 enables position-based and content-based styling:

**StyleFunc Decision Flow**

The `HeaderRow` constant (`-1`) identifies header cells [table/table.go13](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L13-L13)
 Common patterns include:

**Alternating row colors** [table/table\_test.go12-21](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L12-L21)
:

    StyleFunc(func(row, col int) lipgloss.Style {
        switch {
        case row == HeaderRow:
            return HeaderStyle
        case row%2 == 0:
            return EvenRowStyle
        default:
            return OddRowStyle
        }
    })
    

**Column-specific styling** [table/table\_test.go756-808](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L756-L808)
:

    StyleFunc(func(row, col int) lipgloss.Style {
        if row == HeaderRow {
            return lipgloss.NewStyle().Align(lipgloss.Center)
        }
        return lipgloss.NewStyle().Margin(0, 1).Align(lipgloss.Right)
    })
    

Sources: table/table.go, table/table\_test.go

### Border Configuration

Tables support granular border control through individual border component toggles:

| Method | Controls | Default |
| --- | --- | --- |
| `BorderTop()` | Top border | `true` |
| `BorderBottom()` | Bottom border | `true` |
| `BorderLeft()` | Left border | `true` |
| `BorderRight()` | Right border | `true` |
| `BorderHeader()` | Header separator | `true` |
| `BorderColumn()` | Column separators | `true` |
| `BorderRow()` | Row separators | `false` |

**Border Configuration Combinations**

Example with row separators [table/table\_test.go351-367](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L351-L367)
:

    table := New().
        Border(lipgloss.NormalBorder()).
        BorderRow(true).
        Headers("LANGUAGE", "FORMAL", "INFORMAL").
        Rows(rows...)
    

Example with inner borders only [table/table\_test.go240-257](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L240-L257)
:

    table := New().
        BorderTop(false).
        BorderRight(false).
        BorderBottom(false).
        BorderLeft(false).
        BorderRow(true).
        BorderColumn(true)
    

Sources: table/table.go, table/table\_test.go

### Width and Height Constraints

Tables support automatic sizing based on constraints [table/table.go253-308](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L253-L308)
:

**Width Behavior** [table/table\_test.go419-440](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L419-L440)
:

*   **Expand**: If content is narrower than `Width()`, columns expand evenly
*   **Shrink**: If content is wider than `Width()`, columns shrink intelligently using median-based algorithm [table/resizing.go23-46](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L23-L46)
    
*   **Smart crop**: Large columns are reduced first to minimize data truncation

**Height Behavior** [table/table\_test.go633-661](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L633-L661)
:

*   **Exact**: Table renders exactly to `Height()` if possible
*   **Truncate**: Excess rows are hidden with overflow indicator ("…")
*   **YOffset**: Scrolls table vertically when combined with `Height()` [table/table\_test.go112-126](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L112-L126)
    

**Table Resizing Algorithm**

The resizing algorithm [table/resizing.go47-108](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L47-L108)
 ensures optimal column widths by:

1.  Calculating median content width per column [table/resizing.go178-184](https://github.com/charmbracelet/lipgloss/blob/30441468/table/resizing.go#L178-L184)
    
2.  Shrinking columns with largest deviation from median first
3.  Expanding shorter columns when extra space is available

Sources: table/table.go, table/resizing.go, table/table\_test.go

### Data Abstraction and Filtering

Tables accept data through the `Data` interface [table/rows.go3-13](https://github.com/charmbracelet/lipgloss/blob/30441468/table/rows.go#L3-L13)
 enabling custom data sources:

**Data Interface Implementation**

**StringData** provides basic storage [table/rows.go15-63](https://github.com/charmbracelet/lipgloss/blob/30441468/table/rows.go#L15-L63)
:

    data := NewStringData().
        Item("Chinese", "Nǐn hǎo", "Nǐ hǎo").
        Item("French", "Bonjour", "Salut")
    

**Filter** enables data filtering [table/rows.go65-113](https://github.com/charmbracelet/lipgloss/blob/30441468/table/rows.go#L65-L113)
:

    filter := NewFilter(data).Filter(func(row int) bool {
        return data.At(row, 0) != "French"  // Exclude French row
    })
    table := New().Data(filter)
    

The filter pattern is demonstrated in tests [table/table\_test.go571-590](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L571-L590)
 showing how to dynamically exclude rows based on content.

Sources: table/rows.go, table/table\_test.go

### Content Wrapping and Truncation

Tables handle long content through wrapping or truncation [table/table.go302-308](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L302-L308)
:

**Content Handling Modes**

**Wrapping** expands row height to fit content [table/table\_test.go828-926](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L828-L926)
:

    table := New().
        Headers("Name", "Description", "Type", "Required", "Default").
        Rows(longContent...).
        Width(80).
        Wrap(true)  // Enable wrapping
    

**Truncation** uses ellipsis for overflow [table/table.go548-561](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L548-L561)
:

*   Headers are always truncated to single line [table/table.go555-557](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L555-L557)
    
*   Cell width accounts for padding and margins [table/table.go559](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L559-L559)
    
*   Uses `ansi.Truncate()` to preserve ANSI codes [table/table.go560](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L560-L560)
    

Sources: table/table.go, table/table\_test.go

### Base Style and Border Style

Tables support two style layers [table/table.go100-106](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L100-L106)
 [table/table.go212-216](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L212-L216)
:

**Style Inheritance Hierarchy**

**BaseStyle** [table/table\_test.go37-52](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table_test.go#L37-L52)
 sets background for entire table:

    table := New().
        BaseStyle(lipgloss.NewStyle().Background(lipgloss.Color("18"))).
        BorderStyle(lipgloss.NewStyle().Foreground(lipgloss.Color("15")))
    

This ensures consistent background color across all cells and borders, with `BorderStyle` inheriting from `BaseStyle` [table/table.go104](https://github.com/charmbracelet/lipgloss/blob/30441468/table/table.go#L104-L104)

Sources: table/table.go, table/table\_test.go

### Custom Enumerators and Item Styling

Lists support custom enumerator functions and style functions for dynamic rendering:

**Custom Enumerator Pattern**

**Checklist enumerator** [examples/list/sublist/main.go45-52](https://github.com/charmbracelet/lipgloss/blob/30441468/examples/list/sublist/main.go#L45-L52)

    checklistEnum := func(items list.Items, index int) string {

### Gradient and Background Styling

The example demonstrates advanced styling with color gradients [examples/list/sublist/main.go65-76](https://github.com/charmbracelet/lipgloss/blob/30441468/examples/list/sublist/main.go#L65-L76)

### Component Configuration Chains

**Table chaining pattern:**

    table.New().
        Border().
        Headers().
        Rows().
        StyleFunc()
    

**Tree chaining pattern:**

    tree.Root().
        Child().
        ItemStyle().
        EnumeratorStyle()
