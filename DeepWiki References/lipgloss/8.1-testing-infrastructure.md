This page documents the testing infrastructure for lipgloss, focusing on test organization, golden file testing patterns, and benchmark tests. For information about CI/CD workflows and code quality tools, see page 8.2.

Purpose and Scope
-----------------

The lipgloss testing infrastructure validates correctness through unit tests, golden file comparisons, and performance benchmarks. Tests are organized by component, use table-driven patterns for maintainability, and include race detection for thread-safety validation.

Golden File Testing
-------------------

Golden file testing validates rendered output against known-good reference files stored in `testdata/` directories. This approach is used extensively in the component packages (table, list, tree).

### Golden File Structure

**Testdata Directory Organization**

**Sources:** [tree/testdata/TestRootStyle.golden1-3](https://github.com/charmbracelet/lipgloss/blob/30441468/tree/testdata/TestRootStyle.golden#L1-L3)

### Example: Tree Golden File

The tree package uses golden files to validate tree rendering. The file `tree/testdata/TestRootStyle.golden` contains:

    Root
    ├── Foo
    └── Baz
    

This represents the expected output for the `TestRootStyle` test. The test renders a tree structure and compares the output against this golden file.

**Sources:** [tree/testdata/TestRootStyle.golden1-3](https://github.com/charmbracelet/lipgloss/blob/30441468/tree/testdata/TestRootStyle.golden#L1-L3)

### Golden File Test Pattern

**Typical Golden File Test Flow**

**Common Pattern:**

Best Practices for Contributors
-------------------------------

### Writing New Tests

1.  **Use table-driven tests** for multiple scenarios
2.  **Configure renderer explicitly** with `NewRenderer(io.Discard)` and `SetColorProfile()`
3.  **Set background mode** with `SetHasDarkBackground()` for deterministic color tests
4.  **Call `t.Parallel()`** for tests that can run concurrently
5.  **Use helper functions** (`requireEqual`, `requireTrue`, etc.) for cleaner assertions
6.  **Format output** with `formatEscapes()` when debugging ANSI sequences

Summary
-------

The lipgloss testing infrastructure provides comprehensive validation through:

*   **Extensive unit test coverage** across all major components
*   **Race condition detection** with `-race` flag in CI
*   **Atomic coverage tracking** with Coveralls integration
*   **Profile-specific testing** for terminal capability validation
*   **Parallel test execution** for faster feedback
*   **Table-driven patterns** for maintainable test suites

All tests run automatically on every push and pull request, ensuring code quality and preventing regressions.
