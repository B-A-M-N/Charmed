Purpose and Scope
-----------------

The Canvas and Layer system provides advanced composition capabilities for building complex layered UIs with Z-ordering, absolute positioning, and hit testing support. This system enables character-level drawing operations and hierarchical layer management for applications that need precise control over terminal output composition.

For simpler text arrangement without Z-ordering, see [Joining and Alignment](https://deepwiki.com/charmbracelet/lipgloss/5.2-joining-and-alignment)
 and [Placement](https://deepwiki.com/charmbracelet/lipgloss/5.3-placement)
. For text measurement utilities, see [Measurement Utilities](https://deepwiki.com/charmbracelet/lipgloss/5.1-measurement-utilities)
.

System Architecture
-------------------

The Canvas and Layer system consists of three primary components that work together to enable complex terminal compositions:

**Component Responsibilities:**

| Component | Type | Purpose | Key Methods |
| --- | --- | --- | --- |
| `Layer` | Data Structure | Represents positioned content with relative coordinates and hierarchy | `X()`, `Y()`, `Z()`, `AddLayers()` |
| `Compositor` | Computation Engine | Flattens layer hierarchy, sorts by Z-index, manages hit testing | `NewCompositor()`, `flatten()`, `Hit()` |
| `Canvas` | Drawing Buffer | Provides cell-level buffer for compositing drawables | `Compose()`, `CellAt()`, `Render()` |

Canvas: Cell-Based Drawing Buffer
---------------------------------

The `Canvas` type wraps `uv.ScreenBuffer` to provide a character-cell buffer for compositing operations.

### Canvas Structure

### Canvas Operations

The Canvas provides three levels of interaction:

**Creation and Sizing:**

*   `NewCanvas(width, height int)` - Creates a canvas with specified dimensions [canvas.go24-29](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L24-L29)
    
*   `Resize(width, height int)` - Adjusts canvas dimensions [canvas.go32-34](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L32-L34)
    
*   `Clear()` - Clears all cell contents [canvas.go37-39](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L37-L39)
    

**Cell-Level Access:**

*   `CellAt(x, y int)` - Returns pointer to cell at coordinates for direct manipulation [canvas.go57-59](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L57-L59)
    
*   `SetCell(x, y, cell)` - Sets cell content at coordinates [canvas.go62-64](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L62-L64)
    
*   `WidthMethod()` - Returns the Unicode width calculation method (defaults to `ansi.GraphemeWidth`) [canvas.go27](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L27-L27)
    

**Composition and Output:**

*   `Compose(drawer uv.Drawable)` - Draws any `uv.Drawable` (including `Layer`) onto the canvas [canvas.go72-75](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L72-L75)
    
*   `Draw(scr, area)` - Implements `uv.Drawable` to allow canvas as a drawable [canvas.go81-83](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L81-L83)
    
*   `Render()` - Converts canvas contents to styled string [canvas.go86-88](https://github.com/charmbracelet/lipgloss/blob/30441468/canvas.go#L86-L88)

Layer: Positioned Content with Hierarchy
----------------------------------------

The `Layer` type is a pure data structure representing positioned content that can contain child layers.

### Layer Data Structure

**Sources:** [layer.go13-19](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L13-L19)
 [layer.go22-100](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L22-L100)

### Layer Coordinate System

Layers use **relative positioning** - each layer's X, Y, Z coordinates are relative to its parent:

| Coordinate | Default | Purpose | Range |
| --- | --- | --- | --- |
| X   | 0   | Horizontal offset from parent | Any integer |
| Y   | 0   | Vertical offset from parent | Any integer |
| Z   | 0   | Stacking order (higher = on top) | Any integer |

**Bounds Calculation:** The `boundsWithOffset()` method recursively calculates absolute positions and combines content bounds with all child bounds using `image.Rectangle.Union()` [layer.go133-148](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L133-L148)

**Sources:** [layer.go56-72](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L56-L72)
 [layer.go133-148](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L133-L148)

### Layer Methods

**Positioning Methods (Fluent API):**

    ID(id string) *Layer        // Set identifier for retrieval and hit testing
    X(x int) *Layer             // Set x-coordinate relative to parent
    Y(y int) *Layer             // Set y-coordinate relative to parent  
    Z(z int) *Layer             // Set z-index relative to parent
    

**Hierarchy Methods:**

    AddLayers(layers ...*Layer) *Layer  // Add child layers
    GetLayer(id string) *Layer          // Find descendant by ID
    MaxZ() int                          // Get maximum z-index in subtree
    

**Sources:** [layer.go51-100](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L51-L100)

### Compositor Methods

**Construction:**

| Method | Purpose | When to Use |
| --- | --- | --- |
| `NewCompositor(layers...)` | Creates compositor and flattens hierarchy | Initial setup |
| `AddLayers(layers...)` | Adds layers to root and re-flattens | Adding more layers |
| `Refresh()` | Re-flattens hierarchy | After modifying layer positions |

**Drawing and Output:**

| Method | Purpose | Returns |
| --- | --- | --- |
| `Draw(scr, area)` | Draws all layers in z-order to screen | void |

**Querying:**

| Method | Purpose | Returns |

| `GetLayer(id)` | Retrieve layer by ID | `*Layer` or `nil` |

Advanced Features
-----------------

### Hit Testing

The `Hit(x, y)` method performs point-in-rectangle tests to determine which layer (if any) contains the given coordinates:

**LayerHit Structure:**

| Method | Returns | Purpose |
| --- | --- | --- |
| `Empty()` | `bool` | True if no layer was hit |
| `ID()` | `string` | ID of hit layer |
| `Layer()` | `*Layer` | Reference to hit layer |
| `Bounds()` | `image.Rectangle` | Absolute bounds of hit layer |

**Important:** Only layers with non-empty IDs are returned by hit testing. Layers without IDs are skipped [layer.go295](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L295-L295)

**Sources:** [layer.go158-183](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L158-L183)
 [layer.go289-303](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L289-L303)

### Layer Retrieval by ID

Two methods provide layer retrieval:

**Compositor.GetLayer(id):**

*   Uses pre-built index for O(1) lookup [layer.go307-312](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L307-L312)
    
*   Only searches indexed layers (those with IDs set during flattening) [layer.go263-265](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L263-L265)
    

**Layer.GetLayer(id):**

*   Recursively searches layer hierarchy [layer.go103-118](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L103-L118)
    
*   Useful for searching within a specific subtree
*   Returns first match in depth-first order

**Sources:** [layer.go103-118](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L103-L118)
 [layer.go263-265](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L263-L265)
 [layer.go307-312](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L307-L312)

### Bounds Calculation

The compositor calculates overall bounds by taking the union of all layer bounds:

| Operation | Method | Purpose |
| --- | --- | --- |
| Per-Layer Bounds | `boundsWithOffset(parentX, parentY)` | Calculate absolute rectangle for single layer |
| Overall Bounds | `flatten()` final step | Union all layer bounds |
| Bounds Query | `Bounds()` | Return cached overall bounds |

**Bounds Union Process:**

    bounds = layers[0].bounds
    for each remaining layer:
        bounds = bounds.Union(layer.bounds)
    

**Sources:** [layer.go133-148](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L133-L148)
 [layer.go236-241](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L236-L241)
 [layer.go273-275](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L273-L275)

### Refresh Mechanism

The `Refresh()` method re-flattens the layer hierarchy after modifications:

**When to Call Refresh:**

*   After changing layer positions (X, Y, Z)
*   After adding/removing child layers via direct manipulation
*   Not needed after `AddLayers()` (automatically refreshes)

**What Refresh Does:**

1.  Clears cached flattened layers
2.  Clears ID index
3.  Re-runs `flattenRecursive()`
4.  Re-sorts by z-index
5.  Recalculates overall bounds

**Sources:** [layer.go314-318](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L314-L318)
 [layer.go225-242](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L225-L242)

Integration with Style System
-----------------------------

Layers can contain any styled content:

**The layer system is styling-agnostic** - it positions and composites any string content, whether styled or plain text. All ANSI escape sequences in the content are preserved during composition [layer.go154-156](https://github.com/charmbracelet/lipgloss/blob/30441468/layer.go#L154-L156)

Relationship to ultraviolet Package
-----------------------------------

The Canvas and Layer system is built on the `ultraviolet` package:

| lipgloss Type | ultraviolet Type | Relationship |
| --- | --- | --- |
| `Canvas` | `uv.Screen` | Implements interface |
| `Canvas` | `uv.Drawable` | Implements interface |
| `Canvas.scr` | `uv.ScreenBuffer` | Wraps instance |
| `Layer` | `uv.Drawable` | Implements interface |
| `Compositor` | `uv.Drawable` | Implements interface (via `Draw` method) |

**Key ultraviolet interfaces:**

*   `uv.Screen` - Provides cell-level access and bounds
*   `uv.Drawable` - Provides `Draw(screen, area)` method
*   `uv.ScreenBuffer` - Concrete cell buffer implementation
