Shell Completions
=================

Relevant source files

*   [fang.go](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go)
    
*   [testdata/TestSetup/without\_completions/help.golden](https://github.com/charmbracelet/fang/blob/d89b30af/testdata/TestSetup/without_completions/help.golden)
    

This document describes Fang's integration with shell completion functionality. Fang leverages Cobra's built-in completion system, which generates autocompletion scripts for various shells (bash, zsh, fish, PowerShell). The completion feature can be enabled or disabled via configuration options.

For information about other optional features like manpage generation, see [Manpage Generation](https://deepwiki.com/charmbracelet/fang/5.2-manpage-generation)
. For general configuration patterns, see [Configuration Options](https://deepwiki.com/charmbracelet/fang/4-configuration-options)
.

Overview
--------

Shell completions allow users to press Tab to autocomplete commands, subcommands, and flags in their terminal. Cobra provides a default `completion` command that generates shell-specific completion scripts, and Fang exposes this functionality through its configuration system.

By default, Fang enables the completion command. Applications can disable this feature using the `WithoutCompletions()` option when calling `fang.Execute`.

Sources: [fang.go1-209](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L1-L209)

Default Behavior
----------------

When Fang is initialized without the `WithoutCompletions()` option, the completion command is available to end users. The settings struct initializes with `completions: true`:

**Completion Command Enabled by Default**

Sources: [fang.go111-116](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L111-L116)

When enabled, users can run:

*   `<app> completion bash` - Generate bash completion script
*   `<app> completion zsh` - Generate zsh completion script
*   `<app> completion fish` - Generate fish completion script
*   `<app> completion powershell` - Generate PowerShell completion script

The completion command appears in the application's help output alongside other commands.

### Cobra Integration

Fang modifies Cobra's completion behavior through the `CompletionOptions` field:

**Cobra CompletionOptions Control Flow**

Sources: [fang.go163-165](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L163-L165)

The integration point is minimal and non-invasive - Fang simply sets a flag on the existing Cobra command structure. When `DisableDefaultCmd` is true, Cobra's completion command registration is suppressed.

Verification
------------

The test suite validates completion behavior through golden file testing:

| Test Case | Golden File | Validation |
| --- | --- | --- |
| Completions disabled | [testdata/TestSetup/without\_completions/help.golden](https://github.com/charmbracelet/fang/blob/d89b30af/testdata/TestSetup/without_completions/help.golden) | Verifies completion command absent from help output |
| Default behavior | Other help.golden files | Verifies completion command present in help output |

The `without_completions` test case demonstrates that when completions are disabled, the help output shows only the `help` command under COMMANDS, with no `completion` command listed.

Sources: [testdata/TestSetup/without\_completions/help.golden1-17](https://github.com/charmbracelet/fang/blob/d89b30af/testdata/TestSetup/without_completions/help.golden#L1-L17)

Summary
-------

| Aspect | Details |
| --- | --- |
| Default state | Enabled (`completions: true`) |
| Disable option | `fang.WithoutCompletions()` |
| Implementation | Sets `cobra.Command.CompletionOptions.DisableDefaultCmd` |
| Shells supported | bash, zsh, fish, PowerShell (via Cobra) |
| User interface | `<app> completion <shell>` command |

Fang's approach to completions is minimal and delegates entirely to Cobra's well-tested completion system. The framework simply provides a clean interface for enabling or disabling this feature through its functional options pattern.
