Styling and Themes
==================

Relevant source files

*   [help.go](https://github.com/charmbracelet/fang/blob/d89b30af/help.go)
    
*   [theme.go](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go)
    

This document explains the theming and styling system in fang, which provides consistent, terminal-aware visual output for CLI applications. The system implements a two-stage architecture that first selects a color scheme based on terminal capabilities, then transforms that scheme into concrete Lipgloss styles for rendering. For information about how these styles are applied during help rendering, see [Help Rendering System](https://deepwiki.com/charmbracelet/fang/3.2-help-rendering-system)
. For error-specific styling, see [Error Handling](https://deepwiki.com/charmbracelet/fang/3.4-error-handling)
.

Purpose and Architecture
------------------------

The styling system separates color definitions from style application, enabling fang to adapt its visual output to different terminal capabilities while maintaining consistent aesthetics. The architecture consists of two main stages:

1.  **Color Scheme Selection**: Determines which color palette to use based on terminal background (light/dark) and color support
2.  **Style Generation**: Converts the selected color scheme into context-specific Lipgloss styles

**Sources:** [theme.go1-209](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L1-L209)
 [help.go42-108](https://github.com/charmbracelet/fang/blob/d89b30af/help.go#L42-L108)

ColorScheme Structure
---------------------

The `ColorScheme` struct defines the base color palette used throughout the application. It contains 16 semantic color fields that describe the visual identity of different UI elements.

| Field | Purpose | Example Usage |
| --- | --- | --- |
| `Base` | Default text color | Body text, descriptions |
| `Title` | Section headers | "USAGE", "EXAMPLES", "FLAGS" |
| `Description` | Help text | Command and flag descriptions |
| `Codeblock` | Code block background | Usage examples, command syntax |
| `Program` | Program name color | Main executable name |
| `DimmedArgument` | Optional elements | `[args]`, `[--flags]` placeholders |
| `Comment` | Comment text | Lines starting with `#` |
| `Flag` | Flag identifiers | `--help`, `-v` |
| `FlagDefault` | Default value hints | "(default value)" annotations |
| `Command` | Subcommand names | Command arguments in examples |
| `QuotedString` | Quoted strings | Arguments in quotes |
| `Argument` | Regular arguments | Non-flag parameters |
| `Help` | Help-specific text | Reserved for future use |
| `Dash` | Dash characters | Reserved for future use |
| `ErrorHeader` | Error banner | `[fg, bg]` pair for "ERROR" badge |
| `ErrorDetails` | Error message text | Error descriptions |

The `ColorScheme` struct is defined at [theme.go15-33](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L15-L33)

**Sources:** [theme.go15-33](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L15-L33)

Color Scheme Selection
----------------------

Fang uses a `ColorSchemeFunc` type to dynamically select color schemes based on terminal capabilities. The selection process involves detecting the terminal's background color (light or dark) and choosing an appropriate palette.

### Background Detection

The `mustColorscheme()` function determines whether the terminal has a dark background by:

1.  Checking if stdout is a terminal using `term.IsTerminal()`
2.  Calling `lipgloss.HasDarkBackground()` which analyzes the terminal's background color
3.  Creating a `lipgloss.LightDarkFunc` that selects colors based on the result

This detection happens at [theme.go115-121](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L115-L121)

**Sources:** [theme.go115-121](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L115-L121)

Built-in Color Schemes
----------------------

Fang provides two built-in color scheme functions:

### DefaultColorScheme

The `DefaultColorScheme()` function returns a rich color palette using the charmtone color library, which provides carefully selected colors designed for terminal interfaces.

Key characteristics:

*   Uses 16 million colors (TrueColor support)
*   Provides light and dark mode variants for each color
*   Optimized for readability and aesthetics
*   Primary palette: charmtone colors (Charple, Malibu, Pony, etc.)

Example color mappings for dark mode:

*   Title: `charmtone.Charple` (purple)
*   Program: `charmtone.Guppy` (cyan)
*   Flag: `charmtone.Guac` (green)
*   Command: `charmtone.Cheeky` (magenta)

The implementation is at [theme.go42-62](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L42-L62)

### AnsiColorScheme

The `AnsiColorScheme()` function returns a basic color scheme using standard ANSI colors for maximum compatibility with older terminals.

Key characteristics:

*   Uses 16 basic ANSI colors
*   Works on terminals with limited color support
*   Simpler aesthetic but wider compatibility
*   Color mappings: Blue (title), Magenta (flags), Cyan (commands), Green (strings)

The implementation is at [theme.go64-82](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L64-L82)

| Element | DefaultColorScheme (dark) | AnsiColorScheme |
| --- | --- | --- |
| Title | `charmtone.Charple` | `lipgloss.Blue` |
| Program | `charmtone.Guppy` | Dynamic (base color) |
| Flag | `charmtone.Guac` | `lipgloss.Magenta` |
| Command | `charmtone.Cheeky` | `lipgloss.Cyan` |
| QuotedString | `charmtone.Salmon` | `lipgloss.Green` |
| Error Header BG | `charmtone.Cherry` | `lipgloss.Red` |

**Sources:** [theme.go42-82](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L42-L82)

*   `QuotedString`: Quoted string arguments

The `Program` struct appears twice in `Styles`: once nested in `Codeblock` (with background) and once at the top level (without background) for inline usage.

Context-Specific Styling
------------------------

The styling system provides different visual contexts for the same elements depending on where they appear. This is primarily managed through the dual `Program` styles:

### Code Block Context

When syntax elements appear in code blocks (usage examples), they receive the `Codeblock` background:

This styling is applied during example rendering in [help.go224-244](https://github.com/charmbracelet/fang/blob/d89b30af/help.go#L224-L244)
 by `styleExamples()` and [help.go246-364](https://github.com/charmbracelet/fang/blob/d89b30af/help.go#L246-L364)
 by `styleExample()`.

### Inline Context

When the same elements appear inline (like in flag descriptions), they use styles without backgrounds from [theme.go170-183](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L170-L183)
:

*   `Styles.Program.Name` - no background
*   `Styles.Program.Flag` - no background
*   `Styles.Program.Command` - no background

This allows inline code references to maintain visual consistency with surrounding text while still being syntactically highlighted.

**Sources:** [theme.go138-183](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L138-L183)
 [help.go224-364](https://github.com/charmbracelet/fang/blob/d89b30af/help.go#L224-L364)

Background Color Handling
-------------------------

The styling system adapts to terminals without background color support or non-TTY output. The `helpFn()` function at [help.go55-59](https://github.com/charmbracelet/fang/blob/d89b30af/help.go#L55-L59)
 removes vertical padding from code blocks when:

1.  The color profile is `colorprofile.Ascii` or `colorprofile.NoTTY`
2.  The block has no background color set (detected via `reflect.DeepEqual()`)

This prevents unnecessary whitespace in plain text output.

**Sources:** [help.go55-59](https://github.com/charmbracelet/fang/blob/d89b30af/help.go#L55-L59)

Lipgloss Integration
--------------------

Fang's styling system is built on [Lipgloss v2](https://github.com/charmbracelet/fang/blob/d89b30af/Lipgloss%20v2)
 which provides:

*   Terminal color profile detection
*   Style composition and rendering
*   Automatic color adaptation based on terminal capabilities
*   ANSI code generation

Key Lipgloss features used:

*   `lipgloss.Style` - The base styling primitive
*   `lipgloss.LightDarkFunc` - Light/dark mode color selection
*   `lipgloss.HasDarkBackground()` - Terminal background detection
*   `lipgloss.Width()` - ANSI-aware width calculation
*   `lipgloss.JoinHorizontal()` - Layout composition

The integration points are at [theme.go8](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L8-L8)
 (import) and throughout [help.go](https://github.com/charmbracelet/fang/blob/d89b30af/help.go)
 for rendering.

**Sources:** [theme.go8](https://github.com/charmbracelet/fang/blob/d89b30af/theme.go#L8-L8)
