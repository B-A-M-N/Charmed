Manpage Generation
==================

Relevant source files

*   [LICENSE.md](https://github.com/charmbracelet/fang/blob/d89b30af/LICENSE.md?plain=1)
    
*   [README.md](https://github.com/charmbracelet/fang/blob/d89b30af/README.md?plain=1)

*   [testdata/TestSetup/complete/man.golden](https://github.com/charmbracelet/fang/blob/d89b30af/testdata/TestSetup/complete/man.golden)
    

Fang automatically adds manpage generation capabilities to Cobra applications through a hidden `man` command. This feature integrates the [mango-cobra](https://github.com/charmbracelet/fang/blob/d89b30af/mango-cobra)
 library to generate Unix manual pages in roff format from Cobra command structures.

For information about shell completions, see [Shell Completions](https://deepwiki.com/charmbracelet/fang/5.3-shell-completions)
. For details about command-line help output, see [Help Rendering System](https://deepwiki.com/charmbracelet/fang/3.2-help-rendering-system)
.

Overview
--------

By default, Fang adds a hidden `man` command to every Cobra application. When invoked, this command generates a complete manpage for the entire command tree and outputs it to stdout in roff format. Unlike Cobra's built-in manpage generation which creates separate pages for each subcommand, Fang uses mango-cobra to generate a single consolidated manpage, making it more suitable for smaller applications.

The manpage generation system consists of three main components:

| Component | Purpose | Library |
| --- | --- | --- |
| Command Registration | Adds hidden `man` command to root | fang.Execute |
| Manpage Generation | Converts Cobra structure to manpage | mango-cobra |
| Roff Formatting | Formats output in roff markup | muesli/roff |

Sources: [fang.go1-209](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L1-L209)

The Hidden 'man' Command
------------------------

The `man` command is registered as a hidden subcommand with specific characteristics that make it suitable for programmatic use while keeping it out of normal help output.

### Command Specification

The command is defined with the following properties:

**Diagram 3: Command Properties and Behavior**

| Property | Value | Purpose |
| --- | --- | --- |
| `Use` | `"man"` | Command name users invoke |
| `Short` | `"Generates manpages"` | Brief description |
| `Hidden` | `true` | Excluded from help output |
| `SilenceUsage` | `true` | Don't show usage on error |
| `DisableFlagsInUseLine` | `true` | Clean usage line |
| `Args` | `cobra.NoArgs` | Command takes no arguments |

Implementation Details
----------------------

The manpage generation implementation is concise and delegates most work to the mango-cobra library:

**Key Code Entities:**

*   `mango.NewManPage(1, cmd.Root())` - Creates a ManPage with section 1 (user commands) from the root command
*   `roff.NewDocument()` - Creates a new roff document for formatting
*   `page.Build(roff.NewDocument())` - Builds the final roff-formatted string
*   `fmt.Fprint(os.Stdout, ...)` - Outputs the manpage to standard output

The manpage section number is hardcoded to `1`, which is the standard Unix manual section for user commands and executable programs.

Sources: [fang.go151-156](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L151-L156)

Configuration
-------------

Manpage generation is controlled through the settings system.

### Default Behavior

Manpage generation is **enabled by default**. The `settings` struct initializes with `manpages: true`.

Sources: [fang.go112](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L112-L112)

### Disabling Manpages

To disable manpage generation, use the `WithoutManpage()` option:

**Option Function:**

The `WithoutManpage()` function returns an `Option` that sets the `manpages` field to `false`:

    WithoutManpage() Option
      └─> Sets s.manpages = false
    

**Usage Pattern:**

When disabled, the hidden `man` command is not added to the command tree, and users cannot invoke it.

Generated Output Format
-----------------------

The manpage is generated in roff format, which is the standard Unix manual page markup language. The output includes:

*   **Header**: Command name, section number
*   **Name Section**: Brief description
*   **Synopsis**: Usage patterns
*   **Description**: Detailed command information
*   **Options**: Available flags and their descriptions
*   **Commands**: Subcommands (if any)
*   **Examples**: Usage examples (if provided in Cobra metadata)

The exact format depends on the content of the Cobra command structure. The mango-cobra library automatically extracts information from:

*   `cmd.Use` - Command name and arguments
*   `cmd.Short` - Brief description
*   `cmd.Long` - Detailed description
*   `cmd.Example` - Usage examples
*   `cmd.Flags()` - Available flags
*   `cmd.Commands()` - Subcommands

Sources: [fang.go151-156](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L151-L156)
 [README.md33-38](https://github.com/charmbracelet/fang/blob/d89b30af/README.md?plain=1#L33-L38)

Comparison with Cobra Default
-----------------------------

The table below contrasts Fang's manpage generation approach with Cobra's built-in functionality:

| Aspect | Fang (mango-cobra) | Cobra Default |
| --- | --- | --- |
| Output Format | Single consolidated page | One page per command |
| Generation Method | Direct roff formatting | Markdown to man conversion |
| Best Suited For | Smaller applications | Large multi-command apps |
| Integration | Hidden command | Separate function call |
| Manual Section | 1 (hardcoded) | Configurable |

Sources: [README.md33-38](https://github.com/charmbracelet/fang/blob/d89b30af/README.md?plain=1#L33-L38)
