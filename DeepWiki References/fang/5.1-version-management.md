Version Management
==================

Relevant source files

*   [fang.go](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go)
    
*   [testdata/TestSetup/with\_version\_and\_hash/help.golden](https://github.com/charmbracelet/fang/blob/d89b30af/testdata/TestSetup/with_version_and_hash/help.golden)
    

This document explains Fang's automatic version management system, which adds `--version` and `-v` flags to Cobra applications. The system can automatically extract version information from Go's build metadata or accept custom version strings.

For information about other command-line features like manpage generation, see [5.2](https://deepwiki.com/charmbracelet/fang/5.2-manpage-generation)
. For configuration options overview, see [4](https://deepwiki.com/charmbracelet/fang/4-configuration-options)
.

* * *

Overview
--------

Fang automatically enables version reporting for CLI applications by:

1.  Adding `--version` and `-v` flags to the root command
2.  Extracting version information from Go build metadata when available
3.  Formatting version strings with optional commit SHA
4.  Providing configuration options to customize or disable version handling

The version management system integrates seamlessly with Cobra's existing flag system and can be configured through functional options passed to `fang.Execute`.

**Sources:** [fang.go1-209](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L1-L209)

Version Sources
---------------

Fang supports three version information sources, in order of precedence:

| Source | Priority | Configuration | Example Output |
| --- | --- | --- | --- |
| Custom version string | 1 (Highest) | `WithVersion("v1.2.3")` | `v1.2.3` or `v1.2.3 (abc1234)` |
| Build metadata | 2   | Automatic (default) | `v0.1.0` or `v0.1.0 (abc1234)` |
| Fallback | 3 (Lowest) | When no metadata available | `unknown (built from source)` |

### Automatic Version Detection

When no custom version is provided via `WithVersion`, Fang attempts to extract version information from Go's build metadata using `runtime/debug.ReadBuildInfo()` [fang.go185-190](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L185-L190)
:

This works when:

*   The binary is built with Go modules
*   The module has a valid `go.sum` entry
*   VCS information is embedded during build

Version String Formatting
-------------------------

### Format Logic

The `buildVersion` function [fang.go181-196](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L181-L196)
 constructs the final version string:

1.  **Base version determination:**
    
    *   If `opts.version` is set, use it
    *   Else if `debug.BuildInfo` is available and has valid `Main.Sum`, use `Main.Version`
    *   Else use `"unknown (built from source)"`
2.  **Commit SHA appending:**
    
    *   If commit SHA has at least 7 characters, append first 7 in parentheses
    *   Format: `{version} ({commit[:7]})`

### Version String Examples

| Scenario | Version | Commit | Output |
| --- | --- | --- | --- |
| Custom version + commit | `"v1.2.3"` | `"abc1234567890"` | `v1.2.3 (abc1234)` |
| Custom version only | `"v1.2.3"` | `""` | `v1.2.3` |
| BuildInfo version + VCS | (from BuildInfo) | (from VCS) | `v0.1.0 (def5678)` |
| BuildInfo version only | (from BuildInfo) | `""` | `v0.1.0` |
| No metadata | `""` | `""` | `unknown (built from source)` |
| Short commit | `"v1.0.0"` | `"abc"` | `v1.0.0` |

Integration with Cobra
----------------------

### Version Assignment Flow

When `root.Version` is set, Cobra automatically:

*   Adds `--version` and `-v` flags to the root command
*   Displays the version string when either flag is used
*   Exits after printing the version

Build Metadata Extraction
-------------------------

### getKey Helper Function

The `getKey` function [fang.go198-208](https://github.com/charmbracelet/fang/blob/d89b30af/fang.go#L198-L208)
 extracts specific values from Go's build metadata:

**Commonly extracted keys:**

*   `"vcs.revision"`: Git commit SHA
*   `"vcs.time"`: Commit timestamp
*   `"vcs.modified"`: Whether working tree was dirty
