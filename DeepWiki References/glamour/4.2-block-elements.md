Purpose and Scope
-----------------

This document covers the rendering of **block-level elements** in Glamour's Markdown rendering pipeline. Block elements are structural components that occupy their own vertical space and typically contain other elements. This includes: Document, Paragraph, BlockQuote, Heading, and Definition Lists.

For information about the general rendering lifecycle (Render/Finish phases), see [Rendering Lifecycle](https://deepwiki.com/charmbracelet/glamour/4.1-rendering-lifecycle)
. For specialized block types like Tables, Code Blocks, and Lists, see [Tables](https://deepwiki.com/charmbracelet/glamour/4.3-tables)
, [Code Blocks](https://deepwiki.com/charmbracelet/glamour/4.4-code-blocks)
, and [Lists and Tasks](https://deepwiki.com/charmbracelet/glamour/4.6-lists-and-tasks)
 respectively.

**Sources:** [ansi/elements.go41-479](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L41-L479)

BlockElement - Generic Block Container
--------------------------------------

`BlockElement` is the most fundamental block renderer, serving as a generic container for elements that need indentation, margins, and style application but no specialized layout logic.

### Structure

    BlockElement {
        Block   *bytes.Buffer  // Accumulates rendered child content
        Style   StyleBlock     // Block-level styling (indent, margin, colors)
        Margin  bool           // Apply margin/padding around content
        Newline bool           // Add newline after content
    }
    

**Sources:** [ansi/blockelement.go14-19](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L14-L19)

### Rendering Lifecycle

#### Render Phase

The `Render` method [ansi/blockelement.go22-29](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L22-L29)
 performs initial setup:

1.  Pushes the `BlockElement` onto the `BlockStack` to establish a new layout context
2.  Renders `BlockPrefix` to the parent writer with parent style
3.  Renders `Prefix` to the current block's buffer with current style

This creates a nested rendering context where child elements write to `Block` (the buffer).

#### Finish Phase

The `Finish` method [ansi/blockelement.go32-65](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L32-L65)
 completes rendering:

1.  **If `Margin` is true:**
    
    *   Applies word wrapping using `ansi.Wordwrap()` with the available width from `BlockStack`
    *   Creates a `MarginWriter` to apply indentation and margins
    *   Writes wrapped content through the margin writer
    *   Optionally adds a newline if `Newline` is true
2.  **If `Margin` is false:**
    
    *   Directly writes the block's buffer to the parent block (no margin processing)
3.  Renders `Suffix` and `BlockSuffix`
    
4.  Resets the buffer and pops from `BlockStack`
    

**Sources:** [ansi/blockelement.go22-65](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L22-L65)

HeadingElement - Specialized Heading Renderer
---------------------------------------------

`HeadingElement` handles H1-H6 headings with level-specific styling.

### Structure

    HeadingElement {
        Level int   // Heading level (1-6)
        First bool  // Whether this is the first element in parent
    }
    

**Sources:** [ansi/heading.go12-15](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/heading.go#L12-L15)

### Style Resolution

Headings use a two-tier style cascade [ansi/heading.go27-44](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/heading.go#L27-L44)
:

1.  Base `Heading` style from `StyleConfig`
2.  Level-specific override (H1, H2, H3, H4, H5, H6)

The `cascadeStyles` function merges these, with level-specific styles taking precedence.

### Rendering Process

#### Render Phase [ansi/heading.go27-59](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/heading.go#L27-L59)

1.  Resolves the appropriate style for the heading level
2.  Adds a newline separator if not the first element (`!e.First`)
3.  Creates a `BlockElement` with the cascaded style
4.  Pushes it onto `BlockStack`
5.  Renders `BlockPrefix` and `Prefix`

#### Finish Phase [ansi/heading.go62-87](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/heading.go#L62-L87)

1.  Creates a `MarginWriter` for the heading
2.  Applies word wrapping with `wordwrap.NewWriter` (respects terminal width)
3.  Writes wrapped content through the margin writer
4.  Renders `Suffix` and `BlockSuffix`
5.  Pops from `BlockStack`

**Sources:** [ansi/heading.go27-87](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/heading.go#L27-L87)

ParagraphElement - Text Paragraph Renderer
------------------------------------------

`ParagraphElement` renders standard text paragraphs with proper spacing and word wrapping.

### Structure

    ParagraphElement {
        First bool  // Whether this is the first element in parent
    }
    

**Sources:** [ansi/paragraph.go13-15](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/paragraph.go#L13-L15)

### Special Handling

Paragraphs inside list items are **not** rendered as separate blocks [ansi/elements.go72-77](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L72-L77)
 Instead, they return an empty `Element{}`, allowing their text content to flow directly into the list item.

### Rendering Process

#### Render Phase [ansi/paragraph.go18-34](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/paragraph.go#L18-L34)

1.  Adds a newline separator if not the first element
2.  Creates a `BlockElement` with paragraph style (cascaded from parent)
3.  Pushes onto `BlockStack`
4.  Renders `BlockPrefix` and `Prefix`

#### Finish Phase [ansi/paragraph.go37-63](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/paragraph.go#L37-L63)

1.  Creates a `MarginWriter` for the paragraph
2.  **If the paragraph has content** (non-whitespace):
    *   Creates a `wordwrap.NewWriter` with terminal width
    *   Enables `KeepNewlines` if `PreserveNewLines` option is set
    *   Writes wrapped content through the margin writer
    *   Adds a trailing newline
3.  Renders `Suffix` and `BlockSuffix`
4.  Pops from `BlockStack`

**Sources:** [ansi/paragraph.go18-63](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/paragraph.go#L18-L63)
 [ansi/elements.go72-77](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L72-L77)

Style Cascading in Block Elements
---------------------------------

**Diagram: Style Cascading Process for Block Elements**

All block elements inherit styles from their parent through the `BlockStack`. The `cascadeStyle` function merges parent and element-specific styles, with element styles overriding parent values when present.

**Example from BlockQuote** [ansi/elements.go89](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L89-L89)
:

    cascadeStyle(ctx.blockStack.Current().Style, ctx.options.Styles.BlockQuote, false)
    

This takes the current block's style (which itself cascaded from parents) and merges it with the BlockQuote-specific style.

Layout Management Integration
-----------------------------

Block elements interact closely with three layout management systems:

### BlockStack

The `BlockStack` maintains a stack of active block contexts [ansi/elements.go89](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L89-L89)
 Each block element:

1.  **On Render:** Pushes itself onto the stack via `bs.Push(*e)`
2.  **During Child Rendering:** Children write to the current block's buffer
3.  **On Finish:** Pops itself from the stack via `bs.Pop()`

This allows nested blocks (e.g., blockquotes containing paragraphs) to properly manage indentation and width.

### MarginWriter

The `MarginWriter` [ansi/blockelement.go42](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L42-L42)
 applies indentation and margins when writing final content. It uses the block's `StyleBlock` to determine:

*   Left margin/indent from `Indent` field
*   Additional padding from `Margin` field
*   Line-by-line indentation for the entire block

### Word Wrapping

Block elements use two word-wrapping approaches:

| Element Type | Wrapping Method | Source |
| --- | --- | --- |
| `BlockElement` | `ansi.Wordwrap()` | [ansi/blockelement.go36-40](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L36-L40) |
| `HeadingElement` | `wordwrap.NewWriter()` | [ansi/heading.go67-74](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/heading.go#L67-L74) |
| `ParagraphElement` | `wordwrap.NewWriter()` | [ansi/paragraph.go43-48](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/paragraph.go#L43-L48) |

Both methods respect `bs.Width(ctx)` to calculate available terminal width after accounting for indentation.

**Sources:** [ansi/blockelement.go36-42](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockelement.go#L36-L42)
