Purpose and Scope
-----------------

This document explains how Glamour renders fenced code blocks with syntax highlighting. It covers the `CodeBlockElement` type, its integration with the Chroma syntax highlighter, style configuration, and the rendering pipeline from Markdown code blocks to styled terminal output.

For information about inline code (backtick-wrapped text within paragraphs), see the inline text rendering section. For table rendering, which also uses code-like monospace styling, see [Tables](https://deepwiki.com/charmbracelet/glamour/4.3-tables)
.

CodeBlockElement Structure
--------------------------

The `CodeBlockElement` struct represents a fenced code block in the Markdown AST:

| Field | Type | Purpose |
| --- | --- | --- |
| `Code` | `string` | The raw code content to be rendered |
| `Language` | `string` | The language identifier for syntax highlighting (e.g., "go", "python", "javascript") |

When the ANSIRenderer encounters an `ast.KindCodeBlock` node during AST traversal, it creates a `CodeBlockElement` and populates these fields from the node's attributes.

**Sources:** [ansi/codeblock.go27-31](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/codeblock.go#L27-L31)

Rendering Pipeline
------------------

The following diagram shows how a code block flows through the rendering system:

**Diagram: Code Block Rendering Flow**

The rendering process follows these steps:

1.  **Style Resolution**: Extract `StyleCodeBlock` configuration from `ctx.options.Styles.CodeBlock`
2.  **Indentation Setup**: Create an `indent.NewWriterPipe` with configured indentation and margin
3.  **Theme Selection**: Use Chroma theme if configured, otherwise fallback to plain rendering
4.  **Highlighting**: Call `quick.Highlight()` with the code, language, formatter, and theme
5.  **Output**: Write prefix, highlighted code, and suffix to the indented writer

**Sources:** [ansi/codeblock.go68-152](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/codeblock.go#L68-L152)

Chroma Integration
------------------

### Style Registration

Glamour uses the Chroma library for syntax highlighting. When rendering a code block with Chroma enabled, it dynamically registers a custom Chroma style called `"charm"`:

**Diagram: Chroma Style Registration and Token Mapping**

The registration process (protected by a `sync.Mutex` for thread safety) maps 31 Chroma token types to custom styles:

| Category | Token Types |
| --- | --- |
| **Basic** | Text, Error |
| **Comments** | Comment, CommentPreproc |
| **Keywords** | Keyword, KeywordReserved, KeywordNamespace, KeywordType |
| **Operators** | Operator, Punctuation |
| **Names** | Name, NameBuiltin, NameTag, NameAttribute, NameClass, NameConstant, NameDecorator, NameException, NameFunction, NameOther |
| **Literals** | Literal, LiteralNumber, LiteralDate, LiteralString, LiteralStringEscape |
| **Generic** | GenericDeleted, GenericEmph, GenericInserted, GenericStrong, GenericSubheading |
| **Background** | Background |

**Sources:** [ansi/codeblock.go86-128](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/codeblock.go#L86-L128)

### chromaStyle Function

The `chromaStyle()` helper function converts a `StylePrimitive` into a Chroma-compatible style string:

**Diagram: chromaStyle Conversion Logic**

Example conversions:

*   `{Color: "203", BackgroundColor: "236"}` → `"203 bg:236"`
*   `{Color: "39", Bold: true, Italic: true}` → `"39 bold italic"`
*   `{Color: "#C4C4C4"}` → `"#C4C4C4"`

**Sources:** [ansi/codeblock.go33-65](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/codeblock.go#L33-L65)

Style Configuration
-------------------

Code blocks are styled using the `StyleCodeBlock` configuration from the active theme. Key configuration fields include:

| Field | Type | Description |
| --- | --- | --- |
| `Indent` | `*uint` | Indentation level for the entire code block |
| `Margin` | `*uint` | Left margin spacing |
| `Theme` | `string` | Chroma theme name (used when Chroma is nil) |
| `Chroma` | `*Chroma` | Token-specific style definitions |
| `BlockPrefix` | `string` | Text rendered before the code block |
| `BlockSuffix` | `string` | Text rendered after the code block |
| `StylePrimitive` | `StylePrimitive` | Fallback styling (color, bold, etc.) |

Fallback Rendering
------------------

When Chroma highlighting is disabled or unavailable, Glamour falls back to plain text rendering:

**Diagram: Fallback Rendering Flow**

The fallback creates a `BaseElement` with the raw code as the token and applies the `StylePrimitive` from `rules.StylePrimitive`. This provides basic color and formatting without language-specific syntax highlighting.

**Sources:** [ansi/codeblock.go145-152](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/codeblock.go#L145-L152)

Indentation and Margins
-----------------------

Code blocks use the `indent.NewWriterPipe` to apply indentation and margins:

**Diagram: Code Block Indentation**

The indentation callback renders spaces with the current block style from the `BlockStack`, ensuring that indentation matches the surrounding context's styling. This is the same pattern used by other block elements.

Integration with Rendering Context
----------------------------------

Code blocks receive configuration and context through the `RenderContext` parameter:

| Context Field | Usage |
| --- | --- |
| `ctx.blockStack` | Manages nested block indentation and style cascading |
| `ctx.options.Styles.CodeBlock` | Provides `StyleCodeBlock` configuration |
| `ctx.options.ColorProfile` | Determines whether to use Chroma (skipped for ASCII profiles) |
| `ctx.options.ChromaFormatter` | Optional override for the Chroma formatter |

The block stack tracks the current rendering depth and inherited styles, allowing code blocks to properly indent within lists, blockquotes, and other nested structures.

**Sources:** [ansi/codeblock.go68-72](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/codeblock.go#L68-L72)
