This document provides a comprehensive overview of the pre-defined styling themes available in Glamour. These themes control the visual appearance of rendered Markdown content in the terminal. For information about creating custom styles, see [Custom Styles](https://deepwiki.com/charmbracelet/glamour/3.3-custom-styles)
.

Theme Overview
--------------

Glamour includes several built-in themes that provide ready-to-use styling configurations for different terminal environments and aesthetic preferences. These themes are defined as JSON files that specify styling attributes for various Markdown elements.

Sources: [styles/dark.json](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/dark.json)

Available Themes
----------------

Glamour provides the following built-in themes:

| Theme Name | Description | Best For |
| --- | --- | --- |
| `dark` | Dark background with light text | Dark terminal themes |
| `light` | Light background with dark text | Light terminal themes |
| `notty` | No color, uses plain text with Markdown syntax | Terminals without ANSI support |
| `dracula` | Based on the popular Dracula color scheme | Dark theme with vibrant colors |
| `pink` | Pink accents with minimal styling | Custom aesthetic preference |
| `simple` | Minimalist styling | Clean, unobtrusive output |
| `tokyonight` | Based on Tokyo Night color scheme | Dark theme with blue accents |

Sources: [styles/dark.json](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/dark.json)

Theme Structure
---------------

All themes in Glamour follow a consistent structure based on the `StyleConfig` type defined in [ansi/style.go97-139](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L97-L139)
 Each theme is a JSON file that contains styling configurations for different Markdown elements.

Theme Examples
--------------

Below are brief examples of how different elements appear in various themes:

### Dark Theme

The dark theme uses light text on dark backgrounds with blue headings and colorful syntax highlighting.

Key features:

*   Document color: Light gray (252)
*   Headings: Blue (39), with H1 having yellow (228) text on blue (63) background
*   Links: Underlined with color 30
*   Code: Red text (203) on dark background (236)
*   Block quotes: Indented with vertical line

Sources: [styles/dark.json](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/dark.json)
 [styles/gallery/dark.png](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/gallery/dark.png)

### Light Theme

The light theme uses dark text on light backgrounds with blue headings and more subdued syntax highlighting.

Key features:

*   Document color: Dark gray (234)
*   Headings: Blue (27), with H1 having yellow (228) text on blue (63) background
*   Links: Underlined with color 36
*   Code: Red text (203) on light background (254)
*   Block quotes: Indented with vertical line

Sources: [styles/light.json](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/light.json)
 [styles/gallery/light.png](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/gallery/light.png)

### NoTTY Theme

The notty theme is designed for terminals without ANSI color support. It uses plain text with Markdown syntax preserved.

Key features:

*   No colors used
*   Markdown syntax visible (e.g., `**bold**`, `*italic*`)
*   Simple ASCII characters for block quotes and lists
*   Minimal styling overall

Sources: [styles/notty.json](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/notty.json)

### Dracula Theme

Based on the popular Dracula color scheme, this theme uses vibrant colors on a dark background.

Key features:

*   Document color: Light gray (#f8f8f2) on dark background
*   Purple headings (#bd93f9)
*   Cyan links (#8be9fd)
*   Yellow emphasis (#f1fa8c)

Using Built-in Themes
---------------------

Built-in themes can be specified when creating a new renderer using the appropriate configuration functions:

### Theme Selection Methods

The following functions can be used to select a theme:

1.  `WithAutoStyle()` - Automatically selects a theme based on terminal capabilities
2.  `WithDarkStyle()` - Uses the dark theme
3.  `WithLightStyle()` - Uses the light theme
4.  `WithStandardStyle(name)` - Uses a specific built-in theme by name:
    *   "dark"
    *   "light"
    *   "notty"
    *   "dracula"
    *   "pink"
    *   "simple"
    *   "tokyonight"

Style Inheritance and Cascading
-------------------------------

Styles in Glamour follow a cascading pattern, where more specific styles inherit from and override more general styles. This is implemented through the `cascadeStyles`, `cascadeStylePrimitives`, and related functions in [ansi/style.go141-257](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L141-L257)

For example, heading styles (h1-h6) inherit from the general heading style, which allows for consistent styling while enabling specific customizations for each heading level.

Sources: [ansi/style.go141-157](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L141-L157)
 [ansi/style.go157-241](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L157-L241)
 [ansi/style.go243-257](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L243-L257)

Template Functionality
----------------------

Theme styles can include template format strings for more dynamic styling. For example, the image text format in most themes is defined as:

    "image_text": {
      "format": "Image: {{.text}} →"
    }
    

This uses Go's template syntax to insert the image's text. Glamour provides various template helper functions defined in [ansi/templatehelper.go](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/templatehelper.go)
 for advanced formatting needs.

Sources: [ansi/templatehelper.go13-86](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/templatehelper.go#L13-L86)
