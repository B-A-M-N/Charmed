This page documents the various rendering functions provided by the Glamour library for converting Markdown content into styled terminal output. For information about customizing styles, see [Styling System](https://deepwiki.com/charmbracelet/glamour/3-styling-system)
 and for details about the TermRenderer itself, see [TermRenderer](https://deepwiki.com/charmbracelet/glamour/2.1-termrenderer)
.

Overview of Rendering Functions
-------------------------------

Glamour provides two main approaches to rendering Markdown:

1.  **High-level render functions** - Quick and simple functions for basic rendering needs
2.  **Custom renderer approach** - More configurable approach using a `TermRenderer` instance

Sources: [glamour.go44-67](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L44-L67)
 [glamour.go285-295](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L285-L295)

High-Level Render Functions
---------------------------

These functions provide a streamlined way to render Markdown without creating and configuring a renderer instance.

### Render

This function renders a Markdown string with a specified style and returns the ANSI-formatted output as a string.

**Parameters:**

*   `in` - Markdown content to render
*   `stylePath` - Either a built-in style name (e.g., "dark", "light") or a path to a JSON style file

### TermRenderer Methods

Once you have a `TermRenderer` instance, you can use these methods to render Markdown:

| Method | Signature | Description |
| --- | --- | --- |
| `Render` | `func (tr *TermRenderer) Render(in string) (string, error)` | Renders Markdown string to styled output |
| `RenderBytes` | `func (tr *TermRenderer) RenderBytes(in []byte) ([]byte, error)` | Renders Markdown byte slice to styled byte slice |
| `Write` | `func (tr *TermRenderer) Write(b []byte) (int, error)` | Implements `io.Writer` for use in streaming scenarios |
| `Read` | `func (tr *TermRenderer) Read(b []byte) (int, error)` | Implements `io.Reader` for reading rendered output |
| `Close` | `func (tr *TermRenderer) Close() error` | Finalizes rendering when using `Writer`/`Reader` interface |

Sources: [glamour.go253-295](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L253-L295)

Configuration Options
---------------------

When creating a `TermRenderer`, you can customize its behavior using these configuration options:

| Option Function | Description |
| --- | --- |
| `WithAutoStyle()` | Automatically selects dark or light style based on terminal background |
| `WithStandardStyle(style string)` | Uses a built-in style (e.g., "dark", "light", "notty") |
| `WithEnvironmentConfig()` | Uses style from `GLAMOUR_STYLE` environment variable |
| `WithStylePath(stylePath string)` | Uses style from file path or built-in style |
| `WithStyles(styles ansi.StyleConfig)` | Uses provided style configuration directly |
| `WithStylesFromJSONBytes(jsonBytes []byte)` | Parses styles from JSON bytes |
| `WithStylesFromJSONFile(filename string)` | Loads styles from JSON file |
| `WithWordWrap(wordWrap int)` | Sets word wrap width (default: 80) |
| `WithTableWrap(tableWrap bool)` | Controls whether table content wraps or truncates |
| `WithInlineTableLinks(inlineTableLinks bool)` | Controls table link rendering style |
| `WithPreservedNewLines()` | Preserves newlines in the Markdown source |
| `WithEmoji()` | Enables emoji rendering |
| `WithChromaFormatter(formatter string)` | Sets formatter for code syntax highlighting |
| `WithBaseURL(baseURL string)` | Sets base URL for resolving relative links |
| `WithColorProfile(profile termenv.Profile)` | Sets color profile (TrueColor, ANSI256, etc.) |

Sources: [glamour.go102-251](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L102-L251)

Conclusion
----------

Glamour provides flexible rendering functions for converting Markdown to styled terminal output. The high-level functions (`Render`, `RenderWithEnvironmentConfig`, `RenderBytes`) offer simplicity for basic needs, while the custom renderer approach provides extensive configurability for more advanced use cases.

For more information about the styling system and available themes, refer to the [Styling System](https://deepwiki.com/charmbracelet/glamour/3-styling-system)
 section. For detailed configuration options, see [Configuration Options](https://deepwiki.com/charmbracelet/glamour/2.3-configuration-options)
