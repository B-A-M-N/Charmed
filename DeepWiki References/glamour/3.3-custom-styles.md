This page explains how to create and apply custom styles to Glamour's markdown rendering. Custom styles allow you to define your own color schemes, formatting rules, and visual presentation for all markdown elements. For information about the structure of `StyleConfig` and style properties, see [Style Configuration](https://deepwiki.com/charmbracelet/glamour/3.1-style-configuration)
. For pre-built themes, see [Built-in Themes](https://deepwiki.com/charmbracelet/glamour/3.2-built-in-themes)
.

Overview
--------

Glamour supports custom styling through JSON files or programmatic `StyleConfig` objects. Custom styles can be loaded from files, environment variables, or passed directly as Go structs. The styling system uses a cascading model where block elements define default properties inherited by their children.

### WithStylePath

`WithStylePath` is the most flexible option, attempting to interpret the path first as a built-in style name, then as a file path:

**Behavior:**

1.  Calls `getDefaultStyle(stylePath)` to check if it's a built-in style
2.  If not found, treats it as a file path and calls `os.ReadFile(stylePath)`
3.  Unmarshals the JSON into `tr.ansiOptions.Styles`
4.  Returns an error if both attempts fail

**Sources:** [glamour.go147-161](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L147-L161)

### WithStyles

`WithStyles` accepts a fully constructed `ansi.StyleConfig` object, useful when building styles programmatically:

This method bypasses file I/O and JSON parsing, directly assigning the style configuration.

**Sources:** [glamour.go164-169](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L164-L169)

### WithStylesFromJSONBytes

`WithStylesFromJSONBytes` parses JSON from a byte slice:

Useful when you have JSON data in memory (e.g., embedded resources, network responses).

**Sources:** [glamour.go173-177](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L173-L177)

### WithStylesFromJSONFile

`WithStylesFromJSONFile` loads JSON directly from a file path:

Unlike `WithStylePath`, this always treats the argument as a file path and never falls back to built-in styles.

**Sources:** [glamour.go180-188](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L180-L188)

### WithEnvironmentConfig

`WithEnvironmentConfig` reads the `GLAMOUR_STYLE` environment variable and delegates to `WithStylePath`:

If `GLAMOUR_STYLE` is unset or empty, defaults to `"auto"`.

**Sources:** [glamour.go140-142](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L140-L142)

JSON Style Format
-----------------

### Structure Overview

Custom style JSON files define styling for block and inline markdown elements. The structure maps element types to style configurations:

**Sources:** [styles/README.md8-566](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/README.md?plain=1#L8-L566)

### Style Attributes

Each element in the JSON can specify these attributes:

| Category | Attribute | Type | Description |
| --- | --- | --- | --- |
| **Positioning** | `indent` | number | Indentation level |
|     | `indent_token` | string | Character(s) for indentation |
|     | `margin` | number | Spacing around element |
| **Prefix/Suffix** | `block_prefix` | string | Printed before (in parent's style) |
|     | `block_suffix` | string | Printed after (in parent's style) |
|     | `prefix` | string | Printed before (in element's style) |
|     | `suffix` | string | Printed after (in element's style) |
| **Colors** | `color` | color | Text color |
|     | `background_color` | color | Background color |
| **Text Formatting** | `bold` | bool | Bold text |
|     | `italic` | bool | Italic text |
|     | `underline` | bool | Underline decoration |
|     | `faint` | bool | Reduced intensity |
|     | `crossed_out` | bool | Strikethrough |
|     | `overlined` | bool | Overline decoration |
|     | `blink` | bool | Blinking text |
|     | `inverse` | bool | Swap foreground/background |
|     | `conceal` | bool | Hide text |
| **Capitalization** | `upper` | bool | Convert to uppercase |
|     | `lower` | bool | Convert to lowercase |
|     | `title` | bool | Convert to title case |

**Sources:** [styles/README.md10-40](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/README.md?plain=1#L10-L40)

### Element-Specific Attributes

Some elements have specialized attributes:

| Element | Attribute | Type | Description |
| --- | --- | --- | --- |
| `list` | `level_indent` | number | Indentation for nested lists |
| `code_block` | `theme` | string | Chroma syntax highlighting theme |
| `task` | `ticked` | string | Prefix for checked tasks |
| `task` | `unticked` | string | Prefix for unchecked tasks |

**Sources:** [styles/README.md152-183](https://github.com/charmbracelet/glamour/blob/69661fd5/styles/README.md?plain=1#L152-L183)

Color Specifications
--------------------

Colors in custom styles can be specified as:

| Format | Example | Description |
| --- | --- | --- |
| ANSI 256 | `"39"` | ANSI 256-color palette index |
| Hex | `"#FF5733"` | Hexadecimal RGB color |
| Named | `"red"` | Named color (termenv) |

The renderer automatically adapts colors based on terminal capabilities via `ColorProfile` (TrueColor, ANSI256, or ANSI).
