Configuration Options
---------------------

`TermRendererOption` functions configure `NewTermRenderer()`:

| Option Function | Parameters | Description |
| --- | --- | --- |
| `WithAutoStyle()` | none | Auto-detect terminal background and use `"dark"` or `"light"` style |
| `WithStandardStyle(style)` | `string` | Use a built-in style name (`"dark"`, `"light"`, `"dracula"`, `"pink"`, etc.) |
| `WithStylePath(path)` | `string` | Load custom style from JSON file path |
| `WithStyles(config)` | `ansi.StyleConfig` | Use a custom `StyleConfig` struct directly |
| `WithWordWrap(width)` | `int` | Set word wrap width (default: 80) |
| `WithEmoji()` | none | Enable emoji rendering using `:emoji_name:` syntax |
| `WithPreservedNewLines()` | none | Preserve newlines from input (default: collapse) |
| `WithTableWrap(wrap)` | `bool` | Enable/disable table content wrapping (default: `true`) |
| `WithInlineTableLinks(inline)` | `bool` | Render links inline in tables vs. footer references (default: `false`) |
| `WithColorProfile(profile)` | `termenv.Profile` | Set terminal color capabilities (`TrueColor`, `ANSI256`, `ANSI`, `Ascii`) |
| `WithChromaFormatter(formatter)` | `string` | Set code block syntax highlighting formatter |
| `WithEnvironmentConfig()` | none | Configure using `GLAMOUR_STYLE` environment variable |

Sources: [README.md40-65](https://github.com/charmbracelet/glamour/blob/69661fd5/README.md?plain=1#L40-L65)

Available Styles
----------------

Glamour includes several built-in styles that you can use:

| Style Name | Description |
| --- | --- |
| `dark` | Dark theme for terminals with dark backgrounds |
| `light` | Light theme for terminals with light backgrounds |
| `notty` | Theme for non-terminal environments |
| `dracula` | Dracula-inspired dark theme |
| `auto` | Automatically selects dark or light based on terminal background |

You can specify these style names when using `glamour.Render()` or set them in the `GLAMOUR_STYLE` environment variable.

Rendering Pipeline
------------------

This diagram illustrates how Glamour processes Markdown internally:

Internal Rendering Pipeline:

The rendering pipeline involves:

1.  Parsing Markdown into an Abstract Syntax Tree (AST) using the goldmark parser
2.  Configuring an ANSI renderer with style settings
3.  Processing different element types (headings, paragraphs, code blocks, etc.)
4.  Rendering the AST to ANSI-styled terminal output

Example Output
--------------

When rendering a sample Markdown document, you'll get styled terminal output. For example, rendering the contents of `testdata/example.md` would produce styled content with:

*   Emphasized text (italic, bold)
*   Lists (ordered and unordered)
*   Task lists with checkboxes
*   Tables with proper alignment
*   Code blocks with syntax highlighting
*   Block quotes with styling
*   Hyperlinks
