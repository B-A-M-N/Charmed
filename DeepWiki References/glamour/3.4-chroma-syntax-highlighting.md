This page documents Glamour's integration with the Chroma syntax highlighting library for code blocks. It covers the token type system, configuration structure, and customization of syntax highlighting styles. For general style configuration concepts, see [Style Configuration](https://deepwiki.com/charmbracelet/glamour/3.1-style-configuration)
. For information about rendering code blocks, see [Code Blocks](https://deepwiki.com/charmbracelet/glamour/4.4-code-blocks)
.

Overview
--------

Glamour uses the Chroma library (alecthomas/chroma) to provide syntax highlighting for fenced code blocks. The highlighting system defines 31 distinct token types, each with customizable styling properties. These token types are organized within the `Chroma` struct and configured as part of the `StyleCodeBlock` definition.

The syntax highlighting system supports two configuration approaches:

*   **Theme-based**: Reference a named Chroma theme via the `Theme` field
*   **Custom**: Define explicit token styles via the `Chroma` field

**Sources:** [ansi/style.go3-36](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L3-L36)
 [ansi/style.go76-81](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L76-L81)

Chroma Struct and Token Types
-----------------------------

The `Chroma` struct in [ansi/style.go3-36](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L3-L36)
 defines all available token types. Each field maps to a `StylePrimitive` that controls the visual appearance of that token category when rendered in code blocks.

**Sources:** [ansi/style.go3-36](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L3-L36)

StyleCodeBlock Integration
--------------------------

The `StyleCodeBlock` struct embeds `Chroma` configuration within the broader code block styling system. It extends `StyleBlock` and adds two fields for syntax highlighting control:

The `StyleCodeBlock` structure [ansi/style.go76-81](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L76-L81)
:

    type StyleCodeBlock struct {
        StyleBlock
        Theme  string  `json:"theme,omitempty"`
        Chroma *Chroma `json:"chroma,omitempty"`
    }
    

*   **`Theme`**: References a named Chroma theme (e.g., "monokai", "dracula")
*   **`Chroma`**: Provides explicit token-by-token styling configuration

When both fields are present, the `Chroma` field takes precedence, allowing theme-based defaults to be overridden with custom token styles.

**Sources:** [ansi/style.go76-81](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L76-L81)
 [ansi/style.go68-74](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L68-L74)

| Foreground Color | `color` | string | Text color (hex or ANSI code) |
| Background Color | `background_color` | string | Background color |
| Bold | `bold` | bool | Bold text |
| Italic | `italic` | bool | Italic text |
| Underline | `underline` | bool | Underlined text |
| Crossed Out | `crossed_out` | bool | Strikethrough text |
| Faint | `faint` | bool | Faint/dim text |
| Blink | `blink` | bool | Blinking text |
| Inverse | `inverse` | bool | Swap foreground/background |
| Conceal | `conceal` | bool | Hidden text |
| Overlined | `overlined` | bool | Overlined text |
| Upper | `upper` | bool | Convert to uppercase |
| Lower | `lower` | bool | Convert to lowercase |
| Title | `title` | bool | Convert to title case |
| Prefix/Suffix | `prefix`, `suffix` | string | Text to prepend/append |
| Block Prefix/Suffix | `block_prefix`, `block_suffix` | string | Block-level prefix/suffix |
| Format | `format` | string | Template string for formatting |

Color Format Support
--------------------

Token colors in the `Chroma` configuration support two formats:

*   **Hex Colors**: Full 6-digit hex colors (e.g., `"#FF5555"`, `"#50FA7B"`)
*   **ANSI Color Codes**: Terminal color codes as strings (e.g., `"212"`, `"244"`, `"39"`)

The hex format provides precise color control but requires terminal support for true color. The ANSI format provides compatibility with a wider range of terminals using the standard 256-color palette.
