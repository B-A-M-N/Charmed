===================

The Style Configuration system in Glamour defines how different Markdown elements are visually rendered in the terminal. This page explains the styling architecture, how styles are defined, and how they cascade through the rendering process.

For information about built-in themes, see [Built-in Themes](https://deepwiki.com/charmbracelet/glamour/3.2-built-in-themes)
, and for creating custom styles, see [Custom Styles](https://deepwiki.com/charmbracelet/glamour/3.3-custom-styles)
.

Style Configuration Overview
----------------------------

The Style Configuration in Glamour provides a structured approach to define the appearance of rendered Markdown. It allows customization of colors, formatting, indentation, margins, and other visual aspects for each Markdown element type.

Sources: [ansi/style.go98-139](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L98-L139)

StyleConfig Structure
---------------------

At the core of the styling system is the `StyleConfig` struct, which defines styles for all renderable Markdown elements.

The `StyleConfig` structure contains fields for each type of Markdown element, from document-level properties to specific elements like headings, links, and code blocks.

Sources: [ansi/style.go98-139](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L98-L139)

Style Primitives
----------------

The fundamental building block of Glamour's style system is the `StylePrimitive` struct:

Key properties in `StylePrimitive` include:

| Property | Type | Description |
| --- | --- | --- |
| BlockPrefix/BlockSuffix | string | Text to add before/after the entire block |
| Prefix/Suffix | string | Text to add before/after each line |
| Color/BackgroundColor | \*string | Text and background colors (ANSI color codes or names) |
| Bold, Italic, Underline, etc. | \*bool | Text formatting options |
| Format | string | Custom format string that can use template functions |

The pointer types (\*string, \*bool) allow for distinguishing between unset values and explicit false/empty values, which is important for style cascading.

Sources: [ansi/style.go38-59](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L38-L59)

Block Styles
------------

Block elements require additional styling properties beyond what `StylePrimitive` provides:

### StyleBlock

`StyleBlock` extends `StylePrimitive` with block-specific properties:

| Property | Type | Description |
| --- | --- | --- |
| Indent | \*uint | Amount of indentation for the block |
| IndentToken | \*string | String used for indentation (default is space) |
| Margin | \*uint | Vertical margin around the block |

Sources: [ansi/style.go68-74](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L68-L74)

### Specialized Block Styles

Glamour provides several specialized style structs for different types of block elements:

Each specialized style struct adds properties specific to its element type:

*   **StyleCodeBlock**: Adds theme selection and syntax highlighting options
*   **StyleList**: Adds indentation control for nested lists
*   **StyleTable**: Adds customization for table borders and separators
*   **StyleTask**: Adds checklist item appearance customization

Style Cascading
---------------

Glamour implements style cascading, where styles inherit and override properties at various levels. The cascading mechanism works as follows:

When styles are cascaded:

1.  The process starts with the parent style's properties
2.  Child-specific properties are applied, overriding parent values when present
3.  Properties that are not specified in the child remain inherited from the parent

Glamour uses the following cascading functions:

*   `cascadeStyles`: Cascades multiple `StyleBlock` instances
*   `cascadeStylePrimitives`: Cascades multiple `StylePrimitive` instances
*   `cascadeStylePrimitive`: Merges a parent and child `StylePrimitive`
*   `cascadeStyle`: Merges a parent and child `StyleBlock`

This cascading approach allows for concise style definitions, as elements can inherit common properties and only override what needs to be different.

Sources: [ansi/style.go141-257](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L141-L257)

Style Application in the Rendering Pipeline
-------------------------------------------

Style configuration integrates with Glamour's rendering pipeline as follows:

1.  The TermRenderer is created with a StyleConfig
2.  The Markdown is parsed into an Abstract Syntax Tree (AST)
3.  Element renderers apply styles from StyleConfig to each node in the AST
4.  The result is ANSI-styled text ready for terminal display

The Style Configuration acts as a central reference that determines how each element appears in the final output.

Sources: [ansi/style.go98-139](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/style.go#L98-L139)

Summary
-------

The Style Configuration system in Glamour provides a robust and flexible way to define how Markdown elements are rendered in the terminal. Through a hierarchy of style structures, cascading inheritance, and JSON-based configuration, it allows for both simple theming and advanced customization of the rendering process.
