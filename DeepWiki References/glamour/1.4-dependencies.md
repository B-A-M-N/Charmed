Dependencies
============

Relevant source files

This page documents all external libraries that Glamour depends on, their versions, and their specific roles in the rendering pipeline. For information about how these dependencies are integrated into the architecture, see [Architecture Overview](https://deepwiki.com/charmbracelet/glamour/1.3-architecture-overview)

Overview
--------

Glamour relies on a carefully selected set of external libraries to handle markdown parsing, terminal styling, syntax highlighting, text wrapping, and terminal environment detection. Dependencies are categorized into direct dependencies (explicitly imported by Glamour) and indirect dependencies (transitive requirements).

**Sources:** [go.mod1-37](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L1-L37)

Core Rendering Dependencies
---------------------------

These libraries form the backbone of Glamour's markdown-to-terminal rendering pipeline.

### goldmark (v1.7.13)

**Package:** `github.com/yuin/goldmark`  
**Role:** Markdown parser that converts markdown text into an Abstract Syntax Tree (AST)

Goldmark is the primary markdown parser used by Glamour. It provides:

*   CommonMark-compliant parsing
*   Extensible architecture for custom renderers
*   Support for GitHub Flavored Markdown (GFM) extensions
*   AST representation of markdown documents

The `ANSIRenderer` in Glamour walks the goldmark AST and dispatches to element-specific renderers. See [Rendering Lifecycle](https://deepwiki.com/charmbracelet/glamour/4.1-rendering-lifecycle)
 for details on AST traversal.

**Sources:** [go.mod14](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L14-L14)

### goldmark-emoji (v1.0.6)

**Package:** `github.com/yuin/goldmark-emoji`  
**Role:** Goldmark extension for emoji shortcode support

Adds emoji parsing capabilities to goldmark, allowing markdown like `:smile:` to be rendered as emoji characters. Can be enabled/disabled via `WithEmoji()` option.

**Sources:** [go.mod15](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L15-L15)

### lipgloss (v1.1.1)

**Package:** `github.com/charmbracelet/lipgloss`  
**Role:** Terminal styling framework for applying colors, formatting, and layout

Lipgloss provides the styling primitives used throughout Glamour:

*   Color application (foreground/background)
*   Text formatting (bold, italic, underline, strikethrough)
*   Layout primitives (borders, margins, padding)
*   ANSI escape code generation

The `StylePrimitive` and `StyleBlock` types in Glamour's style system map directly to lipgloss styling functions.

**Sources:** [go.mod7](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L7-L7)

### chroma (v2.20.0)

**Package:** `github.com/alecthomas/chroma/v2`  
**Role:** Syntax highlighter for code blocks

Chroma provides syntax highlighting for code blocks with:

*   Support for 200+ programming languages
*   Multiple color schemes/themes
*   Token-level styling (keywords, strings, comments, etc.)
*   Integration via `StyleCodeBlock.Chroma` configuration

See [Chroma Syntax Highlighting](https://deepwiki.com/charmbracelet/glamour/3.4-chroma-syntax-highlighting)
 for detailed configuration options.

**Sources:** [go.mod6](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L6-L6)

Terminal Environment Dependencies
---------------------------------

These libraries handle terminal detection, environment capabilities, and terminal I/O operations.

### termenv (v0.16.0)

**Package:** `github.com/muesli/termenv`  
**Role:** Terminal environment detection and capability detection

Termenv detects:

*   Terminal color profile (TrueColor, ANSI256, ANSI, ASCII)
*   Background color (light/dark detection)
*   Terminal features and capabilities
*   ANSI support level

Used by `WithAutoStyle()` to automatically select appropriate themes based on terminal capabilities.

**Sources:** [go.mod13](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L13-L13)

### golang.org/x/term (v0.36.0)

**Package:** `golang.org/x/term`  
**Role:** Terminal I/O and size detection

Provides low-level terminal operations:

*   Terminal size detection (width/height)
*   Raw mode control
*   Terminal state management

Used to determine default word wrap width when not explicitly specified.

**Sources:** [go.mod16](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L16-L16)

### charmbracelet/x/ansi (v0.10.2)

**Package:** `github.com/charmbracelet/x/ansi`  
**Role:** ANSI escape sequence utilities

Provides utilities for working with ANSI escape codes:

*   ANSI code stripping
*   Text width calculation (ignoring ANSI codes)
*   ANSI sequence parsing

**Sources:** [go.mod8](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L8-L8)

Text Processing Dependencies
----------------------------

These libraries handle text wrapping, layout, and Unicode operations.

### reflow (v0.3.0)

**Package:** `github.com/muesli/reflow`  
**Role:** Text wrapping and reflow operations

Reflow provides:

*   Word wrapping with configurable width
*   Indentation and padding support
*   Line breaking algorithms
*   Support for ANSI-styled text

The `WordWrap` function from reflow is used throughout element rendering to fit text to terminal width. The `MarginWriter` wraps reflow functionality for consistent indentation.

**Sources:** [go.mod12](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L12-L12)

### golang.org/x/text (v0.30.0)

**Package:** `golang.org/x/text`  
**Role:** Unicode text processing and transformation

Provides:

*   Unicode normalization
*   Text transformation
*   Encoding detection and conversion
*   Language-specific text processing

**Sources:** [go.mod17](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L17-L17)

Dependency Management
---------------------

Glamour uses Go modules for dependency management with:

*   **Minimum Go version:** 1.24.0
*   **Dependency updates:** Automated via Dependabot (weekly checks)
*   **Version pinning:** Direct dependencies use semantic versioning; some development dependencies use commit hashes
*   **License compliance:** All dependencies use permissive licenses compatible with MIT

For development workflow and dependency updates, see [Contributing](https://deepwiki.com/charmbracelet/glamour/7.3-contributing)
.
