Purpose and Scope
-----------------

This document describes the table rendering system in Glamour, which converts Markdown tables into styled ANSI terminal output. The system handles cell content rendering, alignment, borders, and a specialized feature for rendering links and images as footer references.

For information about the general element rendering lifecycle, see [Rendering Lifecycle](https://deepwiki.com/charmbracelet/glamour/4.1-rendering-lifecycle)
. For details on how links and images are rendered outside of tables, see [Links and Images](https://deepwiki.com/charmbracelet/glamour/4.5-links-and-images)
.

Table Rendering Architecture
----------------------------

**Sources:** [ansi/table.go1-200](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L1-L200)
 [ansi/table\_links.go1-236](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L1-L236)

The table rendering system uses a two-level architecture: Glamour's element types (`TableElement`, `TableRowElement`, `TableHeadElement`, `TableCellElement`) parse the Goldmark AST and manage state, while the `lipgloss/table.Table` library handles the actual grid layout and ANSI rendering. The `TableElement` maintains link collections that are rendered as footer references after the table completes.

Core Components
---------------

### TableElement

The `TableElement` struct is the primary container for table rendering state:

| Field | Type | Purpose |
| --- | --- | --- |
| `lipgloss` | `*table.Table` | Lipgloss table instance for rendering |
| `table` | `*astext.Table` | Reference to Goldmark AST node |
| `header` | `[]string` | Accumulated header cell content |
| `row` | `[]string` | Accumulated row cell content |
| `source` | `[]byte` | Original markdown source for link extraction |
| `tableImages` | `[]tableLink` | Collected images for footer rendering |
| `tableLinks` | `[]tableLink` | Collected links for footer rendering |

**Sources:** [ansi/table.go14-24](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L14-L24)

The `TableElement` is stored in the `RenderContext.table` field and accessed by child elements during rendering. Links and images are collected during the `Render` phase [ansi/table.go68-70](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L68-L70)
 and printed in the `Finish` phase [ansi/table.go141](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L141-L141)

### TableCellElement

The `TableCellElement` renders individual cell content:

    type TableCellElement struct {
        Children []ElementRenderer
        Head     bool
    }
    

**Sources:** [ansi/table.go32-36](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L32-L36)

The `Children` field contains nested element renderers (text, links, images, etc.) that make up the cell content. The `Head` boolean determines whether content is appended to the header or row buffer [ansi/table.go192-196](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L192-L196)

During rendering, cells iterate through their children and apply table-specific styling via `StyleOverrideRender` [ansi/table.go173-176](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L173-L176)
 ensuring consistent appearance regardless of the element type within the cell.

### TableRowElement and TableHeadElement

These lightweight elements serve as lifecycle hooks:

*   `TableRowElement`: Appends the accumulated `ctx.table.row` to the lipgloss table [ansi/table.go152](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L152-L152)
    
*   `TableHeadElement`: Sets the accumulated `ctx.table.header` as table headers [ansi/table.go163](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L163-L163)
    

**Sources:** [ansi/table.go26-30](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L26-L30)
 [ansi/table.go146-166](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L146-L166)

Rendering Lifecycle
-------------------

**Sources:** [ansi/table.go38-73](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L38-L73)
 [ansi/table.go120-144](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L120-L144)
 [ansi/table.go168-199](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L168-L199)

The rendering process follows this sequence:

1.  **Initialization** (`Render` phase): Creates lipgloss table instance, sets dimensions, collects all links/images
2.  **Cell Accumulation**: Cells render their children into string buffers and append to header or row arrays
3.  **Row Completion**: After each row's cells complete, `TableRowElement.Finish()` calls `lipgloss.Row()`
4.  **Finalization** (`Finish` phase): Applies styles, borders, renders the lipgloss table, then prints footer links

Cell Alignment
--------------

Cell alignment is determined by the Goldmark table's `Alignments` field and applied via lipgloss styles:

**Sources:** [ansi/table.go85-94](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L85-L94)

The alignment is set in `setStyles()` using a `StyleFunc` that receives the row and column indices [ansi/table.go76](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L76-L76)
 and returns the appropriate lipgloss style. Default margins are set at 0,1 [ansi/table.go79](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L79-L79)
 but can be overridden by the style configuration [ansi/table.go82-84](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L82-L84)

Example output from [ansi/testdata/TestRenderer/table\_align.golden1-8](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/testdata/TestRenderer/table_align.golden#L1-L8)
:

         Label                │          Value          │                 URL       
    ──────────────────────────┼─────────────────────────┼─────────────────────────  
         First                │           foo           │            charm.sh       
         Second               │           bar           │         charm.sh[1]       
    

Borders and Separators
----------------------

Table borders are configured through the `StyleTable` configuration and applied via lipgloss borders:

**Sources:** [ansi/table.go100-118](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L100-L118)

The border configuration uses five separators:

*   `RowSeparator`: Horizontal lines (top/bottom)
*   `ColumnSeparator`: Vertical lines (left/right)
*   `CenterSeparator`: Cross junction for interior cells

By default, lipgloss uses `NormalBorder()` [ansi/table.go102](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L102-L102)
 but external borders are disabled [ansi/table.go114-117](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L114-L117)
 leaving only the interior separators visible. This creates the characteristic table appearance with column dividers and row separators.

Links and Images in Tables
--------------------------

A distinctive feature of Glamour's table rendering is automatic footer reference generation for links and images within table cells. This keeps the table grid clean while preserving link information.

### Link Collection

The `collectLinksAndImages()` method walks the table AST and extracts three types of references:

| Link Type | Detection | Content Display |
| --- | --- | --- |
| `linkTypeAuto` | `ast.AutoLink` nodes | Domain extracted via `linkDomain()` or shortened by autolink detection |
| `linkTypeRegular` | `ast.Link` nodes | Link text from AST node content |
| `linkTypeImage` | `ast.Image` nodes | Alt text or domain if empty |

**Sources:** [ansi/table\_links.go122-181](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L122-L181)

The collection process uses `ast.Walk()` to traverse the table subtree [ansi/table\_links.go126](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L126-L126)
 extracting URLs from nodes [ansi/table\_links.go133-170](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L133-L170)
 and building `tableLink` structs [ansi/table\_links.go19-24](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L19-L24)
 Duplicate links are removed via `slice.Uniq()` [ansi/table\_links.go178-179](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L178-L179)

### Footer Reference Format

**Sources:** [ansi/table\_links.go35-110](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L35-L110)

Footer links are rendered below the table with numbered references. The format depends on link type:

*   **Regular/Auto links**: `[N]: content url`
*   **Images**: `[N]: Image: domain → url`

The link text includes position-based padding [ansi/table\_links.go82-87](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L82-L87)
 to align reference numbers vertically. URLs are truncated if they exceed available terminal width [ansi/table\_links.go71-72](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L71-L72)

Example from [ansi/testdata/TestRendererIssues/316.golden1-11](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/testdata/TestRendererIssues/316.golden#L1-L11)
:

                                   Here[1] │ hello
                           autolink.com[2] │ world
    
    [1]: Here https://example.com
    [2]: autolink.com https://autolink.com
    

### Inline vs Footer Links

The rendering mode is controlled by `ctx.options.InlineTableLinks`:

**Sources:** [ansi/table\_links.go112-120](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L112-L120)

When `InlineTableLinks` is false (default), links appear as numbered references with footer details. When true, links render inline within cells, potentially creating wider tables.

Table Styling
-------------

Table appearance is controlled by the `StyleTable` configuration structure:

| Style Property | Type | Effect |
| --- | --- | --- |
| `StylePrimitive` | `StylePrimitive` | Text color, bold, italic for cell content |
| `Indent` | `*uint` | Left indentation for entire table |
| `Margin` | `*uint` | Additional left margin; also used for cell padding |
| `Prefix`/`Suffix` | `string` | Text before/after table content |
| `BlockPrefix`/`BlockSuffix` | `string` | Text before/after entire table block |
| `RowSeparator` | `*string` | Horizontal separator character |
| `ColumnSeparator` | `*string` | Vertical separator character |
| `CenterSeparator` | `*string` | Junction character for intersections |

**Sources:** [ansi/table.go44-60](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L44-L60)
 [ansi/table.go75-98](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L75-L98)

The table style is accessed via `ctx.options.Styles.Table` [ansi/table.go44](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L44-L44)
 and applied at multiple stages:

*   Indentation/margin during initial rendering [ansi/table.go45-54](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L45-L54)
    
*   Cell-level styling in `StyleFunc` [ansi/table.go76-97](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L76-L97)
    
*   Border configuration [ansi/table.go100-118](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L100-L118)
    
*   Prefix/suffix text [ansi/table.go58-59](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L58-L59)
     [ansi/table.go138-139](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L138-L139)
    

Layout and Wrapping
-------------------

Table width and wrapping behavior are determined by the rendering context:

**Sources:** [ansi/table.go60-66](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L60-L66)

The table width is constrained by the `BlockStack` width calculation [ansi/table.go60](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L60-L60)
 which accounts for parent element indentation and margins (see [BlockStack](https://deepwiki.com/charmbracelet/glamour/5.1-blockstack)
). The `TableWrap` option controls whether cell content wraps to multiple lines or truncates.

Indentation is applied using the `indent.NewWriterPipe` wrapper [ansi/table.go52-54](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L52-L54)
 which prepends spaces before each line of table output. The indentation amount combines the style's `Indent` and `Margin` values [ansi/table.go42-50](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L42-L50)

**Sources:** [ansi/table.go38-73](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table.go#L38-L73)
 [ansi/margin.go1-58](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/margin.go#L1-L58)
