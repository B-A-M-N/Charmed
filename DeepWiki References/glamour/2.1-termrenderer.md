Purpose and Scope
-----------------

This document describes the `TermRenderer` type, which is the primary interface for rendering Markdown content to ANSI-styled terminal output in Glamour. `TermRenderer` acts as a configuration wrapper around the lower-level rendering pipeline, managing options, styling, and the interaction with the Goldmark parser and ANSIRenderer.

For information about the underlying ANSI rendering engine that TermRenderer delegates to, see [Element Renderers](https://deepwiki.com/charmbracelet/glamour/4-element-renderers)
. For details about configuring styles and themes, see [Styling System](https://deepwiki.com/charmbracelet/glamour/3-styling-system)
. For simple one-off rendering functions that don't require creating a TermRenderer instance, see [Render Functions](https://deepwiki.com/charmbracelet/glamour/2.2-render-functions)
.

**Sources:** [glamour.go1-323](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L1-L323)

Overview
--------

The `TermRenderer` type serves as the main entry point for Glamour's rendering functionality. It combines three key responsibilities:

1.  **Configuration Management**: Stores rendering options including word wrap width, color profile, style configuration, and various feature flags
2.  **Parser Integration**: Wraps a configured `goldmark.Markdown` instance with appropriate extensions (GFM, definition lists) and an `ANSIRenderer`
3.  **Rendering Interface**: Provides both simple (`Render`, `RenderBytes`) and streaming (`Write`, `Read`, `Close`) APIs for converting Markdown to styled output

**Sources:** [glamour.go34-41](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L34-L41)
 [glamour.go70-100](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L70-L100)

TermRenderer Structure
----------------------

The `TermRenderer` struct contains four fields that manage the rendering state:

| Field | Type | Purpose |
| --- | --- | --- |
| `md` | `goldmark.Markdown` | Configured Goldmark parser with ANSIRenderer attached |
| `ansiOptions` | `ansi.Options` | Configuration passed to ANSIRenderer (styles, width, colors) |
| `buf` | `bytes.Buffer` | Input buffer for streaming Write/Close/Read API |
| `renderBuf` | `bytes.Buffer` | Output buffer for streaming Write/Close/Read API |

**Sources:** [glamour.go36-41](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L36-L41)

Construction with NewTermRenderer
---------------------------------

The `NewTermRenderer` function creates and configures a new `TermRenderer` instance by applying a variadic list of `TermRendererOption` functions.

### Default Configuration

When created without options, `NewTermRenderer` initializes with:

*   **Goldmark Extensions**: `extension.GFM` and `extension.DefinitionList` enabled
*   **Parser Options**: `parser.WithAutoHeadingID()` enabled
*   **Word Wrap**: `80` characters (constant `defaultWidth`)
*   **Color Profile**: `termenv.TrueColor`
*   **Priority**: `1000` (constant `highPriority`) for the ANSIRenderer registration

**Sources:** [glamour.go70-100](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L70-L100)
 [glamour.go26-29](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L26-L29)

Configuration Options
---------------------

`TermRenderer` is configured through the functional options pattern. Each `TermRendererOption` is a function with signature `func(*TermRenderer) error` that modifies the renderer's configuration.

### Style Configuration Options

| Option Function | Purpose | Example |
| --- | --- | --- |
| `WithStandardStyle(style string)` | Use a built-in style ("dark", "light", "dracula", etc.) | `WithStandardStyle("dark")` |
| `WithAutoStyle()` | Auto-detect terminal background and use dark or light style | `WithAutoStyle()` |
| `WithEnvironmentConfig()` | Read style from `GLAMOUR_STYLE` environment variable | `WithEnvironmentConfig()` |
| `WithStylePath(stylePath string)` | Load style from JSON file or use built-in style name | `WithStylePath("mystyle.json")` |
| `WithStyles(styles ansi.StyleConfig)` | Set style configuration directly | `WithStyles(customConfig)` |
| `WithStylesFromJSONBytes(jsonBytes []byte)` | Parse style from JSON bytes | `WithStylesFromJSONBytes(data)` |
| `WithStylesFromJSONFile(filename string)` | Load and parse style from JSON file | `WithStylesFromJSONFile("style.json")` |

**Sources:** [glamour.go119-188](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L119-L188)

### Layout and Formatting Options

| Option Function | Purpose | Default |
| --- | --- | --- |
| `WithWordWrap(wordWrap int)` | Set line wrapping width in characters | `80` |
| `WithTableWrap(tableWrap bool)` | Enable/disable table content wrapping | `true` |
| `WithInlineTableLinks(inlineTableLinks bool)` | Render table links inline vs. as footer references | `false` |
| `WithPreservedNewLines()` | Preserve newlines from source Markdown | `false` |

**Sources:** [glamour.go190-223](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L190-L223)

### Terminal and Rendering Options

| Option Function | Purpose |
| --- | --- |
| `WithColorProfile(profile termenv.Profile)` | Set color profile (TrueColor, ANSI256, ANSI) |
| `WithBaseURL(baseURL string)` | Set base URL for resolving relative links |
| `WithEmoji()` | Enable emoji rendering (adds goldmark-emoji extension) |
| `WithChromaFormatter(formatter string)` | Set Chroma formatter for code syntax highlighting |

**Sources:** [glamour.go102-239](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L102-L239)

Rendering Methods
-----------------

`TermRenderer` provides two primary rendering methods for immediate, stateless conversion of Markdown to styled output.

### Render Method

The `Render` method [glamour.go285-288](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L285-L288)
 is a convenience wrapper that:

1.  Converts the input string to `[]byte`
2.  Calls `RenderBytes`
3.  Converts the output back to `string`

### RenderBytes Method

The `RenderBytes` method [glamour.go291-295](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L291-L295)
 is the core rendering implementation:

1.  Creates a temporary `bytes.Buffer`
2.  Calls `tr.md.Convert(in, &buf)` which runs the Goldmark parser and ANSIRenderer
3.  Returns `buf.Bytes()` and any error from conversion

Both methods are stateless and can be called multiple times on the same `TermRenderer` instance.

**Sources:** [glamour.go284-295](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L284-L295)

Streaming API
-------------

For processing Markdown in chunks or integrating with I/O pipelines, `TermRenderer` implements a streaming API through the `io.Writer` and `io.Reader` interfaces.

### Write Method

The `Write` method [glamour.go264-270](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L264-L270)
 implements `io.Writer`:

*   Accepts a byte slice and writes it to the internal `buf` buffer
*   Returns the number of bytes written and any error
*   Can be called multiple times to accumulate input

### Close Method

The `Close` method [glamour.go274-282](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L274-L282)
 triggers the actual rendering:

*   Calls `tr.md.Convert(tr.buf.Bytes(), &tr.renderBuf)` to render accumulated input
*   Resets `tr.buf` to free memory
*   Must be called before reading output
*   Returns any error from the conversion process

### Read Method

The `Read` method [glamour.go253-262](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L253-L262)
 implements `io.Reader`:

*   Reads rendered output from `tr.renderBuf`
*   Returns number of bytes read and `io.EOF` when exhausted
*   Can only be called after `Close()`

**Sources:** [glamour.go253-282](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L253-L282)
 [glamour\_test.go19-47](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour_test.go#L19-L47)

Relationship to Other Components
--------------------------------

The `TermRenderer` orchestrates several components:

1.  **Wraps `goldmark.Markdown`**: Configured with GFM extensions and custom renderer
2.  **Configures `ANSIRenderer`**: Passes `ansi.Options` to the ANSI rendering engine
3.  **Manages Buffers**: Handles input/output buffering for both simple and streaming APIs
4.  **Used by Package-Level Functions**: The `Render` and `RenderBytes` functions [glamour.go45-67](https://github.com/charmbracelet/glamour/blob/69661fd5/glamour.go#L45-L67)
     internally create temporary `TermRenderer` instances
