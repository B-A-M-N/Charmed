Lists and Tasks
===============

Relevant source files

This document describes how Glamour renders Markdown lists, including ordered (numbered) lists, unordered (bulleted) lists, and task lists with checkboxes. It covers the element dispatching logic, the rendering lifecycle for list items, and how indentation works for nested lists.

For information about other block-level elements, see [Block Elements](https://deepwiki.com/charmbracelet/glamour/4.2-block-elements)
. For details on the BlockStack indentation system, see [BlockStack](https://deepwiki.com/charmbracelet/glamour/5.1-blockstack)
.

* * *

Overview
--------

Glamour handles three types of list-related Markdown elements:

*   **Unordered lists** - Rendered with bullets (e.g., `•`, `-`)
*   **Ordered lists** - Rendered with enumeration (e.g., `1.`, `2.`, `3.`)
*   **Task lists** - Rendered with checkboxes (e.g., `☐`, `☑`)

These elements are parsed by Goldmark into AST nodes (`ast.KindList`, `ast.KindListItem`, `astext.KindTaskCheckBox`) and then dispatched to specialized renderers (`ItemElement`, `TaskElement`).

List Item Rendering
-------------------

### ItemElement Structure

The `ItemElement` struct handles both ordered and unordered list items:

| Field | Type | Purpose |
| --- | --- | --- |
| `IsOrdered` | `bool` | Whether the list is ordered (numbered) |
| `Enumeration` | `uint` | The item number for ordered lists |

**Sources:** [ansi/listitem.go9-12](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/listitem.go#L9-L12)

### Item Numbering Logic

For ordered lists, Glamour calculates the enumeration number by:

1.  Counting siblings via `PreviousSibling()` traversal
2.  Adding the list's `Start` value (if not 1)

**Sources:** [ansi/elements.go127-168](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L127-L168)

### Rendering Output

When `ItemElement.Render()` is called:

**For unordered lists:**

    BaseElement with Styles.Item (typically "• " or "- ")
    

**For ordered lists:**

    BaseElement with Styles.Enumeration and Prefix = "1.", "2.", etc.
    

The style configuration determines the actual bullet character or number formatting.

**Sources:** [ansi/listitem.go15-29](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/listitem.go#L15-L29)

Task Lists
----------

### Task Checkbox Detection

Task lists are detected when a `ListItem` has a `TaskCheckBox` as its first grandchild:

This traversal happens because Goldmark's AST structure places the `TaskCheckBox` node as:

    ListItem
      └─ (first child - paragraph or text block)
          └─ TaskCheckBox (first child of that)
    

**Sources:** [ansi/elements.go149-159](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L149-L159)

### TaskElement Rendering

The `TaskElement` struct is simple:

| Field | Type | Purpose |
| --- | --- | --- |
| `Checked` | `bool` | Whether the checkbox is checked |

When rendered, it selects the appropriate prefix from style configuration:

The actual checkbox characters (e.g., `☐` vs `☑`, or `[ ]` vs `[x]`) are defined in the style configuration via `Styles.Task.Ticked` and `Styles.Task.Unticked`.

**Sources:** [ansi/task.go1-28](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/task.go#L1-L28)

List Nesting and Indentation
----------------------------

### BlockStack Integration

Lists use the `BlockStack` to manage nested indentation. When a list is pushed onto the stack:

The `BlockStack.Indent()` method sums all `Style.Indent` values from elements in the stack, providing the cumulative indentation for the current rendering position.

**Sources:** [ansi/blockstack.go33-44](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockstack.go#L33-L44)

### Available Width Calculation

As lists nest, the available rendering width decreases:

    available_width = WordWrap - BlockStack.Indent() - BlockStack.Margin() * 2
    

This ensures that deeply nested lists don't overflow the terminal width.

**Sources:** [ansi/blockstack.go61-66](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/blockstack.go#L61-L66)

### Post-Item Newlines

List items control their trailing newline behavior based on context:

| Condition | Post-Item String |
| --- | --- |
| Last child is a nested list | `""` (no newline) |
| Last item in list | `""` (no newline) |
| Otherwise | `"\n"` (add newline) |

This logic creates proper spacing between list items while avoiding extra blank lines at list boundaries.

**Sources:** [ansi/elements.go143-147](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/elements.go#L143-L147)

Style Configuration Reference
-----------------------------

### List Styles

Lists reference these style configuration paths:

| Style Path | Purpose | Type |
| --- | --- | --- |
| `Styles.List.StyleBlock` | Container block styles | `StyleBlock` |
| `Styles.List.LevelIndent` | Indentation per nesting level | `uint` |
| `Styles.Item` | Unordered list item prefix | `StylePrimitive` |
| `Styles.Enumeration` | Ordered list number formatting | `StylePrimitive` |

### Task Styles

Task lists reference:

| Style Path | Purpose | Type |
| --- | --- | --- |
| `Styles.Task.StylePrimitive` | Base task item styling | `StylePrimitive` |
| `Styles.Task.Ticked` | Checked checkbox prefix (e.g., `"[✓] "`) | `string` |
| `Styles.Task.Unticked` | Unchecked checkbox prefix (e.g., `"[ ] "`) | `string` |

These styles are typically defined in theme JSON files (see [Built-in Themes](https://deepwiki.com/charmbracelet/glamour/3.2-built-in-themes)
 and [Custom Styles](https://deepwiki.com/charmbracelet/glamour/3.3-custom-styles)
).
