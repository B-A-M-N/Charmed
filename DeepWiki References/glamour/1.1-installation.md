This document provides comprehensive instructions for installing and integrating the Glamour library into Go projects. Glamour is a stylesheet-based Markdown rendering library for terminal applications. For information about using Glamour after installation, see [Quick Start](https://deepwiki.com/charmbracelet/glamour/1.2-quick-start)
.

Prerequisites
-------------

Before installing Glamour, ensure you have:

*   Go 1.24.0 or later (required minimum version)
*   A Go project initialized with `go mod init`

Sources: [go.mod3](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L3-L3)

Dependency Structure
--------------------

When you install Glamour, it brings in several direct dependencies that handle various aspects of Markdown rendering and terminal styling:

| Dependency | Version | Purpose |
| --- | --- | --- |
| `github.com/yuin/goldmark` | v1.7.13 | Markdown parsing to AST |
| `github.com/yuin/goldmark-emoji` | v1.0.6 | Emoji support for Markdown |
| `github.com/alecthomas/chroma/v2` | v2.20.0 | Syntax highlighting for code blocks |
| `github.com/charmbracelet/lipgloss` | v1.1.1 | Terminal styling and ANSI rendering |
| `github.com/charmbracelet/x/ansi` | v0.10.2 | ANSI escape code utilities |
| `github.com/muesli/termenv` | v0.16.0 | Terminal environment detection |
| `github.com/muesli/reflow` | v0.3.0 | Text wrapping and reflow |
| `github.com/microcosm-cc/bluemonday` | v1.0.27 | HTML sanitization |
| `golang.org/x/term` | v0.36.0 | Terminal control |
| `golang.org/x/text` | v0.30.0 | Text processing |

Sources: [go.mod5-18](https://github.com/charmbracelet/glamour/blob/69661fd5/go.mod#L5-L18)

Common Installation Issues
--------------------------

### Dependency Conflicts

If you encounter dependency conflicts, try these solutions:

1.  Run `go mod tidy` to resolve dependency issues
2.  Check compatibility with other dependencies that might use different versions of shared libraries
3.  Use `go get github.com/charmbracelet/glamour@vX.Y.Z` to install a specific version

### Importing Issues

If you have issues importing Glamour:

1.  Ensure your imports are correctly formatted: `import "github.com/charmbracelet/glamour"`
2.  Check that your module can access the dependency (no network issues or proxy problems)

Using Vendored Dependencies
---------------------------

If you prefer to vendor your dependencies:

This command will copy all dependencies, including Glamour, into a `vendor` directory in your project, ensuring availability even without external network access.

API Entry Points
----------------

After installation, Glamour provides two primary API entry points:

### Quick Rendering Functions

| Function | Signature | Purpose |
| --- | --- | --- |
| `glamour.Render()` | `func(in, styleName string) (string, error)` | One-shot rendering with named style |
| `glamour.RenderBytes()` | `func(in []byte, styleName string) ([]byte, error)` | One-shot rendering from bytes |
| `glamour.RenderWithEnvironmentConfig()` | `func(in string) (string, error)` | Render using `GLAMOUR_STYLE` environment variable |
