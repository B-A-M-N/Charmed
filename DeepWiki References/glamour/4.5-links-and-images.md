This page documents how Glamour renders Markdown links and images to styled terminal output. It covers the `LinkElement` and `ImageElement` renderers, URL resolution, and the special table footer links feature that consolidates links and images outside of table cells.

For information about table rendering structure, see [Tables](https://deepwiki.com/charmbracelet/glamour/4.3-tables)
. For general element rendering lifecycle, see [Rendering Lifecycle](https://deepwiki.com/charmbracelet/glamour/4.1-rendering-lifecycle)
.

* * *

Overview
--------

Glamour handles three types of link-like elements:

*   **Regular links** - Standard Markdown `<FileRef file-url="https://github.com/charmbracelet/glamour/blob/69661fd5/text" undefined file-path="text">Hii</FileRef>` syntax
*   **Autolinks** - URLs wrapped in angle brackets `<url>`
*   **Images** - Image syntax `!<FileRef file-url="https://github.com/charmbracelet/glamour/blob/69661fd5/alt" undefined file-path="alt">Hii</FileRef>`

Links and images are rendered with separate styling for their text content and their URL/href portions. When links or images appear inside tables, they can optionally be extracted to a footer section below the table to prevent cell overflow.

**Sources**: [ansi/link.go1-77](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L1-L77)
 [ansi/image.go1-53](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/image.go#L1-L53)
 [ansi/table\_links.go1-236](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L1-L236)

Link Element Architecture
-------------------------

### LinkElement Structure

The `LinkElement` type handles hyperlink rendering with two distinct parts: the link text and the URL.

| Field | Type | Purpose |
| --- | --- | --- |
| `BaseURL` | `string` | Base URL for resolving relative links |
| `URL` | `string` | The hyperlink destination |
| `Children` | `[]ElementRenderer` | Child elements that form the link text |
| `SkipText` | `bool` | If true, omit rendering the link text |
| `SkipHref` | `bool` | If true, omit rendering the URL |

**Sources**: [ansi/link.go10-17](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L10-L17)

* * *

### Link Text Rendering

The link text is rendered by processing child elements with the `LinkText` style applied. The implementation supports two rendering paths:

The key logic in [ansi/link.go34-56](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L34-L56)
 handles two cases:

1.  If the child implements `StyleOverriderElementRenderer`, it calls `StyleOverrideRender` with the `LinkText` style
2.  Otherwise, it renders the child to a buffer and wraps it in a `BaseElement` with the `LinkText` style

**Sources**: [ansi/link.go34-56](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L34-L56)

* * *

### URL/Href Rendering

The href portion is rendered after the text with special handling for anchor-only links:

Key behaviors:

*   **Anchor-only links** (e.g., `#section`) are ignored and not rendered ([ansi/link.go65](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L65-L65)
    )
*   **Relative URLs** are resolved against the `BaseURL` using `resolveRelativeURL()`
*   A **space prefix** is added if text was rendered (`!e.SkipText`) ([ansi/link.go59-62](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L59-L62)
    )

**Sources**: [ansi/link.go58-76](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L58-L76)

Image Element Architecture
--------------------------

### ImageElement Structure

Images are rendered with alt text and an optional URL arrow indicator:

The `TextOnly` flag controls whether the image URL is rendered. When true, the arrow suffix (`" →"`) is stripped from the `ImageText` style format ([ansi/image.go20-22](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/image.go#L20-L22)
).

**Sources**: [ansi/image.go8-52](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/image.go#L8-L52)

* * *

### Image Rendering Flow

| Step | Condition | Action |
| --- | --- | --- |
| 1   | `len(e.Text) > 0` | Render text with `Styles.ImageText` |
| 2   | `e.TextOnly == true` | Strip `" →"` suffix from format and return |
| 3   | `len(e.URL) > 0` | Resolve URL and render with `Styles.Image` |

**Example Output** (from default dark style):

    Image: Example → https://example.com/image.png
    

**Sources**: [ansi/image.go18-52](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/image.go#L18-L52)

Table Footer Links Feature
--------------------------

### Purpose and Architecture

When links or images appear in table cells, they can cause cell overflow and formatting issues. Glamour provides a footer links feature that extracts these elements to a numbered reference list below the table.

**Sources**: [ansi/table\_links.go35-110](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L35-L110)
 [ansi/table\_links.go122-181](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L122-L181)

* * *

### tableLink Structure

The internal `tableLink` type stores extracted link information:

Link types are distinguished by a `linkType` enum ([ansi/table\_links.go26-33](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L26-L33)
):

| Type | Value | Description |
| --- | --- | --- |
| `linkTypeAuto` | 1   | Autolink `<url>` |
| `linkTypeImage` | 2   | Image `!<FileRef file-url="https://github.com/charmbracelet/glamour/blob/69661fd5/alt" undefined file-path="alt">Hii</FileRef>` |
| `linkTypeRegular` | 3   | Regular link `<FileRef file-url="https://github.com/charmbracelet/glamour/blob/69661fd5/text" undefined file-path="text">Hii</FileRef>` |

**Sources**: [ansi/table\_links.go19-33](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L19-L33)

* * *

### Link Collection Algorithm

The collection process walks the table's AST and extracts all links and images:

Key functions:

*   **`nodeContent()`** ([ansi/table\_links.go196-220](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L196-L220)
    ) - Recursively extracts text content from AST nodes
*   **`linkDomain()`** ([ansi/table\_links.go222-227](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L222-L227)
    ) - Extracts hostname from URL as fallback content
*   **`autolink.Detect()`** - Shortens common platform URLs (e.g., "github.com/user/repo")

**Sources**: [ansi/table\_links.go122-181](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L122-L181)
 [ansi/table\_links.go196-227](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L196-L227)

* * *

### Footer Rendering Format

The footer renders two separate numbered lists:

**Example output** from [ansi/testdata/TestRenderer/table\_with\_footer\_links\_complex.golden39-62](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/testdata/TestRenderer/table_with_footer_links_complex.golden#L39-L62)
:

    [1]: @andreynering https://github.com/andreynering
    [2]: GitHub projects https://github.com/andreynering
    ...
    
    [1]: Image: github.com → https://github.com/andreynering.png
    [2]: Image: github.com → https://github.com/aymanbagabas.png
    

The rendering includes smart width truncation: URLs are truncated with "…" if they exceed available terminal width ([ansi/table\_links.go71-72](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L71-L72)
).

**Sources**: [ansi/table\_links.go35-110](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L35-L110)

Configuration
-------------

### Style Options

Links and images use four distinct style configurations:

| Style Property | Applied To | Default Colors (Dark) |
| --- | --- | --- |
| `Styles.LinkText` | Link text content | Green, bold |
| `Styles.Link` | Link URL/href | Cyan, underline |
| `Styles.ImageText` | Image alt text + arrow | Gray |
| `Styles.Image` | Image URL | Magenta, underline |

**Sources**: Configuration details in [Styling System](https://deepwiki.com/charmbracelet/glamour/3-styling-system)

* * *

### InlineTableLinks Option

The `InlineTableLinks` option controls footer link behavior:

When `InlineTableLinks` is `true`, links remain inline within table cells instead of being extracted to the footer ([ansi/table\_links.go113-115](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L113-L115)
).

**Sources**: [ansi/table\_links.go112-120](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/table_links.go#L112-L120)

URL Resolution
--------------

Both `LinkElement` and `ImageElement` support relative URL resolution through the `resolveRelativeURL()` function:

This allows markdown files to reference resources with relative paths like `./images/logo.png`, which are resolved against the document's base URL.

**Sources**: [ansi/link.go67](https://github.com/charmbracelet/glamour/blob/69661fd5/ansi/link.go#L67-L67)
