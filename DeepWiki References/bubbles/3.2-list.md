List

The `list` package provides a feature-rich Bubble Tea component for browsing a general-purpose set of items. It includes built-in support for fuzzy filtering, pagination, an activity spinner, status messages, and an integrated help system.

Sources: [list/list.go1-4](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L1-L4)

Component Overview
------------------

The List component manages a collection of items and coordinates several sub-components:

*   **`paginator.Model`**: Handles the logic for slicing items into pages.
*   **`textinput.Model`**: Used for the interactive filter prompt.
*   **`spinner.Model`**: Indicates background activity.
*   **`help.Model`**: Displays contextual keybindings.

### Model Structure and Layout

The `Model` struct tracks the UI state and configuration. Layout calculations are primarily handled in `updatePagination`, which determines how many items fit on a page based on the available height and the delegate's height/spacing.

Sources: [list/list.go147-204](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L147-L204)
 [list/list.go813-845](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L813-L845)

Item and Delegate Interfaces
----------------------------

The list's flexibility stems from its interface-driven design, separating data from presentation.

### Item Interface

To appear in a list, a type must implement `Item`. The `FilterValue` is used by the filtering engine to match against user input.

Sources: [list/list.go34-38](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L34-L38)

### ItemDelegate Interface

The `ItemDelegate` handles the rendering and update logic for items. This allows developers to change how items look or behave (e.g., responding to custom keys) without modifying the item data itself.

Sources: [list/list.go46-61](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L46-L61)

### Default Implementation

The package provides `DefaultItem` and `DefaultDelegate` for common "Title + Description" lists.

*   **`DefaultItem`**: Adds `Title()` and `Description()` methods to the `Item` interface. [list/defaultitem.go67-71](https://github.com/charmbracelet/bubbles/blob/42130e89/list/defaultitem.go#L67-L71)
    
*   **`DefaultDelegate`**: Implements the rendering logic, including truncation and filter-match highlighting using `lipgloss.StyleRunes`. [list/defaultitem.go86-217](https://github.com/charmbracelet/bubbles/blob/42130e89/list/defaultitem.go#L86-L217)
    

Event Handling: Filtering vs. Browsing
--------------------------------------

The `Update` function in `list.go` splits logic based on the `FilterState`.

| State | Constant | Description |
| --- | --- | --- |
| **Unfiltered** | `Unfiltered` | Normal browsing mode. |
| **Filtering** | `Filtering` | User is typing in the filter input. |
| **Filter Applied** | `FilterApplied` | List is restricted to matches; user is browsing results. |

### Event Split Logic

When `filterState` is `Filtering`, most key presses are routed to the `FilterInput` (textinput). When in `Unfiltered` or `FilterApplied`, keys are routed to `handleBrowsing`.

Sources: [list/list.go131-135](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L131-L135)
 [list/list.go858-914](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L858-L914)

Filtering System
----------------

The list uses a `FilterFunc` to rank items. By default, it uses `sahilm/fuzzy`.

### Implementation Details

1.  **`FilterFunc`**: A function type that takes a term and a list of strings, returning a slice of `Rank`. [list/list.go86-94](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L86-L94)
    
2.  **`filteredItem`**: An internal struct that maps a matched item back to its original index in the master `items` slice and stores rune indices for highlighting. [list/list.go63-67](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L63-L67)
    
3.  **`FilterMatchesMsg`**: Sent when a background filter operation completes, containing the new set of matches. [list/list.go81](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L81-L81)
    

Sources: [list/list.go98-123](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L98-L123)
 [list/list.go916-970](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L916-L970)

Pagination and Layout
---------------------

The list relies on `paginator.Model` to manage pages. The number of items per page is calculated dynamically in `updatePagination`.

### Height Calculation

The available height for items is calculated by subtracting the heights of enabled UI elements from the total `Model.height`:

*   Title Bar (if `showTitle`)
*   Filter Input (if `showFilter`)
*   Status Bar (if `showStatusBar`)
*   Pagination (if `showPagination`)
*   Help (if `showHelp`)

The number of items per page is then: `AvailableHeight / (Delegate.Height + Delegate.Spacing)`.

Sources: [list/list.go813-845](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L813-L845)

Key Map and Help System
-----------------------

The list integrates with the `help` component. It defines a `KeyMap` struct containing `key.Binding` fields for navigation (Up, Down, PageUp, etc.) and filtering actions.

### Help Integration

The `Model` implements `help.KeyMap`. If the `ItemDelegate` also implements `help.KeyMap`, the list automatically merges the delegate's keys into the help view. This is seen in `DefaultDelegate` which can provide `ShortHelp` and `FullHelp`.

Sources: [list/list.go736-811](https://github.com/charmbracelet/bubbles/blob/42130e89/list/list.go#L736-L811)
 [list/defaultitem.go220-233](https://github.com/charmbracelet/bubbles/blob/42130e89/list/defaultitem.go#L220-L233)

Styling
-------

Styles are managed via the `Styles` struct and `DefaultStyles` function. This includes:

*   **`Title`** and **`TitleBar`**: Styling for the top header.
*   **`StatusBar`**: Styling for the item count and filter status.
*   **`Filter`**: Inherits from `textinput.Styles`.
*   **`ActivePaginationDot`**: Styling for the paginator indicators.

Sources: [list/style.go15-102](https://github.com/charmbracelet/bubbles/blob/42130e89/list/style.go#L15-L102)
