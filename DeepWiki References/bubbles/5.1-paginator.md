Paginator
=========

Relevant source files

*   [paginator/paginator.go](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go)
    
*   [paginator/paginator\_test.go](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator_test.go)
    

Overview
--------

The `paginator` package provides a Bubble Tea component for calculating pagination state and rendering pagination status indicators. It is important to note that this package does not render the actual content pages; it is purely for handling keystrokes related to pagination and rendering the pagination status (e.g., "1/5" or "••○").

Sources: [paginator/paginator.go1-4](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L1-L4)

Architecture
------------

The Paginator follows the standard Bubble Tea Model-View-Update (MVU) pattern. It maintains the current state of navigation (current page, items per page) and provides helpers to map that state to data slices.

### The Model Struct

The `Model` struct contains the configuration and state for the pagination logic.

Sources: [paginator/paginator.go15-27](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L15-L27)
 [paginator/paginator.go39-57](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L39-L57)
 [paginator/paginator.go128-145](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L128-L145)

Core Concepts
-------------

### Display Types

The `Type` field in the `Model` determines how the `View()` method renders the pagination status.

*   **`Arabic`**: Renders using a numeric format (e.g., "1/10"). The format can be customized via the `ArabicFormat` field (default: `"%d/%d"`).
*   **`Dots`**: Renders using characters to represent pages (e.g., "•○"). The characters are controlled by `ActiveDot` (default: `"•"`) and `InactiveDot` (default: `"○"`).

Sources: [paginator/paginator.go15-21](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L15-L21)

### Data Flow and Slicing

The Paginator bridges the gap between the UI state and the data array using `GetSliceBounds`.

Key Functions
-------------

### Initialization

The `New` function initializes a `Model` with sensible defaults: `Arabic` type, 1 item per page, and default key bindings.

| Option | Description |
| --- | --- |
| `WithTotalPages(int)` | Sets the initial `TotalPages` count. |
| `WithPerPage(int)` | Sets the number of items per page. |

Sources: [paginator/paginator.go128-159](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L128-L159)

### Pagination Helpers

*   **`SetTotalPages(items int) int`**: Calculates the number of pages needed for a given number of items based on `PerPage` and updates the model.
*   **`GetSliceBounds(length int) (int, int)`**: Returns the `start` and `end` indices for slicing a data array of the given `length`.
*   **`ItemsOnPage(totalItems int) int`**: Calculates how many items should appear on the current page.
*   **`PrevPage()` / `NextPage()`**: Programmatically change the current page while respecting bounds (0 to `TotalPages-1`).

Sources: [paginator/paginator.go63-73](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L63-L73)

Key Bindings
------------

The `KeyMap` struct defines the keys used to trigger page changes during `Update`.

| Action | Default Keys | Code Identifier |
| --- | --- | --- |
| Previous Page | `pgup`, `left`, `h` | `KeyMap.PrevPage` |
| Next Page | `pgdown`, `right`, `l` | `KeyMap.NextPage` |

Sources: [paginator/paginator.go24-36](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L24-L36)

Implementation Details
----------------------

### Rendering Pipeline

The `View()` method switches between rendering strategies based on `Model.Type`.

1.  **`arabicView()`**: Uses `fmt.Sprintf` with `ArabicFormat`, `Page+1` (for 1-based display), and `TotalPages`.
2.  **`dotsView()`**: Iterates from `0` to `TotalPages-1`, appending `ActiveDot` if the index matches `Page`, otherwise appending `InactiveDot`.

Sources: [paginator/paginator.go177-200](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L177-L200)

### Internal Usage in Lists

The `list` component (documented in section 3.2) uses the `paginator` package internally to handle its pagination logic. When the list's width or height changes, it calls `SetTotalPages` and `GetSliceBounds` to determine which items from the list's data source should be rendered in the viewport.

 [paginator/paginator.go92-96](https://github.com/charmbracelet/bubbles/blob/42130e89/paginator/paginator.go#L92-L96)
