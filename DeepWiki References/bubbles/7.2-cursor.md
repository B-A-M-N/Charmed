Cursor
======

Relevant source files

*   [cursor/cursor.go](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go)
    
*   [cursor/cursor\_test.go](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor_test.go)
    

The `cursor` package provides a virtual cursor primitive designed to support text-based input components like `textinput` and `textarea`. It manages the visual representation, blinking logic, and focus state of a cursor independently of the terminal's native hardware cursor.

[cursor/cursor.go1-3](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L1-L3)

Model and State
---------------

The `Model` struct encapsulates the state of a virtual cursor. It tracks its own visual styles, blinking interval, and focus status.

| Field | Type | Description |
| --- | --- | --- |
| `Style` | `lipgloss.Style` | The style applied to the cursor block itself. |
| `TextStyle` | `lipgloss.Style` | The style used for the character under the cursor when the cursor is "blinked" (hidden). |
| `BlinkSpeed` | `time.Duration` | The interval between blink cycles (default: 530ms). |
| `IsBlinked` | `bool` | Current visibility state; `true` means the cursor block is hidden. |
| `mode` | `Mode` | Determines if the cursor blinks, stays static, or is hidden. |
| `focus` | `bool` | Whether the parent component is focused. |

### Focus and Blur

The cursor's behavior is tied to its focus state. When focused via `Focus()`, the cursor becomes visible and initiates its blinking cycle if the mode is set to `CursorBlink` [cursor/cursor.go214-222](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L214-L222)
 When `Blur()` is called, the cursor is hidden (`IsBlinked = true`) and blinking stops [cursor/cursor.go225-228](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L225-L228)

### Cursor Modes

The `Mode` type defines how the cursor behaves visually:

*   **`CursorBlink`**: The cursor toggles visibility based on `BlinkSpeed` [cursor/cursor.go48](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L48-L48)
    
*   **`CursorStatic`**: The cursor remains visible without blinking [cursor/cursor.go49](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L49-L49)
    
*   **`CursorHide`**: The cursor is never rendered [cursor/cursor.go50](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L50-L50)

Animation and Lifecycle
-----------------------

The cursor uses a specialized message-passing loop to handle blinking. This lifecycle is managed using `context` to allow for cancellation of pending blinks, preventing "ghost" blink messages from updating the model after the mode or focus has changed.

### Multi-Instance Safety

To support multiple cursors in a single application (e.g., multiple text inputs), the package uses an `id` and `tag` pattern:

1.  **`id`**: Assigned at creation via `nextID()` to uniquely identify the `Model` instance [cursor/cursor.go20-22](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L20-L22)
     [cursor/cursor.go102](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L102-L102)
    
2.  **`tag`**: Incremented every time a new blink cycle starts [cursor/cursor.go195](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L195-L195)
    
3.  **Validation**: When `Update` receives a `BlinkMsg`, it verifies that both the `id` and `tag` match the current state. If they do not, the message is ignored, preventing stale animations from affecting the UI [cursor/cursor.go143-146](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L143-L146)
    

Rendering
---------

The `View()` method implements a "virtual cursor." Unlike `tea.Cursor` which moves the terminal's hardware cursor, this package renders the cursor by manipulating the styling of a single character.

*   **Blinked State**: Renders the character using `TextStyle` [cursor/cursor.go237-239](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L237-L239)
    
*   **Visible State**: Renders the character using `Style` with `Reverse(true)` applied, creating the "block" effect [cursor/cursor.go240-241](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L240-L241)

### Context Cancellation

The `blinkCtx` struct holds a `context.CancelFunc` [cursor/cursor.go38-41](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L38-L41)
 Whenever a new blink is requested (via `Blink()`), the previous context is explicitly cancelled [cursor/cursor.go188-190](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L188-L190)
 This ensures that if the user types rapidly (restarting the blink timer frequently), only the most recent timer actually resolves to a `BlinkMsg`. If a timer is cancelled, it returns a `blinkCanceled` no-op message instead [cursor/cursor.go204](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L204-L204)
