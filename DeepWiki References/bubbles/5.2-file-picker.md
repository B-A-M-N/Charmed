File Picker
===========

Relevant source files

*   [filepicker/filepicker.go](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go)

The File Picker component provides a terminal-based file system navigation and selection interface for Bubble Tea applications. It allows users to browse directories, view file metadata, and select files or directories based on configurable criteria.

Sources: [filepicker/filepicker.go1-3](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L1-L3)

Overview
--------

The File Picker presents a navigable list of files and directories with capabilities to:

*   Navigate through the file system hierarchy using a stack-based history.
*   Show file metadata including Unix permissions and human-readable sizes.
*   Filter displayed files by extension via `AllowedTypes`.
*   Support selection of files, directories, or both.
*   Customize appearance through a comprehensive `Styles` system.
*   Handle asynchronous directory reading to maintain UI responsiveness.

Sources: [filepicker/filepicker.go5-18](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L5-L18)
 [filepicker/filepicker.go127-162](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L127-L162)

Data Flow and Selection Lifecycle
---------------------------------

The File Picker uses a specific message-passing flow for directory reads and selection events.

### Asynchronous Directory Reads

Directory contents are fetched via the `readDir` command [filepicker/filepicker.go197-225](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L197-L225)
 To prevent race conditions or stale data from overlapping reads, each `Model` has a unique `id` [filepicker/filepicker.go22-24](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L22-L24)
 The `readDirMsg` includes this ID, and the `Update` function discards any messages that do not match the current model's ID [filepicker/filepicker.go248-251](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L248-L251)

### Selection Detection

The parent application identifies selections by calling helper methods on the model after the `Update` loop:

*   `DidSelectFile(msg)`: Returns `true` and the file path if a valid file/directory was selected [filepicker/filepicker.go447-465](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L447-L465)
    
*   `DidSelectDisabledFile(msg)`: Returns `true` if the user tried to select a file not matching `AllowedTypes` [filepicker/filepicker.go468-479](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L468-L479)

Navigation and Stack Management
-------------------------------

The File Picker maintains three internal stacks to restore the user's cursor position and scroll state when navigating back up the directory tree:

1.  `selectedStack`: Stores the index of the selected item [filepicker/filepicker.go150](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L150-L150)
    
2.  `minStack`: Stores the top-most visible index (scroll offset) [filepicker/filepicker.go155](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L155-L155)
    
3.  `maxStack`: Stores the bottom-most visible index [filepicker/filepicker.go154](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L154-L154)
    

When a directory is opened, the current state is pushed via `pushView` [filepicker/filepicker.go187-191](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L187-L191)
 When the user navigates back (e.g., `h` or `Backspace`), the state is restored via `popView` [filepicker/filepicker.go193-195](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L193-L195)

Configuration and Styling
-------------------------

### Initialization

`New()` initializes the model with default styles, keybindings, and sets `AutoHeight` to `true` [filepicker/filepicker.go27-49](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L27-L49)

### Filtering and Permissions

*   `AllowedTypes`: A slice of strings (e.g., `[]string{".png", ".jpg"}`). If populated, files not matching these extensions are styled with `Styles.DisabledFile` and cannot be selected [filepicker/filepicker.go138](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L138-L138)
    
*   `IsHidden`: A platform-specific utility determines if a file is hidden. On Unix, it checks for a leading dot [filepicker/hidden\_unix.go9-11](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/hidden_unix.go#L9-L11)
    ; on Windows, it uses `syscall.GetFileAttributes` [filepicker/hidden\_windows.go11-21](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/hidden_windows.go#L11-L21)

View Rendering Pipeline
-----------------------

The `View()` method constructs the file list based on the current scroll window (`minIdx` to `maxIdx`).

1.  **Height Calculation**: If `AutoHeight` is true, the height is adjusted based on `tea.WindowSizeMsg` minus a `marginBottom` of 5 [filepicker/filepicker.go60-61](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L60-L61)
     [filepicker/filepicker.go255-257](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L255-L257)
    
2.  **Entry Formatting**: Each entry is formatted using `humanize.Bytes` for size and `os.FileMode.String()` for permissions [filepicker/filepicker.go17](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L17-L17)
    
3.  **Symlink Resolution**: If an entry is a symlink, the picker attempts to read the link target using `os.Readlink` and displays it alongside the name [filepicker/filepicker.go8](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L8-L8)
    
4.  **Empty State**: If no files match the current filters or the directory is empty, `Styles.EmptyDirectory` is rendered [filepicker/filepicker.go106](https://github.com/charmbracelet/bubbles/blob/42130e89/filepicker/filepicker.go#L106-L106)
