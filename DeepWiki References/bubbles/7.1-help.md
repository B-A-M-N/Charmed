The `help` package provides a standardized component for displaying key bindings in Bubble Tea applications. It supports two distinct display modes: a compact, single-line "short" help and a comprehensive, multi-column "full" help. The component is width-aware, automatically truncating content with an ellipsis when horizontal space is constrained.

----------------------

The help system relies on a decoupled architecture where the `Model` handles rendering logic and layout, while the data is provided via the `KeyMap` interface.

Implementation Details
----------------------

### Width-Aware Rendering Pipeline

The `help` component is designed to be responsive. When `SetWidth(int)` is called, the rendering methods use `shouldAddItem` to determine if the next key binding fits within the allocated horizontal space.

### The KeyMap Interface

To display help, an application or component must implement `KeyMap`. If a `key.Binding` is disabled via `SetEnabled(false)`, it is automatically omitted from the rendered view.

*   `ShortHelp()`: Returns `[]key.Binding`. These are rendered horizontally separated by `ShortSeparator`.
*   `FullHelp()`: Returns `[][]key.Binding`. Each inner slice represents a column in the full help view.

Styling and Appearance
----------------------

The component provides a `Styles` struct to customize every visual aspect.

### Default Styles

The package provides `DefaultStyles(isDark bool)`, along with helpers `DefaultDarkStyles()` and `DefaultLightStyles()`. These styles use `lipgloss.LightDark` to provide accessible contrast for both background types.

| Style Field | Default Dark Color | Default Light Color |
| --- | --- | --- |
| `ShortKey` / `FullKey` | `#626262` | `#909090` |
| `ShortDesc` / `FullDesc` | `#4A4A4A` | `#B2B2B2` |
| `ShortSeparator` / `Ellipsis` | `#3C3C3C` | `#DADADA` |

Sources: [help/help.go31-43](https://github.com/charmbracelet/bubbles/blob/42130e89/help/help.go#L31-L43)
 [help/help.go48-64](https://github.com/charmbracelet/bubbles/blob/42130e89/help/help.go#L48-L64)

### Configuration Defaults

When initialized via `New()`, the model uses the following defaults:

*   `ShortSeparator`: `" • "`
*   `FullSeparator`: `" "` (4 spaces)
*   `Ellipsis`: `"…"`
*   `Styles`: `DefaultDarkStyles()`

Sources: [help/help.go93-100](https://github.com/charmbracelet/bubbles/blob/42130e89/help/help.go#L93-L100)

Integration in Other Components
-------------------------------

The `help` component is a core utility used by complex Bubbles components to provide contextual instructions.

### List Component Integration

The `list` component defines its own `KeyMap` struct that satisfies `help.KeyMap`. It categorizes bindings into browsing, filtering, and help-toggle groups.

Sources: [list/keys.go7-31](https://github.com/charmbracelet/bubbles/blob/42130e89/list/keys.go#L7-L31)

### Table Component Integration

Similar to the list, the `table` component utilizes the help system to display navigation keys (Up/Down) and exit commands.

Testing and Golden Files
------------------------

The `help` package uses golden file testing to ensure rendering consistency across different widths. Tests in `help_test.go` validate that `FullHelpView` correctly wraps or truncates columns when the width is set to 20, 30, or 40 characters.

**Example Golden Output (Width 30):**

    enter continue | esc back …
                     ?   help
