Progress Bar
============

The Progress Bar component provides a customizable, animated progress indicator for terminal applications built with the Bubble Tea framework. It features smooth, spring-based animations and high-resolution color blending.

Overview
--------

The Progress Bar component in the `progress` package displays progress with a horizontal bar that fills from left to right. It is built on the Bubble Tea Model-View-Update (MVU) pattern and utilizes the `harmonica` library for physics-based animations.

Key features include:

*   **High-Resolution Rendering**: Uses half-block characters to double the vertical color resolution. [progress/progress.go32-37](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L32-L37)
    
*   **Spring Physics**: Smooth transitions using frequency and damping controls. [progress/progress.go157-167](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L157-L167)
    
*   **Flexible Coloring**: Supports solid colors, multi-color gradients, and dynamic color functions. [progress/progress.go81-130](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L81-L130)
    
*   **Message Routing**: Internal ID and tag management to prevent message leakage between multiple progress bars. [progress/progress.go24-30](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L24-L30)

Visual Pipeline
---------------

### Half-Block Rendering

By default, the component uses the `DefaultFullCharHalfBlock` ('▌'). [progress/progress.go37](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L37-L37)
 This allows the renderer to set different colors for the foreground and background of a single cell, effectively doubling the horizontal resolution of color transitions. [progress/progress.go33-36](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L33-L36)

### Color Blending and Gradients

The progress bar supports three primary coloring modes:

1.  **Solid**: A single `FullColor` used for the entire filled portion. [progress/progress.go96-102](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L96-L102)
    
2.  **Gradient (Blend)**: A slice of `color.Color` values. If `WithScaled(true)` is used, the gradient fits the filled portion; otherwise, it spans the entire width of the bar. [progress/progress.go103-106](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L103-L106)
     [progress/progress.go169-178](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L169-L178)
    
3.  **ColorFunc**: A dynamic callback `func(total, current float64) color.Color` that determines color based on the fill state. [progress/progress.go18-22](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L18-L22)
     [progress/progress.go108-130](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L108-L130)

Limitations and Considerations
------------------------------

*   **FPS**: The animation loop is hardcoded to 60 FPS. [progress/progress.go48](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L48-L48)
    
*   **Message IDs**: Each model generated by `New()` gets a unique ID to ensure `FrameMsg` is routed correctly. Cloning a model via struct literal instead of `New()` or `SetPercent` may cause animation routing issues. [progress/progress.go28-30](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L28-L30)
     [progress/progress.go234-235](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L234-L235)
    
*   **Unicode**: The default half-block character requires a terminal with proper Unicode support for the best visual fidelity. [progress/progress.go32-37](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L32-L37)
