Viewport
========

Relevant source files

*   [textarea/textarea\_test.go](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea_test.go)

The Viewport component is a scrollable container for displaying content that exceeds the available screen space. It supports both vertical and horizontal scrolling, mouse wheel interaction, and advanced features like soft-wrapping, line-number gutters, and text highlighting.

Sources: [viewport/viewport.go51-116](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L51-L116)
 [viewport/viewport.go1-13](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L1-L13)

Model and Initialization
------------------------

The `Model` struct is the core of the viewport, managing state for dimensions, offsets, and content rendering.

Sources: [viewport/viewport.go52-116](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L52-L116)

Viewports are typically initialized using the `New` function with functional options:

*   `WithWidth(int)`: Sets the viewport width [viewport/viewport.go27-31](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L27-L31)
    
*   `WithHeight(int)`: Sets the viewport height [viewport/viewport.go33-39](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L33-L39)
    

The `setInitialValues` method provides defaults: `MouseWheelEnabled` is `true`, `MouseWheelDelta` is `3`, and the `KeyMap` is populated with pager-like bindings [viewport/viewport.go148-155](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L148-L155)

Coordinate Management
---------------------

The viewport tracks two primary offsets to manage the "window" into the content:

*   `yOffset`: The vertical scroll position (top-most visible line) [viewport/viewport.go72](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L72-L72)
    
*   `xOffset`: The horizontal scroll position (left-most visible column) [viewport/viewport.go75](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L75-L75)
    

### Navigation Logic

The component provides methods to manipulate these offsets while respecting content boundaries:

*   `ScrollDown(n int)` / `ScrollUp(n int)`: Adjusts `yOffset` [viewport/viewport.go196-245](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L196-L245)
    
*   `ScrollLeft(n int)` / `ScrollRight(n int)`: Adjusts `xOffset` [viewport/viewport.go299-349](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L299-L349)
    
*   `EnsureVisible(line, column int)`: Automatically adjusts offsets to make a specific coordinate visible [viewport/viewport.go375-399](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L375-L399)
    

Sources: [viewport/viewport.go71-80](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L71-L80)
 [viewport/viewport.go196-399](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L196-L399)

Rendering Pipeline
------------------

The rendering process transforms raw content lines into a styled, scrollable view.

Sources: [viewport/viewport.go500-527](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L500-L527)
 [viewport/viewport.go529-623](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L529-L623)

### SoftWrap vs. Horizontal Scrolling

*   **SoftWrap (`true`)**: Lines are wrapped to fit the viewport width. Horizontal scrolling is disabled (`xOffset` ignored) [viewport/viewport.go57-60](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L57-L60)
    
*   **Horizontal Scrolling (`SoftWrap: false`)**: Lines maintain their original length. Users scroll horizontally using `xOffset` [viewport/viewport.go74-75](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L74-L75)
    

### Gutter System

The `LeftGutterFunc` allows prepending a static column (e.g., line numbers) that remains visible during horizontal scrolling.

The `GutterContext` provides the line `Index`, `TotalLines`, and whether the line is a `Soft` wrapped continuation [viewport/viewport.go131-146](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L131-L146)

Sources: [viewport/viewport.go57-60](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L57-L60)
 [viewport/viewport.go89-95](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L89-L95)
 [viewport/viewport.go131-146](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L131-L146)

Highlighting Engine
-------------------

The viewport supports a sophisticated highlighting system for search results or syntax.

1.  **SetHighlights**: Accepts a string and byte-offset matches (e.g., from `regexp.FindAllStringIndex`) and converts them into `highlightInfo` using `parseMatches` [viewport/viewport.go417-425](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L417-L425)
     [viewport/highlight.go20-102](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/highlight.go#L20-L102)
    
2.  **Navigation**: `HighlightNext()` and `HighlightPrevious()` move the selection index and automatically call `EnsureVisible` to bring the match into view [viewport/viewport.go434-471](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L434-L471)
    
3.  **Styles**: `HighlightStyle` is applied to all matches, while `SelectedHighlightStyle` is applied to the currently focused match [viewport/viewport.go101-108](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L101-L108)
    

Sources: [viewport/viewport.go417-471](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L417-L471)
 [viewport/highlight.go9-102](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/highlight.go#L9-L102)

KeyMap and Mouse Handling
-------------------------

The viewport handles `tea.KeyPressMsg` and `tea.MouseMsg` automatically in its `Update` method.

### KeyBindings

The `DefaultKeyMap` includes standard pager keys:

*   **Vertical**: `j`/`k`, `pgup`/`pgdown`, `u`/`d` (half-page) [viewport/keymap.go25-48](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/keymap.go#L25-L48)
    
*   **Horizontal**: `h`/`l`, `left`/`right` [viewport/keymap.go49-56](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/keymap.go#L49-L56)
    

### Mouse Interaction

If `MouseWheelEnabled` is true, the viewport responds to `tea.MouseWheelUp` and `tea.MouseWheelDown`.

*   Standard scroll moves the `yOffset` by `MouseWheelDelta` [viewport/viewport.go150-151](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L150-L151)
    
*   Shift+Scroll or horizontal wheel events (on supported terminals) move the `xOffset` [viewport/viewport.go64-70](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L64-L70)
    

Sources: [viewport/keymap.go23-58](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/keymap.go#L23-L58)
 [viewport/viewport.go64-70](https://github.com/charmbracelet/bubbles/blob/42130e89/viewport/viewport.go#L64-L70)
