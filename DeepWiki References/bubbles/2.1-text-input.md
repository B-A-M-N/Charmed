Text Input
==========

The Text Input component (`textinput` package) provides a flexible, feature-rich single-line text input field for terminal-based applications built with the Bubble Tea framework. It handles user keyboard input, cursor management, text editing operations, and offers various customization options to create both simple and complex text input experiences.

For multi-line text input, see the [Text Area](https://deepwiki.com/charmbracelet/bubbles/2.2-text-area)
 component.

Core Features
-------------

The Text Input component supports:

1.  **Text Entry and Editing**: Basic text entry with various editing operations using `[]rune` for Unicode safety [textinput/textinput.go123](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L123-L123)
    
2.  **Cursor Navigation**: Move the cursor by character, word, or to the start/end of input [textinput/textinput.go47-64](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L47-L64)
    
3.  **Echo Modes**: Normal text display, password masking, or completely hidden input [textinput/textinput.go27-41](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L27-L41)
    
4.  **Validation**: Custom `ValidateFunc` to enforce input constraints [textinput/textinput.go43-44](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L43-L44)
    
5.  **Suggestions**: Prefix-based auto-completion engine [textinput/textinput.go147-154](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L147-L154)
    
6.  **Styling**: Distinct styles for focused and blurred states via `Styles` and `StyleState` [textinput/styles.go50-68](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/styles.go#L50-L68)
    
7.  **Viewport Logic**: Horizontally scrolling viewport for long inputs constrained by `Width` [textinput/textinput.go114-117](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L114-L117)

Echo Modes
----------

The `EchoMode` determines how characters are rendered in the `View()`.

| Constant | Value | Description |
| --- | --- | --- |
| `EchoNormal` | `0` | Displays text as is. Default behavior. |
| `EchoPassword` | `1` | Displays `EchoCharacter` (default `*`) instead of actual characters. |
| `EchoNone` | `2` | Displays nothing as characters are entered. |

Sources: [textinput/textinput.go27-41](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L27-L41)
 [textinput/textinput.go159-160](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L159-L160)

Cursor Management
-----------------

The component supports two types of cursors:

1.  **Virtual Cursor**: A `cursor.Model` managed internally that renders a styled block or character within the terminal string [textinput/textinput.go103-104](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L103-L104)
    
2.  **Real Cursor**: When `useVirtualCursor` is false, the component relies on the terminal's native cursor via `tea.Cursor` [textinput/textinput.go99-101](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L99-L101)
    

The virtual cursor handles its own blinking lifecycle via `BlinkMsg` and `blinkCtx` [cursor/cursor.go27-41](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L27-L41)

Sources: [textinput/textinput.go176-187](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L176-L187)
 [cursor/cursor.go63-97](https://github.com/charmbracelet/bubbles/blob/42130e89/cursor/cursor.go#L63-L97)

Viewport and Scrolling
----------------------

When a `Width` is set [textinput/textinput.go114-117](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L114-L117)
 the input acts as a horizontally scrolling window.

*   `offset`: The index of the first visible rune in `value` [textinput/textinput.go134](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L134-L134)
    
*   `offsetRight`: The index of the last visible rune [textinput/textinput.go135](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L135-L135)
    

As the cursor (`pos`) moves beyond the visible bounds, the `Update` logic adjusts `offset` to ensure the cursor remains visible, effectively "scrolling" the text.

Suggestions and Autocompletion
------------------------------

The suggestion engine allows for IDE-like tab completion:

*   **Storage**: `suggestions` holds the master list; `matchedSuggestions` holds those matching the current input [textinput/textinput.go149-152](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L149-L152)
    
*   **Trigger**: Suggestions are updated as the user types if `ShowSuggestions` is true [textinput/textinput.go147](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L147-L147)
    
*   **Navigation**: `NextSuggestion` and `PrevSuggestion` cycle through the matches [textinput/textinput.go62-63](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L62-L63)
    
*   **Acceptance**: `AcceptSuggestion` (default `Tab`) completes the input with the current selection [textinput/textinput.go61](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L61-L61)

Key Functions
-------------

### SetValue and Validation

`SetValue(string)` sanitizes input using `runeutil` to remove control characters and then runs the `Validate` callback [textinput/textinput.go209-216](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L209-L216)
 If validation fails, the `Err` field is populated [textinput/textinput.go218-219](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L218-L219)

### Paste

The `Paste()` command uses `github.com/atotto/clipboard` to fetch system clipboard content and returns a `tea.Cmd` that produces a `pasteMsg` [textinput/textinput.go16](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L16-L16)
 [textinput/textinput.go21-25](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L21-L25)

### Key Bindings

The `KeyMap` defines how keyboard events map to internal actions like `WordForward` or `DeleteBeforeCursor` [textinput/textinput.go47-64](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L47-L64)
 Default bindings include standard Emacs-style shortcuts (e.g., `Ctrl+A` for line start, `Ctrl+K` for delete after cursor) [textinput/textinput.go68-87](https://github.com/charmbracelet/bubbles/blob/42130e89/textinput/textinput.go#L68-L87)
