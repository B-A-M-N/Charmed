Stopwatch and Timer
===================

Relevant source files

*   [stopwatch/stopwatch.go](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go)
    
*   [timer/timer.go](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go)
    

This page documents the Stopwatch and Timer components in the Bubbles UI library. These components provide time tracking functionality for terminal-based applications, with the stopwatch measuring elapsed time and the timer counting down to zero.

Overview
--------

The Bubbles library provides two distinct time tracking components:

1.  **Stopwatch**: Counts up from zero, tracking elapsed time [stopwatch/stopwatch.go1-2](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L1-L2)
    
2.  **Timer**: Counts down from a specified duration, signaling when time is up [timer/timer.go1-2](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L1-L2)
    

Both components follow the Bubble Tea Model-View-Update pattern and utilize internal ID and tag management to ensure multi-instance safety [stopwatch/stopwatch.go11-15](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L11-L15)
 [timer/timer.go11-15](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L11-L15)

Component Structure
-------------------

Both components share a similar internal structure, maintaining a running state, a unique ID, and a tag for message synchronization.

Message Types
-------------

Both components define specific message types to communicate state changes. While they share names like `TickMsg`, they are defined within their respective packages and carry unique IDs to prevent crosstalk.

| Component | Message Type | Purpose |
| --- | --- | --- |
| **Stopwatch** | `TickMsg` | Sent on every interval tick [stopwatch/stopwatch.go31-41](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L31-L41) |
| **Stopwatch** | `StartStopMsg` | Controls the running state [stopwatch/stopwatch.go44-47](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L44-L47) |
| **Stopwatch** | `ResetMsg` | Resets the elapsed duration to zero [stopwatch/stopwatch.go50-52](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L50-L52) |
| **Timer** | `TickMsg` | Sent on every interval tick; includes `Timeout` bool [timer/timer.go64-79](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L64-L79) |
| **Timer** | `TimeoutMsg` | Convenience message sent exactly once on expiry [timer/timer.go85-87](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L85-L87) |
| **Timer** | `StartStopMsg` | Controls the running state [timer/timer.go58-61](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L58-L61) |

Sources: [stopwatch/stopwatch.go31-52](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L31-L52)
 [timer/timer.go58-87](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L58-L87)

Stopwatch Component
-------------------

The stopwatch tracks elapsed time by incrementing its internal duration `d` [stopwatch/stopwatch.go56](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L56-L56)
 on every `TickMsg`.

### Initialization and Options

A stopwatch is created using `New()`. It defaults to a 1-second tick interval [stopwatch/stopwatch.go66-75](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L66-L75)

*   **WithInterval**: Sets the frequency of `TickMsg` generation [stopwatch/stopwatch.go24-28](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L24-L28)
    

### Control Methods

*   `Start()`: Sends a `StartStopMsg` with `running: true` and triggers the first tick [stopwatch/stopwatch.go88-92](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L88-L92)
    
*   `Stop()`: Sends a `StartStopMsg` with `running: false` [stopwatch/stopwatch.go95-99](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L95-L99)
    
*   `Reset()`: Sends a `ResetMsg` to clear the duration [stopwatch/stopwatch.go110-114](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L110-L114)
    
*   `Elapsed()`: Returns the current `time.Duration` [stopwatch/stopwatch.go155-157](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L155-L157)
    

### Update Logic

The `Update` function handles state transitions based on the received ID. It ignores messages intended for other stopwatch instances [stopwatch/stopwatch.go125-127](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L125-L127)
 When a `TickMsg` is accepted, it increments the duration by the configured `Interval` [stopwatch/stopwatch.go146](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L146-L146)

Sources: [stopwatch/stopwatch.go66-152](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L66-L152)

Timer Component
---------------

The timer counts down from a specified `Timeout` duration [timer/timer.go92](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L92-L92)

### Initialization and Options

A timer is created using `New(timeout time.Duration, opts ...Option)` [timer/timer.go103](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L103-L103)

*   **WithInterval**: Configures how often the timer decrements its internal value (default 1s) [timer/timer.go24-28](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L24-L28)
    

### Control and State

*   `Timedout()`: Returns `true` if `Timeout <= 0` [timer/timer.go132-134](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L132-L134)
    
*   `Running()`: Returns `false` if the timer is paused or has timed out [timer/timer.go124-129](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L124-L129)
    
*   `Start()` / `Stop()`: Controls the `running` field via `StartStopMsg` [timer/timer.go175-182](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L175-L182)

Shared Patterns
---------------

### ID and Multi-Instance Safety

Both components use a global atomic counter to assign a unique `id` to every new instance [stopwatch/stopwatch.go11-15](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L11-L15)
 [timer/timer.go11-15](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L11-L15)

*   The `Update` methods check `msg.ID == m.id` before processing [stopwatch/stopwatch.go125](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L125-L125)
     [timer/timer.go145](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L145-L145)
    
*   This allows developers to run multiple timers/stopwatches in a single application without message collisions.

### The Tag System

To prevent "tick drift" or excessive updates if the system is under load, both components use a `tag` system [stopwatch/stopwatch.go58](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L58-L58)
 [timer/timer.go98](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L98-L98)

1.  Each `TickMsg` carries a `tag`.
2.  When `Update` processes a `TickMsg`, it increments the model's internal `tag` [stopwatch/stopwatch.go147](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L147-L147)
    
3.  If a `TickMsg` arrives with an old `tag`, it is rejected [stopwatch/stopwatch.go142-144](https://github.com/charmbracelet/bubbles/blob/42130e89/stopwatch/stopwatch.go#L142-L144)
     [timer/timer.go158-160](https://github.com/charmbracelet/bubbles/blob/42130e89/timer/timer.go#L158-L160)
