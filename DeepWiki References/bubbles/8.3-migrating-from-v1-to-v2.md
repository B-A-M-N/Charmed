Migrating from v1 to v2
=======================

Relevant source files

*   [UPGRADE\_GUIDE\_V2.md](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1)
    
*   [go.mod](https://github.com/charmbracelet/bubbles/blob/42130e89/go.mod)
    
*   [go.sum](https://github.com/charmbracelet/bubbles/blob/42130e89/go.sum)
    

This page provides a comprehensive technical guide for developers migrating applications from Bubbles v1 (`github.com/charmbracelet/bubbles`) to Bubbles v2 (`charm.land/bubbles/v2`). Bubbles v2 is built upon Bubble Tea v2 and Lip Gloss v2, introducing significant improvements to type safety, styling APIs, and message handling.

1\. Import Path Changes
-----------------------

The primary breaking change is the move to a new vanity URL and versioned module path. All imports must be updated to use the `v2` suffix.

| Component | v1 Import Path | v2 Import Path |
| --- | --- | --- |
| **Module** | `github.com/charmbracelet/bubbles` | `charm.land/bubbles/v2` |
| **Cursor** | `github.com/charmbracelet/bubbles/cursor` | `charm.land/bubbles/v2/cursor` |
| **Help** | `github.com/charmbracelet/bubbles/help` | `charm.land/bubbles/v2/help` |
| **Key** | `github.com/charmbracelet/bubbles/key` | `charm.land/bubbles/v2/key` |
| **List** | `github.com/charmbracelet/bubbles/list` | `charm.land/bubbles/v2/list` |
| **Paginator** | `github.com/charmbracelet/bubbles/paginator` | `charm.land/bubbles/v2/paginator` |
| **Progress** | `github.com/charmbracelet/bubbles/progress` | `charm.land/bubbles/v2/progress` |
| **Spinner** | `github.com/charmbracelet/bubbles/spinner` | `charm.land/bubbles/v2/spinner` |
| **Table** | `github.com/charmbracelet/bubbles/table` | `charm.land/bubbles/v2/table` |
| **Textarea** | `github.com/charmbracelet/bubbles/textarea` | `charm.land/bubbles/v2/textarea` |
| **Textinput** | `github.com/charmbracelet/bubbles/textinput` | `charm.land/bubbles/v2/textinput` |
| **Viewport** | `github.com/charmbracelet/bubbles/viewport` | `charm.land/bubbles/v2/viewport` |

> **Note:** Internal utility packages `runeutil` and `memoization` are no longer exported in v2 [UPGRADE\_GUIDE\_V2.md79](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L79-L79)

2\. Global Breaking Changes
---------------------------

Several architectural patterns have been standardized across the entire library to improve consistency and prevent accidental state mutation.

### 2.1 tea.KeyMsg to tea.KeyPressMsg

In alignment with Bubble Tea v2, all Bubbles components now listen for `tea.KeyPressMsg` instead of `tea.KeyMsg`.

Sources: [UPGRADE\_GUIDE\_V2.md94-104](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L94-L104)

### 2.2 Encapsulated Dimensions

Exported `Width` and `Height` fields have been replaced with getter and setter methods to allow components to trigger internal re-calculations (such as re-wrapping text or updating viewports) when dimensions change.

**Affected Components:** `filepicker`, `help`, `progress`, `table`, `textinput`, `viewport`.

Sources: [UPGRADE\_GUIDE\_V2.md106-122](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L106-L122)

### 2.3 KeyMap Immutability

Global `DefaultKeyMap` variables are now functions. This prevents one component instance from accidentally modifying the keybindings of all other instances in the same process.

**Affected Components:** `paginator`, `textarea`, `textinput`.

Sources: [UPGRADE\_GUIDE\_V2.md124-138](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L124-L138)

3\. Light and Dark Styles API
-----------------------------

Lip Gloss v2 removed `AdaptiveColor`. Consequently, Bubbles v2 components that provide default styling now require an explicit `isDark bool` parameter to determine which color palette to apply.

### Implementation Pattern

Components typically provide three styling functions:

1.  `DefaultStyles(isDark bool) Styles`: Returns styles based on the provided background luminosity.
2.  `DefaultDarkStyles() Styles`: Returns styles optimized for dark backgrounds.
3.  `DefaultLightStyles() Styles`: Returns styles optimized for light backgrounds.

4\. Per-Component Migration Notes
---------------------------------

### 4.1 Progress Bar

The `progress` component underwent the most significant refactoring, moving from string-based hex codes to `image/color.Color` for better integration with Lip Gloss v2's color system.

| v1 Symbol | v2 Replacement |
| --- | --- |
| `FullColor` (string) | `FullColor` (color.Color) |
| `EmptyColor` (string) | `EmptyColor` (color.Color) |
| `WithGradient(a, b)` | `WithColors(colorA, colorB)` |
| `WithScaledGradient(a, b)` | `WithScaledColors(colorA, colorB)` |

Sources: [UPGRADE\_GUIDE\_V2.md219-259](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L219-L259)

### 4.2 List and Textinput

The `list` component's internal `textinput` styling has been consolidated. Previously, `list.Styles` had individual fields for the filter prompt and cursor. These are now contained within a standard `textinput.Styles` struct.

Sources: [UPGRADE\_GUIDE\_V2.md200-205](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L200-L205)

### 4.3 Paginator

The boolean toggles for specific key sets (e.g., `UsePgUpPgDownKeys`) have been removed. Developers should now modify the `KeyMap` directly for granular control.

Sources: [UPGRADE\_GUIDE\_V2.md211-216](https://github.com/charmbracelet/bubbles/blob/42130e89/UPGRADE_GUIDE_V2.md?plain=1#L211-L216)

5\. Removed Symbols Reference
-----------------------------

The following table tracks deprecated or removed symbols and their v2 equivalents.

| Component | Removed Symbol | v2 Equivalent / Action |
| --- | --- | --- |
| **All** | `NewModel` | Use `New` |
| **Cursor** | `model.Blink` | `model.IsBlinked` |
| **Cursor** | `model.BlinkCmd()` | `model.Blink()` |
| **Filepicker** | `DefaultStylesWithRenderer(r)` | `DefaultStyles()` |
| **Paginator** | `UseHLKeys`, `UseJKKeys`, etc. | Edit `model.KeyMap` directly |
| **Progress** | `WithDefaultGradient` | `WithDefaultColors()` |
| **Progress** | `WithSolidFill` | `WithColors(color, color)` |
| **Table** | `model.Focus()`, `model.Blur()` | `model.SetFocused(bool)` |
| **Textarea** | `model.Cursor` | `model.Cursor()` (Getter) |
