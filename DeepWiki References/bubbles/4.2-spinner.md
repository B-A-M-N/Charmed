The Spinner component in the Bubbles library provides animated loading indicators for terminal applications. It offers a simple way to show that an operation is in progress, giving users visual feedback during potentially long-running tasks. For other progress indicators like progress bars, see [Progress Bar](https://deepwiki.com/charmbracelet/bubbles/4.1-progress-bar)
.

Overview
--------

The Spinner is a self-contained UI component designed to animate a sequence of frames at a specified speed. It's built to work within the Bubble Tea framework, implementing the standard Model-View-Update pattern.

Technical Implementation
------------------------

### The Animation Loop

The spinner animation is driven by a continuous loop of `TickMsg` messages. The cycle begins with an initial call to `Tick()`, which returns a message that the parent model should return as a command in its `Init` or `Update` function.

**Data Flow: Animation Tick Loop**

 [spinner/spinner.go170-182](https://github.com/charmbracelet/bubbles/blob/42130e89/spinner/spinner.go#L170-L182)
 [spinner/spinner.go184-192](https://github.com/charmbracelet/bubbles/blob/42130e89/spinner/spinner.go#L184-L192)

### ID and Tag Management

To support multiple spinners in a single application without message leakage, the component uses two internal identifiers:

1.  **ID**: A unique integer generated via `nextID()` using `atomic.AddInt64` when a new model is created. This ensures that `TickMsg` messages are routed to the correct `Model` instance. [spinner/spinner.go12-18](https://github.com/charmbracelet/bubbles/blob/42130e89/spinner/spinner.go#L12-L18)
     [spinner/spinner.go113](https://github.com/charmbracelet/bubbles/blob/42130e89/spinner/spinner.go#L113-L113)
    
2.  **Tag**: A sequence counter that increments with every processed tick. This prevents "double-ticking" where multiple messages queued in the Bubble Tea runtime might cause the spinner to spin faster than its defined `FPS`. [spinner/spinner.go142-145](https://github.com/charmbracelet/bubbles/blob/42130e89/spinner/spinner.go#L142-L145)
     [spinner/spinner.go152-153](https://github.com/charmbracelet/bubbles/blob/42130e89/spinner/spinner.go#L152-L153)

Comparison with Progress Bar
----------------------------

While both are progress indicators, they serve different UX purposes:

*   **Spinner**: Used for indeterminate tasks where the total duration is unknown. Relies on `tea.Tick` for constant-rate animation.
*   **Progress Bar**: Used for determinate tasks. Uses the `harmonica` library for spring-based physics animations and `FrameMsg` for variable-rate updates. [progress/progress.go181-184](https://github.com/charmbracelet/bubbles/blob/42130e89/progress/progress.go#L181-L184)
