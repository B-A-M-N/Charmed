The Text Area component provides a multi-line text input field for terminal applications built with the Bubble Tea framework. It supports rich editing capabilities including cursor navigation, text selection, line wrapping, scrolling, and styling. For single-line text input, see [Text Input](https://deepwiki.com/charmbracelet/bubbles/2.1-text-input)
.

Sources: [textarea/textarea.go1-3](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L1-L3)

Architecture
------------

The Text Area component follows a Model-View-Update pattern consistent with the Bubble Tea framework. It internally manages a `viewport.Model` for rendering and a `cursor.Model` for text insertion logic.

### Core Components

1.  **Model**: The main struct representing the Text Area. It stores the value as `[][]rune` to distinguish between logical lines (hard breaks) and visual lines (soft wraps). [textarea/textarea.go264-265](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L264-L265)
    
2.  **KeyMap**: Defines key bindings for actions like `InsertNewline` (Enter) and `DeleteWordBackward` (Alt+Backspace). [textarea/textarea.go78-107](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L78-L107)
    
3.  **Styles**: Stores `StyleState` for focused and blurred states, allowing the UI to change appearance based on focus. [textarea/textarea.go178-186](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L178-L186)
    
4.  **MemoCache**: An LRU cache used to store line-wrapping results, significantly improving performance for large blocks of text. [internal/memoization/memoization.go28-34](https://github.com/charmbracelet/bubbles/blob/42130e89/internal/memoization/memoization.go#L28-L34)
    
5.  **Viewport**: Manages the visible window of text and handles vertical scrolling when the cursor moves beyond the `Height`. [textarea/textarea.go267-280](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L267-L280)

Text Handling and Soft Wrapping
-------------------------------

The Text Area handles text as a `[][]rune`. This structure treats each element in the outer slice as a logical line (separated by `\n`).

### Soft Wrapping Pipeline

When rendering, logical lines are "soft-wrapped" to fit the `Width`. This process is computationally expensive, so results are cached using the `MemoCache`.

The `wrap` function handles:

*   **Word Wrapping**: Breaking lines at spaces. [textarea/textarea.go1398-1461](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L1398-L1461)
    
*   **Character Wrapping**: Breaking mid-word if a single word exceeds the component width. [textarea/textarea.go1438-1445](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L1438-L1445)
    
*   **Rune Width**: Utilizing `go-runewidth` to account for CJK or Emoji characters that occupy two cells. [textarea/textarea.go1425-1435](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L1425-L1435)
    

Sources: [textarea/textarea.go1398-1461](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L1398-L1461)
 [internal/memoization/memoization.go63-74](https://github.com/charmbracelet/bubbles/blob/42130e89/internal/memoization/memoization.go#L63-L74)

Coordinate Mapping
------------------

Text Area manages three distinct coordinate spaces:

1.  **Logical Space**: `(row, col)` indices into the `[][]rune`. [textarea/textarea.go261-262](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L261-L262)
    
2.  **Visual Space**: The lines as they appear after soft-wrapping.
3.  **Viewport Space**: The lines currently visible on the screen based on `yOffset`.

The `LineInfo()` method provides a bridge between these spaces, calculating `ColumnOffset` and `RowOffset` for the cursor relative to the start of a soft-wrapped visual line. [textarea/textarea.go111-138](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L111-L138)

Configuration Options

| Option | Description | Default |
| --- | --- | --- |
| `CharLimit` | Total character limit across all lines. | 0 (No limit) |
| `MaxHeight` | Maximum height the component can grow to. | 99  |
| `MaxWidth` | Maximum width the component can grow to. | 500 |

| `Prompt` | String displayed before every line. | `│` |

Style System
------------

The `Styles` struct defines the appearance for `Focused` and `Blurred` states. Each state contains a `StyleState` which styles sub-elements:

*   **Base**: The overall container. [textarea/textarea.go130-131](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L130-L131)
    
*   **CursorLine**: The line where the cursor currently resides. [textarea/textarea.go134-135](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L134-L135)
    
*   **Placeholder**: Text shown when the value is empty. [textarea/textarea.go146-147](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L146-L147)
    
*   **EndOfBuffer**: The character shown for lines beyond the content (e.g., `~`). [textarea/textarea.go140-141](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L140-L141)

Viewport Integration
--------------------

Text Area uses an internal `viewport.Model` to handle scrolling. When the cursor moves vertically (e.g., `LineNext`), the component checks if the new visual row is within the viewport's range. If not, it calls `viewport.SetYOffset()` or `viewport.ScrollDown()` to keep the cursor visible. [textarea/textarea.go857-866](https://github.com/charmbracelet/bubbles/blob/42130e89/textarea/textarea.go#L857-L866)
