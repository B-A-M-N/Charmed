Markdown Processing
===================

This document details how Glow processes and renders Markdown content from various sources. The Markdown Processing system is responsible for acquiring content, preprocessing it, and rendering it for display using the Glamour library. For information about how the rendered content is presented in the user interface, see [Terminal User Interface](https://deepwiki.com/charmbracelet/glow/2.2-terminal-user-interface)
.

Source Acquisition
------------------

Glow can process markdown content from multiple sources:

Sources: [main.go73-149](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L73-L149)

The `sourceFromArg` function handles different input types:

1.  **Standard Input**: When the input argument is "-", content is read from stdin
2.  **URLs**:
    *   GitHub/GitLab URLs are processed to fetch README files using their APIs
    *   HTTP/HTTPS URLs are fetched directly
3.  **Directories**:
    *   When a directory is provided, Glow searches for files named README.md, README, etc.
4.  **Local Files**:
    *   Any other argument is treated as a path to a local file

### Content Type Detection

Glow determines whether the content is markdown or code based on the file extension:

Sources: [utils/utils.go53-70](https://github.com/charmbracelet/glow/blob/752de97c/utils/utils.go#L53-L70)

The system recognizes the following extensions as markdown: `.md`, `.mdown`, `.mkdn`, `.mkd`, and `.markdown`.

Rendering with Glamour
----------------------

The core of Glow's markdown processing is the rendering done by the Glamour library. Two main functions handle rendering:

1.  `executeCLI` in main.go for CLI mode
2.  `glamourRender` in ui/pager.go for TUI mode

### Style Selection and Configuration

Glow offers multiple styling options for rendered markdown:

The available built-in styles include:

*   `auto`: Automatically selects light or dark based on terminal
*   `dark`: Dark theme
*   `light`: Light theme
*   `notty`: Style for non-terminal output
*   Other styles like `dracula` and `pink`

Custom styles can be provided as paths to JSON style definition files.

Special Handling for Code Files
-------------------------------

When processing non-markdown files (code files), Glow applies special handling:

1.  The content is wrapped in a markdown code block with the appropriate language based on file extension
2.  Style configuration is modified to remove indentation from code blocks
3.  Line wrapping may be disabled for code content

This allows Glow to display syntax-highlighted code with appropriate formatting.

Sources: [utils/utils.go44-46](https://github.com/charmbracelet/glow/blob/752de97c/utils/utils.go#L44-L46)

Output Methods
--------------

After rendering, content can be displayed in three ways:

| Output Method | Description | Implementation |
| --- | --- | --- |
| Direct Output | Content is written directly to stdout | Simple print to terminal |
| External Pager | Content is displayed via an external pager (e.g., `less`) | Uses `$PAGER` environment variable or defaults to `less -r` |
| TUI Document View | Content is displayed in Glow's TUI interface | Uses the `pagerModel` to render and display markdown with interactive features |

The output method is determined by command-line flags, with the TUI being the default when viewing directories.

Sources: [main.go314-340](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L314-L340)
