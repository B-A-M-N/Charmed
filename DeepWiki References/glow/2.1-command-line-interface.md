Purpose and Scope
-----------------

This document details the Command-Line Interface (CLI) of Glow, a terminal-based markdown renderer. The CLI handles user input parsing, markdown source resolution, rendering, and output presentation. For information about the Terminal User Interface (TUI), see [Terminal User Interface](https://deepwiki.com/charmbracelet/glow/2.2-terminal-user-interface)
.

Overview
--------

Glow's CLI provides a flexible interface for viewing markdown content from various sources with customizable rendering options. It serves as the primary entry point for the application and determines whether to render markdown directly, use a pager, or launch the interactive TUI.

Command Structure
-----------------

Glow's command structure is built using Cobra, with a root command and subcommands for configuration and documentation.

Sources: [main.go46-63](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L46-L63)
 [config\_cmd.go28-54](https://github.com/charmbracelet/glow/blob/752de97c/config_cmd.go#L28-L54)

### Root Command

The root command accepts an optional source or directory argument and various flags that control rendering behavior:

| Flag | Short | Description | Default |
| --- | --- | --- | --- |
| `--pager` | `-p` | Display with pager | `false` |
| `--tui` | `-t` | Display with TUI | `false` |
| `--style` | `-s` | Style name or JSON path | `"auto"` |
| `--width` | `-w` | Word-wrap at width | `0` (auto) |
| `--all` | `-a` | Show system files and directories (TUI-mode only) | `false` |
| `--line-numbers` | `-l` | Show line numbers (TUI-mode only) | `false` |
| `--preserve-new-lines` | `-n` | Preserve newlines in the output | `false` |
| `--mouse` | `-m` | Enable mouse wheel (TUI-mode only) | `false` |
| `--config` |     | Config file path | System default |

Sources: [main.go396-405](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L396-L405)
 [main.go407-416](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L407-L416)

### Config Command

The `config` command opens the configuration file in an editor, creating the file with default values if it doesn't exist.

    glow config [--config path/to/config.yml]

Input Source Resolution
-----------------------

One of the key features of Glow's CLI is its ability to resolve markdown content from various sources.

### Local File and Directory Handling

When given a file path, Glow opens and reads the file directly. If a directory is provided, Glow:

1.  Walks through the directory tree
2.  Looks for common README filenames (e.g., `README.md`, `Readme.md`)
3.  Uses the first matching file found

If no argument is provided, Glow treats the current directory (`.`) as the input.

Sources: [main.go104-138](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L104-L138)
 [main.go139-148](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L139-L148)

### Remote Source Handling

Glow supports fetching markdown content from:

1.  **GitHub repositories**: Resolves to the repository's README using the GitHub API
2.  **GitLab repositories**: Resolves to the repository's README using the GitLab API
3.  **HTTP(S) URLs**: Directly fetches content from the specified URL

URL handling supports both explicit URL formats (`https://github.com/user/repo`) and abbreviated forms (`github://user/repo`).

Markdown Processing and Output
------------------------------

After obtaining markdown content from the source, the CLI processes and displays it using one of three methods.

Sources: [main.go272-341](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L272-L341)

### Rendering Configuration

The rendering process is customized using several parameters:

1.  **Style**: Determines the color scheme and formatting (default: `auto`)
2.  **Width**: Controls word-wrapping width (auto-detected if not specified)
3.  **Base URL**: Used for relative links, derived from the source URL
4.  **Preserved new lines**: Optional preservation of newline characters

For code files (non-markdown), Glow wraps the content in a code block based on the file extension.

Sources: [main.go290-297](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L290-L297)
 [main.go302-306](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L302-L306)

### Output Methods

Glow offers three output methods:

1.  **Direct output**: Renders to stdout (default)
2.  **Pager**: Pipes rendered output through an external pager (using `$PAGER` or `less -r`)
3.  **TUI mode**: Launches the interactive Terminal User Interface

Sources: [main.go314-340](https://github.com/charmbracelet/glow/blob/752de97c/main.go#L314-L340)

Configuration System
--------------------

Glow's CLI behavior is influenced by configuration from multiple sources, with command-line flags taking precedence.

### Configuration Sources

Glow's configuration is loaded from the following sources, in order of precedence:

1.  Command-line flags
2.  Environment variables prefixed with `GLOW_`
3.  Configuration file (`glow.yml`)

The default configuration file is located in the user's config directory and contains these settings:

 [config\_cmd.go16-26](https://github.com/charmbracelet/glow/blob/752de97c/config_cmd.go#L16-L26)

### Configuration File Management

The `glow config` command provides a simple way to edit the configuration file. When executed:

1.  Ensures the configuration file exists (creates it with defaults if needed)
2.  Opens the file in the user's preferred editor (using `$EDITOR`)
3.  Saves changes automatically when the editor closes

### GitHub and GitLab Integration

For repository URLs, Glow makes API requests to GitHub or GitLab to:

1.  Locate the repository's README file
2.  Retrieve download URLs for the raw content
3.  Fetch the markdown content

This enables viewing READMEs directly from repositories without manual downloading.

### External Programs

The CLI can interact with external programs:

1.  **Pager**: When using `--pager`, output is piped to `$PAGER` or `less -r`
2.  **Editor**: The `config` command opens the config file in the user's `$EDITOR`
