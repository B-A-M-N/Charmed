Terminal User Interface
=======================

Relevant source files

*   [console\_windows.go](https://github.com/charmbracelet/glow/blob/752de97c/console_windows.go)
    
*   [go.mod](https://github.com/charmbracelet/glow/blob/752de97c/go.mod)

This document details the Terminal User Interface (TUI) system in Glow, explaining its architecture, components, state management, and message handling. The TUI provides an interactive way to browse and view markdown files in the terminal. For details on the Command Line Interface that works alongside the TUI, see [Command-Line Interface](https://deepwiki.com/charmbracelet/glow/2.1-command-line-interface)
.

Overview
--------

Glow's TUI is built using the [Bubble Tea](https://github.com/charmbracelet/glow/blob/752de97c/Bubble%20Tea)
 library, which implements an Elm-like architecture (Model-View-Update pattern). The interface offers two main views:

1.  **Stash View**: A browsable list of markdown files with filtering capabilities
2.  **Document View**: A rendered view of a selected markdown document

The TUI handles navigation between these views, keyboard shortcuts, window resizing, and file watching for live updates.

Architecture
------------

The TUI follows the Model-View-Update (MVU) architecture pattern, which consists of three main components:

1.  **Model**: Data structure that holds application state
2.  **View**: Function that renders the model state as a string
3.  **Update**: Function that handles messages and updates the model

The TUI system is composed of these core components:

The `model` struct is the top-level component that manages the overall application state and contains two sub-models:

1.  `stashModel`: Manages the file listing view
2.  `pagerModel`: Manages the document viewing experience

The `commonModel` contains shared data like configuration and window dimensions that both sub-models need access to.

State Management
----------------

The TUI has two primary states, controlled by the `state` type in the main model:

| State | Constant | Description |
| --- | --- | --- |
| Stash View | `stateShowStash` | Shows a listing of markdown files for browsing |
| Document View | `stateShowDocument` | Shows a rendered markdown document |

Key Commands and Message Types
------------------------------

The TUI uses various commands to handle asynchronous operations:

| Command | Purpose |
| --- | --- |
| `findLocalFiles` | Searches for markdown files in the local directory |
| `findNextLocalFile` | Retrieves the next markdown file from the finder channel |
| `renderWithGlamour` | Renders markdown content using Glamour library |
| `waitForStatusMessageTimeout` | Sets a timeout for status messages |

Message types that flow through the system include:

*   `tea.KeyMsg`: Keyboard input
*   `tea.WindowSizeMsg`: Window size changes
*   `foundLocalFileMsg`: A local markdown file has been found
*   `filteredMarkdownMsg`: Markdown list has been filtered
*   `contentRenderedMsg`: Markdown content has been rendered
*   `statusMessageTimeoutMsg`: Status message timeout triggered

Sources: [ui/ui.go356-419](https://github.com/charmbracelet/glow/blob/752de97c/ui/ui.go#L356-L419)

Keyboard Shortcuts
------------------

The TUI responds to the following keyboard shortcuts:

| Key | Context | Action |
| --- | --- | --- |
| `q` | Both views | Quit the application |
| `Esc` | Document view | Return to stash view |
| `Ctrl+c` | Both views | Quit the application |
| `r` | Stash view | Refresh file listing |
| `left`/`h`/`delete` | Document view | Return to stash view |
| `Ctrl+z` | Both views | Suspend application |

Sources: [ui/ui.go216-261](https://github.com/charmbracelet/glow/blob/752de97c/ui/ui.go#L216-L261)

Extending the TUI
-----------------

The modular design of the TUI makes it extensible in several ways:

1.  Adding new views by creating new models that implement the tea.Model interface
2.  Adding new message types for new functionality
3.  Extending existing models with additional state and behavior

For detailed guidance on extending Glow, see [Extending Glow](https://deepwiki.com/charmbracelet/glow/4.2-code-quality-and-testing)
