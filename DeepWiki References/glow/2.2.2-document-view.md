Document View
=============

Relevant source files

*   [ui/editor.go](https://github.com/charmbracelet/glow/blob/752de97c/ui/editor.go)

Purpose and Scope
-----------------

The Document View system implements the markdown document viewer in Glow's TUI mode. It is responsible for displaying rendered markdown content, managing viewport scrolling, watching files for changes, and handling user interactions like editing and copying. This component is implemented by the `pagerModel` struct and associated functions in [ui/pager.go](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go)

For information about the file browser interface, see [Stash View](https://deepwiki.com/charmbracelet/glow/2.2.1-stash-view)
. For details on the markdown rendering pipeline, see [Markdown Processing](https://deepwiki.com/charmbracelet/glow/2.3-markdown-processing)
.

Component Architecture
----------------------

The `pagerModel` struct serves as the core component of the document viewer, managing display state, viewport operations, and file watching capabilities.

### Core Structure

**Sources:** [ui/pager.go92-106](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L92-L106)

The `pagerModel` is initialized via `newPagerModel()` which creates a viewport with configurable high-performance rendering and initializes the file system watcher.

| Field | Type | Purpose |
| --- | --- | --- |
| `common` | `*commonModel` | Shared state including terminal dimensions and configuration |
| `viewport` | `viewport.Model` | Bubbles component for scrollable content display |
| `state` | `pagerState` | Current state (browsing or showing status message) |
| `showHelp` | `bool` | Whether the help overlay is visible |
| `statusMessage` | `string` | Current status message text |
| `statusMessageTimer` | `*time.Timer` | Timer for auto-dismissing status messages |
| `currentDocument` | `markdown` | Cached document for re-rendering on resize |
| `watcher` | `*fsnotify.Watcher` | File system watcher for live reload |

**Sources:** [ui/pager.go92-106](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L92-L106)
 [ui/pager.go108-121](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L108-L121)

Viewport Management and Rendering
---------------------------------

The document view uses the `viewport.Model` from the Bubbles library to handle scrollable content display with optional high-performance rendering.

### Viewport Configuration

**Sources:** [ui/pager.go108-121](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L108-L121)
 [ui/pager.go123-133](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L123-L133)

The viewport height calculation accounts for the status bar (1 line) and optionally the help view. The constant `statusBarHeight` is defined as 1, and `pagerHelpHeight` is calculated dynamically based on the help text.

### Glamour Rendering Pipeline

The `glamourRender()` function performs the markdown-to-ANSI conversion with several configurable options.

**Sources:** [ui/pager.go422-480](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L422-L480)

Key rendering parameters:

*   **Width calculation**: `min(GlamourMaxWidth, viewport.Width)` for markdown, 0 for code [ui/pager.go430-433](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L430-L433)
    
*   **Style selection**: `GlamourStyle()` utility function with code detection [ui/pager.go436](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L436-L436)
    
*   **Line numbers**: Added when `ShowLineNumbers` is enabled or viewing non-markdown files [ui/pager.go466-468](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L466-L468)
    
*   **Truncation**: Lines are truncated to `viewport.Width - lineNumberWidth` [ui/pager.go423](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L423-L423)
    

File Watching and Live Reload
-----------------------------

The pager implements automatic reloading when the displayed file is modified on disk using the `fsnotify` library.

### Watcher Lifecycle

**Sources:** [ui/pager.go482-535](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L482-L535)

### Implementation Details

| Function | Purpose | Implementation |
| --- | --- | --- |
| `initWatcher()` | Creates fsnotify watcher instance | [ui/pager.go482-488](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L482-L488) |
| `watchFile()` | Adds directory to watcher, loops on events | [ui/pager.go490-520](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L490-L520) |
| `unwatchFile()` | Removes directory from watcher | [ui/pager.go522-531](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L522-L531) |
| `localDir()` | Extracts directory from document path | [ui/pager.go533-535](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L533-L535) |

The watcher monitors the **directory** containing the file (not the file itself) and filters events by checking `event.Name != m.currentDocument.localPath`. Only `fsnotify.Write` and `fsnotify.Create` events trigger a reload via `reloadMsg{}`.

**Sources:** [ui/pager.go502-512](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L502-L512)

External Editor Integration
---------------------------

The pager can launch an external editor for the current document, opening it at the current scroll position.

### Editor Flow

**Sources:** [ui/pager.go218-228](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L218-L228)
 [ui/editor.go10-19](https://github.com/charmbracelet/glow/blob/752de97c/ui/editor.go#L10-L19)

### Line Number Calculation

The cursor position in the editor is determined by converting the viewport's scroll percentage to a line number:

    lineno = TotalLineCount() × ScrollPercent()
    

Special case: If at the top of the document (`AtTop()`), `lineno` is forced to 0.

**Sources:** [ui/pager.go219-222](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L219-L222)

The `openEditor()` function uses the `charmbracelet/x/editor` package to respect the user's `$EDITOR` environment variable. The process is launched via `tea.ExecProcess()` which suspends the TUI, runs the editor, and resumes when the editor exits.

### Copy Functionality

The copy operation uses a dual-clipboard approach for maximum compatibility:

1.  **OSC 52 escape sequences**: `termenv.Copy()` for terminal-based copy

After copying, a status message "Copied contents" is displayed.

Status Bar and Help System
--------------------------

The status bar provides persistent feedback about scroll position and document information, while the help overlay offers detailed key binding reference.

### Status Bar Layout

**Sources:** [ui/pager.go296-366](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L296-L366)

The status bar adapts its styling based on the current state:

| State | Text Source | Style Variables |
| --- | --- | --- |
| `pagerStateBrowse` | `currentDocument.Note` | `statusBarNoteStyle`, `statusBarScrollPosStyle`, `statusBarHelpStyle` |
| `pagerStateStatusMessage` | `statusMessage` | `statusBarMessageStyle`, `statusBarMessageScrollPosStyle`, `statusBarMessageHelpStyle` |

The document note is truncated with ellipsis to fit the available width, calculated by subtracting the widths of all other components from `common.width`.

**Sources:** [ui/pager.go326-342](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L326-L342)

### Color Definitions

| Style Element | Light Mode | Dark Mode |

| `mintGreen` (message fg) | #89F0CB | #89F0CB |
| `darkGreen` (message bg) | #1C8760 | #1C8760 |
| `lineNumberFg` | #656565 | #7D7D7D |
| `statusBarNoteFg` | #656565 | #7D7D7D |
| `statusBarBg` | #E6E6E6 | #242424 |

**Sources:** [ui/pager.go32-77](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L32-L77)

### Help View

The help overlay is toggled with the `?` key and displays a two-column layout of key bindings. The `toggleHelp()` function recalculates viewport height to accommodate the help text.

**Sources:** [ui/pager.go139-145](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L139-L145)
 [ui/pager.go368-406](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L368-L406)

The help view content is generated by `helpView()`, which formats key bindings in a grid layout and pads lines with spaces for consistent background coloring. The `pagerHelpHeight` is cached after first render to avoid recalculation.

**Sources:** [ui/pager.go127-132](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L127-L132)
 [ui/pager.go368-406](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L368-L406)

Unload and Cleanup
------------------

When the user exits the pager (typically by pressing ESC to return to the stash view), the `unload()` method performs cleanup operations.

### Cleanup Sequence

**Sources:** [ui/pager.go167-179](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go#L167-L179)

The cleanup ensures:

*   Help overlay is hidden
*   Status message timer is stopped to prevent memory leaks
*   State is reset to browse mode
*   Viewport is cleared
*   Scroll position is reset
*   File watcher is removed from the directory

This prepares the pager for the next document load while preventing resource leaks.
