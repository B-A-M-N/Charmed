UI Components and Styling
=========================

Relevant source files

*   [console\_windows.go](https://github.com/charmbracelet/glow/blob/752de97c/console_windows.go)
    
*   [go.mod](https://github.com/charmbracelet/glow/blob/752de97c/go.mod)

This document covers the shared UI components and styling infrastructure used across Glow's terminal interface. It focuses on the `markdown` struct that represents document metadata, the lipgloss-based styling system that provides consistent visual appearance, and common rendering utilities for text normalization and time formatting.

For information about how these components are used in specific views, see [Stash View](https://deepwiki.com/charmbracelet/glow/2.2.1-stash-view)
 and [Document View](https://deepwiki.com/charmbracelet/glow/2.2.2-document-view)
. For the markdown rendering pipeline that generates content, see [Markdown Processing](https://deepwiki.com/charmbracelet/glow/2.3-markdown-processing)
.

The markdown Struct
-------------------

The `markdown` struct serves as the core data model for documents throughout the TUI, storing both content and metadata for markdown files.

### Structure Definition

**Sources:** [ui/markdown.go16-29](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L16-L29)

| Field | Type | Purpose |
| --- | --- | --- |
| `localPath` | `string` | Full path for local markdown files; only relevant for local documents and stashed items |
| `filterValue` | `string` | Normalized text used during filtering; ephemeral field maintained only during active filtering |
| `Body` | `string` | Rendered markdown content |
| `Note` | `string` | Display name or title for the document |
| `Modtime` | `time.Time` | Last modification timestamp for relative time display |

**Sources:** [ui/markdown.go16-29](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L16-L29)

### Filter Value Generation

The `buildFilterValue()` method prepares text for fuzzy matching by normalizing the document's note field. This normalization removes diacritics and other marks to improve search accuracy across different character sets.

**Sources:** [ui/markdown.go32-40](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L32-L40)

The normalization process uses Unicode transformation chains:

1.  Decompose characters into base + combining marks (NFD)
2.  Remove nonspacing marks (Mn category)
3.  Recompose into canonical form (NFC)

**Sources:** [ui/markdown.go46-56](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L46-L56)

### Relative Time Formatting

The `relativeTime()` method provides human-readable timestamps that adapt based on how recent the modification time is:

| Time Range | Format | Example |
| --- | --- | --- |
| < 1 minute | "just now" | "just now" |
| < 1 week | Relative (custom) | "5 minutes ago", "2 hours ago" |
| ≥ 1 week | Absolute | "02 Jan 2006 15:04 MST" |

**Sources:** [ui/markdown.go42-44](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L42-L44)
 [ui/markdown.go59-67](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L59-L67)

The relative time magnitudes are configured in a table that defines format strings for different time durations:

**Sources:** [ui/markdown.go59-67](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L59-L67)
 [ui/markdown.go69-88](https://github.com/charmbracelet/glow/blob/752de97c/ui/markdown.go#L69-L88)

Platform-Specific Color Support
-------------------------------

### Windows ANSI Color Enablement

On Windows platforms, Glow automatically enables ANSI color sequence support through the Windows Console API. This is necessary for Windows 10+ to properly render colored terminal output.

**Sources:** [console\_windows.go13-23](https://github.com/charmbracelet/glow/blob/752de97c/console_windows.go#L13-L23)

The implementation:

1.  Retrieves the stdout handle as a Windows `Handle`
2.  Gets the current console mode flags
3.  Sets the console mode with `ENABLE_VIRTUAL_TERMINAL_PROCESSING` flag added
4.  Executes automatically via `init()` at program startup

This code is conditionally compiled only for Windows builds using the build tag `// +build windows`.

**Sources:** [console\_windows.go1-23](https://github.com/charmbracelet/glow/blob/752de97c/console_windows.go#L1-L23)

Style Application Pattern
-------------------------

The styling system uses a consistent pattern throughout the codebase where pre-defined style functions are applied to text strings:

**Sources:** [ui/styles.go27-41](https://github.com/charmbracelet/glow/blob/752de97c/ui/styles.go#L27-L41)

This approach provides:

*   **Centralized color management**: All colors defined in one location
*   **Adaptive theming**: Automatic adjustment for light/dark terminals
*   **Type safety**: Compile-time verification of style usage
*   **Reusability**: Styles can be composed and extended

The pattern is consistent with lipgloss best practices where styles are defined once and reused throughout the application, ensuring visual consistency across all TUI components.

**Sources:** [ui/styles.go1-46](https://github.com/charmbracelet/glow/blob/752de97c/ui/styles.go#L1-L46)
