Purpose and Scope
-----------------

The Stash View is the primary file browsing interface in Glow's TUI mode. It displays a paginated, filterable list of local markdown documents and allows users to navigate, search, and select files for viewing. This component implements a three-state machine for managing user interactions during document browsing and filtering operations.

For information about the document viewing interface that appears after selecting a file, see [Document View](https://deepwiki.com/charmbracelet/glow/2.2.2-document-view)
. For the overall TUI architecture and how views are coordinated, see [Terminal User Interface](https://deepwiki.com/charmbracelet/glow/2.2-terminal-user-interface)
.

---------------------

State Management
----------------

### View States (stashViewState)

The `stashViewState` enum controls high-level UI behavior [ui/stash.go60-67](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L60-L67)
:

*   **`stashStateReady`**: Normal browsing mode, accepting user input
*   **`stashStateLoadingDocument`**: Transitioning to pager view, showing loading spinner
*   **`stashStateShowingError`**: Displaying error message, awaiting dismissal

The state determines which update handler processes messages [ui/stash.go446-456](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L446-L456)

### Filter States (filterState)

:

*   **`unfiltered`**: No filter active, displaying all documents
*   **`filtering`**: User actively typing in filter input field
*   **`filterApplied`**: Filter applied, user browsing filtered results

Filter state affects which documents are visible via `getVisibleMarkdowns()` [ui/stash.go309-315](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L309-L315)

Document Management
-------------------

### Adding and Sorting Documents

Documents are added through `addMarkdowns()` [ui/stash.go295-306](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L295-L306)
 which:

1.  Appends new markdown pointers to `m.markdowns`
2.  Sorts documents if no filter is applied using `sortMarkdowns()`
3.  Updates pagination to reflect new document count

### Document Selection

The currently selected document is calculated based on page and cursor position [ui/stash.go278-292](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L278-L292)
:

    selectedIndex = (currentPage × itemsPerPage) + cursorPosition
    

The `selectedMarkdown()` method returns `nil` if the index is out of bounds, which guards against race conditions during filtering.

### Fuzzy Matching Implementation

The filtering system uses the `sahilm/fuzzy` library to perform substring matching [ui/stash.go868-891](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L868-L891)

**Filtering Pipeline**

### Filter State Transitions

Key keyboard interactions during filtering [ui/stash.go600-666](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L600-L666)
:

| Key | Current State | Action |
| --- | --- | --- |
| `/` | `unfiltered` | Enter filtering mode, focus input |
| `Esc` | `filtering` | Cancel filter, return to unfiltered |
| `Enter` | `filtering` | Apply filter (creates filterSection) |
| `Esc` | `filterApplied` | Clear filter, remove filterSection |
| `/` | `filterApplied` | Edit existing filter |

When a filter is applied with only one result, the system automatically opens that document [ui/stash.go626-633](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L626-L633)

Pagination System
-----------------

### Paginator Configuration

The stash view uses bubbles' `paginator.Model` with dot-style indicators [ui/stash.go402-408](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L402-L408)
:

Each section maintains its own paginator instance, preserving position when switching between tabs.

### Dynamic Page Calculation

The `updatePagination()` method recalculates pagination parameters based on available screen space [ui/stash.go255-275](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L255-L275)
:

    availableHeight = totalHeight - topPadding - helpHeight - bottomPadding
    itemsPerPage = max(1, availableHeight / stashViewItemHeight)
    totalPages = ceil(documentCount / itemsPerPage)
    

The constant `stashViewItemHeight` is 3 lines [ui/stash.go24](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L24-L24)
 accounting for the item content plus spacing.

### Cursor Boundary Management

Cursor movement functions ensure the cursor stays within valid bounds [ui/stash.go333-372](https://github.com/charmbracelet/glow/blob/752de97c/ui/stash.go#L333-L372)
:

*   **`moveCursorUp()`**: Wraps to previous page when cursor < 0
*   **`moveCursorDown()`**: Advances to next page when cursor exceeds itemsOnPage
*   Edge cases handle first/last page boundaries
