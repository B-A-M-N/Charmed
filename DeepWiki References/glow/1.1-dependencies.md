Dependencies
============

Relevant source files

*   [console\_windows.go](https://github.com/charmbracelet/glow/blob/752de97c/console_windows.go)
    
*   [go.mod](https://github.com/charmbracelet/glow/blob/752de97c/go.mod)
    

This document provides a comprehensive overview of Glow's external dependencies, cataloging the third-party libraries that power its terminal user interface, command-line interface, and markdown rendering capabilities. The dependencies are organized by functional category, with detailed explanations of their roles in the application architecture.

For information about how these dependencies are configured and used at runtime, see [Configuration System](https://deepwiki.com/charmbracelet/glow/3-configuration-system)
. For details on the build and release process that manages these dependencies, see [Build and Release Process](https://deepwiki.com/charmbracelet/glow/4.1-build-and-release-process)
.

Dependency Overview
-------------------

Glow's dependencies fall into six primary categories, all managed through Go modules in [go.mod1-76](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L1-L76)
:

| Category | Purpose | Key Libraries |
| --- | --- | --- |
| **TUI Framework** | Provides the terminal user interface foundation using the Model-View-Update pattern | `bubbletea`, `bubbles`, `lipgloss` |
| **Markdown Rendering** | Converts markdown to styled ANSI terminal output | `glamour` |
| **CLI Framework** | Handles command-line argument parsing and command structure | `cobra`, `viper` |
| **File System Operations** | File watching, path resolution, and git integration | `fsnotify`, `go-app-paths`, `gitcha` |
| **Text Processing** | Terminal capabilities, text wrapping, and unicode handling | `termenv`, `reflow`, `go-runewidth` |
| **Utilities** | Search, clipboard, humanization, and environment parsing | `fuzzy`, `clipboard`, `humanize`, `env` |

**Sources:** [go.mod1-76](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L1-L76)

Charmbracelet Ecosystem
-----------------------

Glow's core user interface and rendering capabilities are built on the Charmbracelet ecosystem, a collection of Go libraries designed for terminal applications.

### bubbletea (v1.3.10)

The Model-View-Update (MVU) framework that powers Glow's entire TUI. `bubbletea` provides:

*   Event loop management for keyboard and mouse input
*   Window resize handling
*   Terminal state management
*   Message-based state updates

**Usage in Glow:**

*   The `ui.NewProgram` function initializes a `tea.Program` for TUI mode
*   The `model` struct implements `tea.Model` interface with `Init()`, `Update()`, and `View()` methods
*   State transitions between `stashModel` and `pagerModel` are handled through the bubbletea message system

**Sources:** [go.mod11](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L11-L11)

### bubbles (v0.21.0)

Pre-built, composable TUI components that integrate with bubbletea. Glow uses:

*   `viewport.Model` for scrollable content in the pager view
*   `spinner.Model` for loading indicators during file operations
*   `textinput.Model` for filtering and search functionality in the stash view

**Usage in Glow:**

*   The `pagerModel` contains a `viewport.Model` at [ui/pager.go](https://github.com/charmbracelet/glow/blob/752de97c/ui/pager.go)
    
*   The `stashModel` uses `textinput.Model` for filtering file lists

**Sources:** [go.mod10](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L10-L10)

### lipgloss (v1.1.1)

Terminal styling library using a CSS-like API. Provides:

*   Color and style definitions
*   Layout primitives (borders, padding, alignment)
*   Adaptive color profiles based on terminal capabilities

**Usage in Glow:**

*   [ui/styles.go](https://github.com/charmbracelet/glow/blob/752de97c/ui/styles.go)
     defines all color schemes and text styles
*   Styles are applied throughout the UI for consistent theming
*   Used extensively in both `stashModel.View()` and `pagerModel.View()` rendering

**Sources:** [go.mod13](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L13-L13)

### glamour (v0.10.0)

Markdown rendering engine that converts markdown to styled terminal output. Features:

*   ANSI escape sequence generation
*   Syntax highlighting for code blocks (via chroma)
*   Style customization through JSON style files
*   Support for various markdown extensions

**Usage in Glow:**

*   [main.go](https://github.com/charmbracelet/glow/blob/752de97c/main.go)
     uses `glamour.NewTermRenderer()` for CLI mode rendering
*   The `pagerModel.glamourRender()` method renders markdown for TUI display
*   Style selection is configured via `GLAMOUR_STYLE` environment variable

**Sources:** [go.mod12](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L12-L12)

### Additional Charmbracelet Libraries

**log (v0.4.2):** Structured logging with level-based filtering.

**x/editor (v0.1.0):** Opens files in the user's preferred editor (`$EDITOR` or `$VISUAL`). Used by the pager view to enable in-place markdown editing.

**Sources:** [go.mod14-15](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L14-L15)

CLI and Configuration Framework
-------------------------------

Glow's command-line interface is built on industry-standard CLI and configuration libraries.

### cobra (v1.10.2)

Command-line interface framework providing:

*   Subcommand structure (`glow config`, `glow man`)
*   Flag parsing and validation
*   Help text generation
*   Shell completion support

**Code Entities:**

*   `rootCmd` in [main.go](https://github.com/charmbracelet/glow/blob/752de97c/main.go)
     defines the root command
*   `configCmd` subcommand for editing configuration
*   `manCmd` subcommand for generating man pages

**Sources:** [go.mod27](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L27-L27)

### viper (v1.21.0)

Configuration management library that supports:

*   Multiple configuration formats (YAML, TOML, JSON)
*   Environment variable binding
*   Configuration file discovery
*   Layered configuration merging

**Usage in Glow:**

*   Loads `glow.yml` from standard config directories
*   Reads `GLOW_*` and `GLAMOUR_STYLE` environment variables
*   Provides defaults for `style`, `width`, `pager`, and other settings
*   The `validateOptions()` function in [main.go](https://github.com/charmbracelet/glow/blob/752de97c/main.go)
     retrieves viper values

**Sources:** [go.mod28](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L28-L28)

File System and Version Control
-------------------------------

These dependencies handle file operations, path resolution, and git repository integration.

### fsnotify (v1.9.0)

Cross-platform file system notification library. Enables:

*   Real-time file change detection
*   Directory watching
*   Event filtering (create, write, remove, rename)

**Usage in Glow:**

*   The `pagerModel` uses fsnotify to implement live reload functionality
*   When a markdown file is edited externally, the pager automatically re-renders the content
*   Watcher is created per-file when viewing documents

**Sources:** [go.mod17](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L17-L17)

### go-app-paths (v0.2.2)

XDG Base Directory specification implementation. Provides:

*   Config directory resolution (`~/.config/glow/`)
*   Data directory resolution (`~/.local/share/glow/`)
*   Cache directory resolution
*   Cross-platform path handling (Windows, macOS, Linux)

**Usage in Glow:**

*   Locates `glow.yml` configuration file
*   Stores stash database and cached files
*   Used in the `stashModel` for file discovery

**Sources:** [go.mod21](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L21-L21)

### go-homedir (v1.1.0)

Reliable home directory resolution across platforms. Handles:

*   `~` expansion in file paths
*   Windows `%USERPROFILE%` resolution
*   Fallback mechanisms when `$HOME` is unset

**Sources:** [go.mod19](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L19-L19)

### gitcha (v0.3.0)

Git repository information extraction. Provides:

*   Repository root detection
*   Commit information retrieval
*   Branch name detection
*   Remote URL parsing

**Usage in Glow:**

*   Used by the stash system to discover markdown files in git repositories
*   Enables filtering by git-tracked files only
*   Supports GitHub/GitLab URL resolution

**Sources:** [go.mod20](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L20-L20)

Text Processing and Terminal Capabilities
-----------------------------------------

These libraries handle text layout, terminal feature detection, and unicode processing.

### termenv (v0.16.0)

Terminal capability detection and color profile management:

*   ANSI/256-color/TrueColor support detection
*   Background color detection
*   Terminal feature querying
*   Color adaptation based on terminal capabilities

**Usage in Glow:**

*   Used by lipgloss to adapt colors to terminal capabilities
*   Enables proper styling across different terminal emulators
*   Handles dark/light mode detection for automatic theme adjustment

**Sources:** [go.mod25](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L25-L25)

### reflow (v0.3.0)

Text wrapping and formatting utilities:

*   Word wrapping with configurable width
*   Unicode-aware text truncation
*   Indentation and padding
*   ANSI sequence preservation during wrapping

**Usage in Glow:**

*   Glamour uses reflow internally for markdown text flow
*   Ensures markdown content fits within terminal width
*   Handles code block and list indentation

**Sources:** [go.mod23](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L23-L23)

### go-runewidth (v0.0.19)

East Asian width character handling:

*   Correct width calculation for CJK characters
*   Emoji width detection
*   Combining character support
*   Terminal alignment for multi-width characters

**Usage in Glow:**

*   Ensures proper text alignment in the TUI
*   Critical for correct rendering of markdown tables
*   Used by lipgloss for layout calculations

**Sources:** [go.mod18](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L18-L18)

### golang.org/x/term (v0.38.0)

Low-level terminal control:

*   Raw mode management
*   Terminal size detection
*   Password input (no echo)
*   Platform-specific terminal operations

**Sources:** [go.mod30](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L30-L30)

### golang.org/x/text (v0.32.0)

Unicode and internationalization support:

*   Character encoding conversion
*   Unicode normalization
*   Locale-aware text processing
*   Language detection

**Sources:** [go.mod31](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L31-L31)

Utility Libraries

Additional libraries that provide specialized functionality.

### fuzzy (v0.1.1)

Fuzzy string matching for search functionality:

*   Subsequence matching algorithm
*   Score-based result ranking
*   Case-insensitive matching

**Usage in Glow:**

*   Powers the stash view's filter mode
*   Allows users to quickly find files by typing partial names
*   The `stashModel` uses `fuzzy.Find()` for file filtering

**Sources:** [go.mod26](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L26-L26)

### clipboard (v0.1.4)

Cross-platform clipboard access:

*   Copy text to system clipboard
*   Read clipboard contents
*   macOS/Windows/Linux support

**Usage in Glow:**

*   Enables copying file paths and content from the TUI
*   Keyboard shortcut integration in the pager view

**Sources:** [go.mod8](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L8-L8)

### go-humanize (v1.0.1)

Human-readable formatting:

*   File size formatting (KB, MB, GB)
*   Time duration formatting
*   Number formatting with separators

**Usage in Glow:**

*   Displays file sizes in the stash view
*   Formats timestamps for last modified dates

**Sources:** [go.mod16](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L16-L16)

### env (v11.3.1)

Environment variable parsing into Go structs:

*   Type-safe environment variable loading
*   Default value support
*   Required variable validation

**Usage in Glow:**

*   Parses environment variables into configuration structs
*   Validates environment-based configuration

**Sources:** [go.mod9](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L9-L9)

### mango-cobra (v1.3.0)

Man page generation for cobra commands:

*   Converts cobra command structure to roff format
*   Generates Unix man pages
*   Integrates with documentation systems

**Usage in Glow:**

*   The `manCmd` subcommand uses this to generate man pages
*   Enables `glow man` to output documentation in man page format

**Sources:** [go.mod22](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L22-L22)

### roff (v0.1.0)

Low-level roff format generation library used by mango-cobra.

Dependency Version Management
-----------------------------

The [go.mod3-5](https://github.com/charmbracelet/glow/blob/752de97c/go.mod#L3-L5)
 file specifies Go 1.24.0 as the minimum required version with toolchain 1.24.1:

    go 1.24.0
    toolchain go1.24.1
    

Version selection follows these principles:

*   **Direct dependencies** are pinned to specific minor versions (e.g., `v1.3.10`)
*   **Semantic versioning** is enforced through Go modules
*   **Breaking changes** are tracked via major version increments (e.g., `v2` module path)
*   **Security updates** are applied via dependabot (see [Code Quality and Testing](https://deepwiki.com/charmbracelet/glow/4.2-code-quality-and-testing)
    )

For information on how dependencies are tested and validated during CI/CD, see [Code Quality and Testing](https://deepwiki.com/charmbracelet/glow/4.2-code-quality-and-testing)
.
