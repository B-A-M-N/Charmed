This page details how to set up Soft Serve as a system service for automatic startup and background operation on Linux systems. The focus is primarily on systemd-based distributions, which covers most modern Linux environments.

For other installation options that don't involve system services, see [Installation Methods](https://deepwiki.com/charmbracelet/soft-serve/2.1-installation-methods)
. For detailed configuration options after installation, see [Configuration Options](https://deepwiki.com/charmbracelet/soft-serve/2.2-configuration-options)

### What Package Installation Does

The package installation process automatically:

1.  Creates a dedicated system user and group named `soft-serve`
2.  Sets up the data directory at `/var/lib/soft-serve`
3.  Installs and enables the systemd service
4.  Configures service security hardening

These steps are handled by the package's post-installation scripts.

### Environment Variables Configuration

When running as a system service, the most common way to configure Soft Serve is by adding environment variables to `/etc/soft-serve.conf`:

    # Example /etc/soft-serve.conf
    SOFT_SERVE_SSH_LISTEN_ADDR=:2222
    SOFT_SERVE_HTTP_LISTEN_ADDR=:8080
    SOFT_SERVE_INITIAL_ADMIN_KEYS='ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...'
    

These settings override any values in the configuration file at `/var/lib/soft-serve/config.yaml`.

Sources: [.nfpm/soft-serve.conf](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/.nfpm/soft-serve.conf)

### Environment to Configuration Mapping

| Environment Variable | Config Property | Description |
| --- | --- | --- |
| `SOFT_SERVE_NAME` | `Config.Name` | Server name displayed in UI |
| `SOFT_SERVE_DATA_PATH` | `Config.DataPath` | Path to data directory |
| `SOFT_SERVE_SSH_LISTEN_ADDR` | `Config.SSH.ListenAddr` | SSH server listen address |
| `SOFT_SERVE_SSH_PUBLIC_URL` | `Config.SSH.PublicURL` | Public SSH URL |
| `SOFT_SERVE_HTTP_LISTEN_ADDR` | `Config.HTTP.ListenAddr` | HTTP server listen address |
| `SOFT_SERVE_HTTP_PUBLIC_URL` | `Config.HTTP.PublicURL` | Public HTTP URL |
| `SOFT_SERVE_GIT_LISTEN_ADDR` | `Config.Git.ListenAddr` | Git daemon listen address |
| `SOFT_SERVE_DB_DRIVER` | `Config.DB.Driver` | Database driver (sqlite/postgres) |
| `SOFT_SERVE_DB_DATA_SOURCE` | `Config.DB.DataSource` | Database connection string |
| `SOFT_SERVE_INITIAL_ADMIN_KEYS` | `Config.InitialAdminKeys` | Initial admin SSH keys |

Security Hardening
------------------

The systemd service configuration includes extensive security measures:

| Category | Directives | Purpose |
| --- | --- | --- |
| Filesystem access | ReadWritePaths, ProtectSystem, ProtectHome | Restrict file access |
| Resource limits | LimitNOFILE | Prevent resource exhaustion |
| Isolation | PrivateUsers, PrivateTmp, PrivateDevices | Isolate from system |
| System protection | ProtectKernel\* directives | Prevent system modification |
| Capability restrictions | CapabilityBoundingSet | Restrict privileged operations |
| System call filtering | SystemCallFilter | Limit available system calls |

Sources: [.nfpm/soft-serve.service18-45](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/.nfpm/soft-serve.service#L18-L45)

Advanced Configuration
----------------------

### Changing Default Ports

To change the default ports in the system service:

    # In /etc/soft-serve.conf
    SOFT_SERVE_SSH_LISTEN_ADDR=:2222
    SOFT_SERVE_HTTP_LISTEN_ADDR=:8080
    SOFT_SERVE_GIT_LISTEN_ADDR=:9419
    

### TLS Configuration

To enable HTTPS for the system service:

    # In /etc/soft-serve.conf
    SOFT_SERVE_HTTP_TLS_KEY_PATH=/path/to/key.pem
    SOFT_SERVE_HTTP_TLS_CERT_PATH=/path/to/cert.pem
    SOFT_SERVE_HTTP_PUBLIC_URL=https://your-domain.com:23232
    

Sources: [pkg/config/config.go65-80](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L65-L80)

### Database Configuration

To use PostgreSQL instead of SQLite in the system service:

    # In /etc/soft-serve.conf
    SOFT_SERVE_DB_DRIVER=postgres
    SOFT_SERVE_DB_DATA_SOURCE=postgres://user:password@localhost:5432/soft_serve
    

Sources: [README.md248-274](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L248-L274)

Troubleshooting
---------------

### Common Issues

| Issue | Possible Solution |
| --- | --- |
| Service fails to start | Check logs with `journalctl -u soft-serve` |
| Permission errors | Ensure `/var/lib/soft-serve` is owned by the `soft-serve` user |
| Network connection issues | Check firewall settings for ports 23231 (SSH), 23232 (HTTP), 9418 (Git) |
| SSH authentication fails | Verify that admin keys are correctly configured |
| Changes to config.yaml not taking effect | Environment variables may be overriding settings |
