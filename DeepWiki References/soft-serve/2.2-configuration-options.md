This page documents the configuration options available in Soft Serve and how to customize your Soft Serve instance. Configuration allows you to control server behavior, network settings, authentication, and various other aspects of the system.

For information on installing Soft Serve, see [Installation Methods](https://deepwiki.com/charmbracelet/soft-serve/2.1-installation-methods)
. For setting up Soft Serve as a system service, see [System Service Setup](https://deepwiki.com/charmbracelet/soft-serve/2.3-system-service-setup)
.

Configuration Overview
----------------------

Soft Serve can be configured in three ways:

1.  **Configuration file**: By default, a `config.yaml` file is created in the data directory
2.  **Environment variables**: All settings can be overridden using environment variables
3.  **Default values**: Built-in defaults are used when no custom configuration is provided

Environment Variables
---------------------

All configuration options can be set via environment variables with the prefix `SOFT_SERVE_`. Nested configuration options use underscores to separate levels:

| Config Path | Environment Variable |
| --- | --- |
| name | SOFT\_SERVE\_NAME |
| ssh.listen\_addr | SOFT\_SERVE\_SSH\_LISTEN\_ADDR |
| git.max\_connections | SOFT\_SERVE\_GIT\_MAX\_CONNECTIONS |
| http.tls\_cert\_path | SOFT\_SERVE\_HTTP\_TLS\_CERT\_PATH |

Sources: [README.md238-246](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L238-L246)
 [pkg/config/config.go166-211](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L166-L211)

### Data Path

The data directory location can be configured with the `SOFT_SERVE_DATA_PATH` environment variable. This directory contains:

*   Configuration file
*   Repositories
*   SSH keys
*   Database

By default, if not specified, Soft Serve uses `./data` as the data directory.

Server Core Configuration
-------------------------

### Server Name

The `name` setting defines the name of your Soft Serve instance that will be displayed in the UI.

Environment variable: `SOFT_SERVE_NAME`

### Logging Configuration

The `log` section controls how log messages are formatted and stored:

Environment variables:

*   `SOFT_SERVE_LOG_FORMAT`
*   `SOFT_SERVE_LOG_TIME_FORMAT`
*   `SOFT_SERVE_LOG_PATH`

Sources: [pkg/config/config.go92-104](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L92-L104)

### Initial Admin Keys

The `initial_admin_keys` setting is a list of SSH public keys that will be granted admin privileges when the server starts:

You can also set this via the `SOFT_SERVE_INITIAL_ADMIN_KEYS` environment variable, with keys separated by newlines.

This is particularly important when first setting up a server, as these keys will be used to create the initial admin user.

Debug Configuration
-------------------

Soft Serve includes several debug-related environment variables:

*   `SOFT_SERVE_DEBUG`: Enable debug logging
*   `SOFT_SERVE_VERBOSE`: Enable verbose logging (only works when debug is enabled)
*   `SOFT_SERVE_NO_COLOR`: Disable colored output
