CLI Tool Structure
==================

Relevant source files

*   [.gitignore](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/.gitignore)
    
*   [cmd/soft/main.go](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/cmd/soft/main.go)
    

This page documents the structure and functionality of the Soft Serve command-line interface (CLI) tool. The CLI tool serves as the primary entry point for users to interact with Soft Serve, providing commands for running the server, browsing repositories, and performing administrative tasks. For information about the server components that are started by the CLI, see [Server Components](https://deepwiki.com/charmbracelet/soft-serve/3.2-server-components)
.

Overview
--------

The Soft Serve CLI tool is built using the [Cobra](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/Cobra)
 command-line framework and provides a hierarchical structure of commands. The main executable is named `soft`, and it exposes various subcommands for different functionalities.

_CLI Command Structure_

Sources: [cmd/soft/main.go39-79](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/cmd/soft/main.go#L39-L79)

Root Command
------------

The root command (`soft`) serves as the entry point for all functionality. When executed without any subcommands, it defaults to running the `browse` command, which connects to a Soft Serve server.

The root command is defined in the main package and includes version information that is set during build time via ldflags:

Subcommands
-----------

The Soft Serve CLI tool includes several key subcommands, each responsible for a specific aspect of the system:

| Command | Description | Package Path |
| --- | --- | --- |
| `serve` | Starts the Soft Serve server | `cmd/soft/serve` |
| `browse` | Connects to a Soft Serve server (default) | `cmd/soft/browse` |
| `hook` | Git hook commands | `cmd/soft/hook` |
| `admin` | Administrative commands | `cmd/soft/admin` |
| `man` | Generates man pages (hidden) | Defined in main |

Each subcommand is defined in its own package and imported into the main package, where they are added to the root command.

Command Implementation Details
------------------------------

Each subcommand is implemented in its own package, with its functionality defined through Cobra commands:

*   `serve`: Starts the Soft Serve server with configurable options
*   `browse`: Connects to a running Soft Serve server
*   `hook`: Handles Git hook operations
*   `admin`: Provides administrative commands for managing repositories and users
*   `man`: Generates man pages for the CLI documentation
