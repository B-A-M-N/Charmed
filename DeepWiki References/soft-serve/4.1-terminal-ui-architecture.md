Overview
--------

The Terminal UI (TUI) system in Soft Serve provides an interactive interface for users to browse repositories via SSH connections. This document explains the architecture of the TUI system, focusing on how different components work together to create a responsive and informative user experience. For information about the command-line interface, see [Command Line Interface](https://deepwiki.com/charmbracelet/soft-serve/4.3-command-line-interface)
.

The TUI is built using [Bubble Tea](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/Bubble%20Tea)
 a Go framework that follows the Model-View-Update (MVU) pattern. This architecture allows for a declarative approach to UI development, where state transitions drive UI updates.

Component Architecture
----------------------

The TUI consists of a hierarchy of components, with each component responsible for a specific part of the interface.

Model-View-Update Pattern
-------------------------

The Terminal UI follows the MVU pattern where:

1.  **Model**: Represents the state of the application
2.  **View**: Renders the model as a terminal UI
3.  **Update**: Handles messages to produce a new model and optional commands

Key Bindings
------------

The TUI provides key bindings for navigation and interaction:

| Key | Action | Context |
| --- | --- | --- |
| ESC | Back to previous view | Global |
| Tab | Switch tab | Repository Page |
| Enter | Select item | Selection, Lists |
| Up/Down | Navigate lists | Selection, Lists |
| PgUp/PgDn | Page up/down | Lists, Viewport |
| Home/End | Go to top/bottom | Viewport |
| c   | Copy (hash, URL, etc.) | Repository Page |
| /   | Filter | Selection Page |
| q   | Quit | Global |
| ?   | Toggle help | Global |

Key Technical Insights
----------------------

1.  **Component Composition**: The TUI makes extensive use of component composition, with each component responsible for a specific part of the UI and delegating to subcomponents.
    
2.  **Message Passing**: Communication between components happens through messages, which are passed up and down the component hierarchy.
    
3.  **Viewport Management**: The TUI uses viewport components to handle scrolling for content that exceeds the terminal size.
    
4.  **Styling**: The TUI uses LipGloss for styling components with consistent colors, margins, and padding.
    
5.  **Syntax Highlighting**: Code and diff views use syntax highlighting provided by Glamour and Chroma.

Conclusion
----------

The Terminal UI architecture in Soft Serve provides a responsive and informative interface for browsing git repositories via SSH. The Model-View-Update pattern, combined with component composition and message passing, creates a maintainable and extensible codebase.

The TUI system is designed to work seamlessly with the other components of Soft Serve, such as the git backend and SSH server, to provide a complete self-hostable git server solution.
