The SSH Server is a core component of Soft Serve that enables Git operations over SSH, provides a terminal-based user interface, and allows administrative control of the server through SSH commands. This document explains how the SSH server is implemented, configured, and its role within the Soft Serve architecture.

For information about the Git daemon that handles the git:// protocol, see [Git Daemon](https://deepwiki.com/charmbracelet/soft-serve/3.2.2-git-daemon)
.

Overview
--------

The SSH Server in Soft Serve provides several functions:

1.  Git protocol operations over SSH (git clone, push, pull)
2.  Interactive Terminal UI for browsing repositories
3.  Command-line interface for administrative tasks
4.  Authentication and authorization for users
5.  Git LFS SSH backend (when enabled)

Configuration
-------------

The SSH Server is highly configurable through a YAML configuration file and environment variables. The default configuration starts the SSH server on port 23231.

### Configuration Options

| Option | Description | Default Value | Environment Variable |
| --- | --- | --- | --- |
| Enabled | Toggles the SSH server on/off | `true` | `SOFT_SERVE_SSH_ENABLED` |
| ListenAddr | The address on which the SSH server will listen | `:23231` | `SOFT_SERVE_SSH_LISTEN_ADDR` |
| PublicURL | The public URL of the SSH server (used for clone URLs) | `ssh://localhost:23231` | `SOFT_SERVE_SSH_PUBLIC_URL` |
| KeyPath | Path to the SSH server's host key | `ssh/soft_serve_host_ed25519` | `SOFT_SERVE_SSH_KEY_PATH` |
| ClientKeyPath | Path to the SSH server's client key (for Git requests to SSH remotes) | `ssh/soft_serve_client_ed25519` | `SOFT_SERVE_SSH_CLIENT_KEY_PATH` |
| MaxTimeout | Maximum duration for a connection in seconds (0 = no timeout) | `0` | `SOFT_SERVE_SSH_MAX_TIMEOUT` |
| IdleTimeout | Idle timeout in seconds before a connection is closed | `600` (10 minutes) | `SOFT_SERVE_SSH_IDLE_TIMEOUT` |

Authentication
--------------

Authentication in the SSH Server is handled through SSH public keys. Users prove their identity by using their private SSH key to connect to the server.

### Initial Admin Setup

When Soft Serve is first set up, admin keys can be provided through the `SOFT_SERVE_INITIAL_ADMIN_KEYS` environment variable. These keys are granted full administrative privileges:

Sources: [README.md122-129](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L122-L129)
 [pkg/config/config.go159-161](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L159-L161)

### Public Key Authentication

Soft Serve authenticates users by their SSH public keys. Each user can have multiple public keys associated with their account, but a public key can only be associated with one user. This ensures that SSH connections are properly mapped to user accounts.

### Access Control Levels

Soft Serve implements four access levels that determine what actions a user can perform:

| Access Level | Description |
| --- | --- |
| no-access | Denies access to all repositories |
| read-only | Can read public repositories |
| read-write | Has full control of repositories |
| admin-access | Has full control of the server, including user and repository management |

Security Considerations
-----------------------

### Host Keys

The SSH Server generates and uses ED25519 SSH host keys by default. These keys are stored in the data directory and are used to identify the server to clients. Clients will be prompted to verify the host key fingerprint on first connection.

### Client Keys

The server also maintains a client key pair that it uses when acting as a client to other SSH servers, such as when mirroring repositories from remote SSH sources.

### Key Limitations

Soft Serve currently has a limitation regarding RSA keys: only SHA-1 RSA keys are supported, not newer RSA keys. This is due to limitations in Go's `x/crypto/ssh` package. Users with newer RSA keys will need to use a different key type (like Ed25519) or generate an older SHA-1 RSA key.

Sources: [README.md754-772](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L754-L772)

Conclusion
----------

The SSH Server is a central component of Soft Serve, providing secure access to Git repositories and administrative functionality. Its configuration options allow for flexibility in deployment, while its authentication and authorization mechanisms ensure proper access control.

By using the SSH protocol, Soft Serve provides a familiar interface for Git users while adding powerful features like an interactive terminal UI and administrative commands.
