Binary Download Installation
----------------------------

You can download pre-compiled binaries directly from the GitHub releases page. This method is useful when package managers are not available or when you need a specific version.

To download a binary:

1.  Visit the [releases page](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/releases%20page)
    
2.  Choose the appropriate package for your system
3.  Download and install according to your platform's standards:
    *   For raw binaries: extract and place in a directory in your PATH
    *   For package files (.deb, .rpm, .apk): use your system's package manager

Post-Installation Setup
-----------------------

After installing Soft Serve through any of the methods above, you'll need to set up and configure your server.

### Data Directory Structure

Soft Serve creates a data directory to store its data, including repositories, SSH keys, and the database. By default, this directory is named `data` in the current working directory, but you can customize it using the `SOFT_SERVE_DATA_PATH` environment variable.

### First-Time Configuration

When you run Soft Serve for the first time, make sure the `SOFT_SERVE_INITIAL_ADMIN_KEYS` environment variable is set to your SSH public key. This will create an admin user with full privileges.

After the first run, a `config.yaml` file will be created in your data directory with default settings that you can customize.

Configuration Sources and Precedence
------------------------------------

Soft Serve loads its configuration from multiple sources with a specific precedence order:

The precedence order (highest to lowest):

1.  Environment variables (SOFT\_SERVE\_\*)
2.  Config file (specified by SOFT\_SERVE\_CONFIG\_LOCATION or in data directory)
3.  Internal defaults

Sources: [pkg/config/config.go226-323](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L226-L323)

Summary of Installation Methods
-------------------------------

| Method | Platforms | Benefits | Considerations |
| --- | --- | --- | --- |
| Package Managers | Platform-specific | Easy installation, handles dependencies | May not have latest version |
| Binary Download | All | Latest releases, specific versions | Manual installation, no auto-updates |
| Go Install | All (with Go installed) | Simple for Go developers | Requires Go toolchain |
| Docker | All (with Docker) | Isolated environment, easy deployment | Container overhead, volume mapping needed |

Each installation method results in the same `soft` binary with identical functionality, so choose the method that best fits your environment and workflow.

Sources: [README.md54-102](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L54-L102)
