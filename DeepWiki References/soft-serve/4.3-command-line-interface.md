This page documents the command-line interface for Soft Serve, a self-hostable Git server. It covers both the `soft` command-line tool as well as the SSH command interface used to interact with a running Soft Serve instance. For information about the Terminal User Interface, see [Terminal UI Architecture](https://deepwiki.com/charmbracelet/soft-serve/4.1-terminal-ui-architecture)
.

CLI Structure
-------------

Soft Serve provides a command-line tool named `soft` with several subcommands for different functionality. The main command structure is designed to be simple and intuitive.

Sources: [cmd/soft/main.go39-65](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/cmd/soft/main.go#L39-L65)
 [cmd/soft/main.go73-79](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/cmd/soft/main.go#L73-L79)

### Command Overview

| Command | Description |
| --- | --- |
| `soft` | When run without arguments, defaults to `soft browse` for the current directory |
| `soft serve` | Starts the Soft Serve Git server |
| `soft browse [directory]` | Opens the Terminal UI to browse repositories |
| `soft hook` | Manages Git hooks functionality |
| `soft admin` | Performs administrative operations |

The primary commands you'll use most frequently are `serve` (to run the server) and `browse` (to browse repositories locally).

Server Command (`soft serve`)
-----------------------------

The `serve` command starts the Soft Serve Git server. This command initializes all server components, including SSH, Git protocol, HTTP, and Stats servers.

When you run `soft serve` for the first time, it creates a `data` directory that stores repositories, SSH keys, and the database. It also generates a default configuration file (`config.yaml`) in this directory.

### Environment Variables

All configuration options can be set using environment variables prefixed with `SOFT_SERVE_`. These take precedence over config file settings. Key variables include:

| Environment Variable | Description |
| --- | --- |
| `SOFT_SERVE_DATA_PATH` | Path where Soft Serve stores its data |
| `SOFT_SERVE_CONFIG_LOCATION` | Path to custom config file |
| `SOFT_SERVE_NAME` | Server name displayed in the UI |
| `SOFT_SERVE_INITIAL_ADMIN_KEYS` | Initial admin SSH public keys |
| `SOFT_SERVE_SSH_LISTEN_ADDR` | SSH server listen address |
| `SOFT_SERVE_HTTP_LISTEN_ADDR` | HTTP server listen address |
| `SOFT_SERVE_GIT_MAX_CONNECTIONS` | Max Git daemon connections |
| `SOFT_SERVE_DEBUG` | Enable debug logging |
| `SOFT_SERVE_VERBOSE` | Enable verbose logging (requires debug mode) |

For a complete list of environment variables, see the Config struct in [pkg/config/config.go130-161](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L130-L161)

Configuration Loading Sequence
------------------------------

The configuration is loaded in this sequence:

1.  Default configuration is created
2.  If a config file exists, it is loaded and merged with defaults
3.  Environment variables are processed and override file settings
4.  Configuration validation is performed
5.  All paths are converted to absolute paths relative to the data directory

Summary
-------

The Soft Serve command-line interface consists of:

1.  The `soft` command with subcommands for running the server and browsing repositories
2.  SSH commands for managing repositories, users, and server settings
3.  Configuration through both environment variables and a YAML config file

This CLI design allows for both interactive use and scripted automation while keeping the interface simple and consistent.
