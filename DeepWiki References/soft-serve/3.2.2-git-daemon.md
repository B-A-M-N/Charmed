The Git Daemon is a core server component in Soft Serve that enables Git clients to communicate with the server using the native Git protocol (`git://`). This component provides read-only access to Git repositories over port 9418 without requiring SSH authentication, making it ideal for public repository access.

For information about the SSH-based Git access, see [SSH Server](https://deepwiki.com/charmbracelet/soft-serve/3.2.1-ssh-server)
.

Overview
--------

The Git Daemon implements the Git transport protocol, handling `git-upload-pack` and `git-upload-archive` requests. It provides an efficient and lightweight mechanism for users to clone and fetch from Soft Serve repositories without SSH credentials.

Sources: [pkg/daemon/daemon.go44-56](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/daemon/daemon.go#L44-L56)

Architecture
------------

### Key Components

1.  **TCP Listener**: Accepts connections on the configured port (default: 9418)
2.  **Connection Manager**: Tracks and limits active connections
3.  **Request Handler**: Parses and processes Git protocol requests
4.  **Backend Integration**: Verifies repository access permissions

The daemon integrates with Soft Serve's backend system for authentication, authorization, and repository access.

Supported Git Services
----------------------

The Git Daemon supports two primary Git services:

| Service | Description | Metric Counter |
| --- | --- | --- |
| `git-upload-pack` | Handles `git fetch` and `git clone` operations | `soft_serve_git_git_upload_pack_total` |
| `git-upload-archive` | Handles `git archive` operations | `soft_serve_git_git_upload_archive_total` |

Both services are implemented through the Git protocol and are tracked with Prometheus metrics.

Connection Management
---------------------

The Git Daemon implements several mechanisms to manage client connections:

1.  **Max Connections**: Limits the total number of simultaneous connections
2.  **Idle Timeout**: Closes inactive connections after a configurable period
3.  **Max Timeout**: Sets an absolute maximum time for any connection

When a limit is reached, the daemon closes the connection and returns an appropriate error message to the client.

Sources: [pkg/daemon/daemon.go97-134](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/daemon/daemon.go#L97-L134)

Security and Access Control
---------------------------

The Git Daemon provides read-only access to repositories, with the following security considerations:

1.  **Access Control**: Repository access is verified through the backend
2.  **Path Validation**: Ensures repository paths are properly sanitized
3.  **Anonymous Access**: Only allows access if keyless access is enabled

For enhanced security, repositories are only accessible if they have been properly registered in the Soft Serve backend.

Sources: [pkg/daemon/daemon.go242-269](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/daemon/daemon.go#L242-L269)

Configuration Options
---------------------

The Git Daemon can be configured through various options in the Soft Serve configuration:

| Option | Environment Variable | Description | Default |
| --- | --- | --- | --- |
| Enabled | `SOFT_SERVE_GIT_ENABLED` | Enable/disable the Git daemon | `true` |
| Listen Address | `SOFT_SERVE_GIT_LISTEN_ADDR` | TCP address to listen on | `:9418` |
| Max Connections | `SOFT_SERVE_GIT_MAX_CONNECTIONS` | Maximum number of concurrent connections | `32` |
| Idle Timeout | `SOFT_SERVE_GIT_IDLE_TIMEOUT` | Timeout for idle connections (seconds) | `60` |
| Max Timeout | `SOFT_SERVE_GIT_MAX_TIMEOUT` | Maximum connection duration (seconds) | `600` |

Lifecycle Management
--------------------

The Git Daemon provides methods for managing its lifecycle:

1.  **ListenAndServe()**: Starts the daemon and begins accepting connections
2.  **Shutdown(ctx)**: Gracefully shuts down the daemon, waiting for active connections to complete
3.  **Close()**: Forcefully closes the daemon and all active connections

The daemon integrates with the main Soft Serve server's lifecycle management for coordinated startup and shutdown.

Git Protocol Specifics
----------------------

The Git protocol used by the daemon follows this format:

    <service> <repository-path>\0host=<hostname>\0[<extra-parameters>]
    

Where:

*   `<service>` is either `git-upload-pack` or `git-upload-archive`
*   `<repository-path>` is the path to the repository
*   `<hostname>` is the server hostname
*   `<extra-parameters>` may include protocol version and other options

The daemon parses this request format and sets up appropriate environment variables for Git services.

Error Handling
--------------

The Git Daemon implements comprehensive error handling for various scenarios:

| Error Type | Error Message | Cause |
| --- | --- | --- |
| ErrTimeout | `fatal: timeout` | Connection idle or maximum timeout exceeded |
| ErrInvalidRequest | `fatal: invalid request` | Malformed Git protocol request |
| ErrInvalidRepo | `fatal: invalid repository` | Repository doesn't exist or path is invalid |
| ErrNotAuthed | `fatal: not authorized` | User doesn't have permission to access repository |
| ErrMaxConnections | `fatal: too many connections` | Maximum connections limit reached |

Performance Considerations
--------------------------

The Git Daemon is designed for performance with the following optimizations:

1.  **Asynchronous Processing**: Handles each client connection in a separate goroutine
2.  **Efficient Protocol**: Uses the native Git protocol for optimal transfer performance
3.  **Connection Limits**: Prevents server overload with configurable connection limits
4.  **Metrics**: Provides Prometheus metrics for monitoring performance
