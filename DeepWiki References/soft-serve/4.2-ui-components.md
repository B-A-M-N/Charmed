This page documents the user interface components in Soft Serve, their structure, interactions, and how they work together to create the terminal-based UI. For information about the overall UI architecture, see [Terminal UI Architecture](https://deepwiki.com/charmbracelet/soft-serve/4.1-terminal-ui-architecture)
.

Overview
--------

Soft Serve's user interface is built using the [Bubble Tea](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/Bubble%20Tea)
 framework following the Model-View-Update (MVU) pattern. UI components in Soft Serve are organized hierarchically, with reusable components that can be composed to create complex interfaces.

The UI components are primarily located in two directories:

*   `pkg/ui/components/`: Reusable UI primitives like code viewers, selectors, and status bars
*   `pkg/ui/pages/`: Higher-level views like repository and selection pages

Key UI Components
-----------------

### Repository Page

The Repository Page is the central component for viewing a Git repository. It manages multiple tab components and a status bar.

Sources: [pkg/ui/pages/repo/repo.go49-85](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/repo.go#L49-L85)
 [pkg/ui/pages/repo/repo.go305-323](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/repo.go#L305-L323)

#### State Management

The Repository component manages its state through the following types:

    state int
    const (
        loadingState state = iota
        readyState
    )
    

The Repository Page cycles through these states during initialization and when switching repositories.

Sources: [pkg/ui/pages/repo/repo.go21-26](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/repo.go#L21-L26)

### Code Component

The Code component displays syntax-highlighted code and supports various formatting options:

Sources: [pkg/ui/components/code/code.go22-257](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/components/code/code.go#L22-L257)

Key features:

*   Syntax highlighting for various languages
*   Markdown rendering via Glamour
*   Line numbers
*   Side notes
*   Viewport navigation

### File Item Delegate

The `FileItemDelegate` handles the rendering of files in a file browser:

Sources: [pkg/ui/pages/repo/filesitem.go67-156](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/filesitem.go#L67-L156)

The delegate handles:

*   Rendering file names, sizes, and modes
*   Styling based on file type (directory vs. file)
*   Copy operations for file names

### Status Bar

The Status Bar component displays contextual information about the current view:

Sources: [pkg/ui/components/statusbar/statusbar.go10-93](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/components/statusbar/statusbar.go#L10-L93)

The Status Bar adapts its content based on the active tab and context.

### Commit Log View

The Log component displays commit history and diffs:

Sources: [pkg/ui/pages/repo/log.go25-31](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/log.go#L25-L31)
 [pkg/ui/pages/repo/log.go183-198](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/log.go#L183-L198)

The Log component has three main states:

*   Loading: Shows a spinner while fetching commits
*   Commits: Displays a list of commits
*   Diff: Shows the selected commit's diff

### Stash View

The Stash component displays stashed changes:

Sources: [pkg/ui/pages/repo/stash.go18-23](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/stash.go#L18-L23)

Similar to the Log component, the Stash view transitions between loading, list, and patch views.

Component Communication
-----------------------

Components in Soft Serve communicate through messages passed up and down the component hierarchy. The diagram below illustrates this flow:

Sources: [pkg/ui/pages/repo/repo.go150-291](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/repo.go#L150-L291)

### Message Types

The UI uses various message types to communicate state changes:

| Message Type | Purpose |
| --- | --- |
| `RepoMsg` | Set the current repository |
| `RefMsg` | Set the current Git reference |
| `LogItemsMsg` | Update the commit log items |
| `FileItemsMsg` | Update the file browser items |
| `CopyMsg` | Indicate text has been copied |
| `GoBackMsg` | Navigate back to the previous view |
| `EmptyRepoMsg` | Indicate the repository is empty |

Sources: [pkg/ui/pages/repo/repo.go28-47](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/ui/pages/repo/repo.go#L28-L47)

UI Interaction Patterns
-----------------------

### Navigation

Navigation in Soft Serve follows these patterns:

1.  **Tab Navigation**: Switch between tabs in the repository view
2.  **Back Navigation**: Return to previous views
3.  **Selection Navigation**: Navigate through lists of items
4.  **Viewport Navigation**: Scroll through content

The UI supports both keyboard and mouse interactions:

Conclusion
----------

The UI Components in Soft Serve form a cohesive system built on the Bubble Tea framework. They follow consistent patterns for rendering, state management, and user interaction. Components are designed to be reusable and composable, allowing for a flexible and maintainable UI architecture.

The main components (Repository Page, Code Component, File Browser, etc.) work together to provide a seamless terminal-based Git repository browsing experience, with features comparable to graphical Git clients despite running entirely in the terminal.
