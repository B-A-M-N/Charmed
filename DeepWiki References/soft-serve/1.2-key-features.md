Key Features

This document provides an overview of the core features and capabilities of Soft Serve, a self-hostable Git server for the command line. It focuses on what Soft Serve can do, rather than implementation details or setup instructions. For installation and configuration information, see [Installation and Configuration](https://deepwiki.com/charmbracelet/soft-serve/2-installation-and-configuration)
.

1\. Multiprotocol Support
-------------------------

Soft Serve stands out by offering multiple ways to interact with Git repositories:

Sources: [README.md19-26](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L19-L26)
 [pkg/config/config.go19-80](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L19-L80)

### SSH Protocol

*   Interactive TUI for browsing repositories
*   Command-line interface for server management
*   Git operations (clone, pull, push)
*   Public key authentication
*   Support for viewing files with syntax highlighting

### Git Protocol

*   Native Git protocol support via the Git daemon
*   Lightweight and efficient for read operations
*   Anonymous access for public repositories

### HTTP Protocol

*   Standard Git over HTTP/HTTPS support
*   Git LFS (Large File Storage) support
*   Authentication through access tokens
*   Web-based access to repositories

2\. Terminal User Interface (TUI)
---------------------------------

One of Soft Serve's standout features is its rich terminal user interface accessible via SSH.

Sources: [README.md669-693](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L669-L693)

### TUI Capabilities

*   Repository browsing and navigation
*   File viewing with syntax highlighting and line numbers
*   Commit history browsing
*   Branch and tag exploration
*   Diff viewing for changes
*   Server and repository management commands

### TUI Accessibility

*   Access by connecting via SSH: `ssh <server-address>`
*   Direct repository access: `ssh <server-address> -t <repo-name>`
*   File viewing commands: `ssh <server-address> repo blob <repo> <file-path> -c -l`
*   Directory tree viewing: `ssh <server-address> repo tree <repo> [path]`

3\. Repository Management
-------------------------

Soft Serve provides comprehensive repository management features accessible through SSH commands or direct Git operations.

Sources: [README.md446-604](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L446-L604)

### Repository Creation

*   Create via SSH command: `ssh <server> repo create <name>`
*   Create on-demand via Git push to a non-existent repository
*   Import from remote Git repositories: `ssh <server> repo import <name> <url>`
*   Support for nested repositories (e.g., `organization/repo`)

### Repository Configuration

*   Set repository visibility (public/private)
*   Hide repositories from listings
*   Configure repository description and project name
*   Manage default branch
*   Configure repository as a mirror

### Content Management

*   Browse repository files and directories
*   View file contents with or without syntax highlighting
*   Explore commit history
*   Examine branches and tags

4\. Access Control
------------------

Soft Serve implements a comprehensive access control system to manage user permissions and repository access.

Sources: [README.md284-400](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L284-L400)

### Authentication

*   SSH public key authentication for users
*   User access tokens for HTTP authentication
*   Anonymous access configuration for public repositories
*   Keyless access control for non-SSH protocols

### User Management

*   Add, remove, and modify users
*   Manage user SSH public keys
*   Generate and manage user access tokens
*   Set user access levels

### Repository Permissions

*   Four access levels: no-access, read-only, read-write, admin-access
*   Public and private repositories
*   Repository collaborators with specific access levels
*   Repository-specific access control

5\. Git LFS Support
-------------------

Soft Serve includes full Git Large File Storage (LFS) support, allowing efficient handling of large binary files.

Sources: [README.md276-282](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L276-L282)
 [pkg/config/config.go116-123](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L116-L123)

### LFS Features

*   Support for both HTTP and SSH LFS protocols
*   Automatic handling of large binary files
*   Integrated with Soft Serve's authentication system
*   Configurable enablement of protocols

6\. Hooks and Webhooks
----------------------

Soft Serve provides extensive support for Git hooks and webhooks to extend functionality and integrate with external systems.

Sources: [README.md695-752](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L695-L752)
 [README.md643-667](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L643-L667)

### Git Hooks

*   Support for standard Git server-side hooks (pre-receive, update, post-update, post-receive)
*   Global hooks that apply to all repositories
*   Repository-specific hooks
*   Hooks for integrating with CI/CD systems

### Webhooks

*   Repository event notifications to external services
*   Support for push, collaborator, and branch/tag creation events
*   Webhook creation and management via SSH commands
*   Delivery management and monitoring

7\. Server Configuration and Flexibility
----------------------------------------

Soft Serve offers extensive configuration options to adapt to different deployment scenarios.

Sources: [README.md136-282](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/README.md?plain=1#L136-L282)
 [pkg/config/config.go130-164](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L130-L164)
 [pkg/config/config.go334-379](https://github.com/charmbracelet/soft-serve/blob/767bdd5a/pkg/config/config.go#L334-L379)

### Configuration Options

*   Server name and branding
*   SSH, Git, HTTP, and Stats server settings
*   Database configuration (SQLite or PostgreSQL)
*   Git LFS enablement and protocol selection
*   Logging configuration
*   Initial admin keys setup
*   Custom data path location

### Configuration Methods

*   YAML configuration file
*   Environment variables
*   Default values for easy setup
*   Runtime configuration via SSH commands

### Deployment Flexibility

*   Standalone binary
*   Docker container
*   System service (Systemd support)
*   Package manager installation options

Summary of Features
-------------------

| Feature Category | Key Capabilities |
| --- | --- |
| Protocol Support | SSH, Git, HTTP/HTTPS |
| Terminal UI | Repository browsing, file viewing, commit history |
| Repository Management | Create, import, configure, collaborate |
| Access Control | SSH keys, tokens, permission levels, collaborators |
| Git LFS | HTTP and SSH protocols for large files |
| Hooks & Webhooks | Server-side hooks, event notifications, integrations |
| Configuration | YAML, environment variables, multiple database options |
| Deployment | Binary, Docker, system service, package managers |

Soft Serve combines these features into a cohesive, self-hostable Git server that emphasizes command-line access and a terminal-based user experience, while still providing the flexibility to use standard Git clients and protocols.
