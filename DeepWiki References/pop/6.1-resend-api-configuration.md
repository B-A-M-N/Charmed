========================

This document details how to configure and use the Resend API delivery method in Pop, a command-line email client. For information about SMTP configuration, see [SMTP Configuration](https://deepwiki.com/charmbracelet/pop/6.2-smtp-configuration)

Overview
--------

Resend is the primary email delivery method for Pop. It provides a modern email API that allows sending emails via HTTP requests. Pop integrates with Resend through the official Go client library.

Sources: [main.go23-25](https://github.com/charmbracelet/pop/blob/cdaa7255/main.go#L23-L25)

Obtaining a Resend API Key
--------------------------

To use Pop with Resend, you need to obtain an API key:

1.  Create an account at [Resend](https://resend.com/)
    
2.  Navigate to the API Keys section
3.  Generate a new API key

Configuration Methods
---------------------

There are two ways to configure the Resend API key in Pop:

| Method | Description | Example |
| --- | --- | --- |
| Environment Variable | Set using `RESEND_API_KEY` | `export RESEND_API_KEY=re_123abc` |
| Command-line Flag | Use `--resend.key` or `-r` | `pop -r re_123abc` |

Sources: [main.go23-25](https://github.com/charmbracelet/pop/blob/cdaa7255/main.go#L23-L25)

------------------------

If both Resend API key and SMTP credentials are configured, Pop will report an error and ask the user to set only one delivery method.

Using Resend with Custom or Default Domains
-------------------------------------------

When using Resend, you have two options for the sender domain:

1.  **Custom domain**: Configure your domain with Resend for better deliverability
2.  **Default domain**: Use `onboarding@resend.dev` if you don't have a custom domain set up

    export POP_FROM=onboarding@resend.dev

Troubleshooting
---------------

### Common Issues

1.  **Missing API Key**: If no delivery method is configured, Pop will display an error message:
    
        RESEND_API_KEY environment variable is required.
        You can grab one at https://resend.com/api-keys
        
    
2.  **Multiple Delivery Methods**: If both Resend and SMTP are configured, Pop will display:
    
        Unknown delivery method.
        You have set both RESEND_API_KEY and POP_SMPT_* delivery methods.
        Set only one of these environment variables.
        
    
3.  **Email Sending Failure**: In case of an error, Pop saves the email body to a temporary file and displays the file path.

Markdown to HTML Conversion
---------------------------

Resend supports both HTML and plain text emails. Pop converts your Markdown content to HTML with two modes:

| Mode | Description | Enabled By |
| --- | --- | --- |
| Standard | Safe HTML conversion | Default |
| Unsafe | Allows unsafe HTML and additional Markdown features | `--unsafe` flag or `POP_UNSAFE_HTML=true` |

The unsafe mode enables:

*   Raw HTML in Markdown
*   Strikethrough formatting
*   Tables
*   Automatic link detection
