User Interface Components
=========================

This document explains the user interface architecture of Pop, focusing on the model, key bindings, and styling system that power both the Terminal User Interface (TUI) mode. For information about using the Command Line Interface, see [Command Line Interface](https://deepwiki.com/charmbracelet/pop/4-command-line-interface)
, and for details on using the TUI as an end-user, see [Terminal User Interface](https://deepwiki.com/charmbracelet/pop/5-terminal-user-interface)
.

Component Architecture
----------------------

Pop's user interface is built on the Bubble Tea framework, which implements the Model-View-Update (MVU) pattern. The central component is the `Model` struct, which contains all the UI components, manages state transitions, and handles rendering.

Sources: [model.go52-85](https://github.com/charmbracelet/pop/blob/cdaa7255/model.go#L52-L85)

State Management
----------------

The UI state is managed through the `State` enumeration, which determines which component is currently active and how user input is processed. State transitions are triggered by key presses and are managed in the `Update` method.

State transitions handle focus management through the `blurInputs()` and `focusActiveInput()` methods, which update the visual state of UI components.

Sources: [model.go230-362](https://github.com/charmbracelet/pop/blob/cdaa7255/model.go#L230-L362)
 [model.go364-423](https://github.com/charmbracelet/pop/blob/cdaa7255/model.go#L364-L423)

Input Components
----------------

Pop uses several UI components from the Bubble Tea ecosystem:

1.  **Text Inputs** (`textinput.Model`): Used for single-line fields like From, To, Subject
2.  **Text Area** (`textarea.Model`): Used for the email body with markdown support
3.  **List** (`list.Model`): Used for displaying attachments
4.  **File Picker** (`filepicker.Model`): Used for selecting attachment files
5.  **Spinner** (`spinner.Model`): Used during email sending
6.  **Help** (`help.Model`): Displays key bindings

These components are initialized in the `NewModel` function and updated in the `Update` method.

Sources: [model.go88-212](https://github.com/charmbracelet/pop/blob/cdaa7255/model.go#L88-L212)
 [model.go425-473](https://github.com/charmbracelet/pop/blob/cdaa7255/model.go#L425-L473)

Key Binding System
------------------

Pop uses a key binding system defined in the `KeyMap` struct. Key bindings change based on the current state and are updated through the `updateKeymap()` method.

| Key Binding | Default Keys | Function | Enabled Conditions |
| --- | --- | --- | --- |
| NextInput | Tab | Move to next input | Always |
| PrevInput | Shift+Tab | Move to previous input | Always |
| Send | Enter, Ctrl+D | Send email | When hovering send button and can send |
| Attach | Enter | Attach file | When editing attachments |
| Unattach | X   | Remove attachment | When editing attachments with items |
| Back | Esc | Return from file picker | When picking file |
| Quit | Ctrl+C | Quit application | Always |

Styling System
--------------

Pop uses the Lipgloss library for UI styling. Styles are defined in `style.go` and applied based on component focus.

| Style Name | Description | Usage |
| --- | --- | --- |
| activeTextStyle | White text | Active input text |
| textStyle | Light gray text | Inactive input text |
| activeLabelStyle | Accent-colored text | Active input label |
| labelStyle | Gray text | Inactive input label |
| placeholderStyle | Dark gray text | Placeholder text |
| cursorStyle | White text | Cursor |
| sendButtonActiveStyle | Yellow text on accent background | Active send button |
| sendButtonInactiveStyle | Light gray text on dark gray background | Inactive send button |
| sendButtonStyle | Gray text on dark gray background | Default send button |

The styling system works in conjunction with the focus management methods:

*   `blurInputs()`: Applies inactive styles to all components
*   `focusActiveInput()`: Applies active styles to the current component based on state

Rendering Pipeline
------------------

The rendering pipeline is implemented in the `View` method of the `Model` struct. It handles different rendering based on the current state:

1.  File picking state: Shows the file picker
2.  Sending email state: Shows a loading spinner
3.  Normal state: Shows all input components and the send button

Attachments System
------------------

The attachments system consists of a list component to display attachments and a file picker to select them. Attachments are represented by the `attachment` type (a string alias) and rendered using the `attachmentDelegate` struct.

The attachment list and file picker are integrated into the state management system, with dedicated states for editing attachments (`editingAttachments`) and picking files (`pickingFile`).
