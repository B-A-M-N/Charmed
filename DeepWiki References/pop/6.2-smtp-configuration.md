SMTP Configuration
==================

Relevant source files

*   [LICENSE](https://github.com/charmbracelet/pop/blob/cdaa7255/LICENSE)

This page documents how to configure and use SMTP servers for sending emails with Pop. For information about using the Resend API delivery method instead, see [Resend API Configuration](https://deepwiki.com/charmbracelet/pop/6.1-resend-api-configuration)
.

Overview
--------

Pop supports sending emails through SMTP (Simple Mail Transfer Protocol) servers, allowing you to use your existing email provider's infrastructure to send emails directly from your terminal. This configuration method is particularly useful when you want to send emails from your personal or organizational email address without using third-party services.

Sources: [main.go33-50](https://github.com/charmbracelet/pop/blob/cdaa7255/main.go#L33-L50)
 [email.go66-132](https://github.com/charmbracelet/pop/blob/cdaa7255/email.go#L66-L132)

Configuration Methods
---------------------

Pop offers two methods to configure SMTP settings:

### 1\. Environment Variables

Setting environment variables is the recommended approach for persistent configuration across multiple uses of Pop.

| Variable | Description | Example |
| --- | --- | --- |
| `POP_SMTP_HOST` | SMTP server hostname | `smtp.gmail.com` |
| `POP_SMTP_PORT` | SMTP server port | `587` |
| `POP_SMTP_USERNAME` | SMTP authentication username | `user@example.com` |
| `POP_SMTP_PASSWORD` | SMTP authentication password | `yourpassword` |
| `POP_SMTP_ENCRYPTION` | Encryption type: `starttls`, `ssl`, or `none` | `starttls` |
| `POP_SMTP_INSECURE_SKIP_VERIFY` | Skip TLS certificate verification (true/false) | `false` |

### 2\. Command-Line Flags

For one-time configurations or to override environment variables, you can use command-line flags:

| Flag | Short Flag | Description |
| --- | --- | --- |
| `--smtp.host` | `-H` | SMTP server hostname |
| `--smtp.port` | `-P` | SMTP server port |
| `--smtp.username` | `-U` | SMTP authentication username |
| `--smtp.password` | `-p` | SMTP authentication password |
| `--smtp.encryption` | `-e` | Encryption type (starttls, ssl, none) |
| `--smtp.insecure` | `-i` | Skip TLS certificate verification |

Encryption and Security Settings
--------------------------------

Pop supports three encryption types for SMTP connections:

| Encryption Type | Description | Typical Port |
| --- | --- | --- |
| `starttls` (default) | Start with plaintext and upgrade to TLS | 587 |
| `ssl` | Direct SSL/TLS connection | 465 |
| `none` | No encryption (insecure) | 25  |

### TLS Configuration

The TLS configuration for the SMTP connection is defined in [email.go97-100](https://github.com/charmbracelet/pop/blob/cdaa7255/email.go#L97-L100)
:

*   A timeout of 10 seconds is set for both connection and sending
*   The `POP_SMTP_INSECURE_SKIP_VERIFY` environment variable or `--smtp.insecure` flag controls whether to skip certificate verification
    *   Setting this to `true` bypasses certificate validation, which can be useful for self-signed certificates but is less secure
    *   Default is `false`, which enforces certificate validation

Sources: [email.go85-100](https://github.com/charmbracelet/pop/blob/cdaa7255/email.go#L85-L100)

Gmail-Specific Configuration
----------------------------

Pop includes special handling for Gmail accounts:

*   If the username ends with `@gmail.com` and other settings are not specified:
    *   `smtp.gmail.com` is used as the default host
    *   `587` is used as the default port

Gmail requires special considerations:

1.  Use an App Password instead of your regular password for accounts with 2FA enabled
2.  Make sure "Less Secure App Access" is enabled for accounts without 2FA (though Google is phasing this out)

Sources: [email.go62-83](https://github.com/charmbracelet/pop/blob/cdaa7255/email.go#L62-L83)

Email Delivery Process
----------------------

When sending an email using the SMTP method, Pop performs the following steps:

1.  Creates an SMTP client with the configured settings
2.  Establishes a connection to the server
3.  Creates a new email message with the provided details
4.  Converts Markdown content to HTML (if possible)
5.  Attaches any files specified
6.  Sends the email through the SMTP connection

If the email fails to send, Pop saves the email body to a temporary file and returns an error message with the path to that file, allowing you to recover your content.

Common SMTP Server Configurations
---------------------------------

Here are configurations for popular email providers:

| Provider | Host | Port | Encryption |
| --- | --- | --- | --- |
| Gmail | smtp.gmail.com | 587 | starttls |
| Outlook/Office 365 | smtp.office365.com | 587 | starttls |
| Yahoo Mail | smtp.mail.yahoo.com | 587 | starttls |
| ProtonMail (with Bridge) | 127.0.0.1 | 1025 | starttls |
| Amazon SES (US East) | email-smtp.us-east-1.amazonaws.com | 587 | starttls |

Troubleshooting
---------------

If you encounter issues sending emails via SMTP:

1.  **Connection failures**: Verify host, port, and network connectivity
2.  **Authentication failures**: Check username and password
3.  **Encryption issues**: Try different encryption types
4.  **Certificate errors**: If using a self-signed certificate, you may need to set `POP_SMTP_INSECURE_SKIP_VERIFY=true`
5.  **Provider-specific restrictions**: Some providers have additional security measures or limits

For Gmail users:

*   Create an App Password: [https://myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
    
*   Enable "Less Secure App Access" if not using 2FA (being deprecated by Google)
