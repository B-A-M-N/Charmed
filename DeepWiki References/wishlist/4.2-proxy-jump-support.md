Purpose and Scope
-----------------

This document describes Wishlist's implementation of SSH proxy jump functionality, which allows users to connect to remote hosts through one or more intermediate jump servers. Proxy jumping provides a secure way to access hosts that are not directly accessible from the client's network. For general information about SSH client implementation, see [SSH Client Implementation](https://deepwiki.com/charmbracelet/wishlist/4-ssh-client-implementation)
.

Overview of Proxy Jump
----------------------

Proxy jump (sometimes referred to as "jump host" or "bastion host") is an SSH feature that enables connecting to a target server by first establishing a connection through an intermediate server. This is particularly useful in secure environments where direct access to internal servers is restricted.

Sources: [jump.go](https://github.com/charmbracelet/wishlist/blob/5fe34a29/jump.go)

Usage
-----

### Endpoint Configuration

To use proxy jump in Wishlist, specify the `ProxyJump` field in an endpoint configuration:

| Field | Description | Example |
| --- | --- | --- |
| `ProxyJump` | Jump host specification in the format `[user@]host[:port]` | `jumpuser@jump.example.com` |
| `Address` | Target host address | `internal.example.com:22` |

Sources: [client.go26-41](https://github.com/charmbracelet/wishlist/blob/5fe34a29/client.go#L26-L41)

Error Handling and Connection Management
----------------------------------------

Wishlist implements robust error handling and connection management for proxy jumps:

1.  All connections are tracked using a `closers` slice that contains closure functions
2.  Connections are properly closed in case of errors
3.  Detailed error messages provide context about which part of the proxy chain failed

Sources: [jump.go12-36](https://github.com/charmbracelet/wishlist/blob/5fe34a29/jump.go#L12-L36)
 [client.go100-110](https://github.com/charmbracelet/wishlist/blob/5fe34a29/client.go#L100-L110)

Integration with SSH Client Interface
-------------------------------------

The proxy jump functionality is seamlessly integrated with Wishlist's SSH client interface. When a client requests a connection to an endpoint with a `ProxyJump` field, the system automatically handles the proxy jump without requiring additional user intervention.

Sources: [client.go15-18](https://github.com/charmbracelet/wishlist/blob/5fe34a29/client.go#L15-L18)

Conclusion
----------

Wishlist's proxy jump support provides a flexible and robust mechanism for connecting to remote hosts through intermediate jump servers. This implementation follows SSH best practices and integrates seamlessly with the overall Wishlist architecture.
