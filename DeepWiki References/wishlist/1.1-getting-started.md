Configuration
-------------

Wishlist can be configured in several ways. By default, it will look for configuration files in the following order:

1.  Path specified with `-config` flag
2.  `.wishlist/config.yaml`
3.  `.wishlist/config.yml`
4.  `.wishlist/config`
5.  `[[user config dir]]/wishlist/config.yaml`
6.  `[[user config dir]]/wishlist/config.yml`
7.  `[[user config dir]]/wishlist/config`
8.  `$HOME/.ssh/config`
9.  `/etc/ssh/ssh_config`

The first configuration file that is found and parsed without errors will be used.

Authentication
--------------

Authentication methods differ between local and server modes:

### Local Mode Authentication

When running in local mode, Wishlist uses the following authentication methods in order:

1.  If the endpoint has `IdentityFile` specified, it will try to use that
2.  If an SSH Agent is available, it will be used
3.  Otherwise, common key names in `~/.ssh` will be tried

### Server Mode Authentication

When running as a server, Wishlist:

1.  First tries to forward the current SSH Agent
2.  If no agent is available, it creates or uses an existing ed25519 key in `.wishlist/client_ed25519`

Note: Password authentication is not supported.

Sources: [README.md111-151](https://github.com/charmbracelet/wishlist/blob/5fe34a29/README.md?plain=1#L111-L151)
