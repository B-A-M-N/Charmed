Operation Modes

This document explains the two primary operation modes of Wishlist: Local Mode and Server Mode. These modes determine how users interact with the Wishlist SSH directory system. For information about the overall architecture, see [Architecture Overview](https://deepwiki.com/charmbracelet/wishlist/2-architecture-overview)
.

Mode Overview
-------------

Wishlist operates in two distinct modes:

1.  **Local Mode** - The default mode where Wishlist runs locally as a terminal application on your machine
2.  **Server Mode** - Wishlist runs as an SSH server that clients can connect to remotely

The operational mode is determined by the command used to start Wishlist.

Sources: [cmd/wishlist/main.go45-106](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L45-L106)
 [cmd/wishlist/main.go126-195](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L126-L195)

Local Mode
----------

Local Mode is the default operation mode, invoked when running the `wishlist` command without a `serve` subcommand. In this mode, Wishlist presents a Terminal User Interface (TUI) directly on the user's local machine.

### TUI Navigation

When no specific endpoint is provided (`wishlist` or `wishlist list`), Wishlist displays an interactive list of available endpoints. Users can navigate this list and select an endpoint to connect to. This functionality is implemented in the `workLocally` function.

### Direct Connection

If a specific endpoint name is provided (`wishlist endpoint-name`), Wishlist will attempt to directly connect to that endpoint without showing the TUI list.

Sources: [cmd/wishlist/main.go395-419](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L395-L419)
 [cmd/wishlist/main.go421-430](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L421-L430)

Server Mode
-----------

Server Mode is activated with the `wishlist serve` command (or its aliases `server`, `s`). In this mode, Wishlist starts an SSH server that clients can connect to remotely to access the endpoint list.

### SSH Server Setup

When operating in Server Mode, Wishlist:

1.  Creates the necessary keypair if it doesn't exist (in `.wishlist/server_ed25519`)
2.  Configures an SSH server for each endpoint defined in the configuration
3.  Sets up a special "list" endpoint that presents the TUI over SSH

Sources: [cmd/wishlist/main.go126-195](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L126-L195)
 [server.go22-95](https://github.com/charmbracelet/wishlist/blob/5fe34a29/server.go#L22-L95)

### Endpoint Management

Each endpoint can have its own configuration, including:

*   A unique address (hostname:port)
*   Custom middlewares for extending functionality
*   Authentication settings

Wishlist automatically assigns ports to endpoints if not explicitly configured, starting from the base port defined in the configuration.

### Dynamic Endpoint Refresh

Server Mode supports dynamic endpoint refreshing via the `--endpoints.refresh.interval` flag, which allows the list of available endpoints to be updated periodically without restarting the server.

Sources: [cmd/wishlist/main.go142-164](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L142-L164)
 [server.go44-52](https://github.com/charmbracelet/wishlist/blob/5fe34a29/server.go#L44-L52)

Implementation Details
----------------------

### Server Factory

In Server Mode, Wishlist uses a factory function to create SSH servers for each endpoint. This factory function applies appropriate middlewares for each endpoint, including the endpoint-specific middlewares and common ones like logging and terminal handling.

Sources: [cmd/wishlist/main.go176-188](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L176-L188)
 [server.go98-121](https://github.com/charmbracelet/wishlist/blob/5fe34a29/server.go#L98-L121)

### Mode Comparison

The following table summarizes the key differences between the two operation modes:

| Feature | Local Mode | Server Mode |
| --- | --- | --- |
| Invocation | `wishlist` or `wishlist list` | `wishlist serve` |
| UI Location | Local terminal | Remote SSH client |
| Authentication | Uses local SSH client credentials | Requires server-side auth configuration |
| Use Case | Personal use on local machine | Team-wide SSH directory service |
| Connection Flow | Direct connection to endpoints | Two-step: SSH to Wishlist then to target |
| Dynamic Updates | Requires restart to refresh | Supports automatic refreshing |

Sources: [cmd/wishlist/main.go45-106](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L45-L106)
 [cmd/wishlist/main.go126-195](https://github.com/charmbracelet/wishlist/blob/5fe34a29/cmd/wishlist/main.go#L126-L195)
 [server.go22-95](https://github.com/charmbracelet/wishlist/blob/5fe34a29/server.go#L22-L95)

Example Usage
-------------

### Local Mode

    # Launch TUI with list of endpoints
    wishlist
    
    # Directly connect to a specific endpoint
    wishlist server-name
    

### Server Mode

    # Start Wishlist as an SSH server
    wishlist serve
    
    # Connect to Wishlist server from an SSH client
    ssh user@wishlist-host
