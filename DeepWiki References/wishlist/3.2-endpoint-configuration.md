This page documents how endpoints are defined, discovered, and configured in Wishlist. Endpoints represent SSH connections that can be listed and accessed through the Wishlist interface. For information about how Wishlist integrates with standard SSH configuration files, see [SSH Config Integration](https://deepwiki.com/charmbracelet/wishlist/3.1-ssh-config-integration)
.

Endpoint Structure
------------------

In Wishlist, an endpoint represents a target SSH connection with various properties that define how to connect to it. Each endpoint has a unique name, an address in the format `host:port`, and additional configuration options.

Sources: [config.go39-144](https://github.com/charmbracelet/wishlist/blob/5fe34a29/config.go#L39-L144)

The `Endpoint` struct contains several fields that mirror standard SSH client configuration options:

| Field | Description | SSH Equivalent |
| --- | --- | --- |
| `Name` | Unique identifier for the endpoint | \-  |
| `Address` | Host and port in the format `host:port` | `HostName` and `Port` |
| `User` | Username for authentication | `User` |
| `ForwardAgent` | Whether to forward the SSH agent | `ForwardAgent` |
| `RequestTTY` | Whether to request a TTY | `RequestTTY` |
| `RemoteCommand` | Command to execute on the remote host | Command specified after the host |
| `ProxyJump` | Host to use as a jump server | `ProxyJump` |
| `SendEnv` | Environment variables to send to the server | `SendEnv` |
| `SetEnv` | Environment variables to set on the server | `SetEnv` |
| `PreferredAuthentications` | Authentication methods to try | `PreferredAuthentications` |
| `IdentityFiles` | SSH keys to use for authentication | `IdentityFile` |
| `Timeout` | Connection timeout | `ConnectTimeout` |

Sources: [config.go39-56](https://github.com/charmbracelet/wishlist/blob/5fe34a29/config.go#L39-L56)
 [\_example/config.yaml14-74](https://github.com/charmbracelet/wishlist/blob/5fe34a29/_example/config.yaml#L14-L74)

Endpoint Discovery Methods
--------------------------

Wishlist can automatically discover endpoints from several sources, reducing the need for manual configuration.

Sources: [tailscale/tailscale.go](https://github.com/charmbracelet/wishlist/blob/5fe34a29/tailscale/tailscale.go)

### Tailscale Discovery

Wishlist can discover endpoints from Tailscale by querying the Tailscale API for devices in a specified tailnet.

Sources: [tailscale/tailscale.go18-69](https://github.com/charmbracelet/wishlist/blob/5fe34a29/tailscale/tailscale.go#L18-L69)

The Tailscale discovery method uses either API key or OAuth2 client credentials for authentication with the Tailscale API.

### Zeroconf/mDNS Discovery

Wishlist can discover SSH services advertised on the local network using Zeroconf/mDNS (Multicast DNS).

Sources: [zeroconf/zeroconf.go18-45](https://github.com/charmbracelet/wishlist/blob/5fe34a29/zeroconf/zeroconf.go#L18-L45)

Endpoint Hints
--------------

Endpoint hints provide a way to enhance discovered endpoints by applying additional configuration based on matching patterns.

Sources: [config.go58-75](https://github.com/charmbracelet/wishlist/blob/5fe34a29/config.go#L58-L75)
 [\_example/config.yaml75-108](https://github.com/charmbracelet/wishlist/blob/5fe34a29/_example/config.yaml#L75-L108)

### EndpointHint Structure

The `EndpointHint` struct contains pattern matching rules and configuration values to apply to matching endpoints.

Sources: [config.go58-75](https://github.com/charmbracelet/wishlist/blob/5fe34a29/config.go#L58-L75)

The `Match` field uses glob patterns to match endpoint names. For example, `"*.local"` would match all endpoints with names ending in `.local`.

### Configuring Hints

Hints are defined in the Wishlist YAML configuration file:

Sources: [\_example/config.yaml75-108](https://github.com/charmbracelet/wishlist/blob/5fe34a29/_example/config.yaml#L75-L108)

Multiple hints can be defined, and if an endpoint matches multiple hints, all matched hints will be applied with the last matching hint taking precedence.

Environment Variables Handling
------------------------------

Wishlist provides flexible handling of environment variables through two mechanisms:

1.  `SetEnv`: Sets specific environment variables in the SSH session
2.  `SendEnv`: Sends specified environment variables from the local environment to the SSH session

Sources: [config.go86-129](https://github.com/charmbracelet/wishlist/blob/5fe34a29/config.go#L86-L129)

The `Environment()` method on the `Endpoint` struct processes these environment variables:

1.  It first evaluates the local environment variables against the `SendEnv` patterns
2.  It then applies the `SetEnv` values, which take precedence over any values from `SendEnv`

By default, Wishlist sends the `LC_*` and `LANG` environment variables, similar to the default behavior of OpenSSH.

Authentication Methods
----------------------

Endpoints can specify preferred authentication methods through the `PreferredAuthentications` field. If not specified, Wishlist defaults to using `publickey` followed by `keyboard-interactive`.

The `Authentications()` method on the `Endpoint` struct returns the list of authentication methods to try.

Sources: [config.go79-84](https://github.com/charmbracelet/wishlist/blob/5fe34a29/config.go#L79-L84)
