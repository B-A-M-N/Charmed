Purpose and Scope
-----------------

This document explains how Sequin processes and interprets SGR (Select Graphic Rendition) escape sequences, which are used for text styling in terminal applications. SGR sequences control visual attributes such as bold, italic, underline, colors, and other text effects. This page covers the implementation of SGR handling in Sequin, the supported SGR parameters, and how they are translated into human-readable descriptions.

For information about cursor-related ANSI commands, see [Cursor Commands](https://deepwiki.com/charmbracelet/sequin/3.2-cursor-commands)
. For information about screen and line commands, see [Screen and Line Commands](https://deepwiki.com/charmbracelet/sequin/3.3-screen-and-line-commands)
.

What is SGR?
------------

SGR (Select Graphic Rendition) is part of the ANSI escape code standard that controls the appearance of text in the terminal. SGR sequences start with the CSI (Control Sequence Introducer) escape sequence `ESC[` followed by one or more numeric parameters separated by semicolons, and ending with the letter 'm'.\
\
For example, the sequence `ESC[1;31m` sets text to be bold and red.\
\
Sources: [sgr.go12-111](https://github.com/charmbracelet/sequin/blob/025641c1/sgr.go#L12-L111)\
\
SGR Parameter Processing\
------------------------\
\
Sequin's `handleSgr` function in `sgr.go` receives an ANSI parser object and processes the SGR parameters to generate human-readable descriptions. The function identifies various text styling attributes and color settings based on the numeric parameter values.\
\
Sources: [sgr.go12-111](https://github.com/charmbracelet/sequin/blob/025641c1/sgr.go#L12-L111)\

Color Handling\
--------------\
\
Sequin supports three types of color formats in SGR:\
\
1.  Basic ANSI colors (8 colors)\
2.  Extended ANSI256 colors (256 colors)\
3.  24-bit RGB colors (16+ million colors)\
\

    \
    In the broader context of Sequin, the SGR handling is one specialized part of the ANSI sequence processing pipeline:\

    ```mermaid\
    flowchart TD\
        Input["Input Stream"] --> Parser["ansi.Parser"]\
        Parser --> Dispatch{"Handler Dispatch"}\

        Dispatch -->|"CSI m"| SGRHandler["handleSgr()\nText styling"]\
        Dispatch -->|"Other CSI"| OtherHandlers["Other CSI handlers"]\

Summary\
-------\
\
The SGR handling in Sequin provides comprehensive support for ANSI text styling, including:\
\
1.  Basic text attributes (bold, italic, underline, etc.)\
2.  Multiple underline styles (single, double, curly, dotted, dashed)\
3.  Color support at three levels (basic ANSI, ANSI256, and 24-bit RGB)\
4.  Style reset capabilities\
\
This implementation allows Sequin to provide detailed, human-readable descriptions of text styling codes found in terminal output, making it easier for developers to understand and debug ANSI-based terminal applications.\
