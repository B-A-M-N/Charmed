ANSI Sequence Processing
========================

Relevant source files

*   [handlers.go](https://github.com/charmbracelet/sequin/blob/025641c1/handlers.go)
    
*   [main.go](https://github.com/charmbracelet/sequin/blob/025641c1/main.go)
    

This document details how Sequin processes ANSI escape sequences from input to formatted output. It covers the core processing pipeline that takes raw text with embedded ANSI sequences, parses them, and renders them in a human-readable format. For information about command execution to capture input for processing, see [Command Execution](https://deepwiki.com/charmbracelet/sequin/2.2-command-execution)
.

Overview
--------

At its core, Sequin analyzes input text byte by byte, identifies ANSI escape sequences, decodes them, and formats them with explanations. The system follows a clear pipeline design that transforms raw terminal output into a structured explanation of each control sequence.

Sources: [main.go68-287](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L68-L287)

Sequence Processing Pipeline
----------------------------

The main processing function (`process`) in Sequin manages the entire pipeline for transforming raw input containing ANSI sequences into human-readable output.

Sources: [main.go68-287](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L68-L287)

Parser Initialization and Sequence Decoding
-------------------------------------------

Sequin uses the `ansi.Parser` from the `github.com/charmbracelet/x/ansi` package to decode ANSI sequences. The parser is obtained and released using a pool pattern to optimize memory usage.

Sources: [main.go205-283](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L205-L283)

Sequence Type Identification
----------------------------

Sequin identifies different types of ANSI sequences using prefix detection functions:

| Sequence Type | Identification Function | Handler Map |
| --- | --- | --- |
| CSI | `ansi.HasCsiPrefix(seq)` | `csiHandlers` |
| DCS | `ansi.HasDcsPrefix(seq)` | `dcsHandlers` |
| OSC | `ansi.HasOscPrefix(seq)` | `oscHandlers` |
| PM  | `ansi.HasPmPrefix(seq)` | _Direct handling_ |
| SOS | `ansi.HasSosPrefix(seq)` | _Direct handling_ |
| APC | `ansi.HasApcPrefix(seq)` | _Direct handling_ |
| ESC | `ansi.HasEscPrefix(seq)` | `escHandler` |
| Control Code | `width == 0 && len(seq) == 1` | `ctrlCodes` |
| Text | `width > 0` | _Buffered_ |

Sources: [main.go212-282](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L212-L282)

Handler Dispatch System
-----------------------

After detecting the sequence type, Sequin dispatches the sequence to the appropriate handler based on the command code. Handlers are organized in maps by sequence type:

The key algorithm for handler dispatch:

1.  Extract the command code using `p.Command()`
2.  Look up the appropriate handler in the handler map
3.  Call the handler function with the parser
4.  Format and print the result

Sources: [main.go187-203](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L187-L203)
 [handlers.go10-92](https://github.com/charmbracelet/sequin/blob/025641c1/handlers.go#L10-L92)

Handler Tables
--------------

Sequin organizes handlers in map tables based on sequence type. The command code serves as the key in these maps.

Sources: [handlers.go10-85](https://github.com/charmbracelet/sequin/blob/025641c1/handlers.go#L10-L85)

Text and Formatting Management
------------------------------

Sequin manages two primary formatting functions:

1.  `seqPrint`: Formats and prints sequence data with explanation
2.  `flushPrint`: Prints accumulated text when a sequence is encountered

Sources: [main.go80-172](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L80-L172)
 [main.go174-185](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L174-L185)

Theme Application and Styling
-----------------------------

The output is styled using a theme system that applies different styles based on the content type:

| Content Type | Theme Style |
| --- | --- |
| Sequence | `t.sequence` |
| Text | `t.text` |
| Explanation | `t.explanation` |
| Error | `t.error` |
| Kind label | `t.kindStyle(kind)` |

The theme is selected based on the environment variable `SEQUIN_THEME` and can be customized for dark/light backgrounds.

Sources: [main.go69-78](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L69-L78)

Control Code Handling
---------------------

Sequin understands and explains a wide range of control codes (both C0 and C1 sets), providing human-readable explanations for them.

Sources: [main.go289-361](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L289-L361)

Raw Mode
--------

Sequin provides a "raw mode" (`--raw` flag) that displays sequences without explanations, which is useful for more technical displays or when only interested in the sequence values themselves.

Sources: [main.go84-86](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L84-L86)
 [main.go178-182](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L178-L182)

ANSI Sequence Type Overview

Conclusion
----------

The ANSI sequence processing system in Sequin follows a clear pipeline design that transforms raw terminal output into a structured explanation of each control sequence. It leverages the `ansi.Parser` for sequence decoding and dispatches each sequence to specialized handlers that provide human-readable explanations.
