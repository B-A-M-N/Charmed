Purpose and Overview
--------------------

This document explains how Sequin interprets and displays FinalTerm shell integration commands. FinalTerm commands are specialized Operating System Command (OSC) sequences used for shell integration that help track command execution status and prompt locations. These commands enable terminal emulators to understand the context of shell activity, enhancing the user experience by providing information about command execution status.

For information about other OSC commands like window titles and colors, see [Window Title and Colors](https://deepwiki.com/charmbracelet/sequin/3.4-window-title-and-colors)
.

Sources: [final\_term.go1-2](https://github.com/charmbracelet/sequin/blob/025641c1/final_term.go#L1-L2)

FinalTerm Command Structure
---------------------------

FinalTerm commands in Sequin follow the OSC 133 format with specific parameters that indicate different shell states. The general structure is:

    OSC 133;<TYPE>[;<ADDITIONAL_DATA>] ST
    

Where:

*   `OSC` is the Operating System Command escape sequence introducer
*   `133` is the specific code for FinalTerm shell integration
*   `<TYPE>` is a single character indicating the command type
*   `<ADDITIONAL_DATA>` is optional data (such as exit codes)
*   `ST` is the string terminator

Sources: [final\_term.go11-16](https://github.com/charmbracelet/sequin/blob/025641c1/final_term.go#L11-L16)

Command Types
-------------

Sequin recognizes four distinct types of FinalTerm commands, each representing a different state in shell operation:

| Command Type | Format | Description |
| --- | --- | --- |
| Prompt start | OSC 133;A | Indicates the beginning of a shell prompt |
| Command start | OSC 133;B | Marks the beginning of command input |
| Command executed | OSC 133;C | Signals that a command has been executed |
| Command finished | OSC 133;D\[;<EXIT\_CODE>\] | Indicates a command has finished, optionally with exit code |

Sources: [final\_term.go23-38](https://github.com/charmbracelet/sequin/blob/025641c1/final_term.go#L23-L38)

Implementation Details
----------------------

Sequin processes FinalTerm commands through a dedicated handler function that parses and formats these commands into human-readable explanations.

The handler function follows these steps:

1.  Splits the command data by semicolons to extract the command type and any additional parameters
2.  Validates that there are at least two parts and the command type is a single character
3.  Generates appropriate descriptive text based on the command type
4.  For "Command finished" commands, includes the exit code if present

Sources: [final\_term.go12-39](https://github.com/charmbracelet/sequin/blob/025641c1/final_term.go#L12-L39)

Handling Mechanism
------------------

The FinalTerm command handler processes OSC 133 sequences by parsing the data portion of the sequence and extracting meaningful information:

The command parameters are extracted using a byte splitting operation on the parser data. The handler checks for validity of these parameters and returns an appropriate description or an error if the sequence is invalid.

Sources: [final\_term.go12-40](https://github.com/charmbracelet/sequin/blob/025641c1/final_term.go#L12-L40)

Error Handling
--------------

When encountering invalid FinalTerm commands, Sequin returns an error message. A command may be considered invalid if:

1.  It doesn't have at least two parts separated by semicolons
2.  The command type (second part) is not a single character
3.  The command type character is not one of the recognized types (A, B, C, or D)

This error handling ensures that users are aware when they encounter malformed or unsupported FinalTerm commands.
