Pointer, Clipboard, and Links
=============================

Relevant source files

Purpose and Scope
-----------------

This document details how Sequin processes and interprets specialized ANSI escape sequences related to pointer shapes, clipboard operations, hyperlinks, and system notifications. These handlers extend Sequin's capabilities beyond basic text styling and cursor movements (covered in [Cursor Commands](https://deepwiki.com/charmbracelet/sequin/3.2-cursor-commands)
) to include more advanced terminal features.

Overview of Specialized Handlers
--------------------------------

Sequin includes several specialized handlers for interpreting modern terminal capabilities that go beyond traditional ANSI functionality. These handlers process Operating System Commands (OSC) and Control Sequence Introducer (CSI) sequences related to pointer appearance, clipboard interaction, hyperlink embedding, and notification triggers.

Sources: [pointer.go](https://github.com/charmbracelet/sequin/blob/025641c1/pointer.go)
 [clipboard.go](https://github.com/charmbracelet/sequin/blob/025641c1/clipboard.go)
 [hyperlink.go](https://github.com/charmbracelet/sequin/blob/025641c1/hyperlink.go)
 [notify.go](https://github.com/charmbracelet/sequin/blob/025641c1/notify.go)

Pointer Shape Handler
---------------------

The pointer shape handler interprets ANSI sequences that modify the mouse cursor appearance in the terminal. This functionality allows terminal applications to change the cursor shape for different contexts (e.g., indicating text selection mode vs. normal mode).

### Implementation

The `handlePointerShape` function splits the sequence data using semicolons and expects exactly two parts. It extracts the pointer shape description and formats it into a human-readable explanation.

Sources: [pointer.go11-19](https://github.com/charmbracelet/sequin/blob/025641c1/pointer.go#L11-L19)

Clipboard Operations
--------------------

Sequin supports interpreting clipboard-related sequences that interact with the system clipboard. These sequences allow terminal applications to read from or write to the clipboard.

### Clipboard Types

Sequin recognizes two types of clipboards:

*   **System clipboard** (denoted by 'c'): The standard clipboard used by most applications
*   **Primary clipboard** (denoted by 'p'): The X11 primary selection (typically paste with middle click)

### Operations

The clipboard handler supports two operations:

1.  **Request** - Ask for clipboard content
2.  **Set** - Put new content into the clipboard

When setting content, the data is expected to be base64-encoded in the ANSI sequence and is decoded by the handler.

Sources: [clipboard.go11-36](https://github.com/charmbracelet/sequin/blob/025641c1/clipboard.go#L11-L36)

Hyperlink Handler
-----------------

The hyperlink handler processes ANSI sequences that embed clickable links in terminal output. Modern terminals support this feature to create interactive text that can open URLs when clicked.

### Implementation

The `handleHyperlink` function splits the input sequence by semicolons, expecting three parts:

1.  The command identifier
2.  Options (which may be further split by colons)
3.  The URL to link to

The handler formats these components into a comprehensive description of the hyperlink being created.

Sources: [hyperlink.go11-29](https://github.com/charmbracelet/sequin/blob/025641c1/hyperlink.go#L11-L29)

Notification Handler
--------------------

The notification handler interprets ANSI sequences that trigger system notifications. This allows terminal applications to request attention from the user even when the terminal is not in focus.

### Implementation

The `handleNotify` function processes notification sequences by splitting the input by semicolons and expecting exactly two parts. It extracts the notification message and returns a human-readable description.

Sources: [notify.go11-19](https://github.com/charmbracelet/sequin/blob/025641c1/notify.go#L11-L19)

Integration with Sequin's Architecture
--------------------------------------

These specialized handlers are part of Sequin's modular design for processing ANSI sequences. They integrate with the core parser and dispatcher system to provide comprehensive coverage of modern terminal capabilities.

Example Sequence Handling
-------------------------

The table below shows example ANSI sequences and how Sequin interprets them:

| Sequence Type | Example ANSI Sequence | Sequin Interpretation |
| --- | --- | --- |
| Pointer Shape | `OSC 22;shape=hand` | Set pointer shape to "hand" |
| Clipboard Request | `OSC 52;c;?` | Request "system" clipboard |
| Clipboard Set | `OSC 52;p;SGVsbG8=` | Set clipboard "primary" to "Hello" |
| Hyperlink | `OSC 8;id=doc:help;https://example.com` | Set hyperlink, id:doc:help to "[https://example.com](https://example.com/)<br>" |
| Notification | `OSC 9;New message received` | Notify "New message received" |

Sources: [pointer.go](https://github.com/charmbracelet/sequin/blob/025641c1/pointer.go)
 [clipboard.go](https://github.com/charmbracelet/sequin/blob/025641c1/clipboard.go)

 [notify.go](https://github.com/charmbracelet/sequin/blob/025641c1/notify.go)

Error Handling
--------------

All handlers follow a consistent error handling pattern:

1.  They check if the sequence format is valid (correct number of parts)
2.  They return `errInvalid` if the format doesn't match expectations
3.  For clipboard operations, they additionally handle base64 decoding errors

This consistent approach ensures that malformed sequences don't cause Sequin to crash and instead provide informative error feedback.

Sources: [pointer.go14-16](https://github.com/charmbracelet/sequin/blob/025641c1/pointer.go#L14-L16)
 [clipboard.go19-22](https://github.com/charmbracelet/sequin/blob/025641c1/clipboard.go#L19-L22)

Summary
-------

Sequin's specialized handlers for pointer shapes, clipboard operations, hyperlinks, and notifications extend its capabilities to interpret modern terminal features. These handlers follow a consistent design pattern:

1.  Parse the ANSI sequence using the parser
2.  Validate the sequence format
3.  Extract relevant information
4.  Format a human-readable explanation
5.  Handle errors appropriately

This systematic approach allows Sequin to provide comprehensive debugging information for a wide range of terminal capabilities beyond basic styling and cursor commands.
