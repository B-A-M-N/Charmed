Purpose and Scope
-----------------

This document details how Sequin processes ANSI escape sequences related to screen and line manipulation. These commands control how terminal content is erased, scrolled, or otherwise manipulated at the screen or line level. They are a subset of the ANSI control sequences that Sequin can parse and explain.

For information about cursor movement commands, see [Cursor Commands](https://deepwiki.com/charmbracelet/sequin/3.2-cursor-commands)
. For information about terminal modes, see [Terminal Modes](https://deepwiki.com/charmbracelet/sequin/3.5-terminal-modes)
.

Overview
--------

Screen and line commands allow terminal applications to clear portions of the display, define scrollable regions, insert or delete lines, and scroll content. Sequin processes these commands using dedicated handlers that interpret the parameters and produce human-readable explanations.

Sources: [screen.go10-45](https://github.com/charmbracelet/sequin/blob/025641c1/screen.go#L10-L45)
 [line.go10-36](https://github.com/charmbracelet/sequin/blob/025641c1/line.go#L10-L36)

Screen Commands
---------------

Screen commands primarily deal with clearing portions of the screen or defining scrollable regions. They are processed by the `handleScreen` function.

### Supported Screen Commands

| Command | Parameters | Description |
| --- | --- | --- |
| CSI 0J | None | Erase screen below cursor |
| CSI 1J | None | Erase screen above cursor |
| CSI 2J | None | Erase entire screen |
| CSI 3J | None | Erase entire display (including scrollback buffer) |
| CSI r | top, bottom | Set scrolling region from row 'top' to row 'bottom' |

Sources: [screen.go10-45](https://github.com/charmbracelet/sequin/blob/025641c1/screen.go#L10-L45)

### Screen Command Implementation

The screen command handler evaluates the command's final character and any parameters to determine the specific operation. For erasing operations (J), a single parameter dictates which portion of the screen to erase. For setting scrolling regions (r), two parameters define the top and bottom boundaries.

Sources: [screen.go10-45](https://github.com/charmbracelet/sequin/blob/025641c1/screen.go#L10-L45)

Line Commands
-------------

Line commands manipulate individual lines or groups of lines on the screen. They are processed by the `handleLine` function.

### Supported Line Commands

| Command | Parameters | Description |
| --- | --- | --- |
| CSI 0K | None | Erase from cursor to end of line |
| CSI 1K | None | Erase from start of line to cursor |
| CSI 2K | None | Erase entire line |
| CSI nL | n (default 1) | Insert n blank lines |
| CSI nM | n (default 1) | Delete n lines |
| CSI nS | n (default 1) | Scroll up n lines |
| CSI nT | n (default 1) | Scroll down n lines |

Sources: [line.go10-36](https://github.com/charmbracelet/sequin/blob/025641c1/line.go#L10-L36)

### Line Command Implementation

The line command handler examines the command character and any parameters to determine the specific operation. For line erasing (K), a parameter indicates which portion of the line to erase. For line insertion, deletion, and scrolling commands (L, M, S, T), a parameter specifies the number of lines affected, defaulting to 1 if not specified.

Sources: [line.go10-36](https://github.com/charmbracelet/sequin/blob/025641c1/line.go#L10-L36)

Integration with ANSI Parser
----------------------------

Both screen and line command handlers integrate with Sequin's ANSI parser system. The parser extracts the command characters and parameters, which the handlers then interpret to generate human-readable descriptions.

Sources: [screen.go10-45](https://github.com/charmbracelet/sequin/blob/025641c1/screen.go#L10-L45)
 [line.go10-36](https://github.com/charmbracelet/sequin/blob/025641c1/line.go#L10-L36)

Parameter Handling
------------------

Both handlers use the `ansi.Parser` to extract parameters with default values. The parameter extraction pattern is consistent across both handlers:

1.  Check if a parameter exists at a specific index
2.  If it exists, use the provided value
3.  If it doesn't exist, use a default value

Error Handling
--------------

Both handlers return an `errUnhandled` error when they encounter a command they cannot process. This error is defined elsewhere in the codebase and is used as a signal to the calling code that the specific sequence wasn't recognized by the handler.

Sources: [screen.go44](https://github.com/charmbracelet/sequin/blob/025641c1/screen.go#L44-L44)

Summary
-------

Screen and Line commands in Sequin provide terminal applications with the ability to manipulate the display at different granularities. The implementation in Sequin translates these technical ANSI sequences into human-readable explanations, making it easier for developers to understand what's happening in terminal applications.

These commands form a critical part of terminal control alongside cursor movement commands ([Cursor Commands](https://deepwiki.com/charmbracelet/sequin/3.2-cursor-commands)
) and terminal mode settings ([Terminal Modes](https://deepwiki.com/charmbracelet/sequin/3.5-terminal-modes)
).
