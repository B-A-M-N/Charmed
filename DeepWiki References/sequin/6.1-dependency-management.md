Dependency Management
=====================

Relevant source files

*   [.github/dependabot.yml](https://github.com/charmbracelet/sequin/blob/025641c1/.github/dependabot.yml)
    
*   [.github/workflows/dependabot-sync.yml](https://github.com/charmbracelet/sequin/blob/025641c1/.github/workflows/dependabot-sync.yml)

This document describes the dependency management approach used in the Sequin project. It explains how Go module dependencies are configured, versioned, and updated, as well as the automated processes in place to maintain dependencies. For information about the release process, see [Release Process](https://deepwiki.com/charmbracelet/sequin/6.2-release-process)
.

Go Module System
----------------

Sequin uses Go's built-in module system for dependency management. This provides version-aware dependency resolution and reproducible builds.

### Module Configuration

The project's dependencies are defined in the `go.mod` file, which specifies:

*   The module path (`github.com/charmbracelet/sequin`)
*   The Go version requirement (1.23.0)
*   The specific toolchain version (1.24.1)
*   All direct dependencies with their versions
*   Indirect dependencies (automatically managed)

Sources: [go.mod1-36](https://github.com/charmbracelet/sequin/blob/025641c1/go.mod#L1-L36)

### Core Dependencies

Sequin relies on several key dependencies to implement its functionality:

| Dependency | Purpose |
| --- | --- |
| github.com/charmbracelet/x/ansi | ANSI sequence parsing and processing |
| github.com/charmbracelet/lipgloss/v2 | Terminal formatting and styling |
| github.com/charmbracelet/colorprofile | Terminal color profile detection |
| github.com/charmbracelet/x/term | Terminal interaction utilities |
| github.com/charmbracelet/x/xpty | Pseudo-terminal handling |
| github.com/spf13/cobra | Command-line interface framework |
| github.com/charmbracelet/x/exp/golden | Testing utilities |
| github.com/stretchr/testify | Testing assertion framework |

Most dependencies come from the Charmbracelet ecosystem, ensuring consistent design and integration.

Sources: [go.mod7-16](https://github.com/charmbracelet/sequin/blob/025641c1/go.mod#L7-L16)

Automated Dependency Management
-------------------------------

Sequin uses GitHub's Dependabot to automate dependency updates, ensuring the project stays current with security patches and improvements.

### Dependabot Configuration

Dependabot is configured in `.github/dependabot.yml` to monitor and update three types of dependencies:

1.  **Go Modules**: Updates to Go libraries and packages
2.  **GitHub Actions**: Updates to CI/CD workflows
3.  **Docker**: Updates to Docker-related dependencies

Sources: [.github/dependabot.yml1-42](https://github.com/charmbracelet/sequin/blob/025641c1/.github/dependabot.yml#L1-L42)

Each ecosystem is configured with the following settings:

*   Weekly update schedule (Mondays at 5:00 AM Eastern Time)
*   Dependency updates labeled with "dependencies"
*   Customized commit message prefixes

### Dependabot Sync Workflow

Sequin maintains consistent Dependabot configurations across repositories through a sync workflow:

Sources: [.github/workflows/dependabot-sync.yml1-18](https://github.com/charmbracelet/sequin/blob/025641c1/.github/workflows/dependabot-sync.yml#L1-L18)

This workflow ensures that Dependabot configurations remain standardized across Charmbracelet projects, by:

*   Running weekly on Sundays
*   Supporting manual triggering when needed
*   Using a centralized workflow definition from the `charmbracelet/meta` repository

Version Pinning Strategy
------------------------

Sequin uses explicit version pinning in the `go.mod` file to ensure build reproducibility. Dependencies are generally pinned to specific versions, including pre-release versions when necessary.

For example:

*   Standard releases: `github.com/charmbracelet/colorprofile v0.3.0`
*   Beta releases: `github.com/charmbracelet/lipgloss/v2 v2.0.0-beta1`
*   Git commits: `github.com/charmbracelet/x/ansi v0.8.1-0.20250219181712-d0cc2ccd9c9c`

The `go.sum` file acts as a checksumming mechanism to verify that the downloaded modules match exactly what was used during development, preventing supply chain attacks and ensuring consistent builds.

Sources: [go.mod7-16](https://github.com/charmbracelet/sequin/blob/025641c1/go.mod#L7-L16)
 [go.sum1-58](https://github.com/charmbracelet/sequin/blob/025641c1/go.sum#L1-L58)

Manual Dependency Updates
-------------------------

While Dependabot handles routine updates, developers can manually update dependencies when necessary:

1.  Using `go get` to update a specific dependency:
    
        go get github.com/example/dependency@v1.2.3
        
    
2.  Using `go get -u` to update all dependencies to their latest minor or patch versions:
    
        go get -u ./...
        
    
3.  Using `go mod tidy` to clean up the module file:
    
        go mod tidy
        
    

After manual updates, developers should run the test suite to verify compatibility before committing changes.

Sources: [go.mod1-36](https://github.com/charmbracelet/sequin/blob/025641c1/go.mod#L1-L36)

Dependency Governance
---------------------

The overall dependency management strategy follows these principles:

1.  **Ecosystem Alignment**: Preference for dependencies within the Charmbracelet ecosystem to ensure consistent behavior and integration
2.  **Minimal Dependency Tree**: Carefully evaluating new dependencies to avoid bloating the project
3.  **Security**: Regular automated updates via Dependabot to address security vulnerabilities
4.  **Stability**: Pinning dependencies to specific versions to ensure reproducible builds

This approach balances staying current with maintaining stability and security.

Sources: [go.mod1-36](https://github.com/charmbracelet/sequin/blob/025641c1/go.mod#L1-L36)
