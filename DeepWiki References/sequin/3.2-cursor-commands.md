This page documents how Sequin processes and displays ANSI cursor control sequences. Cursor commands are a category of terminal control sequences that manipulate the cursor position, appearance, and state. For other categories of ANSI sequences such as styling text, see [SGR (Select Graphic Rendition)](https://deepwiki.com/charmbracelet/sequin/3.1-sgr-(select-graphic-rendition))
, or for screen manipulation commands, see [Screen and Line Commands](https://deepwiki.com/charmbracelet/sequin/3.3-screen-and-line-commands)
.

Overview of Cursor Commands
---------------------------

Cursor commands allow terminal applications to:

*   Move the cursor in various directions
*   Position the cursor at specific coordinates
*   Save and restore cursor positions
*   Change the cursor's visual appearance
*   Query cursor position information

These commands are critical for building interactive terminal interfaces, enabling applications to create complex visual layouts and UI elements.

Sources: [cursor.go10-60](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L10-L60)

Implementation in Sequin
------------------------

Sequin processes cursor commands through the `handleCursor` function in the cursor.go file. This function interprets the command type based on the final character of the CSI (Control Sequence Introducer) sequence and any provided parameters.

Sources: [cursor.go10-60](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L10-L60)

### Movement Commands

Sequin handles the following cursor movement commands:

| Final Character | Command | Description | Implementation |
| --- | --- | --- | --- |
| A   | CUU | Cursor Up | Moves cursor up by specified number of rows |
| B   | CUD | Cursor Down | Moves cursor down by specified number of rows |
| C   | CUF | Cursor Forward | Moves cursor right by specified number of columns |
| D   | CUB | Cursor Back | Moves cursor left by specified number of columns |
| E   | CNL | Cursor Next Line | Moves cursor to beginning of line n lines down |
| F   | CPL | Cursor Previous Line | Moves cursor to beginning of line n lines up |

For movement commands, Sequin extracts the parameter value (defaulting to 1 if not specified) and generates a human-readable description of the operation.

Sources: [cursor.go19-34](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L19-L34)

### Cursor Positioning

The most common positioning command uses the final character 'H' and allows setting the cursor to an absolute position specified by row and column coordinates:

    CSI row;column H
    

Sequin extracts both parameters with defaults of 1,1 (the top-left corner of the screen) if parameters are not provided.

Sources: [cursor.go35-43](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L35-L43)

### Position Queries

Terminals can request the cursor position using the 'n' command with parameter 6:

| Final Character | Command | Description | Implementation |
| --- | --- | --- | --- |
| n   | DSR | Device Status Report | Used with parameter 6 to request cursor position |
| n with '?' prefix | Extended DSR | Extended status report | Requests extended cursor position information |

Sources: [cursor.go44-51](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L44-L51)

### Save and Restore Position

These commands allow storing the current cursor position and returning to it later:

| Final Character | Command | Description |
| --- | --- | --- |
| s   | SCP | Save Cursor Position |
| u   | RCP | Restore Cursor Position |

Sources: [cursor.go52-55](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L52-L55)

Cursor Style Commands
---------------------

The command with final character 'q' (DECSCUSR - Set Cursor Style) allows changing the cursor's visual appearance. Sequin supports the following styles:

The style parameter determines both the cursor shape and whether it blinks:

| Parameter | Style Description | Visual Appearance |
| --- | --- | --- |
| 0 or 1 | Blinking block | █ (blinking) |
| 2   | Steady block | █ (non-blinking) |
| 3   | Blinking underline | \_ (blinking) |
| 4   | Steady underline | \_ (non-blinking) |
| 5   | Blinking bar | \| (blinking) |
| 6   | Steady bar | \| (non-blinking) |

These styles are described by the `descCursorStyle` function which converts the numeric parameter to a human-readable description.

Sources: [cursor.go56-80](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L56-L80)

| ANSI Sequence | Sequin Output |
| --- | --- |
| CSI 3 A | Cursor up 3 |

| CSI 5 C | Cursor right 5 |
| CSI 2 D | Cursor left 2 |
| CSI 10;20 H | Set cursor position row=10 col=20 |
| CSI s | Save cursor position |
| CSI u | Restore cursor position |
| CSI 3 q | Set cursor style Blinking underline |

Integration with Sequin's Processing Pipeline
---------------------------------------------

Cursor command handling is one component of Sequin's ANSI sequence processing system. The handle function is registered with the appropriate dispatch table and called when cursor-related CSI sequences are detected.

Sources: [cursor.go10-60](https://github.com/charmbracelet/sequin/blob/025641c1/cursor.go#L10-L60)

Sequin's handling of cursor commands is important for providing developers with clear insights into how terminal applications manipulate cursor position and appearance, making it easier to debug and understand complex terminal interfaces.
