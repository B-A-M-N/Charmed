Release Process
===============

Relevant source files

*   [.github/workflows/release.yml](https://github.com/charmbracelet/sequin/blob/025641c1/.github/workflows/release.yml)
    

This document outlines the automated and manual processes involved in releasing new versions of Sequin. It covers how releases are triggered, built, signed, and distributed to various package managers and platforms. For information about the general build and development process, see [Build and Development](https://deepwiki.com/charmbracelet/sequin/6-build-and-development)
.

Release Triggers and Versioning
-------------------------------

Sequin follows semantic versioning (SemVer) for its releases. New releases are triggered automatically when a git tag matching the pattern `v*.*.*` (e.g., `v1.2.3`) is pushed to the repository.

Release Authentication and Secrets
----------------------------------

The release process requires authentication with various services. The following secrets are used in the GitHub Actions workflow:

| Secret | Purpose |
| --- | --- |
| `DOCKERHUB_USERNAME`, `DOCKERHUB_TOKEN` | Authentication for publishing Docker images |
| `PERSONAL_ACCESS_TOKEN` | GitHub token for creating releases |
| `GORELEASER_KEY` | Key for signing releases |
| `AUR_KEY` | SSH key for publishing to Arch User Repository |
| `FURY_TOKEN` | API token for Package Fury |
| `NFPM_GPG_KEY`, `NFPM_PASSPHRASE` | GPG key for signing Linux packages |
| `MACOS_SIGN_P12`, `MACOS_SIGN_PASSWORD` | Certificate for signing macOS binaries |
| `MACOS_NOTARY_*` | Credentials for Apple's notarization service |
| Various social media tokens | For posting release announcements |

Sources: [.github/workflows/release.yml20-42](https://github.com/charmbracelet/sequin/blob/025641c1/.github/workflows/release.yml#L20-L42)

Release Verification
--------------------

After the automated release process completes, it's important to verify that the release was successful:

1.  Check the GitHub Actions workflow logs for any errors
2.  Verify the GitHub release page has been created with the correct artifacts
3.  Confirm that Docker images are available on Docker Hub
4.  Test installation from various package managers:
    *   `brew install charmbracelet/tap/sequin`
    *   `apt-get install sequin` (if using the APT repository)
    *   `yum install sequin` (if using the YUM repository)

Troubleshooting Release Issues
------------------------------

If issues occur during the release process:

1.  Check the GitHub Actions workflow logs for detailed error information
2.  For package manager issues, verify that the corresponding secrets are correct
3.  For code signing issues, ensure the certificates are valid and not expired
4.  If needed, delete the problematic GitHub tag, fix the issues, and push a new tag
