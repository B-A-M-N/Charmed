Command Execution
=================

This page documents the command execution subsystem in Sequin, which enables the application to run external commands and capture their output for ANSI sequence analysis. Understanding this system is essential for comprehending how Sequin processes terminal output from other programs. For information about how the captured ANSI sequences are processed, see [ANSI Sequence Processing](https://deepwiki.com/charmbracelet/sequin/2.1-ansi-sequence-processing)
.

Overview
--------

Sequin supports two primary input methods:

1.  Processing text piped directly into the application
2.  Executing a command and processing its output

The command execution functionality enables the second approach, allowing users to analyze terminal output from any command without having to manually capture and pipe it. This feature is invoked when arguments are provided to Sequin after a double dash (`--`).

Sources: [main.go47-60](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L47-L60)
 [exec.go19-51](https://github.com/charmbracelet/sequin/blob/025641c1/exec.go#L19-L51)

Command Execution Flow
----------------------

When Sequin is invoked with command arguments, it creates a pseudo-terminal (PTY) to run the command, which ensures that the command behaves as if it were running in an actual terminal, including the output of ANSI escape sequences.

Sources: [main.go47-60](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L47-L60)
 [exec.go19-51](https://github.com/charmbracelet/sequin/blob/025641c1/exec.go#L19-L51)

Implementation Details
----------------------

### Command Execution Function

The core functionality is implemented in the `executeCommand` function in `exec.go`. This function:

1.  Determines the appropriate terminal dimensions
2.  Creates a pseudo-terminal (PTY)
3.  Executes the requested command
4.  Captures all output including ANSI escape sequences
5.  Returns the captured output

Sources: [exec.go19-51](https://github.com/charmbracelet/sequin/blob/025641c1/exec.go#L19-L51)

### Terminal Size Handling

The command execution process begins by determining the dimensions of the current terminal. This ensures that the executed command receives the same terminal dimensions as the user's terminal, which is important for programs that adjust their output based on the terminal width.

Example Usage
-------------

To use Sequin's command execution functionality, users provide the command and its arguments after a double dash:

    sequin -- ls --color=always
    

This executes the `ls` command with color enabled and processes its output through Sequin, displaying all ANSI sequences in a human-readable format.

Sources: [main.go42-46](https://github.com/charmbracelet/sequin/blob/025641c1/main.go#L42-L46)

Error Handling
--------------

The command execution system handles errors in several ways:

1.  **Terminal Size Errors**: Falls back to default dimensions (80x24)
2.  **PTY Creation Errors**: Returns an error and aborts execution
3.  **Command Execution Errors**: Returns both the error and any output that was captured before the error

When a command fails, Sequin still attempts to provide any output that was generated, which may contain useful error information with ANSI sequences that Sequin can interpret.

Technical Implementation
------------------------

The command execution leverages two key external packages:

1.  `github.com/charmbracelet/x/term`: For detecting terminal dimensions
2.  `github.com/charmbracelet/x/xpty`: For PTY creation and management

These packages handle the complex task of creating and managing pseudo-terminals across different operating systems, allowing Sequin to focus on its core functionality of ANSI sequence analysis.
