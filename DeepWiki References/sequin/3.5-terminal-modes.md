Purpose and Scope
-----------------

This document explains how Sequin handles terminal mode commands. Terminal modes are special settings that modify terminal behavior, including cursor appearance, mouse tracking, alternate screen buffers, and input handling options. This page covers how Sequin processes, interprets, and displays these mode-related ANSI escape sequences.

For information about cursor positioning and movement commands, see [Cursor Commands](https://deepwiki.com/charmbracelet/sequin/3.2-cursor-commands)
. For information about screen clearing and manipulation, see [Screen and Line Commands](https://deepwiki.com/charmbracelet/sequin/3.3-screen-and-line-commands)
.

Terminal Modes Overview
-----------------------

Terminal modes are special states that can be enabled or disabled to change terminal behavior. Applications use ANSI escape sequences to request information about modes, enable modes, or disable modes. Sequin interprets these sequences and displays human-readable explanations of what mode is being modified and how.

Sources: [mode.go9-30](https://github.com/charmbracelet/sequin/blob/025641c1/mode.go#L9-L30)

Mode Handler Implementation
---------------------------

In Sequin, terminal modes are handled by the `handleMode` function which processes CSI (Control Sequence Introducer) sequences related to terminal mode operations. The function supports three primary operations:

1.  **Enable Mode** (CSI h or CSI ? h): Turns on a specific terminal mode
2.  **Disable Mode** (CSI l or CSI ? l): Turns off a specific terminal mode
3.  **Request Mode** (CSI p or CSI ? p): Queries the current state of a mode

The handler also distinguishes between standard modes and private modes (identified by the '?' prefix).

The mode handler function extracts the mode number from the first parameter of the sequence, determines if it's a private mode, and then checks the final character of the command to decide which operation is being performed.

Sources: [mode.go9-30](https://github.com/charmbracelet/sequin/blob/025641c1/mode.go#L9-L30)

Supported Terminal Modes
------------------------

Sequin recognizes and provides descriptions for a range of common terminal modes through the `modeDesc` function. Each mode is identified by a numeric code:

| Mode Code | Description | Purpose |
| --- | --- | --- |
| 1   | cursor keys | Controls how cursor keys are reported |
| 25  | cursor visibility | Shows or hides the cursor |
| 1000 | show mouse | Basic mouse reporting |
| 1001 | mouse hilite | Mouse highlighting tracking |
| 1002 | mouse cell motion | Reports mouse movement when buttons are pressed |
| 1003 | mouse all motion | Reports all mouse movement |
| 1004 | report focus | Reports window focus/unfocus events |
| 1006 | mouse SGR ext | Extended mouse reporting |
| 1049 | altscreen | Enables/disables alternate screen buffer |
| 2004 | bracketed paste | Marks pasted text with special sequences |
| 2026 | synchronized output | Synchronizes terminal updates |
| 2027 | grapheme clustering | Controls how grapheme clusters are handled |
| 9001 | win32 input | Windows-specific input mode |

If a mode code is not recognized, Sequin will display it as "unknown."

Sources: [mode.go32-64](https://github.com/charmbracelet/sequin/blob/025641c1/mode.go#L32-L64)

Common Mode Sequences
---------------------

### Alternate Screen Buffer

The alternate screen buffer (mode 1049) is one of the most commonly used terminal modes. Applications like vim, less, and many TUIs use it to create a temporary view that can be discarded to return to the previous terminal content.

    CSI ? 1049 h - Enable alternate screen buffer
    CSI ? 1049 l - Disable alternate screen buffer (return to normal screen)
    

### Mouse Tracking

Mouse tracking modes allow terminal applications to receive mouse events:

    CSI ? 1000 h - Enable basic mouse reporting (button clicks)
    CSI ? 1002 h - Enable mouse cell motion reporting (movement while buttons pressed)
    CSI ? 1003 h - Enable all mouse motion reporting (any movement)
    CSI ? 1006 h - Enable SGR extended mouse reporting format
    

### Cursor Visibility

Applications can hide the cursor during operations where it might be distracting:

    CSI ? 25 l - Hide cursor
    CSI ? 25 h - Show cursor
    

Sources: [mode.go32-64](https://github.com/charmbracelet/sequin/blob/025641c1/mode.go#L32-L64)

Special Keypad Modes
--------------------

In addition to the CSI mode sequences, Sequin also recognizes two special ESC sequences for keypad modes:

1.  **Application Keypad Mode** (`ESC =`): Makes the numeric keypad send special application codes
2.  **Normal Keypad Mode** (`ESC >`): Returns the numeric keypad to normal numeric operation

Sources: [testdata/TestSequences/keypad/normal\_keypad.golden](https://github.com/charmbracelet/sequin/blob/025641c1/testdata/TestSequences/keypad/normal_keypad.golden)

Integration in Sequin Architecture
----------------------------------

The terminal mode handling is part of Sequin's ANSI sequence processing system. It's registered in the handler table to process specific CSI sequences.

The mode handler is specifically responsible for interpreting terminal mode changes and presenting them in a human-readable format. It's part of a larger system that comprehensively decodes all types of ANSI sequences.

Sources: [mode.go9-30](https://github.com/charmbracelet/sequin/blob/025641c1/mode.go#L9-L30)
