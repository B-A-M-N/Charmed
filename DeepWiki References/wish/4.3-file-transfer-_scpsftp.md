File Transfer (SCP/SFTP)
========================

This document describes how to implement file transfer capabilities in SSH servers built with wish. The wish library provides support for the SCP (Secure Copy Protocol) which allows file transfers over SSH connections.

Overview
--------

The `scp` package in wish provides middleware for handling SCP commands, allowing SSH clients to securely copy files to and from the server. The implementation supports both uploading files to the server and downloading files from the server, with options for recursive transfers and directory operations.

Sources: [scp/scp.go1-306](https://github.com/charmbracelet/wish/blob/19f43236/scp/scp.go#L1-L306)
 [scp/filesystem.go1-134](https://github.com/charmbracelet/wish/blob/19f43236/scp/filesystem.go#L1-L134)

SCP Protocol Support
--------------------

The SCP middleware in wish supports two primary operations:

1.  **Copy to Client** (`scp -f`) - Transfers files from the server to the client
2.  **Copy from Client** (`scp -t`) - Transfers files from the client to the server

Both operations can be performed recursively with the `-r` flag to include directories and their contents.

Sources: [scp/scp.go58-90](https://github.com/charmbracelet/wish/blob/19f43236/scp/scp.go#L58-L90)

Core Components
---------------

### SCP Middleware

The SCP middleware is the central component that handles SCP commands. It identifies SCP operations based on command-line arguments and routes them to appropriate handlers.

Sources: [scp/scp.go58-90](https://github.com/charmbracelet/wish/blob/19f43236/scp/scp.go#L58-L90)

### Handler Interfaces

The SCP implementation defines several interfaces for handling file operations:

1.  **CopyToClientHandler** - For transferring files from server to client
2.  **CopyFromClientHandler** - For transferring files from client to server
3.  **Handler** - A combined interface supporting both directions

Sources: [scp/scp.go17-56](https://github.com/charmbracelet/wish/blob/19f43236/scp/scp.go#L17-L56)

### Entry Types

The SCP implementation uses several entry types to represent files and directories:

1.  **FileEntry** - Represents a file with its contents
2.  **DirEntry** - Represents a directory, which can contain other entries
3.  **RootEntry** - A special entry type that serves as the root of an entry hierarchy

These types implement the `Entry` and/or `AppendableEntry` interfaces.

Sources: [scp/scp.go96-243](https://github.com/charmbracelet/wish/blob/19f43236/scp/scp.go#L96-L243)

Built-in Handlers
-----------------

### FileSystem Handler

The `fileSystemHandler` provides an implementation of the `Handler` interface that works with the local file system. This handler allows direct access to files on the server's file system.

The file system handler provides implementations for all required methods:

*   File reading via `NewFileEntry` and `NewDirEntry`
*   File writing via `Write`
*   Directory creation via `Mkdir`
*   Path globbing via `Glob`
*   Directory traversal via `WalkDir`

Sources: [scp/filesystem.go15-133](https://github.com/charmbracelet/wish/blob/19f43236/scp/filesystem.go#L15-L133)

Usage Example
-------------

Here's a basic example of how to add SCP support to a wish SSH server:

With this setup, clients can use standard SCP commands to transfer files:

    # Download a file from the server
    scp -P 2222 user@server:/path/to/file.txt local-file.txt
    
    # Upload a file to the server
    scp -P 2222 local-file.txt user@server:/path/to/
    
    # Recursively download a directory
    scp -r -P 2222 user@server:/path/to/dir/ local-dir/
    
    # Recursively upload a directory
    scp -r -P 2222 local-dir/ user@server:/path/to/

### File Transfer Flow

The `copyToClient` and `copyFromClient` functions (not shown in the provided code) handle the actual transfer of files, implementing the SCP protocol which includes:

1.  Command acknowledgment bytes
2.  File/directory entry headers
3.  File content transfers
4.  Directory recursion

The SCP protocol is a text-based protocol that sends control information and binary data over the same channel, with specific byte sequences to indicate operations and statuses.

Limitations
-----------

1.  The current implementation supports SCP but not SFTP (SSH File Transfer Protocol), which provides more features like resuming transfers and directory listings.
    
2.  Although the section title mentions SFTP, the codebase primarily implements SCP. SFTP support may be planned for future versions.
    
3.  The file system handler operates directly on the server's file system without additional sandboxing, so care should be taken to ensure proper access controls are in place.
    

Security Considerations
-----------------------

When implementing file transfer capabilities in your SSH server:

1.  **Access Control**: Ensure that users can only access files they have permission to access.
    
2.  **Path Validation**: The implementation includes path normalization to prevent path traversal attacks.
    
3.  **Resource Limits**: Consider implementing resource limits to prevent excessive disk usage or denial of service attacks.
    
4.  **Authentication**: File transfer operations should be properly authenticated through SSH's authentication mechanisms.
    

Sources: [scp/scp.go299-305](https://github.com/charmbracelet/wish/blob/19f43236/scp/scp.go#L299-L305)

Summary
-------

The SCP middleware in wish provides a flexible framework for implementing file transfer capabilities in your SSH server. It supports both directions of file transfer and can work with the local file system or custom backends. By implementing the appropriate handler interfaces, you can customize how files are stored, accessed, and secured in your application.
