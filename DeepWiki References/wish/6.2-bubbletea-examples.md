Introduction
------------

Bubble Tea is a powerful framework for building terminal applications with rich, responsive user interfaces. The wish library provides seamless integration with Bubble Tea through its middleware system, allowing you to serve your TUI applications over SSH connections. This document demonstrates practical examples of this integration.

Best Practices
--------------

### Required Middleware Components

For optimal functionality of Bubble Tea applications over SSH, include these middlewares:

| Middleware | Purpose |
| --- | --- |
| `bubbletea.Middleware()` | Manages Bubble Tea programs within SSH sessions |
| `activeterm.Middleware()` | Ensures a PTY is active and available (required for TUI apps) |
| `logging.Middleware()` | Provides useful logging of SSH sessions |

### Handling Terminal Information

Always check for an active PTY before creating a Bubble Tea program:

Sources: [examples/bubbletea/main.go64-100](https://github.com/charmbracelet/wish/blob/19f43236/examples/bubbletea/main.go#L64-L100)
 [examples/bubbleteaprogram/main.go75-78](https://github.com/charmbracelet/wish/blob/19f43236/examples/bubbleteaprogram/main.go#L75-L78)

### Color Profile Handling

When running a Bubble Tea app over SSH, never use the default `lipgloss.NewStyle()` function as it will use the color profile from the server, not the client. Instead, use the `bubbletea.MakeRenderer()` function to get the correct renderer for the current session.

Sources: [examples/bubbletea/main.go72-83](https://github.com/charmbracelet/wish/blob/19f43236/examples/bubbletea/main.go#L72-L83)
