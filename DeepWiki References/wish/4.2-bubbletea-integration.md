Purpose and Overview
--------------------

The BubbleTea integration module enables running Terminal User Interface (TUI) applications over SSH connections using the [Bubble Tea](https://github.com/charmbracelet/wish/blob/19f43236/Bubble%20Tea)
 framework. This documentation covers how to integrate Bubble Tea apps with wish, handle terminal capabilities, manage window resize events, and properly configure terminal rendering based on client capabilities.

For general SSH server setup, see [Core SSH Server Framework](https://deepwiki.com/charmbracelet/wish/2-core-ssh-server-framework)
, or for file transfer capabilities, see [File Transfer (SCP/SFTP)](https://deepwiki.com/charmbracelet/wish/4.3-file-transfer-(scpsftp))
.

Middleware Function Types
-------------------------

The package provides three primary middleware functions:

| Function | Purpose | Signature |
| --- | --- | --- |
| `Middleware` | Standard middleware using a Handler | `func Middleware(handler Handler) wish.Middleware` |
| `MiddlewareWithColorProfile` | Specifies minimum color requirements | `func MiddlewareWithColorProfile(handler Handler, profile termenv.Profile) wish.Middleware` |
| `MiddlewareWithProgramHandler` | Advanced usage with direct program access | `func MiddlewareWithProgramHandler(handler ProgramHandler, profile termenv.Profile) wish.Middleware` |

Sources: [bubbletea/tea.go38-69](https://github.com/charmbracelet/wish/blob/19f43236/bubbletea/tea.go#L38-L69)

Handler Types
-------------

The BubbleTea middleware supports two types of handlers:

1.  **Handler**: A function that returns a model and program options.
    
2.  **ProgramHandler**: A function that returns a complete tea.Program.
    

The first approach is simpler and suitable for most use cases, while the second gives you more control over the program's lifecycle.

Sources: [bubbletea/tea.go17-36](https://github.com/charmbracelet/wish/blob/19f43236/bubbletea/tea.go#L17-L36)

Terminal Handling
-----------------

The middleware automatically handles several terminal-related aspects:

1.  **PTY Detection**: Checks for an active terminal and configures I/O appropriately
2.  **Window Size Events**: Captures resize events and forwards them as `tea.WindowSizeMsg`
3.  **Color Profile**: Detects and adapts to the client's terminal capabilities
4.  **Background Color**: Can detect whether the client has a light or dark background

Color Profile Handling
----------------------

The middleware automatically detects the client's terminal capabilities and configures the rendering appropriately. It can:

1.  Detect terminal type from the PTY information
2.  Query the background color to determine if it's dark or light
3.  Determine the color profile (TrueColor, ANSI256, ANSI, or ASCII)
4.  Enforce a minimum color profile if needed

Sources: [bubbletea/tea.go112-134](https://github.com/charmbracelet/wish/blob/19f43236/bubbletea/tea.go#L112-L134)

Best Practices
--------------

1.  **Always check for active terminal**: Use the `activeterm.Middleware()` to ensure clients have an active terminal.
2.  **Use MakeRenderer**: Always use `bubbletea.MakeRenderer()` instead of `lipgloss.NewStyle()` directly.
3.  **Handle window resize events**: Design your models to respond to `tea.WindowSizeMsg` for proper terminal resizing.
4.  **Consider color compatibility**: Be mindful of terminals with limited color support.
5.  **Cleanup resources**: The middleware handles basic cleanup, but ensure any additional resources are properly closed.

Sources: [examples/bubbletea/main.go64-99](https://github.com/charmbracelet/wish/blob/19f43236/examples/bubbletea/main.go#L64-L99)

Implementation Details
----------------------

When the middleware runs:

1.  It checks if the session has an active PTY
2.  It calls the handler to get a model and options (or a program)
3.  It sets up a goroutine to listen for window resize events
4.  It runs the tea.Program and handles its lifecycle
5.  When the program exits, it ensures proper cleanup before calling the next handler

The middleware integrates terminal capabilities detection to provide the best possible experience for users with different terminal emulators and configurations.
