Command Execution
=================

This page documents how command execution works within SSH sessions in the Wish library. Command execution is a core functionality that allows SSH clients to run commands on the server, whether interactively through a terminal or programmatically.

Overview
--------

The Wish library provides an abstraction over the standard Go `exec.Cmd` mechanism, tailored specifically for SSH sessions. This allows for seamless integration between SSH sessions and local command execution, handling both PTY (pseudo-terminal) and non-PTY scenarios.

Sources: [cmd.go11-35](https://github.com/charmbracelet/wish/blob/19f43236/cmd.go#L11-L35)
 [cmd\_test.go14-34](https://github.com/charmbracelet/wish/blob/19f43236/cmd_test.go#L14-L34)

### Creating Commands

Wish provides two main functions for creating command objects:

Error Handling
--------------

Error handling is an important aspect of command execution. When a command fails, you have several options:

1.  Return the error to the caller
2.  Use `wish.Fatal` to log the error and exit the session
3.  Use `sess.Exit(code)` to set a specific exit code

Sources: [cmd\_test.go76-78](https://github.com/charmbracelet/wish/blob/19f43236/cmd_test.go#L76-L78)

Summary
-------

Command execution in Wish provides a flexible, SSH-aware wrapper around Go's standard `exec.Cmd`. It automatically handles differences between PTY and non-PTY sessions, works across different platforms, and integrates well with command-line frameworks like Cobra and UI frameworks like Bubbletea.
