Server Configuration and Options
================================

Relevant source files

*   [README.md](https://github.com/charmbracelet/wish/blob/19f43236/README.md?plain=1)
    
*   [examples/simple/main.go](https://github.com/charmbracelet/wish/blob/19f43236/examples/simple/main.go)

This document covers the various configuration options available when creating an SSH server with the wish framework. These options allow you to customize everything from network settings and authentication methods to session behavior and middleware configuration.

For information about middleware implementation itself, see [Middleware System](https://deepwiki.com/charmbracelet/wish/3-middleware-system)
.

Configuration Overview
----------------------

The wish library provides a set of configuration options through functions that follow the naming pattern `With*`. These functions return `ssh.Option` values that can be passed to `wish.NewServer()` to configure your SSH server instance.

Sources: [options.go](https://github.com/charmbracelet/wish/blob/19f43236/options.go)

### Using Authorized Keys Files

The `WithAuthorizedKeys` option allows authentication using standard OpenSSH authorized\_keys files:

This option verifies if the connecting client's public key is present in the specified authorized\_keys file. If the file cannot be read or doesn't contain the client's key, authentication fails.

### Using Certificate Authorities

The `WithTrustedUserCAKeys` option implements certificate-based authentication:

This allows users to authenticate with certificates signed by trusted Certificate Authorities. The server will verify:

1.  The certificate is signed by one of the CA keys in the specified file
2.  The certificate is valid (not expired)
3.  The certificate's principals include the username being used to connect

Sources: [options.go99-170](https://github.com/charmbracelet/wish/blob/19f43236/options.go#L99-L170)
 [options\_test.go127-208](https://github.com/charmbracelet/wish/blob/19f43236/options_test.go#L127-L208)

Middleware Configuration
------------------------

The `WithMiddleware` option is central to wish's functionality, allowing you to compose multiple middlewares into a processing chain:

| Option | Description | File Reference |
| --- | --- | --- |
| `WithMiddleware(mw ...Middleware)` | Composes multiple middleware functions into a single SSH handler | [options.go51-65](https://github.com/charmbracelet/wish/blob/19f43236/options.go#L51-L65) |

**Important note**: Middlewares are composed from first to last in the function call, but executed from last to first in the middleware chain. This means the last middleware specified is the first one to process an incoming session.
