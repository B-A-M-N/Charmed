Building from Source
====================

This page provides comprehensive instructions for building VHS from source code. This is particularly useful for developers who want to contribute to VHS, make custom modifications, or build VHS on platforms without pre-compiled binaries. For installing pre-built binaries, see [Installation and Setup](https://deepwiki.com/charmbracelet/vhs/1.1-installation-and-setup)
.

Prerequisites
-------------

Before building VHS from source, you need to install several dependencies:

| Dependency | Purpose | Required Version |
| --- | --- | --- |
| Go  | Programming language | 1.16+ |
| ffmpeg | Video processing | Latest stable |
| ttyd | Terminal sharing | Latest stable |
| Git | Source code management | Latest stable |

### System-Specific Dependencies

VHS requires a browser to render the terminal. The default configuration uses Chromium/Chrome, which must be installed on your system.

#### Font Requirements

For proper terminal rendering, you may want to install various monospace fonts. The Docker configuration uses the following fonts:

*   Source Code Pro
*   DejaVu Sans Mono
*   Fira Code
*   Hack
*   IBM Plex Mono
*   Inconsolata
*   JetBrains Mono
*   Liberation Mono
*   Noto fonts
*   Roboto Mono
*   Ubuntu Mono

Sources: [Dockerfile4-20](https://github.com/charmbracelet/vhs/blob/f28239f3/Dockerfile#L4-L20)

Development Considerations
--------------------------

When building from source for development purposes, keep in mind:

1.  The `.gitignore` file lists several directories that are created during development and testing:
    
    *   `captures/` - Capture output
    *   `frames/` - Individual frames
    *   `tmp/` - Temporary files
    *   `dist/` - Distribution files
2.  Changes to system components should be tested carefully as they may affect multiple parts of the system.
    

Sources: [.gitignore1-12](https://github.com/charmbracelet/vhs/blob/f28239f3/.gitignore#L1-L12)

Troubleshooting
---------------

### Common Build Issues

| Issue | Potential Solution |
| --- | --- |
| Missing ffmpeg | Install ffmpeg using your system's package manager |
| Missing ttyd | Build ttyd from source or install via package manager |
| Browser automation fails | Ensure Chromium/Chrome is installed and accessible |
| Permission issues with Docker | Run Docker with appropriate permissions |
