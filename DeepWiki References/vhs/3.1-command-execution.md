Purpose and Scope
-----------------

This document explains how VHS executes commands from a tape file, simulating keyboard inputs and controlling terminal behavior to create recordings. We cover the command execution pipeline, how different types of commands are processed, and the mechanisms used to translate commands into terminal actions.

For information about the tape file format itself, see [Tape Format](https://deepwiki.com/charmbracelet/vhs/2-tape-format)
. For details on terminal emulation implementation, see [Terminal Emulation](https://deepwiki.com/charmbracelet/vhs/3.2-terminal-emulation)
.

Command Execution Overview
--------------------------

When VHS reads a tape file, it parses the commands and executes them sequentially. Command execution is the bridge between the parsed tape file and the actual terminal simulation that gets recorded.

Sources: [command.go20-34](https://github.com/charmbracelet/vhs/blob/f28239f3/command.go#L20-L34)

Command Types
-------------

VHS supports a variety of commands that can be categorized into several types:

| Category | Description | Examples |
| --- | --- | --- |
| Keyboard Input | Basic keystrokes | Type, Enter, Backspace, Space |
| Key Combinations | Multiple keys pressed together | Ctrl+C, Alt+Tab, Shift+Arrow |
| Control Commands | Control the recording process | Hide, Show |
| Setup Commands | Configure terminal settings | Set, Output |
| Timing Commands | Control timing | Sleep, Wait |
| Clipboard Operations | Interact with system clipboard | Copy, Paste |
| System Commands | Environment and requirements | Env, Require |

The `CommandFuncs` map links each command type to its specific executor function:

Conclusion
----------

The Command Execution system in VHS translates tape file commands into terminal actions. It uses a combination of keyboard simulation, terminal control, and configuration management to create realistic terminal recordings. The system is built around a central `Execute` function that delegates to specific command executors, with support for options like typing speed and repeat counts.
