For information about building and setting up a development environment, see [Building from Source](https://deepwiki.com/charmbracelet/vhs/6.1-building-from-source)

### GoReleaser Configuration

VHS uses GoReleaser for building and distributing releases. The configuration extends from a template in the Charmbracelet meta repository.

    includes:
      - from_url:
          url: charmbracelet/meta/main/goreleaser-vhs.yaml
    
    variables:
      main: "."
      description: "A tool for recording terminal GIFs"
      github_url: "https://github.com/charmbracelet/vhs"
      maintainer: "Maas Lalani <maas@charm.sh>"

Special File Handling
---------------------

VHS uses specific file handling configurations for various file types:

| File Type | Handling |
| --- | --- |
| `.gif`, `.mp4`, `.webm`, `.png` | Managed with Git LFS |
| `.tape` | Linguistically classified as Elixir (for syntax highlighting) |
| `themes*.json`, `THEMES.md` | Marked as generated files |

Component Responsibilities
--------------------------

The following table summarizes the responsibilities of key components in the VHS codebase:

| Component | Location | Responsibility |
| --- | --- | --- |
| CLI | `cmd/vhs/` | Entry point for command-line interface |
| Parser | `pkg/parser/` | Parses tape files into executable commands |
| Lexer | `pkg/lexer/` | Tokenizes tape file content |
| TTYD Manager | `pkg/ttyd/` | Manages the ttyd process for terminal emulation |
| Recorder | `pkg/recorder/` | Handles recording frames from the terminal |
| Publisher | `pkg/publisher/` | Manages uploading and sharing GIFs |
| Command Executor | Internal | Executes commands on the virtual terminal |
| Themes | `pkg/themes/` | Manages terminal visual themes |

Contributing to VHS
-------------------

When contributing to VHS, it's important to understand where to make changes based on the component you're working with. The modular structure means that most features are isolated in their own packages, making the codebase easier to navigate and modify.
