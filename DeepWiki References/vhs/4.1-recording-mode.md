Recording Mode
==============

Recording Mode is a feature in VHS that allows users to capture their terminal interactions and automatically generate a tape file. Instead of manually writing VHS tape commands, users can simply perform the actions they want to record, and VHS will convert those keystrokes and commands into a proper tape file format.

Purpose and Functionality
-------------------------

Recording Mode addresses a common challenge with creating terminal recordings: writing out every command, keystroke, and timing can be tedious. By providing a "what you see is what you get" approach, Recording Mode makes it easy to create complex tape files without needing to understand the tape syntax in detail.

For information about the tape format itself, see [Tape Format](https://deepwiki.com/charmbracelet/vhs/2-tape-format)
.

Sources: [record.go65-71](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L65-L71)

Usage
-----

To use Recording Mode, simply run the `record` command and redirect its output to a file:

This starts a new terminal session where everything you type is recorded. When you're finished, simply exit the shell (type `exit` or press Ctrl+D), and VHS will output the generated tape file.

The recording environment sets a special environment variable `VHS_RECORD=true` that applications can detect to modify their behavior during recording.

Sources: [record.go72-75](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L72-L75)

How Recording Mode Works
------------------------

Recording Mode creates a pseudo-terminal (PTY) that captures all input and converts it into VHS tape commands.

### Recording Mode Architecture

Sources: [record.go65-123](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L65-L123)
 [record.go20-22](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L20-L22)

### Process Flow

1.  Start a shell process in a pseudo-terminal
2.  Set the terminal to raw mode
3.  Capture all user input and terminal output
4.  Monitor for pauses in activity to insert `Sleep` commands
5.  When the shell exits, process the captured input
6.  Convert terminal escape sequences to VHS commands
7.  Consolidate repeated commands to improve readability
8.  Format and output the final tape file

Sources: [record.go72-123](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L72-L123)

### Input Processing and Tape Generation

The `inputToTape` function handles the conversion of raw terminal input to a structured tape file:

Key processing steps:

1.  Remove the terminal `exit` command from the end of the input
2.  Clean up terminal control sequences that aren't relevant
3.  Replace escape sequences with their VHS command equivalents
4.  Compress repeated commands (e.g., multiple key presses)
5.  Format commands with appropriate arguments and options

Sources: [record.go131-201](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L131-L201)

Advanced Features
-----------------

### Command Compression

VHS automatically compresses repeated commands in the generated tape file. For example, pressing the Left arrow key 5 times is represented as `Left 5` rather than five separate `Left` commands.

This makes the resulting tape file more concise and readable.

Sources: [record.go154-163](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L154-L163)

### Special Character Handling

VHS uses different quoting styles to handle special characters in the input:

This ensures that strings containing quotes or other special characters are properly escaped in the generated tape file.

Sources: [record.go203-212](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L203-L212)

Limitations
-----------

*   The exit detection is simple and can be fooled if you type characters that look like "exit" but aren't (e.g., typing "exi" then backspace then "t")
*   Some complex terminal applications may not record perfectly due to their use of advanced terminal features
*   Custom keyboard shortcuts or locale-specific characters might not be correctly mapped

Sources: [record.go134-138](https://github.com/charmbracelet/vhs/blob/f28239f3/record.go#L134-L138)

Example Output
--------------

A simple session where a user types "echo hello", presses Enter, waits briefly, then types "ls" might generate a tape file like:

    Type "echo hello"
    Enter
    Sleep 500ms
    Type "ls"
    Enter
    

This tape file can then be used with the main VHS command to generate a GIF or other output format.
