This document explains how VHS captures frames from the terminal sessions and processes them into output videos (GIF, MP4, WebM). It covers the frame recording mechanism, the rendering pipeline, and the various options that control this process. For information about how VHS controls the terminal itself, see [Terminal Emulation](https://deepwiki.com/charmbracelet/vhs/3.2-terminal-emulation)
.

Frame Capture Mechanism
-----------------------

The frame capture process is one of the most critical components of VHS. It continuously takes snapshots of the terminal's state at regular intervals.

Sources: [vhs.go323-388](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L323-L388)
 [evaluator.go113-138](https://github.com/charmbracelet/vhs/blob/f28239f3/evaluator.go#L113-L138)

### How Frames Are Captured

VHS captures two separate canvas elements from the terminal:

1.  **Text Canvas**: Contains the text content of the terminal
2.  **Cursor Canvas**: Contains only the cursor element

This separation allows for proper layering and composition during the rendering phase. Each frame is saved in PNG format with a sequential numbering scheme:

*   Text frames: `frame-text-00001.png`, `frame-text-00002.png`, etc.
*   Cursor frames: `frame-cursor-00001.png`, `frame-cursor-00002.png`, etc.

The frame capture rate is determined by the `Framerate` option, which defaults to 50 frames per second.

Sources: [vhs.go323-388](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L323-L388)
 [video.go21-22](https://github.com/charmbracelet/vhs/blob/f28239f3/video.go#L21-L22)

Loop Offset Feature
-------------------

The Loop Offset feature helps create seamlessly looping GIFs by moving a percentage of frames from the beginning to the end of the sequence.

Sources: [vhs.go249-318](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L249-L318)

### Implementation Details

The `ApplyLoopOffset` method performs the following steps:

1.  Calculates the number of frames to offset based on the percentage and total frame count
2.  Adjusts the starting frame number for rendering
3.  Renames the frame files to create the modified sequence

This process is handled concurrently to improve performance when dealing with large numbers of frames.

Sources: [vhs.go249-318](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L249-L318)

Rendering Pipeline
------------------

After capturing frames, VHS processes them into various output formats using ffmpeg.

### ffmpeg Command Generation

VHS dynamically builds ffmpeg commands based on the desired output format and styling options. The process involves:

1.  Creating input streams for the text and cursor frame sequences
2.  Adding streams for styling elements (margins, window bars, corners)
3.  Building filter chains to compose the elements and apply format-specific processing
4.  Executing the final command to generate the output

The `buildFFopts` function is central to this process, constructing the command arguments that will be passed to ffmpeg.

Sources: [video.go92-136](https://github.com/charmbracelet/vhs/blob/f28239f3/video.go#L92-L136)

Configurable Options
--------------------

VHS provides numerous options to control the recording and rendering process:

| Option | Description | Default |
| --- | --- | --- |
| `Framerate` | Frames captured/rendered per second | 50  |
| `PlaybackSpeed` | Speed multiplier for the output video | 1.0 |
| `MaxColors` | Maximum colors in the GIF palette | 256 |
| `LoopOffset` | Percentage of frames to move for loop smoothing | 0   |
| `TypingSpeed` | Delay between typed characters | 50ms |

### Style Options

VHS also provides extensive styling options that affect the rendering process:

| Style Option | Description | Default |
| --- | --- | --- |
| `Width` | Output video width | 1200 |
| `Height` | Output video height | 600 |
| `Padding` | Terminal padding | 60  |
| `Margin` | Outer margin size | 0   |
| `BorderRadius` | Corner radius for terminal | 0   |
| `WindowBar` | Type of window bar to display | "" (none) |
| `WindowBarSize` | Height of window bar | 30  |

Render Method Implementation
----------------------------

The `Render` method orchestrates the entire rendering process:

1.  Applies any configured loop offset to the frame sequence
2.  Creates ffmpeg commands for each requested output format
3.  Executes the commands to generate the output files
4.  Handles any errors that occur during rendering

This method is called after all frames have been captured, and it produces the final output files (GIF, MP4, WebM, screenshots).

Sources: [vhs.go222-247](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L222-L247)
 [evaluator.go184-187](https://github.com/charmbracelet/vhs/blob/f28239f3/evaluator.go#L184-L187)

System Integration
------------------

The recording and rendering system integrates with other components of VHS:

*   The **Command Execution** system triggers frame recording start/stop
*   The **Terminal Emulation** system provides the canvas elements to capture
*   The **Evaluator** manages the recording and rendering lifecycle

When using VHS through SSH or as a server, the rendering process works the same way but outputs to temporary files that are then transmitted to the client.
