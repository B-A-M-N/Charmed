Purpose and Scope
-----------------

This document details how VHS implements terminal emulation to create high-fidelity recordings of terminal sessions. It explains the architecture that enables VHS to generate, control, and capture terminal sessions using ttyd and browser automation. For information about how VHS executes commands within this terminal environment, see [Command Execution](https://deepwiki.com/charmbracelet/vhs/3.1-command-execution)
. For details on how terminal output is recorded and rendered into videos, see [Recording and Rendering](https://deepwiki.com/charmbracelet/vhs/3.3-recording-and-rendering)
.

Terminal Emulation Architecture
-------------------------------

VHS uses a combination of ttyd (a terminal sharing tool) and browser automation to create a controllable, recordable terminal environment. The system doesn't rely on traditional terminal emulators like xterm or iTerm2, but instead creates a web-based terminal that can be manipulated programmatically.

Sources: [vhs.go22-36](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L22-L36)
 [tty.go19-46](https://github.com/charmbracelet/vhs/blob/f28239f3/tty.go#L19-L46)

Terminal Initialization Process
-------------------------------

The VHS terminal emulation is initialized through a sequence of steps that create and connect to a web-based terminal:

1.  **Port Selection**: A random available port is selected for ttyd
2.  **TTY Process**: The ttyd process is started on the selected port
3.  **Browser Launch**: A headless browser is started using the Rod library
4.  **Terminal Connection**: The browser connects to the ttyd URL
5.  **Terminal Configuration**: Font family, size, theme, and other settings are applied

Sources: [vhs.go124-156](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L124-L156)
 [tty.go19-24](https://github.com/charmbracelet/vhs/blob/f28239f3/tty.go#L19-L24)

### TTY Initialization

The terminal is created using ttyd, which provides a web interface to a terminal. VHS starts ttyd with specific options to make it suitable for recording:

    ttyd --port={random_port} --interface 127.0.0.1 -t rendererType=canvas -t disableResizeOverlay=true -t enableSixel=true -t customGlyphs=true --once --writable {shell_command}
    

The key options include:

*   `--interface 127.0.0.1` - Only accepts local connections
*   `--once` - Allows only one connection before exiting
*   `--writable` - Enables write access to the terminal
*   `-t rendererType=canvas` - Uses canvas rendering for better capture quality

Sources: [tty.go27-46](https://github.com/charmbracelet/vhs/blob/f28239f3/tty.go#L27-L46)

Terminal Interaction and Capture
--------------------------------

### Terminal Canvases

The terminal rendering is split across two HTML canvas elements that VHS captures separately:

1.  **Text Canvas** (`xterm-text-layer`): Contains all the text content
2.  **Cursor Canvas** (`xterm-cursor-layer`): Contains only the cursor

This separation allows VHS to manipulate the text and cursor independently during processing.

Sources: [vhs.go177-178](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L177-L178)

Terminal Manipulation
---------------------

VHS manipulates the terminal through several methods:

1.  **Direct Input**: Sending keystrokes via browser automation
2.  **Visibility Control**: Pausing and resuming the recording process
3.  **State Inspection**: Reading terminal buffer content for commands like `Wait`

Technical Implementation Notes
------------------------------

### Browser Automation

VHS uses the Rod library (a Devtools Protocol client) to control the browser, which connects to the ttyd web interface. This provides programmatic control of the terminal while allowing high-quality rendering through the browser.

Sources: [vhs.go17-20](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L17-L20)
 [vhs.go139-154](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L139-L154)

### Default Configuration

VHS sets sensible defaults for the terminal emulation including:

*   Font size: 22px
*   Font family: A fallback chain including "JetBrains Mono", "DejaVu Sans Mono", and other monospace fonts
*   Cursor blinking: Enabled
*   Line height: 1.0
*   Letter spacing: 1.0

These defaults ensure consistent and readable terminal recordings.

Sources: [vhs.go57-110](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L57-L110)

### Clean Termination

When recording is complete, VHS properly cleans up the terminal emulation resources:

1.  Terminates the browser instance
2.  Kills the ttyd process
3.  Cleans up temporary frame files if needed

Sources: [vhs.go198-220](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L198-L220)
