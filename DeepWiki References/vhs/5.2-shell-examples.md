Introduction
------------

This page documents how to use different shell configurations with VHS. VHS supports specifying alternate shells for your terminal recordings, allowing you to create demonstrations tailored to specific shell environments. For general information about VHS commands, see [Commands Reference](https://deepwiki.com/charmbracelet/vhs/2.1-commands-reference)
.

Shell Configuration Overview
----------------------------

VHS uses the `Set Shell` command to specify which shell environment to use for your recording session. By default, VHS uses your system's default shell, but you can explicitly configure it to use alternative shells like Oil Shell (OSH), Xonsh, Bash, Zsh, Fish, and others.

Sources: [examples/settings/set-shell-osh.tape1-11](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/settings/set-shell-osh.tape#L1-L11)
 [examples/settings/set-shell-xonsh.tape1-11](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/settings/set-shell-xonsh.tape#L1-L11)

Using the `Set Shell` Command
-----------------------------

The `Set Shell` command is straightforward to use in your tape files:

    Set Shell <shell-name>
    

Where `<shell-name>` is the name or path of the shell you want to use. This command should be placed at the beginning of your tape file, typically after any output configuration.

Example Shells
--------------

VHS supports a variety of shells. Here are examples of some alternative shells you can use:

### Oil Shell (OSH)

Oil Shell is a bash-compatible shell with modern features:

    Output examples/settings/set-shell-osh.gif
    
    Set FontSize 38
    Set Height 225
    
    Set Shell osh
    
    Sleep 1s
    Type "I am using OSH."
    Sleep 2s
    

Sources: [examples/settings/set-shell-osh.tape1-11](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/settings/set-shell-osh.tape#L1-L11)

### Xonsh Shell

Xonsh is a Python-powered shell language:

    Output examples/settings/set-shell-xonsh.gif
    
    Set FontSize 38
    Set Height 225
    
    Set Shell xonsh
    
    Sleep 1s
    Type "I am using xonsh."
    Sleep 2s
    

Sources: [examples/settings/set-shell-xonsh.tape1-11](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/settings/set-shell-xonsh.tape#L1-L11)

Comparison of Shell Options
---------------------------

The table below lists common shells that can be used with VHS:

| Shell Name | Command | Description | Special Considerations |
| --- | --- | --- | --- |
| Bash | `Set Shell bash` | Default UNIX shell on many systems | Widely compatible |
| Zsh | `Set Shell zsh` | Extended Bourne shell with improvements | May have different completion behavior |
| Fish | `Set Shell fish` | User-friendly shell with enhanced features | Syntax differs from POSIX shells |
| OSH | `Set Shell osh` | Bash-compatible shell with modern features | Requires Oil Shell installation |
| Xonsh | `Set Shell xonsh` | Python-powered shell | Combines Python and shell syntax |
| PowerShell | `Set Shell pwsh` | Modern shell and scripting language from Microsoft | Windows native commands differ |

Shell-Specific Considerations
-----------------------------

When using different shells in your VHS recordings, keep these considerations in mind:

1.  **Availability**: The shell must be installed and available in the PATH of the system running VHS.
2.  **Initialization**: Different shells may have different startup scripts (e.g., `.bashrc`, `.zshrc`).
3.  **Syntax differences**: Commands that work in one shell may not work in others.
4.  **Prompt differences**: Shell prompts may vary, affecting the visual appearance of your recording.
5.  **Timing**: Different shells may have different startup times, which may require adjusting sleep durations.

Advanced Usage
--------------

### Using a Shell with Custom Path

If the shell you want to use is not in the PATH or has a custom name, you can specify the full path:

    Set Shell /usr/local/bin/custom-shell
    

### Shell with Arguments

Some shells can accept startup arguments:

    Set Shell "zsh --no-rcs"
    

This example starts Zsh without loading any resource/startup files.

Best Practices
--------------

1.  **Test your shell environment** before creating the final recording to ensure all commands work as expected.
2.  **Consider portability** if your tape files will be run on different systems.
3.  **Document shell-specific features** if your recording demonstrates functionality that's unique to a particular shell.
4.  **Use appropriate timing** as different shells may process commands at different speeds.
