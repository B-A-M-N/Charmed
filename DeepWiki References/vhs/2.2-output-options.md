This page documents the various output formats and configuration options available in VHS for rendering your terminal recordings. VHS offers multiple output formats and extensive customization options to tailor the appearance and behavior of your recordings. For information about styling the terminal appearance itself (themes, borders, etc.), see [Themes and Styling](https://deepwiki.com/charmbracelet/vhs/2.3-themes-and-styling)
.

Overview of Output Capabilities
-------------------------------

VHS can generate recordings in three primary video formats and one frame sequence format:

*   GIF: Animated GIF format (most commonly used)
*   MP4: H.264 video format
*   WebM: Open web video format
*   Frames: PNG sequence of individual frames

You can specify one or multiple output formats for a single recording session, allowing you to generate different formats simultaneously from the same tape.

The Output Command
------------------

The `Output` command in VHS specifies where and in what format the recording should be saved. This command must be declared at the beginning of your tape file, before any non-setting commands.

### Basic Syntax

    Output <path>.<format>
    

Where:

*   `<path>` is the file path where the output should be saved
*   `<format>` determines the output format (gif, mp4, webm, or directory path for frames)

### Multiple Outputs

You can specify multiple outputs in a single tape file:

    Output demo.gif
    Output demo.mp4
    Output demo.webm
    Output frames/
    

This will generate all specified formats from a single recording session. Each format uses the same sequence of captured frames, processed differently by FFmpeg.

Output Format Details
---------------------

Each output format has different characteristics that make it suitable for various use cases:

### GIF Format

GIF is the default and most common output format for VHS. It's widely supported across platforms and websites, making it ideal for documentation, readme files, and social media.

Characteristics:

*   File extension: `.gif`
*   Supports animation with looping
*   Limited color palette (configurable with MaxColors setting)
*   Larger file size than video formats for the same content
*   No audio support
*   Universal compatibility

### MP4 Format

MP4 provides better quality and compression than GIF, making it suitable for longer recordings or when file size is a concern.

Characteristics:

*   File extension: `.mp4`
*   H.264 video codec
*   Smaller file size than GIF for the same content
*   High visual quality
*   Supported by most modern web browsers and platforms

### WebM Format

WebM is an open video format optimized for the web with excellent compression.

Characteristics:

*   File extension: `.webm`
*   VP9 video codec
*   Often smaller than MP4 at similar quality
*   Good support in modern browsers
*   Open format (no licensing fees)

### Frames Directory

You can output the raw frame sequence as individual PNG files, which is useful for custom post-processing.

Characteristics:

*   Specified as a directory path
*   Produces numbered PNG files
*   Highest quality (lossless)
*   Allows for manual frame editing or custom processing

Output Configuration Options
----------------------------

Various settings affect how your outputs are generated. These settings should be placed at the beginning of your tape file, along with your `Output` commands.

Sources: [video.go41-59](https://github.com/charmbracelet/vhs/blob/f28239f3/video.go#L41-L59)
 [vhs.go53-55](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L53-L55)

### Framerate Control

#### Set Framerate

Sets the capture framerate of the recording:

    Set Framerate 60
    

Higher framerates result in smoother animations but larger file sizes. The default is 50 frames per second.

#### Set PlaybackSpeed

Controls the speed of the final output:

    Set PlaybackSpeed 0.5  # Make output 2 times slower
    Set PlaybackSpeed 1.0  # Keep output at normal speed (default)
    Set PlaybackSpeed 2.0  # Make output 2 times faster
    

This is particularly useful when demonstrating complex sequences that would be too fast at normal speed, or for speeding up slow processes.

#### Set LoopOffset

Controls where the GIF loop should begin, allowing you to set a more interesting initial frame:

    Set LoopOffset 5    # Start the GIF at the 5th frame
    Set LoopOffset 50%  # Start the GIF halfway through
    

This is particularly important for GIFs, as the first frame is often used as a preview when embedded in documentation or websites.

### Visual Styling Options

Several settings affect the visual appearance of the output:

#### Set Margin and MarginFill

Adds a colored or image margin around the terminal:

    Set Margin 60
    Set MarginFill "#6B50FF"
    

This creates space around the terminal in the final output. `MarginFill` can be a color code or an image file.

#### Set BorderRadius

Rounds the corners of the terminal window:

    Set BorderRadius 10
    

Best used in combination with `Margin` and `MarginFill`.

#### Set WindowBar

Adds a decorative bar to the top of the terminal window:

    Set WindowBar Colorful
    

Options include:

*   Colorful
*   ColorfulRight
*   Rings
*   RingsRight

#### Set WindowBarSize

Controls the height of the window bar in pixels:

    Set WindowBarSize 40  # Default is 40

Technical Details of Output Generation
--------------------------------------

VHS generates video outputs through a multi-step process:

1.  Frames are captured from the terminal session using browser automation
2.  Text and cursor layers are captured separately as PNG files
3.  Frames are processed using FFmpeg with various filters based on your settings
4.  The final output is rendered in the specified format(s)

### Frame Capturing Process

VHS captures two layers for each frame:

*   Text canvas: Contains the text content
*   Cursor canvas: Contains the cursor position

These layers are combined during the rendering process to create the final output frames.

Sources: [vhs.go321-388](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L321-L388)
 [vhs.go249-318](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L249-L318)

### FFmpeg Command Generation

VHS uses FFmpeg to process the captured frames into the final video output. The FFmpeg commands are constructed dynamically based on your configuration settings:

1.  Input streams are set up for text and cursor layers
2.  Filters are applied based on style settings (margins, border radius, etc.)
3.  Format-specific options are added (GIF palette, video codec settings, etc.)
4.  The final command is executed to generate the output

The command generation is handled by the `buildFFopts` function, which assembles the appropriate FFmpeg arguments based on the target format and other options.

Best Practices
--------------

1.  **File Size Optimization**:
    
    *   Use MP4 or WebM for longer recordings to reduce file size
    *   For GIFs, consider reducing the capture framerate
    *   Use `PlaybackSpeed` to speed up slow sections
2.  **Visual Quality**:
    
    *   Use suitable dimensions for your target platform
    *   Add margin and border radius for a more polished look
    *   Consider window bars for a more realistic terminal appearance
3.  **Looping Considerations**:
    
    *   For GIFs, use `LoopOffset` to ensure the most interesting frame is shown first
    *   For demonstrations, ensure the loop transitions smoothly
4.  **Flexibility**:
    
    *   Generate multiple formats to have options for different platforms
    *   Use the frame sequence output for custom post-processing

Implementation Notes
--------------------

The output generation in VHS is primarily handled by the following components:

1.  The `VideoOptions` struct in [video.go50-59](https://github.com/charmbracelet/vhs/blob/f28239f3/video.go#L50-L59)
     which defines the configuration for video rendering
2.  The `VideoOutputs` struct in [video.go41-48](https://github.com/charmbracelet/vhs/blob/f28239f3/video.go#L41-L48)
     which maps output formats to file paths
3.  The `Render` method in [vhs.go222-247](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L222-L247)
     which triggers the video generation process
4.  The format-specific generator functions in [video.go139-185](https://github.com/charmbracelet/vhs/blob/f28239f3/video.go#L139-L185)
     which create the appropriate FFmpeg commands

All of these components work together to transform the captured terminal frames into the final output formats based on the settings specified in your tape file.

The loop offset feature, which is particularly useful for GIFs, is implemented in the `ApplyLoopOffset` method in [vhs.go249-318](https://github.com/charmbracelet/vhs/blob/f28239f3/vhs.go#L249-L318)
 This function rearranges the frame sequence to start from a more interesting point in the recording.
