Installation and Setup
======================

Relevant source files

*   [.gitignore](https://github.com/charmbracelet/vhs/blob/f28239f3/.gitignore)
    
*   [Dockerfile](https://github.com/charmbracelet/vhs/blob/f28239f3/Dockerfile)

This page provides comprehensive instructions for installing VHS and its dependencies on various platforms. VHS requires specific system dependencies to function correctly, and this guide covers all supported installation methods. For information about using VHS after installation, see [Quick Start](https://deepwiki.com/charmbracelet/vhs/1.2-quick-start)
.

System Requirements
-------------------

VHS has two primary dependencies that must be installed and available in your system PATH:

1.  **ttyd** - A terminal sharing tool that VHS uses for terminal emulation
2.  **ffmpeg** - Used for video processing and GIF generation

Sources: [README.md77-78](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L77-L78)

Installation Methods
--------------------

VHS can be installed through several methods depending on your platform and preferences.

Sources: [README.md80-110](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L80-L110)

### Package Managers

#### macOS and Linux with Homebrew

    brew install vhs
    

#### Arch Linux

    pacman -S vhs
    

#### NixOS

    nix-env -iA nixpkgs.vhs
    

#### Windows with Scoop

    scoop install vhs
    

Sources: [README.md82-93](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L82-L93)

### Docker

Docker provides a convenient way to run VHS without installing dependencies:

    docker run --rm -v $PWD:/vhs ghcr.io/charmbracelet/vhs <cassette>.tape
    

The Docker image includes all necessary dependencies, including fonts, ttyd, ffmpeg, and chromium.

Sources: [README.md96-99](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L96-L99)
 [Dockerfile1-57](https://github.com/charmbracelet/vhs/blob/f28239f3/Dockerfile#L1-L57)

### Binary Downloads

Pre-built binaries and packages are available for various platforms:

*   **Packages**: Available in Debian (.deb) and RPM formats
*   **Binaries**: Available for Linux, macOS, and Windows

Download the appropriate binary or package from the [releases page](https://github.com/charmbracelet/vhs/blob/f28239f3/releases%20page)

Sources: [README.md102-106](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L102-L106)

### Installing from Source with Go

If you have Go installed, you can build VHS from source:

    go install github.com/charmbracelet/vhs@latest
    

Sources: [README.md108-110](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L108-L110)

Platform-Specific Installation
------------------------------

### Debian/Ubuntu

    sudo mkdir -p /etc/apt/keyrings
    curl -fsSL https://repo.charm.sh/apt/gpg.key | sudo gpg --dearmor -o /etc/apt/keyrings/charm.gpg
    echo "deb [signed-by=/etc/apt/keyrings/charm.gpg] https://repo.charm.sh/apt/ * *" | sudo tee /etc/apt/sources.list.d/charm.list
    # Install ttyd from https://github.com/tsl0922/ttyd/releases
    sudo apt update && sudo apt install vhs ffmpeg
    

### Fedora/RHEL

    echo '[charm]
    name=Charm
    baseurl=https://repo.charm.sh/yum/
    enabled=1
    gpgcheck=1
    gpgkey=https://repo.charm.sh/yum/gpg.key' | sudo tee /etc/yum.repos.d/charm.repo
    # Install ttyd from https://github.com/tsl0922/ttyd/releases
    sudo yum install vhs ffmpeg
    

### Void Linux

    sudo xbps-install vhs
    

### Windows

    winget install charmbracelet.vhs
    # or
    scoop install vhs
    

Sources: [README.md113-154](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L113-L154)

Installation Verification
-------------------------

To verify that VHS and its dependencies are correctly installed:

1.  **Check VHS installation**:
    
        vhs --version
        
    
2.  **Verify dependencies**:
    
        which ttyd
        which ffmpeg
        
    
3.  **Create a simple test tape**:
    
        vhs new test.tape
        
    
4.  **Run the test tape**:
    
        vhs test.tape
        
    

If all steps succeed, VHS is properly installed and configured.

Docker Environment Variables
----------------------------

When using the Docker container, you can configure VHS with these environment variables:

| Variable | Default | Description |
| --- | --- | --- |
| VHS\_PORT | 1976 | The port for VHS server to listen on |
| VHS\_HOST | 0.0.0.0 | The host for VHS server to bind to |
| VHS\_GID | 1976 | The group ID to run the server as |
| VHS\_UID | 1976 | The user ID to run the server as |
| VHS\_KEY\_PATH | /vhs/vhs | The path to the SSH key |
| VHS\_AUTHORIZED\_KEYS\_PATH | (empty) | Path to authorized keys file |
| VHS\_NO\_SANDBOX | true | Disables Chrome sandbox for containerized environments |

Sources: [Dockerfile49-55](https://github.com/charmbracelet/vhs/blob/f28239f3/Dockerfile#L49-L55)

Troubleshooting Common Installation Issues
------------------------------------------

### Missing Dependencies

If you encounter errors related to missing dependencies:

1.  **ttyd not found**: Install ttyd from the [ttyd GitHub releases page](https://github.com/charmbracelet/vhs/blob/f28239f3/ttyd%20GitHub%20releases%20page)
    
2.  **ffmpeg not found**: Install ffmpeg using your system's package manager:
    *   macOS: `brew install ffmpeg`
    *   Debian/Ubuntu: `sudo apt install ffmpeg`
    *   Fedora/RHEL: `sudo yum install ffmpeg`
    *   Windows: `scoop install ffmpeg` or download from [ffmpeg.org](https://ffmpeg.org/)
        

### Browser Automation Issues

VHS uses browser automation to control the terminal emulator. If you encounter browser-related errors:

1.  Ensure you have sufficient permissions
2.  Try running with the `VHS_NO_SANDBOX=true` environment variable
3.  For Docker installations, this is already configured

Sources: [README.md77-78](https://github.com/charmbracelet/vhs/blob/f28239f3/README.md?plain=1#L77-L78)
 [Dockerfile40](https://github.com/charmbracelet/vhs/blob/f28239f3/Dockerfile#L40-L40)
