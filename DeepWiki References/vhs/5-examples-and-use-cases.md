Purpose and Scope
-----------------

This page provides a comprehensive collection of example tape files and common use cases for VHS. These examples demonstrate how to create effective terminal recordings for various scenarios, from simple command demonstrations to complex interactive sessions.

For information on the tape file format and available commands, see [Tape Format](https://deepwiki.com/charmbracelet/vhs/2-tape-format)
 and [Commands Reference](https://deepwiki.com/charmbracelet/vhs/2.1-commands-reference)
. For styling options, see [Themes and Styling](https://deepwiki.com/charmbracelet/vhs/2.3-themes-and-styling)
.

Common Use Cases
----------------

VHS can be used in a variety of contexts:

*   **Tool Demos**: Create demos for CLI tools to showcase features
*   **Documentation**: Generate visual examples for technical documentation
*   **Tutorials**: Build step-by-step guides with visual feedback
*   **Marketing**: Create appealing GIFs for product announcements
*   **Testing**: Visual verification of terminal-based applications

Sources: [examples/README.md](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/README.md?plain=1)

| --- | --- | --- |

Common Patterns and Best Practices
----------------------------------

Based on the examples provided, here are some common patterns and best practices for creating effective VHS recordings:

### Output Format Patterns

| Pattern | Description | Example |
| --- | --- | --- |
| Multiple formats | Specify multiple output formats | `Output demo.gif`  <br>`Output demo.mp4` |
| Frame capture | Save individual frames | `Output frames/` |
| ASCII output | Generate text-only output | `Output demo.ascii` |

### Timing and Pacing Patterns

| Pattern | Description | Example |
| --- | --- | --- |
| Natural pauses | Add pauses between actions | `Sleep 1s` |
| Variable typing speed | Adjust typing speed for different sections | `Type@150ms "text"` |
| Timed key presses | Control the speed of repeated keys | `Down@200ms 10` |

### Interaction Patterns

| Pattern | Description | Example |
| --- | --- | --- |
| Basic command | Simple command execution | `Type "command"`  <br>`Enter` |
| Navigation | Moving through UI elements | `Up/Down/Left/Right` keys |
| Text editing | Demonstrating text modification | `Ctrl+A`, `Backspace` |
| Hidden setup | Hide preparation commands | `Hide`  <br>_commands_  <br>`Show` |

### Visual Enhancement Patterns

| Pattern | Description | Example |
| --- | --- | --- |
| Terminal customization | Set dimensions and appearance | `Set Width 1300`  <br>`Set Height 650`  <br>`Set FontSize 22` |
| Framerate control | Adjust animation smoothness | `Set Framerate 40` |
| Typing speed | Control pace of typing | `Set TypingSpeed 75ms` |
| Padding | Add spacing around content | `Set Padding 20` |

Conclusion
----------

The examples provided in the VHS repository demonstrate various use cases and techniques for creating effective terminal recordings. From simple command demonstrations to complex interactive applications, VHS offers a flexible system for recording and sharing terminal sessions.

To create your own VHS recordings, you can use these examples as templates, adapting them to your specific needs. The patterns and techniques shown here can be combined and extended to create compelling visual documentation for terminal applications and workflows.
