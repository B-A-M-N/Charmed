This page documents advanced techniques for creating sophisticated and professional VHS tape scripts. While the [Commands Reference](https://deepwiki.com/charmbracelet/vhs/2.1-commands-reference)
 covers individual command syntax and [Tape Format](https://deepwiki.com/charmbracelet/vhs/2-tape-format)
 explains the fundamentals, this section focuses on combining commands and leveraging special features to create complex, polished terminal recordings.

Command Composition and Flow Control
------------------------------------

### Nested Tape Execution

One powerful technique in VHS is the ability to execute tape files from within other tape files, enabling modular design and reuse.

Example from meta.tape:

    Type "vhs < cassette.tape" Sleep 0.5 Enter
    

This technique allows you to:

*   Create reusable tape components
*   Chain multiple recordings together
*   Create meta-demonstrations of VHS itself

Sources: [examples/meta.tape14](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.tape#L14-L14)

### Strategic Visibility Control

Professional recordings typically hide setup and cleanup phases from the final output.

Example usage pattern:

    Hide
    Type "cp examples/welcome.tape cassette.tape && clear" Enter
    Show
    Type "vhs < cassette.tape" Enter
    Sleep 30s
    Hide
    Type "rm cassette.tape" Enter
    

Sources: [examples/meta.tape10-12](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.tape#L10-L12)
 [examples/meta.tape18-20](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.tape#L18-L20)
 [examples/commands/hide.gif](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/commands/hide.gif)

### Dynamic Timing Control

Strategic use of Sleep commands creates natural pacing in demonstrations:

| Purpose | Sleep Duration | Example |
| --- | --- | --- |
| Brief pause between commands | 0.5s - 1s | `Type "ls" Enter Sleep 0.7s` |
| Time to read command output | 2s - 5s | `Type "git log" Enter Sleep 4s` |
| Emphasis on important result | 3s - 8s | `Type "npm test" Enter Sleep 6s` |
| Wait for long-running process | Variable | `Type "./build.sh" Enter Sleep 30s` |

Example from meta.tape:

    Sleep 30s
    

Sources: [examples/meta.tape16](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.tape#L16-L16)

Advanced Input Techniques
-------------------------

### Special Keys and Key Combinations

Examples:

    # Navigation in command line
    Left Left Left  # Move cursor left three positions
    Backspace       # Delete character before cursor
    Ctrl+a          # Move to beginning of line
    Ctrl+e          # Move to end of line
    Ctrl+c          # Interrupt running process

### Multi-Stage Command Building

Creating complex command entry with editing:

    Type "git comit -m 'Fix typo in README'" 
    Sleep 1s
    Left Left Left Left Left Left Left Left Left Left Left Left Left
    Type "m"  # Fixing "comit" to "commit"
    Sleep 0.5s
    End       # Move to end of line
    Enter
    

Sources: [examples/commands/arrow.gif](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/commands/arrow.gif)
 [examples/commands/type.gif](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/commands/type.gif)

### Natural-Looking Interactions

Making demonstrations appear more human by simulating hesitation and correction:

    Type "grpe -r 'function' ."
    Sleep 1s
    Ctrl+a    # Move to beginning of line  
    Right Right Right Right  # Move to position of typo
    Backspace
    Type "e"  # Fix "grpe" to "grep"
    End
    Enter
    

Sources: [examples/commands/backspace.gif](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/commands/backspace.gif)

Visual Customization
--------------------

### Dynamic Theme Switching

For tutorials that need to demonstrate different visual settings:

    Set Theme "Dracula"
    Type "cat code.js" Enter
    Sleep 2s
    
    Set Theme { "brightGreen": "#00D787", "brightYellow": "#FE5F86", "yellow": "#FFFFFF", "brightBlue": "#875FFF" }
    Type "cat code.js" Enter
    Sleep 2s
    

Example from meta.tape:

    Set Theme { "brightGreen": "#00D787", "brightYellow": "#FE5F86", "yellow": "#FFFFFF", "brightBlue": "#875FFF" }
    

Sources: [examples/meta.tape8](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.tape#L8-L8)

### Terminal Resizing

Changing terminal dimensions during recording for different views:

    Set Width 1920
    Set Height 1080
    Type "ls -la" Enter
    Sleep 2s
    
    # Change to a narrower view to demonstrate text wrapping
    Set Width 800
    Set Height 600
    Type "cat README.md" Enter
    Sleep 2s
    

Example from meta.tape:

    Set Width 1920
    Set Height 1080

Output Optimization Techniques
------------------------------

### Creating Perfect Loops

For GIFs that need to loop seamlessly:

Example:

    Type "echo 'Step 1'" Enter
    Sleep 1s
    Type "echo 'Step 2'" Enter
    Sleep 1s
    Type "echo 'Step 3'" Enter
    Sleep 1s
    Type "clear" Enter  # Return to initial state for perfect loop
    

### File Size Optimization

Techniques to keep output files small and efficient:

| Technique | Effect | Implementation |
| --- | --- | --- |
| Minimal terminal size | Reduces pixel count | `Set Width 80 Height 20` |
| Reduced framerate | Smaller file size | `Set Framerate 15` |
| Output format | Video vs GIF tradeoffs | `Output demo.mp4` or `Output demo.webm` |
| Compression | Post-processing | `Output demo.gif --quality 75` |
| Focused recording | Show only what's needed | Use Hide/Show strategically |

Example from meta.tape:

    Output examples/meta.webm
    Output examples/meta.mp4
    

Sources: [examples/meta.tape1-2](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.tape#L1-L2)
 [examples/meta.gif](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/meta.gif)

Multi-Context Demonstrations
----------------------------

### Split Terminal Scenarios

Demonstrating client-server interactions or parallel processes:

Example implementation:

    Hide
    Type "tmux new-session -d" Enter
    Type "tmux split-window -h" Enter
    Type "tmux select-pane -t 0" Enter
    Show
    
    # Left pane - Server
    Type "cd server" Enter
    Type "python -m http.server 8000" Enter
    Sleep 3s
    
    # Right pane - Client
    Type "tmux select-pane -t 1" Enter
    Type "curl http://localhost:8000" Enter
    Sleep 2s
    

### Multi-Step Workflow Demonstrations

For complex demonstrations involving several tools in sequence:

    # Step 1: Code editing
    Type "vim app.js" Enter
    Sleep 1s
    Type "ifunction hello() { console.log('Hello, world!'); }" Escape
    Type ":wq" Enter
    Sleep 1s
    
    # Step 2: Running tests
    Type "npm test" Enter
    Sleep 3s
    
    # Step 3: Deployment
    Type "git add app.js" Enter
    Type "git commit -m 'Add hello function'" Enter
    Sleep 2s
    Type "git push origin main" Enter
    Sleep 3s
    

Best Practices and Patterns
---------------------------

### Complete Demo Pattern

A standard pattern for professional demos:

Example implementation:

    # 1. Setup (hidden)
    Hide
    Type "git clone https://github.com/example/repo.git" Enter
    Type "cd repo" Enter
    Type "npm install" Enter
    Sleep 5s
    Type "clear" Enter
    Show
    
    # 2. Demo (visible)
    Type "# Let's run our application" Enter
    Type "node index.js" Enter
    Sleep 3s
    Type "# Now let's test the API" Enter
    Type "curl localhost:3000/api/data" Enter
    Sleep 2s
    
    # 3. Cleanup (hidden)
    Hide
    Type "cd .." Enter
    Type "rm -rf repo" Enter
    

### Visual Markers and Annotations

Adding comments or visual cues to guide viewers:

    Type "# First, let's install the dependencies" Enter
    Type "npm install" Enter
    Sleep 3s
    Type "# Now we can start the application" Enter
    Type "npm start" Enter
    

### Error Recovery and Handling

Demonstrating how to handle errors gracefully:

    Type "invalid_command" Enter
    Sleep 1s
    Type "# Oops, that command doesn't exist! Let's try this instead:" Enter
    Sleep 1s
    Type "valid_command" Enter
    

Specialized Use Cases
---------------------

### Interactive Tutorial Style

Simulating an interactive tutorial experience:

    Type "python" Enter
    Sleep 1s
    Type "# Let's learn about Python lists" Enter
    Sleep 2s
    Type "fruits = ['apple', 'banana', 'cherry']" Enter
    Sleep 1s
    Type "print(fruits[0])" Enter
    Sleep 2s
    Type "# Now let's add an item to our list" Enter
    Sleep 1s
    Type "fruits.append('orange')" Enter
    Type "print(fruits)" Enter
    

### Comparative Demonstrations

Showing before/after or A/B comparisons:

    Type "# Method A: Sequential processing" Enter
    Type "time ./process_sequential.sh" Enter
    Sleep 5s
    
    Type "# Method B: Parallel processing" Enter
    Type "time ./process_parallel.sh" Enter
    Sleep 5s
    
    Type "# As you can see, parallel processing is 3x faster" Enter
    Sleep 3s
