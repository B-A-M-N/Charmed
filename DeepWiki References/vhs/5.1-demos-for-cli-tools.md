Purpose and Scope
-----------------

This document demonstrates how to use VHS to create visually appealing demonstrations of command-line tools. It provides practical examples and techniques for showcasing CLI applications' features through automated terminal recordings. For information about shell-specific examples, see [Shell Examples](https://deepwiki.com/charmbracelet/vhs/5.2-shell-examples)
, and for advanced tape creation techniques, see [Advanced Tape Techniques](https://deepwiki.com/charmbracelet/vhs/5.3-advanced-tape-techniques)
.

Creating CLI Tool Demos with VHS
--------------------------------

VHS excels at creating visually consistent, repeatable demos of command-line tools. These demos are particularly valuable for:

*   Documentation of CLI tools
*   Feature showcases in README files
*   Product marketing materials
*   Tutorial content

Sources: [examples/README.md1-252](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/README.md?plain=1#L1-L252)

Demo Creation Workflow
----------------------

The process for creating a CLI tool demo follows this general pattern:

1.  Define the output settings (dimensions, framerate, output file)
2.  Write the command sequence to showcase your tool
3.  Add appropriate pauses for readability (Sleep commands)
4.  Include simulated user interactions (typing, keyboard navigation)
5.  Run the tape file with VHS to generate the output

Sources: [examples/README.md1-252](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/README.md?plain=1#L1-L252)

### Gum

[Gum](https://github.com/charmbracelet/vhs/blob/f28239f3/Gum)
 is a tool for creating attractive command-line interfaces. Here are examples of VHS demos for various Gum features:

#### Gum File Browser

    Output file.gif
    
    Type "gum file ./src"
    Sleep 1s
    Enter
    Sleep 2s
    Down@500ms 4
    Up@500ms 1
    Sleep 1s
    Enter
    Sleep 1s
    Down@500ms 4
    Sleep 1s
    Up@500ms 2
    Sleep 2s
    

This tape shows how to navigate through a file browser created with Gum, demonstrating keyboard navigation and selection.

#### Gum Table

    Output table.gif
    
    Type "gum table < superhero.csv -w 2,12,5,6,6,8,4,20 --height 10"
    Enter
    Sleep 1s
    Down@200ms 10
    Sleep 1s
    Down@200ms 10
    Sleep 1s
    Up@200ms 10
    Sleep 1s
    Enter
    Sleep 3s
    

This example demonstrates Gum's table component, showing scrolling through rows and selecting an item.

### Bubble Tea Applications

[Bubble Tea](https://github.com/charmbracelet/vhs/blob/f28239f3/Bubble%20Tea)
 is a framework for building terminal user interfaces. VHS is particularly effective for showcasing Bubble Tea applications:

#### Editor Integration Example

    Output examples/bubbletea/exec.gif
    
    Hide
    Type "go build -o exec ."
    Enter
    Type "clear"
    Enter
    Show
    
    Type "EDITOR=nano ./exec"
    Enter
    Sleep 0.5
    
    Type@0.5 "aaaa"
    Sleep 1
    
    Type@0.5 "e"
    Type "Hello, EDITOR!"
    Sleep 1
    Ctrl+X
    Sleep 0.5
    Type "n"
    Sleep 1
    
    Hide
    Type "q"
    Type "rm exec"
    Enter
    

This example demonstrates a Bubble Tea application that integrates with an external editor (nano), showing how to record both the TUI application and the editor interaction.

Best Practices for Creating Effective CLI Demos
-----------------------------------------------

Based on the examples in the repository, here are some best practices for creating effective CLI tool demos with VHS:

### 1\. Set Appropriate Display Dimensions

    Set Width 1600
    Set Height 1040
    

Tailor your terminal dimensions to show your tool effectively. Wider terminals are better for tables and complex outputs, while a more square format might be better for focusing on simple commands.

### 2\. Control Timing for Readability

    Type "gh issue list"
    Sleep 500ms
    Enter
    Sleep 4s
    

Use `Sleep` commands strategically to:

*   Pause after typing commands (before pressing Enter) to show what's being run
*   Allow time for output to be read
*   Create natural pacing that mimics human interaction

### 3\. Demonstrate User Interaction

    Down@200ms 10
    Sleep 1s
    Up@200ms 10
    

Show navigation and interaction with your tool. The `@` syntax controls the speed of repeated actions, making them look more natural.

### 4\. Use Hide/Show for Setup and Cleanup

    Hide
    Type "go build -o exec ."
    Enter
    Type "clear"
    Enter
    Show
    

Hide terminal output during setup or cleanup phases to keep the demo focused on the important parts.

### 5\. Showcase Multiple Features

Rather than showing just a basic command, demonstrate multiple features of your tool in a single demo. This provides more value and shows how the features work together.

### 6\. Control Typing Speed

    Type@500ms "todo"
    

Vary typing speed to emphasize different parts of the demo. Slower typing draws attention to important commands.

Creating Complex Interaction Sequences
--------------------------------------

For tools with rich UI interactions, you can create complex sequences that show realistic usage patterns:

The Glow example demonstrates this approach, showing a natural progression through different features of the tool:

1.  Launch application
2.  Navigate through content
3.  Search for specific text
4.  Edit content
5.  Navigate between different views
6.  Exit the application

This creates a much more compelling demo than simply showing static output.

Sources: [examples/glow/glow.tape1-130](https://github.com/charmbracelet/vhs/blob/f28239f3/examples/glow/glow.tape#L1-L130)

Summary
-------

Creating effective CLI tool demos with VHS involves:

1.  Planning your demo to showcase key features
2.  Writing tape files with appropriate commands, timing, and interactions
3.  Using VHS features like Hide/Show and variable typing speeds
4.  Generating high-quality output that can be embedded in documentation

The examples in the VHS repository provide excellent templates for creating your own demos of various CLI tools, from simple command-line utilities to complex interactive applications.
