======================

This page covers installing `mods`, configuring API keys for LLM providers, and understanding the four-tier configuration hierarchy that controls application behavior.

Overview
--------

Installation consists of three stages:

1.  Installing the binary via package manager, pre-built package, or building from source
2.  Initial configuration file creation (automatic on first run)
3.  API key configuration for desired providers

The configuration system uses a precedence hierarchy: embedded template → user config file → environment variables → CLI flags.

Sources: [config.go208-252](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L208-L252)
 [README.md28-104](https://github.com/charmbracelet/mods/blob/07a05d5b/README.md?plain=1#L28-L104)

--------------------

### Package Managers

| Platform | Command |
| --- | --- |
| macOS/Linux (Homebrew) | `brew install charmbracelet/tap/mods` |
| Windows (Winget) | `winget install charmbracelet.mods` |
| Arch Linux | `yay -S mods` |
| Nix | `nix-shell -p mods` |

Package manager installations automatically configure shell completions.

Shell Completions
-----------------

### Automatic Setup

Package manager installations (Homebrew, APT, YUM, etc.) automatically configure shell completions if your shell is properly configured.

### Manual Generation

If you built from source or need to regenerate completions:

| Shell | Command |
| --- | --- |
| Bash | `mods completion bash -h` |
| Zsh | `mods completion zsh -h` |
| Fish | `mods completion fish -h` |
| PowerShell | `mods completion powershell -h` |

Each command displays instructions for setting up completions in the respective shell.
