Configuration System
====================

Relevant source files

*   [config.go](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go)
    
*   [config\_template.yml](https://github.com/charmbracelet/mods/blob/07a05d5b/config_template.yml)
    

The Configuration System manages all settings for the mods CLI tool, including LLM provider configurations, user preferences, and external tool integrations. It implements a multi-layered configuration approach with clear precedence rules and automatic defaults.

For information about how configuration interacts with LLM providers, see [Provider Architecture](https://deepwiki.com/charmbracelet/mods/3.1-provider-configuration)
. For details about MCP tool configuration, see [MCP Tool Integration](https://deepwiki.com/charmbracelet/mods/5.1-conversation-database)
.

Configuration Structure
-----------------------

The configuration is organized into several key structures that handle different aspects of the system.

### Core Config Struct

The `Config` struct serves as the main configuration container:

| Field | Type | Purpose | Environment Variable |
| --- | --- | --- | --- |
| `API` | `string` | Default API provider | `MODS_API` |
| `Model` | `string` | Default model name | `MODS_MODEL` |
| `MaxInputChars` | `int64` | Character limit for input | `MODS_MAX_INPUT_CHARS` |
| `Temperature` | `float64` | Model temperature setting | `MODS_TEMP` |
| `APIs` | `APIs` | Provider configurations | \-  |
| `MCPServers` | `map[string]MCPServerConfig` | External tool servers | \-  |
| `CachePath` | `string` | Cache directory path | `MODS_CACHE_PATH` |

Environment Variable Support
----------------------------

The configuration system automatically maps environment variables with the `MODS_` prefix to configuration fields:

| Configuration Field | Environment Variable | Example |
| --- | --- | --- |
| `Model` | `MODS_MODEL` | `export MODS_MODEL=gpt-4` |
| `API` | `MODS_API` | `export MODS_API=anthropic` |
| `Temperature` | `MODS_TEMP` | `export MODS_TEMP=0.7` |
| `MaxInputChars` | `MODS_MAX_INPUT_CHARS` | `export MODS_MAX_INPUT_CHARS=50000` |
| `CachePath` | `MODS_CACHE_PATH` | `export MODS_CACHE_PATH=/custom/path` |

The environment parsing is handled by the `env.ParseWithOptions` function with prefix support.

Sources: [config.go232-234](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L232-L234)

File Management and XDG Compliance
----------------------------------

The configuration system follows XDG Base Directory Specification for file organization:

The `ensureConfig` function handles directory creation and file initialization:

1.  Uses `xdg.ConfigFile()` to determine the config file path
2.  Creates necessary directories with `os.MkdirAll()`
3.  Checks for existing configuration with `writeConfigFile()`
4.  Creates default configuration from template if needed

Sources: [config.go208-252](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L208-L252)
 [config.go236-245](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L236-L245)

Default Configuration Template
------------------------------

The embedded `config_template.yml` provides comprehensive defaults for all supported LLM providers:

### Major Provider Categories

*   **OpenAI**: GPT models including o1, o3-mini series
*   **Anthropic**: Claude models including Sonnet variants
*   **Google**: Gemini models including 2.0-flash
*   **Local/Open**: Ollama, LocalAI configurations
*   **Cloud Providers**: Cohere, Perplexity, Groq, Cerebras

### Template Processing

The template uses Go's `text/template` package for dynamic content generation:

Template variables include `Config` defaults and `Help` descriptions for each setting.

Sources: [config\_template.yml1-547](https://github.com/charmbracelet/mods/blob/07a05d5b/config_template.yml#L1-L547)
 [config.go263-282](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L263-L282)

Format Text Configuration
-------------------------

The `FormatText` type supports flexible output formatting configuration:

It supports both simple string and complex map configurations:

*   Simple: `format-text: "Format as markdown"`
*   Complex: `format-text: { markdown: "...", json: "..." }`

The `UnmarshalYAML` method handles both formats seamlessly.

Sources: [config.go118-137](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L118-L137)
 [config.go287-291](https://github.com/charmbracelet/mods/blob/07a05d5b/config.go#L287-L291)

Configuration Validation and Defaults
-------------------------------------

The system applies sensible defaults when values are missing or invalid:

*   `WordWrap`: Defaults to 80 characters
*   `CachePath`: Uses XDG data directory if not specified
*   `MCPTimeout`: Defaults to 15 seconds for tool operations
*   Directory creation ensures cache and conversation storage
