Bubble Tea UI Model
===================

Relevant source files

*   [mods.go](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go)
    

This document describes the `Mods` struct and its implementation of the Bubble Tea UI framework. The `Mods` struct serves as the core state machine that orchestrates user input, AI provider communication, and output rendering. It implements the `tea.Model` interface with `Init()`, `Update()`, and `View()` methods that handle the complete lifecycle of an AI interaction session.

For details on how the model is instantiated and invoked from the CLI entry point, see [Application Entry Point](https://deepwiki.com/charmbracelet/mods/2.1-application-entry-point)
. For information about how messages are prepared for AI providers, see [Message Stream Context](https://deepwiki.com/charmbracelet/mods/2.4-message-stream-context)
. For rendering and terminal output, see [Output Rendering and Formatting](https://deepwiki.com/charmbracelet/mods/4.3-output-rendering-and-formatting)
.

Mods Struct Overview
--------------------

The `Mods` struct is the central Bubble Tea model that maintains all application state during execution. It holds configuration, UI components, state machine variables, and connections to persistence systems.

**Sources:** [mods.go52-78](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L52-L78)

The key fields include:

| Field | Type | Purpose |
| --- | --- | --- |
| `state` | `state` | Current position in state machine (enum) |
| `Output` | `string` | Accumulated response text from AI |
| `Input` | `string` | User input after whitespace normalization |
| `messages` | `[]proto.Message` | Full conversation context for AI provider |
| `Config` | `*Config` | Configuration including API keys, model selection |
| `db` | `*convoDB` | SQLite database for conversation metadata |
| `cache` | `*cache.Conversations` | File-based cache for full message history |
| `glam` | `*glamour.TermRenderer` | Markdown renderer for terminal output |
| `glamViewport` | `viewport.Model` | Scrollable viewport for long responses |
| `anim` | `tea.Model` | Animated spinner during request phase |
| `cancelRequest` | `[]context.CancelFunc` | Context cancellation functions for cleanup |

**Sources:** [mods.go52-78](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L52-L78)

State Machine
-------------

The application progresses through six distinct states during execution. State transitions are triggered by incoming Bubble Tea messages in the `Update()` method.

### State Enumeration

**Sources:** [mods.go39-48](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L39-L48)
 [mods.go125-215](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L125-L215)

| State | Constant | Description |
| --- | --- | --- |
| `startState` | `0` | Initial state before cache operations resolved |
| `configLoadedState` | `1` | Cache details loaded, ready to read stdin |
| `requestState` | `2` | Request sent to AI provider, awaiting first response |
| `responseState` | `3` | Receiving streaming chunks from AI provider |
| `doneState` | `4` | Operation complete, ready to exit |
| `errorState` | `5` | Error occurred, will exit with error message |

**Sources:** [mods.go39-48](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L39-L48)

| `API` | `string` | API provider name (overrides config) |
| `Model` | `string` | Model name (overrides config) |

Init Method
-----------

The `Init()` method is called once when the Bubble Tea program starts. It returns a command that resolves cache operation details.

The `findCacheOpsDetails()` method performs several resolution steps:

1.  Determines if continuing last conversation (`--continue-last` flag)
2.  Resolves read ID from `--continue` or `--show` flags
3.  Determines write ID from `--title` or `--continue` flags
4.  Looks up existing conversation in database to load model/API
5.  Generates new SHA-1 ID if creating new conversation
6.  Returns `cacheDetailsMsg` with all resolved values

**Sources:** [mods.go120-122](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L120-L122)
 [mods.go534-587](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L534-L587)

View Method
-----------

The `View()` method renders the current state to the terminal. It returns different output based on the current state and TTY configuration.

**Sources:** [mods.go222-257](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L222-L257)

### View State Behavior

| State | TTY Output | Non-TTY Output |
| --- | --- | --- |
| `requestState` | Animation spinner (unless quiet) | Empty |
| `responseState` | Glamour-formatted markdown in viewport | Raw chunks printed immediately |
| `doneState` | Empty | Final newline |
| `errorState` | Empty (error shown separately) | Empty |

**Sources:** [mods.go222-257](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L222-L257)

Command Methods
---------------

The `Mods` model defines several command-generating methods that perform I/O operations and return messages.

### readStdinCmd

Reads user input from stdin if not a TTY. Returns `completionInput` message.

**Sources:** [mods.go604-615](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L604-L615)

### startCompletionCmd

Initializes the AI provider client and starts the streaming request. This is a complex method that:

1.  Resolves the model and API using `resolveModel()`
2.  Ensures API keys are available via `ensureKey()`
3.  Configures provider-specific clients (OpenAI, Anthropic, Cohere, Ollama, Google)
4.  Sets up HTTP proxy if configured
5.  Retrieves MCP tools via `mcpTools()`
6.  Builds message context via `setupStreamContext()`
7.  Constructs `proto.Request` with all parameters
8.  Initializes provider client and starts streaming
9.  Returns result from `receiveCompletionStreamCmd()`

**Sources:** [mods.go276-454](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L276-L454)

### receiveCompletionStreamCmd

Processes streaming chunks from the AI provider. This method:

1.  Calls `stream.Next()` to get next chunk
2.  Extracts content from `stream.Current()`
3.  Returns `completionOutput` with content and stream
4.  On stream end, checks for errors with `stream.Err()`
5.  Processes tool calls with `stream.CallTools()`
6.  Updates `m.messages` with complete conversation
7.  Returns `completionOutput` with nil stream when done

**Sources:** [mods.go492-528](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L492-L528)

### findCacheOpsDetails

Resolves conversation IDs and loads settings from database. See the [Init Method](https://github.com/charmbracelet/mods/blob/07a05d5b/Init%20Method)
 section above for details.

**Sources:** [mods.go534-587](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L534-L587)

### readFromCache

Loads a conversation from cache and displays it. Used for `--show` operations.

**Sources:** [mods.go617-631](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L617-L631)

Helper Methods
--------------

### quit

Cancels all active requests and returns `tea.Quit()` message.

**Sources:** [mods.go259-264](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L259-L264)

### retry

Implements exponential backoff for retrying failed requests. Waits `100ms * 2^retries` before returning new `completionInput` to retry.

**Sources:** [mods.go266-274](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L266-L274)

### appendToOutput

Accumulates response text and updates the glamour-rendered output. For TTY output, it:

1.  Appends to `m.Output` string
2.  Renders full output with glamour to `m.glamOutput`
3.  Updates viewport content
4.  Auto-scrolls to bottom if viewport was already at bottom

For non-TTY output, it:

1.  Appends chunks to `m.content` slice (mutex-protected)
2.  Chunks are printed immediately in `View()` method

**Sources:** [mods.go635-661](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L635-L661)

### viewportNeeded

Returns true if the glamour-rendered output height exceeds the terminal height, indicating scrolling is needed.

**Sources:** [mods.go217-219](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L217-L219)

### resolveModel

Resolves model aliases to canonical names and locates the model in the API configuration. Returns the matched `API` and `Model` structs. For details, see [Model Resolution and Fallbacks](https://deepwiki.com/charmbracelet/mods/3.3-model-resolution-and-fallbacks)
.

**Sources:** [mods.go704-747](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L704-L747)

### ensureKey

Retrieves API key from multiple sources in priority order:

1.  `api.APIKey` field (explicit key)
2.  Environment variable specified in `api.APIKeyEnv`
3.  External command specified in `api.APIKeyCmd`
4.  Default environment variable for provider (e.g., `OPENAI_API_KEY`)

Returns error with helpful message if no key found.

**Sources:** [mods.go456-490](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L456-L490)

Integration with Other Systems
------------------------------

### Configuration System

The `Config` field provides all runtime settings including API keys, model selection, formatting options, and conversation IDs. It is modified during `cacheDetailsMsg` handling to set resolved cache IDs.

**Sources:** [mods.go72](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L72-L72)
 [mods.go128-133](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L128-L133)

### Conversation Management

The model interacts with both the database and cache:

*   `db.Find()` locates conversations by ID or title
*   `db.FindHEAD()` retrieves the most recent conversation
*   `cache.Read()` loads full message history
*   Messages are saved automatically by the provider streaming logic

**Sources:** [mods.go70-71](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L70-L71)
 [mods.go589-602](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L589-L602)
 [mods.go617-631](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L617-L631)

### Provider Streaming

The model delegates to provider-specific clients but handles the stream uniformly:

**Sources:** [mods.go276-454](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L276-L454)

### Animation System

The animation model is a separate Bubble Tea component managed as a child model. It is initialized with configuration and styles, updated only during loading states, and rendered via its own `View()` method.

**Sources:** [mods.go66](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L66-L66)

### Terminal Output

The model uses several components for output rendering:

*   `glamour.TermRenderer` converts markdown to ANSI-styled terminal output
*   `viewport.Model` provides scrolling for long responses
*   `lipgloss.Renderer` provides styling capabilities
*   TTY detection determines whether to use formatted or raw output
