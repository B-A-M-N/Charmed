Terminal Capabilities and Styling
=================================

Relevant source files

*   [.gitignore](https://github.com/charmbracelet/mods/blob/07a05d5b/.gitignore)
    
*   [styles.go](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go)
    
*   [term.go](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go)
    

Purpose and Scope
-----------------

This document details the terminal capability detection and styling system in Mods. The system uses the `mattn/go-isatty` library to detect TTY connections, `charmbracelet/lipgloss` for terminal-aware rendering, and `muesli/termenv` for terminal capability detection. The styling infrastructure provides adaptive color schemes that automatically adjust based on terminal background (light/dark) and capabilities (TrueColor support).

The system determines appropriate rendering behavior based on whether stdin/stdout are connected to a TTY, initializes separate renderers for stdout and stderr, and maintains a `styles` struct with 17 different style definitions used throughout the application. For information about how these styles are applied during animation rendering, see page 4.2. For details on how styled content is processed and displayed, see page 4.3.

TTY Detection System
--------------------

The terminal detection system determines whether stdin and stdout are connected to a terminal (TTY) or redirected (pipes, files). This detection controls whether the application renders styled output or emits plain text.

**TTY Detection Functions**

Sources: [term.go12-18](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L12-L18)

Both `isInputTTY` and `isOutputTTY` are defined as `sync.OnceValue` functions, which execute the detection logic only once and cache the result. This is critical for performance since TTY status is checked throughout the application lifecycle.

Sources: [term.go12-18](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L12-L18)

**TTY Detection Usage Patterns**

| Scenario | Input TTY | Output TTY | Application Behavior |
| --- | --- | --- | --- |
| Interactive Terminal | ✓   | ✓   | Full styling, Bubble Tea UI, animations |
| Pipe Output | ✓   | ✗   | No styling, plain text output (e.g., `mods "query" \| grep`) |
| Pipe Input | ✗   | ✓   | Read from stdin, styled output (e.g., `cat file \| mods`) |
| Script Mode | ✗   | ✗   | Plain text I/O (e.g., `cat input \| mods > output`) |

The most common check is `isOutputTTY()`, used to determine whether to apply lipgloss styling and glamour markdown rendering, or to emit raw text for piping to other commands.

Sources: [term.go12-18](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L12-L18)

The `styles` Struct
-------------------

The `styles` struct defined in [styles.go10-28](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L10-L28)
 contains 17 pre-configured `lipgloss.Style` fields used throughout the application for consistent visual presentation.

**`styles` Struct Definition**

Sources: [styles.go10-28](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L10-L28)

**Style Field Categories**

| Category | Fields | Usage Context |
| --- | --- | --- |
| Application Identity | `AppName` | Application name display in help text |
| CLI Elements | `CliArgs`, `Flag`, `FlagComma`, `FlagDesc` | Flag rendering in help/error messages |
| Content Formatting | `InlineCode`, `Link`, `Quote`, `Pipe` | Inline text elements in output |
| Error Display | `ErrorHeader`, `ErrorDetails`, `ErrPadding` | Error message formatting |
| Interactive UI | `CyclingChars`, `ConversationList` | Animation characters, conversation lists |
| Metadata | `SHA1`, `Timeago`, `Comment` | Conversation IDs, timestamps, annotations |

Sources: [styles.go10-28](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L10-L28)

**The `makeStyles` Function**

The `makeStyles(r *lipgloss.Renderer)` function at [styles.go30-50](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L30-L50)
 constructs a `styles` struct by calling `r.NewStyle()` for each field and configuring it with colors, padding, and text properties.

Sources: [styles.go30-50](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L30-L50)

Key configuration patterns in `makeStyles`:

*   **Foreground colors**: Set using `lipgloss.Color("#RRGGBB")` or `lipgloss.AdaptiveColor`
*   **Background colors**: Applied for headers and inline code blocks
*   **Padding**: Horizontal padding (`Padding(0, horizontalEdgePadding)`)
*   **Text properties**: `Bold(true)`, `Underline(true)`, `SetString()`

The constant `horizontalEdgePadding = 2` is used for consistent spacing in error displays.

Sources: [styles.go30-50](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L30-L50)

Renderer Initialization
-----------------------

The system maintains four `sync.OnceValue` global variables in [term.go20-34](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L20-L34)
 that lazily initialize renderers and styles for stdout and stderr.

**Dual Renderer System**

Sources: [term.go20-34](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L20-L34)

**Renderer Implementation Details**

| Variable | Return Type | Initialization | Purpose |
| --- | --- | --- | --- |
| `stdoutRenderer` | `*lipgloss.Renderer` | `lipgloss.DefaultRenderer()` | Renders to stdout with auto-detected color profile |
| `stderrRenderer` | `*lipgloss.Renderer` | `lipgloss.NewRenderer(os.Stderr, termenv.WithColorCache(true))` | Renders to stderr with color caching for performance |
| `stdoutStyles` | `styles` | `makeStyles(stdoutRenderer())` | Pre-configured styles for main output |
| `stderrStyles` | `styles` | `makeStyles(stderrRenderer())` | Pre-configured styles for error output |

Sources: [term.go20-34](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L20-L34)

The `stdoutRenderer` uses `lipgloss.DefaultRenderer()`, which automatically detects the terminal's color capabilities and output stream. The `stderrRenderer` explicitly targets `os.Stderr` and enables color caching via `termenv.WithColorCache(true)` to improve performance when rendering multiple error messages.

Each renderer has an associated styles struct created by calling `makeStyles()` with that renderer. This ensures that styles are bound to the correct output stream and use the appropriate color profile.

Sources: [term.go20-34](https://github.com/charmbracelet/mods/blob/07a05d5b/term.go#L20-L34)

Style Field Details
-------------------

Each field in the `styles` struct has specific configuration details defined in `makeStyles`. This section documents the key style properties.

**Error Display Styles**

Sources: [styles.go36-38](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L36-L38)

The `ErrorHeader` style at [styles.go36](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L36-L36)
 is pre-configured with the string "ERROR" via `SetString()`, making it ready for immediate use in error displays. The `ErrorDetails` style is set to the same configuration as `Comment`, providing a muted appearance for error explanations. The `ErrPadding` style provides horizontal spacing.

**CLI and Flag Styles**

| Field | Configuration | Line |
| --- | --- | --- |
| `AppName` | `Bold(true)` | [styles.go32](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L32-L32) |

Action Confirmation Header
--------------------------

In addition to the `styles` struct, [styles.go54-64](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L54-L64)
 defines a module-level `outputHeader` style and `printConfirmation` function for displaying user action confirmations.

**`printConfirmation` Function**

Sources: [styles.go58-64](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L58-L64)

The `outputHeader` variable at [styles.go56](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L56-L56)
 is a `lipgloss.Style` with:

*   Foreground: `#F1F1F1` (white)
*   Background: `#6C50FF` (purple)
*   Bold: `true`
*   Padding: `0, 1`
*   Margin right: `1`

The `printConfirmation` function formats and displays confirmation messages after actions like saving conversations. It defaults to "WROTE" if no action is specified, converts the action to uppercase, sets it in the header style, and joins it horizontally with the content string.

Sources: [styles.go54-64](https://github.com/charmbracelet/mods/blob/07a05d5b/styles.go#L54-L64)

Initialization and Update Cycle
-------------------------------

The rendering components are initialized during Mods creation and updated during the application lifecycle:

Sources: [mods.go70-85](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L70-L85)
 [mods.go109-194](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L109-L194)

During initialization, the system creates necessary renderers and styles. When window size changes, the viewport dimensions are updated to match the new terminal dimensions.

Text Processing Details
-----------------------

Several text processing operations are performed during rendering:

1.  Tab replacement:
    
        m.glamOutput = strings.ReplaceAll(m.glamOutput, "\t", strings.Repeat(" ", tabWidth))
        
    
2.  Whitespace handling:
    
        m.glamOutput = strings.TrimRightFunc(m.glamOutput, unicode.IsSpace)
        
    
3.  Width truncation:
    
        truncatedGlamOutput := m.renderer.NewStyle().MaxWidth(m.width).Render(m.glamOutput)
        
    

Sources: [mods.go665-669](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L665-L669)

These operations ensure that text displays correctly in the terminal regardless of content type.
