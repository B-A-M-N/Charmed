Application Entry Point
=======================

Relevant source files

*   [examples.go](https://github.com/charmbracelet/mods/blob/07a05d5b/examples.go)
    
*   [main.go](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go)
    

This document describes the application's entry point in `main.go`, including the `main()` function, Cobra command structure, initialization sequence, flag binding system, and the primary execution flow through `rootCmd.RunE`. For information about how the configuration system loads settings, see [Configuration System](https://deepwiki.com/charmbracelet/mods/2.2-configuration-system)
. For details on the Bubble Tea UI lifecycle, see [Bubble Tea UI Model](https://deepwiki.com/charmbracelet/mods/2.3-bubble-tea-ui-model)
.

Overview
--------

The mods application uses the Cobra framework for command-line interface handling. The entry point orchestrates initialization, flag parsing, database setup, and delegates to either special commands (list, show, delete) or the main Bubble Tea UI for AI interaction.

Main Function Flow
------------------

The `main()` function at [main.go336-399](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L336-L399)
 serves as the application's entry point and follows this sequence:

**Sources:** [main.go336-399](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L336-L399)

### Configuration Loading

The application first loads configuration via `ensureConfig()` at [main.go339](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L339-L339)
 If configuration loading fails, the error is handled but execution continues if the user invoked `--settings`, allowing them to edit a malformed config file.

**Sources:** [main.go339-347](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L339-L347)

### Flag Initialization

After configuration loads, `initFlags()` is called at [main.go350](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L350-L350)
 to bind all command-line flags. This must occur after configuration loading because default values are derived from the loaded config.

**Sources:** [main.go350](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L350-L350)

### Database Initialization

For normal commands (not completion, man, version, or help), the application opens a SQLite database at [main.go353](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L353-L353)
:

The database is deferred for closure at [main.go358](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L358-L358)
 The global `db` variable of type `*convoDB` is used throughout the application for conversation management.

**Sources:** [main.go352-359](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L352-L359)

### Special Command Detection

The application checks for special commands using helper functions:

*   `isCompletionCmd()` at [main.go361](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L361-L361)
     - detects shell completion requests
*   `isManCmd()` at [main.go373](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L373-L373)
     - detects man page generation
*   `isVersionOrHelpCmd()` at [main.go352](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L352-L352)
     - detects version/help flags

These checks optimize startup by skipping database initialization when unnecessary.

**Sources:** [main.go361-392](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L361-L392)

Cobra Command Structure

**Sources:** [main.go70-242](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L70-L242)

### Root Command Configuration

| Field | Value | Purpose |
| --- | --- | --- |
| `Use` | `"mods"` | Command name |
| `Short` | `"GPT on the command line. Built for pipelines."` | Brief description |
| `SilenceUsage` | `true` | Suppress usage on errors |
| `SilenceErrors` | `true` | Custom error handling |
| `Example` | `randomExample()` | Random usage example |
| `RunE` | Main execution function | Core logic |

The command has no subcommands in normal operation. Completion and man page commands are added dynamically only when needed.

Command Execution Flow
----------------------

The `rootCmd.RunE` function at [main.go80-240](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L80-L240)
 implements the main execution logic:

**Sources:** [main.go80-240](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L80-L240)

### Execution Phases

#### Phase 1: Prefix Setup (Lines 81)

The user's prompt arguments are joined and stored in `config.Prefix` after whitespace removal.

**Sources:** [main.go81](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L81-L81)

#### Phase 2: Tea Program Options (Lines 83-95)

Bubble Tea program options are configured based on TTY status:

*   Non-TTY input or `--raw`: disable input with `tea.WithInput(nil)`
*   TTY output and not `--raw`: send output to stderr with `tea.WithOutput(os.Stderr)`
*   Non-TTY output or `--raw`: disable renderer with `tea.WithoutRenderer()`
*   If `VIMRUNTIME` environment variable is set: enable quiet mode

**Sources:** [main.go83-95](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L83-L95)

#### Phase 3: Editor Mode (Lines 97-103)

If no arguments provided, input is TTY, and `--editor` flag is set, opens user's `$EDITOR` to compose the prompt via `prefixFromEditor()`.

**Sources:** [main.go97-103](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L97-L103)

#### Phase 4: Interactive Prompt (Lines 105-117)

If no arguments or `--ask-model` flag, prompts user interactively via `askInfo()` to select API, model, and enter prompt text.

**Sources:** [main.go105-117](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L105-L117)

#### Phase 5: Bubble Tea Execution (Lines 119-133)

Creates cache, initializes `Mods` struct, creates Bubble Tea program, and runs it. The returned model is cast back to `*Mods` to check for errors.

**Sources:** [main.go119-133](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L119-L133)

#### Phase 6: Post-Execution Routing (Lines 135-239)

After Bubble Tea completes, handles various flags:

*   `--dirs`: prints config and cache directories
*   `--settings`: opens config file in editor
*   `--reset-settings`: resets config to defaults
*   `--list-roles`: lists available roles
*   `--list`: lists conversations
*   `--mcp-list` / `--mcp-list-tools`: MCP operations
*   `--delete`: deletes specified conversations
*   `--delete-older-than`: deletes old conversations
*   Default: prints output and saves conversation if needed

**Sources:** [main.go135-239](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L135-L239)

### Error Types

| Type | Fields | Usage |
| --- | --- | --- |
| `flagParseError` | Flag name, reason | Invalid flag values |
| `modsError` | `err`, `reason` | Application errors with user-friendly messages |
| Generic `error` | \-  | Unexpected errors |

The handler exhausts stdin at [main.go443-445](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L443-L445)
 before printing errors to prevent stdin data from interfering with error display.

Key Helper Functions
--------------------

### isNoArgs()

Defined at [main.go767-781](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L767-L781)
 checks if the user provided any meaningful arguments or flags. Returns `true` when no operation is requested (no prompt, no show, no list, etc.).

**Sources:** [main.go767-781](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L767-L781)

### askInfo()

Interactive prompt function at [main.go783-851](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L783-L851)
 using the `huh` form library. Prompts for:

1.  API selection (if `--ask-model` or no valid default model)
2.  Model selection for chosen API
3.  Prompt text (if not provided)

**Sources:** [main.go783-851](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L783-L851)

### prefixFromEditor()

Function at [main.go924-949](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L924-L949)
 that:

1.  Creates temporary file
2.  Opens it in `$EDITOR`
3.  Reads contents as prompt
4.  Deletes temporary file

**Sources:** [main.go924-949](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L924-L949)
