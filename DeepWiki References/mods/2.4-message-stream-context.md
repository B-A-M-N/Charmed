Message Stream Context
======================

Relevant source files

*   [mods.go](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go)
    
*   [stream.go](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go)
    

Purpose and Scope
-----------------

This document describes the `setupStreamContext` system responsible for constructing the message array sent to AI providers. The function builds a `[]proto.Message` by combining system prompts, role definitions, conversation history, and user input, while enforcing content limits. For information about how these messages are sent to providers and streamed back, see [Client Initialization and Streaming](https://deepwiki.com/charmbracelet/mods/3.2-client-initialization-and-streaming)
. For details on how conversation history is stored and loaded, see [Cache System](https://deepwiki.com/charmbracelet/mods/5.2-cache-system)
.

**Sources:** [stream.go1-71](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L1-L71)
 [mods.go402-404](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L402-L404)

* * *

Overview
--------

The message stream context is built by the `setupStreamContext` method, which prepares the complete message array before sending a request to an AI provider. This function orchestrates several configuration-driven transformations to construct a properly ordered sequence of system and user messages.

**Sources:** [stream.go10-70](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L10-L70)

* * *

Message Structure
-----------------

Messages use the `proto.Message` type with two primary fields:

| Field | Type | Description |
| --- | --- | --- |
| `Role` | `proto.Role` | Message role: `RoleSystem`, `RoleUser`, or `RoleAssistant` |
| `Content` | `string` | The text content of the message |

The `m.messages` field in the `Mods` struct stores the complete message array that will be sent in the `proto.Request`.

**Sources:** [stream.go12-67](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L12-L67)
 [mods.go64](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L64-L64)

* * *

Message Construction Flow
-------------------------

**Sources:** [stream.go10-70](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L10-L70)

* * *

Construction Phases
-------------------

### Phase 1: Format System Message

If formatting is enabled (`cfg.Format` is true), a system message is prepended containing formatting instructions from the configuration.

The format text is retrieved from the `FormatText` map using `FormatAs` as the key. This system message guides the AI to produce output in the specified format.

**Sources:** [stream.go13-18](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L13-L18)

* * *

### Phase 2: Role System Messages

When a role is specified (`cfg.Role != ""`), the system loads role definitions from `cfg.Roles` and adds them as system messages.

The `loadMsg` function handles loading role content from strings, files, or URLs. Each message in the role setup becomes a separate system message.

**Sources:** [stream.go20-41](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L20-L41)

* * *

### Phase 3: Prefix Application

If a prefix is configured, it is prepended to the user content with double newline separation:

    content = strings.TrimSpace(prefix + "\n\n" + content)
    

This allows users to add context or instructions before the main input content.

**Sources:** [stream.go43-45](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L43-L45)

* * *

### Phase 4: Content Length Enforcement

Unless `cfg.NoLimit` is true, the content is truncated to fit within the model's `MaxChars` limit:

The `mod.MaxChars` value comes from the resolved model configuration. If `MaxChars` is 0, it defaults to `cfg.MaxInputChars`.

**Sources:** [stream.go47-49](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L47-L49)
 [mods.go382-384](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L382-L384)

* * *

### Phase 5: Cache Loading

When continuing a conversation, previous messages are loaded from the cache and prepended to `m.messages`:

The cache read populates `m.messages` with the complete conversation history, including previous system messages, user prompts, and assistant responses. The cache read can fail with a `modsError` if the conversation cannot be loaded.

**Sources:** [stream.go51-62](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L51-L62)

* * *

### Phase 6: User Message Addition

Finally, the processed content is added as a user message:

    m.messages = append(m.messages, proto.Message{
        Role:    proto.RoleUser,
        Content: content,
    })
    

This is always the final step, ensuring the user's input is the last message in the context.

**Sources:** [stream.go64-67](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L64-L67)

* * *

Message Array Structure
-----------------------

The final `m.messages` array follows this structure:

**Example message array when continuing a conversation with a role:**

| Index | Role | Content Source |
| --- | --- | --- |
| 0   | System | Format text (e.g., "Output as JSON") |
| 1   | System | Role setup message 1 |
| 2   | System | Role setup message 2 |
| 3   | User | Previous user prompt (from cache) |
| 4   | Assistant | Previous AI response (from cache) |
| 5   | User | Current user input |

**Sources:** [stream.go12-67](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L12-L67)

* * *

Integration with Request Flow
-----------------------------

The constructed message array is used to build the `proto.Request` sent to AI providers:

After `setupStreamContext` completes successfully, the `proto.Request` struct is populated with the message array along with other parameters like temperature, top-p, and max tokens.

**Sources:** [mods.go402-446](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L402-L446)

* * *

Configuration Fields
--------------------

The following `Config` fields affect message stream context construction:

| Field | Type | Purpose |
| --- | --- | --- |
| `Format` | `bool` | Enable format system message |
| `FormatAs` | `string` | Format type (e.g., "json") |
| `FormatText` | `map[string]string` | Format instruction text by type |
| `Role` | `string` | Selected role name |
| `Roles` | `map[string][]string` | Role definitions |
| `Prefix` | `string` | Text to prepend to user input |
| `NoLimit` | `bool` | Disable max-chars enforcement |
| `NoCache` | `bool` | Disable cache loading |
| `cacheReadFromID` | `string` | Conversation ID to load |

**Sources:** [stream.go11-62](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L11-L62)

* * *

Error Handling
--------------

The function returns a `modsError` in three scenarios:

1.  **Role not found**: When `cfg.Role` is specified but doesn't exist in `cfg.Roles`
    
2.  **Role loading failure**: When `loadMsg` fails to load role content
    
3.  **Cache read failure**: When conversation history cannot be loaded
    

All errors are wrapped in `modsError` with both technical details (`err`) and user-friendly explanations (`reason`).

**Sources:** [stream.go23-26](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L23-L26)
 [stream.go31-35](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L31-L35)
 [stream.go53-61](https://github.com/charmbracelet/mods/blob/07a05d5b/stream.go#L53-L61)

* * *

Post-Stream Message Updates
---------------------------

After the AI provider stream completes, the messages array is updated with the complete conversation:

This updated array includes the assistant's response and is used when saving the conversation to the cache and database.

**Sources:** [mods.go521](https://github.com/charmbracelet/mods/blob/07a05d5b/mods.go#L521-L521)

* * *

Related Functions
-----------------

*   `loadMsg`: Loads content from strings, files (file://), or URLs (http:/https:) for role setup
*   `m.cache.Read`: Loads previous messages from cache directory
*   `removeWhitespace`: Sanitizes input by removing whitespace-only content
*   `increaseIndent`: Adds tab indentation to stdin input
