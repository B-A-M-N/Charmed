Message Utilities
=================

Relevant source files

*   [messages\_test.go](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go)
    

Purpose and Scope
-----------------

This document describes utility functions that support message manipulation and preprocessing in the mods codebase. These functions handle common tasks such as extracting prompts from message arrays, truncating content to fit token limits, and normalizing text formatting.

For the broader context of how messages flow through the system, see [Message Stream Context](https://deepwiki.com/charmbracelet/mods/2.4-message-stream-context)
. For conversation-level message management, see [Conversation Management](https://deepwiki.com/charmbracelet/mods/2.5-conversation-management)
.

* * *

Overview
--------

Message utilities provide low-level text processing operations used throughout the application to prepare, format, and extract information from user and AI messages. These functions serve three primary purposes:

1.  **Extraction**: Retrieving specific content from message arrays (e.g., the last user prompt)
2.  **Formatting**: Normalizing text for display or title generation (e.g., extracting first lines)
3.  **Truncation**: Managing content length to respect token limits

The utilities are stateless, pure functions that operate on `proto.Message` arrays or raw strings.

**Sources**: [messages\_test.go1-61](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go#L1-L61)

* * *

Function Inventory
------------------

The following table summarizes the core message utility functions:

| Function | Input | Output | Purpose |
| --- | --- | --- | --- |
| `lastPrompt` | `[]proto.Message` | `string` | Extracts content from the last user message |
| `firstLine` | `string` | `string` | Returns the first line of text, trimming newlines |
| `cutPrompt` | `string, int` | `string` | Truncates text to fit within character limits |
| `removeWhitespace` | `string` | `string` | Strips extraneous whitespace from text |

**Sources**: [messages\_test.go10-60](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go#L10-L60)

* * *

Last Prompt Extraction
----------------------

### Function Signature and Behavior

    lastPrompt(messages []proto.Message) string
    

The `lastPrompt` function iterates through a message array in reverse order to find the most recent message with `Role == proto.RoleUser`, returning its content. This is commonly used to generate conversation titles or to display context in the UI.

#### Message Structure

**Sources**: [messages\_test.go10-48](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go#L10-L48)

### Use Cases

The function handles three scenarios:

1.  **Empty Array**: Returns empty string when no messages exist
2.  **Single User Message**: Returns the content of that message
3.  **Multiple Messages**: Traverses backwards to find the last user prompt, ignoring assistant and system messages

#### Test Coverage

The test suite validates these scenarios:

| Test Case | Input Messages | Expected Output |
| --- | --- | --- |
| No prompt | `nil` or empty array | `""` |
| Single prompt | 1 user message: "single" | `"single"` |
| Multiple prompts | 5 messages, last user: "last" | `"last"` |

**Sources**: [messages\_test.go10-48](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go#L10-L48)

First Line Extraction
---------------------

### Function Signature and Behavior

    firstLine(text string) string
    

The `firstLine` function splits input text on newline characters (`\n`) and returns the first segment, with trailing newlines removed. This is used primarily for generating conversation titles from multi-line prompts.

#### Text Processing Flow

**Sources**: [messages\_test.go50-60](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go#L50-L60)

### Test Coverage

The test suite validates three text patterns:

1.  **Single Line**: Input with no newlines returns unchanged
2.  **Single Line with Trailing Newline**: Trailing `\n` is removed
3.  **Multiple Lines**: Only the first line is returned, regardless of subsequent content

    Test Input: "line\nsomething else\nline3\nfoo\nends with a double \n\n"
    Expected:   "line"
    

**Sources**: [messages\_test.go50-60](https://github.com/charmbracelet/mods/blob/07a05d5b/messages_test.go#L50-L60)

Token Limit Handling
--------------------

### The cutPrompt Function

While not directly tested in the provided test file, the `cutPrompt` function is referenced in the system architecture as handling token limit enforcement. This function likely truncates prompt content to fit within the `max-input-chars` constraint defined in model configurations.

#### Conceptual Flow

**Sources**: Inferred from TOC description

### Integration with Model Configuration

The function works in conjunction with the `Model.MaxInputChars` field from the configuration system (see [Provider Configuration](https://deepwiki.com/charmbracelet/mods/3.1-provider-configuration)
). When a prompt exceeds this limit, `cutPrompt` truncates the text while attempting to preserve semantic meaning.

**Sources**: Inferred from system architecture

Whitespace Preprocessing
------------------------

### The removeWhitespace Function

The `removeWhitespace` utility normalizes text by removing or collapsing extraneous whitespace. This preprocessing step ensures consistent text formatting before messages are sent to AI providers.

#### Common Whitespace Scenarios

| Input Pattern | Processing | Output |
| --- | --- | --- |
| Multiple spaces | Collapse to single | `"a b"` → `"a b"` |
| Leading/trailing | Trim | `" text "` → `"text"` |
| Multiple newlines | Normalize | `"a\n\n\nb"` → `"a\n\nb"` |

**Sources**: Inferred from TOC description

Summary
-------

Message utilities provide essential text processing operations that support the broader message handling system. These lightweight, stateless functions enable:

*   **Title Generation**: Using `firstLine` and `lastPrompt` to create human-readable conversation titles
*   **Token Management**: Applying `cutPrompt` to respect model input limits
*   **Text Normalization**: Using `removeWhitespace` for consistent formatting

The utilities are thoroughly tested and serve as building blocks for higher-level components like `setupStreamContext` and conversation management.
