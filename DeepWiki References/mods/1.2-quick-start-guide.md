Basic Usage
-----------

### Simple Prompts

The most basic usage is to pass a prompt directly as arguments to `mods`:

This command flows through [main.go80-81](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L80-L81)
 where `config.Prefix` is set from the joined arguments. The application then initializes the Bubble Tea UI model [main.go123](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L123-L123)
 and executes the prompt against the configured LLM.

When running with a TTY output, mods displays:

*   An animated spinner on stderr [main.go88-89](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L88-L89)
    
*   Formatted markdown output using Glamour [main.go224-228](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L224-L228)
    
*   Conversation metadata upon completion [main.go756-763](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L756-L763)

### Interactive Mode

When no arguments are provided and stdin is a TTY, mods enters interactive mode using the `huh` form library:

This triggers [main.go105-117](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L105-L117)
 which calls `askInfo()` [main.go783-851](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L783-L851)
 to display interactive prompts for:

*   API selection (OpenAI, Anthropic, Cohere, etc.)
*   Model selection within the chosen API
*   Prompt text input

The `--ask-model` / `-M` flag forces interactive model selection even when a default is configured [main.go249](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L249-L249)

**Sources:** [main.go105-117](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L105-L117)

### Editor Mode

Use the `-e` / `--editor` flag to compose prompts in your default editor:

This checks [main.go97-103](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L97-L103)
 and calls `prefixFromEditor()` [main.go924-949](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L924-L949)
 which:

1.  Creates a temporary file [main.go925-929](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L925-L929)
    
2.  Opens it in `$EDITOR` [main.go931-943](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L931-L943)
    
3.  Reads the contents as the prompt [main.go944-948](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L944-L948)
    

**Sources:** [main.go97-103](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L97-L103)

Pipeline Integration
--------------------

### Reading from STDIN

Mods integrates seamlessly with Unix pipelines by reading from stdin:

The TTY detection logic [main.go85-86](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L85-L86)
 determines if stdin is a pipe. If not a TTY, stdin is read and combined with the prompt prefix. This happens in the Bubble Tea model's `Init()` method where `mods.Input` is populated from stdin.

**Sources:** [main.go85-86](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L85-L86)

### Writing to STDOUT

When stdout is piped or redirected, mods operates in raw mode:

The output mode is determined by [main.go88-92](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L88-L92)
:

*   **TTY Output**: Formatted markdown on stderr [main.go89](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L89-L89)
     with renderer enabled
*   **Piped Output**: Raw text on stdout [main.go91](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L91-L91)
     renderer disabled with `tea.WithoutRenderer()`

The final output logic [main.go222-229](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L222-L229)
 prints `glamOutput` (formatted) or `Output` (raw) based on TTY status.

**Sources:** [main.go88-92](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L88-L92)
 [main.go222-229](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L222-L229)

Conversation Management
-----------------------

Mods automatically saves conversations with SHA-1 identifiers and titles, similar to git commits.

### Automatic Saving

By default, all conversations are saved [main.go717-765](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L717-L765)
:

The saving process:

1.  Generates SHA-1 ID from conversation content [main.go731](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L731-L731)
    
2.  Extracts title from first line of prompt [main.go735](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L735-L735)
    
3.  Writes full message array to cache [main.go748](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L748-L748)
    
4.  Saves metadata to SQLite database [main.go751](https://github.com/charmbracelet/mods/blob/07a05d5b/main.go#L751-L751)

Common Command Examples
-----------------------

The following table shows typical mods usage patterns:

| Use Case | Command | Description |
| --- | --- | --- |
| Code review | `git diff \| mods "review this"` | Analyze git changes |
| Error analysis | `cat error.log \| mods "summarize"` | Summarize log errors |
| Documentation | `mods "explain how REST APIs work"` | Generate explanations |
| Data transformation | `cat data.json \| mods -f "as CSV"` | Convert between formats |
| Command help | `mods --role shell "archive old files"` | Get shell commands |
| Commit messages | `git diff \| mods -r "commit msg" > msg.txt` | Generate commit messages |
| Code generation | `mods "python script to parse CSV" > script.py` | Create code files |
| Interactive query | `mods -e` | Compose complex prompts |
| Conversation review | `mods --list` | Browse previous conversations |
| Continued discussion | `mods -C "more details please"` | Follow-up on last response |
