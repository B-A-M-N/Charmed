Animation System
================

Relevant source files

*   [anim.go](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go)
    

The Animation System provides visual feedback to users during AI request processing by displaying animated text with cycling characters, gradient colors, and a spinner. This system is implemented as a Bubble Tea model that runs concurrently with the AI streaming operation to indicate that work is in progress.

This document covers the animation logic and rendering mechanics. For information about how animations integrate with the overall UI state machine, see [Bubble Tea UI Model](https://deepwiki.com/charmbracelet/mods/2.3-bubble-tea-ui-model)
. For terminal styling capabilities, see [Terminal Capabilities and Styling](https://deepwiki.com/charmbracelet/mods/4.1-terminal-capabilities-and-styling)
.

Purpose and Scope
-----------------

The animation displays while the application is in `requestState` waiting for AI provider responses. It consists of three visual elements: randomly cycling characters, a text label that gradually stabilizes, and an ellipsis spinner that activates after the label completes. The system adapts its appearance based on terminal capabilities, using gradient colors in TrueColor terminals and falling back to plain text otherwise.

Sources: [anim.go70-81](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L70-L81)

Architecture Overview
---------------------

The animation system is centered around the `anim` struct which implements the `tea.Model` interface, allowing it to participate in the Bubble Tea update loop.

**anim Struct Components**

| Field | Type | Purpose |
| --- | --- | --- |
| `start` | `time.Time` | Animation start timestamp for calculating character states |
| `cyclingChars` | `[]cyclingChar` | Characters that cycle indefinitely with random values |
| `labelChars` | `[]cyclingChar` | Characters that cycle briefly then settle to final label text |
| `ramp` | `[]lipgloss.Style` | Gradient color styles for TrueColor terminals (cycled for animation) |
| `label` | `[]rune` | Final label text to display |
| `ellipsis` | `spinner.Model` | Bubbles spinner for ellipsis animation after label completes |
| `ellipsisStarted` | `bool` | Flag indicating whether ellipsis spinner has begun |
| `styles` | `styles` | Styling configuration from parent UI |

Sources: [anim.go70-81](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L70-L81)
 [anim.go32-37](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L32-L37)

Character Lifecycle State Machine
---------------------------------

Each `cyclingChar` progresses through a three-state lifecycle controlled by timing parameters. The state determines what character value is rendered at each frame.

**Character State Logic**

The `cyclingChar.state()` method determines the current state based on elapsed time:

*   **charInitialState**: Before `initialDelay` has elapsed, renders as `'.'` to create a staggered start effect [anim.go198-199](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L198-L199)
    
*   **charCyclingState**: After `initialDelay`, renders random characters from `charRunes` which contains alphanumeric and symbol characters [anim.go21](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L21-L21)
     [anim.go200-201](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L200-L201)
    
*   **charEndOfLifeState**: After `lifetime` elapses (only if `finalValue > 0`), renders the stable `finalValue` character [anim.go202-203](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L202-L203)
    

Characters with `finalValue < 0` cycle indefinitely and never reach end-of-life state [anim.go129](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L129-L129)

Sources: [anim.go23-29](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L23-L29)
 [anim.go43-52](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L43-L52)
 [anim.go195-206](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L195-L206)

Animation Components
--------------------

### Cycling Characters

The `cyclingChars` array provides the random character prefix that cycles continuously throughout the animation's lifetime. These characters never stabilize and create a dynamic visual effect.

**Initialization**: Created in `newAnim()` with count limited by `maxCyclingChars` constant (120) [anim.go86-87](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L86-L87)
:

    cyclingCharsSize (parameter) → bounded to maxCyclingChars → n
    

Each cycling character is initialized with:

*   `finalValue = -1` to indicate infinite cycling [anim.go129](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L129-L129)
    
*   `initialDelay` randomized between 0-480ms [anim.go120-122](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L120-L122)
    

Sources: [anim.go83-132](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L83-L132)

### Label Characters

The `labelChars` array implements the text label that gradually resolves from random cycling to the stable label text. This creates a "materializing" effect where the label appears to form letter by letter.

**Character Specification**:

*   One `cyclingChar` per rune in the label [anim.go135-143](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L135-L143)
    
*   `finalValue` set to the target character from the label [anim.go139](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L139-L139)
    
*   `lifetime` randomized between 0-900ms [anim.go141](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L141-L141)
    

When all label characters reach `charEndOfLifeState`, the ellipsis spinner activates [anim.go161-175](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L161-L175)

Sources: [anim.go134-143](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L134-L143)
 [anim.go161-176](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L161-L176)

### Color Gradient System

In TrueColor-capable terminals, cycling characters are rendered with a gradient that shifts over time, creating a colorful wave effect.

**Gradient Construction**: The `makeGradientRamp()` function creates a gradient between pink (#F967DC) and purple (#6B50FF) using the Luv color space for perceptually uniform interpolation [anim.go227-239](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L227-L239)

**Seamless Cycling**: The ramp is doubled by appending its reverse, creating a palindromic color sequence that loops smoothly without jarring transitions [anim.go113](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L113-L113)

**Rotation Timing**: Colors shift at 5 FPS via `colorCycleFPS` constant [anim.go17](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L17-L17)
 Each `colorCycleMsg` rotates the ramp array left by one position [anim.go184](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L184-L184)

Sources: [anim.go102-114](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L102-L114)
 [anim.go227-240](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L227-L240)
 [anim.go179-185](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L179-L185)

### Ellipsis Spinner

The `ellipsis` field uses the Bubbles spinner component configured with the `spinner.Ellipsis` style. This provides the familiar "..." animation pattern that activates after the label stabilizes.

**Activation Condition**: The spinner starts after all `labelChars` reach `charEndOfLifeState` [anim.go163-175](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L163-L175)
:

    Count chars in charEndOfLifeState == len(label) 
    → ellipsisStarted = true 
    → Tick() command sent after 220ms delay
    

The delay creates a brief pause between label completion and spinner start [anim.go172](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L172-L172)

Sources: [anim.go98](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L98-L98)
 [anim.go161-176](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L161-L176)
 [anim.go186-189](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L186-L189)

Timing and Message System
-------------------------

The animation uses two concurrent timing loops implemented as Bubble Tea commands that send messages at regular intervals.

**Timing Constants**:

| Constant | Value | Purpose |
| --- | --- | --- |
| `charCyclingFPS` | 22 FPS (45ms) | Character update rate [anim.go16](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L16-L16) |
| `colorCycleFPS` | 5 FPS (200ms) | Color gradient rotation rate [anim.go17](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L17-L17) |

**Message Handlers**:

*   **stepCharsMsg**: Updates all `cyclingChars` and `labelChars` by calling `updateChars()` twice, checks if ellipsis should start, and reschedules itself [anim.go157-178](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L157-L178)
    
*   **colorCycleMsg**: Rotates the `ramp` array if it has at least 2 elements, then reschedules itself [anim.go179-185](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L179-L185)
    
*   **spinner.TickMsg**: Forwarded to the `ellipsis` spinner model for its internal animation [anim.go186-189](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L186-L189)
    

Sources: [anim.go15-19](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L15-L19)
 [anim.go54-68](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L54-L68)
 [anim.go149-151](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L149-L151)
 [anim.go154-192](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L154-L192)

Rendering Process
-----------------

The `View()` method constructs the animation output by concatenating three sections in order:

**Rendering Logic** [anim.go209-225](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L209-L225)
:

1.  **Cycling Characters**: Each character is rendered with gradient color if `ramp[i]` exists, otherwise rendered plain [anim.go212-218](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L212-L218)
    
2.  **Label Characters**: All label characters are appended as plain text (colors would interfere with readability) [anim.go220-222](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L220-L222)
    
3.  **Ellipsis**: The spinner's view is appended to the end [anim.go224](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L224-L224)
    

The entire output is a single line of text that updates in place during the animation.

Sources: [anim.go209-225](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L209-L225)

Construction and Initialization
-------------------------------

The `newAnim()` constructor configures all animation parameters based on terminal capabilities and input parameters.

**Function Signature**: `newAnim(cyclingCharsSize uint, label string, r *lipgloss.Renderer, s styles) anim`

**Initialization Flow**:

1.  Bound `cyclingCharsSize` to `maxCyclingChars` (120) [anim.go85-88](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L85-L88)
    
2.  Add gap space before label if cycling characters exist [anim.go90-93](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L90-L93)
    
3.  Initialize timing: `start = time.Now()` [anim.go96](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L96-L96)
    
4.  Create gradient ramp if TrueColor supported and enough characters [anim.go105-114](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L105-L114)
    
5.  Create `cyclingChars` with `finalValue = -1` and randomized delays [anim.go125-132](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L125-L132)
    
6.  Create `labelChars` with target characters and randomized lifetimes [anim.go135-143](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L135-L143)
    

**Randomization**: Initial delays and lifetimes use `makeDelay()` helper with `rand.Int31n()` to stagger character start times and create organic motion [anim.go116-122](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L116-L122)

The `Init()` method launches both timing loops via `tea.Batch()` [anim.go149-151](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L149-L151)

Sources: [anim.go83-146](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L83-L146)
 [anim.go149-151](https://github.com/charmbracelet/mods/blob/07a05d5b/anim.go#L149-L151)
